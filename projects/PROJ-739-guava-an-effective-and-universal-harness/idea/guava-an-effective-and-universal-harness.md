# Guava: An Effective and Universal Harness for Embodied Manipulation  

**Field**: computer science  

## Research question  

How does integrating a language‑model‑driven “harness” for tool‑use affect the success rate of long‑horizon embodied manipulation tasks compared with a baseline pipeline that invokes the same language model without the harness?  

## Motivation  

Recent vision‑language models (VLMs) can generate high‑level tool commands, yet it remains unclear whether a structured harness that formalises tool APIs and enforces safety checks yields measurable gains on challenging, multi‑step manipulation problems. Demonstrating a clear benefit would justify the added engineering effort and guide future system design for embodied agents.  

## Related work  

- [Guava: An Effective and Universal Harness for Embodied Manipulation (2026)](https://arxiv.org/abs/2606.18363) — Introduces the Guava harness and reports promising performance, but lacks independent statistical validation and a comparison to a “no‑harness” baseline.  
- [A Conversational Framework for Human‑Robot Collaborative Manipulation with Distributed Generative AI models (2026)](https://arxiv.org/abs/2606.06061) — Proposes a distributed conversational architecture that integrates VLMs with ROS2 for collaborative manipulation, providing a relevant alternative pipeline without a dedicated harness.  
- [Adaptive Inverse Kinematics Framework for Learning Variable‑Length Tool Manipulation in Robotics (2025)](https://arxiv.org/abs/2510.26551) – Presents a tool‑centric kinematic learning approach, illustrating the importance of explicit tool abstractions for flexible manipulation, which motivates evaluating a harness that standardises such abstractions.  

## Expected results  

We expect the harness‑enabled pipeline to achieve a statistically significant improvement (≥ 5 percentage‑point absolute gain) in task‑success rates on a suite of unseen long‑horizon manipulation benchmarks. Significance will be confirmed with 95 % confidence intervals obtained via paired bootstrap resampling across evaluation episodes. A null result (no reliable gain) would indicate that the additional harness overhead does not translate into practical benefits.  

## Methodology sketch  

1. **Dataset acquisition** – Download the publicly released RoboSuite manipulation suite (≈ 2 K simulated trajectories) from the official RoboSuite release page.  
2. **Baseline pipeline** – Implement a “direct‑call” pipeline that feeds the same open‑source VLM (e.g., LLaVA‑13B) raw task instructions to generate tool commands, then executes them in the RoboSuite simulator.  
3. **Harness pipeline** – Extend the baseline with the Guava‑style harness:  
   - Define a JSON schema for each tool (e.g., `grasp`, `move`, `rotate`).  
   - Wrap VLM outputs in API calls, enforce parameter bounds, and insert safety checks (collision‑free predicates).  
4. **Task set** – Select 20 long‑horizon tasks (e.g., “stack‑and‑place”, “assemble‑tower”) that were **not** used during the original Guava training.  
5. **Evaluation runs** – For each task, run 30 independent episodes per pipeline (different random seeds for scene randomisation). Record binary success/failure and episode‑level metrics (time to completion, number of tool calls).  
6. **Statistical analysis** – Compute per‑task success rates for both pipelines. Apply paired bootstrap (10 000 resamples) to obtain 95 % confidence intervals for the difference in success rates. Perform a two‑sided hypothesis test (α = 0.05) and report p‑values with Holm‑Bonferroni correction for multiple tasks.  
7. **Ablation** – Repeat steps 3–6 with the harness **disabled** for safety checks (i.e., only API wrapping) to isolate the contribution of the safety layer.  
8. **Reproducibility package** – Provide a `requirements.txt`, an `environment.yml`, and a lightweight Dockerfile (Python 3.10, no GPU) that downloads the dataset, runs the two pipelines, and outputs a CSV of results. All scripts are ≤ 200 LOC each.
