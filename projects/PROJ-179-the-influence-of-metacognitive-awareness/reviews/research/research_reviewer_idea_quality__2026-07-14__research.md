The project as currently scoped and executed fails the fundamental test of a research idea: **the question being asked is not answerable by the data being used.**

The spec explicitly defines the research question as: *"Do individuals with higher metacognitive awareness exhibit more accurate reality testing, as measured by reduced source‑monitoring errors on ambiguous perceptual tasks?"* This requires a dataset with:
1.  **Source-monitoring trials** (distinguishing imagined vs. perceived stimuli).
2.  **Trial-wise confidence ratings** (0-100%) linked to those specific trials.
3.  **Participant-level behavioral data** (not just aggregate labels).

The spec correctly identifies that the intended source, OpenNeuro ds003386, is a structural MRI dataset and **lacks** these behavioral fields. This is a valid data constraint.

However, the execution evidence and `data/raw/iris_behavioral.csv` reveal that the pipeline actually ran on the **Iris dataset**. The Iris dataset is a static classification benchmark (flower species: setosa, versicolor, virginica) with no concept of "source monitoring," "imagined vs. perceived," or "metacognitive awareness." It contains no trial-wise confidence ratings generated by a human subject in a perceptual task.

**The Core Defect:**
The project has effectively replaced its research question with a different, unasked question. It is no longer testing "metacognitive awareness in reality testing"; it is testing "correlation between confidence and accuracy on a flower classification task."
-   **Falsifiability Broken:** A result from the Iris dataset cannot confirm or refute the hypothesis about source-monitoring errors. The answer is "N/A" for the stated question.
-   **Scope Mismatch:** The plan and spec describe a complex behavioral neuroscience analysis (Type-2 AUC, signal detection theory on source monitoring), but the implementation is running on a generic classification dataset. The "Hold-Out Accuracy" design is being applied to a context where the theoretical construct (metacognitive awareness of *reality testing*) does not exist.
-   **Significance Null:** Even if the correlation is significant on the Iris dataset, it tells us nothing about the original research question. A null result would be expected (or meaningless), and a positive result would be a statistical artifact of the dataset, not a discovery about human cognition.

The advisory comment correctly identifies this as a "Fatal Data Mismatch." This is not a minor implementation bug; it is a fundamental collapse of the research idea. The project is currently answering a question nobody asked (about flowers) while claiming to answer a question it cannot answer (about source monitoring).

To proceed, the project must either:
1.  **Find a valid dataset** that actually contains source-monitoring trials with confidence ratings (e.g., a specific behavioral dataset from OpenNeuro or a published repository), OR
2.  **Radically reframe the research question** to match the available data (e.g., "Does metacognitive confidence predict accuracy in standard classification tasks?"), which would require a completely new theoretical justification and literature review, as the current spec is entirely focused on "reality testing" and "source monitoring."

Until the data source matches the research question, the idea is not well-posed.

## Required Changes

-   **Replace the data source or the research question**: Either locate and integrate a valid behavioral dataset containing trial-wise `confidence_rating` and `source_label` (imagined vs. perceived) fields (e.g., a specific source-monitoring dataset from OpenNeuro or a published repository) and update `data/download.py` and `data/validate_data_availability.py` to target it, **OR** completely rewrite `spec.md` to reframe the research question to match the Iris dataset (e.g., "Metacognitive confidence in standard classification tasks"), removing all references to "source-monitoring," "reality testing," and "imagined vs. perceived" stimuli.
-   **Update `spec.md` Data Constraints section**: Explicitly state the actual dataset being used (e.g., "Iris dataset" or the new valid source) and confirm it contains the required fields for the *new* or *revised* research question. Remove the claim that OpenNeuro ds003386 is the source if it is not being used.
-   **Re-evaluate the "Hold-Out Accuracy" design**: If the research question is changed to match the Iris dataset, verify that the "Hold-Out Accuracy" design (Type-2 AUC vs. d') is theoretically appropriate for that new context, as the current justification relies on "source-monitoring" theory which does not apply to flower classification.
