## Statistical Rigor

### Power Analysis (FR-025)
To detect an inconsistency proportion of at least 0.10 (double the baseline 0.05) with power ≥ 0.80 at α = 0.05, a sample size of **N ≥ 300** is required. This is implemented in Phase 0, R04.

### Multiple Comparison Correction (FR-032)
Subgroup analyses (domain/year) involve multiple hypothesis tests. **Bonferroni correction** is applied: adjusted α = 0.05 / number_of_subgroups. This controls family-wise error rate.

### Baseline Handling (FR-012)
When baseline conversion rate is missing, the system uses the **average of the two variant rates**. This is a heuristic approximation that **affects effect size reconstruction uncertainty**. The uncertainty impact is quantified in audit notes per scientific_soundness-ad85a63f resolution.

### Statistical Validation (FR-026)
Monte Carlo simulation (numerous replicates) validates `scipy.stats` implementations. Acceptance criterion: absolute difference ≤ 0.005.

### Consistency Threshold (Constitution VI)
**Absolute p-value difference threshold is fixed at 0.05**. No relative or log-scale thresholds are permitted for p-value discrepancy (Constitution VI, methodology-5bca6076 resolution). This is a hard requirement that cannot be overridden by statistical considerations.

### Bias Adjustment (FR-027)
Bias-adjusted rate uses **domain-weighted averaging with equal weighting per domain** (scientific_soundness-64bd4091 resolution). This provides reproducible methodology for handling domain imbalance.

### Construct Validity Limitation
This project tests **internal consistency** of reported statistics, not ground-truth statistical validity. Without access to raw data, we cannot verify whether the reported p-values correctly reflect the actual statistical evidence. This limitation is acknowledged per methodology-c014d2d7.

## Compute Feasibility

- **Library**: `scipy`, `pandas`, `numpy` (CPU-optimized).
- **Memory**: A substantial number of synthetic rows plus a set of audit rows fit easily within available memory constraints.
- **Runtime**: Extraction is network-bound. Statistical reconstruction is O(N). Monte Carlo is parallelizable but CPU-only. Total estimated time < 6 hours on 2 vCPU.
- **GPU**: Not used.

## Risk Mitigation

| Risk | Mitigation |
|------|------------|
| **Dead URLs** | Logged as parsing failures (FR-007). Excluded from aggregates. |
| **Domain Bias** | Subsampling if >30% from one domain (FR-027). Bias-adjusted rate reported with equal domain weighting. |
| **Extraction Accuracy** | Real-world validation set (≥100 samples) ensures ≥85% precision (SC-031b, methodology-42b0e48f resolution). |
| **Statistical Bugs** | Monte Carlo validation (FR-026) + Independent synthetic generator (FR-030). |
| **Constitution Violation** | Task T099 (Constitution Compliance Audit) verifies all Principles I-VII. |
| **Baseline Uncertainty** | Heuristic baseline handling quantifies effect size reconstruction uncertainty (FR-012, scientific_soundness-ad85a63f resolution). |
