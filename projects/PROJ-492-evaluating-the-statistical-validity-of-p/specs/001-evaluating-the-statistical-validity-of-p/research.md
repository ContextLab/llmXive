# Research: Evaluating the Statistical Validity of Public A/B Test Summaries

## Overview

This research document details the methodology, dataset strategy, statistical approach, and validation plan for auditing public A/B test summaries for statistical consistency. The core research question is: **What is the prevalence of statistical inconsistencies in publicly available A/B test summaries?**

**Methodological Limitations** (CRITICAL): Internal consistency ≠ external accuracy. This audit validates whether reported numbers are mathematically consistent with each other, NOT whether they accurately reflect the actual experiment. A summary can be internally consistent but externally inaccurate (e.g., fabricated data that happens to satisfy statistical relationships). This constraint MUST be prominently acknowledged when interpreting prevalence estimates.

## Statistical Methodology

### 1. P-Value Reconstruction (FR-003)

| Outcome Type | Statistical Test | Implementation |
|--------------|------------------|----------------|
| Binary (conversion rates) | Two-proportion z-test | `scipy.stats.proportions_ztest` |
| Binary (cell count ≤5) | Fisher's exact test | `scipy.stats.fisher_exact` |
| Continuous (mean difference) | Welch's two-sample t-test | `scipy.stats.ttest_ind` (equal_var=False) |

**Monte Carlo Validation (FR-026)**:
- Each statistical test validated with a sufficient number of replicates.
- Acceptance criterion: Absolute difference between library result and Monte Carlo estimate ≤0.005 (SC-003, SC-026).
- Validation report saved to `output/monte_carlo_validation_report.json`.

### 2. Inconsistency Detection (FR-004)

A summary is flagged **inconsistent** if ANY of the following holds (evaluated at α=0.05):

| Criterion | Threshold | Rationale |
|-----------|-----------|-----------|
| |Δp| (absolute difference) | >0.05 | Constitution Principle VI mandates absolute threshold (policy choice, not statistical optimization) |
| Inequality p-value (e.g., "p<0.001") | Reconstructed p > bound | Only flag if reconstructed exceeds reported bound |
| |Δeffect_size| (relative difference) | >5% of larger magnitude | Industry reporting variance (FR-004); not constrained by Constitution |

**Construct Validity Limitation**: The absolute 0.05 threshold creates non-uniform inconsistency detection across the p-value distribution. For p=0.001, a 0.05 difference is 50x the reported value; for p=0.4, it's only [deferred]. This biases prevalence estimates toward studies with smaller p-values. Documented as policy choice per Constitution Principle VI, not statistical optimization.

### 3. Prevalence Estimation (FR-005a)

- **Test**: Two-sided binomial test against baseline proportion consistent with prior research (John et al., 2022 meta-analysis).
- **Confidence Interval**: 95% Wilson score interval (preferred for proportions near boundaries).
- **Output**: p-value (3 decimal places), Wilson CI width ≤0.10 (SC-014), raw inconsistency rate.

### 4. Sensitivity Analysis (FR-005b)

- **Baseline range**: 0.02 to 0.10 (step 0.01) — **Multiple different baseline proportions**.
- **Multiple comparison correction**: Bonferroni correction applied (adjusted α = 0.05/9 ≈ 0.0056) to control family-wise error rate. Testing multiple hypotheses inflates Type I error rate without correction.
- **Acceptance criterion**: Estimated prevalence varies by <0.02 across range (SC-015).

### 5. Domain Bias Assessment (FR-027)

- **Dominance threshold**: No single domain >30% of corpus.
- **Action**: Subsample dominant domain if exceeded (T044).
- **Output**: Bias-adjusted inconsistency rate (domain-weighted average), raw rate, domain proportions.

### 6. Subgroup Analysis (FR-032)

- **Subgroups**: By source domain and publication year.
- **Test**: Fisher's exact test for subgroups with ≥10 summaries.
- **Multiple comparison correction**: Bonferroni (adjusted α = 0.05 / number_of_subgroups).
- **Output**: Subgroup prevalence, adjusted p-value, significance flag.

## Power Analysis (FR-025)

**Parameters**:
- Baseline proportion (null): a literature-based estimate (John et al., 2022)
- Effect size to detect: a small effect size (double baseline)
- Power: ≥0.80
- Significance level (α): 0.05

**Calculation**:
Using binomial power analysis formula:
```
N = (Z_α/2 + Z_β)² × [p₀(1-p₀) + p₁(1-p₁)] / (p₁ - p₀)²
```
Where:
- Z_α/2 = 1.96 (for α=0.05 two-sided)
- Z_β = 0.84 (for power=0.80)
- p₀ = 0.05 (null)
- p₁ = 0.10 (alternative)

**Result**: N ≥ 300 summaries required. This is mandated by FR-025 and SC-025.

**Limitation**: If corpus N<300 is achievable, document power limitation explicitly (cannot claim statistical significance for prevalence estimates).

## Edge Cases & Handling

| Edge Case | Handling | FR Reference |
|-----------|----------|--------------|
| Missing metric (e.g., effect size without p-value) | Flag "missing metric"; record in audit notes; exclude from prevalence; **perform sensitivity analysis comparing prevalence with/without missing-metric entries** | FR-004, T022b |
| Non-binary outcomes (continuous) | Use Welch's t-test instead of z-test | FR-003 |
| Rounded/inequality p-values (e.g., "p<0.001") | Treat bound as upper limit; flag only if reconstructed > bound | FR-004 |
| Conflicting sample sizes (>5% difference) | Flag "size mismatch"; exclude from prevalence aggregates | FR-004b |
| Missing baseline conversion rate | Use average of two variant rates (heuristic); flag in notes | FR-012 |
| Dead/non-HTML URLs | Record parsing failure with ERR-### code; exclude from aggregates | FR-007, SC-005 |
| Domain dominance (>30%) | Subsample to equalize proportions; report bias-adjusted rate | FR-027 |

**Selection Bias Note**: Entries with missing required metrics are excluded from prevalence calculations. If missing metrics correlate with inconsistency rates (e.g., inconsistent studies less likely to report all metrics), this introduces selection bias. T022b performs sensitivity analysis to quantify this bias.

## Statistical Rigor Checklist

- [x] **Multiple-comparison correction**: Bonferroni applied to subgroup tests (FR-032) AND sensitivity analysis (FR-005b); baseline binomial test is single test (no correction needed).
- [x] **Sample-size/power justification**: N≥300 calculated for power≥0.80 at α=0.05 (FR-025).
- [x] **Causal framing**: Audit is observational; claims are associational (internal consistency only, not external accuracy).
- [x] **Measurement validity**: Statistical tests validated via Monte Carlo (FR-026); extraction validated via human annotation (FR-031b).
- [x] **Predictor collinearity**: No regression model; direct reconstruction from reported numbers. No collinearity issues.
- [x] **Sensitivity analysis correction**: Testing 9 baseline proportions requires Bonferroni correction (α=0.0056) to control family-wise error rate.
- [x] **Selection bias mitigation**: T022b sensitivity analysis compares prevalence with/without missing-metric entries.
