# Implementation Plan: Robotic AI Sensory Fidelity Ablation Study

**Branch**: `001-sensory-fidelity-ablation` | **Date**: 2026-07-10 | **Spec**: `specs/001-sensory-fidelity-ablation/spec.md`
**Input**: Feature specification from `/specs/001-sensory-fidelity-ablation/spec.md`

## Summary

This feature implements a CPU-tractable ablation study to quantify how sensory representation fidelity (Raw RGB vs. Downsampled Depth vs. 2D Occupancy Grids) impacts sample efficiency and generalization in autonomous navigation. The study utilizes a lightweight DQN agent with a pruned MobileNetV backbone, constrained to a multi-core CPU, 7GB RAM, and -hour wall-clock limit. It establishes baselines via classical (Pure Pursuit/Dijkstra) and stochastic planners, executes a rigorous statistical analysis (Welch's ANOVA/Kruskal-Wallis) on learning curves with explicit censored-data handling, and explicitly frames findings as associational.

**Testing**: `pytest` with contract tests against YAML schemas.  
**Target Platform**: Linux (GitHub Actions -core runner, x64).  
**Project Type**: Computational research / simulation pipeline.  
**Performance Goals**: Training completion ≤ 6 hours; Peak RAM ≤ 7GB; Inference latency ≤ 33ms/step.  
**Constraints**: No GPU/CUDA; no 8-bit quantization libraries; no deep-net training from scratch; deterministic seeding.  
**Scale/Scope**: N=30 randomized seeds (distinct environment realizations); modalities; ~M parameter model; A substantial number of training steps per run (capped by time).

## Implementation Phases

### Phase 0: Environment & Calibration (Prerequisites)
- **Phase 0.1**: Run `scripts/validate_calibration.py` to perform extrinsic calibration and coordinate transformation validation (FR-009). Generate `results/calibration_report.json`. **Block** if validation fails.
- **Phase 0.5**: Run `scripts/sweep_thresholds.py` to perform sensitivity analysis on occupancy grid generation threshold (FR-008). Select optimal threshold and log to `results/threshold_sweep.json`.

### Phase 1: Baselines & Data Generation
- **Phase 1.1**: Execute `scripts/run_baselines.py` to establish classical (Pure Pursuit/Dijkstra) and stochastic baselines (US-1).
- **Phase 1.2**: Execute `scripts/generate_modalities.py` to generate RGB, Depth, and Occupancy Grid tensors from simulated ground truth (US-2).

### Phase 2: Training
- **Phase 2.1**: Train DQN agents for each modality (30 seeds each) with hard 6-hour time limit (FR-003).
- **Phase 2.2**: Log peak RAM and CPU usage (FR-006). Handle memory pressure crashes via checkpointing.

### Phase 3: Analysis & Generalization
- **Phase 3.1**: Execute statistical analysis on learning curves (AUC, censored time-to-convergence) using Welch's ANOVA/Kruskal-Wallis (FR-005).
- **Phase 3.2**: Execute `scripts/eval_generalization.py` to test agents on novel, high-obstacle-density scenarios (SC-002).
- **Phase 3.3**: Generate final report with associational framing (FR-007).

### Phase 4: Versioning
- **Phase 4.1**: Run `scripts/hash_artifacts.py` to compute checksums and update `state/` YAML file (Principle V).
