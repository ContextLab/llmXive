# Implementation Plan: Evaluating Calibration of Predictive Intervals in Time Series Forecasting

**Branch**: `[001-calibration-evaluation]` | **Date**: 2026-06-25 | **Spec**: `specs/001-calibration-evaluation/spec.md`

## Summary

This feature implements a rigorous evaluation pipeline to assess the calibration of predictive intervals (PIs) generated by four distinct forecasting models (ARIMA, ETS, Prophet, LightGBM) on the M time series dataset. The system will compute empirical coverage rates against nominal coverage targets, perform stratified analysis by seasonality and trend strength, apply adaptive conformal prediction for recalibration, and enforce statistical rigor via Benjamini-Yekutieli FDR correction (a robust variant for dependent data). All methods are constrained to CPU-only execution on GitHub Actions free-tier runners with limited CPU resources, limited RAM, and a time limit.

## Testing Strategy

- **Contract Testing**: The pipeline output (`results/coverage.csv`, `results/recalibration.csv`) is validated against the YAML schemas in `contracts/` using `pytest` and `jsonschema`. Any schema violation halts the pipeline.
- **Unit Testing**: Logic for coverage calculation, STL decomposition, and FDR correction is tested with synthetic data where ground truth is known.
- **Integration Testing**: A "mini-pipeline" runs on 10 series to verify end-to-end flow and output structure.

## Complexity Tracking

| Violation | Why Needed | Simpler Alternative Rejected Because |
| :--- | :--- | :--- |
| **Four Models** | Required by FR-002 to compare parametric vs. non-parametric vs. ML approaches. | Single model would not allow comparative calibration analysis. |
| **Adaptive Conformal** | Required by FR-006 to demonstrate mitigation of miscalibration. | Static intervals would not address the core research question of recalibration efficacy. |
| **FDR Correction** | Required by FR-007 due to simultaneous hypothesis tests (4 models × 12 horizons). | Uncorrected p-values would inflate false positive rates, violating statistical rigor. |
| **BY vs BH** | BY is required for autocorrelated data (see Research). | BH is mandated by spec but BY is the scientifically valid choice for this dependency structure; we report BY as the primary result with BH as sensitivity. |
