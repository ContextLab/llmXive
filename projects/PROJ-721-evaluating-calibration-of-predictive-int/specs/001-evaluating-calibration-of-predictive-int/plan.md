# Implementation Plan: Evaluating Calibration of Predictive Intervals in Time Series Forecasting

**Branch**: `[001-calibration-evaluation]` | **Date**: 2026-06-25 | **Spec**: `specs/001-calibration-evaluation/spec.md`

## Summary

This feature implements a rigorous evaluation pipeline to assess the calibration of predictive intervals for time series forecasting. The system ingests a subset of the M4 competition dataset, fits four distinct models (ARIMA, ETS, Prophet, LightGBM with quantile regression), and generates prediction intervals at nominal [deferred] and [deferred] levels. It calculates empirical coverage rates, performs stratified analysis by seasonality and trend strength, applies adaptive conformal prediction for recalibration, and reports results with Benjamini-Hochberg FDR correction. All findings are framed as associational comparisons. The pipeline is designed to run entirely on CPU within the GitHub Actions free-tier limits (limited CPU, constrained RAM, 6 hours).

## Computational Feasibility & Methodological Rigor

### Compute Feasibility (GitHub Actions Free Tier)
*   **Resource Limits**: 2 CPU cores, ~7 GB RAM, ~14 GB disk, ≤6 hours.
*   **Strategy**:
 * **Dataset Subset**: The pipeline processes a representative subset of **[deferred] time series** (FR-001), not the full M4 dataset ([deferred]+ series). This subset is selected to maintain the frequency distribution (SC-005).
    *   **Model Selection**:
        *   **ARIMA/ETS**: `statsmodels` implementations are CPU-efficient for short horizons.
        *   **Prophet**: Runs on CPU; configured for fast fitting (no GPU).
        *   **LightGBM**: Uses quantile regression objective (`objective='quantile'`). Trained on CPU with limited iterations to ensure runtime < 6h.
    *   **Memory Management**: Data is loaded in chunks; large intermediate arrays are garbage collected.
    *   **No GPU**: All libraries are pinned to CPU-compatible versions. No CUDA dependencies.

### Methodological Rigor
*   **FR/SC Coverage**:
    *   **FR-001 to FR-010**: Each requirement is mapped to a specific phase/step in the plan (e.g., FR-005 -> `data/preprocessing.py` for STL decomposition; FR-007 -> `utils/stats.py` for FDR).
    *   **SC-001 to SC-005**: Success criteria are explicitly measured (e.g., runtime logged, coverage deviation calculated, FDR applied).
*   **Statistical Rigor**:
    *   **Multiple Comparisons**: Benjamini-Hochberg FDR (FR-007) is applied to all p-values across 4 models × 12 horizons.
 * **Power Justification**: The plan acknowledges the sample size ([deferred] series) and notes that power is sufficient for detecting large calibration gaps (>2%) but may be limited for small effects.
    *   **Causal Inference**: All claims are framed as associational (FR-009, Assumption: Inference Framing). No causal claims about model superiority are made.
    *   **Measurement Validity**: M4 metadata is used for seasonality; trend strength is derived via STL (FR-005, Assumption: Metadata Validity).
    *   **Collinearity**: No collinearity issues expected as models are distinct; however, feature engineering for LightGBM avoids redundant predictors.
*   **Dataset-Variable Fit**:
    *   **M4 Dataset**: Verified to contain `seasonality` and `frequency` fields.
    *   **Trend Strength**: Not in metadata; derived via STL decomposition (Assumption: Metadata Validity).
    *   **Gap Handling**: If a series lacks required metadata, it is skipped and logged (Edge Case).

## Phase Ordering
1.  **Data Download**: Fetch M4 dataset and verify checksums (FR-001).
2.  **Preprocessing**: Split into train/test, derive trend strength via STL (FR-005).
3.  **Model Fitting**: Fit ARIMA, ETS, Prophet, LightGBM on training data.
4.  **Forecasting**: Generate point forecasts and prediction intervals for horizons 1-12 (FR-002, FR-003).
5.  **Evaluation**: Calculate empirical coverage rates (FR-004).
6.  **Stratification**: Group results by seasonality and trend strength (FR-005).
7.  **Recalibration**: Apply adaptive conformal prediction if deviation > 2% (FR-006).
8.  **Statistical Testing**: Apply FDR correction and bootstrap tests (10,000 resamples) (FR-007, T039).
9.  **Reporting**: Generate final CSVs and figures (SC-001 to SC-005).
