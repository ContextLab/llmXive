# Data Model Documentation

**Purpose** – This document describes the core data entities produced and consumed by the
*Evaluating the Impact of Code Duplication on LLM Code Understanding* pipeline and
visualises the flow of data between them. It is referenced by the implementation
tasks in Phases 2‑5 and serves as the single source of truth for downstream
validation, checksum tracking and reproducibility (see SC‑001 – SC‑007).

---

## 1. Entity Definitions

| Entity | Storage Path (relative to project root) | Format | Primary Fields | Description |
|--------|------------------------------------------|--------|----------------|-------------|
| **Raw Corpus** | `data/raw/github-code-sample.csv` | CSV (streamed) | `file_path`, `content` | Sub‑sample (≈500 MB) of the `codeparrot/github-code` dataset streamed from HuggingFace. [UNRESOLVED-CLAIM: c_ab82b230 — status=not_enough_info] |
| **Parse Failures Log** | `data/parse_failures.csv` | CSV | `file_path`, `error_type`, `error_message`, `timestamp` | Records any file that could not be parsed by the AST cloner (syntax errors, encoding issues, etc.). |
| **Clone Metrics** | `data/processed/clone_metrics.csv` | CSV | `file_path`, `total_nodes`, `clone_nodes`, `clone_density` | Output of `ast_cloner.py`. `clone_density = clone_nodes / total_nodes`. |
| **Perplexity Scores** | `data/processed/perplexity_scores.csv` | CSV | `file_path`, `token_count`, `perplexity` | Per‑file token‑level perplexity computed by `model_metrics.py` using the 8‑bit `codegen-350M-mono` model. |

| **Correlation Results** | `data/analysis/correlation_results.csv` | CSV | `metric_x`, `metric_y`, `spearman_rho`, `p_value`, `n` | Spearman rank correlation between duplication density and (a) perplexity, (b) bug‑detection accuracy (produced by `correlation_analysis.py`). |
| **PII Scan Report** | `data/analysis/pii_scan_report.csv` | CSV | `file_path`, `pii_type`, `match`, `line_number` | Findings of `pii_scanner.py` across raw, processed, and analysis directories. |
| **Visualization Figures** | `data/analysis/figures/` | PNG, PDF | *filename* | Scatter plots and regression lines that visualise the relationships captured in the correlation results (generated by `visualization.py`). |

## 3. Versioning & Checksums

Every artifact produced by the pipeline (raw, processed, analysis, and figure files)
is recorded in the **checksum manifest** (`code/checksum_manifest.py`). The manifest
stores SHA‑256 hashes keyed by the relative file path, enabling:

* Re‑producibility verification (SC‑005)
* Integrity validation in downstream tests (e.g., T025, T036, T044)
* Auditable change tracking across runs
