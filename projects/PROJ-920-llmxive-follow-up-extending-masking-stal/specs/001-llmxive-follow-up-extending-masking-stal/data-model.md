# Data Model: llmXive follow-up: extending "Masking Stale Observations Helps Search Agents -- Until It Doesn't"

## Overview

This document defines the data structures for the synthetic trajectory generation, simulation logs, and regression outputs. All data is stored in JSON or CSV formats to ensure compatibility with the CPU-only pipeline.

## Entity Definitions

### 1. Trajectory
A single search session generated by the simulator.
-   **ID**: Unique identifier (UUID or hash).
-   **Turns**: List of turn objects.
-   **Metadata**: Ground truth labels (critical turn, density level).

### 2. Simulation Log
The result of running an agent on a trajectory with a specific retention horizon.
-   **Trajectory ID**: Reference to the source trajectory.
-   **Horizon**: The retention window size used.
-   **Outcome**: Binary success/failure.
-   **Density**: The density of the critical evidence block.
-   **H_min**: The calculated minimum required horizon for the evidence.
-   **Age**: The age of the evidence at the time of simulation (current turn - critical turn).
-   **Decay_Factor**: The calculated focus decay factor.

### 3. Regression Output
The results of the statistical analysis.
-   **Coefficients**: Model parameters.
-   **P-values**: Significance tests.
-   **Plot Path**: Location of the generated surface plot.

## Schema Definitions

### Trajectory Schema
```yaml
type: object
properties:
  trajectory_id:
    type: string
    description: "Unique identifier for the trajectory."
  turns:
    type: array
    description: "List of turns in the search session."
    items:
      type: object
      properties:
        turn_index:
          type: integer
          description: "Index of the turn (0-based)."
        text:
          type: string
          description: "Text content of the turn."
        entropy:
          type: number
          description: "Shannon entropy of the text block (bits/token)."
        technical_ratio:
          type: number
          description: "Proportion of technical tokens."
        density:
          type: number
          description: "Composite density metric (0.6*entropy + 0.4*ratio)."
  ground_truth:
    type: object
    properties:
      critical_turn_index:
        type: integer
        description: "Index of the turn containing critical evidence."
      density_level:
        type: string
        enum: [low, medium, high]
        description: "Qualitative density label."
      target_success:
        type: boolean
        description: "Theoretical success if horizon is infinite."
      h_min:
        type: integer
        description: "Calculated minimum required horizon based on density."
```

### Simulation Log Schema
```yaml
type: object
properties:
  simulation_id:
    type: string
    description: "Unique identifier for the simulation run."
  trajectory_id:
    type: string
    description: "Reference to the source trajectory."
  retention_horizon:
    type: integer
    description: "Number of turns retained (1 to T)."
  success:
    type: boolean
    description: "Binary outcome (1 if success, 0 if failure)."
  evidence_visible:
    type: boolean
    description: "Whether the critical evidence was within the retention window (Horizon >= H_min)."
  heuristic_success:
    type: boolean
    description: "Whether the probabilistic solver succeeded."
  density_value:
    type: number
    description: "Density of the critical evidence block."
  h_min:
    type: integer
    description: "Minimum required horizon for this evidence."
  evidence_age:
    type: integer
    description: "Age of the evidence (current turn - critical turn)."
  decay_factor:
    type: number
    description: "Calculated focus decay factor."
```

### Regression Output Schema
```yaml
type: object
properties:
  model_summary:
    type: object
    properties:
      formula:
        type: string
        description: "GLM formula used."
      n_observations:
        type: integer
        description: "Total number of simulation records."
      pseudo_r_squared:
        type: number
        description: "McFadden's R-squared."
  coefficients:
    type: array
    description: "List of coefficient estimates."
    items:
      type: object
      properties:
        term:
          type: string
        estimate:
          type: number
        std_error:
          type: number
        z_value:
          type: number
        p_value:
          type: number
  interaction_significance:
    type: object
    properties:
      p_value:
        type: number
        description: "P-value for the density*horizon interaction term."
      supported_hypothesis:
        type: boolean
        description: "True if p < 0.05."
  plot_path:
    type: string
    description: "Relative path to the generated PNG file."
```

## Data Flow

1. **Generation**: `generate_trajectories.py` reads configuration -> writes `data/raw/trajectories.json` ([deferred] records).
2.  **Simulation**: `simulate_agent.py` reads `trajectories.json` -> loops over horizons -> appends to `data/processed/simulation_logs.csv` (streamed).
3.  **Analysis**: `analyze_results.py` reads `simulation_logs.csv` -> fits GLM -> writes `data/processed/regression_results.json`.
4.  **Visualization**: `visualize_results.py` reads `regression_results.json` (or raw logs) -> generates `output/plots/surface_plot.png`.
