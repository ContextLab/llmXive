## Unresolved concerns

- The plan proposes a Pearson correlation between 'nodes_visited' and 'accuracy'. However, accuracy is a binary outcome (0.0 or 1.0) per task. Pearson correlation is inappropriate for binary dependent variables; it assumes linearity and normality of residuals which binary data violates. The methodology should specify Point-Biserial correlation or a logistic regression model to properly assess the relationship between traversal depth and success probability.
- The plan relies on 'Synthetic Noisy Graphs' generated by 'injecting irrelevant edges'. The methodology does not define the distribution or density of this noise. If the noise is too sparse, the robustness check fails to stress the heuristic; if too dense, the graph becomes unrecognizable. The plan must specify the noise injection rate (e.g., '10% of edges are random') to ensure the robustness test is scientifically valid and reproducible.
- The plan defines the 'inflection point' as the node count where accuracy drops below 95% of baseline. This is a derived metric from a step-function (accuracy is 0 or 1). Identifying a precise 'point' on a step function is statistically unstable without smoothing or binning. The methodology should specify how this threshold is estimated (e.g., 'binning tasks by node count and finding the first bin with <95% mean accuracy') to ensure the metric is robust to noise.
