## Summary

This project implements a rigorous evaluation pipeline to benchmark "Evidence Attribution" in document understanding. The core hypothesis is that text-only Large Language Models (LLMs), despite strong reasoning capabilities, will fail to accurately attribute answers to specific visual locations (bounding boxes) compared to Multimodal LLMs (MLLMs). 

The system will execute:
1.  **Text-Only Pipeline**: A two-stage pipeline (Retrieval + Reasoning) using `all-MiniLM-L6-v2` and `Phi-3-mini`. Performance is measured using **Strict Attributed Accuracy (SAA)**, a composite metric requiring:
    *   **Answer Correctness**: Exact Match OR (Semantic Similarity ≥ 0.85 **AND** Ground Truth Answer present in retrieved context).
    *   **Spatial Correctness**: Intersection-over-Union (IoU) > 0.5.
2.  **Visual-Only Control**: A separate experiment where a model receives only the full-page image to predict a bounding box. Performance is measured using **Visual Localization Accuracy (VLA)**, distinct from SAA, to isolate the modality gap.

All experiments will be conducted on CPU-only infrastructure (GitHub Actions free tier) with strict memory and time constraints.

**Critical Dependency**: The project requires the **CiteVQA** dataset (QA pairs + Bounding Boxes). As this dataset is not currently listed in the project's "Verified datasets" block, this plan is **blocked** until a verified source URL is provided. No synthetic or proxy datasets will be used as they cannot replicate the ground-truth bounding box distribution required for the SAA metric.

## Complexity Tracking

| Violation | Why Needed | Simpler Alternative Rejected Because |
|-----------|------------|-------------------------------------|
| **Dual Pipeline Architecture** (Text vs. Visual) | Required to isolate the "modality gap" hypothesis (US-3). | A single multimodal model would conflate retrieval and reasoning capabilities, failing to diagnose whether errors stem from lack of text context or lack of visual grounding. |
| **Strict SAA Metric** (Intersection of Answer + Box + Context) | Required by Constitution Principle VI to detect "Attribution Hallucination". | Using only answer accuracy (Exact Match) would mask the core failure mode where the model knows the answer but points to the wrong location or didn't retrieve the evidence. |
| **Bootstrap CI Analysis** | Required to validly compare a sample mean against a fixed scalar baseline without assuming zero variance for the baseline. | A one-sample t-test is statistically invalid for comparing a sample against a point estimate from a different study. |
| **CPU-Only Constraints** | Required by GitHub Actions free-tier limits (7GB RAM, no GPU). | GPU-based models (e.g., standard Llama-7B) would crash or exceed the 6h runtime limit, making the project unexecutable in CI. |
| **Blocking on Verified Dataset** | Required by Constitution Principle II (Verified Accuracy). | Proceeding with synthetic or unverified data would invalidate the SAA metric (which requires ground-truth boxes) and the reproducibility of the results. |
