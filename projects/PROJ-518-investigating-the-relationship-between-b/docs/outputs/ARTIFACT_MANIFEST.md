# Project Artifacts Manifest

This document describes the specific files generated by the PROJ-518 pipeline, their locations, and their contents.

## 1. Data Interim Files

### `data/interim/permutation_results.csv`
- **Source**: Generated by `code/analysis/statistics.py` (via `run_permutation_test`) and saved by `code/scripts/save_analysis_results.py`.
- **Description**: Contains the empirical p-values derived from the permutation test (shuffling creativity scores) to assess the significance of the relationship between network flexibility and creativity.
- **Columns**:
 - `iteration`: The permutation index.
 - `p_value`: The calculated p-value for that iteration.
- **Format**: CSV with headers.

### `data/interim/sensitivity_summary.csv`
- **Source**: Generated by `code/analysis/sensitivity.py` (`run_sensitivity_analysis`) and saved by `code/scripts/save_analysis_results.py`.
- **Description**: Summarizes the robustness of the findings across different sliding window parameters.
- **Columns**:
 - `window_length`: The window size used (e.g., 20, 30, 40).
 - `correlation`: The Pearson correlation coefficient for that window.
 - `p_value`: The significance level for that window.
- **Format**: CSV with headers.

## 2. Visualization Outputs (in `docs/outputs/`)

### `flexibility_vs_creativity.png`
- **Source**: `code/viz/plots.py` (`plot_flexibility_vs_creativity`).
- **Description**: A scatter plot visualizing the primary relationship of interest.
 - X-axis: Network Flexibility.
 - Y-axis: Creativity Score (CAQ).
 - Overlaid: Best-fit regression line with a 95% confidence interval band.
- **Constraints**: Compressed to ensure file size ≤ 5 MB.

### `model_residuals.png`
- **Source**: `code/viz/plots.py` (`plot_residuals`).
- **Description**: Diagnostic plot showing residuals vs. fitted values to check for heteroscedasticity and model fit issues.

### `model_qq.png`
- **Source**: `code/viz/plots.py` (`plot_residuals`).
- **Description**: Quantile-Quantile plot to assess the normality of the regression residuals.

## 3. Logs and State

### `data/data_exclusion_log.txt`
- **Source**: `code/utils/logging.py` (`log_exclusion`).
- **Description**: Records subjects excluded from the analysis due to missing scans, missing scores, or high motion, along with the reason code.

### `state/projects/PROJ-518...yaml`
- **Source**: `code/utils/versioning.py` (`update_state_file`).
- **Description**: Contains SHA-256 hashes of key data and code files to ensure reproducibility and track data versioning.

## 4. Configuration
- **`.env`**: (Optional) Environment variable overrides for paths and parameters.
- **`code/config.py`**: Defines default `WINDOW_SIZES`, `STEP`, and paths.
