# Implementation Plan: Predicting Insect Pollinator Networks from Floral Trait Data

**Branch**: `001-predict-pollinator-networks` | **Date**: 2026-07-14 | **Spec**: `spec.md`

## Summary

This feature implements a computational pipeline to predict plant-pollinator interaction networks using static floral trait data. The approach ingests bipartite matrices from the Web of Life database and floral trait metadata, preprocesses them (imputation, normalization, encoding), and trains a Random Forest classifier to predict link probabilities. 

**Key Methodological Shift**: The model uses a **Trait-Only** feature set (morphology, color, scent, sampling effort) to ensure generalizability to unseen species in held-out ecosystems. The validation strategy is a **Leave-One-Ecosystem-Out (LOEO)** cross-validation loop, replacing the statistically invalid single-point held-out test. This allows for a robust estimate of generalization error across the N~8-10 ecosystems. The plan implements a **Trait-Shuffled Null Model** as the primary validation for trait efficacy, addressing the circularity of degree-preserving nulls.

**Scope Reframing**: Given the limited number of ecosystems (N ~ 8-10), the project aims to **demonstrate proof-of-concept generalizability** rather than prove universal biological laws. Claims regarding "universal trait rules" are reframed as "demonstration of generalizability potential" within the observed dataset distribution.

The model is validated via stratified k-fold cross-validation (internal) and LOEO (external generalization).. The output includes performance metrics (AUC-ROC), trait importance rankings, and network visualizations.

## Implementation Phases

### Phase 0: Data Ingestion & Heuristic Mapping
*   **Goal**: Download interaction matrices and trait metadata for 10 target ecosystems.
*   **FR-001**: Implement downloader for Web of Life.
*   **Data Strategy**: Instead of manual curation, the pipeline uses a **Heuristic Mapping** strategy:
    1.  Attempt to download a known 'trait mapping file' from Web of Life metadata.
    2.  If unavailable, scrape the specific ecosystem's Web of Life study page for linked DOIs.
    3.  If still unavailable, search the Dryad API for the ecosystem name + "traits" and download the first match.
    4.  If no data is found, log a warning and skip the ecosystem.
*   **Risk Mitigation**: If < 8 ecosystems have complete trait data, log warning and proceed with available data (as per spec).
*   **Output**: `data/raw/` directory with raw files.

### Phase 1: Preprocessing & Feature Engineering
*   **Goal**: Create unified feature matrix (rows: pairs, cols: traits + label).
*   **FR-002**: Implement median imputation, winsorization (1st/99th), Z-score normalization, one-hot encoding.
*   **FR-007**: Generate negative samples (co-occurring unobserved pairs).
    *   **Co-occurrence Logic**: Co-occurrence is **derived strictly from the interaction matrix**. A pair `(plant, pollinator)` is defined as co-occurring if both species appear in the same interaction matrix file. Temporal metadata is used only if explicitly provided in the source; otherwise, spatial co-occurrence (same file) is the sole proxy, aligning with the spec's assumption that spatial metadata is sufficient.
    *   **Trait-Only Strategy**: To ensure generalization to unseen species, the feature matrix will **exclude** species IDs and ecosystem IDs. Only intrinsic plant traits (morphology, color, scent) and **sampling effort** are used as predictors.
    *   **Sampling Effort**: Extract 'sampling effort' (e.g., number of visits) from metadata. If unavailable, use a default or flag as missing. This helps the model distinguish true negatives from unobserved positives.
*   **Label Noise Mitigation (Critical)**: 
    *   **Problem**: Unobserved pairs are treated as "probabilistic negatives" (likely non-interactions), but many are false negatives (missed interactions). This biases the model.
    *   **Mitigation**: 
        1.  **Sampling Effort Covariate**: Explicitly include `sampling_effort` as a feature to help the model learn that low-sampling-effort pairs are less likely to be observed even if they interact.
        2.  **Sensitivity Analysis**: A secondary evaluation will be run on a subset of "high-confidence" negatives (pairs with high sampling effort but no interaction) to assess the impact of label noise on trait importance.
        3.  **Framing**: The model output is framed as "Probability of Observed Interaction" rather than "Probability of True Biological Link".
*   **Output**: `data/processed/feature_matrix.csv` and `train_test_splits.json`.

### Phase 2: Model Training & Cross-Validation
*   **Goal**: Train Random Forest with stratified k-fold cross-validation.
*   **FR-003**: Configure `RandomForestClassifier` (class_weight='balanced' or stratified sampling).
*   **FR-005**: Calculate permutation importance.
*   **Output**: Trained model artifact, CV metrics (AUC-ROC mean/std).

### Phase 3: Generalization Validation & Null Models
*   **Goal**: Evaluate generalizability using LOEO and compare against null models.
*   **FR-004**: Run **Leave-One-Ecosystem-Out (LOEO)** cross-validation.
    *   **Process**: Train on N-1 ecosystems, test on the held-out one. Repeat for all ecosystems.
    *   **Metric**: Calculate mean AUC-ROC across all LOEO folds.
*   **SC-001**: Implement **Trait-Shuffled Null Model** as the primary validation for trait efficacy.
    *   **Method**: Shuffle trait values among species within the ecosystem while keeping the network structure fixed. Retrain and evaluate.
    *   **Secondary**: Implement **Degree-Preserving Null Model** (as per spec) but explicitly state in the report that it tests network topology, not trait efficacy.
    *   **Permutations**: Default a sufficient number of iterations to ensure convergence. **Fallback**: If runtime > 3 hours, reduce to 100 iterations or single-shuffle approximation. Flag the result as "approximate" in the report.
*   **Sensitivity Analysis**: Re-run evaluation on a subset of "high-confidence" negatives (pairs with high sampling effort) to assess the impact of label noise (false negatives) on trait importance.
*   **Output**: Generalization metrics (mean AUC-LOEO), p-values (trait-shuffled), comparison plots.

### Phase 4: Reporting & Visualization
*   **Goal**: Generate final report and network plots.
*   **FR-006**: Generate NetworkX plots (observed vs. predicted).
*   **SC-002, SC-003, SC-004**: Compile metrics into Markdown summary.
*   **Constitution Check**: Ensure all citations in the report are validated by the Reference-Validator Agent (CI step).
*   **Output**: `results/report.md`, `results/plots/`.

## Compute Feasibility Strategy
*   **CPU-First**: Random Forest, Z-score, and NetworkX are CPU-tractable. The dataset size fits within 7 GB RAM.
*   **Dynamic Iteration Scaling**: For the permutation test, the pipeline monitors runtime. If it exceeds 3 hours, it automatically reduces iterations to 100 or switches to a single-shuffle approximation to stay within the 6-hour limit.
*   **Streaming**: If any ecosystem dataset is unusually large, the ingestion script will use chunked reading (`pandas.read_csv(chunksize=...)`) to stay within memory limits.
*   **Timeout Mitigation**: The fallback strategy ensures the pipeline completes even if the full permutation test is too slow.
