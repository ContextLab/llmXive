# Phenomenological AI: First-Person Experience Modeling
## Quick Start Guide

This guide provides instructions for setting up the environment and running the phenomenological AI research pipeline.

### Environment Setup

1. **Clone the repository** (if not already done):
 ```bash
 git clone <repository-url>
 cd PROJ-592-phenomenological-ai-first-person-experie
 ```

2. **Create a virtual environment**:
 ```bash
 python -m venv.venv
 source.venv/bin/activate # On Windows:.venv\Scripts\activate
 ```

3. **Install dependencies**:
 ```bash
 pip install -r code/requirements.txt
 ```

4. **Verify installation**:
 ```bash
 python -c "import llama_cpp; import numpy; import pandas; print('All dependencies installed successfully.')"
 ```

### Pipeline Execution

The main entry point is `code/main.py`. Use the `--task` flag to specify which phase of the pipeline to run.

#### 1. Generation Phase

Generate phenomenological reports using the TinyLlama model (CPU-tractable) and four prompting strategies.

```bash
python code/main.py --task generate --config code/config.py
```

**Output**: Generated reports saved to `data/raw/`.

#### 2. Control Corpus Generation

Generate control samples from the ArXiv NLP dataset for discriminant validity testing.

```bash
python code/main.py --task generate_control --config code/config.py
```

**Output**: Control samples saved to `data/raw/control_corpus.json`.

#### 3. Analysis Phase

Compute phenomenological metrics (Consistency, Stability, Marker Presence) and perform statistical analysis.

```bash
python code/main.py --task analyze --config code/config.py
```

**Output**: Validity scores and statistical results saved to `data/processed/validity_scores.csv` and `data/processed/stats_results.json`.

#### 4. Validation Phase

Select representative samples for human rating and calculate inter-rater reliability.

```bash
python code/main.py --task validate_human --config code/config.py
```

**Output**: Stratified sample saved to `data/qualitative/sampled_reports.json`.

#### 5. Statistics Phase

Run full statistical analysis including normality tests, ANOVA/Kruskal-Wallis, and post-hoc tests.

```bash
python code/main.py --task stats --config code/config.py
```

**Output**: Comprehensive statistical report saved to `data/processed/statistical_analysis.json`.

#### 6. Full Pipeline

Execute the entire pipeline from generation to validation in one command.

```bash
python code/main.py --task full --config code/config.py
```

### Data Outputs

After successful execution, the following artifacts will be available:

- `data/raw/`: Generated phenomenological reports and control corpus
- `data/processed/`: Validity scores, statistical results, and analysis outputs
- `data/qualitative/`: Stratified samples for human evaluation
- `figures/`: Visualizations (if generated by analysis scripts)

### Troubleshooting

- **CUDA Errors**: Ensure you are running on a CPU-only environment. The pipeline is configured for TinyLlama with `llama-cpp-python` on CPU.
- **Import Errors**: Verify that all dependencies are installed and the virtual environment is activated.
- **Missing Output Files**: Check that the `--config` path is correct and that the script completed without errors.

### Additional Scripts

- **Consistency Analysis**: `python code/analysis/consistency.py`
- **Stability Analysis**: `python code/analysis/stability.py`

For detailed usage of each script, refer to the docstrings within the respective files.
