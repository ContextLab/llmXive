# Data Model: llmXive Automation System (001-gene-regulation)

## Overview

This document defines the data entities, relationships, and schemas for the `llmXive` automation pipeline. The model supports hypothesis generation, novelty scoring, code execution, and statistical analysis.

## Entities

### 1. Hypothesis

A research claim generated by the LLM agent.

- **id**: Unique identifier (UUID).
- **text**: The hypothesis string (e.g., "Variable X is positively correlated with Variable Y").
- **variables**: List of variables involved (extracted from text or dataset schema).
- **novelty_score**: Float (0.0–1.0) from semantic similarity.
- **novelty_status**: Enum: `novel`, `non_novel`, `inconclusive` (for default 0.5 scores).
- **plausibility_score**: Float (perplexity-based, used for filtering).
- **arch_config**: JSON object defining the LLM parameters used (model, temp, rag, context_window).
- **source**: "generated" or "human_baseline".

### 2. ExecutionLog

A record of the code execution attempt for a hypothesis.

- **id**: Unique identifier (UUID).
- **hypothesis_id**: Reference to the Hypothesis.
- **code**: The generated Python script string.
- **status**: Enum: `Pass`, `Fail`.
- **error_type**: Enum: `None`, `ModuleNotFoundError`, `RuntimeError`, `NonDeterministic`, `MemoryError`, `LogicalError`, `Timeout`.
- **runtime_duration**: Float (seconds).
- **output**: stdout/stderr snippet.

### 3. Dataset

The input data for hypothesis generation.

- **id**: Unique identifier (UUID).
- **source_url**: Verified URL (e.g., HuggingFace).
- **file_path**: Local path to the file.
- **checksum**: SHA-256 hash of the file.
- **schema**: List of column names and types.
- **sample_size**: Number of rows used (if sampled).
- **splits**: List of splits fetched (e.g., `["train", "test"]`).

### 4. ArchConfig

A set of parameters defining the LLM run.

- **id**: Unique identifier (UUID).
- **model**: String (e.g., "llama-3-8b", "distilbert").
- **temperature**: Float.
- **retrieval_augmentation**: Boolean.
- **embedding_model**: String (for novelty scoring).
- **context_window**: Integer (structural parameter).
- **model_size**: String (e.g., "8B", "70B" - structural parameter).

## Relationships

- **Dataset** (1) → (N) **Hypothesis**: One dataset generates many hypotheses.
- **Hypothesis** (1) → (1) **ExecutionLog**: One hypothesis has one execution log (or retry logs).
- **ArchConfig** (1) → (N) **Hypothesis**: One configuration generates many hypotheses.

## Data Flow

1. **Ingest**: `Dataset` is downloaded and checksummed.
2. **Generate**: `Hypothesis` is created using `ArchConfig` and `Dataset` schema.
3. **Score**: `Hypothesis.novelty_score` is computed against literature corpus. **Plausibility** is checked.
4. **Execute**: `ExecutionLog` is created by running generated code.
5. **Analyze**: `ExecutionLog` and `Hypothesis` are aggregated for statistical analysis (LMM).

## Storage

- **Raw Data**: `data/raw/` (checksummed, immutable).
- **Processed Data**: `data/processed/` (sampled, derived).
- **Generated Code**: `code/generated/` (per hypothesis).
- **Logs**: `data/logs/` (execution logs, novelty scores).
- **Vector Index**: `data/lit_search/` (corpus index).
