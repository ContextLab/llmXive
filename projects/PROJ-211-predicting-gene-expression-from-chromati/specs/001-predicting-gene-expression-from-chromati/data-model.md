# Data Model: Predicting Gene Expression from Chromatin Accessibility

## Overview

This document defines the data structures, schemas, and relationships required for the pipeline. All data flows from `raw/` (synthetic) to `processed/` to `models/` and finally to `results/`.

## Entity Definitions

### 1. Cell Line
- **Description**: A biological sample identifier (e.g., GM12878).
- **Attributes**:
    - `cell_line_id`: String (unique identifier).
    - `source`: String (e.g., "Synthetic").
    - `replicates`: Integer (≥3).

### 2. Gene
- **Description**: A genomic feature with expression values.
- **Attributes**:
    - `gene_id`: String (e.g., "ENSG00000123456").
    - `symbol`: String (e.g., "TP53").
    - `tss_start`: Integer (1-based coordinate).
    - `tss_end`: Integer.
    - `chromosome`: String.

### 3. Peak
- **Description**: A genomic region with accessibility signal.
- **Attributes**:
    - `peak_id`: String (e.g., "peak_chr1_1000_2000").
    - `chromosome`: String.
    - `start`: Integer.
    - `end`: Integer.
    - `distance_to_tss`: Integer (calculated relative to nearest gene).

### 4. Expression Matrix
- **Description**: Log-transformed gene expression counts.
- **Dimensions**: Rows = Genes, Columns = Cell Lines.
- **Transformation**: $log_2(counts + 1)$.
- **Filtering**: Genes with zero expression in all samples are removed.

### 5. Accessibility Matrix
- **Description**: Aggregated peak accessibility signal.
- **Dimensions**: Rows = Genes (mapped to peaks), Columns = Cell Lines.
- **Aggregation**: Sum of reads within ±50kb of TSS.
- **Filtering**: Peaks with zero accessibility in all samples are removed.

## Data Flow

1.  **Ingestion**: Synthetic data is generated by `code/generate_data.py` into `raw/synthetic_multiomic.jsonl`.
2.  **Preprocessing**:
    -   TSS coordinates are loaded from the synthetic generation context.
    -   Peaks are mapped to genes if `|peak_center - tss_center| <= 50000`.
    -   Matrices are constructed and filtered (zero-expression genes).
3.  **Modeling**:
    -   Input: `accessibility_matrix` (X), `expression_matrix` (y).
    -   Output: `model.pkl` (sklearn ElasticNet object), `coefficients.csv`.
4.  **Evaluation**:
    -   Metrics: R², Pearson Correlation, Bonferroni-corrected p-values.
    -   Output: `results/metrics.csv`, `results/feature_importance.csv`.

## Constraints

-   **Memory**: Matrices must be stored as `float32` if `float64` exceeds 7GB RAM.
-   **Integrity**: All derived files must have a checksum recorded in `state.yaml`.
-   **Schema Validation**: All intermediate files must validate against `contracts/dataset_schema.schema.yaml`.
