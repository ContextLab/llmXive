## Unresolved concerns

- The plan states: 'Perform power_analysis.py to estimate effect size... If N > 500 is required, flag feasibility risk.' This is methodologically unsound for a fixed-constraint study. If the pilot indicates N=1000 is needed for 80% power, the study is underpowered at N=500. The plan must explicitly state that the study will report the *achieved* power (post-hoc) or the minimum detectable effect size (MDES) at N=500, rather than treating N=500 as a hard stop that invalidates the study if the effect is smaller than expected. A 'flag' does not resolve the statistical validity of the final conclusion.
- FR-008 requires validating the 'Generalization Gap' against an independent ground-truth metric (e.g., human evaluations or external unbiased benchmark). The plan.md 'Methodology' section mentions a 'Human Proxy' (HuggingFaceH4/image-reward) in Phase 3, but the 'Constitution' (Principle VII) and 'Spec' (FR-008) imply a need for a distinct ground truth to rule out circularity. The plan does not explicitly define a task or data artifact that satisfies FR-008's requirement for an *independent* validation source distinct from the VLMs used in FR-005. The 'Human Proxy' is another VLM, which may not satisfy the 'independent ground-truth' requirement if the hypothesis is about VLM circularity. The plan lacks a clear element addressing how FR-008 is satisfied if the proxy is not truly independent.
- SC-002 requires measuring statistical power against a minimum sample size for a medium effect size. The plan.md 'Methodology' describes a 'Pilot Phase' (Phase 1) to calculate power, but the 'Project Structure' and 'Quickstart' do not explicitly map a specific output artifact (e.g., a 'power_analysis_report.json') that serves as the measurable evidence for SC-002. While the script `pilot_power_analysis.py` exists, the plan does not explicitly state that its output is the *measured* artifact for SC-002, creating a gap between the success criterion and the plan's evidence trail.
- FR-009 mandates verifying OOD prompt independence (cosine similarity < 0.3). The plan.md 'Methodology' Phase 0 includes `validate_ood.py`, but the 'Constitution' (Principle VI) and 'Spec' (FR-009) require this to be a hard gate. The plan's 'Task' list (referenced in the kickback comments as T015/T016) indicates a potential race condition or lack of explicit sequential enforcement in the *current* plan text (which is missing the explicit task list here, but the kickback comments imply the plan's task ordering is flawed). The plan.md text provided does not explicitly list the tasks T015/T016 with their dependencies, making it impossible to verify if the plan element *explicitly* enforces the FR-009 gate before generation (FR-003/FR-004). The plan relies on the 'Methodology' narrative rather than a structured task dependency graph to satisfy FR-009.
- FR-007 requires a Wilcoxon signed-rank test with bootstrap resampling. The plan.md 'Methodology' Phase 3 mentions 'Wilcoxon signed-rank test' and 'bootstrap resampling', but the 'Constitution' (Principle VII) explicitly mandates a 'paired t-test' for the Generalization Gap claim. This is a direct contradiction between the Spec/Plan (Wilcoxon) and the Constitution (t-test). The plan does not resolve this conflict or provide a rationale for deviating from the Constitution's mandated statistical test, leaving FR-007 potentially unbacked by a compliant plan element if the Constitution is the source of truth for the test type.
- The plan cites 'Turkish Maps Reviews' (opdullah/turkish-google-maps-reviews) as a source for 'ID Prompts' to mirror Qwen-Image-Bench. This dataset contains short, location-specific user reviews in Turkish. It is semantically and linguistically mismatched with the likely English, descriptive, and diverse nature of an image generation benchmark (Qwen-Image-Bench). Using this to define the 'In-Distribution' centroid will result in a biased baseline that does not represent the actual training distribution of the image model, invalidating the 'Generalization Gap' comparison.
- The plan relies on 'HC3 French OOD' (Human vs. AI chat data) for 'OOD Prompts'. This dataset contains conversational text, not visual concepts. Converting chat logs into 'abstract physics' or 'obscure historical artifact' image prompts requires a translation step not described. Furthermore, the semantic distance of chat logs to image prompts is undefined. The cited datasets do not provide the necessary visual-semantic coverage to rigorously test 'zero-shot generalization' on image generation tasks.
- The 'Dataset Variable Fit' section claims the cited text-only datasets provide 'semantic distance suitable for OOD testing' for image generation. This is an unverified assertion. The plan must cite a source or methodology that explicitly links these text datasets to the visual domain (e.g., a paper using these for image prompt generation) or replace them with datasets that contain visual-semantic pairs (e.g., LAION subsets, COCO-OOD, or a custom curated image-caption set).
- Constitution Principle VII mandates a 'paired t-test' for the Generalization Gap. The plan explicitly implements a 'Wilcoxon signed-rank test' (Plan: Statistical Analysis, FR-007). This is a direct violation of the per-project constitution's statistical requirement.
- The 'Constitution Check' table lists Principle VII as 'PASS' despite the plan implementing Wilcoxon instead of the mandated t-test. This is a false positive compliance check. The 'Implementation Detail' column cites FR-007 (Wilcoxon) which contradicts the Principle text (t-test).
- Principle V (Versioning Discipline) requires updating the `state/` file. The plan mentions `state/` in the structure but does not explicitly detail the mechanism for updating the `updated_at` timestamp in the `Constitution Check` table, unlike other principles which have specific implementation details.
- The plan defines 'Generalization Gap' as (OOD Degradation) - (ID Degradation) and tests this against zero. However, the 'Human Proxy' validation (FR-008) is described as comparing results against an external metric to 'rule out circular dependency'. If the Human Proxy is a *different* metric (e.g., human ratings) than the VLM scores used to calculate the Gap, the test is not circular but the plan fails to define how the *Gap* itself is validated against the proxy. If the proxy is the *same* metric type (VLM), it is circular. The plan must explicitly state: Is the 'Generalization Gap' calculated on the VLM scores, and is the 'Human Proxy' a *separate* ground-truth calculation of the *same* Gap metric? If the proxy is just a correlation check, it does not validate the *existence* of the gap, only its correlation with another noisy signal. The current phrasing implies a tautological check or a category error in validation.
- The plan proposes a Wilcoxon signed-rank test on the 'Generalization Gap' (OOD Degradation - ID Degradation). This implies a paired test where each 'pair' is a single prompt's degradation difference. However, the 'Generalization Gap' is a *derived* quantity (Difference of Differences). The statistical independence of the 'Degradation' values for ID and OOD sets is not guaranteed if the same prompts are used for both (which they are not, per design) or if the VLM scoring model has a systematic bias that affects both sets similarly. More critically, the plan treats the 'Gap' as a single scalar per prompt, but the degradation is calculated as (Base - RL). If the VLM scores are bounded (0-1) and non-normal, the *difference* of two such scores (Degradation) may have a distribution that violates the assumptions of the Wilcoxon test (which assumes the distribution of differences is symmetric). The plan does not address the symmetry assumption of the Wilcoxon test for the derived 'Degradation' variable.
- The plan uses a pilot (N=20) to estimate effect size (Cohen's d) for a power analysis. However, the pilot uses the *same* VLM reward models that will be used for the full study. If the VLM models have a systematic bias or low variance (e.g., they always score 0.8 for 'aesthetics'), the pilot will estimate a near-zero effect size or near-zero variance, leading to an incorrect sample size calculation. The power analysis assumes the pilot's variance is representative of the full study's variance. If the VLM is 'saturated' (scores cluster at extremes), the pilot will fail to detect the true effect size, leading to an underpowered study. The plan does not account for the potential 'ceiling/floor effects' of the VLM reward models in the power analysis.
