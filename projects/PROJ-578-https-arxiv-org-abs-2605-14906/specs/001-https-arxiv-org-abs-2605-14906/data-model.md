# Data Model: Reproduce & Validate MemLens Benchmark

## Overview

The data model for the MemLens benchmark reproduction consists of three primary entities: `EvaluationRun`, `Question`, and `Result`. These entities capture the configuration of the benchmark, the specific test items, and the outcomes of the evaluation.

## Entities

### EvaluationRun

Represents a single execution of the benchmark with specific configuration parameters.

| Attribute | Type | Description | Constraints |
| :--- | :--- | :--- | :--- |
| `run_id` | string | Unique identifier for the run (UUID) | Required, Unique |
| `model_name` | string | Name of the model used (e.g., "gpt-4", "llava-7b") | Required |
| `context_length` | integer | Context window size in tokens (32000, 64000, etc.) | Required, Enum: 32000, 64000, 128000, 256000 |
| `ablation_mode` | boolean | Whether image-ablation was enabled | Required, Default: false |
| `cpu_only` | boolean | Whether the run was executed in CPU-only mode | Required, Default: true |
| `timestamp` | datetime | ISO 8601 timestamp of the run start | Required |
| `status` | string | Status of the run (e.g., "completed", "oom", "rate_limited") | Required |
| `error_code` | string | Specific error code if status is not "completed" | Optional |

### Question

Represents a single test item from the MemLens benchmark dataset.

| Attribute | Type | Description | Constraints |
| :--- | :--- | :--- | :--- |
| `question_id` | string | Unique identifier for the question | Required, Unique |
| `memory_ability` | string | Category of memory ability (e.g., "information_extraction") | Required, Enum: 5 defined types |
| `has_image_evidence` | boolean | Whether the question requires visual evidence | Required |
| `context_tokens` | integer | Number of tokens in the provided context | Required |
| `ground_truth` | string | The correct answer (if available) | Optional |

### Result

Represents the outcome of evaluating a specific question within an EvaluationRun.

| Attribute | Type | Description | Constraints |
| :--- | :--- | :--- | :--- |
| `result_id` | string | Unique identifier for the result | Required, Unique |
| `run_id` | string | Foreign key to EvaluationRun | Required |
| `question_id` | string | Foreign key to Question | Required |
| `predicted_answer` | string | The model's generated answer | Required |
| `is_correct` | boolean | Whether the prediction matches ground truth | Required |
| `latency_ms` | integer | Time taken to generate the answer (ms) | Optional |
| `tokens_used` | integer | Number of tokens consumed in the request | Optional |

## Relationships

-   **EvaluationRun** `1` -- `N` **Result**: One run produces many results.
-   **Question** `1` -- `N` **Result**: One question can be evaluated in many runs.
-   **Result** `N` -- `1` **EvaluationRun**: Each result belongs to one run.
-   **Result** `N` -- `1` **Question**: Each result corresponds to one question.

## Data Flow

1.  **Input**: The `EvaluationRun` configuration is initialized via CLI arguments.
2.  **Load**: The `Question` entities are loaded from the vendored dataset.
3.  **Process**: For each question, the `Result` is generated by the model (local or API).
4.  **Output**: The `Result` entities are aggregated into a JSON artifact (`results.json`).

## Schema Validation

The output `results.json` must conform to the schema defined in `contracts/result_metric.schema.yaml` and `contracts/evaluation_run.schema.yaml`. The input `questions.jsonl` must conform to `contracts/question.schema.yaml`.
