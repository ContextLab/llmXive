# Assessing Uncertainty Quantification Techniques for Materials Property Predictions  

**Field**: materials science  

## Research question

How do Gaussian‑process regression, Monte‑Carlo dropout, deep ensembles, and conformal prediction differ in calibration error and prediction‑interval sharpness when applied to machine‑learning models that predict materials properties such as elastic modulus, band gap, and thermal conductivity?

## Motivation

Machine‑learning models are increasingly used to predict materials properties, but most reports provide only point estimates, leaving designers without reliable risk information. Systematically comparing lightweight uncertainty‑quantification (UQ) techniques will reveal which methods give trustworthy intervals for different property classes, guiding risk‑aware materials discovery pipelines.

## Related work

- [Dual Accuracy-Quality-Driven Neural Network for Prediction Interval Generation (2022)](https://arxiv.org/abs/2212.06370) — Introduces a neural‑network architecture that directly optimizes prediction‑interval coverage and width, providing a methodological baseline for evaluating interval‑based UQ.  
- [The Deep Arbitrary Polynomial Chaos Neural Network or how Deep Artificial Neural Networks could benefit from Data-Driven Homogeneous Chaos Theory (2023)](https://arxiv.org/abs/2306.14753) — Explores a chaos‑theory‑inspired network that yields analytically tractable uncertainty estimates, relevant as an alternative UQ approach for materials‑property regression.  
- [Bayesian Neural Networks (2020)](https://arxiv.org/abs/2006.01490) — Reviews Bayesian neural‑network formulations for regression, a classic probabilistic UQ technique that can be compared with more recent lightweight methods.

## Expected results

We expect to observe systematic trade‑offs: methods such as deep ensembles may achieve lower expected calibration error but produce wider intervals (lower sharpness), whereas conformal prediction may offer tighter intervals with slightly higher mis‑coverage. Confirmation will come from statistically significant differences (paired t‑tests or Wilcoxon signed‑rank tests) in calibration error and interval width across the three public materials datasets; a null result (no clear ranking) would still be informative for the community.

## Methodology sketch

- **Data acquisition**:  
  - Download elastic‑modulus data from the Materials Project (`https://materialsproject.org/downloads`).  
  - Download band‑gap data from OQMD (`https://oqmd.org/static/oqmd.db`).  
  - Download thermal‑conductivity data from AFLOW (`https://aflow.org/`).  
- **Pre‑processing**: Standardize compositions, compute feature vectors using the `matminer` featurizers (e.g., `composition`, `structure` descriptors). Split each dataset into 70 % train / 15 % validation / 15 % test (stratified by property range).  
- **Baseline model**: Train a small graph‑neural‑network (e.g., CGCNN with ≤2 M parameters) or a Gradient‑Boosting Regressor (XGBoost) on the training split; select hyperparameters on the validation set.  
- **Apply UQ techniques** (each added to the same baseline predictor):  
  1. **Gaussian‑process regression (GPR)** – train directly on the same features.  
  2. **Monte‑Carlo dropout** – enable dropout at inference, draw 30 stochastic forward passes per test point.  
  3. **Deep ensembles** – train 5 independently initialized baseline models; aggregate predictions.  
  4. **Conformal prediction** – use split‑conformal method on validation residuals to construct 90 % prediction intervals for the test set.  
- **Evaluation metrics** (computed on the held‑out test set, independent of training):  
  - Expected Calibration Error (ECE) for prediction intervals.  
  - Reliability diagrams (visual check).  
  - Average prediction‑interval width (sharpness).  
  - Computational cost (training + inference time, peak RAM).  
- **Statistical analysis**: For each metric, perform paired statistical tests across the three datasets to assess whether differences between UQ methods are significant (α = 0.05).  
- **Reproducibility**: All scripts will be written in Python, using `pandas`, `scikit‑learn`, `torch`, and `matminer`; environment captured via a `requirements.txt`. The full workflow will be orchestrated with a single `bash` script that runs in ≤1 hour on a GitHub‑Actions runner (2 CPU cores, ≤2 GB RAM).
