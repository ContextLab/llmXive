# Design Document: Evaluating the Impact of Code Generation Models on Code Testability

## 1. Introduction

This document outlines the architectural design and data flow for the project "Evaluating the Impact of Code Generation Models on Code Testability". The goal is to systematically compare code generated by LLMs against human-written benchmarks using rigorous statistical methods.

## 2. System Architecture

The system follows a linear pipeline architecture with distinct stages:

1. **Data Ingestion**: Downloads and samples the HumanEval dataset.
2. **Generation**: Produces code candidates using specified LLMs.
3. **Analysis**: Computes static and dynamic metrics.
4. **Statistical Validation**: Applies hypothesis tests to compare distributions.
5. **Reporting**: Visualizes results and generates a narrative report.

## 3. Data Model

### 3.1 Input Data
- **HumanEval**: JSONL format containing `task_id`, `prompt`, `test`, and `entry_point`.

### 3.2 Intermediate Data
- **Generated Code**: Stored in `data/generated/` keyed by `task_id` and `source_type`.
- **Metrics**: `data/analysis/metrics.json` containing:
 - `task_id`: Unique identifier.
 - `source_type`: "human", "codegen", "codellama", etc.
 - `cyclomatic_complexity`: Integer.
 - `halstead_volume`: Float.
 - `branch_coverage_pct`: Float or `[deferred]`.
 - `pass_rate`: Binary (0/1).

### 3.3 Output Data
- **Statistical Results**: `data/analysis/sensitivity_metrics.json` with p-values, effect sizes (Cohen's d).
- **Report**: `results_report.md`.

## 4. Component Design

### 4.1 `download_data.py`
- **Responsibility**: Fetches HumanEval from HuggingFace, verifies SHA256, applies stratified sampling.
- **Dependencies**: `huggingface_hub`, `pandas`.

### 4.2 `generate_code.py`
- **Responsibility**: Loads LLMs (local or API), generates code, handles retries.
- **Dependencies**: `transformers`, `torch`, `requests` (for API).

### 4.3 `analyze_metrics.py`
- **Responsibility**: Runs `radon` for complexity, `pytest` for coverage/pass rate.
- **Dependencies**: `radon`, `pytest`.

### 4.4 `statistical_tests.py`
- **Responsibility**: Implements Wilcoxon, McNemar, Fisher, Permutation tests, and Power Analysis.
- **Dependencies**: `scipy`, `numpy`.

### 4.5 `report_generator.py`
- **Responsibility**: Generates plots via `matplotlib`, renders Jinja2 templates.
- **Dependencies**: `matplotlib`, `jinja2`.

## 5. Error Handling Strategy

- **Data Fetch Failures**: Retry with exponential backoff; fail loudly if source unreachable.
- **Generation Failures**: Log to `errors.log`, mark sample as missing, continue pipeline.
- **Metric Calculation Failures**: Record `[deferred]` for coverage, log warning, proceed.
- **Statistical Edge Cases**: Handle zero-variance cases in permutation tests.

## 6. Security and Integrity

- **Checksums**: All downloaded artifacts verified via SHA256.
- **State Tracking**: `state/artifact_hashes.yaml` tracks integrity of all outputs.
- **Citation Validation**: `validate_citations.py` ensures all references are verified.

## 7. Future Considerations

- Support for additional datasets (e.g., MBPP).
- Integration with CI/CD for automated re-evaluation.
- Extension to multi-language support (Java, C++).
