# Research Protocol: Evaluating Code Generation Impact on Testability

## 1. Research Question

Does the choice of code generation model (parameter size and architecture) significantly impact the **testability** of the resulting code, as measured by structural complexity and functional test pass rates?

## 2. Hypotheses

- **H1 (Structural Complexity)**: Code generated by larger models (e.g., CodeLlama-7B) will exhibit significantly lower cyclomatic complexity and Halstead volume compared to smaller models (e.g., CodeGen-350M).
- **H2 (Functional Success)**: Code generated by larger models will have a significantly higher pass rate on the HumanEval test suite.
- **H3 (Correlation)**: There is a significant negative correlation between structural complexity metrics and functional pass rates.

## 3. Methodology

### 3.1 Dataset
- **Source**: HumanEval (HuggingFace: `openai_humaneval`)
- **Sampling**: Stratified sampling based on human pass-rate quartiles to ensure representative difficulty distribution.
- **Target Sample Size**: 50 tasks (minimum 38 required for 80% power at d=0.5).

### 3.2 Models
- **Primary Model**: Salesforce/codegen-mono-350M (Local)
- **Sensitivity Models**:
 - CodeLlama-3B (Local, 4-bit quantization)
 - CodeLlama-7B (HuggingFace Inference API fallback)

### 3.3 Metrics
- **Complexity**:
 - Cyclomatic Complexity (via `radon`)
 - Halstead Volume (via `radon`)
- **Functional**:
 - Pass Rate (Binary: 1 if all tests pass, 0 otherwise)
 - Branch Coverage (via `pytest --cov`)

### 3.4 Statistical Analysis
- **Paired Tests**: Wilcoxon Signed-Rank Test (Complexity), Permutation Test (Coverage)
- **Binary Tests**: McNemar's Test (Pass Rate)
- **Correlation**: Spearman's Rank, Point-Biserial
- **Power Analysis**: A Priori (validate n) and Post-Hoc (observed power)

## 4. Execution Workflow

1. **Data Acquisition**: Download and verify HumanEval.
2. **Generation**: Generate code for each task using the primary model.
3. **Analysis**: Compute metrics and execute tests.
4. **Sensitivity**: Repeat generation/analysis for sensitivity models if resources allow.
5. **Aggregation**: Merge all results into `data/analysis/metrics.json`.
6. **Testing**: Run statistical tests on the aggregated data.
7. **Reporting**: Generate `results_report.md` with visualizations.

## 5. Reproducibility

- **Integrity**: All downloaded datasets and generated artifacts are verified via SHA256 checksums stored in `state/artifact_hashes.yaml`.
- **Logging**: All operations are logged with timestamps and task IDs.
- **Code**: All scripts are version-controlled and dependencies are pinned in `code/requirements.txt`.

## 6. Limitations

- **Compute Constraints**: Sensitivity analysis with larger models may be limited by local RAM or API rate limits.
- **Model Bias**: Results are specific to the selected models and may not generalize to all LLMs.
- **Test Suite**: HumanEval is a synthetic benchmark; real-world testability may differ.
