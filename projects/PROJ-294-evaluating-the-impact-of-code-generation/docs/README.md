# Evaluating the Impact of Code Generation Models on Code Testability

## Project Overview

This project investigates the relationship between code complexity metrics and testability in code generated by Large Language Models (LLMs). We analyze HumanEval tasks generated by different CodeLlama models (350M, 3B, 7B) to determine if structural complexity correlates with functional success (test pass rates).

## Research Questions

1. Does generated code complexity (Cyclomatic, Halstead) correlate with test pass rates?
2. Do larger models (7B) produce more testable code than smaller models (350M)?
3. Is there a trade-off between code complexity and functional correctness?

## Usage

### Full Pipeline Execution

Run the entire pipeline from data download to final report:

```bash
# 1. Download and sample HumanEval dataset
python code/download_data.py

# 2. Generate code using CodeLlama models
python code/generate_code.py

# 3. Analyze metrics (complexity, coverage, pass rates)
python code/analyze_metrics.py

# 4. Run statistical tests
python code/statistical_tests.py

# 5. Generate final report with visualizations
python code/report_generator.py
```

### Individual Components

#### Data Acquisition
```bash
python code/download_data.py
# Outputs: data/raw/humaneval.json, data/analysis/sampled_tasks.json
```

#### Code Generation
```bash
python code/generate_code.py
# Outputs: data/generated/code_samples.json
# Models: CodeLlama-350M (local), CodeLlama-3B (local/quantized), CodeLlama-7B (API fallback)
```

```bash
python code/analyze_metrics.py

# Metrics: cyclomatic_complexity, halstead_volume, branch_coverage_pct, pass_rate
```

#### Statistical Testing
```bash
python code/statistical_tests.py

# Tests: Wilcoxon, McNemar, Fisher's Exact, Permutation, Spearman correlation
```

#### Report Generation
```bash
python code/report_generator.py
# Outputs: results/report.md, results/figures/*.png
```

## Statistical Methods

- **Wilcoxon Signed-Rank Test**: Paired comparison of continuous metrics (complexity, coverage)
- **McNemar's Test**: Binary pass-rate comparison between models
- **Fisher's Exact Test**: Unpaired exploratory analysis
- **Permutation Test**: Robust paired comparison for branch coverage
- **Spearman's Rank Correlation**: Testing independence of complexity from functional success
- **Power Analysis**: A priori (sample size validation) and post-hoc (achieved power)

## Sensitivity Analysis

The project includes a sensitivity analysis comparing:
- **CodeLlama-350M**: Base model (local execution)
- **CodeLlama-3B**: Medium model (local with 4-bit quantization)
- **CodeLlama-7B**: Large model (HuggingFace Inference API fallback)

Results are merged into `data/analysis/metrics.json` and included in the final report.

## Error Handling

- **Missing LLM Samples**: Logged to `errors.log`, marked as `[missing]` in metrics
- **Execution Failures**: Coverage marked as `[deferred]`, pass_rate still recorded
- **API Failures**: Automatic fallback to local models, logged with timestamps
- **Data Integrity**: SHA256 checksums verified at download and before analysis

## Logging

All tasks are logged with timestamps and task IDs to `logs/pipeline.log`.
Configure logging level via environment variable: `LOG_LEVEL=DEBUG`

## Reproducibility

- All artifacts are versioned with SHA256 hashes in `state/artifact_hashes.yaml`
- Random seeds are fixed for sampling (seed=42)
- Dependency versions are pinned in `requirements.txt`

## Known Limitations

1. **API Rate Limits**: CodeLlama-7B generation may be throttled by HuggingFace API
2. **Memory Constraints**: Local 3B model requires ~6GB RAM with 4-bit quantization
3. **Execution Time**: Full pipeline may take 30+ minutes on standard hardware
4. **Coverage Measurement**: Branch coverage may fail for complex control flow (marked as `[deferred]`)
