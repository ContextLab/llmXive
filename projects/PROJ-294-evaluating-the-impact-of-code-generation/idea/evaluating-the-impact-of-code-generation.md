# Evaluating the Impact of Code Generation Models on Code Testability

**Field**: computer science

## Research question

Does code generated by Large Language Models (LLMs) exhibit statistically significant differences in structural testability metrics (e.g., cyclomatic complexity, branch coverage potential) compared to human-written code on standardized benchmarks?

## Motivation

While LLMs accelerate development, their impact on software quality assurance remains uncertain. If generated code systematically lacks testability or introduces hidden complexity, it could degrade maintainability and increase testing costs. This study addresses the gap in understanding how AI-assisted code affects downstream verification efforts.

## Related work

- [Execution-based Evaluation for Data Science Code Generation Models (2022)](http://arxiv.org/abs/2211.09374v1) — Establishes execution-based metrics for evaluating code generation, relevant for measuring functional correctness alongside structural properties.
- [ReCode: Robustness Evaluation of Code Generation Models (2022)](http://arxiv.org/abs/2212.10264v1) — Highlights the brittleness of LLM code, suggesting potential variance in stability that may correlate with testability challenges.
- [Multi-lingual Evaluation of Code Generation Models (2022)](http://arxiv.org/abs/2210.14868v3) — Provides benchmark datasets (MBXP, HumanEval) that serve as the foundation for comparing human vs. model-generated code.
- [SIMCOPILOT: Evaluating Large Language Models for Copilot-Style Code Generation (2025)](http://arxiv.org/abs/2505.21514v1) — Introduces interactive evaluation scenarios, offering context on how generated code fits into developer workflows.

## Expected results

We expect to find that LLM-generated code has lower cyclomatic complexity but higher redundancy compared to human code. Confirmation will rely on statistically significant differences in static analysis metrics and unit test pass rates across the HumanEval benchmark subset. Evidence will be considered strong if p-values < 0.05 in non-parametric tests comparing the two distributions.

## Methodology sketch

- Download the HumanEval dataset from HuggingFace Datasets (URL: `https://huggingface.co/datasets/openai_humaneval`).
- Install static analysis tools (`radon`, `coverage.py`) compatible with CPU-only execution on GitHub Actions.
- Run `radon` to compute cyclomatic complexity and halstead metrics on the reference human solutions.
- Generate code for a subset (n=50) of HumanEval tasks using `Salesforce/codegen-350M-mono` (CPU-optimized, fits within 7GB RAM).
- Run the same static analysis pipeline on the generated code samples.
- Attempt to execute tests for both human and generated code using the provided unit test harnesses.
- Perform Mann-Whitney U tests to compare complexity and coverage distributions between human and LLM code groups.
- Visualize results using `matplotlib` to generate comparison histograms for the final report.
