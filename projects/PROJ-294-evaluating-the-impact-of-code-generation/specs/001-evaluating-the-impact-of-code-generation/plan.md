# Implementation Plan: Evaluating the Impact of Code Generation Models on Code Testability

## Summary

The project evaluates whether code generated by the `Salesforce/codegen-350M-mono` model exhibits different testability characteristics compared to humanвЂ‘written reference solutions from the HumanEval benchmark. The approach involves downloading a reproducible subset of HumanEval tasks, generating code solutions with the 350M model on CPU, running static analysis (`radon`) and **dynamic execution** (`coverage.py` in a sandbox) to obtain structural and functional metrics, and performing **paired** nonвЂ‘parametric statistical comparisons with appropriate postвЂ‘hoc power analysis. All steps are fully automated and designed to run within the GitHub Actions freeвЂ‘tier constraints.

## Technical Context

- **Language/Version**: Python 3.10+ (compatible with GitHub Actions runners)  
- **Primary Dependencies**: `datasets`, `torch==2.3.0` (CPUвЂ‘only), `transformers`, `radon`, `coverage`, `pytest`, `scipy`, `statsmodels`, `matplotlib`, `pandas`, `numpy`, `seaborn` (all pinned in `requirements.txt`)

- **Target Platform**: GitHub Actions Free Tier (Linux, 2 CPU, ~7 GB RAM, в‰¤6 h runtime)  
- **Constraints**: No GPU, no largeвЂ‘model inference; all models must fit in в‰¤7 GB RAM.

## Complexity Tracking

| Violation | Why Needed | Simpler Alternative Rejected |
|-----------|------------|------------------------------|
| Paired Design (Wilcoxon SignedвЂ‘Rank) | Controls for task difficulty; avoids selection bias when LLM fails on hard tasks. | Unpaired MannвЂ‘Whitney would confound task difficulty. |
| Fisher's Exact Test for `pass_rate` | Appropriate for binary paired outcomes. | MannвЂ‘Whitney on binary data is statistically inefficient. |
| Dynamic `dynamic_branch_coverage` | True measure of executed branches; avoids static proxy pitfalls. | Static "potential" coverage is not a valid testability metric. |
| Excluding CodeLlamaвЂ‘7B execution | Infeasible on freeвЂ‘tier RAM; prevents CI failure. | Attempting largeвЂ‘model inference would exceed resource limits. |
| Logging of failures | Ensures transparency per FRвЂ‘007 and SCвЂ‘005. | Silent dropping would hide data quality issues. |

## Methodology Overview

1. **Data Acquisition**  
   - Download HumanEval (`openai/openai_humaneval`) from the verified HuggingFace URL.  
   - Pin the dataset commit hash; compute SHA256 checksum stored in `data/metadata.yaml`.  
   - Randomly select 50 tasks (seed = 42) and save as `data/raw/subset.parquet`.

2. **Citation Validation**  
   - Run `validate_citations.py` which calls the ReferenceвЂ‘Validator Agent on all external URLs (dataset, model). Execution halts on any validation failure.

3. **Code Generation**  
   - Load `Salesforce/codegen-350M-mono` via `transformers` on CPU.  
   - Generate **one** solution per task, with up to 3 retries on timeout or generation error.  
   - SyntaxвЂ‘check each snippet; failures are logged (`errors.log`) and marked `generation_status = "syntax_error"`.

4. **Static & Dynamic Analysis**  
   - **Static**: `radon` computes cyclomatic complexity (`cc`) and full Halstead suite.  
   - **Dynamic**: Execute each generated (and human) snippet against its HumanEval test harness inside a sandboxed subprocess while `coverage.py` records **dynamic_branch_coverage** (percentage of branches actually executed).  
   - Record `pass_rate` (1.0 if all tests pass, else 0.0).  
   - Any execution crash or timeout is logged; the sample is excluded from coverage analysis but may still contribute to passвЂ‘rate if it fails.

5. **Paired Filtering**  
   - Retain only tasks where *both* human and LLM samples have `generation_status == "success"` and successful execution.  
   - Minimum viable paired set: n в‰Ґ 40 (SCвЂ‘001). If fewer, the pipeline aborts with a clear log entry.

6. **Statistical Analysis**  
   - **Continuous metrics** (`cyclomatic_complexity`, `halstead_volume`, `dynamic_branch_coverage`): Wilcoxon SignedвЂ‘Rank Test (paired, nonвЂ‘parametric).  
   - **Binary metric** (`pass_rate`): Fisher's Exact Test on the 2Г—2 contingency of pass/fail vs. source.  
   - **Significance threshold**: О± = 0.05 (no multipleвЂ‘comparison correction required because only three primary metrics are preвЂ‘registered).  
   - **Effect size**: Report `r` for Wilcoxon and odds ratio for Fisher's.  
   - **Power analysis**: PostвЂ‘hoc calculation using `statsmodels.stats.power`. The study is powered to detect **large** effects (Cohen's d в‰€ 0.8) with nв‰€50. If achieved power < 0.8 for moderate effects, results are labeled "inconclusive".  
   - **Decoupling analysis**: Partial correlation of `pass_rate` with `source_type` controlling for `cyclomatic_complexity`; logistic regression `pass_rate ~ source_type + cyclomatic_complexity` with coefficients reported to test independence of structural metrics from functional success.

7. **Versioning Update**  
   - `update_state.py` computes SHA256 hashes of all generated files and writes them to `state/projects/PROJ-294-evaluating-the-impact-of-code-generation.yaml` with a timestamp, satisfying Constitution Principle V.

8. **Visualization & Reporting**  
   - `visualize.py` creates histograms and boxвЂ‘plots (PNG) for each continuous metric, saved under `results/plots/`.  
   - `report.py` compiles `results/results_report.md` including all figures, statistical tables, power analysis summary, decoupling results, and a **limitations** section discussing the inability to run a largeвЂ‘model sensitivity analysis (qualitative discussion only).

## EdgeвЂ‘Case Handling

| Situation | Handling |
|-----------|----------|
| Generation produces syntactically invalid Python | Log to `errors.log`, mark sample as `syntax_error`, exclude from metric computation. |
| `radon` fails to parse a snippet | Catch exception, log, increment `parsing_failure` counter, exclude from static metrics. |
| Execution crashes or times out | Record `pass_rate = 0.0`, `dynamic_branch_coverage = null`; log the failure; sample may still be used for passвЂ‘rate comparison if needed. |
| HumanEval dataset version changes | Dataset version is pinned; any mismatch triggers a failure in `validate_citations.py`. |
