# PROJ-294: Evaluating the Impact of Code Generation Models on Code Testability

## Overview

This project evaluates how code generated by Large Language Models (LLMs) impacts code testability. We compare metrics like Cyclomatic Complexity, Halstead Volume, and Branch Coverage between human-written code (HumanEval) and code generated by models such as CodeGen-350M, CodeLlama-7B, and CodeLlama-3B.

## Quick Start

### Prerequisites
- Python 3.8+
- pip
- Access to HuggingFace (for datasets and models)

### Installation
```bash
pip install -r code/requirements.txt
```

### Running the Pipeline
To run the entire analysis pipeline from data download to report generation:
```bash
python code/run_pipeline.py
```

### Generating the Report
After the pipeline completes, view the results:
- **Metrics**: `data/analysis/metrics.json`
- **Sensitivity Analysis**: `data/analysis/sensitivity_metrics.json`
- **Final Report**: `results_report.md`
- **Figures**: `results/figures/`

## Project Structure

- `code/`: Source code for data processing, analysis, and reporting.
- `data/`: Raw and processed data artifacts.
- `results/`: Generated figures and reports.
- `state/`: Pipeline state and integrity tracking.
- `tests/`: Unit and integration tests.
- `docs/`: Detailed documentation.

## Key Features

- **Automated Pipeline**: End-to-end execution from data acquisition to statistical analysis.
- **Sensitivity Analysis**: Comparison of multiple LLM models.
- **Statistical Rigor**: Wilcoxon, McNemar, Permutation tests, and Power Analysis.
- **Comprehensive Reporting**: Automated Markdown report with visualizations.
