## Unresolved concerns

- The plan states: 'If no such single-source paired dataset is found... the system logs the specific reason, halts the statistical pipeline, and outputs a 'Hypothesis Unanswerable' report.' This design conflates 'data unavailability' with 'hypothesis unanswerability'. The research question (Can EEG predict tDCS response?) is scientifically valid; the current public data is merely insufficient. Concluding the hypothesis is 'unanswerable' based on a missing dataset is a logical error (confounding data scarcity with theoretical invalidity). The methodology should frame this as 'Data Insufficient for Current Analysis' rather than declaring the hypothesis unanswerable, which misrepresents the scientific conclusion.
- The plan relies on a 'Hypothesis Unanswerable Mode' as a primary outcome. However, the spec (FR-001, US-1) requires the system to 'explicitly state that the primary research question... is UNANSWERABLE with current public data'. This is a methodological flaw: the inability to find a specific dataset does not prove the biomarker relationship doesn't exist. The methodology should distinguish between 'No data found' (a logistical finding) and 'Hypothesis rejected' (a scientific finding). The current approach risks a false negative conclusion about the biomarker's validity.
- The 'Dual Mode Logic' is justified by the need to avoid 'invalid data assumptions'. However, the plan does not address the confound of 'publication bias' or 'dataset heterogeneity'. If the only available paired datasets are small or from specific populations (e.g., healthy young adults), the power analysis (FR-007) might be satisfied, but the model would still fail to generalize. The methodology lacks a strategy to assess dataset representativeness before running the power analysis, potentially leading to a 'Sufficient Power' flag on a biased sample.
- The research phase concludes 'no single-source paired dataset... has been identified' and decides to operate in 'Hypothesis Unanswerable Mode'. This is a premature conclusion. The methodology should include a systematic search strategy (e.g., searching OpenNeuro, PhysioNet, and specific tDCS repositories for 'EEG' AND 'tDCS' AND 'motor') rather than relying on a static list of 'verified sources'. The current approach assumes the absence of evidence is evidence of absence, which is a methodological error in the research design.
- FR-010 requires the system to 'define the primary outcome and analysis plan as pre-registered before data processing begins'. The plan.md describes the pipeline logic (ingestion, preprocessing, modeling) but contains no mechanism, task, or output artifact for generating or storing a 'pre-registration' document or hash. The plan assumes the analysis plan is implicit in the code, but the spec explicitly requires a pre-registration step distinct from the code execution.
- FR-012 requires the system to 'explicitly identify the canonical source for the tDCS motor performance data'. The plan.md and research.md state that 'NO verified source found' and the system will switch to 'Hypothesis Unanswerable Mode'. While the plan handles the *absence* of data, it does not define a task to *perform the explicit identification and verification* of the tDCS source as a distinct step before concluding it is missing. The logic assumes the absence rather than executing the verification task required by FR-012.
- FR-013 requires the system to 'explicitly identify the canonical source for the raw EEG time-series data'. Similar to FR-012, the plan assumes the PhysioNet source is the target but does not explicitly define a task to 'identify and verify' this source as a distinct step with an output artifact (e.g., a verified source manifest) before proceeding to ingestion. The plan jumps to 'Data Ingestion Gate' without the explicit identification step mandated by the spec.
- FR-009 requires the system to 'perform a normality check... If non-normal... apply a rank-based transformation or use a non-parametric test'. The plan.md mentions 'Normality Check' in the research section but the 'Technical Context' and 'Project Structure' do not explicitly map a task or module to the *conditional logic* of switching to a non-parametric test (Spearman) if the check fails. The plan implies the Ridge Regression (FR-004) is the only model, potentially missing the fallback path required by FR-009.
- SC-006 requires measuring 'Null distribution validity' against the 'Kolmogorov-Smirnov test for uniformity'. The plan.md mentions 'Permutation Testing' in the research section but does not explicitly state that the output will include the KS-test statistic or p-value for the null distribution. The 'Analysis Result' schema in data-model.md includes 'permutation_p_value' but does not explicitly capture the KS-test result required to satisfy SC-006.
- The plan cites a HuggingFace parquet link for 'PhysioNet EEG' (JasiekKaczmarczyk/physionet-preprocessed) as the primary source for raw EEG time-series. This dataset is preprocessed and lacks the raw voltage time-series required for the MNE pipeline (filtering, re-referencing, bad channel detection) mandated by FR-002 and Constitution Principle VI. The plan assumes this source provides 'Raw voltage time-series (BIDS/EDF)', which is factually incorrect for a preprocessed parquet file. The pipeline cannot execute the required preprocessing steps on this specific artifact.
- The plan states 'OpenNeuro (Parquet): The verified link points to openneuro-fslr64k, which is structural/functional MRI data, not EEG/tDCS.' This confirms the plan has correctly identified that the cited OpenNeuro source is inappropriate for the required EEG modality. However, the plan fails to identify ANY alternative single-source dataset containing paired EEG and tDCS outcomes. While the 'Hypothesis Unanswerable' mode is a valid fallback, the plan's 'Primary Mode' path relies on a dataset (the parquet file) that cannot support the required preprocessing, making the 'Primary Mode' technically infeasible even if a paired dataset were found in that specific format.
- Constitution Principle VII (Biomarker Validation) requires evaluation on an 'independent public dataset distinct from the primary'. The plan's 'Technical Context' and 'Summary' explicitly state: 'No cross-dataset subject matching' and 'No synthetic data or cross-dataset pairing is performed'. The plan lacks a defined mechanism to execute the mandatory generalization check on a second dataset (e.g., Kaggle EEG Motor Imagery) without violating the single-source ingestion gate. This is a direct contradiction between the plan's execution logic and the constitutional requirement.
- Constitution Principle IV (Single Source of Truth) requires figures/stats to trace to 'one row in data/ and one block in code/'. The plan mentions generating a 'manifest.json' but does not explicitly map this artifact to the specific 'Analysis Result' entity defined in data-model.md or the 'model_output' contract. The plan should explicitly state that the manifest maps output metrics to the specific schema fields (e.g., 'adjusted_r2' in model_output.schema.yaml) to ensure full traceability compliance.
- The 'Project Structure' lists 'contracts/dataset.schema.yaml', 'contracts/feature.schema.yaml', and 'contracts/output.schema.yaml'. However, the 'contracts/' directory in the input also contains 'eeg-features.schema.yaml', 'model_output.schema.yaml', and 'tdc-response.schema.yaml'. The plan's structure list is incomplete and inconsistent with the actual contract files provided, potentially leading to missing validation steps for 'model_output' and 'tdc-response' entities.
- Constitution Principle VII mandates evaluation on an 'independent public dataset distinct from the primary... to demonstrate generalizability.' However, the plan's 'Hypothesis Unanswerable Mode' (triggered by lack of a single-source paired dataset) halts the pipeline before any modeling occurs. If the primary dataset is missing the tDCS outcome (as stated in research.md), the model cannot be trained, and thus cannot be evaluated on a secondary dataset. The plan creates a logical contradiction: it requires a secondary validation step that is impossible to execute if the primary training data is missing the target variable. This renders the 'Generalization' requirement untestable under the plan's own failure conditions.
- The plan identifies the PhysioNet dataset as 'EEG only' and states 'NO verified source found' for tDCS outcomes. It then proposes to run the pipeline in 'Hypothesis Unanswerable Mode' if no paired data exists. However, the research question is 'Can EEG predict tDCS response?' If the only available data lacks the response variable, the hypothesis is not just 'unanswerable' due to data absence; the *metric* (predictive power) is undefined. The plan treats the absence of data as a valid 'result' (Hypothesis Unanswerable Mode), but scientifically, this is a data availability failure, not a scientific finding about the biomarker. The plan conflates 'data not found' with 'hypothesis tested and failed,' which is a category error in scientific reporting.
- The plan states: 'A power analysis (FR-007) runs before modeling. If the dataset N is insufficient... the system flags Underpowered and suppresses inferential statistics.' However, FR-007 specifies calculating N based on 'Cohen's d=0.5'. Cohen's d is a measure of effect size for *group differences* (e.g., tDCS vs. Sham), not for *predictive regression* (EEG features -> continuous response). Using a t-test effect size (d) to power a regression model (R²) is a methodological mismatch. The power analysis should be based on the expected R² or the correlation coefficient (r) between the strongest EEG feature and the tDCS response, not a group difference metric.
- FR-009 mandates: 'If the data is non-normal... apply a rank-based transformation or use a non-parametric test (Spearman) instead of Pearson/linear regression.' The plan then specifies using 'Ridge Regression' (FR-004). Ridge regression is a parametric linear model. If the data is non-normal, switching to Spearman correlation (a rank-based correlation coefficient) is not equivalent to running a Ridge Regression model. Spearman provides a correlation coefficient, not a multivariate predictive model with coefficients for multiple features. The plan proposes a 'fallback' that abandons the core modeling requirement (multivariate prediction) rather than adapting the model (e.g., using Rank-Ridge or robust regression). This breaks the scientific continuity of the analysis.
