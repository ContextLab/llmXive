## Unresolved concerns

- The plan proposes using Ridge Regression with 5-fold CV on a dataset where the target variable (tDCS response) is synthetic and mathematically decoupled from the features in Fallback Mode. While this validates code execution, it provides zero statistical power to detect the effect size of interest (Cohen's d = 0.5) because the signal-to-noise ratio is intentionally set to zero. The methodology fails to address how the project will proceed if no real paired data is found, as the fallback does not support the research hypothesis.
- SC-002-FB defines success as R² ≈ 0.0 (±0.05) in Fallback Mode. This confirms the synthetic data is decoupled, but it does not validate the biomarker hypothesis. The plan lacks a methodology for power analysis or sample size estimation for the Primary Mode, acknowledging 'Sample size is determined by available public data.' Without a minimum N to detect the expected effect size, the study is underpowered to answer the research question even if paired data is found.
- FR-006 mandates FDR correction for multiple comparisons. While 'validation.py' is listed in the plan, there is no explicit phase, contract, or data-model entity description confirming the implementation of the Benjamini-Hochberg procedure or the output of FDR-corrected p-values as required by the spec.
- The 'OpenNeuro tDCS' dataset cited (URL: huggingface.co/datasets/clane9/openneuro-fslr64k/...) is inappropriate for the research question. The URL points to 'fslr64k', which is a standard structural MRI template space (FSLR 64k), not a tDCS motor performance dataset. Furthermore, the plan relies on this source for 'tDCS motor performance scores' while simultaneously admitting in the 'Dataset Fit Analysis' that 'OpenNeuro: Contains tDCS outcomes (potentially). Missing: Paired EEG data'. Relying on a structural MRI template URL for behavioral tDCS outcomes is a fundamental data mismatch. The plan must identify a verified source containing actual tDCS behavioral outcomes (pre/post motor scores) paired or unpaired with EEG, or explicitly confirm the synthetic data generation (FR-002) is the ONLY path because no such real dataset exists.
- The 'EEG Motor Movement/Imagery' dataset citation (URL: huggingface.co/datasets/neurofusion/eeg-restingstate/...) is a HuggingFace proxy for a CSV file, not the canonical PhysioNet EEG Motor Movement/Imagery Dataset (which is a BIDS/EDF repository). While the URL may reach, the plan cites 'PhysioNet' in the text but provides a HuggingFace CSV link. This CSV likely contains only event markers or pre-processed features, not the raw EEG time-series required for the 'band-pass filter (1–45 Hz)' and 'bad channel rejection' steps defined in FR-003 and the Constitution (Principle VI). If the source is only a CSV of events, the preprocessing pipeline described in the plan is infeasible. The plan must cite the actual raw EEG source (e.g., PhysioNet accession) or confirm the CSV contains the necessary raw voltage data.
- The 'Constitution Check' section lists 'Reproducibility' as item 1, but the Constitution (FR-030) labels 'Reproducibility' as Principle I. The Plan's numbering (1-7) does not align with the Constitution's Roman Numeral structure (I-VII), creating potential confusion in traceability. While the content is present, the mapping should be explicit (e.g., 'Principle I: Reproducibility').
- The plan relies on 'PhysioNet Motor Movement/Imagery' for EEG and 'OpenNeuro tDCS' for outcomes, assuming they can be paired or that synthetic data is the only fallback. These are distinct public repositories with different subject populations and experimental protocols. There is no mechanism described to align subjects across these datasets (e.g., via a shared ID or demographic matching). If no paired data exists (as the plan admits is likely), the 'Primary Mode' is impossible. The plan treats the existence of paired data as a binary switch, but the scientific reality is that the 'Primary Mode' hypothesis (EEG predicts tDCS response) is untestable with the *specific* datasets listed unless a third, paired dataset is identified. The current dataset strategy makes the primary research question empirically unanswerable with the proposed sources.
