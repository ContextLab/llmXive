# Data Model: llmXive follow-up: extending "Qwen-Image-Flash: Beyond Objective Design"

## 1. Overview

This document defines the data structures for the synthetic logical reasoning pipeline. All data is stored in `data/` and follows the schemas defined in `contracts/`.

## 2. Entity Definitions

### 2.1 SyntheticProblem
A single logical reasoning instance generated by the rule‑based engine.

* **Attributes**:
  * `id`: Unique string identifier (UUID).
  * `entropy_level`: Enum {`high`, `low`, `target`}.
  * `structure_type`: String (e.g., "modus_ponens", "arithmetic_chain") to enable stratified sampling and distinctness checks (addresses methodology-4223fb8c).
  * `premises`: List of strings (the logical premises).
  * `operator`: String (logical operator or arithmetic function).
  * `question`: String (the query).
  * `ground_truth`: String (the correct answer).
  * `entropy_score`: Float (calculated metric of randomness/structure).
  * `trace_entropy`: Float (entropy of the teacher's CoT trace, if available).
  * `trace_consistent`: Boolean (true if trace entropy matches problem entropy; required by FR‑009).
  * `is_valid`: Boolean (flag for logical consistency; must be true for training).
  * `set_type`: Enum {`train`, `test_generalization`} (distinguishes training data from the balanced Generalization Set required by FR‑008).
  * `set_name`: String (optional human‑readable tag, e.g., `high_entropy`, `low_entropy`, `target_specific`, `generalization`).

* **Required**: `id`, `entropy_level`, `structure_type`, `premises`, `question`, `ground_truth`, `entropy_score`, `is_valid`, `set_type`.

### 2.2 DistillationRun
A record of a single training session for one entropy subset.

* **Attributes**:
  * `run_id`: Unique string identifier.
  * `entropy_subset`: Enum {`high`, `low`, `target`}.
  * `model_config`: JSON object (architecture, params).
  * `training_loss_curve`: List of floats (loss per epoch).
  * `convergence_epoch`: Integer (epoch where **validation loss first falls ≤ 0.1**; threshold‑based definition, matching research.md).
  * `final_accuracy`: Float (accuracy on the Generalization Set).
  * `total_runtime_seconds`: Float.
  * `peak_ram_gb`: Float.
  * `status`: Enum {`completed`, `failed_timeout`, `failed_non_converge`}.

* **Required**: `run_id`, `entropy_subset`, `convergence_epoch`, `final_accuracy`, `total_runtime_seconds`, `peak_ram_gb`, `status`.

### 2.3 StatisticalResult
The output of the analysis phase.

* **Attributes**:
  * `analysis_id`: Unique string identifier.
  * `test_type`: Enum {`anova`, `t_test_pairwise`}.
  * `statistic`: Float (F‑value or t‑value).
  * `p_value`: Float (raw).
  * `p_value_corrected`: Float (Bonferroni‑adjusted).
  * `correction_method`: String (e.g., "bonferroni").
  * `conclusion`: String (e.g., "Significant difference found").
  * `causal_framing`: Boolean (always true for this study).

* **Required**: `analysis_id`, `test_type`, `statistic`, `p_value`, `p_value_corrected`, `correction_method`, `conclusion`, `causal_framing`.

## 3. Data Flow

1. **Generation**: `logic_generator.py` produces `SyntheticProblem` records → `data/raw/*.csv`.
2. **Filtering**: Invalid or `trace_consistent = false` problems are removed.
3. **Training**: `distill_loop.py` consumes the filtered training set → produces `DistillationRun` records → `data/processed/runs.jsonl`.
4. **Evaluation**: Models are evaluated on the **Generalization Set** (`set_type: test_generalization`) → accuracy and convergence epoch recorded.
5. **Analysis**: `stats.py` consumes `DistillationRun` records → produces `StatisticalResult` → `data/processed/stats.json`.

## 4. Validation Rules

* **Entropy Separation**: The mean `entropy_score` of `low` subset must be < `high` subset (t‑test, p < 0.05) before training.
* **Trace Consistency**: For each training problem, `trace_consistent` must be true; otherwise the problem is excluded (FR‑009).
* **Generalization Set Balance**: The Generalization Set must contain **equal counts** of High, Low, and Target problems, and each problem must be **structurally distinct** from any training instance (verified by `structure_type` mismatch) (FR‑008).
* **Resource Limits**: `DistillationRun.peak_ram_gb` must be < 7.0.
* **Convergence Definition**: `convergence_epoch` is the first epoch where validation loss ≤ 0.1 (threshold‑based, matching research methodology).
