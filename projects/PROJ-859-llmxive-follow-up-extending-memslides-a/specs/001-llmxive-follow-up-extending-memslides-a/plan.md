# Implementation Plan: Trace Compressibility Analysis

## Summary

This project investigates the structural properties of multi-turn tool-execution traces that determine their compressibility into symbolic rules without degrading persona-aligned agent behavior. The approach involves generating a synthetic dataset of revision sessions based on the MemSlides schema, extracting structural metrics (entropy, repetition, variance), and performing **per-trace rule induction** to calculate a specific "Compressibility Score" for each trace. A separate global model is used for latency/accuracy benchmarking. The core analysis correlates structural metrics with the per-trace Compressibility Score.

## Phase Plan

### Phase 0: Research & Data Strategy (Addressing FR-001, FR-002, FR-003)
1. **Synthetic Data Generation**: Implement `synthetic_trace.py` to generate multi-turn sessions mimicking MemSlides schema.
   - *Constraint*: Must vary sequence length, tool types, and argument variance.
   - *Output*: Raw trace JSON files containing `exact_tool_sequence` and `raw_arg_variance`.
2. **Metric Extraction**: Implement `extract.py` to compute sequence entropy, tool-repetition frequency, and argument semantic variance (using `sentence-transformers/all-MiniLM-L6-v2`).
   - *Handling Edge Cases*: Define default values for undefined variance; handle zero-repetition traces.
3. **Per-Trace Rule Induction**: Implement `rule_induction.py` to induce a rule set **specifically for each trace** (or small batch) to reproduce its `final_state`.
   - *Target*: The model predicts the `final_state` (ground truth) from structural metrics.
   - *Compressibility Score*: Calculated as `RuleSetSize / TraceLength` conditioned on `Fidelity >= 90%`.
   - *Constraint*: Must be CPU-tractable.
   - *Output*: Per-trace compressibility scores and rule sets.

### Phase 1: Benchmarking & Statistical Analysis (Addressing FR-004, FR-005, FR-006, FR-007)
1. **Agent Implementation**: Implement `baseline.py` (raw memory) and `compressed.py` (using a **global** rule set induced from training data for latency comparison).
2. **Benchmark Execution**: Run both agents on held-out test set. Measure Edit Accuracy and Retrieval Latency.
3. **Statistical Analysis**:
   - *Primary*: Beta regression of **Fidelity Loss** (1 - CompressedAccuracy) on Structural Metrics. (Fidelity Loss is bounded [0,1], satisfying Beta regression assumptions).
   - *Secondary*: Multiple Linear Regression (MLR) if Fidelity Loss is transformed to log-odds, satisfying Constitution Principle VII as a robustness check.
   - *Correlation*: Spearman correlation between structural metrics and the per-trace Compressibility Score.
4. **Sensitivity Analysis**: Sweep compression thresholds to analyze fidelity stability.

### Phase 2: Validation & Reporting
1. **Contract Validation**: Ensure all outputs match `contracts/*.yaml` at the root level.
2. **Reproducibility Check**: Run full pipeline from scratch with pinned seeds.
3. **Documentation**: Update `research.md`, `data-model.md`, `quickstart.md`.
