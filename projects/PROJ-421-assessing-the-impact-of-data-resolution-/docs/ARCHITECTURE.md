# Architecture Documentation

## System Overview

This project implements an automated pipeline to assess the impact of spatial data resolution on statistical power. The system is modular, with distinct components for data ingestion, resampling, analysis, and visualization.

## Component Breakdown

### 1. Data Ingestion (`data_ingestion.py`)
- **Purpose**: Download and validate high-resolution NLCD data.
- **Key Functions**:
 - `download_with_progress`: Retrieves data from HuggingFace with retry logic.
 - `verify_download`: Validates checksums.
- **Dependencies**: `utils.py` (retry logic, checksumming).

### 2. Resampling (`resampling.py`)
- **Purpose**: Generate coarser resolution rasters.
- **Key Functions**:
 - `generate_resolution`: Applies nearest-neighbor resampling.
 - Windowed reading to manage memory.
- **Dependencies**: `config.py` (resolution factors), `utils.py` (windowed I/O).

### 3. Analysis (`analysis.py`)
- **Purpose**: Compute spatial autocorrelation and statistical power.
- **Key Functions**:
 - `create_binary_indicator_map`: Transforms land cover to binary (Forest=1).
 - `calculate_moran_i`: Computes Moran's I statistic.
 - `generate_null_distribution`: H0 via 1,000 permutations.
 - `simulate_h1_gibbs`: H1 via Gibbs Sampler with calibrated λ.
 - `calculate_statistical_power`: Rejection rate of H1.
- **Dependencies**: `pysal`, `numpy`, `config.py`.

### 4. Visualization (`visualization.py`)
- **Purpose**: Generate power curves and identify thresholds.
- **Key Functions**:
 - `find_threshold`: Locates resolution where power < 0.80.
- **Dependencies**: `pandas`, `matplotlib`.

### 5. Sensitivity Analysis (`sensitivity_analysis.py`)
- **Purpose**: Validate threshold stability.
- **Key Functions**:
 - `run_sensitivity_sweep`: ±10% factor variation.
 - `write_sensitivity_report`: Outputs stability confirmation.

### 6. Reporting (`generate_final_report.py`, `type2_error_analysis.py`)
- **Purpose**: Synthesize results into human-readable reports.
- **Key Outputs**:
 - `data/results/final_report.md`
 - `data/results/threshold_report.txt`

## Data Flow

1. **Raw Data** (`data/raw/`) → Downloaded by `data_ingestion.py`.
2. **Derived Data** (`data/derived/`) → Generated by `resampling.py`.
3. **Results** (`data/results/`) → Generated by `analysis.py`, `visualization.py`, `sensitivity_analysis.py`.

## Configuration

All paths, seeds, and resolution factors are managed in `config.py`.

## Error Handling

- Retry logic with exponential backoff for network operations.
- Memory-mapped I/O for large raster files.
- Checksum validation for data integrity.
