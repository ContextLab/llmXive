# Research: The Influence of Narrative Framing on Attitudes Towards AI Assistance

## Research Question
Does framing LLM assistance as a "collaborative partner" versus an "automated tool" lead to more positive attitudes, higher perceived usefulness, and greater trust among lay users?

## Theoretical Framework
This study is grounded in **Technology Acceptance Model (TAM)** and **Human-Computer Interaction (HCI)** theories regarding anthropomorphism and trust.
- **Davis (1989)**: Perceived Usefulness and Attitude towards Use are primary predictors of technology adoption.
- **McKnight et al. (2002)**: Trust in technology is a multi-dimensional construct (competence, integrity, benevolence).
- **Framing Effect**: The linguistic framing of an agent (social vs. mechanical) may prime different mental models, altering trust and perceived utility.

## Dataset Strategy

Since this is a primary data collection study (survey-based), there is no pre-existing public dataset for this specific experiment. The "dataset" will be generated via:
1.  **Stimulus Generation**: Controlled creation of two vignette texts (Partner vs. Tool) using Python scripts.
2.  **Data Collection Simulation/Import**: The system will support importing raw CSVs from survey platforms (e.g., Qualtrics, Google Forms) or generating synthetic data for pipeline testing (as per US-1, US-2).
3.  **Verified Sources**:
    -   *No external public dataset URL is applicable for the primary outcome data*, as this is a novel experiment.
    -   *Scale Validation Sources*: The instruments used (TAM, Trust Scale) are validated against primary literature (Davis, McKnight), not a raw dataset.

**Data Generation Plan**:
-   **Vignettes**: Generated by `code/01_stimulus_generation.py`.
-   **Responses**: Simulated for testing (using `numpy` with known effect sizes) and imported for real runs.
-   **Variables**:
    -   `condition`: Categorical (Partner, Tool)
    -   `attitude_score`: Continuous (7-item Likert sum)
    -   `usefulness_score`: Continuous (3-item Likert sum)
    -   `trust_score`: Continuous (4-item Likert sum)
    -   `manipulation_check`: Binary (Pass/Fail)

## Methodology & Statistical Plan

### 1. Stimulus Validation (FR-001, FR-010, SC-001, SC-005)
-   **Readability**: Flesch-Kincaid Grade Level calculated via `textstat`. Target: Difference ≤ 2.0.
-   **Sentiment**: VADER Compound Score calculated via `vaderSentiment`. Target: Difference ≤ 0.05.
-   **Semantic Similarity**: Sentence embeddings (`sentence-transformers/all-MiniLM-L6-v2`) used to compute cosine similarity. Target: High similarity (>0.95) except for the framing keywords, ensuring no other semantic drift.
-   **Human Pre-Test**: A small pilot (n=20) will rate vignettes on "Friendliness" and "Competence" (Likert 1-7) to validate that the framing difference is perceived as intended, addressing the limitation of automated lexicons.
-   **Action**: If constraints are not met, the vignette template is iteratively adjusted until criteria are satisfied.

### 2. Randomization (FR-002)
-   **Method**: Simple randomization (50/50 split) using a fixed seed for reproducibility.
-   **Verification**: Binomial test on assignment distribution (N=10,000 simulated runs) to ensure 1:1 ratio within tolerance.

### 3. Pilot Study (FR-011)
-   **Execution**: Run `code/03_pilot_study.py` with n≥30 participants.
-   **Validation**: Calculate the discrimination power of the manipulation check (e.g., Cohen's d between those who passed/failed on a dummy attention check, or AUC).
-   **Gate**: Full study deployment is blocked if the manipulation check does not show significant discrimination (p < 0.05).

### 4. Data Cleaning & Analysis Strategy (US-2, Edge Cases, FR-007)
-   **Primary Analysis Population (Intent-to-Treat)**: All randomized participants are included in the primary analysis, regardless of manipulation check failure. This avoids post-treatment bias where exclusion might correlate with the outcome.
-   **Exclusion Criteria**: Only participants with missing >50% of primary DV items or obvious bot responses (e.g., straight-lining) are excluded *before* analysis.
-   **Sensitivity Analysis (FR-007)**: A secondary analysis re-runs the primary tests excluding participants who failed the manipulation check. This assesses robustness but does not replace the ITT primary claim.
-   **Data Transformation**: Summed Likert scores (Attitude, Usefulness, Trust) are treated as continuous variables. Justification: With N=300 and multi-item scales, the Central Limit Theorem supports the normality of the sampling distribution of the mean/sum.

### 5. Statistical Analysis (FR-004, FR-005, FR-006, FR-008)
-   **Primary Test**: **Welch's t-test** (unpaired, unequal variances assumed) for each DV (Attitude, Usefulness, Trust).
    -   *Rationale*: Pre-specified to avoid the Type I error inflation associated with conditional test selection based on Shapiro-Wilk/Levene's results. Welch's test is robust to variance heterogeneity.
-   **Robustness Checks**:
    -   **Ordinal Regression**: Ordered logit/probit models to verify results hold when treating data as strictly ordinal.
    -   **Mann-Whitney U**: Non-parametric test as a secondary check.
-   **Effect Size**: Cohen's d (with 95% CI) for t-tests; Rank-biserial correlation for U-tests.
-   **Multiple Comparisons**: Benjamini-Hochberg (FDR) correction applied to the family of three tests (Attitude, Usefulness, Trust).
-   **Covariates**: Linear regression controlling for Age/Gender as a secondary robustness check.

### 6. Power Analysis & Sample Size (FR-009, SC-002)
- **Target**: Recruit N=350 (anticipating [deferred] attrition/failures) to ensure final analyzed N ≥ 300.
-   **Justification**: Power analysis (G*Power logic) targeting 80% power to detect d=0.4 at α=0.05 requires N≈310.
-   **Reporting Logic (SC-002)**:
    -   If final N ≥ 300: Report standard power (≥80% for d=0.4).
 - If final N < 300: **Do NOT calculate post-hoc power** (tautological). Instead, calculate and report the **Minimum Detectable Effect Size (MDES)** for [deferred] power given the achieved N. If MDES > 0.4, the report explicitly flags the study as "Underpowered for target effect size (d=0.4)".

## Decision Rationale (Compute Feasibility)
-   **CPU-Only**: All statistical operations (`scipy.stats`, `statsmodels`, `sentence-transformers` in CPU mode) are lightweight and run efficiently on 2-core CPU. No deep learning models or GPU inference is used.
-   **Data Volume**: Dataset size (N=350 rows, ~20 columns) is negligible (<1 MB), well within 7GB RAM limits.
-   **Runtime**: Analysis scripts are expected to complete in < 2 minutes, far below the established CI time limit.

## Risks & Mitigations
-   **Risk**: Recruitment failure (N < 300).
    -   *Mitigation*: Over-recruit to N=350. If N < 300, report MDES and interpret findings with appropriate caution regarding precision.
-   **Risk**: Stimuli not balanced on sentiment/semantics.
    -   *Mitigation*: Automated validation loop + Human Pre-test prevents deployment if SC-005 or semantic similarity criteria are not met.
-   **Risk**: Low manipulation check pass rate.
    -   *Mitigation*: ITT primary analysis ensures unbiased estimates. Sensitivity analysis quantifies the impact of exclusion.
-   **Risk**: Manipulation check invalidity.
    -   *Mitigation*: FR-011 Pilot Study gates the full experiment; if the check doesn't discriminate, the instrument is revised before main data collection.
