\section{Conclusion}

We introduced \method{}, which reframes harness improvement as a retrospective process that an agent can run on its own past experience, rather than a search guided by external ground-truth feedback.
The central idea is that an agent's own trajectories already contain the signal needed to improve it, since re-solving past tasks and comparing the outcomes exposes where the harness fails and what would fix it.
Across all three domains (software engineering, technical work, and knowledge work), this self-supervised loop yields consistent held-out gains and reshapes how the agent works.
We see \method{} as a step toward agents that keep improving from the experience they accumulate in deployment, where labeled validation data is rare.
