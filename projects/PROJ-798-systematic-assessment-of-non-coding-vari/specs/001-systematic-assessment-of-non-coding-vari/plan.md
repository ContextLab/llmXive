# Implementation Plan: Systematic Assessment of Non-Coding Variant Effects on Transcription Factor Binding Affinities

**Branch**: `001-gene-regulation` | **Date**: 2026-07-10 | **Spec**: `spec.md`

## Summary

This project implements a computational pipeline to quantify the effect of non-coding SNPs on Transcription Factor (TF) binding affinities using Position Weight Matrices (PWMs). The workflow ingests common human SNPs (MAF > 1%) from the 1000 Genomes Project, filters them against regulatory regions (promoters/enhancers) from ENCODE, calculates allele-specific binding score differences ($\Delta Score$) using JASPAR PWMs, and performs statistical enrichment analysis against GWAS Catalog loci. The implementation prioritizes CPU-tractable methods (scikit-learn, biopython) to ensure execution within free-tier CI constraints (limited RAM, bounded time).

**Critical Note on Data Sources**: The plan distinguishes between two modes:
1.  **Research Mode**: Uses real biological data (1000 Genomes, JASPAR 2024, ENCODE, GWAS Catalog) to validate the scientific hypothesis. This mode is required for the final research output and satisfies all functional requirements.
2.  **CI Validation Mode**: Uses synthetic data to validate pipeline logic and code correctness. **This mode cannot validate the biological hypothesis** due to the lack of real LD structures and biological signal. It is used solely for automated testing.

## Constitution Check

**Principle I (Reproducibility)**: The plan mandates pinned versions in `requirements.txt`, fixed random seeds for permutation tests, and checksums for all downloaded data in `data/`. For synthetic data, the checksum is generated from the seed and generator code hash.  
**Principle II (Verified Accuracy)**: All dataset citations in `research.md` are restricted to the provided "Verified datasets" list (1000 Genomes, JASPAR 2024, ENCODE, GWAS Catalog). The CI run uses synthetic data *instead* of verified external data, which is a known deviation from Principle II for the CI environment only. Principle II applies to the final research output.  
**Principle III (Data Hygiene)**: The plan defines a strict `data/raw/` vs `data/derived/` separation. No in-place modifications; all transformations produce new files with derivation logs.  
**Principle IV (Single Source of Truth)**: All statistical outputs (p-values, enrichment ratios) will be generated by scripts in `code/` and written to `data/`. The paper will reference these files, not hand-calculated numbers.  
**Principle V (Versioning)**: The plan includes a `checksums.json` generation step. For synthetic data, the checksum is generated from the seed and generator code hash. Artifacts will be versioned via content hashes.  
**Principle VI (Biophysical Proxy Validity)**: The plan strictly uses PWM-based scoring (JASPAR) as the primary method. No deep learning surrogates (e.g., Enformer) will be used for the core $\Delta Score$ calculation, adhering to the biophysical proxy requirement.  
**Principle VII (Population-Genetic Independence)**: The pipeline ensures that the SNP list (1000 Genomes) and the GWAS overlap targets are distinct datasets. The permutation test uses a 'Stratified Label Permutation' to preserve sequence context while breaking the variant-score link, maintaining independence between the variant set and the null generation.

**Spec Gap Flag**: 
- FR-001 mandates dbSNP as the primary source. The plan uses 1000 Genomes as a verified proxy for common human SNPs (MAF > 1%) due to the lack of a direct, programmatic dbSNP URL in the verified list. This is a documented deviation for the implementation, as 1000 Genomes is the standard source for common variants in this context.
- FR-002 mandates a ±10bp window. The plan uses a dynamic window (PWM-length dependent) to ensure accuracy. This is a methodological improvement over the spec's fixed window to avoid systematic bias.
- FR-004 mandates 'shuffling SNP positions'. The plan uses 'Stratified Label Permutation' to address methodological concerns regarding sequence context. This is a methodological improvement.

## Complexity Tracking

| Violation | Why Needed | Simpler Alternative Rejected Because |
|-----------|------------|-------------------------------------|
| Stratified Label Permutation Test (n=1000) | Required by FR-004 (updated to address methodological concerns) to establish a robust null distribution for significance testing while preserving sequence context. | Analytic approximations (e.g., Z-test) are insufficient for the non-Gaussian distribution of $\Delta Scores$ and the complex dependency structure of genomic regions. Shuffling positions changes the underlying sequence context, invalidating the null hypothesis. |
| West-Stephens max-T FDR Correction | Required by FR-006 to handle the dependence between correlated TF motifs. | Standard Benjamini-Hochberg and Storey-Tibshirani assume independence or positive dependence; West-Stephens is specifically designed for correlated tests in permutation frameworks. |
| Dynamic Window Size | Required to ensure the full binding site is captured for all TFs (PWM length varies). | Fixed ±10bp window may truncate longer motifs, introducing systematic bias in the $\Delta Score$ calculation. |
| Memory-Mapped FASTA | Required to handle large genomic sequences within 7 GB RAM. | Loading full chromosomes into RAM would exceed memory limits on the free-tier runner. |
