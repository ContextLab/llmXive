# Data Model: llmXive follow-up: extending "ArcANE"

## 1. Overview

This document defines the data structures used throughout the experiment lifecycle, from axis definition to final statistical results. All data is stored in JSONL (JSON Lines) format for streaming compatibility and easy parsing.

## 2. Core Entities

### 2.1. Character Axis
Defines the psychological profile of a character.
- **Source**: Manual input (FR-001), validated by two researchers.
- **Fields**:
  - `character_id`: Unique identifier (e.g., "elizabeth_bennet").
  - `coarse_axis`: Broad trait description (e.g., "Pride to Humility").
  - `fine_axis`: Nuanced phase description (e.g., "Initial Arrogance to Self-Reflection").
  - `source_text_id`: Reference to the source text segment.
  - `definition_kappa`: Inter-rater reliability score for axis definitions.

### 2.2. Probe
An "Out-of-World" scenario prompt.
- **Source**: Generated by `probe_generator` (FR-002).
- **Fields**:
  - `probe_id`: Unique identifier.
  - `character_id`: Linked character.
  - `scenario_text`: The generated scenario.
  - `cosine_similarity`: Similarity score against source text (must be < 0.3).
  - `status`: "valid", "regenerated", "discarded".

### 2.3. Experiment Result
The output of a single trial.
- **Source**: `experiment_runner` (FR-003).
- **Fields**:
  - `trial_id`: Unique identifier.
  - `probe_id`: Linked probe.
  - `condition`: "coarse", "fine", or "hybrid".
  - `model_response`: Text generated by the target model.
  - `judge_score`: 1-5 Likert score from the Judge model (or null if missing).
  - `rule_score`: 1-5 score from the rule-based metric (sentiment/coherence).
  - `execution_time_ms`: Time taken for generation.
  - `error`: (Optional) Error message if generation failed.
  - `status`: "success", "timeout" (marked as missing).

### 2.4. Calibration Result
Results of the Judge calibration step.
- **Source**: `judge_service` (FR-007).
- **Fields**:
  - `calibration_id`: Unique identifier.
  - `human_scores`: List of human annotations (static responses).
  - `judge_scores`: List of Judge predictions.
  - `kappa_coefficient`: Cohen's Kappa value.
  - `status`: "passed" (>0.6) or "failed".

## 3. Data Flow

1. **Input**: `data/raw/axes.jsonl` (Manual) + `data/raw/source_text.jsonl` (Gutenberg).
2. **Derivation**: `data/derived/probes.jsonl` (Generated).
3. **Execution**: `data/derived/results.jsonl` (Generated).
4. **Output**: `data/derived/stats_summary.json` (Aggregated).

## 4. Constraints

- **Immutability**: Raw data files are never modified. Derivations are written to new files.
- **Checksums**: Every file in `data/` is checksummed (SHA-256).
- **PII**: No personally identifiable information in `data/` (use character names only).
