# Evaluating the Impact of Code Generation on Code Smell Frequency

**Field**: computer science

## Research question

Does code generated by large language models exhibit a different frequency of common code smells compared to human-written code in equivalent software contexts?

## Motivation

LLM-based code generation is increasingly integrated into developer workflows, but there is limited empirical evidence on whether these tools systematically alter code quality patterns. Understanding if generated code introduces distinct smell profiles would inform guidelines for when and how to adopt code generation safely, balancing productivity gains against long-term maintainability costs.

## Literature gap analysis

### What we searched

We queried Semantic Scholar / arXiv / OpenAlex using search terms including "code smell LLM", "code smell frequency", "code smell large language model", and "code smell machine learning". The literature block returned 4 results total.

### What is known

- [Does Code Smell Frequency Have a Relationship with Fault-proneness? (2023)](http://arxiv.org/abs/2305.05572v1) — Establishes that code smell frequency correlates with fault-proneness in human-written code, but does not address code generation sources.
- [Evaluating the impact of code smell refactoring on the energy consumption of Android applications (2025)](http://arxiv.org/abs/2509.26031v1) — Shows code smells affect energy consumption in Android apps, but focuses on refactoring impacts rather than generation sources.
- [On the Prevalence, Evolution, and Impact of Code Smells in Simulation Modelling Software (2024)](http://arxiv.org/abs/2409.03957v1) — Documents code smell prevalence in simulation software, but does not compare LLM-generated vs. human-written code.

### What is NOT known

No published work has directly measured and compared code smell frequencies between LLM-generated and human-written code in equivalent software contexts. Existing literature establishes code smell relationships to faults, energy, and domain-specific prevalence, but leaves open the question of whether automated generation introduces distinct or exacerbated smell patterns.

### Why this gap matters

Software teams adopting LLM code generation need evidence-based guidance on maintainability trade-offs. If generated code systematically exhibits more (or different types of) code smells, this would inform integration strategies, review processes, and refactoring priorities. The answer has direct practical implications for software engineering practice.

### How this project addresses the gap

This project will systematically collect LLM-generated and human-written code samples from equivalent GitHub projects, apply standardized static analysis to measure code smell frequencies, and statistically compare the distributions. The comparison directly produces the previously-unavailable evidence on generation-source differences.

## Expected results

We expect to find either (1) no significant difference in code smell frequency between LLM-generated and human-written code, suggesting generation does not degrade maintainability, or (2) a measurable increase in specific smell types (e.g., duplicated code, long methods) in generated code. Either outcome is publishable: the former would challenge assumptions about LLM quality, while the latter would provide concrete guidance for mitigation strategies.

## Methodology sketch

- Download 50 open-source Python/Java repositories from GitHub (via `git clone`), selecting projects with ≥100 stars and ≥5 years of history to ensure mature codebases.
- For each repository, extract 20 recent commits to identify human-written code samples (approximately 1000 functions/methods total).
- Use the same repositories' issue trackers to identify 50 common programming tasks (e.g., "implement binary search", "parse JSON configuration").
- Generate code for each task using a public LLM API (e.g., HuggingFace Inference API with Gemma-2B or CodeLlama-7B), obtaining 50 LLM-generated code samples.
- Apply static analysis tools (SonarQube CLI or PMD) to all code samples, extracting code smell counts per file/function for categories: Long Method, Duplicated Code, Feature Envy, Long Parameter List.
- Compute smell frequency per 1000 lines of code for each source (human vs. LLM) and each smell category.
- Perform two-sample t-tests (or Mann-Whitney U if distributions are non-normal) to compare smell frequencies between human-written and LLM-generated code for each smell type.
- Generate effect size estimates (Cohen's d) to quantify the magnitude of any detected differences.
- Produce visualizations (box plots, violin plots) showing smell distribution comparisons across sources.
