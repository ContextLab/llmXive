## 1. Current State

**Original Requirement (FR-008):**
> "The model training must use **reaction class stratification** to ensure balanced representation of reaction types in training and test sets."

**Current Implementation Plan (Plan.md):**
> "Use **Scaffold Split** (Murcko Scaffolds) to group molecules and prevent data leakage, ensuring that the model generalizes to unseen chemical structures rather than just memorizing reaction classes."

## 2. Problem Statement

The original specification (FR-008) mandates "reaction class stratification" as the primary splitting strategy. However, recent analysis and the updated implementation plan indicate that reaction class stratification is insufficient for preventing data leakage in molecular reactivity prediction.

Reaction class stratification ensures balanced classes but does not prevent the test set from containing molecules with scaffolds present in the training set. This leads to over-optimistic performance metrics that do not reflect true generalization capability to novel chemical structures.

The implementation plan explicitly requires **Scaffold Split** (grouping by Murcko Scaffolds) to satisfy the scientific validity requirements of the project.

## 3. Proposed Change

**Replace FR-008 with the following:**

> **FR-008: Data Split Strategy**
> "The model training and evaluation must use a **Scaffold Split** strategy based on Murcko Scaffolds. Molecules sharing the same scaffold must be grouped together and assigned to the same fold (training, validation, or test) to prevent data leakage. Reaction class stratification may be used as a secondary check only if scaffold counts are sufficiently large, but scaffold integrity is the primary constraint."

**Additional Clarification:**
- The split must be performed on the **product** molecules' scaffolds.
- The split ratio must be maintained (e.g., 80/10/10 for train/val/test) at the scaffold group level.
- The implementation must verify that no scaffold appears in both training and test sets.

## 4. Justification

1. **Scientific Validity**: Scaffold splitting is the standard for assessing generalization in cheminformatics. It ensures the model learns to predict reactivity for *new* chemical structures, not just new instances of known structures.
2. **Leakage Prevention**: Reaction class stratification alone allows the same scaffold to appear in both train and test, leaking structural information.
3. **Alignment with Plan**: The implementation plan explicitly details the Scaffold Split methodology. This amendment aligns the specification with the agreed-upon implementation strategy.

## 5. Impact Analysis

- **Data Pipeline**: The `src/data/preprocess.py` module must implement the `calculate_scaffold_split` function using RDKit's MurckoScaffold generation.
- **Training Loop**: The `src/analysis/train.py` module must use the scaffold indices for K-Fold Cross-Validation.
- **Metrics**: Expected performance metrics may decrease compared to random or class-stratified splits, but this will reflect true generalization ability.
