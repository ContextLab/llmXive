# Data Model: Kairos: A Native World Model Stack for Physical AI

## Entity Relationship Overview

The data model centers on **Trajectories** generated by the **Simulation Environment** and consumed by **World Models**.

### Core Entities

1.  **Trajectory**: A sequence of time-stamped observations, actions, and rewards.
2.  **State**: A vector representing the robot's configuration (joint angles, end-effector pose) and environment (object positions).
3.  **Action**: A vector representing the control command (joint torques or end-effector velocity).
4.  **ModelCheckpoint**: Serialized weights of a trained model (Baseline or Kairos).
5.  **EvaluationResult**: Aggregated metrics for a specific test run.

## Data Schema Definitions

### 1. Trajectory Schema
A trajectory is a list of steps. Each step contains a state, action, and optional reward.

```yaml
type: object
properties:
  trajectory_id:
    type: string
    description: "Unique identifier for the trajectory (UUID)"
  source:
    type: string
    enum: ["bridge_data_v2_kitchen"]
    description: "Origin of the data (BridgeData V2 Kitchen subset)"
  steps:
    type: array
    items:
      type: object
      properties:
        t:
          type: integer
          description: "Time step index"
        state:
          type: array
          items:
            type: number
          description: "Flattened state vector (joint angles, object poses)"
        action:
          type: array
          items:
            type: number
          description: "Flattened action vector"
        reward:
          type: number
          description: "Reward signal (if available)"
        physics_error:
          type: boolean
          description: "True if a physics violation (collision, explosion) occurred at this step"
        velocity_deviation:
          type: number
          description: "Deviation in velocity from expected continuity (for real-world data)"
        state_jump:
          type: number
          description: "Magnitude of state discontinuity (teleportation check)"
        kinetic_energy_deviation:
          type: number
          description: "Percentage deviation in kinetic energy from expected conservation (for sim)"
        momentum_error:
          type: number
          description: "Magnitude of momentum conservation error (for sim)"
    minItems: 1
  metadata:
    type: object
    properties:
      seed:
        type: integer
      horizon:
        type: integer
```

### 2. Model Configuration Schema
Defines the hyperparameters and architecture type.

```yaml
type: object
properties:
  model_type:
    type: string
    enum: ["baseline_transformer", "kairos_memory"]
  hyperparameters:
    type: object
    properties:
      hidden_dim:
        type: integer
      num_layers:
        type: integer
      attention_heads:
        type: integer
      memory_scale:
        type: number
        description: "Scale parameter for the multi-scale memory module"
      sparsity:
        type: number
        description: "Sparsity constraint (0.0 to 1.0)"
  training_config:
    type: object
    properties:
      epochs:
        type: integer
      batch_size:
        type: integer
      learning_rate:
        type: number
      seed:
        type: integer
```

### 3. Evaluation Metrics Schema
Aggregated results from a test run.

```yaml
type: object
properties:
  run_id:
    type: string
  model_type:
    type: string
  seed:
    type: integer
  metrics:
    type: object
    properties:
      mse:
        type: number
        description: "Mean Squared Error of state prediction"
      success_rate:
        type: number
        description: "Percentage of tasks completed within N steps"
      pcs:
        type: number
        description: "Physics Consistency Score (0-100%, based on energy/momentum checks for sim, or velocity/state heuristics for real)"
      latency_p95_ms:
        type: number
        description: "95th percentile inference latency in ms"
      peak_ram_gb:
        type: number
        description: "Peak RAM usage in GB"
  statistics:
    type: object
    properties:
      bootstrap_ci_lower:
        type: number
        description: "Lower bound of 95% Bootstrap CI for success rate difference"
      bootstrap_ci_upper:
        type: number
        description: "Upper bound of 95% Bootstrap CI for success rate difference"
      wilcoxon_p_value:
        type: number
      effect_size:
        type: number
```

## Data Flow

1.  **Ingestion**: `data/loader.py` fetches BridgeData V2 -> `Trajectory` objects.
2.  **Preprocessing**: `data/preprocessing.py` normalizes states/actions -> `ProcessedTrajectory`.
3.  **Training**: `models/` consumes batches -> updates `ModelCheckpoint`.
4.  **Evaluation**: `eval/metrics.py` runs `ModelCheckpoint` on `TestTrajectories` -> `EvaluationResult`.
5.  **Aggregation**: `eval/statistical.py` aggregates results across seeds -> `StatisticalReport`.
