# Data Model: Predicting the Influence of Alloying on the Seebeck Coefficient Using Public Data

## Overview

This document defines the data structures used throughout the project. It ensures consistency between the ingestion, feature engineering, modeling, and reporting phases. All data flows through the `data/` directory in CSV or JSON format.

## Entity Definitions

### 1. Raw Material Record (Input)
Represents a single entry from the electronic transport database.
- **Source**: DOI 10.1038/sdata.2017.85
- **Format**: JSON/CSV (parsed by `01_ingest_and_clean.py`)

| Field | Type | Description | Nullable |
| :--- | :--- | :--- | :--- |
| `material_id` | str | Unique identifier from source | No |
| `formula` | str | Chemical formula (e.g., "Bi2Te3") | No |
| `material_family` | str | Inferred family (Bi-Te, Pb-Te, Skutterudite) | Yes (inferred) |
| `seebeck_coefficient` | float | Seebeck coefficient (µV/K) | No |
| `temperature` | float | Measurement temperature (K) | No (Required for analysis) |
| `raw_composition` | str | Raw string of elements (e.g., "Bi:2, Te:3") | No |

### 2. Processed Material Record (Intermediate)
Result of cleaning and filtering.
- **Source**: `01_ingest_and_clean.py`
- **Format**: CSV (`data/processed/cleaned_compositions.csv`)

| Field | Type | Description | Nullable |
| :--- | :--- | :--- | :--- |
| `material_id` | str | Unique identifier | No |
| `formula` | str | Normalized formula | No |
| `material_family` | str | One of: "Bi-Te", "Pb-Te", "Skutterudite" | No |
| `seebeck_coefficient` | float | Target variable | No |
| `temperature` | float | Measurement temperature (K) | No |
| `elements` | list[str] | List of constituent elements (e.g., ["Bi", "Te"]) | No |
| `stoichiometry` | list[float] | List of stoichiometric coefficients | No |

### 3. Feature Engineered Record (Model Input)
Final dataset used for training.
- **Source**: `02_engineer_features.py`
- **Format**: CSV (`data/processed/final_features.csv`)

| Field | Type | Description | Nullable |
| :--- | :--- | :--- | :--- |
| `material_id` | str | Unique identifier | No |
| `material_family_Bi-Te` | int | One-hot encoded (1 if Bi-Te, else 0) | No |
| `material_family_Pb-Te` | int | One-hot encoded (1 if Pb-Te, else 0) | No |
| `material_family_Skutterudite`| int | One-hot encoded (1 if Skutterudite, else 0) | No |
| `mean_atomic_radius` | float | Weighted mean (Å) | No |
| `electronegativity_variance` | float | Variance (Pauling scale) | No |
| `valence_electron_conc` | float | Weighted mean valence electrons | No |
| `atomic_number_variance` | float | Variance of Z | No |
| `temperature` | float | Measurement temperature (K) | No |
| `seebeck_coefficient` | float | Target variable | No |

### 4. Model Output (Results)
Results generated by the training script.
- **Source**: `03_train_and_evaluate.py`
- **Format**: JSON (`data/processed/model_metrics.json`)

| Field | Type | Description | Nullable |
| :--- | :--- | :--- | :--- |
| `r2_mean` | float | Mean R² from CV | No |
| `r2_ci_lower` | float | Lower bound of 95% CI | No |
| `r2_ci_upper` | float | Upper bound of 95% CI | No |
| `mae` | float | Mean Absolute Error | No |
| `permutation_p_value` | float | p-value from permutation test vs Null | No |
| `baseline_p_value` | float | p-value for improvement over Linear Regression | No |
| `feature_importance` | dict | Map of feature name -> importance score | No |
| `pearson_correlations` | dict | Map of feature name -> Pearson r | No |
| `vif_scores` | dict | Map of feature name -> VIF score | No |
| `classification` | str | "Success", "Inconclusive", or "Failure" | No |
| `split_strategy` | str | "80/20" or "RepeatedCV" | No |
| `n_samples` | int | Total number of samples used | No |
| `retention_rate` | float | Global retention rate (aggregated metric) | No |

## Constraints & Validation Rules

1.  **No Nulls in Features**: The `final_features.csv` must contain zero null values in the engineered descriptor columns. If a record has missing periodic data, it is dropped *before* this stage.
2.  **Stoichiometry Sum**: The sum of `stoichiometry` in `Processed Material Record` must be normalized (or consistent) to ensure weighted averages are accurate.
3.  **Family Encoding**: Only the three specified families are allowed. Any record not matching these is dropped in Phase 1.
4.  **Retention Rate**: The global `retention_rate` (in Model Output) must be >= 0.95. If not, the pipeline halts before generating this file.
5.  **R² Threshold**: The `classification` field in `Model Output` is strictly derived from `r2_mean` and `permutation_p_value`:
    -   Success: `r2_mean > 0.2` AND `permutation_p_value < 0.05`.
    -   Inconclusive: `0.2 <= r2_mean < 0.4` OR (`r2_mean > 0.2` AND `p >= 0.05`).
    -   Failure: `r2_mean < 0.2` OR `p >= 0.05`.
