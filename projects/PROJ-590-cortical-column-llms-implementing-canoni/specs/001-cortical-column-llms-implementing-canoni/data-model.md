# Data Model: Cortical Column LLMs

## 1. Overview

This document defines the data schemas for synthetic function generation, model configurations, and experiment results. All data is stored in `data/` as JSON/CSV files with checksums.

## 2. Synthetic Function Dataset

Generated by `code/src/data/benchmarks.py`.

### 2.1. Schema: `raw_function_data.json`
Represents a single sample of a time-series or surface.

```json
{
  "sample_id": "integer",
  "function_type": "string (lorenz, fourier, polynomial)",
  "input_vector": "array[float]",
  "target_vector": "array[float]",
  "metadata": {
    "seed": "integer",
    "parameters": "object (e.g., sigma, rho, beta for Lorenz)"
  }
}
```

### 2.2. Dataset Split
-   **Training Set**: Lorenz Attractor (Chaotic).
- **Validation Set**: [deferred] of Lorenz (Chaotic).
-   **Test Set**: Fourier Surfaces + Polynomial Surfaces (Independent).

## 3. Model Configuration

Stored in `data/configs/`.

### 3.1. Schema: `model_config.yaml`
Defines the architecture and hyperparameters.

```yaml
model_type: "baseline" | "microcircuit" | "ablation_recurrence" | "ablation_inhibition" | "scaling_2x" | "scaling_4x"
parameters:
  embedding_dim: integer
  num_heads: integer
  microcircuit:
    layers: ["L2/3", "L4", "L5", "L6"]
    e_i_ratio: float  # Target 4.0
    homeostasis_enabled: boolean
    ablation_flags:
      remove_recurrence: boolean
      remove_inhibition: boolean
    padding_strategy: "mask" # Explicitly state the parity strategy
training:
  epochs: integer
  learning_rate: float
  batch_size: integer
  seed: integer
```

## 4. Experiment Results

Stored in `data/results/`.

### 4.1. Schema: `experiment_run.json`
Captures the output of a single training run.

```json
{
  "run_id": "string (UUID)",
  "timestamp": "ISO8601",
  "config_hash": "string",
  "metrics": {
    "train_mae": "float",
    "val_mae": "float",
    "test_mae": "float",
    "training_time_seconds": "float",
    "peak_memory_mb": "float",
    "gradient_norm_mean": "float"
  },
  "scaling_analysis": {
    "parameter_count": "integer",
    "columns_count": "integer"
  },
  "status": "completed" | "timeout" | "oom"
}
```

## 5. Data Flow

1.  **Generation**: `benchmarks.py` creates `raw_function_data.json` (Lorenz for train/val, Poly/Fourier for test).
2.  **Checksum**: `data/manifest.json` records SHA256 hashes.
3.  **Training**: `trainer.py` loads data, trains model, outputs `experiment_run.json`.
4.  **Aggregation**: `scripts/analyze_results.py` aggregates runs to compute "Cost of Plausibility" curves and scaling trends.
