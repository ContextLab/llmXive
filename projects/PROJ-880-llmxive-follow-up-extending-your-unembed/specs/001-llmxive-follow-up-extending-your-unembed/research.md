# Research: llmXive follow-up: extending "Your UnEmbedding Matrix is Secretly a Feature Lens for Text Embeddings"

## Hypothesis & Research Questions

**Primary Hypothesis**: The "edge spectrum" subspace (top-$k$ singular vectors of $W_U$) encodes a language-agnostic "common sense" prior, implying high cosine similarity across models regardless of training language.
**Alternative Hypothesis**: The composition of the edge spectrum shifts to reflect language-specific syntactic noise, resulting in significantly lower cross-lingual cosine similarity compared to the variance observed between same-language models (initialization variance).

**Research Questions**:
1. **RQ1**: Is the geometric alignment (cosine similarity) of edge spectrum subspaces between English models (Llama-3, Mistral) and the multilingual model (BLOOM) significantly lower than the alignment between models of the same language family (Llama-3 vs. Mistral)?
2. **RQ2**: Do the tokens with the highest logit weights within the edge spectrum subspace differ semantically between English and non-English (French/Chinese) representations?
3. **RQ3**: Is the observed shift statistically significant against a null distribution generated by the 'Within-Language Baseline' (Llama-3 vs. Mistral)?
4. **RQ4**: Is the subspace orientation correlated with external typological features (WALS)?

## Methodology

### Phase 1: Edge Spectrum Extraction (FR-001)
1. **Load Models**: Load Llama-3, Mistral, and BLOOM `unembedding` ($W_U$) layers using `transformers` in `float32` (CPU).
 * *Constraint Check*: $W_U$ dimensions are typically `[vocab_size, hidden_size]`. For Llama (large vocabulary, large hidden size), this is ~2.1GB. Fits in GB RAM.
2. **SVD**: Perform `scipy.sparse.linalg.svds` on $W_U$.
 * *Target*: Extract top-$k=100$ singular vectors ($U_k$).
 * *Stability*: Handle rank-deficient matrices by checking singular values; if $\sigma_i < \epsilon$, truncate.
 * *Runtime Fallback*: If dense SVD exceeds 2 hours, reduce $k$ to 50 and log as a limitation.
3. **Output**: Save $U_k$ matrices to `data/processed/svd_results/{model}_uk.npy`.

### Phase 2: Subspace Similarity & Token Attribution (FR-002, FR-003, FR-005)
1. **Cosine Similarity**: Compute $S_{ij} = \text{cosine\_similarity}(U_{k,i}, U_{k,j})$ for all model pairs.
 * *Metric*: Mean cosine similarity of the subspace bases.
2. **Token Embedding Projection** (Replaces Pseudo-Inverse):
 * Retrieve token embeddings $E$ from the model.
 * Compute frequency distribution $f_L$ for language $L$ (from RedPajama/OSCAR).
 * Select top-$N$ most frequent tokens (e.g., $N=10,000$) to form a representative set $T_L$.
 * Project these token embeddings onto the subspace: $P_L = U_k^T \cdot E[T_L]$.
 * Compute the centroid (mean) of $P_L$ as the "mean embedding" vector $\hat{v}_L$.
 * *Rationale*: This measures how the model's *own* representation of the language corpus aligns with the anisotropy, avoiding the conflation of corpus stats with the model's basis vectors.
3. **Token Ranking**: Identify tokens with highest weights in $P_L$. Compare top lists.

### Phase 3: Statistical Validation (FR-004)
1. **Null Distribution & Baseline**:
 * Since 'same-language, different-seed' models are unavailable, we use the **Within-Language Baseline** as the empirical proxy for initialization variance.
 * Calculate similarity $S_{EN} = \text{cosine\_similarity}(U_{k, \text{Llama}}, U_{k, \text{Mistral}})$.
 * This $S_{EN}$ represents the variance due to initialization alone.
2. **Bootstrap Resampling (Sampling Stability)**:
 * To estimate the stability of the observed cross-lingual similarity $S_{Cross}$, we perform a bootstrap resampling of the *token frequency vectors*.
 * Generate $N=1000$ bootstrap samples by resampling tokens (with replacement) from the frequency distribution $f_L$.
 * For each sample, recompute the top-$N$ tokens, project them, and calculate $S_{Cross}^{(b)}$.
 * This generates a confidence interval for $S_{Cross}$.
3. **Hypothesis Test**:
 * Compare the observed $S_{Cross}$ against the observed $S_{EN}$.
 * Calculate a p-value based on the overlap of the $S_{Cross}$ confidence interval with $S_{EN}$.
 * *Note*: This tests if the cross-lingual shift is significantly larger than the typical initialization variance observed between English models, accounting for sampling noise.
4. **Threshold**: Flag "Statistically Significant" if the confidence interval of $S_{Cross}$ does not overlap with $S_{EN}$ (or p < 0.05 if a specific test statistic is defined).

### Phase 4: External Validation (SC-004)
1. **Acquire WALS**: Download WALS features for English, French, and Chinese.
2. **Correlation**: Compute the correlation coefficient between the principal component of the subspace orientation and WALS features (e.g., word order).
3. **Output**: Report correlation coefficient and p-value.

## Decision Rationale (Compute Feasibility)

* **CPU-Only SVD**: Using `svds` for $k=100$ reduces runtime to minutes. Fallback to $k=50$ ensures completion within 6 hours.
* **Bootstrap Test**: 1,000 iterations of token resampling and projection is trivial (< 10 mins).
* **Memory**: Loading one model at a time keeps peak RAM usage within manageable limits.
* **Data**: Stratified sampling of 50M tokens from RedPajama ensures long-tail stability without processing the full 1T dataset. OSCAR subsets are pre-filtered and fit within memory.
* **SVD Runtime**: If dense SVD exceeds 2 hours, reduce $k$ to 50 and log the limitation.

## Risks & Mitigations

| Risk | Impact | Mitigation |
|:--- |:--- |:--- |
| **Runtime > 6h** | High: SVD on large vocab models. | Use `svds` with $k=100$. Fallback to $k=50$ if runtime exceeds 2h. Log limitation. |
| **Dataset Mismatch** | Medium: Language purity in OSCAR. | Use `oscar` with explicit language filters. If a language is missing, report "Data Gap" and proceed with available languages. |
| **Seed-Variant Unavailability** | High: No true seed-variant models. | Use 'Within-Language Baseline' (Llama vs. Mistral) as the empirical null for initialization variance. Explicitly state this limitation in the report. |
| **Numerical Instability** | Medium: SVD on near-singular matrices. | Use `scipy.linalg.svd` with `full_matrices=False` and explicit thresholding for singular values. |
| **Collinearity** | Medium: Predictors are definitionally related. | Report descriptive statistics; do not claim independent effects. Acknowledge collinearity in the paper. |
