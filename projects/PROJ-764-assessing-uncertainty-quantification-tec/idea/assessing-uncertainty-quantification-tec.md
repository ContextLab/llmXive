# Assessing Uncertainty Quantification Techniques for Machine‑Learning Predicted Material Properties  

**Field**: materials science  

## Research question  

How accurately do lightweight uncertainty‑quantification methods—deep‑ensemble averaging, Monte‑Carlo dropout, and Gaussian‑process regression—capture predictive uncertainty for machine‑learning models of material properties (e.g., formation energy, bulk modulus, band gap) on the publicly available Materials Project dataset?  

## Motivation  

Most data‑driven materials‑property studies report only point predictions, leaving downstream screening decisions vulnerable to hidden model error. Identifying which inexpensive UQ technique provides reliable confidence estimates will enable safer high‑throughput discovery pipelines without requiring costly Bayesian deep‑learning or large ensembles.  

## Related work  

- [MT‑CGCNN: Integrating Crystal Graph Convolutional Neural Network with Multitask Learning for Material Property Prediction (2018)](https://arxiv.org/abs/1811.05660) — Provides a widely‑used baseline ML architecture for predicting diverse material properties from crystal graphs, establishing the performance floor for any UQ overlay.  
- [Uncertainty Quantification For Turbulent Flows with Machine Learning (2023)](https://arxiv.org/abs/2310.11435) — Demonstrates the application of deep ensembles and MC‑dropout to quantify uncertainty in physics‑based predictions, offering methodological precedent for adapting these techniques to materials‑property regressors.  
- [Quantifying Aleatoric and Epistemic Uncertainty in Machine Learning: Are Conditional Entropy and Mutual Information Appropriate Measures? (2022)](https://arxiv.org/abs/2209.03302) — Discusses proper metrics for separating epistemic and aleatoric components of predictive uncertainty, guiding the choice of calibration and interval‑score evaluations in this project.  

## Expected results  

- Quantitative rankings of the three UQ approaches using calibration curves, expected calibration error, and interval‑score metrics across multiple properties.  
- Evidence of whether any method consistently yields well‑calibrated uncertainties (e.g., 90 % prediction intervals covering ≈90 % of true values) while remaining computationally lightweight (training ≤30 min, UQ evaluation ≤20 min on a GitHub Actions runner).  
- Demonstration that incorporating the best‑performing UQ technique improves downstream screening decisions (e.g., higher precision in selecting thermoelectric candidates at a fixed recall).  

## Methodology sketch  

- **Data acquisition**: `wget` the Materials Project bulk download (≈5 000 inorganic compounds) from `https://materialsproject.org/open`. Extract compositional features (one‑hot element fractions) and simple structural descriptors (e.g., average atomic radius, packing fraction).  
- **Train baseline regressor**: Implement a modest feed‑forward neural network (2 hidden layers, ≤10 k parameters) using PyTorch; train on 80 % of the data, validate on 10 %, test on 10 %.  
- **Apply UQ techniques**:  
  1. **Deep ensembles** – Train 5 independently initialized copies of the baseline network; aggregate predictions to obtain mean and variance.  
  2. **Monte‑Carlo dropout** – Enable dropout (p = 0.2) at inference; draw 30 stochastic forward passes per test sample to estimate predictive variance.  
  3. **Gaussian‑process regression** – Fit a sparse GP (inducing points = 500) on the same feature set using GPyTorch.  
- **Calibration evaluation**:  
  - Compute reliability diagrams and Expected Calibration Error (ECE) for each method.  
  - Calculate the proper interval‑score (negative log‑likelihood of the predictive Gaussian) for 50 % and 90 % confidence intervals.  
  - Separate aleatoric vs. epistemic components where applicable, following the metrics discussed in the 2022 uncertainty‑quantification paper.  
- **Downstream screening case study**: Use the test‑set predictions of formation energy and band gap to rank candidates for stable perovskite structures. Compare selection precision when using point predictions only vs. when filtering by high‑confidence (narrow‑interval) predictions from each UQ method.  
- **Resource monitoring**: Record wall‑clock time and peak RAM for each step; ensure total runtime ≤5 h on a standard GitHub Actions runner (2 CPU, 7 GB RAM).
