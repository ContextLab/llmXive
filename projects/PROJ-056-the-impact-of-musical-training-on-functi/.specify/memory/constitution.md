# The Impact of Musical Training on Functional Connectivity in Adolescent Brains — Research Project Constitution

## Core Principles

### I. Reproducibility (NON-NEGOTIABLE)

Every result reported in this project MUST be reproducible by re-running the
project's `code/` against the project's `data/` on a fresh GitHub Actions
runner. Random seeds MUST be pinned in `code/`. External datasets MUST be
fetched from the same canonical source on every run.

### II. Verified Accuracy (inherits parent Principle II)

Every external citation in `idea/`, `technical-design/`,
`implementation-plan/`, or `paper/` MUST be verified by the
Reference-Validator Agent against the primary source before contributing
review points. Title-token-overlap with the cited source MUST be ≥
`CITATION_TITLE_OVERLAP_THRESHOLD` (default 0.7).

### III. Data Hygiene

Datasets MUST be checksummed and the checksum recorded under `data/`. No
data may be modified in place; every transformation MUST produce a new file
with a documented derivation. Personally identifying information MUST NOT
appear in committed data.

### IV. Single Source of Truth (inherits parent Principle I)

Every figure, statistic, or interpretation in the paper MUST trace back to
exactly one row in this project's `data/` and one block in this project's
`code/`. Derived numbers MUST NOT be hand-typed into the paper.

### V. Versioning Discipline

Every artifact under this project carries a content hash. The
Advancement-Evaluator Agent invalidates stale review records when the
hashed artifact changes. Every research-stage artifact change updates this
project's `state/projects/PROJ-056-the-impact-of-musical-training-on-functi.yaml` `updated_at` timestamp.

### VI. Neuroimaging Data Integrity

All resting‑state fMRI scans used as the predicted variable must be stored
in their raw DICOM/NIfTI form unchanged under `data/`. Preprocessing (e.g.,
motion correction, slice‑timing correction, spatial normalization) must be
performed with a documented pipeline whose scripts reside in `code/`, and
the exact parameters used for each subject must be recorded in a
metadata file. This principle follows directly from the idea’s description
that functional connectivity patterns are derived from resting‑state fMRI
imaging.

### VII. Behavioral Assessment Standardization

Predictor variables (musical training history, years of training, musical
aptitude) derived from behavioral assessments and self‑report data must be
collected using validated questionnaires or structured interview protocols.
Raw questionnaire responses must be saved under `data/` with timestamps and
instrument identifiers, and any derived scores must be generated by scripts
in `code/`. This reflects the idea’s specification that predictors come from
behavioral assessments and self‑report data.

1. On every artifact write that introduces or modifies citations.

# Idea summary (for context)

## Research-question validation

### Phenomenon-vs-method check

**Verdict**: pass

The question asks about a substantive brain-behavior relationship: whether musical training during adolescence affects functional connectivity patterns in specific neural networks. This is independent of any specific ML method or computational constraint—the question is about neuroplasticity mechanisms, not method performance.

### Circularity check

**Verdict**: pass

The predictor (musical training history, years of training, musical aptitude) comes from behavioral assessments and self-report data. The predicted variable (functional connectivity patterns) comes from resting-state fMRI imaging. These are independent measurement modalities with no shared primary signal, so the predictive relationship is empirically testable rather than mechanically guaranteed.

### Triviality check

**Verdict**: pass

Both positive and null results would be informative. A positive finding would support neuroplasticity from musical training in typically developing adolescents, while a null result would suggest either that functional effects are limited to clinical populations (as in the autism literature) or that structural rather than functional changes predominate. The adolescent-specific focus and network-level analysis represent a genuine gap in the literature.

### Question-narrowing check

**Verdict**: pass

The question names a domain relationship (musical training → functional connectivity in auditory, motor, and executive control networks during adolescence) rather than implementation constraints. It does not fixate on specific algorithms, hardware budgets, or runtime limits as the core question.

### Overall verdict

**Verdict**: validated

All four checks pass without significant concerns. The research question targets a substantive brain-behavior relationship with independent data sources, non-trivial outcomes, and clear domain focus. The project can proceed to initialization without requiring reframing.
