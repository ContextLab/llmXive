# Data Model: Evaluating the Impact of Code Complexity on LLM Code Understanding

## 1. Overview

This document defines the data structures for the project. All data flows through `data/` (raw/processed) and `results/` (logs/reports).

## 2. Entity Relationship Diagram (Text)

- **CodeSnippet**: The fundamental unit. Contains source code and computed complexity metrics.
- **TaskInstance**: A specific execution of the Summarization task on a CodeSnippet.
- **InferenceResult**: The output of the LLM for a TaskInstance.
- **AnalysisReport**: Aggregated statistics derived from InferenceResults.

## 3. Schemas

### CodeSnippet (Processed)
Stored in: `data/processed/annotated_snippets.csv`
*Note: Metadata regarding `radon` version and flags is stored in `data/processed/metadata.json` to satisfy Constitution Principle VI.*

| Field | Type | Description |
| :--- | :--- | :--- |
| `snippet_id` | String | Unique identifier (SHA256 of code). |
| `source_code` | String | Raw Python code. |
| `func_name` | String | Function name (if extractable). |
| `docstring` | String | Ground truth summary. |
| `complexity_tertile` | String | "Low", "Medium", or "High" (based on stratified sampling). |
| `cyclomatic_complexity` | Float | Computed by `radon`. |
| `halstead_volume` | Float | Computed by `radon`. |
| `maintainability_index` | Float | Computed by `radon`. |
| `is_valid` | Boolean | True if `radon` parsed successfully. |

### InferenceResult
Stored in: `results/inference_logs.jsonl`

| Field | Type | Description |
| :--- | :--- | :--- |
| `snippet_id` | String | FK to CodeSnippet. |
| `task_type` | String | "summarization". |
| `prompt` | String | The exact prompt sent to the model. |
| `model_output` | String | Raw text generated by the model. |
| `reasoning` | String | Extracted reasoning (if any). |
| `ground_truth` | String | Expected output (docstring). |
| `performance_score_bleu` | Float | BLEU-4 score. |
| `performance_score_rouge` | Float | ROUGE-L score. |
| `execution_status` | String | "success", "failed", "timeout", "oom". |

### AnalysisReport
Stored in: `results/analysis_metrics.csv`

| Field | Type | Description |
| :--- | :--- | :--- |
| `metric_name` | String | e.g., "Spearman_Cyclomatic_Summarization". |
| `correlation_coefficient` | Float | $\rho$ value. |
| `p_value` | Float | Significance value. |
| `adjusted_p_value` | Float | BH-corrected p-value. |
| `sample_size` | Integer | N used for this test. |
| `vif_status` | String | "Pass" or "PCA_Fallback". |

## 4. Data Flow

1.  **Acquisition**: `data/raw/*.parquet` -> `01_data_acquisition.py` -> `data/raw/codesearchnet_raw.parquet`.
2.  **Annotation**: `data/raw/codesearchnet_raw.parquet` -> `02_complexity_annotation.py` -> `data/processed/annotated_snippets.csv` + `data/processed/metadata.json`.
3.  **Inference**: `data/processed/annotated_snippets.csv` -> `03_inference.py` -> `results/inference_logs.jsonl`.
4.  **Analysis**: `results/inference_logs.jsonl` -> `04_analysis.py` -> `results/analysis_metrics.csv`.
5.  **Validation**: `results/analysis_metrics.csv` is validated against `contracts/analysis_schema.yaml`.
