# Research: Evaluating the Impact of LLM-Generated Code Explanations on Comprehension

## Dataset Strategy

The study relies on the **CodeSearchNet** corpus for Python functions. The dataset contains source code, natural language summaries (docstrings), and metadata.

| Dataset Name | Purpose | Source URL | Load Method |
| :--- | :--- | :--- | :--- |
| **CodeSearchNet (Python)** | Source of code snippets and official docstrings. | https://huggingface.co/datasets/codeparrot/codesearchnet | `datasets.load_dataset("codeparrot/codesearchnet", split="train")` |
| **CodeLlama (Reference)** | Reference for model capabilities. Model weights loaded from official Meta repo. | https://huggingface.co/meta-llama/CodeLlama-7b-Instruct-hf | `transformers.AutoModelForCausalLM.from_pretrained("meta-llama/CodeLlama-7b-Instruct-hf")` |

## Methodology & Statistical Rigor

### Experimental Design
- **Independent Variables**: 
  1. *Condition* (3 levels: Code Only, Code+LLM, Code+Docstring).
  2. *Complexity* (3 levels: Low, Medium, High).
- **Dependent Variables**: 
  1. *Accuracy* (Binary: Correct/Incorrect).
  2. *Latency* (Continuous: ms).
- **Design**: 3x3 Mixed Design. *Condition* is a between-subjects factor (randomized). *Complexity* is a within-subjects factor (each participant sees snippets of varying complexity).

### Statistical Analysis Plan
1.  **Preprocessing**: 
    - Exclude participants with total time < 30s or missing data > 80% (FR-004).
    - Handle missing explanations (skipped snippets) by excluding specific rows, not participants.
2.  **Primary Model**: 
    - **Accuracy**: Generalized Linear Mixed Model (GLMM) with **Binomial** family and logit link (addressing methodology-ad7aa92c).
    - **Latency**: Linear Mixed Model (LMM) with Gaussian family.
    - **Fixed Effects**: Condition, Complexity, Condition × Complexity interaction.
    - **Random Effects**: Random intercepts for **ParticipantID** AND **SnippetID** (addressing methodology-ff682b01 and methodology-8f553867). This accounts for the non-independence of repeated measures per participant and controls for the specific 30 snippets chosen (confounding of Snippet and Complexity).
    - **Note**: The spec (FR-005) requests "random intercepts for participants only". The plan overrides this to include **SnippetID** for statistical validity. This is a **spec-root cause** conflict; the Spec must be updated to match the rigorous methodology.
3.  **Post-Hoc**: 
    - Tukey’s HSD for pairwise comparisons of Condition levels (using `emmeans` or `statsmodels` equivalent).
    - **Multiplicity Correction**: Applied automatically by Tukey method to control Family-Wise Error Rate (FWER) (SC-005).
4.  **Sensitivity Analysis**: 
    - Sweep BLEU threshold across a range of values.
    - Re-run GLMM/LMM on subsets of data where generated explanation BLEU score ≥ threshold.
    - Report variance in F-statistics and p-values (SC-002).
    - **Limitation Acknowledgement**: The plan explicitly acknowledges that BLEU measures fidelity to the docstring (the control condition). High BLEU implies similarity to the control, creating a tautological risk (scientific_soundness-82faf682). The analysis is framed as testing stability relative to this metric, not independent quality.

### Power & Sample Size
- **Power Limitation**: With N=30 snippets (the unit of analysis for the within-subjects factor), the study has limited power to detect small interaction effects (methodology-d0a9d38b). The plan frames the interaction test as exploratory and emphasizes effect size estimation.
- **Participant Power**: A sample size sufficient for detecting main effects but potentially limited for the interaction term given the low snippet count.

### Assumptions & Limitations
- **Causal Inference**: Complexity is an observed covariate, not manipulated. All claims regarding complexity are **associational**, not causal.
- **Model Feasibility**: CodeLlama on CPU is borderline. The plan includes a fallback to a smaller variant of TinyLlama if OOM occurs.
- **BLEU Validity**: BLEU measures fidelity to the docstring, not intrinsic quality. This limitation is explicitly reported (FR-009).

## Computational Feasibility

- **Inference**: CodeLlama-7B (4-bit) on CPU with `transformers` (default precision if 4-bit fails) or TinyLlama fallback. Batch size = 1.
- **Memory**: Aggressive garbage collection (`gc.collect()`) between snippets.
- **Time**: Inference for 30 snippets is estimated at ~1-2 hours (CodeLlama) or <30 mins (TinyLlama). Analysis is negligible. Total time within a practical operational window.
- **Constraints**: No GPU. No training.

## Data Collection Validity (SC-003)
- The system will calculate the **participant pass rate** (percentage of recruited participants passing quality filters: >30s time, <80% missing).
- A check will be performed: if pass rate < 90%, a warning will be logged in the final report, and the validity of the data collection will be flagged.
