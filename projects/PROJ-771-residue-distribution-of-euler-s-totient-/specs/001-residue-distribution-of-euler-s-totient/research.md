# Research: Residue Distribution of Euler's Totient Function Modulo Small Primes

## Problem Definition

The research investigates the **rate of convergence** and **deviation magnitude** of the residues of Euler's totient function $\phi(n)$ modulo small primes $p \in \{3, 5, 7, 11\}$. Specifically, we test whether the observed deviations from the theoretical asymptotic distribution match the predicted error bounds $O(N^{1-\delta})$ described in Lebowitz-Lockard et al. (2021) and Pollack & Roy (2024).

While analytic number theory suggests that $\phi(n) \pmod p$ becomes uniform as $N \to \infty$ (a result by Erdős, Pomerance, etc.), this project does **not** treat uniformity as an open hypothesis to be "validated" (which would be tautological). Instead, the analysis focuses on:
1.  Calculating the **theoretical expected count** $E_k^{theo}$ for finite $N$ using the product formula with the specific error term correction.
2.  Measuring the **residual deviation** $|O_k - E_k^{theo}|$.
3.  Testing if this residual is consistent with the predicted error bound $O(N^{1-\delta})$ using a **Block Bootstrap** procedure to account for the dependence structure of the $\phi(n)$ sequence.

The null hypothesis is: **The observed deviation magnitude is consistent with the theoretical error bound.**

## Methodological Approach

### 1. Data Generation (Linear Sieve)
To compute $\phi(n)$ for $n \in [1, N]$ efficiently ($O(N)$), a **Linear Sieve** (also known as the Euler Sieve) is employed.
- **Algorithm**: Maintain an array `phi` where `phi[1] = 1`. Iterate $i$ from 2 to $N$. If `phi[i]` is unset, $i$ is prime; set `phi[i] = i-1`. For each prime $p \le \text{smallest prime factor of } i$, update `phi[i*p]`.
- **Arithmetic Integrity**: Python's native `int` type handles arbitrary precision automatically, satisfying Constitution Principle VI. No overflow checks are needed as Python integers grow as required.
- **Modulo Reduction**: Immediately after computing $\phi(n)$, compute $r = \phi(n) \pmod p$ and increment the count for bin $r$. This avoids storing the full $\phi(n)$ array, reducing memory footprint to $O(N)$ for the sieve state and $O(p)$ for counts.

### 2. Statistical Testing: Deviation vs. Error Bounds
The primary hypothesis is **not** simple uniformity, but rather:
$H_0$: The observed deviation magnitude $|O_k - E_k^{theo}|$ is consistent with the theoretical error bound $C \cdot N^{1-\delta}$.

- **Theoretical Expectation ($E_k^{theo}$)**:
  Instead of $N/p$, we calculate $E_k^{theo}$ using the product formula derived from the distribution of prime factors, adjusted by the error term from Lebowitz-Lockard (2021):
  $$ E_k^{theo} = \frac{N}{p} \left( 1 + \epsilon_k(N) \right) $$
  where $\epsilon_k(N)$ represents the specific bias predicted for residue $k$ at scale $N$. If specific $\epsilon_k$ formulas are not available for all $k$, we use the asymptotic limit $N/p$ but explicitly test the residual against the global error bound $O(N^{1-\delta})$.

- **Handling Dependence (Block Bootstrap)**:
  Since $\phi(n)$ values are not independent (they depend on the prime factorization of $n$), standard $\chi^2$ p-values are invalid.
  - **Procedure**:
    1.  Divide the sequence of $N$ residues into $B$ contiguous blocks of size $L$.
    2.  Resample these blocks with replacement to create $M$ synthetic datasets (e.g., $M=1000$).
    3.  Compute the test statistic (e.g., $\chi^2$-like sum of squared residuals) for each synthetic dataset.
    4.  The empirical distribution of these statistics forms the null distribution.
    5.  The p-value is the proportion of synthetic statistics exceeding the observed statistic.
  - **Fallback for Small $N$**: If $N$ is too small to form valid blocks (e.g., $N < 100$), the system will issue a warning and report the raw deviation magnitude without a p-value, or use a permutation test on the available data if feasible.
  - **No Circular Simulation**: We do not simulate uniform data. We resample the *observed* data to estimate the null distribution of the statistic under the assumption that the observed structure is representative of the error process.

- **Multiple Testing**: Since 4 primes are tested, a Bonferroni correction ($\alpha_{adj} = 0.05/4 = 0.0125$) will be noted in the report, though the primary analysis uses $\alpha=0.05$ as per the spec.

### 3. Visualization & Theoretical Comparison
- **Bar Plots**: Observed frequencies vs. $E_k^{theo}$ (theoretical expectation with error bounds).
- **Residual Plots**: Plot of $|O_k - E_k^{theo}|$ against the predicted error bound $C \cdot N^{1-\delta}$ for each residue class.
- **QQ-Plots**: Quantile-Quantile plots of the Block Bootstrap test statistics to assess the fit of the null distribution.

**Target Environment**: GitHub Actions Free Tier (2 CPU, ~7 GB RAM, 6h limit).
- **Memory**: The linear sieve requires storing an array of size $N$ for the `phi` values and `is_prime` flags. For $N=5,000,000$, this is approximately $5 \times 10^6 \times 8$ bytes $\approx 40$ MB (using 64-bit integers). Even with Python's object overhead, this fits comfortably within 7 GB.
- **Time**: The sieve is $O(N)$. With $N=5,000,000$, Python operations should complete in < 10 minutes. The Block Bootstrap (1000 iterations) is $O(M \cdot B)$, which is negligible compared to the sieve.
- **Constraints**: No GPU required. No heavy ML models. Pure CPU arithmetic.

## Decision Rationale

1.  **Why Linear Sieve?** Standard trial division is $O(N \sqrt{N})$, which is too slow for $N=5 \times 10^6$. The Linear Sieve is $O(N)$, ensuring the job completes within the 6-hour CI limit.
2.  **Why Python?** Python's native arbitrary-precision integers simplify the implementation of Constitution Principle VI (no need for external BigInt libraries like GMP). `numpy` and `scipy` provide optimized array operations and statistical functions.
3.  **Why Block Bootstrap?** The standard $\chi^2$ test assumes independence, which is violated by the deterministic, factor-dependent nature of $\phi(n)$. The Block Bootstrap preserves the local dependence structure, providing a valid null distribution for the test statistic.
4.  **Why Deviation vs. Error Bounds?** Testing for "uniformity" is trivial for a proven theorem. The scientific value lies in verifying the *rate* of convergence and the *validity* of the theoretical error bounds for finite $N$.
