# Data Model: Predictive Modeling of Host Immune Response

## Entities

### 1. ViralGenome
Represents a single viral genome sequence.
- **Attributes**:
  - `accession_id`: string (NCBI accession)
  - `virus_strain`: string (Strain name)
  - `family`: string (Viral family)
  - `fasta_path`: string (Relative path to raw FASTA)
  - `features`: dict (CAI, GC, k-mers, stability, repeat_density)

### 2. HostExpressionSample
Represents a host transcriptomic sample.
- **Attributes**:
  - `sample_id`: string (GEO sample ID)
  - `virus_strain_accession`: string (Link to virus)
  - `host_species`: string (e.g., "Homo sapiens")
  - `raw_counts_path`: string (Path to count matrix)
  - `isg_pc1`: float (Computed interferon score)
  - `status`: enum (valid, excluded, unknown)

### 3. FeatureMatrix
The merged dataset for modeling.
- **Attributes**:
  - `strain_id`: string (Primary key for modeling unit)
  - `target`: float (Aggregated ISG-PC1)
  - `features`: array (Vector of all extracted features)
  - `metadata`: dict (Strain info)

## Data Flow

1.  **Raw Ingestion**:
    - `data/raw/ncbi_viruses.fasta` (or local)
    - `data/raw/geo_counts.tsv` (or local)
2.  **Preprocessing**:
    - `data/processed/normalized_counts.tsv` (TMM normalized via edgeR)
    - `data/processed/isg_scores.csv` (ISG-PC1 per sample)
    - `data/processed/sample_variance.csv` (Preserved sample-level variance)
3.  **Feature Extraction**:
    - `data/processed/viral_features.csv` (CAI, GC, k-mers, repeat_density, stability)
4.  **Merging**:
    - `data/processed/merged_dataset.csv` (Join on `virus_strain`)
    - `data/processed/aggregated_strain_data.csv` (One row per strain)
5.  **Model Output**:
    - `artifacts/model.pkl`
    - `artifacts/metrics.json`
    - `artifacts/plots/feature_importance.png`
6.  **Manifest**:
    - `data/manifest.yaml` (Generated by download.py)

## Assumptions & Constraints

- **Strain Aggregation**: Multiple samples per strain are averaged.
- **Missing Data**: Samples with missing `virus_strain_accession` are excluded.
- **Species**: Only Human/Mouse supported for standard ISG sets; others require ortholog mapping.
- **Real Data**: The pipeline requires real data. No simulation mode.
