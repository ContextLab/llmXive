# Implementation Plan: Predictive Modeling of Host Immune Response from Viral Sequence Features

**Branch**: `001-predict-immune-response` | **Date**: 2026-06-24 | **Spec**: `spec.md`
**Input**: Feature specification from `/specs/001-predict-immune-response/spec.md`

## Summary

This project implements a computational pipeline to investigate whether viral sequence features (codon usage, GC-content, k-mers, protein stability) can predict host immune response scores (Interferon-Response PC1) derived from transcriptomic data. The system downloads viral genomes from NCBI, processes GEO transcriptomic data, extracts features, trains an Elastic Net regression model with rigorous statistical validation (permutation tests, Stability Selection, VIF checks), and generates interpretability visualizations.

**Critical Note on Data**: The pipeline requires real paired viral genome and host transcriptomic data. If the specific GEO accessions and NCBI Virus search terms are not provided, the pipeline will abort with a fatal error. No synthetic data generation is permitted for the primary scientific validation.

## Methodological Adjustments

1.  **Sample Size & Power**: The plan acknowledges that N ~30-50 is underpowered for high-dimensional regression (P >> N). To mitigate, **Stability Selection** is used as the primary feature selection method to reduce dimensionality before inference. The pipeline will calculate effective degrees of freedom and abort if the effective N is insufficient for the target R².
2.  **Feature Selection**: Debiased Lasso is replaced by **Stability Selection** for primary inference (FR-012) due to N << P. Debiased Lasso is retained only as an exploratory check if N > 200.
3.  **Negative Control**: A negative control (shuffling viral sequences) is added to the permutation test to distinguish true signal from spurious correlations.
4.  **Host Covariates**: Host codon usage bias is included as a covariate to control for tautological correlations.
5.  **Strain-Level Splitting**: The split is performed by shuffling unique strain IDs, ensuring the test set has a representative number of strains sufficient for evaluation (or more if total allows). The pipeline aborts if total strains < 10.
6.  **ESM-1b Fallback**: If ESM-1b fails, the pipeline attempts a CPU mode or a proxy (AAIndex) ONLY if `--proxy_mode` is set. Otherwise, it aborts.
7.  **TMM Normalization**: Implemented via `rpy2` calling `edgeR::calcNormFactors`.
8.  **Repeat Density**: Implemented via `pybedtools` calling `bedtools` or `RepeatMasker`.
9.  **Ortholog Mapping**: Implemented via Ensembl Compara v109 API (via `pyensembl` or similar).
10. **Manifest Generation**: `download.py` explicitly writes `data/manifest.yaml` with NCBI accessions and release version.
