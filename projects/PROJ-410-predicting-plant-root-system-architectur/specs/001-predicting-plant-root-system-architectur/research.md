# Research: Predicting Plant Root System Architecture from Genomic Data

## Research Strategy

This research aims to determine if genomic variation in *Arabidopsis thaliana* can predict root system architecture (RSA) traits (length, branching, angle) across varying nutrient conditions. The study is observational; all claims will be framed as associational.

The spec requires data from the **1001 Genomes Project** (Genomic) and **ATRDB/PhenomicsDB** (Phenotypic).

### Statistical Methodology

1.  **Data Harmonization**:
    *   Match by *Accession ID*.
    *   Encode SNPs as allele counts (0, 1, 2).
    *   **Missing Data**: Exclude markers with >5% missingness. Exclude accessions with missing phenotypes. **No imputation** in the primary pipeline (FR-002).

2.  **Modeling**:
    *   **Stratification**: Separate models for each nutrient condition (e.g., Low N, High N, Control).
    *   **Algorithms**:
        *   **Linear Regression**: Use **L1 (Lasso) or ElasticNet** regularization. *Rationale*: PCA was rejected for linear models because it transforms features into orthogonal components, destroying the biological interpretability of specific genomic markers (US-3). Lasso performs feature selection while preserving marker identity.
        *   **Random Forest (RF)**: For non-linear capture.
        *   **Gradient Boosting (XGBoost/LightGBM CPU version)**: For performance.
    *   **Dimensionality Reduction**:
        *   For **Linear Models**: Use Lasso (no PCA).
        *   For **Tree Models**: If feature count > 5000, PCA may be applied *only* for computational speed, with the explicit caveat that **marker-level interpretability is forfeited** and only component importance is reported.

3.  **Evaluation**:
    *   **Metrics**: R², Mean Absolute Error (MAE).
    *   **Baseline**: Compare against a null model (intercept only).
    *   **Significance**: Permutation test to generate a null distribution of R².
        *   **Null Hypothesis**: The observed R² is compared against the distribution of R² values generated by shuffling the target variable *within the same modeling pipeline*.
        *   **Data Leakage Prevention**: Feature selection (variance filtering) and dimensionality reduction MUST be performed **inside** the cross-validation and permutation loops. Performing these steps on the full dataset before splitting leads to circularly optimistic p-values.
        *   **P-value Calculation**: Proportion of permuted R² values >= observed R². Handle negative R² appropriately (test against R²=0 or distribution).
    *   **Correction**: Apply Benjamini-Hochberg (FDR) correction **only** to the set of *model-level* hypothesis tests (one per nutrient condition). Applying correction to ~200k markers directly is methodologically unsound for tree-based models without a stability selection framework.

4.  **Feature Importance**:
    *   Permutation importance for tree-based models.
    *   SHAP values (KernelSHAP for CPU efficiency) for interpretability.
    *   **Stability Selection**: Perform **100 bootstrap samples** (not 10) to rank marker stability. This ensures the stability metric has low variance and is statistically robust.
    *   **Multiple Testing**: Stability selection serves as the robustness check for marker significance, avoiding the need for direct p-value correction on thousands of markers in a p>>n setting.

### Decision Rationale & Feasibility

*   **CPU-Only Constraint**: The plan avoids deep learning (CNNs/RNNs) and GPU-dependent libraries (PyTorch with CUDA). `scikit-learn` and `xgboost` (CPU build) are selected for their efficiency on 2 vCPU.
*   **Memory Management**: The pipeline will load data in chunks. If the full SNP matrix exceeds available memory capacity, the plan triggers a PCA reduction step (for tree models) or relies on Lasso sparsity (for linear models).
*   **Statistical Rigor**:
    *   *Multiple Comparisons*: FDR applied to model-level tests. Stability selection used for marker-level robustness.
    *   *Power*: Acknowledged limitation: Sample size (a moderate number of accessions) may limit power for detecting small effect sizes in high-dimensional data. The permutation test provides a robust significance estimate without power assumptions.
    *   *Causality*: Explicit disclaimer added to all outputs (FR-009).
    *   *Collinearity*: If SNPs are in linkage disequilibrium (LD), independent effects cannot be claimed. SHAP values will be interpreted as "predictive contribution" rather than "causal effect."
    *   *Data Validity*: The plan explicitly acknowledges that if real data is unavailable, the biological conclusions are invalid. The pipeline validation on mock data proves *methodological* soundness, not *biological* truth.

### Dataset-Variable Fit Analysis

*   **Required Variables**: Genotype (SNP), Phenotype (Root Length, Branching), Environment (Nutrient Condition).
*   **Gap Analysis**: The provided `# Verified datasets` block **does not verify** the existence of a dataset containing *both* 1001 Genomes SNPs and specific RSA phenotypes.
*   **Risk**: If the real data is unavailable, the pipeline will fail to produce biological results.
*   **Mitigation**: The implementation will include a "Mock Data Generator" that creates synthetic data with the correct schema to prove the *pipeline* works, while the report will explicitly state that the *biological* results are pending real data ingestion. This satisfies the "Compute Feasibility" requirement (CI passes) while acknowledging the "Dataset-Variable Fit" gap.
