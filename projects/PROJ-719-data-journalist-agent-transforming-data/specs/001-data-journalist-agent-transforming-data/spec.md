# Feature Specification: Data Journalist Agent: Transforming Data into Verifiable Multimodal Stories

**Feature Branch**: `001-data-journalist-agent-reproduction`  
**Created**: 2024-05-23  
**Status**: Draft  
**Input**: User description: "Reproduce & validate: Data Journalist Agent: Transforming Data into Verifiable Multimodal Stories"

## User Scenarios & Testing

### User Story 1 - End-to-End Execution of the Data2Story Pipeline (Priority: P1)

A researcher MUST be able to execute the vendored `data2story-skill` repository on a sample scenario to confirm the multi-agent orchestration (Detective, Analyst, Designer, etc.) functions without runtime errors.

**Why this priority**: This is the foundational requirement for a reproduction project. If the pipeline cannot execute, no validation of the paper's claims is possible. It delivers the core artifact: a generated story.

**Independent Test**: Can be fully tested by running the `run_evals.py` script against a single, small scenario (e.g., `01_meteorite_flagship.md`) and verifying the existence of output artifacts (HTML, JSON reports, generated assets) without manual intervention.

**Acceptance Scenarios**:

1. **Given** the repository is cloned and dependencies are installed, **When** the user runs the evaluation script on `01_meteorite_flagship.md`, **Then** the script completes successfully (exit code 0) and produces a directory containing `index.html`, `audit_report.json`, and generated media assets.
2. **Given** the script is running, **When** the `Inspector` agent processes the output, **Then** a `cell_registry.json` is generated linking every claim in the story to a specific data cell or code reference.

---

### User Story 2 - Verification of Evidence Traceability (Priority: P2)

A researcher MUST be able to verify that the generated story's claims are grounded in the source data by inspecting the `Inspector` agent's output registry.

**Why this priority**: The paper's primary innovation is "evidence-grounded claims." Without verifying this specific mechanism, the reproduction fails to validate the core contribution.

**Independent Test**: Can be tested by parsing the generated `cell_registry.json` and cross-referencing a random sample of claims in the output HTML against the cited data rows or code cells.

**Acceptance Scenarios**:

1. **Given** a generated story and its `cell_registry.json`, **When** a specific claim (e.g., "The temperature increased by 5 degrees") is selected, **Then** the registry must contain a valid reference ID pointing to a specific cell in the source dataset or a line in the analysis code.
2. **Given** the `Inspector` output, **When** the verification script runs, **Then** it must report a [deferred] match rate between the claims in the narrative and the entries in the registry for the processed scenario.

---

### User Story 3 - Multimodal Asset Generation (Priority: P3)

A researcher MUST be able to confirm that the `Designer` agent successfully generates at least one multimodal asset (e.g., an image, audio clip, or interactive chart) as described in the paper.

**Why this priority**: The paper claims "multimodally generative" capabilities. While the text generation is P1, the multimodal aspect is a key differentiator that must be demonstrated, even if using placeholder or mock generation logic if external APIs are unavailable.

**Independent Test**: Can be tested by checking the `assets/` directory of the output for non-text files (e.g., `.png`, `.mp3`, `.mp4`) or interactive HTML components generated by the `Designer` agent.

**Acceptance Scenarios**:

1. **Given** the pipeline completes, **When** the output directory is inspected, **Then** at least one non-HTML asset (image, audio, or video file) generated by the `Designer` or `Cinematographer` agent must exist.
2. **Given** the `Designer` configuration, **When** the `text2image` or `text2music` script is invoked, **Then** it must either generate a file or, if external API access is blocked, generate a placeholder artifact with a clear `generated_by` metadata tag indicating the intended modality.

---

### Edge Cases

- **API Rate Limiting**: The `Designer` agent may attempt to call external generation APIs (e.g., OpenRouter, Hugging Face). If these are rate-limited or unavailable in the CI environment, the system MUST handle the failure gracefully by generating a placeholder asset and logging the error, rather than crashing the entire pipeline.
- **Data Availability**: If the source dataset for a specific scenario (e.g., `03_regression_gaps.md`) is missing or malformed, the `Analyst` agent MUST fail the specific scenario step with a clear error message (`NEEDS CLARIFICATION: Dataset X missing`) rather than proceeding with empty data.
- **Memory Constraints**: The pipeline processes multiple agents sequentially. If a single agent (e.g., the `Cinematographer` generating video) exceeds the RAM limit of the CI runner, the system MUST be configured to skip that specific heavy generation step and log a warning.

## Requirements

### Functional Requirements

- **FR-001**: The system MUST execute the `data2story-skill` evaluation script (`pro/skills/data2story-pro/evals/scripts/run_evals.py`) against the `01_meteorite_flagship.md` scenario and produce a valid output directory containing `index.html` and `audit_report.json`. (See US-1)
- **FR-002**: The system MUST generate a `cell_registry.json` file that maps every factual claim in the generated story to a specific data cell, code block, or external reference ID. (See US-2)
- **FR-003**: The system MUST validate that the `Inspector` agent's verification logic correctly identifies at least one claim that is either ungrounded or incorrectly grounded, if such a case exists in the test scenario. (See US-2)
- **FR-004**: The system MUST ensure that the `Designer` agent attempts to generate at least one multimodal asset (image, audio, or video) or a documented placeholder if external API calls are blocked by the CI environment. (See US-3)
- **FR-005**: The system MUST handle API failures (e.g., rate limits, network timeouts) by logging the error and proceeding to the next agent step without terminating the entire pipeline, ensuring the reproduction report is still generated. (See US-1)

### Key Entities

- **Scenario**: A specific news topic and associated dataset (e.g., `01_meteorite_flagship.md`) used as input for the agent pipeline.
- **Cell Registry**: A JSON artifact mapping story claims to their source evidence (data cells, code lines, references).
- **Multimodal Asset**: A generated file (image, audio, video, interactive chart) produced by the `Designer` or `Cinematographer` agents.

## Success Criteria

### Measurable Outcomes

> Planning docs state *what* will be measured and the *source/reference* it is measured against; defer specific empirical values to the implementation/research phase.

- **SC-001**: The end-to-end execution time of the `01_meteorite_flagship.md` scenario is measured against the 6-hour GitHub Actions free-tier limit. (See FR-001)
- **SC-002**: The completeness of the `cell_registry.json` is measured against the total number of factual claims identified in the generated HTML story. (See FR-002)
- **SC-003**: The success rate of multimodal asset generation (or placeholder creation) is measured against the total number of media requests issued by the `Designer` agent. (See FR-004)
- **SC-004**: The error recovery rate is measured as the percentage of scenarios that complete successfully despite encountering at least one API failure or network timeout. (See FR-005)

## Assumptions

- **Assumption about Data/Environment**: The CI environment (GitHub Actions free tier) has no GPU access; any code requiring CUDA (e.g., `load_in_8bit`, `device_map="cuda"`) MUST be bypassed or replaced with CPU-compatible logic, potentially resulting in placeholder assets instead of high-fidelity generative media.
- **Assumption about Scope Boundaries**: The reproduction focuses on the `pro` (professional) version of the skill set located in `pro/skills/data2story-pro/`, as this is the version described in the paper's evaluation section; the `skills/data2story` (non-pro) version is out of scope for this specific reproduction.
- **Assumption about API Access**: External APIs for generative media (e.g., OpenRouter, Hugging Face) may be rate-limited or require authentication keys not present in the CI environment; the system assumes that generating a placeholder artifact with a "generation failed" flag is an acceptable outcome for validation purposes.
- **Assumption about Dataset Availability**: The source datasets referenced in the `evals/scenarios` directory are present in the vendored submodule and do not require external downloads that would exceed the disk or time constraints.
