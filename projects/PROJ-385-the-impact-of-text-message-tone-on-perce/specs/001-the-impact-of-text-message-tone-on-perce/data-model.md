# Data Model: The Impact of Text Message Tone on Perceived Emotional Support

## Overview

This document defines the core data entities, their attributes, and relationships required for the study on text message tone and perceived emotional support. The model supports the generation of controlled stimuli, collection of human ratings, and subsequent statistical analysis using Linear Mixed-Effects Models (LMM).

## Entity Definitions

### 1. Stimulus

Represents a unique text message variant generated for the experiment. Stimuli are constructed using a factorial design manipulating emoji usage, punctuation, length, and context.

**Attributes:**
- `stimulus_id` (string): Unique identifier (e.g., `S001`, `S024`). Format: `S` followed by 3-digit zero-padded integer.
- `text_content` (string): The actual text of the message.
- `emoji_level` (integer): Factor level for emoji usage (0: None, 1: Low, 2: High).
- `punctuation_type` (string): Factor level for punctuation (`period`, `exclamation`).
- `length_category` (string): Factor level for message length (`short`, `long`).
- `context` (string): Relationship context (`friend`, `acquaintance`).
- `cue_intensity` (float): Calculated composite score of non-verbal cues (optional, used for sensitivity analysis).

**Constraints:**
- `stimulus_id` must be unique.
- `text_content` must not be empty.
- Total number of unique stimuli = 24 (3 emoji × 2 punctuation × 2 length × 2 contexts).

**Source:** Generated by `code/01_generate_stimuli.py`.
**Storage:** `data/raw/stimuli.csv`.

### 2. Participant

Represents a human subject (simulated for this study) who provides ratings.

**Attributes:**
- `participant_id` (string): Unique identifier. Format: `P-[A-Z0-9]{8}` (e.g., `P-A1B2C3D4`).
- `relationship_context` (string): The context assigned to this participant for the session (`friend`, `acquaintance`). Randomized per participant.
- `completion_time_seconds` (float): Time taken to complete the survey (optional, for quality control).
- `is_excluded` (boolean): Flag indicating if the participant was excluded during data cleaning (e.g., straight-lining).
- `exclusion_reason` (string): Reason for exclusion if applicable (e.g., `straight_lining`, `incomplete`).

**Constraints:**
- `participant_id` must match the Prolific ID regex pattern.
- `relationship_context` must be one of the defined contexts.

**Source:** Simulated in `code/02_simulate_ratings.py`; Updated in `code/03_clean_data.py`.
**Storage:** Implicit in `data/raw/ratings.csv` (via `participant_id`) and `data/processed/cleaning_log.csv`.

### 3. Rating

Represents a single data point: a participant's evaluation of a specific stimulus.

**Attributes:**
- `rating_id` (string): Unique identifier for the rating record.
- `participant_id` (string): Foreign key referencing the `Participant`.
- `stimulus_id` (string): Foreign key referencing the `Stimulus`.
- `emotional_support_score` (integer): Likert scale rating (1-7) of perceived emotional support.
- `naturalness_score` (integer): Likert scale rating (1-7) of message naturalness (optional secondary outcome).
- `timestamp` (string): ISO 8601 timestamp of when the rating was recorded.

**Constraints:**
- `emotional_support_score` ∈ [1, 7].
- Each `participant_id` must have exactly one rating for each of the 24 `stimulus_id`s (within their assigned context).
- Composite key (`participant_id`, `stimulus_id`) must be unique.

**Source:** Simulated in `code/02_simulate_ratings.py`.
**Storage:** `data/raw/ratings.csv`.

### 4. AnalysisResult

Represents the output of the statistical modeling pipeline (LMM).

**Attributes:**
- `analysis_id` (string): Unique identifier for the analysis run.
- `model_formula` (string): The statistical formula used (e.g., `score ~ emoji * context + (1|participant) + (1|stimulus)`).
- `fixed_effects` (dictionary): Estimates, standard errors, t-values, and p-values for fixed effects.
- `random_effects` (dictionary): Variance components for random intercepts.
- `interaction_p_value` (float): P-value for the interaction term (emoji × context).
- `post_hoc_results` (list): Tukey-corrected pairwise comparisons (if interaction is significant).
- `sample_size` (integer): Number of observations used in the final model (after cleaning).
- `timestamp` (string): ISO 8601 timestamp of the analysis run.

**Constraints:**
- Must contain at least one fixed effect estimate.
- P-values must be in range [0, 1].

**Source:** Generated by `code/04_run_lmm.py`.
**Storage:** `data/processed/analysis_results.json`.

## Entity Relationships

- **Participant 1:N Rating**: A single participant provides multiple ratings (one for each stimulus).
- **Stimulus 1:N Rating**: A single stimulus is rated by multiple participants.
- **Rating N:1 Participant**: Each rating belongs to exactly one participant.
- **Rating N:1 Stimulus**: Each rating corresponds to exactly one stimulus.
- **AnalysisResult 1:N Rating (Aggregated)**: An analysis result is derived from the aggregate of many ratings.

## Data Flow Diagram

1. **Stimulus Generation** → `data/raw/stimuli.csv`
2. **Rating Simulation** (uses Stimuli + Target N) → `data/raw/ratings.csv`
3. **Data Cleaning** (uses Stimuli + Ratings) → `data/processed/cleaning_log.csv` (updates Participant status)
4. **LMM Analysis** (uses Cleaned Ratings + Stimuli) → `data/processed/analysis_results.json`
