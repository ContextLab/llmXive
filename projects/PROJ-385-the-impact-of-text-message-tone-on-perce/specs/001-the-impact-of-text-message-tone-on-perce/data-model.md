# Data Model: The Impact of Text Message Tone on Perceived Emotional Support

This document defines the canonical data structures (entities) used throughout the research pipeline.
It is derived from the requirements in `plan.md` and `spec.md` and serves as the basis for:
1. Database schema design (if applicable)
2. CSV/JSON file formats for data exchange
3. Validation schemas (JSON Schema / YAML) in `contracts/`
4. Statistical analysis inputs

## Validation Rules

All data artifacts generated by the pipeline MUST conform to the schemas defined in:
`specs/001-text-tone-emotional-support/contracts/`

---

## 1. Stimulus

**Description**: Represents a single text message variant generated for the experiment.
Each stimulus is a unique combination of base scenario and tone manipulation factors.
**Source**: Generated by `code/01_generate_stimuli.py`
**Output File**: `data/raw/stimuli.csv`

### Fields

| Field Name | Type | Description | Constraints |
|:--- |:--- |:--- |:--- |
| `stimulus_id` | String | Unique identifier for the stimulus. | Primary Key. Format: `S{00001}` |
| `scenario_id` | String | Identifier for the base scenario (e.g., "bad_day", "good_news"). | Foreign Key to Scenario (implicit) |
| `message_text` | String | The actual text content of the message presented to participants. | Non-empty, max 500 chars |
| `emoji_level` | Integer | Categorical level of emoji usage (0=None, 1=Low, 2=High). | 0, 1, or 2 |
| `punctuation_level` | Integer | Categorical level of punctuation intensity (0=Standard, 1=High, 2=Excessive). | 0, 1, or 2 |
| `length_category` | String | Length category of the message. | "Short", "Medium", "Long" |
| `tone_label` | String | Ground truth label for the intended tone. | e.g., "Supportive", "Neutral", "Dismissive" |

### Example Record
```csv
stimulus_id,scenario_id,message_text,emoji_level,punctuation_level,length_category,tone_label
S00001,scn_bad_day,"Hey, I know things are tough. I'm here for you if you need to talk! 😊",1,1,Medium,Supportive
```

## 2. Participant

**Description**: Represents a unique human participant in the study.
**Source**: Generated by `code/02_simulate_ratings.py` (Mock) or `code/04_collect_real_data.py` (Real)
**Output File**: `data/raw/ratings.csv` (embedded), `data/consent/` (separate records)

### Fields

| Field Name | Type | Description | Constraints |
|:--- |:--- |:--- |:--- |
| `participant_id` | String | Unique anonymized identifier for the participant. | Primary Key. Format: `P{00001}` (Mock) or Prolific ID hash (Real) |
| `relationship_context` | String | The context of the relationship the participant is imagining. | "Friend", "Acquaintance" |
| `completion_time_sec` | Float | Time taken to complete the rating task. | > 0 |
| `exclusion_flag` | Boolean | Flag indicating if the participant was excluded (e.g., straight-lining). | True/False |
| `exclusion_reason` | String | Reason for exclusion if flagged. | "Straight-lining", "Missing Data", "Null" |

---

## 3. Rating

**Description**: Represents a single rating event where a participant evaluates a stimulus.
**Source**: Generated by `code/02_simulate_ratings.py` or `code/04_collect_real_data.py`
**Output File**: `data/raw/ratings.csv`

### Fields

| Field Name | Type | Description | Constraints |
|:--- |:--- |:--- |:--- |
| `rating_id` | String | Unique identifier for the rating event. | Primary Key |
| `participant_id` | String | Foreign Key to Participant. | Required |
| `stimulus_id` | String | Foreign Key to Stimulus. | Required |
| `emotional_support_score` | Integer | Likert scale rating of perceived emotional support. | 1 (Very Low) to 7 (Very High) |
| `clarity_score` | Integer | Likert scale rating of message clarity. | 1 (Very Unclear) to 7 (Very Clear) |
| `timestamp` | String | ISO 8601 timestamp of the rating. | Required |

### Example Record
```csv
rating_id,participant_id,stimulus_id,emotional_support_score,clarity_score,timestamp
R000001,P00001,S00001,6,7,2023-10-27T10:00:00Z
```

## 4. AnalysisResult

**Description**: Aggregated statistical results from the Linear Mixed-Effects Model (LMM).
**Source**: Generated by `code/04_run_lmm.py`
**Output File**: `data/processed/analysis_results.json`

### Structure

This entity is a JSON object containing:

- `model_summary`: String representation of the LMM fit.
- `fixed_effects`: List of dictionaries containing:
 - `term`: Name of the predictor (e.g., "C(emoji_level)", "C(punctuation_level)", "C(relationship_context)").
 - `estimate`: Coefficient value.
 - `std_err`: Standard error.
 - `t_value`: T-statistic.
 - `p_value`: P-value (Satterthwaite approximation).
- `random_effects`: Dictionary of variance components.
 - `Participant`: Variance of random intercepts.
 - `Stimulus`: Variance of random intercepts.
- `interaction_significant`: Boolean indicating if the interaction p-value < 0.05.
- `post_hoc_tests`: (Optional) List of Tukey-corrected pairwise comparisons if interaction is significant.
 - `comparison`: String "Group A vs Group B".
 - `diff`: Mean difference.
 - `p_adj`: Adjusted p-value.

## Validation Against Specification

This data model satisfies the following requirements:
- **FR-001**: Captures all necessary variables for LMM analysis (Stimulus, Participant, Rating).
- **FR-006**: Includes metadata for factorial design (emoji_level, punctuation_level).
- **SC-004**: Ensures traceability via unique IDs (stimulus_id, participant_id).
- **Constitution Principle IV**: `AnalysisResult` provides a single source of truth for statistical findings.
- **Constitution Principle VI**: Participant anonymity is preserved via hashed IDs in `Participant` and separate consent logs.
