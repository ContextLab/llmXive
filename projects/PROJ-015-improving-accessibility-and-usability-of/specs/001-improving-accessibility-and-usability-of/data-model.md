# Data Model: Improving Accessibility and Usability of Complex Computer Systems for People with Disabilities

## Overview
This document defines the schema for data generated by the simulator and consumed by the analysis pipeline. The data model is designed to support the within-subjects experimental design and the specific statistical tests (T-Test/Wilcoxon) required by the specification.

## Entities

### 1. Participant
Represents a unique individual in the study.
- **Attributes**:
  - `participant_id`: Unique string identifier (e.g., `P001`).
  - `disability_type`: Categorical (e.g., `visual`, `motor`, `cognitive`, `hearing`).
  - `interface_sequence`: Categorical (`traditional_first` or `explainable_first`) to control for order effects.
  - `recruitment_source`: String (for tracking Constitution Principle VI compliance).

### 2. Session
Represents a single interaction instance (one interface variant per participant).
- **Attributes**:
  - `session_id`: Unique string identifier.
  - `participant_id`: Foreign key to Participant.
  - `interface_type`: Categorical (`traditional` or `explainable`).
  - `start_time`: ISO 8601 timestamp.
  - `end_time`: ISO 8601 timestamp.
  - `completion_time_seconds`: Float (derived).
  - `error_count`: Integer.
  - `explanation_engagement_time_seconds`: Float (only for `explainable` interface; 0.0 for `traditional`).
  - `sus_score`: Integer (0-100) or `null` if incomplete.
  - `status`: Categorical (`complete`, `incomplete`, `rejected`).
  - `dropout_reason`: String (if `incomplete`).

### 3. Analysis Metric (Derived)
Aggregated statistical results per metric type.
- **Attributes**:
  - `metric_name`: String (e.g., `completion_time`).
  - `test_method`: String (`Paired T-Test` or `Wilcoxon Signed-Rank`).
  - `statistic`: Float.
  - `p_value`: Float.
  - `p_value_adjusted`: Float (Holm-Bonferroni corrected).
  - `mean_diff`: Float.
  - `confidence_interval_lower`: Float.
  - `confidence_interval_upper`: Float.
  - `n_effective`: Integer (count of valid pairs).

## Data Flow
1.  **Simulator** $\rightarrow$ `data/raw/sessions.jsonl` (Raw logs).
2.  **Cleaner** $\rightarrow$ `data/processed/sessions_clean.csv` (Validated, imputed, flagged).
3.  **Analyzer** $\rightarrow$ `data/processed/metrics_summary.csv` (Statistical outputs).
4.  **Reporter** $\rightarrow$ `paper/figures/` (Visualizations).
