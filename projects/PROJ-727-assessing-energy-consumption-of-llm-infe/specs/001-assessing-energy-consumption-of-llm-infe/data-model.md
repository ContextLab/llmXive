# Data Model: Assessing Energy Consumption of LLM Inference for Code Completion

## 1. Overview

This document defines the data structures, schemas, and relationships for the energy consumption benchmarking pipeline. The system produces three primary data artifacts:
1.  **Raw Inference Logs**: Row-level data for every model-problem interaction.
2.  **Aggregated Metrics**: Summary statistics per model.
3.  **Statistical Report**: Text/JSON output of ANOVA and regression results.

## 2. Entities

### 2.1 ModelRun (Row-Level Data)
Represents a single inference event.

| Field | Type | Description | Source |
|-------|------|-------------|--------|
| `run_id` | UUID | Unique identifier for the run. | Generated |
| `model_id` | String | Identifier of the model (e.g., "gpt2-small"). | Config |
| `problem_id` | String | HumanEval task ID (e.g., "HumanEval/0"). | Dataset |
| `prompt_tokens` | Integer | Number of tokens in the input prompt. | `transformers` |
| `tokens_generated` | Integer | Number of tokens generated by the model. | `transformers` |
| `energy_kwh` | Float | Total energy consumed in kWh. | `codecarbon` |
| `runtime_seconds` | Float | Total runtime in seconds. | `codecarbon` |
| `pass_fail_status` | Integer | 1 if solution passes tests, 0 otherwise. | Evaluation |
| `status` | String | "success", "timeout", "oom", "error". | System |

### 2.2 ModelProfile (Static Metadata)
Represents the static characteristics of the model.

| Field | Type | Description |
|-------|------|-------------|
| `model_id` | String | Identifier. |
| `parameter_count` | Integer | Total parameters (e.g., [deferred]). |
| `architecture_type` | String | e.g., "GPT-2", "Transformer", "CodeGen". |

### 2.3 AggregatedMetric (Summary)
Summary statistics per model.

| Field | Type | Description |
|-------|------|-------------|
| `model_id` | String | Identifier. |
| `mean_energy_per_token` | Float | Average Joules/token (excludes nulls). |
| `std_energy_per_token` | Float | Standard deviation. |
| `accuracy_pass_rate` | Float | Percentage of problems solved (0.0 to 1.0). |
| `total_energy_kwh` | Float | Total energy for the run. |

## 3. Data Flow

1.  **Ingestion**: HumanEval dataset loaded from Parquet.
2.  **Processing**:
    -   Loop over Models.
    -   Loop over Problems.
    -   Inference + Energy Logging -> `ModelRun` record.
    -   Evaluation -> Update `pass_fail_status`.
3.  **Aggregation**: Group by `model_id` to compute `AggregatedMetric`.
4.  **Analysis**: Input `ModelRun` (filtered for valid energy) into ANOVA/Regression.
5.  **Output**: Save `results.csv`, `stats_report.json`, and visualizations.

## 4. Constraints & Validation

-   **Null Handling**: `energy_kwh` or `tokens_generated` may be `null` (None). These rows are excluded from mean calculations (FR-011).
-   **Precision**: Energy values stored as `float64`.
-   **Reproducibility**: `run_id` is deterministic based on `model_id` + `problem_id` + `seed`.
