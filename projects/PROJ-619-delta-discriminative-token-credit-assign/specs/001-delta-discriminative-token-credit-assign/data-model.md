# Data Model: DelTA Build Artifacts

## Overview

This document defines the data model for the validation artifacts generated by the DelTA build process. Since this project validates a build pipeline rather than processing a dataset, the "data" consists of:
1.  **Build Artifacts**: The generated PDF files.
2.  **Validation Report**: The `reproduction_report.md` file summarizing the status.
3.  **Source Metadata**: Information about the vendored submodule.

## Entity Definitions

### 1. BuildArtifact
Represents a file generated by the `make.sh` script.

| Attribute | Type | Description | Constraints |
| :--- | :--- | :--- | :--- |
| `filename` | String | Name of the file (e.g., `notation_example.pdf`) | Must exist in output directory |
| `path` | String | Relative path from project root | Must be valid path |
| `size_bytes` | Integer | File size in bytes | Must be > 50,000 (50KB) |
| `created_at` | ISO8601 | Timestamp of generation | Generated at build time |
| `type` | Enum | Type of artifact (`pdf`, `tex`, `log`) | Must be one of: `pdf`, `tex`, `log` |

### 2. ReproductionReport
Represents the summary of the validation run.

| Attribute | Type | Description | Constraints |
| :--- | :--- | :--- | :--- |
| `status` | Enum | Overall status | Must be: `Success`, `Partial`, `Failed` |
| `build_time_seconds` | Float | Time taken for `make.sh` | Must be < 21,600 (6 hours) |
| `artifacts_generated` | List[BuildArtifact] | List of generated files | Non-empty if status is `Success` |
| `errors` | List[String] | List of error messages | Empty if status is `Success` |
| `gpu_detected` | Boolean | Whether GPU code was found | Must be `False` |
| `submodule_verified` | Boolean | Whether submodule hash matches | Must be `True` |

### 3. SubmoduleMetadata
Represents the state of the vendored code.

| Attribute | Type | Description | Constraints |
| :--- | :--- | :--- | :--- |
| `name` | String | Name of the submodule | Must be `dlbook_notation` |
| `url` | String | Remote URL | Must match `.gitmodules` |
| `commit_hash` | String | Git commit hash | 40-character hex string |
| `path` | String | Local path | `external/dlbook_notation` |

## Relationships

- **ReproductionReport** *contains* **BuildArtifact** (1:N)
- **ReproductionReport** *references* **SubmoduleMetadata** (1:1)

## Constraints & Invariants

1.  **Size Constraint**: Any artifact of type `pdf` must have `size_bytes` > 50,000.
2.  **Status Constraint**: If `gpu_detected` is `True`, `status` must be `Failed`.
3.  **Dependency Constraint**: If `submodule_verified` is `False`, `status` must be `Failed`.
4.  **Time Constraint**: `build_time_seconds` must be < 21,600.

## Validation Logic

The `reproduction_report.md` generation script must enforce the following logic:
- If `make.sh` returns non-zero exit code -> `status` = `Failed`.
- If `pdflatex` not found -> `status` = `Failed`, `errors` includes "Missing pdflatex".
- If any PDF size <= 50KB -> `status` = `Partial` or `Failed` (depending on severity).
- If `device="cuda"` found in code -> `status` = `Failed`.
