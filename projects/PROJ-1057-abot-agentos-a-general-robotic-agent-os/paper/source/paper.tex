\title{ABot-AgentOS: A General Robotic Agent OS with Lifelong Multi-modal Memory}

\author{AMAP CV Lab}
\vspace{-11pt}


\contribution{See \hyperref[sec:contributions]{Contributions} section for a full author list.}

\abstract{

% Recent advances in vision-language models (VLMs) have opened the door to general-purpose embodied agents, yet existing PC-side agent frameworks such as OpenClaw and Hermes remain confined to the digital realm and lack the physical grounding required for real-world robotic deployment. In this paper, we present \textbf{ABot-AgentOS}, a general-purpose robotic agent operating system that serves as the Physical System 2 for embodied intelligence, capable of supporting a broad spectrum of robotic platforms including humanoids and quadruped robot dogs. Our work makes three principal contributions. First, we bridge VLMs and the physical world through a holistic agent design that unifies conversational interaction, locomotion, and manipulation within a single decision loop; new robot embodiments can be onboarded rapidly by simply registering body-specific motion and manipulation skills, without modifying the upper-level reasoning stack. Second, we introduce EmbodiedWorldBench, a comprehensive benchmark suite tailored to embodied agent capabilities, covering a diverse set of long-horizon and compositional tasks that jointly stress-test perception, planning, tool use, and physical execution. Third, in contrast to the text-only memory employed by prior PC-side agents, we design a unified multi-modal memory system that jointly records embodied human–robot dialogue, spatial object layouts, person identities, and their relations. The system organizes multi-modal experience into a compact, source-grounded memory graph rather than raw transcripts or raw visual payloads, enabling traceable retrieval over events, places, objects, people, images, and temporal relations. On top of this graph substrate, ABot-AgentOS supports online memory writing and retrieval during interaction, as well as an offline multi-agent self-evolution loop that diagnoses memory failures and compiles safe runtime assets for subsequent deployment. Extensive experiments on real and simulated robots, together with memory-heavy QA benchmarks, demonstrate that ABot-AgentOS provides a portable, extensible, and physically grounded foundation for lifelong embodied agents.

Recent VLM and VLA systems have improved robotic perception and action prediction, yet long-horizon embodied agents still require a general runtime layer for reasoning, memory, tool use, verification, and cross-embodiment execution. We present \textbf{ABot-AgentOS}, a general robotic Agent Operating System that sits above low-level controllers and provides a deliberative agent layer for scene-conditioned planning, context-isolated skill execution, multi-stage verification, multi-modal memory, and edge-cloud collaboration. To evaluate such systems, we introduce \textbf{EmbodiedWorldBench}, an executable benchmark with 16 indoor, outdoor, and hybrid scenes, four difficulty levels, and over 200 tasks involving navigation, object search, NPC dialogue, dynamic events, and trace-grounded scoring.

ABot-AgentOS further introduces \textbf{Universal Multi-modal Graph Memory}, a persistent source-grounded substrate that converts dialogue, visual observations, spatial context, temporal relations, and task traces into typed nodes and edges. A failure-driven self-evolution loop converts diagnosed memory failures into gated runtime evo-assets that are promoted only to later evaluation splits, preventing current-split ground-truth leakage while enabling continual improvement. On an initial EmbodiedWorldBench subset, ABot-AgentOS improves over a single-controller baseline in both task success and goal completion. Across memory benchmarks, ABot-AgentOS Static achieves 87.5 on LoCoMo, 59.9 on OpenEQA EM-EQA, 88.6 on Mem-Gallery, and 76.5 Acc@All on NExT-QA; self-evolution further improves LoCoMo to 88.7, OpenEQA to 60.4, and Mem-Gallery to 89.0. These results suggest that a general Agent OS layer can improve long-horizon embodied execution while providing persistent, auditable memory for continual interaction.
