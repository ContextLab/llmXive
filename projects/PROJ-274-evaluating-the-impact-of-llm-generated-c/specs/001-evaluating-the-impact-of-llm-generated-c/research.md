# Research: Evaluating the Impact of LLM-Generated Code Documentation on Developer Onboarding

## Overview

This research phase defines the datasets, statistical methods, and experimental design required to answer the research question: *How does LLM-generated code documentation compare to human-written documentation (or no documentation) in reducing onboarding time and effort for new developers?*

**Study Status**: **Pilot/Feasibility Study**. The sample size (N=15-20) is intentionally small to test the pipeline and estimate effect sizes. It is **not** statistically powered to detect medium effects (f=0.25) with high power in a 3-group design (which requires N≈159). Results will be reported as exploratory with explicit power limitations.

## Statistical Methodology

### Primary Analysis Plan

1. **Normality Check**: Shapiro-Wilk test on task completion times for each condition (exploratory).
2. **Primary Test**: **Linear Mixed-Effects Model (LMM)**
   - *Fixed Effects*: `Condition` (LLM Docs, Human Docs, No Docs).
   - *Random Effects*: `Repository` (to account for varying difficulty across codebases).
   - *Rationale*: This isolates the 'Condition' effect from the 'Repository' effect, addressing the confounding risk where repo difficulty might correlate with documentation quality.
   - *Alternative*: If LMM assumptions fail, use **Aligned Rank Transform (ART) ANOVA** with blocking by repository.
3. **Post-Hoc Tests**:
   - If LMM significant: Tukey HSD (or `emmeans` with Tukey adjustment) for pairwise comparisons.
   - If ART ANOVA significant: Dunn's test with Bonferroni correction.
4. **Effect Size**: Partial eta-squared (LMM) or epsilon-squared (ART).
5. **Significance Threshold**: α = 0.05 (with sensitivity analysis at 0.01, 0.10).

### Assumptions & Limitations

- **Causal Inference**: The "LLM vs. None" comparison is causal due to random assignment. The "LLM vs. Human" comparison is associational due to potential confounding in human doc quality (mitigated by using the same repo artifacts).
- **Power**: **Explicit Limitation**: N=15-20 is insufficient for medium effects (f≈0.25). The study is designed as a pilot to estimate effect sizes and variance components for a future full-scale study (target N≈sufficient for adequate power).
- **Collinearity**: Not applicable (conditions are mutually exclusive).
- **Multiple Comparisons**: Tukey/Bonferroni corrections applied to control family-wise error rate.
- **Resource Constraints**: The 7GB RAM constraint applies to the *analysis* phase (negligible usage) and the *generation* phase (context window limits, not RAM). The assumption in `spec.md` regarding RAM for the entire codebase is a category error; the analysis is lightweight.

## Experimental Design

### Conditions

1. **LLM Docs**: Participants receive documentation generated by an LLM (pinned model, temperature, prompt).
2. **Human Docs**: Participants receive existing human-written documentation from the **same** repository (verified by rubric).
3. **No Docs**: Participants receive no documentation.

### Tasks

- Standardized onboarding tasks (e.g., "fix bug X", "explain module Y") on selected repositories.
- Each participant completes tasks in one condition only.
- **Documentation Parity**: The 'Human' condition uses the exact same repository artifacts (pinned commit) as the 'LLM' condition's target to ensure the comparison is about the *source* of the documentation, not the *quality* of the repo.

### Metrics

- **Primary**: Task completion time (seconds).
- **Secondary**: Number of clarification questions, subjective helpfulness rating (Likert scale).

## Decision Rationale

- **LMM vs. ANOVA**: LMM is chosen to control for 'Repository' as a random effect, preventing repo difficulty from confounding the 'Condition' effect.
- **CPU-Only Execution**: All statistical tests (LMM, ART ANOVA) are computationally lightweight and will run within 6 hours on free-tier CI.
- **Dataset Generation**: Since no external dataset contains the required variables, the study will generate its own data through controlled experiments.
- **Pilot Scope**: The study is explicitly framed as a pilot to avoid false claims of statistical power.
