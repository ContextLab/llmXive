# Quickstart: Evaluating the Impact of LLM-Generated Code Documentation on Developer Onboarding

## 1. Prerequisites

- Python 3.11+
- Git
- Access to GitHub (for repository cloning)
- LLM API key (optional; fallback to local model)

## 2. Installation

```bash
# Clone the repository
git clone
cd evaluating-llm-docs-impact

# Create virtual environment
python -m venv venv
source venv/bin/activate # On Windows: venv\Scripts\activate

# Install dependencies
pip install -r requirements.txt
```

## 3. Project Structure

```text
projects/PROJ-274-evaluating-the-impact-of-llm-generated-c/
├── data/ # Data storage
├── code/ # Source code
│ ├── experiment/ # Experiment runner
│ ├── generation/ # Documentation generation
│ ├── analysis/ # Statistical analysis
│ └── utils/ # Utilities (logging, anonymization)
├── tests/ # Test suite
├── requirements.txt
└── README.md
```

## 4. Running the Mock Study (Phase 2)

To test the experiment framework with simulated participants:

```bash
# Run mock study with 3 participants (1 per condition)
python code/experiment/experiment.py --mode mock --participants 3
```

**Expected Output**:
- Logs in `data/raw/task_logs.json`
- Summary in `data/raw/mock_summary.json`

## 5. Generating Documentation (Phase 2)

To generate LLM documentation for a repository:

```bash
# Generate docs for a repo (pinned to commit)
python code/generation/doc_pipeline.py \
 --repo \
 --commit abc123def456 \
 --output data/processed/docs/repo_docs.md
```

**Fallback**: If API fails, uses `phi-2` (int4) locally.

## 6. Running Statistical Analysis (Phase 3)

To run the full analysis pipeline:

```bash
# Run analysis on processed data
python code/analysis/stats_runner.py \
 --input data/processed/task_logs_anon.json \
 --output data/processed/analysis_results.json
```

**Output**:
- `analysis_results.json`: Test statistics, p-values, conclusions.
- `report.md`: Final report (generated by `report_gen.py`).

## 7. Monitoring Resource Usage

To monitor runtime and memory:

```bash
# Run with monitoring enabled
python code/utils/logging.py --track \
 --command "python code/analysis/stats_runner.py..."
```

**Output**:
- Runtime (seconds)
- Peak memory (MB)

## 8. Validating Schemas

To validate data against contracts:

```bash
# Run contract tests
pytest tests/contract/
```

**Expected**: All tests pass; data matches schemas in `contracts/`.

## 9. Common Issues

| Issue | Solution |
|-------|----------|
| **API Rate Limit** | Fallback to `phi-2` local model (FR-008). |
| **Memory Exceed** | Reduce dataset size; check `psutil` logs. |
| **Repo Not Found** | Verify commit hash; ensure ≤500 files. |
| **PII Detected** | Run `anonymize.py` before analysis. |
