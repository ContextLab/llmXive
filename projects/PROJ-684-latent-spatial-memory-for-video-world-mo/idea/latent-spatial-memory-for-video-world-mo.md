# Latent Spatial Memory for Video World Models

**Field**: computer science

## Research question

How does maintaining explicit 3D spatial representations in latent memory affect the long-term temporal consistency of video world model predictions compared to purely temporal memory mechanisms?

## Motivation

Video world models must maintain spatial coherence across frames to simulate physically plausible environments, yet existing approaches rely heavily on pixel-space or purely temporal memory that degrades over long horizons. The gap is whether latent-space 3D representations can provide more efficient and stable memory without the computational burden of explicit point-cloud reconstruction. Answering this informs both model architecture design and the theoretical understanding of what memory structures support spatial reasoning.

## Related work

- [Out of Sight but Not Out of Mind: Hybrid Memory for Dynamic Video World Models (2026)](https://arxiv.org/abs/2603.25716) — Addresses dynamic object occlusion in video world models using hybrid memory, relevant to how spatial consistency should be maintained when objects move out of view.
- [From Masks to Worlds: A Hitchhiker's Guide to World Models (2025)](https://arxiv.org/abs/2510.20668) — Provides conceptual framework for world model design and evaluation, useful for understanding the relationship between memory mechanisms and world model performance.
- [Improving World Models using Deep Supervision with Linear Probes (2025)](https://arxiv.org/abs/2504.03861) — Investigates supervision techniques for encoding spatial structure in world models, relevant to how latent representations should be validated.

## Expected results

We expect that explicit 3D latent spatial memory will show measurable improvement in long-horizon prediction consistency (≥50 frames) compared to temporal-only baselines, with the effect being detectable via frame-wise alignment error and object permanence metrics. A null result would indicate that temporal memory alone suffices for spatial consistency at the scales we test, which would be informative for model efficiency trade-offs.

## Methodology sketch

- Download a subset of RealEstate10K (first 100 videos) via public mirror; store in `/data/re10k_subset/`
- Load pre-trained video world model checkpoints (e.g., from HuggingFace) that support CPU inference; record checkpoint URLs and versions
- Implement two memory variants: (a) temporal-only (RNN/Transformer state), (b) latent 3D spatial memory (compressed voxel grid)
- Generate 100-frame rollouts from 20 random prompts per variant; save latent states and reconstructed frames
- Compute frame-wise consistency metrics: PSNR, SSIM, and learned perceptual similarity (LPIPS) on CPU using pre-trained features
- Measure memory footprint (GB) and inference time (s/frame) for each variant across 3 random seeds
- Apply paired bootstrap test (1000 resamples) to assess statistical significance of consistency differences
- Ablate by removing spatial memory component to isolate its contribution to consistency vs. speed
- Document all hyperparameters, random seeds, and hardware configuration in `/docs/protocol.md`
- Release code and artifacts under permissive license with Zenodo DOI for reproducibility
