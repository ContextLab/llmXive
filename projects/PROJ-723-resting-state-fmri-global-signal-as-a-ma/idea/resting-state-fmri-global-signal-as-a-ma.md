# Resting‑State fMRI Global Signal as a Marker of Mind‑Wandering  

**Field**: neuroscience  

## Research question  

Does the amplitude (standard deviation) of the resting‑state fMRI global signal predict individual differences in self‑reported mind‑wandering frequency as measured by the Mind‑Wandering Questionnaire (MWQ)?

## Motivation  

Mind‑wandering influences attention, creativity, and mental‑health outcomes, yet inexpensive neuroimaging biomarkers for its propensity are lacking. The global signal is a low‑dimensional, easily extracted metric from any resting‑state scan; demonstrating its relevance to mind‑wandering would provide a cost‑effective marker for large‑scale studies and clinical screening.

## Related work  

- [Signal Fluctuation Sensitivity: an improved metric for optimizing detection of resting‑state fMRI networks (2015)](https://arxiv.org/abs/1511.04345) — Introduces a sensitivity metric for resting‑state signal fluctuations, showing that simple amplitude measures can enhance detection of functional networks, supporting the use of global‑signal variance as a candidate biomarker.  
- [Mind Drifts, Data Shifts: Utilizing Mind Wandering to Track the Evolution of User Experience with Data Visualizations (2024)](https://arxiv.org/abs/2408.03576) — Demonstrates that self‑report questionnaires reliably capture moment‑to‑moment mind‑wandering, providing validation for using the MWQ as a behavioral ground‑truth.  
- [Automatic Mind Wandering Detection in Educational Settings: A Systematic Review and Multimodal Benchmarking (2026)](https://arxiv.org/abs/2604.09569) — Reviews multimodal approaches to detecting mind‑wandering and highlights the scarcity of neuroimaging‑based markers, underscoring the need for simple fMRI predictors.  
- [Fractal‑driven distortion of resting state functional networks in fMRI: a simulation study (2012)](https://arxiv.org/abs/1208.0924) — Shows that low‑frequency, scale‑invariant fluctuations shape resting‑state connectivity; the global signal captures such low‑frequency power, providing a theoretical link to cognitive states like mind‑wandering.  

## Expected results  

We anticipate a modest but reliable positive correlation (r ≈ 0.2–0.3) between global‑signal amplitude and MWQ scores across participants. Ridge regression with 5‑fold cross‑validation should yield a mean absolute error significantly lower than a null model (permutation of MWQ scores), confirming that the global signal carries predictive information beyond chance. Failure to obtain such a relationship would indicate that the global signal alone is insufficient as a biomarker.

## Methodology sketch  

- **Data acquisition**: Download the minimally preprocessed resting‑state fMRI runs (4 × 15 min) and MWQ scores for the HCP 1200‑subject release (https://db.humanconnectome.org). Subsample ~200 participants with complete data.  
- **Preprocessing**:  
  1. Load each run with Nilearn; apply brain mask.  
  2. Compute the voxel‑wise mean time series (global signal) and calculate its standard deviation (amplitude) per run; average across runs for each subject.  
  3. Extract standard motion confounds (FD, DVARS) provided by HCP; regress them out of the global‑signal amplitude.  
  4. Regress demographic covariates (age, sex, education) from both the global‑signal metric and MWQ scores.  
- **Modeling**:  
  1. Assemble a feature matrix X = {global‑signal amplitude}.  
  2. Target vector y = MWQ total score.  
  3. Perform ridge regression with nested 5‑fold cross‑validation to tune the regularization parameter (α).  
  4. Record out‑of‑fold Pearson r, MAE, and R².  
- **Baseline comparison**: Train the same ridge pipeline on permuted y (100 permutations) to generate a null distribution of performance metrics.  
- **Statistical testing**: Compute a one‑sample t‑test comparing observed MAE to the null distribution; report p‑value and effect size.  
- **Robustness checks**:  
  1. Repeat analysis using global‑signal variance instead of standard deviation.  
  2. Add a control predictor (average framewise displacement) to ensure the effect is not driven by residual motion.  
- **Reproducibility**: All code will be written in Python (numpy, pandas, nilearn, scikit‑learn) and containerized with Docker; the workflow will be orchestrated via a Makefile to run within the 6‑hour GitHub Actions limit.
