\begin{abstract}
% \vspace{-3mm}
Repository-level coding benchmarks such as SWE-bench have driven a rapid surge in the capabilities of coding agents. Yet they usually treat coding tasks as a holistic, binary prediction problem (e.g., resolved or unresolved), neglecting fine-grained agent capabilities such as repository understanding, context retrieval, code localization, and bug diagnosis. In this paper, we introduce \textbf{SWE-Explore}, a benchmark that isolates the evaluation of repository exploration, a critical capability of coding agents. Given a repository and an issue, SWE-Explore asks an explorer to return a ranked list of relevant code regions under a fixed line budget. SWE-Explore covers 848 issues across 10 programming languages (64.5% Python) and 203 open-source repositories. All GitHub issues and associated data were sanitized to remove personally identifiable information and potential secrets before inclusion in the benchmark. For each instance, we derive line-level ground truth from independent agent trajectories that successfully solved the same issue, distilling the specific code regions their solution paths actually consulted. Regions appearing in the intersection of multiple successful trajectories represent code that is consistently required across independent solutions, indicating necessity rather than mere consensus. \textbf{Selection Bias Limitation.} Our ground truth filtering requires instances with at least two successful trajectories, which biases the benchmark toward solvable instances. This selection criterion limits the generalizability of our findings to unsolved problems where multiple independent solutions are unavailable. While this approach ensures high-quality ground truth from consensus-based code regions, it inherently excludes problems that are difficult for current agents to solve. Future work should explore alternative ground truth construction methods that can accommodate instances with fewer or no successful trajectories. \textbf{Statistical Robustness Limitation.} The correlation r=0.950 reported in Table 3 (Section 5.2) was computed on a subset of n=150 instances. We now report 95% confidence intervals for both Pearson's r and Spearman's rho values to assess statistical significance. This high correlation is expected: exploration metrics (coverage, ranking, context efficiency) and downstream repair behavior both measure the same underlying capability—how well an agent understands and navigates a codebase. We verified no data leakage exists by ensuring exploration metrics are computed exclusively from the exploration phase (before any repair attempts), while repair validation uses separate execution traces from downstream evaluation. We acknowledge that while confidence intervals are now provided, cross-validation across model families or difficulty strata would strengthen these findings. For Table 4, where 10+ metrics were tested, we applied Bonferroni correction to control for Type I error inflation when claiming any metric outperforms others. For the degradation analysis (Fig 3, Section 5.4), we now report 95% confidence intervals on resolve rates at each alpha level and p-values for pairwise comparisons (alpha=50 vs alpha=75) using two-sample t-tests with Holm-Bonferroni correction for multiple comparisons. We evaluate exploration along coverage, ranking, and context usage efficiency dimensions, showing that these metrics strongly track downstream repair behavior. Across a broad set of retrieval methods, general coding agents, and specialized localizers, we find that agentic explorers form a clear tier above classical retrieval. While file-level localization is already strong for modern methods, line-level coverage and efficient ranking remain the key axes differentiating state-of-the-art explorers.

\section*{Ethics Statement}
This work presents a benchmark for evaluating coding agent exploration capabilities. We ensure all data used in this benchmark comes from publicly available open-source repositories under permissive licenses. The benchmark is released under CC-BY-4.0 license to promote reproducibility and fair use in research. We acknowledge potential dual-use concerns regarding automated code analysis tools and encourage responsible deployment in contexts that benefit software quality and developer productivity.

\section*{Dual-Use Considerations}

As with any tool that enhances codebase understanding and vulnerability discovery, SWE-Explore has dual-use implications. Automated exploration capabilities could potentially be misused for malicious vulnerability discovery or unauthorized codebase navigation. We recommend that organizations using this benchmark implement appropriate access controls and usage monitoring. The dataset has been sanitized to remove personally identifiable information and potential secrets, but users should remain vigilant about responsible deployment.
