# Implementation Plan: Statistical Analysis of Topic Drift in Academic Abstracts

**Branch**: `001-topic-drift-analysis` | **Date**: 2024-01-15 | **Spec**: `specs/001-topic-drift-analysis/spec.md`
**Input**: Feature specification from `/specs/001-topic-drift-analysis/spec.md`

## Summary

This project implements a statistical analysis pipeline to measure "topic drift" in academic abstracts (arXiv and PubMed) from 2000–2024. The approach involves:
1.  **Data Acquisition**: Fetching abstracts via APIs (primary) with checksummed storage for reproducibility.
2.  **Preprocessing**: Tokenization, lemmatization, and filtering (min 20 tokens) with **window-specific stopword lists** to mitigate language drift.
3.  **Topic Modeling**: Fitting LDA (k=10) per 5-year window, with a **validation step** to ensure k=10 is optimal (within 10% of max coherence).
4.  **Topic Alignment**: Aligning topic indices across windows via cosine similarity of topic-word distributions to resolve label switching.
5.  **Drift Measurement**: Computing Jensen-Shannon (JS) divergence between **aligned** topic distributions.
6.  **Significance Testing**: Permutation testing (n=1000) with **LDA refitting on shuffled data** to generate a null distribution, using **MaxT procedure** for Family-Wise Error Rate (FWER) control to account for dependent windows.
7.  **Confidence Intervals**: Bootstrapping to generate confidence intervals for divergence values.
8.  **Sensitivity Analysis**: Sweeping coherence thresholds across a defined range to report inconsistency rates.

The entire pipeline is designed to run on CPU-only infrastructure within 6 hours and 7GB RAM, utilizing stratified sampling for the permutation test to ensure feasibility.

## Complexity Tracking

| Violation | Why Needed | Simpler Alternative Rejected Because |
|-----------|------------|-------------------------------------|
| **Permutation Test (n=1000) with Refit** | Required by FR-007 for statistical rigor; shuffling abstracts requires refitting LDA to test the null hypothesis correctly. | Parametric tests assume normality of divergence distribution, which is unknown. Shuffling vectors without refit ignores LDA variance. |
| **Topic Alignment** | Required to resolve label switching; Topic 0 in Window 1 is not Topic 0 in Window 2. | Computing JS on unaligned vectors measures arbitrary distance, not semantic drift. |
| **MaxT Correction** | Required because window pairs are dependent (share data). BH assumes independence. | BH on dependent tests risks invalid FDR control. MaxT accounts for correlation structure. |
| **Window-specific Stopwords** | Required by FR-014 to prevent language drift mimicking topic drift. | Global stopword lists fail to capture era-specific terminology (e.g., "deep learning" vs "neural nets"). |
| **Sensitivity Analysis** | Required by FR-009 to ensure robustness of coherence threshold. | Single threshold (0.4) might be arbitrary; sweeping {0.35, 0.40, 0.45} validates stability. |
| **Stratified Sampling for Permutations** | Required to fit 1000 LDA models within 6 hours on CPU. | Full dataset refit would exceed 6 hours; stratified sample maintains representativeness. |
