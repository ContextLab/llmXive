## Summary

This project investigates whether agent failures in long-horizon professional workflows (Agents' Last Exam - ALE) are driven primarily by state persistence errors rather than reasoning deficits. The technical approach involves: (1) generating synthetic execution traces with known ground truth to ensure data granularity (FR-001), (2) implementing a Hybrid Classification Protocol using a local LLM to distinguish "State Persistence Errors" from "Reasoning Deficits" based on semantic intent and reconstructed state (FR-002), (3) running a CPU-optimized Large language model

The research question remains: How does scaling model size affect performance on downstream tasks? The method will involve training models of varying capacities and evaluating them on standard benchmarks. References: [Citation] (via GGUF Q4_K_M) with a Context-Checkpointing wrapper (FR-003, FR-004), and (4) performing Mixed-Effects Logistic Regression to analyze step-level success rates while accounting for trajectory divergence (FR-005).

## Classification Logic (Constitution VI Compliance)

To satisfy Constitution Principle VI and FR-001/FR-002, the classification process is strictly defined:

1.  **Normalization Protocol**: Before comparison, `environment_state` is normalized:
    *   Floating-point values compared with a tolerance threshold.
    *   Timestamps and random IDs stripped.
    *   Object references canonicalized.
2.  **Semantic Classification**:
    *   **State Persistence Error**: Agent action contradicts the *reconstructed* environment state (e.g., editing a file that was deleted in a prior step).
    *   **Reasoning Deficit**: Agent action is logically invalid for the *task goal* **given** the reconstructed environment state. (e.g., Agent attempts to open a file that exists, but opens the wrong one due to poor planning, despite having the correct context).
    *   *Implementation*: A local LLM (distil-llama-instruct, Q4_K_M) is prompted with the task goal, reconstructed state, and agent action to determine the label. This avoids the "rule-based heuristic" limitation of missing semantic context.
