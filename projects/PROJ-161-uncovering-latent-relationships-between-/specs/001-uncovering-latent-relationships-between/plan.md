# Implementation Plan: Uncovering Latent Relationships Between Molecular Descriptors and Antibiotic Resistance

**Branch**: `001-uncovering-latent-relationships` | **Date**: 2023-10-27 | **Spec**: `specs/001-uncovering-latent-relationships/spec.md`

## Summary
This feature implements a computational pipeline to investigate correlations between physicochemical molecular descriptors (e.g., logP, TPSA) and antibiotic resistance phenotypes. The system ingests antibiotic structures from chemical repositories (ChEMBL/ZINC15) and resistance data from NCBI, computes descriptors via RDKit, reduces dimensionality with UMAP, clusters via DBSCAN, and performs rigorous statistical testing (Mann-Whitney U, Fisher's exact) with FDR correction. The implementation strictly adheres to CPU-only constraints for GitHub Actions compatibility.

**Testing**: `pytest` with `pytest-cov`

## Phase Mapping to Requirements

- **FR-001 (Data Acquisition)**: `src/data/download.py` + `src/data/process.py` (Merge on InChIKey, **Aggregation Strategy**: `resistance_frequency = resistant_isolates / total_tested_isolates`, min n=5). **SC-001 Metric**: `src/data/process.py` generates `merge_metrics.json` (total, matches, fraction).
- **FR-002 (Descriptors)**: `src/data/process.py` (RDKit `Descriptors` module).
- **FR-003 (UMAP)**: `src/analysis/dimensionality.py` (n_neighbors=15, min_dist=0.1).
- **FR-004 (DBSCAN + Fisher)**: `src/analysis/clustering.py` (eps=0.5, min_samples=10 + Fisher).
- **FR-005 (Stats + FDR + VIF)**: `src/analysis/statistics.py` (Mann-Whitney, BH-FDR, Cohen's d, **VIF Filtering** to remove collinear descriptors before testing).
- **FR-006 (CPU Feasibility)**: Enforced via library selection (no GPU libs) and sampling logic in `research.md`.
- **FR-007 (Permutation Test)**: `src/analysis/clustering.py::run_label_permutation_test` (Config: `PERMUTATION_ITERATIONS=1000` in `config.py`). **Validation**: Shuffles labels relative to *fixed* UMAP/DBSCAN embedding. Validated by `tests/unit/test_permutation_validation.py`.
- **FR-008 (Sensitivity)**: `src/analysis/statistics.py` (Spearman correlation comparison).
- **FR-009 (Bimodality + Fallback)**: `src/analysis/statistics.py::assess_bimodality_and_fallback` (Hartigan's dip test + GMM + Spearman fallback). **Fallback Logic**: If Dip Test fails -> GMM -> If GMM fails -> Continuous Spearman.
- **SC-001 (Merge Fraction)**: `src/data/process.py` generates `merge_metrics.json` (total, matches, fraction), validated against `contracts/output.schema.yaml`.
- **SC-004 (Runtime)**: `src/main.py` logs duration to `runtime_metrics.json`, validated against `contracts/output.schema.yaml`.
- **SC-005 (Reproducibility)**: `tests/integration/test_pipeline.py` runs the pipeline twice (once with snapshot, once fresh) and compares output hashes to measure reproducibility.

## Risk Mitigation

- **Data Unavailability**: If NCBI fetch fails, pipeline halts with `DATA_SOURCE_UNAVAILABLE`. No synthetic data is generated.
- **Collinearity**: VIF filtering ensures only independent descriptors are tested.
- **Circularity**: Permutation test shuffles labels on fixed embedding, breaking the tautology.
- **Distribution Shape**: Fallback to GMM or Continuous Spearman ensures the study proceeds even if bimodality is absent.
