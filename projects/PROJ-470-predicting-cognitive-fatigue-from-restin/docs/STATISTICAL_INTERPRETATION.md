# Statistical Interpretation Guidelines

This document provides guidelines for interpreting the statistical results generated by the
cognitive fatigue prediction pipeline. It covers correlation coefficients, p-values,
confidence intervals, and multiple comparison corrections.

## Overview of Statistical Methods

The pipeline employs the following statistical techniques:

1. **Correlation Analysis**: Pearson or Spearman correlation between complexity metrics and fatigue scores.
2. **Multiple Comparison Correction**: Benjamini-Hochberg (BH) procedure to control False Discovery Rate (FDR).
3. **Sensitivity Analysis**: Evaluation of results at different significance thresholds.
4. **Collinearity Diagnostics**: Variance Inflation Factor (VIF) to assess multicollinearity.

## Correlation Coefficients

### Pearson Correlation (r)

- **Use Case**: Measures linear relationships between two continuous variables.
- **Assumptions**: Normality, linearity, homoscedasticity.
- **Interpretation**:
 - `r = 1.0`: Perfect positive linear relationship.
 - `r = -1.0`: Perfect negative linear relationship.
 - `r = 0.0`: No linear relationship.
 - `|r| > 0.7`: Strong correlation.
 - `0.4 < |r| ≤ 0.7`: Moderate correlation.
 - `0.2 < |r| ≤ 0.4`: Weak correlation.
 - `|r| ≤ 0.2`: Negligible correlation.

### Spearman Correlation (ρ)

- **Use Case**: Measures monotonic relationships (linear or non-linear).
- **Assumptions**: Ordinal or continuous data, monotonic relationship.
- **Interpretation**: Same as Pearson, but robust to outliers and non-normal distributions.

**Pipeline Default**: Spearman correlation is used by default (`correlation_method: "spearman"`)
due to the non-normal distribution often observed in EEG complexity metrics and fatigue scores.

## P-Values and Statistical Significance

### Definition

The p-value represents the probability of observing a correlation as extreme as the one calculated,
assuming the null hypothesis (no correlation) is true.

### Thresholds

- **p ≤ 0.05**: Conventionally considered statistically significant.
- **p ≤ 0.01**: Highly significant.
- **p ≤ 0.001**: Very highly significant.

**Caution**: A small p-value does not imply a large effect size. Always report both p-values and correlation coefficients.

## Multiple Comparison Correction

### The Problem

When testing multiple electrodes (e.g., 20 channels), the probability of false positives increases.
With α = 0.05 and 20 tests, we expect 1 false positive by chance alone.

### Benjamini-Hochberg (BH) Procedure

The pipeline uses the BH method to control the False Discovery Rate (FDR).

**Steps**:
1. Sort all p-values from smallest to largest.
2. For each p-value at rank `i` (out of `m` total tests), calculate the adjusted threshold: `(i/m) * α`.
3. Find the largest `k` such that `p(k) ≤ (k/m) * α`.
4. Reject all null hypotheses for `i ≤ k`.

**Interpretation**:
- An adjusted p-value (q-value) < 0.05 means the result is significant after controlling FDR at 5%.
- This is less conservative than Bonferroni correction, preserving statistical power.

### Reporting

Always report:
- The raw p-value.
- The adjusted p-value (q-value).
- The correction method used (e.g., "Benjamini-Hochberg FDR").

## Confidence Intervals

### Definition

A 95% confidence interval (CI) provides a range of values within which the true population parameter
is likely to lie, with 95% confidence.

### Interpretation

- If the 95% CI for a correlation coefficient **does not include 0**, the result is statistically significant at α = 0.05.
- Narrower intervals indicate higher precision (larger sample size or lower variance).

**Example**:
- `r = 0.45, 95% CI [0.20, 0.65]`: Significant positive correlation.
- `r = 0.10, 95% CI [-0.15, 0.35]`: Not significant (includes 0).

## Sensitivity Analysis

The pipeline generates a sensitivity table (`data/analysis/sensitivity_table.csv`) showing the number
of significant electrodes at different p-value thresholds (0.01, 0.05, 0.10).

### Interpretation

- **Robustness**: If a result holds across multiple thresholds (e.g., significant at p ≤ 0.01 and p ≤ 0.05),
 it is more robust.
- **Threshold Selection**: Choose a threshold based on the study's tolerance for Type I errors (false positives)
 vs. Type II errors (false negatives).

## Collinearity Diagnostics

### Variance Inflation Factor (VIF)

When combining multiple complexity metrics (e.g., LZC + PE) in a regression model, multicollinearity
can inflate standard errors and reduce reliability.

**Threshold**:
- **VIF < 5**: Acceptable (no severe multicollinearity).
- **5 ≤ VIF < 10**: Moderate multicollinearity.
- **VIF ≥ 10**: Severe multicollinearity; consider removing variables.

## Practical Guidelines for Reporting

1. **Always report effect sizes**: "LZC in channel Fpz was significantly correlated with post-sleep fatigue (ρ = 0.42, p = 0.003, 95% CI [0.15, 0.65])."
2. **Specify correction method**: "Significance was determined using Benjamini-Hochberg FDR correction (q < 0.05)."
3. **Include confidence intervals**: "The correlation coefficient was 0.42 (95% CI [0.15, 0.65])."
4. **Discuss practical significance**: A statistically significant result (p < 0.05) may have a small effect size (r = 0.1) and limited practical relevance.
5. **Acknowledge limitations**: Sample size, data quality, and potential confounders (e.g., age, sex) should be discussed.

## Example Interpretation from Pipeline Output

**Scenario**:
- Channel: Fpz
- Metric: Lempel-Ziv Complexity (LZC)
- Correlation: Spearman ρ = 0.38
- Raw p-value: 0.012
- Adjusted p-value (BH): 0.045
- 95% CI: [0.08, 0.62]

**Interpretation**:
"There is a moderate, statistically significant monotonic correlation between LZC in the Fpz channel
and post-sleep fatigue scores (ρ = 0.38, p = 0.012, q = 0.045, 95% CI [0.08, 0.62]).
This suggests that higher EEG complexity during resting state is associated with increased subjective fatigue
after sleep. The result remains significant after Benjamini-Hochberg correction for multiple comparisons
across electrodes."
