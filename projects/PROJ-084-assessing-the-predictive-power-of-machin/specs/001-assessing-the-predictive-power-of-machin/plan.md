# Implementation Plan: Assessing the Predictive Power of Machine Learning for Organic Reaction Outcomes

**Branch**: `001-assess-ml-predictive-power` | **Date**: 2026-07-09 | **Spec**: `specs/001-assess-ml-predictive-power/spec.md`
**Input**: Feature specification from `/specs/001-assess-ml-predictive-power/spec.md`

## Summary

This project assesses the predictive power of classical machine learning (Random Forest, SVM) for organic reaction yields using molecular fingerprints (ECFP4, MACCS). The approach involves ingesting a verified USPTO yield dataset, sanitizing chemical structures, generating fingerprints, and training models under strict CPU-only constraints (free-tier CI). The plan addresses data quality, **Stratified-by-Class + Intra-Class Scaffold Grouping** splitting to prevent leakage and ensure chemical diversity, and interpretable feature importance analysis via substructure aggregation. All results are reproducible and fit within 7GB RAM / 6h runtime limits.

## Technical Context

**Language/Version**: Python 3.11

**Project Type**: Data Science Pipeline / Research Library  
**Performance Goals**: Process full dataset subset within 6 hours; Peak RAM < 7.0 GB  
**Constraints**: No GPU, no deep learning, no external API calls during execution, strict reproducibility (pinned seeds).  
**Scale/Scope**: A representative fixed subset of reactions (subsetted for training/tuning to ensure feasibility), -bit ECFP4 vectors.

## Complexity Tracking

| Violation | Why Needed | Simpler Alternative Rejected Because |
| :--- | :--- | :--- |
| **Stratified-by-Class + Scaffold Split** | Essential to satisfy Constitution Principle VI (class representation) AND prevent leakage (scaffold grouping). | Pure scaffold split risks omitting entire reaction classes. Pure random split risks leakage. The hybrid approach ensures both. |
| **Fixed Representative Subset** | Required to keep RAM < 7GB and runtime < 6h while maintaining statistical validity for tuning. Tuning on a sample and training on full data introduces distribution mismatch bias. | A fixed subset used for the entire pipeline (tuning + training) is statistically valid and computationally feasible. |
| **Substructure Aggregation** | ECFP4 bits are collinear; individual bit importance is unstable. | Reporting individual bits fails to provide interpretable chemical insight. Aggregating importance across all bits associated with a substructure provides a robust metric. |
