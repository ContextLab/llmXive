## Physical Model Specification (Synthetic Generator)

To ensure the synthetic dataset does not produce trivial or tautological results, the generator implements the following physical models:
1.  **Conductivity**: Uses **Matthiessen's Rule** ($\sigma^{-1} = \sigma_0^{-1} + \sigma_{defect}^{-1}$) where $\sigma_{defect}$ scales with defect density and type.
2.  **Young's Modulus**: Uses the **Rule of Mixtures** and continuum elasticity theory, where $E = E_0 (1 - \alpha \cdot \rho_{defect})$, with $\alpha$ varying by defect type.
3.  **Fracture Strength**: Uses **Griffith's Criterion** scaling, where $\sigma_f = \sigma_{f0} (1 - \beta \cdot \sqrt{\rho_{defect}})$, introducing non-linearity.
These models ensure that the relationships between predictors and targets are physically grounded but non-trivial, allowing the Random Forest to learn meaningful patterns rather than simple linear correlations.

## Statistical Methodology

### 1. Preprocessing & Normalization
- Material‑level pristine reference values (`σ₀`, `E₀`, `σ_f₀`) are fetched once per material from the cached pristine structures and stored in `data/processed/pristine_refs.json`.
- **Consistency**: The synthetic generator must use the *same* $P_0$ values to ensure normalized targets are not trivially linear with predictors.
- Normalized targets are computed as ΔP / P₀ for conductivity, Young's modulus, and fracture strength (FR‑003).
- Entries with missing fracture energy are processed with a mock DFTB+ call (≤ 300 s); failures are flagged `[MISSING: timeout]` and excluded.

### 2. Modeling Strategy
- Random Forest regressors (scikit‑learn) are trained separately for each target (FR‑004). Train/test split: [deferred]/20 % with `seed=42`.
- 5‑fold cross‑validation (k = 5) is performed; **standard deviation of R² (cv_std)** is calculated and reported (SC‑003).
- Null model baseline (mean prediction) is evaluated for R² improvement.

### 3. Collinearity Handling (FR‑008)
- VIF is computed for all predictors.
- **Primary Strategy**: Apply **Ridge Regression** (L2 regularization) to handle collinearity in geometric descriptors (e.g., tilt angle vs. density).
- **Fallback**: If Ridge fails or if a feature is deemed physically redundant (VIF > 5 AND low importance), exclude the lower‑importance feature and re‑train.

### 4. Statistical Inference & Rigor (FR‑005, FR‑011, SC‑004)
- Permutation importance yields feature importance scores and raw p‑values.
- Benjamini-Hochberg FDR correction (q ≤ 0.05) is applied across the three target‑specific tests.
- **Scope Note**: If synthetic data is used, p-values are reported as measures of "Internal Consistency" only.
- Adjusted p‑values are stored in the model output schema.

### 5. Hold‑out Validation (FR‑012)
- **Real defect data**: a distinct random‑seed split (different from the main train/test split) creates an independent hold‑out set.
- **Synthetic data**: Use the **Hold-Out Mode** (Distinct Physics Engine: Analytical vs. Surrogate) generated in Step 1.2.
  - **Training Set**: Generated via **Analytical Continuum Mechanics** (closed-form equations).
  - **Hold-Out Set**: Generated via a **Gaussian Process Surrogate** trained on a separate public DFT dataset (or distinct analytical parameters) to emulate a "Distinct Physics Engine".
- Model performance on this hold‑out is reported alongside the test‑set metrics.

### 6. Confounding Control (FR‑013)
- If `synthesis_method` or `grain_size` columns exist, CV folds are stratified by these variables.
- If absent, the variables are omitted from stratification; any available confounders are included as covariates in the model. No synthetic strata are created.

### 7. Sensitivity Analysis (FR‑007, SC‑005)
- Decision cutoffs (e.g., defect density thresholds) are swept over deciles of the observed density distribution.
- For each threshold, False Positive Rate (FPR) and False Negative Rate (FNR) are computed and tabulated.

### 8. External Validation & Validation Report (FR‑009, SC‑007)
- The pipeline attempts to locate an external validation set (experimental measurements or a distinct DFT dataset).
- **Logic**:
  - If **Synthetic Data** was used as the primary source: Generate `Validation_Report.json` with `status: "SYNTHETIC_FALLBACK"`.
  - If **Real Data** was used but no external validation exists: Generate `Validation_Report.json` with `status: "NO_EXTERNAL_DATA"`.
- If an external set is found, the model is evaluated on it and the results are recorded.

## Scope of Inference & Interpretation Limits

- **Real Data Mode**: Results are framed as **Associational** claims about the influence of defects on material properties. Causal claims are not made without randomization.
- **Synthetic Mode**: Results are framed as **Method Validation**. P-values indicate internal consistency of the generator, not external validity. No claims about real-world defect influence are made.
- **Collinearity**: High VIF indicates coupled geometric descriptors. Ridge regression is used to stabilize estimates; exclusion is a last resort.

## Compute Feasibility & Rationale
- Random Forests are CPU‑efficient for ≤ 10 k rows; permutation testing is parallelized across available cores but limited to a computationally feasible number of permutations to stay within the 6‑hour CI window.
- Synthetic data generation uses deterministic physics‑based formulas and a fixed random seed, ensuring reproducibility without GPU resources.

## Assumptions & Risks
- The 2022 supplementary dataset is assumed to contain all required columns; if not, synthetic fallback guarantees workflow continuity.
- All statistical conclusions are explicitly **associational**; no causal claims are made.
- Synthetic physics models are based on peer‑reviewed literature (citations provided in `research.md`).
- **Scope Limitation**: If synthetic data is used, the project goal is "Method Validation" and "Associational Claims within the Synthetic Domain" only.
