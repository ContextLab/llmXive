# Data Model: Exploring the Relationship Between Prime Gaps and the Riemann Hypothesis

## Overview

This document defines the data structures, schemas, and flow for the project. All data is stored in local CSV/JSON files or in-memory NumPy arrays during processing.

## Entities

### 1. PrimeGap
Represents a single gap between consecutive primes.

| Attribute | Type | Description |
| :--- | :--- | :--- |
| `prime_before` | int | The prime number $p_n$. |
| `prime_after` | int | The next prime $p_{n+1}$. |
| `gap_size` | int | $p_{n+1} - p_n$. |
| `normalized_gap` | float | $g_n / (\log p_n)^2$. |
| `window_id` | int | Identifier for the sliding window this gap belongs to. |

### 2. ZetaZero
Represents a non-trivial zero of the Riemann zeta function.

| Attribute | Type | Description |
| :--- | :--- | :--- |
| `index` | int | The $n$-th zero. |
| `imaginary_part` | float | The value $\gamma_n$. |
| `spacing` | float | $\gamma_{n+1} - \gamma_n$. |
| `normalized_spacing` | float | Spacing normalized by mean spacing at $\gamma_n$. |

### 3. WindowStats
Aggregated statistics for a sliding window.

| Attribute | Type | Description |
| :--- | :--- | :--- |
| `window_id` | int | Unique window identifier. |
| `window_start` | int | Start of the interval (prime index). |
| `window_end` | int | End of the interval. |
| `max_gap` | float | Maximum normalized gap in the window. |
| `mean_spacing` | float | Mean zero spacing (if mapped). |
| `ks_statistic` | float | KS statistic comparing this window's gaps to GUE (if applicable). |

## Data Flow

1.  **Raw Generation/Ingestion**:
    *   `primes_gaps.csv`: Generated by `generate_primes.py`.
    *   `zeta_zeros.csv`: Ingested by `ingest_zeros.py`.
2.  **Preprocessing**:
    *   `normalized_gaps.csv`: Output of `preprocess.py` (calculates `normalized_gap`).
    *   `normalized_spacings.csv`: Output of `preprocess.py` (calculates `normalized_spacing`).
3.  **Analysis**:
    *   `window_maxima.csv`: Aggregated max gaps per window.
    *   `correlation_results.json`: Final statistical outputs (KS stat, p-value).
    *   `null_distribution.json`: Results from Cramér model simulation.

## Storage Constraints

-   **Memory**: Data is processed in chunks. The `primes_gaps.csv` file may be large (hundreds of millions of rows) but is never loaded entirely into RAM.
-   **Disk**: All raw and processed files are stored in `data/`. Checksums are maintained in `state/`.
