# Data Model: Predicting Molecular Packing Efficiency in Crystals from SMILES Representations

## Overview
The pipeline operates on a set of well‑defined data artifacts. Below are the canonical schemas for each major entity. All schemas are expressed in JSON‑compatible terms; the corresponding YAML contracts (see `contracts/`) enforce them at runtime.

---

### 1. `DatasetEntry` (row in `dataset.csv`)

| Field | Type | Description | Constraints |
|-------|------|-------------|-------------|
| `cod_id` | string | Unique COD identifier (e.g., `COD_1234567`). | Non‑empty, matches regex `^COD_\\d+$`. |
| `smiles` | string | Canonical SMILES generated by RDKit. | Valid SMILES (RDKit parsable). |
| `unit_cell_volume` | float | Unit‑cell volume in Å³. | > 0. |
| `vdw_volume_sum` | float | Sum of atomic van der Waals volumes (Å³). |
| `packing_coefficient` | float | `unit_cell_volume / vdw_volume_sum`. | 0 < value ≤ 1. |
| `fingerprint` | list[float] | Fixed‑length fingerprint vector from the frozen SMILES‑Transformer. | Length = 768 (as defined in contract). |
| `provenance` | object | Metadata about source and processing steps. | Contains `source_url` (string), `timestamp` (ISO‑8601). |

*All numeric values are stored with double precision.*

---

### 2. `FingerprintVector`

| Field | Type | Description |
|-------|------|-------------|
| `vector` | list[float] | Raw output of the frozen SMILES‑Transformer (mean‑pooled). |
| `dim` | integer | Dimensionality (fixed at the standard embedding dimension). |
| `model_name` | string | Identifier of the pretrained transformer (e.g., `seyonec/PubChem10M_SMILES_BPE_60k`). |
| `source_smiles` | string | SMILES that produced this vector. |

*Used internally; persisted as JSON column in `dataset.csv`.*

---

### 3. `TrainedModel`

| Field | Type | Description |
|-------|------|-------------|
| `architecture` | string | Description of the MLP (hidden size set to a moderate dimension, single output). |
| `input_dim` | integer | Expected fingerprint size (must be 768). |
| `hidden_dim` | integer | Size of the hidden layer (≤ 128 to keep total params ≤ 100k). |
| `state_dict` | binary (torch `.pt` file) | Serialized model weights. |
| `training_metadata` | object | Includes `epochs_trained` (int), `train_loss` (float), `val_loss` (float), `seed` (int). |
| `checksum` | string | SHA‑256 hash of the `.pt` file for reproducibility. |

---

### 4. `Report`

| Field | Type | Description |
|-------|------|-------------|
| `mae` | float | Mean Absolute Error on validation set. |
| `pearson_r` | float | Pearson correlation coefficient. |
| `spearman_rho` | float | Spearman rank correlation. |
| `shapiro_p` | float | Shapiro‑Wilk test p‑value on residuals. |
| `heteroscedasticity` | boolean | True if Breusch‑Pagan test indicates heteroscedasticity (p < 0.05). |
| `perm_test_p` | float | Permutation test p‑value. |
| `sensitivity` | object | Mapping of α thresholds → significance decision (bool). |
| `timestamp` | string (ISO‑8601) | Generation time of the report. |

All fields are written to `report.txt` in a human‑readable key‑value format and also serialized as JSON for contract validation.

## Relationships
- Each `DatasetEntry` **contains** one `FingerprintVector` (stored inline).  
- `TrainedModel` **consumes** the fingerprint vectors of the training split.  
- `Report` **summarizes** predictions made by `TrainedModel` on the validation split of `DatasetEntry`s.

## Versioning & Checksums
- Every artifact (`dataset.csv`, `model.pt`, `report.txt`) is accompanied by a SHA‑256 checksum stored in `state/projects/PROJ-511-...yaml` under `artifact_hashes`.  
- The checksum is recomputed on each CI run; mismatches cause the pipeline to abort, ensuring reproducibility.
