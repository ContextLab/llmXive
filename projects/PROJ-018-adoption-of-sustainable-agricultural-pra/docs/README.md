# PROJ-018: Adoption of Sustainable Agricultural Practices in Low-Income Areas
## through Community Engagement

**Project ID**: PROJ-018
**Status**: Research Implementation
**Python Version**: 3.11+

## Overview

This project implements a reproducible research pipeline to analyze the association between
community-engagement intensity and the adoption of sustainable agricultural practices in
low-income regions. The pipeline acquires survey data, constructs composite indices, fits
logistic regression models with mediation analysis, and generates a comprehensive report.

## Quick Start

```bash
# Install dependencies
pip install -r code/requirements.txt

# Run the pipeline
python code/00_generate_synthetic_data.py
python code/01_download_data.py
python code/02_clean_data.py
python code/03_engineer_features.py
python code/04_model_analysis.py
python code/05_generate_report.py
```

## Data Provenance

### Data Sources

This research pipeline is designed to ingest data from the following real-world sources:

1. **World Bank Living Standards Measurement Study (LSMS)**
 - Primary source for household-level agricultural survey data.
 - Contains variables on farm size, credit access, and practice adoption.
 - Access: Via World Bank API or direct download from the LSMS repository.
 - *Note*: If the API is unavailable, the pipeline falls back to a synthetic data generator
 (`code/00_generate_synthetic_data.py`) to ensure pipeline reproducibility. This limitation
 is documented in `data/metadata.yaml`.

2. **FAO Famine Early Warning Systems Network (FEWS NET) / FIES**
 - Used for food security and engagement proxy variables.
 - Provides regional-level engagement metrics (extension services, collective action).

3. **Synthetic Data Fallback**
 - Generated by `code/00_generate_synthetic_data.py` when real APIs are unreachable.
 - Simulates a dataset conforming to the schema defined in `specs/018-adoption-sustainable-agriculture/contracts/dataset.schema.yaml`.
 - Ensures the pipeline can be tested and validated without external dependencies.

### Data Processing Flow

| Step | Script | Input | Output | Description |
|------|--------|-------|--------|-------------|
| 1 | `01_download_data.py` | Raw API / Synthetic | `data/raw/raw_survey_data.csv` | Fetches data, validates schema, logs source metadata. |
| 2 | `02_clean_data.py` | `data/raw/raw_survey_data.csv` | `data/processed/cleaned_data.csv` | Handles missing values, normalizes categorical codes. |
| 3 | `03_engineer_features.py` | `data/processed/cleaned_data.csv` | `data/processed/engineered_data.csv` | Creates `engagement_score` and `adoption_binary`. |
| 4 | `04_model_analysis.py` | `data/processed/engineered_data.csv` | `results/model_results.yaml` | Logistic regression, VIF, mediation, ROC analysis. |
| 5 | `05_generate_report.py` | All processed artifacts | `results/final_report.pdf` | Comprehensive PDF report generation. |

### File Locations

- **Raw Data**: `data/raw/`
- **Processed Data**: `data/processed/`
- **Results**: `results/`
- **Logs**: `modeling_log.yaml` (root of `code/` or project root depending on config)
- **Metadata**: `data/metadata.yaml`
