# Research: Testing Hubble Constant Isotropy Using Pantheon Supernova Sample

## Overview

This research plan outlines the methodology for testing the isotropy of the Hubble constant ($H_0$) using the Pantheon+ Type Ia supernova compilation. The core hypothesis is that if the universe is isotropic, $H_0$ estimates derived from different sky regions (HEALPix pixels) should be consistent within statistical uncertainties.

## Dataset Strategy

### Primary Dataset: Pantheon+
- **Source**: The Pantheon+ sample (Scolnic et al.) is the standard for low-redshift cosmology.
- **Access**: The dataset will be fetched from the verified Zenodo repository.
 - **URL**: ` (Scolnic et al. 2021, "The Pantheon+ Analysis: The Full Data Set and Cosmological Constraints").
 - **Loader**: The implementation will use `pandas.read_csv` to load the `PANTHEON+` sample file.
- **Variables Required**:
 - `SN`: Supernova identifier.
 - `RA`, `Dec`: Celestial coordinates (degrees).
 - `z`: Redshift.
 - `mu`: Distance modulus.
 - `mu_err`: Uncertainty in distance modulus.
 - `flags`: Quality flags (e.g., `IS_HOSTLESS`, `IS_SN`).
- **Verification**: The plan assumes the Zenodo dataset contains all required variables. If `RA`, `Dec`, `z`, or `mu` are missing, the pipeline halts with a specific error message as per the spec assumptions.

### HEALPix Library
- **Source**: `healpy` (Python wrapper for HEALPix).
- **Access**: No external URL required; installed via `pip`.
- **Usage**: Used to map (RA, Dec) to pixel indices (Nside=4, NESTED) and perform spherical harmonic transforms.

### Peculiar Velocity Model
- **Source**: CosmicFlows-3 flow model.
- **Implementation**: The pipeline will use the `pecvel` Python package (or a fallback analytical implementation based on the CosmicFlows-3 velocity field parameters) to calculate radial peculiar velocities based on RA, Dec, and distance. This correction is applied as a redshift perturbation ($z_{obs} \to z_{cos}$) before $H_0$ estimation.

## Methodology

### Phase 1: Data Ingestion and Cleaning
1. **Load**: Read the Pantheon+ CSV from Zenodo. The dataset is loaded into memory as a single DataFrame, which fits comfortably within the available RAM limit.
2. **Filter**:
 - Apply redshift cut: $z < 0.15$ (FR-001).
 - Filter by quality flags (remove outliers, invalid data).
 - Handle missing `RA`, `Dec`, or `z` by dropping rows and logging counts (Edge Case).
3. **Spatial Indexing**:
 - Convert (RA, Dec) to (Theta, Phi).
 - Assign HEALPix pixel index using `healpy.ang2pix(nside=4, nest=True)` (FR-002).
 - Validate that every row has a valid pixel index within the expected range.

### Phase 2: $H_0$ Estimation
1. **Global Estimate**: Fit the luminosity distance model $d_L(z) = \frac{c}{H_0} \int \frac{dz'}{E(z')}$ to the full sample to derive a global $H_0$.
2. **Local Estimates**:
 - Group supernovae by HEALPix pixel.
 - **Peculiar Velocity Correction**: Apply the CosmicFlows-3 correction to the redshifts ($z \to z_{cos}$) for all supernovae. Uncertainty from the velocity model is propagated into the final error budget.
 - **High-N Pixels (N ≥ 30)**: Fit the full non-linear luminosity distance model to derive local $H_0$ (FR-003).
 - **Low-N Pixels (N < 30)**: Instead of skipping, apply a **Hierarchical Bayesian Shrinkage (Empirical Bayes)** estimator. This method borrows strength from the global mean and neighboring pixels to stabilize the estimate, ensuring full-sky coverage for the anisotropy map (Edge Case, FR-003).
 - **Validation**: Validate the *methodology* using a synthetic dataset with a known ground-truth $H_0$ generated by the pipeline itself, rather than comparing the result to a literature value (which may be biased).
3. **Output**: Store estimates in `h0_estimates.parquet` with a `shrinkage_factor` field for low-N pixels.

### Phase 3: Anisotropy Quantification
1. **Spherical Harmonics**:
 - Create a map of $H_0$ deviations ($\Delta H_0 = H_{0, local} - H_{0, global}$).
 - Compute dipole ($\ell=1$) and quadrupole ($\ell=2$) coefficients using `healpy.sphtfunc.map2alm` (FR-005).
2. **Monte Carlo Simulations**:
 - **Null Hypothesis Generation**: To account for the complex selection function and spatial clustering, the simulation will **shuffle supernova labels (distance moduli) among the fixed observed positions (RA/Dec)**. This preserves the exact survey geometry and clustering while randomizing the cosmological signal.
 - **Approximation**: To meet the 6-hour runtime constraint, the Monte Carlo simulations will use a **linearized Hubble diagram approximation** ($H_0 \approx c \cdot z_{cos} / d_L$) for the rapid generation of the null distribution. The full non-linear fit is only applied to the observed data.
 - Generate a sufficient number of realizations to ensure statistical robustness. For each, compute dipole and quadrupole amplitudes.
 - **Null Distribution**: Build the distribution of amplitudes under the null hypothesis (isotropy).
3. **Significance Testing**:
 - Calculate the p-value for the observed dipole/quadrupole amplitudes against the null distribution (SC-003).
 - Apply Benjamini-Hochberg FDR correction (q=0.05) to the joint test of dipole and quadrupole (FR-006, SC-005).

### Phase 4: Sensitivity Analysis
- **Redshift Cut**: Vary the redshift cut threshold ($z < 0.10, 0.15, 0.20$) and re-run the analysis to assess stability (FR-007, SC-004).
- **Peculiar Velocity**: Run the full analysis **without** the CosmicFlows-3 correction to quantify the bias introduced by the correction model itself. This isolates the anisotropy signal from the correction assumptions.

## Statistical Rigor & Constraints

- **Multiple Comparisons**: FDR correction is explicitly applied to the dipole and quadrupole tests to control the family-wise error rate.
- **Power Justification**: The sample size ([deferred] SNe) is large, but the effective sample size per pixel at Nside=4 is small. The Empirical Bayes shrinkage and Monte Carlo approach account for this by stabilizing low-N estimates and simulating the exact noise structure.
- **Causal Inference**: The study is observational. Any detected anisotropy will be framed as an **associational** relationship, not a causal effect, per the project assumptions.
- **Measurement Validity**: The Pantheon+ distance modulus is a validated standard for Type Ia SNe.
- **Collinearity**: Predictors (RA, Dec) are spatial coordinates. We do not claim independent effects of RA and Dec; rather, we analyze the spatial field as a whole.
- **Compute Feasibility**:
 - **No GPU**: All operations (fitting, Monte Carlo) are CPU-based.
 - **Memory**: The dataset is small (~200 MB). The Monte Carlo loop will be optimized (vectorized operations) to fit within 7 GB RAM.
 - **Runtime**: [deferred] iterations of a *linearized* regression on [deferred] points is computationally trivial (< 1 hour). The full non-linear fit is only run once on the observed data.

## Decision Rationale

- **Nside=4**: Chosen to ensure at least ~30 SNe per pixel on average. Higher resolution would result in many empty or low-count pixels.
- **Empirical Bayes Shrinkage**: Chosen over "skipping" to ensure full-sky coverage and avoid selection bias in the dipole/quadrupole calculation.
- **Monte Carlo Strategy**: Label shuffling within fixed geometry is chosen to accurately replicate the survey selection function and spatial clustering in the null distribution.
- **Linearized Approximation**: Chosen for Monte Carlo to ensure the analysis completes within the 6-hour CPU constraint while maintaining statistical validity for the null distribution.
