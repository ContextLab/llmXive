# Testing Hubble Constant Isotropy Using Pantheon Supernova Sample  

**Field**: physics  

## Research question  

Does the locally measured Hubble constant (H₀) show statistically significant directional variation across the sky when estimated from the Pantheon+ Type Ia supernova compilation?

## Motivation  

Large‑scale isotropy is a cornerstone of the standard cosmological model. Recent H₀ tension debates have focused on possible systematic biases, yet a direct test of spatial uniformity of H₀ using a homogeneous supernova sample has received limited attention. Demonstrating isotropy (or detecting anisotropy) would either reinforce the ΛCDM assumption of homogeneity or point to new physics or unmodeled systematics affecting local expansion measurements.

- [Testing the Hubble law with Pantheon+ (2022)](https://arxiv.org/abs/2208.07271) — Shows how Pantheon+ can be used to probe the low‑z Hubble law, providing a methodological baseline for fitting H₀ with this dataset.  
- [Anisotropy in Pantheon+ supernovae (2024)](https://arxiv.org/abs/2411.10838) — Directly investigates large‑scale anisotropies in the expansion rate using Pantheon+, offering a recent precedent and comparison point for our HEALPix‑grid approach.

## Expected results  

We expect the distribution of regional H₀ estimates to be consistent with statistical fluctuations around the global mean, yielding a null‑hypothesis p‑value > 0.05. A statistically significant excess of variance (p < 0.01) or a non‑zero low‑ℓ spherical‑harmonic component would constitute evidence for anisotropy, prompting further investigation of systematic or physical origins.

## Methodology sketch  

- **Data acquisition**: Download the Pantheon+ distance‑modulus table (≈ 200 MB) from the official repository (e.g., Zenodo DOI 10.5281/zenodo.XXXXX).  
- **Pre‑processing**: Apply standard quality cuts (e.g., redshift < 0.15, flag = 0) and convert distance moduli to luminosity distances.  
- **Sky partitioning**: Use `healpy` with Nside = 4 (192 pixels) to assign each supernova to a HEALPix pixel based on its celestial coordinates.  
- **Local H₀ estimation**: Within each pixel, perform a linear fit of distance modulus vs. log (z) (Hubble diagram) using ordinary least squares; extract the slope to obtain a local H₀ estimate.  
- **Bootstrap resampling**: For each pixel, resample the supernovae 1 000 times to generate a distribution of H₀ estimates and quantify measurement uncertainty.  
- **Global comparison**: Compute the global H₀ from the full sample using the same linear fit.  
- **Anisotropy metrics**:  
  - Calculate the pixel‑wise deviation ΔH₀ = H₀,local − H₀,global and its variance across the sky.  
  - Perform spherical‑harmonic decomposition of the ΔH₀ map (ℓ = 1–3) with `healpy.anafast` to test for dipole/quadrupole power exceeding that expected from the bootstrap ensembles.  
- **Statistical testing**: Compare observed dipole/quadrupole amplitudes to the null distribution generated by randomizing supernova positions (Monte‑Carlo) to obtain p‑values.  
- **Robustness checks**: Repeat the analysis with alternative HEALPix resolutions (Nside = 2, 8) and with redshift cuts (z < 0.1, z < 0.2) to verify stability of results.  

All steps rely on pure‑Python packages (`numpy`, `pandas`, `scipy`, `healpy`, `matplotlib`) and can be executed within a single GitHub Actions job (< 6 h, < 7 GB RAM).
