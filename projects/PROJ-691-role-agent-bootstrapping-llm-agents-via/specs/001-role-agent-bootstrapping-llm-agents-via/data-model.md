# Data Model: Reproduce & validate: Role-Agent: Bootstrapping LLM Agents via Dual-Role Evolution

## Overview
This document defines the data structures for the reproduction pipeline, focusing on the intermediate logs (`wia_results.json`) and the final report (`aiw_analysis_report.md`).

## Entities

### 1. WIA_Log (JSON)
Generated by the WIA component. Represents a single episode interaction.

- **Structure**:
  - `episode_id`: Integer (unique ID for the episode).
  - `action`: String (the action taken by the agent).
  - `predicted_state`: String (the state predicted by the "Agent" role).
  - `actual_state`: String (the ground truth state from the environment).
  - `alignment_score`: Float (optional, if calculated; otherwise null).
  - `timestamp`: ISO 8601 string.
  - `error`: String (null if successful, error message if failed).

### 2. AIW_Report (Markdown)
Generated by the AIW component. Summarizes the reproduction.

- **Sections**:
  - `sample_size`: Integer (e.g., 5).
  - `total_tasks`: Integer (same as sample_size).
  - `failures_identified`: Integer.
  - `dual_role_observed`: Boolean (True if WIA/AIW flow completed).
  - `limitations`: String (e.g., "Small sample size, CPU-only model, Mechanism Plumbing Verification only").
  - `reproduction_summary`: Text (Qualitative assessment).

## Data Flow

1. **Input**: `ALFWorld` environment (dynamic generation of 5 episodes).
2. **Process**: `WIA` execution -> Generates `WIA_Log` entries.
3. **Process**: `AIW` analysis -> Reads `WIA_Log` -> Generates `AIW_Report`.
