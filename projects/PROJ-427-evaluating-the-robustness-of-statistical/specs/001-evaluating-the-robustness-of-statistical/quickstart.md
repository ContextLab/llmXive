# Quickstart: Evaluating the Robustness of Statistical Methods to Common Data Errors

## Overview

This guide provides step-by-step instructions for setting up the environment, running the simulation, and generating results.

## Prerequisites

-   Python 3.11+
-   Git
-   Access to GitHub Actions (for CI execution) or a local Linux environment with 2+ CPU cores and 7GB+ RAM.

## Installation

1.  **Clone the repository**:
    ```bash
    git clone <repo-url>
    cd projects/PROJ-427-evaluating-the-robustness-of-statistical
    ```

2.  **Create a virtual environment**:
    ```bash
    python -m venv venv
    source venv/bin/activate  # On Windows: venv\Scripts\activate
    ```

3.  **Install dependencies**:
    ```bash
    pip install -r requirements.txt
    ```

## Running the Simulation

### 1. Download Datasets
Download verified datasets to `data/raw/`:
```bash
python code/download_data.py
```
*This script fetches data from the verified URLs and records checksums.*

### 2. Generate Synthetic Data
Create null and effect synthetic datasets:
```bash
python code/generate_synthetic.py
```

### 3. Inject Errors
Run the error injection pipeline:
```bash
python code/inject_errors.py --rates 0.01 0.05 0.10 0.20
```

### 4. Execute Statistical Tests
Run the full simulation suite:
```bash
python code/run_tests.py
```

### 5. Aggregate Results and Generate Plots
Compute metrics and create degradation curves:
```bash
python code/aggregate_results.py
```

## Expected Outputs

-   `data/processed/`: Error-injected datasets.
-   `results/metrics/`: JSON/CSV files with test results.
-   `results/plots/`: PNG files showing degradation curves.
-   `report.md`: Final summary report (generated by `aggregate_results.py`).

## Verification

To verify the simulation:
1.  Check that `results/metrics/` contains files for all error types and rates.
2.  Ensure `results/plots/` contains degradation curves for each test type.
3.  Confirm that Type I error rates for the clean dataset are close to 0.05 (within sampling error).

## Troubleshooting

-   **Memory Error**: Reduce the number of simulation iterations in `config.yaml`.
-   **Dataset Not Found**: Ensure the verified URLs are accessible and the `download_data.py` script has network access.
-   **Import Error**: Verify that all dependencies are installed in the virtual environment.
