# Data Model: Reproduce & Validate Domino Speculative Decoding Framework

## 1. Overview
This document defines the data structures for the benchmark execution, metrics collection, and validation reporting. The data flow is: `Prompt Input` -> `Benchmark Execution` (n=5 runs) -> `Raw Metrics` -> `Structured JSON (Mean/Std)` -> `Validation Report`.

## 2. Input Data
### Prompts
*   **Source**: Subset of `cnn_dailymail` (Verified Source).
*   **Format**: List of strings.
*   **Constraint**: Max 512 tokens per prompt. Filtered in `runner.py`.
*   **Count**: 5 prompts (for 5 runs each = 25 total executions).

## 3. Output Data Structures

### 3.1 Benchmark Metrics (JSON)
Generated by the benchmark script. Contains performance data aggregated over n=5 runs.

| Field | Type | Description | Source |
|-------|------|-------------|--------|
| `model_name` | string | Name of the model used (e.g., "Qwen2-1.8B") | Config |
| `hardware` | object | Hardware details (CPU cores, RAM) | System |
| `baseline_latency_ms` | object | Aggregated baseline latency (mean, std, min, max) | Benchmark (n=5) |
| `domino_latency_ms` | object | Aggregated Domino latency (mean, std, min, max) | Benchmark (n=5) |
| `tokens_per_second_baseline` | object | Aggregated baseline throughput (mean, std) | Benchmark (n=5) |
| `tokens_per_second_domino` | object | Aggregated Domino throughput (mean, std) | Benchmark (n=5) |
| `speedup_ratio` | object | Aggregated speedup (mean, std, min, max) | Calculated |
| `acceptance_rate` | object | Fraction of draft tokens accepted (mean, std, min, max) | Benchmark (n=5) |
| `total_tokens` | int | Total tokens generated | Benchmark |
| `prompt_count` | int | Number of prompts processed | Config |
| `run_count` | int | Number of independent runs (n=5) | Config |
| `timestamp` | string | ISO 8601 timestamp | System |
| `status` | string | "success" or "failed" | System |
| `error_message` | string | Error details if failed | System |

### 3.2 Validation Report (JSON)
Generated by the comparison logic.

| Field | Type | Description |
|-------|------|-------------|
| `feature_id` | string | "001-reproduce-domino-speculative-decoding" |
| `paper_claim_speedup` | float | 5.49 (from spec) |
| `reproduced_speedup_mean` | float | Mean speedup from runs |
| `reproduced_speedup_std` | float | Std dev of speedup |
| `acceptance_rate_mean` | float | Mean acceptance rate |
| `hardware_context` | string | "2 vCPU, 7GB RAM" |
| `model_substitution` | string | "Qwen3 -> Qwen2-1.8B" |
| `pass_status` | string | "PASS" (if mechanism validated) or "FAIL" |
| `statistical_significance` | string | "p < 0.05" or "p >= 0.05" |
| `tolerance_note` | string | Explanation of CPU vs GPU discrepancy |
| `spec_conflict_flag` | boolean | True if FR-007/SC-003 were executed but flagged as invalid |
