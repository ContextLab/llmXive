## Summary

This project implements a CPU-tractable pipeline to evaluate the stability of the AnyFlow video diffusion model under temporal discontinuities (scene cuts). The core hypothesis is that the "flow-map divergence" (L2 distance between the model's predicted latent trajectory and a high-precision numerical baseline) correlates with semantic temporal discontinuity. The plan executes three phases: (1) Data Curation and Ground-Truth Annotation (FR-001, FR-002), (2) CPU-Tractable Latent Divergence Calculation (FR-003, FR-004), and (3) Correlation Analysis and Threshold Sensitivity (FR-005, FR-006, FR-007). All methods are constrained to run on GitHub Actions free-tier runners (A limited number of CPU cores, 7GB RAM, 6h limit) without GPU acceleration.

## Complexity Tracking

| Violation | Why Needed | Simpler Alternative Rejected Because |
| :--- | :--- | :--- |
| **Euler Baseline (N=500)** | Required by FR-004 to establish a high-fidelity numerical baseline for divergence. Default set to N=500 to ensure feasibility within 6h. | Using N=50 or N=100 would not provide sufficient resolution to detect subtle model instability. N=5000 is rejected as infeasible on CPU. |
| **ONNX Runtime** | Required by FR-003 to run the frozen AnyFlow model on CPU without CUDA dependencies. | Standard PyTorch CPU inference is slower and may not be optimized for the specific graph structure required for the 16-frame batch. |
| **Stratified Sampling (% cuts)** | Required by FR-001 and FR-010. The strategy ensures not just the presence of cuts, but a **sufficient distribution of scores** (0.0 to 1.0) to guarantee `variance >= 0.05`. It targets "hard" cuts (score ~high) and "smooth" motion (score ~ qualitative level) to maximize score variance. | Random sampling might result in <5% cuts or a narrow score range, leading to a bimodal or skewed distribution that fails the variance check (FR-010). |
| **Real Measurements Only** | All metrics are computed from real video clips and real model inference. No synthetic data, placeholder metrics, or simulated scores are generated or used. | Using synthetic or hardcoded metrics would violate the "Verified Accuracy" and "Data Hygiene" principles, rendering the study invalid. |
| **Continuous Scoring (-1.0)** | Required to capture the *degree* of discontinuity (e.g., fast motion vs. hard cut). A binary metric would lose the monotonic trend information needed for Pearson/Spearman correlation. | A binary classification (cut/no-cut) would only yield an AUC-ROC, failing to test the hypothesis that *instability scales with discontinuity severity*. |

## Data Flow & Annotation Clarification

1.  **Ingestion**: `download.py` creates `clip_metadata.csv`.
2.  **Annotation Template**: `code/annotate.py` generates the empty `data/annotations/continuity_scores.csv` with correct video IDs and schema.
3.  **Manual Scoring**: A human annotator reviews clips (using a standard player) and fills the CSV. **No model features are used.**
4.  **Validation**: `code/annotate.py` (run again) validates the filled CSV against the schema and FR-002 constraints (range [0.0, 1.0]).
5.  **Inference**: `inference.py` creates `divergence_metrics.csv`.
6.  **Analysis**: `analysis.py` creates `correlation_results.json`, `sensitivity_report.csv`, `variance_report.csv`.
