## Summary

This project implements a computational study to investigate how the quantity of teacher-provided prompt demonstrations influences the sample-efficiency of small language models (SLMs) trained with Proximal Policy Optimization (PPO). The system will execute PPO training loops under four discrete conditions (0k, 10k, 50k, 200k prompts) using the OpenAssistant dataset as a *prompt buffer* for rollouts. The study treats the benchmarks (lambada_openai, truthful_qa, mmlu) as independent *transfer tasks* to measure generalization, avoiding tautological validation. The implementation strictly adheres to GitHub Actions free-tier constraints (limited CPU, 7GB RAM, 6h limit) by utilizing CPU-tractable models (GPT-NeoX-125M), dynamic step adjustment to handle time limits, and **non-parametric statistical tests** (Kendall's Tau, Kruskal-Wallis) to detect monotonic trends and group differences in performance, acknowledging the limited sample size (N=12). The output is a structured dataset of performance curves enabling post-hoc analysis of non-linear trends.

**Testing**: `pytest` (unit tests for reward logic, integration tests for training loop flow), contract tests against YAML schemas.  
**Target Platform**: Linux (GitHub Actions free-tier runner).  
**Project Type**: Computational Research / CLI Tool.  
**Performance Goals**: Complete one training run (up to 1000 steps or 6h) with memory footprint < 7GB; benchmark evaluation < 30 mins per suite.  
**Constraints**: No GPU/CUDA; strict 6-hour wall-clock limit; no external API calls for training data; reproducibility via pinned seeds.  
**Scale/Scope**: 4 prompt-size conditions, 3 seeds each, 3 benchmarks, max 1000 PPO steps per run (dynamic adjustment for time limits).
