# Data Model: OpenThoughts-Agent Data Pipeline Validation & Reproduction

## Overview

This document defines the data structures used in the OpenThoughts-Agent data pipeline validation. The primary entities are `TaskTrajectory`, `DatasetManifest`, and `ReproductionReport`.

## Entity Definitions

### TaskTrajectory

Represents a single agentic task instance generated by the pipeline.

**Fields**:
- `input`: `string` - The input prompt for the task.
- `output`: `string` - The final output generated by the agent.
- `thought_process`: `string` - The chain of thought or reasoning steps taken by the agent.
- `task_id`: `string` - Unique identifier for the task.
- `timestamp`: `datetime` - Timestamp of generation.

**Constraints**:
- `input`, `output`, `thought_process` must be non-empty strings.
- `task_id` must be unique within the dataset.
- `timestamp` must be in ISO 8601 format.

### DatasetManifest

Metadata file tracking all generated task trajectories.

**Fields**:
- `generated_at`: `datetime` - Timestamp of manifest creation.
- `task_count`: `integer` - Total number of tasks generated.
- `tasks`: `array` - List of task metadata (task_id, file_path, size_bytes, validation_status).
- `validation_status`: `string` - Overall status (PASS/FAIL).

**Constraints**:
- `task_count` must match the number of entries in `tasks`.
- Each task in `tasks` must have a valid `file_path` and `validation_status`.

### ReproductionReport

Human-readable summary of the validation results.

**Fields**:
- `execution_status`: `string` - Overall status (PASS/FAIL).
- `artifact_counts`: `object` - Counts of generated files.
- `discrepancies`: `array` - List of discrepancies found.
- `execution_time`: `duration` - Time taken for generation.

**Constraints**:
- `discrepancies` must be non-empty if `execution_status` is FAIL.
- `execution_time` must be a valid duration string.

## Data Flow

1. **Input**: `--task-limit` parameter (e.g., 10).
2. **Process**:
   - Fetch task definitions from API (simulated or real).
   - Generate `TaskTrajectory` objects.
   - Write to JSON/Parquet files in `data/generated/`.
3. **Output**:
   - `DatasetManifest` in `data/`.
   - `ReproductionReport` in `results/`.

## Storage Format

- **Task Trajectories**: JSON files (`task_<id>.json`) or Parquet files (`tasks.parquet`).
- **Manifest**: JSON file (`manifest.json`).
- **Report**: Markdown file (`reproduction_report.md`).

## Validation Rules

- All `TaskTrajectory` fields must be present and non-empty.
- `DatasetManifest` must accurately reflect the number of generated files.
- `ReproductionReport` must include a table of artifact sizes and status flags.
