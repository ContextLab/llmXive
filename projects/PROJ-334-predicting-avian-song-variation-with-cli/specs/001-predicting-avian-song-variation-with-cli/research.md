# Research: Predicting Avian Song Variation with Climatic and Geographic Factors

## Research Question
Does variation in avian song metrics (frequency, duration) correlate with climatic (temperature, precipitation) and geographic (elevation, latitude, longitude) factors, and is this relationship robust to changes in statistical significance thresholds?

## Dataset Strategy

### Source Verification
The project uses two primary, verified real-world datasets: `SongRecord` (avian acoustic metadata) and `ClimateSnapshot` (environmental data).

| Dataset | Required Variables | Verified Source | Strategy |
| :--- | :--- | :--- | :--- |
| **SongRecord** | `species_id`, `song_metric_1`, `song_metric_2`, `location_lat`, `location_lon` | **Xeno-Canto** (via `pyxeno` or `requests` API) | Fetch metadata for bird recordings. Filter for species with available audio and GPS coordinates. |
| **ClimateSnapshot** | `location_lat`, `location_lon`, `temperature`, `precipitation`, `elevation` | **WorldClim v2.1** (30-arc-sec resolution, via `geopandas`/`rasterio`) | Extract climate values at song recording coordinates. |
| **AnalysisDataset** | Merged schema | N/A | Generated by joining SongRecord and ClimateSnapshot. |

**Note**: Both datasets are publicly accessible and verified. The pipeline fetches them directly. If fetch fails, the pipeline aborts with a clear error.

### Data Alignment Logic
1.  **Reprojection**: All coordinates converted to WGS84 (EPSG:4326).
2.  **Spatial Extraction**: Instead of a radius join, the system performs **Nearest Grid Cell Extraction** from the WorldClim v2.1 raster (30-arc-sec, ~1km).
    -   **Method**: **Bilinear Interpolation** is used to estimate values at the exact song coordinate using the four surrounding grid cells.
    -   **Fallback**: If bilinear interpolation fails (e.g., missing neighbors, ocean masks, or grid edge), the system falls back to **Nearest Neighbor** extraction (value of the single closest grid cell) to ensure no NaNs are introduced.
3.  **Join Key**: `species_id` (exact match) + `spatial extraction` (lat/lon).
4.  **Handling Mismatches**: Species with no climate match are excluded; a warning log is generated.
5.  **Missing Values**: Climate gaps handled by the interpolation fallback; if still missing, imputed via median of nearest neighbors or excluded.

## Statistical Methodology

### Model Specification
-   **Type**: Multiple Linear Regression (OLS).
-   **Outcomes**: `song_metric_1`, `song_metric_2` (separate models).
-   **Predictors**: `temperature`, `precipitation`, `elevation`, `latitude`, `longitude`.
-   **Assumption**: Observational study. **No causal claims** will be made. Results are associational only.
-   **Model Strategy**:
    -   **Model A (Primary)**: Predictors = `temperature`, `precipitation`. (Climate Only). Used to isolate climate effects.
    -   **Model B (Exploratory)**: Predictors = `temperature`, `precipitation`, `elevation`, `latitude`, `longitude`. (Climate + Geo). Used for sensitivity analysis.
    -   **Default**: Report **Model A** results as the primary hypothesis test.
    -   **VIF Fallback**: If Model B shows VIF > 5 for any predictor, report Model A results and describe geographic confounders descriptively without claiming independent effects.

### Rigor & Diagnostics
1.  **Multicollinearity**: Variance Inflation Factor (VIF) calculated for all predictors in Model B.
    -   **Threshold**: VIF > 5 flags a pair for descriptive joint interpretation.
    -   **Action**: If VIF > 5, report Model A results; do not claim independent effects for collinear terms in Model B.
2.  **Multiple Comparisons**: Benjamini-Hochberg (FDR) procedure applied **per-outcome** (separately for frequency and duration) across the set of predictors to control False Discovery Rate (FR-006).
3.  **Sensitivity Analysis**:
    -   Sweep p-value thresholds across a range of conventional significance levels.
    -   Metric: Change in number of significant predictors and model R².
    -   Stability: Jaccard index of significant predictors across thresholds.
4.  **Power & Sample Size**:
    -   **MDES**: Minimum Detectable Effect Size (Cohen's f²) calculated based on available N and alpha=0.05.
    -   **Reporting**: If achieved power < 0.80, results are flagged as "preliminary" and the MDES is reported. No claim of "no effect" is made if power is insufficient.

### Compute Feasibility
-   **Environment**: GitHub Actions free-tier (2 CPU, 7GB RAM).
-   **Strategy**:
    -   Data subset to a manageable size if total size exceeds RAM (after join/filter).
    -   Use `statsmodels` (CPU-optimized) for OLS.
    -   No GPU/CUDA; no deep learning.
    -   Runtime target: < 1 hour.

## Decision Rationale
-   **Why Model A/B Split?** To disentangle climate effects from geographic confounders. Model A isolates climate; Model B tests robustness.
-   **Why Bilinear + Fallback?** Maximizes accuracy while ensuring robustness against grid edge cases (prevents NaNs).
-   **Why FDR per-outcome?** Controls false discoveries for the specific hypothesis 'Does predictor X affect outcome Y?' without over-correcting for the entire family of all tests.
-   **Why No Causal Claims?** The study is observational (no randomization). The constitution and spec require treating relationships as associational.

## Limitations
-   **Collinearity**: Geographic variables (lat, lon, elevation) are often correlated. VIF diagnostics will flag these, but interpretation must remain descriptive.
-   **Causality**: Cannot infer that climate *causes* song variation; only association can be reported.
-   **Data Quality**: Relies on the accuracy of GPS metadata in Xeno-Canto and the resolution of WorldClim.
