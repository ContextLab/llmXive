# Data Model: Figure ↔ Data Binding

This document defines the binding between the paper's required figures and the specific data artifacts generated by the research pipeline. Every figure in the paper MUST be generated from these exact files.

## 1. Figure 1: The SAA Calculation Workflow
**Type**: Flowchart / Diagram
**Source**: Logic in `eval/saa_scoring.py` and `eval/metrics.py`.
**Data Binding**: No external data file. This figure is a **logic diagram** derived from the code structure.
**Generation Script**: `scripts/plot_workflow.py` (generates a Mermaid or TikZ representation of the decision tree).
**Mandatory Content**:
- Nodes: `Prediction`, `Answer Correct?`, `Region Correct (IoU >= 0.5)?`, `SAA Score`.
- Branches: Explicitly label "Answer Correct/Region Wrong" as **Attribution Hallucination**.
- Legend: Must define SAA, IoU, and Attribution Hallucination without referencing the Methods section.

## 2. Figure 2: Error Distribution Bar Chart
**Type**: Bar Chart
**Source**: `outputs/evaluation_report.json`
**Data Binding**:
```json
{
 "figure_id": "fig_error_distribution",
 "input_file": "outputs/evaluation_report.json",
 "fields": [
 "both_correct",
 "answer_only_correct",
 "region_only_correct",
 "both_wrong"
 ]
}
```
**Generation Script**: `scripts/plot_error_distribution.py` (uses `seaborn.barplot`).
**Mandatory Caption Requirement**: Must explicitly state that the `region_only_correct` bar represents the rate of Attribution Hallucination.

## 3. Figure 3: Dataset Integrity & Skipped Records
**Type**: Stacked Bar Chart
**Source**: `outputs/validation_log.json`
**Data Binding**:
```json
{
 "figure_id": "fig_data_integrity",
 "input_file": "outputs/validation_log.json",
 "fields": [
 "skipped_reasons" // Object mapping reason string to count
 ]
}
```
**Generation Script**: `scripts/plot_integrity.py` (uses `matplotlib.bar` stacked).
**Mandatory Content**: Must show the `skipped_count` total and the breakdown by reason (e.g., "missing_bbox", "missing_image").

## 4. Data Artifacts Schema (Source of Truth)

### `outputs/infer_results.jsonl`
Each line represents a single inference result.
```json
{
 "question_id": "string",
 "question": "string",
 "predicted_answer": "string",
 "predicted_bbox": [float, float, float, float], // [x_min, y_min, x_max, y_max]
 "ground_truth_answer": "string",
 "ground_truth_bbox": [float, float, float, float]
}
```

### `outputs/evaluation_report.json`
The primary metric source for the paper.
```json
{
 "saa_score": float,
 "standard_accuracy": float,
 "attribution_hallucination_rate": float,
 "total_samples": int,
 "skipped_count": int,
 "error_breakdown": {
 "both_correct": int,
 "answer_only_correct": int,
 "region_only_correct": int,
 "both_wrong": int
 },
 "memory_peak_gb": float,
 "execution_time_seconds": float
}
```

### `outputs/validation_log.json`
The integrity source for Figure 3.
```json
{
 "total_records_checked": int,
 "skipped_count": int,
 "skipped_records": [
 {
 "record_id": "string",
 "reason": "missing_bbox | missing_image | ambiguous_bbox"
 }
 ]
}
```
