# Free-form review body

## Strengths
- **Novel Benchmark**: CiteVQA addresses a critical gap in Doc-VQA evaluation by requiring element-level evidence attribution, moving beyond answer-only metrics.
- **Comprehensive Evaluation**: The audit of 20 MLLMs across different tiers (closed/open, large/small) provides a broad view of the current landscape.
- **Clear Phenomenon**: The identification of "Attribution Hallucination" is a significant finding that highlights a specific failure mode in MLLMs.
- **Automation Pipeline**: The automated annotation pipeline is scalable and well-documented, enabling the creation of a large-scale dataset.

## Concerns
- **Ground Truth Validity**: The ground truth citations are generated by an automated pipeline and validated via expert review on only 200 samples (~10% of the dataset). This raises concerns about the reliability of the "Crucial Evidence" labels used for SAA calculation. The prior review `paper_reviewer_scientific_evidence` flagged this as a `full_revision` issue.
- **Model Comparison Confounds**: The evaluation compares closed-source API models (e.g., Gemini, GPT) with open-source local models (e.g., Qwen, Gemma). Differences in input processing (native File API vs. 150 DPI screenshots), resolution scaling, and context window handling introduce confounding variables that may skew the "Performance Disparity" claims.
- **Future-Dated Citations**: The bibliography contains numerous 2025/2026 citations (e.g., `gemini-3.1`, `gpt-5.4`), which, while consistent with the arXiv ID (2605), requires careful handling to avoid confusion regarding the actual availability of these models at the time of the study.

## Recommendation
The paper introduces a valuable benchmark but requires significant scientific validation to support its central claims regarding model performance and attribution hallucinations. The automated ground truth generation needs stronger empirical support (e.g., larger expert validation set, inter-annotator agreement metrics) to ensure the SAA metric is measuring true attribution rather than annotation noise. Additionally, the model comparison methodology needs to be controlled for input processing differences to ensure fair benchmarking. These issues necessitate re-running parts of the research pipeline to strengthen the scientific validity.
