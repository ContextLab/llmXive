# Data Model: CiteVQA Reproduction & Validation

## Entities

### 1. InferenceResult
Represents a single prediction generated by the MLLM.

- `id`: Unique identifier for the sample (string).
- `question`: The input question string.
- `predicted_answer`: The text generated by the model.
- `predicted_bbox`: A list of four floats `[x1, y1, x2, y2]` representing the predicted bounding box (normalized to [0, 1]).
- `image_path`: Path to the image used.
- `inference_time_ms`: Time taken for inference (optional, for diagnostics).

### 2. GroundTruth
Represents the reference data from the dataset.

- `id`: Unique identifier for the sample (string).
- `question`: The ground-truth question string.
- `answer`: The ground-truth answer string.
- `ground_truth_bbox`: A list of four floats `[x1, y1, x2, y2]` representing the correct region (normalized to [0, 1]).
- `image_path`: Path to the image used.

### 3. ValidationLog
Intermediate log of skipped records during dataset validation.

- `skipped_count`: Number of samples skipped due to missing data (integer).
- `skipped_ids`: List of IDs for skipped samples (list of strings).
- `reason`: Reason for skipping (e.g., "missing_bbox", "missing_image") (string).
- `timestamp`: ISO 8601 timestamp of the validation run.

### 4. EvaluationReport
Aggregated metrics from the evaluation pipeline.

- `saa_score`: The Strict Attributed Accuracy score (float, 0.0 to 1.0).
- `total_samples`: Total number of samples processed (integer).
- `correct_count`: Number of samples where both answer and bbox were correct (integer).
- `answer_only_correct`: Number of samples where answer was correct but bbox was wrong (Attribution Hallucination) (integer).
- `bbox_only_correct`: Number of samples where bbox was correct but answer was wrong (integer).
- `both_wrong`: Number of samples where both were wrong (integer).
- `skipped_count`: Number of samples skipped due to missing data (integer).
- `timestamp`: ISO 8601 timestamp of the report generation.

## Data Flow

1.  **Input**: `GroundTruth` records from `external/CiteVQA` dataset.
2.  **Validation**: `data/validate_dataset.py` checks fields and generates `ValidationLog` (`outputs/validation_log.json`).
3.  **Processing**: `infer/run.py` generates `InferenceResult` records for valid samples.
4.  **Evaluation**: `eval/run.py` compares `InferenceResult` against `GroundTruth`, aggregating `skipped_count` from `ValidationLog`.
5.  **Output**: `EvaluationReport` JSON artifact.

## Constraints

- **Bounding Box Format**: All bounding boxes must be normalized to the [0, 1] range relative to image dimensions before IoU calculation.
- **Missing Data**: If `ground_truth_bbox` is missing in the input, the record is skipped, logged in `ValidationLog`, and `skipped_count` is incremented.
- **Memory**: The dataset must be processed in a streaming or batched manner to stay within 7 GB RAM.
