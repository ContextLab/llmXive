# NatureBench: Can Coding Agents Match the Published SOTA of Nature-Family Papers?

**Field**: Computational Linguistics / AI Evaluation

## Research question

How does the "abstraction distance" between a scientific paper's textual method description and standard library implementations predict the failure modes of automated code reconstruction in cross-disciplinary scientific domains?

## Motivation

Current benchmarks for AI coding agents focus heavily on syntax and standard library usage, failing to capture the specific challenge of translating non-standard, text-only methodological descriptions into working scientific code. Understanding how the gap between abstract textual descriptions and concrete implementation (abstraction distance) drives agent failure is critical for determining whether AI can evolve from a code assistant to a genuine research collaborator capable of reproducing complex scientific results.

## Related work

- [NatureBench: Can Coding Agents Match the Published SOTA of Nature-Family Papers? (2026)](https://arxiv.org/abs/2606.24530) — Introduces the NatureBench dataset of 90 cross-disciplinary tasks and reports that agents struggle primarily with "wrong method selection" rather than syntax errors, providing the foundational dataset for this analysis.
- [A Comprehensive Review of State-of-The-Art Methods for Java Code Generation from Natural Language Text (2023)](https://arxiv.org/abs/2306.06371) — Establishes baseline capabilities of LLMs in generating code from natural language for standard programming tasks, serving as a contrast to the domain-specific complexity of scientific modeling.

## Expected results

We expect to find a strong positive correlation between the "abstraction distance" (measured by the deviation of the required algorithm from standard library patterns) and the rate of "wrong method choice" failures. A null result would indicate that agent failures are driven by other factors (e.g., context window limits or data preprocessing errors) rather than the conceptual difficulty of the method translation, which would fundamentally alter the direction of future agent training.

## Methodology sketch

- **Data Acquisition**: Download the NatureBench dataset (90 tasks) and associated ground-truth SOTA values from the official HuggingFace repository linked in the target paper.
- **Abstraction Distance Metric**: Develop a scoring rubric where domain experts rate each task's method description on a scale of 1–5 based on the frequency of standard library functions required versus the need for novel algorithmic derivation.
- **Agent Execution**: Deploy a selection of 3 open-source coding agents (e.g., SWE-agent, OpenHands) to attempt the 90 tasks, enforcing a strict 4-hour wall-clock timeout per task to match the original study's budget, ensuring all runs fit within 6-hour GitHub Actions limits (7GB RAM, 2 CPU).
- **Result Aggregation**: Parse agent output logs to extract final numerical results and compare them against ground-truth SOTA values to calculate the relative error gap ($g$) for each task.
- **Error Categorization**: Classify all failures into categories: "Syntax Error," "Wrong Method Choice," "Data Mismatch," and "Timeout," specifically isolating the "Wrong Method Choice" subset for correlation analysis.
- **Statistical Analysis**: Apply a Pearson correlation test between the "abstraction distance" scores and the "Wrong Method Choice" failure rates, using a binomial proportion confidence interval (Wilson score) to assess significance per domain cluster.
- **Validation Independence**: Validate the agent's success against the *published numerical result* (an independent measurement from the original paper) rather than against the agent's internal confidence scores, intermediate code correctness, or the textual description itself.
