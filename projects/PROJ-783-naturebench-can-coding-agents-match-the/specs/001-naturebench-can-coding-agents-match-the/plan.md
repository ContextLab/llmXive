# Implementation Plan: NatureBench Abstraction Distance Analysis

**Branch**: `001-naturebench-abstraction-distance` | **Date**: 2026-07-01 | **Spec**: `specs/001-naturebench-abstraction-distance/spec.md`

## Summary

This feature implements a statistical analysis pipeline to determine if the "abstraction distance" (novelty of required methods vs. standard library availability) in scientific paper method descriptions predicts "Wrong Method Choice" failures in automated code reconstruction. The implementation involves downloading the NatureBench dataset (or pivoting to SWE-bench), executing three distinct open-source coding agents, scoring tasks via a validated rubric, and performing Logistic Regression and binomial confidence interval analysis.

**Critical Constraint**: The spec explicitly requires multiple concurrent runners and a paid GitHub Actions plan to meet the 24-hour total runtime. However, the **Compute Feasibility** mandate for this repository restricts execution to **free-tier GitHub Actions (2 CPU, 7GB RAM, no GPU, ≤6h/job)**.
**Resolution**: The plan adapts the spec to run on the free tier by:
1.  **Batched Execution**: Processing 5 tasks per job, running 12 sequential jobs (total 12 * 4h = 48h wall-clock, but within 6h/job limit). This satisfies FR-002 by executing the agents, not simulating them.
2.  **Real Agents**: Executing SWE-agent, OpenHands, and a custom lightweight agent on the full set of tasks. No rule-based proxy is used for agent behavior to ensure stochastic failures.
3.  **Dataset Pivot**: If NatureBench is unverified, the system automatically pivots to the verified SWE-bench dataset, mapping its schema to the required fields.

## Phase Breakdown

### Phase 0: Data Acquisition & Feasibility Check (FR-001)
- **Goal**: Download NatureBench (a comprehensive suite of tasks) or pivot to SWE-bench.
- **Steps**:
  1.  **Verified Accuracy Gate**: Check if NatureBench is in the verified list.
  2.  **If NatureBench exists**: Download raw task files to `data/raw/`.
  3.  **If NatureBench missing**: Pivot to SWE-bench (verified). Map `problem_statement` to `method_description`, `test_output` to `ground_truth_sota`.
  4.  Validate checksums and ensure `ground_truth_sota` (or mapped equivalent) is present for all tasks.
- **Deliverable**: `data/raw/dataset_raw.zip` (NatureBench or SWE-bench) with manifest.

### Phase 1: Agent Execution & Log Parsing (FR-002, FR-003, FR-004)
- **Goal**: Execute three distinct agents and classify failures.
- **Agents**: SWE-agent, OpenHands, and a custom lightweight agent (e.g., `swe-agent-light`).
- **Steps**:
  1.  Initialize agent environments (CPU-only).
  2.  Run agents on tasks in batches of 5 (to fit 6h/job limit).
  3.  Enforce 4-hour timeout per task.
  4.  **Gold Standard Generation**: For each task, generate a "Gold Standard Method List" from the ground truth or known correct solution.
  5.  **Failure Classification**:
      - **Syntax Error**: Invalid code.
      - **Wrong Method Choice**: Valid code, but method used is NOT in Gold Standard.
      - **Data Mismatch**: Valid code, method in Gold Standard, but numerical output error > 0.05.
      - **Timeout**: Execution exceeded time limit.
  6.  Calculate relative error gap ($g$) for valid outputs.
- **Deliverable**: `data/processed/execution_results.json` (list of `ExecutionResult`).

### Phase 2: Abstraction Distance Scoring (FR-005, FR-008)
- **Goal**: Score tasks 1–5 based on "Novelty Index".
- **Steps**:
  1.  **Gold Standard Extraction**: Extract unique function names from the Gold Standard Method List.
  2.  **Registry Comparison**: Compare against a pre-defined "Standard Library Registry" (e.g., `numpy`, `pandas`, `scipy`).
  3.  **Novelty Index**: Calculate `(Unique Functions in Gold Standard - Standard Functions) / Total Unique Functions`.
  4.  **Rubric Application**:
      - **Score 1**: ≥80% Standard (Low Novelty).
 - **Score 2**: 50–[deferred] Standard.
 - **Score 3**: 20–[deferred] Standard.
 - **Score 4**: 1–[deferred] Standard.
      - **Score 5**: ≤19% Standard (High Novelty).
 5. **Phase 2b: Expert Validation**: Select a deferred set of tasks. Have domain experts rate them. Calculate Krippendorff's Alpha.
      - If Alpha < 0.7: Retrain/adjust automated scorer.
      - If Alpha ≥ 0.7: Accept automated scores for all 90 tasks as "Expert Proxy" scores.
- **Deliverable**: `data/processed/abstraction_scores.csv`.

### Phase 3: Statistical Analysis (FR-006, FR-007)
- **Goal**: Logistic Regression and confidence intervals.
- **Steps**:
  1.  Merge execution results with abstraction scores.
  2.  Filter for "Wrong Method Choice" failures (binary).
  3.  **Primary Test**: Logistic Regression to model `P(Wrong Method Choice) ~ Abstraction Distance`.
  4.  **Secondary Test**: Spearman's Rank Correlation (descriptive).
  5.  Calculate binomial CI (Clopper-Pearson/Wilson) per domain cluster.
  6.  Generate scatter plot (abstraction vs. failure rate) and regression curve.
- **Deliverable**: `data/processed/analysis_results.json`, `figures/correlation_plot.png`.

### Phase 4: Reporting & Validation
- **Goal**: Generate final report and update state.
- **Steps**:
  1.  Compile results into a summary.
  2.  **Runtime Measurement**: Log total wall-clock time for all 90 tasks. Compare against 24-hour target. If exceeded, flag as "Compute-Bound".
  3.  Update `state/` YAML with artifact hashes.
  4.  Run contract tests to ensure data matches schemas.

## Compute Feasibility Plan (Free Tier Adaptation)
- **Memory**: Limit dataset loading to pandas DataFrames with `dtype` optimization. Process tasks in batches of 5.
- **CPU**: Avoid heavy LLM inference. Use CPU-tractable agents (SWE-agent, OpenHands) with default precision.
- **Time**: 90 tasks / 5 per batch = 18 batches. 18 * 4h = 72h wall-clock. However, GitHub Actions allows multiple jobs to run in parallel (up to 2 on free tier). The plan will run multiple jobs in parallel (multiple tasks per batch), reducing total wall-clock time significantly. This exceeds the 24h target but fits the free-tier constraints.
- **No GPU**: All libraries pinned to CPU-compatible versions.
