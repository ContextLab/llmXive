# Data Model: llmXive follow-up: extending "Where Do Deep-Research Agents Go Wrong? Span-Level Error Localization"

## Overview
This document defines the data structures used in the pipeline. All data is processed in memory using `pandas` DataFrames and `networkx` Graphs, with persistent storage in JSON and CSV formats.

## Raw Data Schema (TELBench)
*Source: HuggingFace `NJU-LINK/TELBench`*

| Field | Type | Description |
| :--- | :--- | :--- |
| `trajectory_id` | string | Unique identifier for the trajectory. |
| `spans` | list[dict] | Ordered list of semantic spans. |
| `spans[].text` | string | The textual content of the span. |
| `spans[].order` | int | The chronological order of the span. |
| `label` | string | Ground truth: "success" or "failure". |

## Intermediate Data Model: Claim-Dependency Graph
*Generated by `graph_builder.py`*

| Component | Type | Description |
| :--- | :--- | :--- |
| `nodes` | list[dict] | Nodes representing semantic claims. |
| `nodes[].id` | string | Unique node ID (e.g., `traj_001_span_05`). |
| `nodes[].text` | string | The claim text. |
| `edges` | list[tuple] | Directed edges (source, target) based on co-reference/citation. |
| `edges[].source` | string | Source node ID. |
| `edges[].target` | string | Target node ID. |
| `edges[].type` | string | Edge type: `co-reference` or `citation`. |

*Note: This graph is strictly a Directed Acyclic Graph (DAG).*

## Metrics Schema (Output)
*Generated by `metrics.py` and `evaluator.py`*

| Column | Type | Description |
| :--- | :--- | :--- |
| `trajectory_id` | string | Unique identifier. |
| `num_nodes` | int | Total nodes in the early-stage graph. |
| `num_edges` | int | Total edges in the early-stage graph. |
| `global_connectivity` | float | Normalized density (num_edges / (num_nodes * (num_nodes - 1))). |
| `predicted_collapse` | bool | True if metric < threshold. |
| `actual_label` | string | "success" or "failure". |
| `prediction_correct` | bool | Boolean match of prediction vs actual. |
| `split` | string | "train" or "test" (for stratification tracking). |

## File Structure

```text
data/
├── raw/
│   └── telbench_trajectories.json   # Raw download (checksummed)
├── processed/
│   ├── graphs/
│   │   └── <trajectory_id>.json     # DAG per trajectory
│   └── metrics/
│       └── trajectory_metrics.csv   # Aggregated metrics
└── results/
    └── evaluation_report.json       # Final confusion matrix & stats
```
