# Audio Interaction Model

**Field**: computer science

## Research question

How does incorporating visual context during self‑supervised pre‑training improve low‑resource audio classification performance compared to audio‑only self‑supervised pre‑training?

## Motivation

Always‑on audio interfaces increasingly operate in multimodal environments where visual cues are available (e.g., smart‑home devices with cameras). Existing audio‑only representation learning struggles on categories with few labelled examples. Demonstrating a tangible benefit of audio‑visual interaction would guide the design of more data‑efficient models for real‑world deployment.

## Related work
- [Learning Speech Representations from Raw Audio by Joint Audiovisual Self‑Supervision (2020)](https://arxiv.org/abs/2007.04134) — Shows that joint audio‑visual self‑supervision yields richer speech embeddings than audio‑only pre‑training, motivating a multimodal extension to low‑resource audio tasks.  
- [DeLoRes: Decorrelating Latent Spaces for Low‑Resource Audio Representation Learning (2022)](https://arxiv.org/abs/2203.13628) — Introduces a self‑supervised method tailored to scarce audio data, providing a strong audio‑only baseline for comparison.  
- [A trained humanoid robot can perform human‑like crossmodal social attention and conflict resolution (2021)](https://arxiv.org/abs/2111.01906) — Demonstrates the practical advantage of integrating visual and auditory cues for interactive agents, supporting the hypothesis that visual context can aid audio‑driven interaction models.

## Expected results

We expect joint audio‑visual pre‑training to yield statistically significant higher classification accuracy (≥2 % absolute gain) on low‑resource audio benchmarks than audio‑only pre‑training, while using comparable model size and compute. Confirmation will come from paired statistical tests across multiple random seeds; a null result (no gain) would indicate limited benefit of visual context for these tasks.

## Methodology sketch
- **Data acquisition**
  - Download the AudioSet‑AV (audio‑visual) subset from OpenSLR/HuggingFace (`https://huggingface.co/datasets/google/avspeech`).
  - Download low‑resource audio classification benchmarks (e.g., ESC‑50 “animal” subset, UrbanSound8K “siren” subset) from OpenML.
- **Pre‑training**
  1. Implement the joint audio‑visual self‑supervision objective from the 2020 paper (contrastive audio‑visual alignment) using a lightweight CNN‑Transformer (~30 M parameters) on the audio‑visual subset (CPU‑friendly batch size = 32, 3 epochs).
  2. Implement the DeLoRes audio‑only self‑supervision objective on the same audio data (excluding video) as a baseline.
- **Fine‑tuning**
  - Freeze the encoder and train a linear classifier on each low‑resource benchmark for 10 epochs (learning rate = 1e‑3) using three random seeds.
- **Evaluation**
  - Compute classification accuracy on the held‑out test split for each seed.
  - Perform a paired two‑tailed t‑test comparing the joint vs. audio‑only accuracies across seeds.
  - Report mean ± standard deviation, effect size (Cohen’s d), and 95 % confidence intervals.
- **Resource constraints**
  - All steps run on a GitHub Actions runner (2 CPU cores, ≤7 GB RAM) using PyTorch‑CPU.
  - Total wall‑time expected ≤ 4 hours (download ≈ 30 min, pre‑training ≈ 2 h, fine‑tuning ≈ 1 h, evaluation ≈ 30 min).
