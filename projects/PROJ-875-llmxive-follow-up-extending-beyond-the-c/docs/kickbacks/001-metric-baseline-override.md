## 1. Executive Summary

This document formally requests a **Spec Kickback** to ratify two critical Plan Overrides identified during the implementation of User Story 3 (Scoring & Statistics) and User Story 2 (Baseline Agent).

1. **Metric Definition Override (FR-006):** Replacing raw Levenshtein distance with **Structured JSON Comparison + Semantic Similarity** to measure "Memory Gap."
2. **Baseline Strategy Override (FR-008):** Mandating the Baseline Agent to process **Visual Inputs** (raw frames) rather than ASCII text, ensuring a fair modality-isolated comparison against the Text Agent.

These changes are necessary to ensure the scientific validity of the "Memory Gap" metric and the fairness of the baseline comparison, addressing specific failures in the original design when applied to multimodal state retention tasks.

---

## 2. Context and Justification

### 2.1. Original Specification (FR-006)
**Original Requirement:** "Calculate Memory Gap using Levenshtein distance between agent output and ground truth."
**Issue:** Levenshtein distance operates on raw string sequences. In this project, the "mental map" is a structured JSON object containing discrete state elements (e.g., `{"key": "found", "door": "locked"}`).
- **Problem A:** Minor formatting differences (whitespace, key ordering) in JSON serialization cause massive Levenshtein penalties despite semantic equivalence.
- **Problem B:** Semantic shifts (e.g., "door is open" vs "door is closed") might have low character distance if the sentence structure is similar, masking critical state errors.
- **Conclusion:** Levenshtein distance is an invalid metric for structured state retention in this context.

### 2.2. Original Specification (FR-008)
**Original Requirement:** "Baseline Agent processes ASCII grids."
**Issue:** The primary hypothesis tests the capability of the Text Agent to retain state *without* visual input. If the Baseline also uses ASCII, both agents are operating on the same degraded modality, failing to establish a "Vision-First" upper bound.
- **Problem:** A fair comparison requires the Baseline to have access to the *original* modality (Visual) that the Text Agent is attempting to reconstruct from ASCII.
- **Conclusion:** The Baseline must operate on Visual Inputs to serve as a valid upper-bound reference for the Text Agent's reconstruction capability.

## 3. Proposed Overrides

### 3.1. Metric Definition: Structured JSON + Semantic Similarity
**Override of FR-006**

**New Definition:** The "Memory Gap" score ($S$) is calculated as:
$$ S = (1 - \text{SemanticSimilarity}) + (\text{Penalty} \times \text{MissingCriticalItems}) $$

Where:
- **SemanticSimilarity:** Cosine similarity between the sentence-transformer embedding of the agent's `mental_map` string and the ground truth `mental_map` string. (Library: `sentence-transformers/all-MiniLM-L6-v2`).
- **MissingCriticalItems:** Count of critical state elements (e.g., `key`, `door`) present in the `masked_ground_truth` but absent or incorrect in the `agent_mental_map`.
- **Penalty:** A scalar weight (default: 1.0) applied per missing critical item to ensure structural integrity is prioritized over semantic nuance.

**Implementation Artifact:** `code/scorer.py` (Task T034) implements this logic.

### 3.2. Baseline Strategy: Visual Input Modality
**Override of FR-008**

**New Definition:** The Baseline Agent (`code/baseline_runner.py`) shall:
1. Ingest raw visual frames (`.png`) generated by the Renderer (Task T015c).
2. Utilize a Vision-Language Model (e.g., `Qwen-VL-Chat-Int4`) to process these frames.
3. Output a structured JSON `mental_map` comparable to the Text Agent's output.
4. Be evaluated against the *same* `masked_ground_truth` used for the Text Agent.

**Implementation Artifact:** `code/baseline_runner.py` (Task T039a) implements this logic.
