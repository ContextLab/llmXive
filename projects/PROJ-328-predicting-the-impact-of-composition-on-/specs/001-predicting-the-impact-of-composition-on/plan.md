# Implementation Plan: Predicting the Impact of Composition on the Vickers Hardness of Solder Alloys

## Summary

This project implements a CPU-tractable regression pipeline to predict Vickers hardness (HV) of solder alloys based on their elemental composition. The approach aggregates open-source materials data via a 'Literature Aggregator' (scraping specific, cited PDFs), engineers compositional descriptors (using Centered Log-Ratio transforms to handle closure), and compares XGBoost against a linear baseline using Bootstrap Model Comparison. The plan strictly adheres to the project constitution regarding reproducibility, data hygiene, and the associational framing of results.

### Sensitivity Analysis (FR-009)
The plan explicitly implements a sensitivity analysis sweeping R² thresholds across a range of moderate to high values.
- **Implementation**: For each threshold `T` in the set, calculate the fraction of 1000 bootstrap samples where the model's R² > `T`.
- **Reporting**: This fraction is reported in `data/processed/sensitivity_analysis.yaml` and visualized in `data/outputs/sensitivity_plot.png`.
- **Success Criterion (SC-005)**: The metric is the "fraction of bootstrap samples exceeding each threshold", directly mapping to SC-005.

## Complexity Tracking

| Violation | Why Needed | Simpler Alternative Rejected Because |
| :--- | :--- | :--- |
| **CLR Transform** | Required by FR-014 to address compositional closure (sum-to-1 constraint). | Standard Euclidean distance/linear regression on raw percentages yields spurious correlations. |
| **XGBoost + Linear** | Required by FR-004 to compare non-linear vs. linear baselines. | Using only one model would fail to identify if non-linear interactions exist. |
| **Bootstrap Model Comparison** | Required to address low power of 5-fold t-test (N < 100). | Paired t-test on 5 folds is underpowered; bootstrap compares full distributions. |
| **Sensitivity Analysis** | Required by FR-009 to sweep R² thresholds. | Single R² point estimate is insufficient for robustness. |

## Feature Engineering Strategy

1.  **Input**: `SolderComposition` (raw percentages).
2.  **CLR Transform**: Apply Centered Log-Ratio transform to elemental composition to map to Euclidean space.
3.  **Descriptor Computation**: Compute weighted mean atomic mass, electronegativity variance, atomic radius variance, weighted average melting point, and valence electron concentration *from the CLR-transformed data* (or as alternative encodings of the composition vector).
4.  **Exclusion**: Raw composition percentages are **excluded** from the model input to prevent re-introducing the closure constraint.
5.  **Collinearity**: VIF scores calculated. If VIF ≥ 5, features are flagged as 'collinear' and interpreted descriptively.
