# Data Model: llmXive follow-up: extending "MemLens: Benchmarking Multimodal Long-Term Memory in Large Vision-Language Models"

## Overview

This document defines the data structures used in the MemLens extension project. It covers the raw dataset schema, the derived memory store schemas, and the output metrics schema. All data is stored in `data/` (raw) and `artifacts/` (derived) directories.

## Raw Dataset Schema

**Source**: MemLens (HuggingFace)
**File Format**: JSON / Parquet

```yaml
type: object
properties:
  id:
    type: string
    description: Unique identifier for the question.
  task_type:
    type: string
    enum: [MSR, TR, Other]
    description: Task category. Only MSR and TR are processed.
  question:
    type: string
    description: The natural language question.
  answer:
    type: string
    description: The ground truth answer.
  images:
    type: array
    items:
      type: string
      description: Paths or URLs to the images involved in the session.
  session_id:
    type: string
    description: Identifier for the multi-session context.
  metadata:
    type: object
    description: Additional metadata (e.g., timestamps, difficulty).
```

## Memory Store Schema

**Derived Data**: Generated by `code/preprocessing.py`
**Storage**: JSONL or Parquet in `data/processed/memory_stores/`

### Coarse Store
```yaml
type: object
properties:
  entry_id:
    type: string
  session_id:
    type: string
  text_summary:
    type: string
  embedding_vector:
    type: array
    items:
      type: number
    description: Dense vector from sentence-transformers.
```

### Medium Store
```yaml
type: object
properties:
  entry_id:
    type: string
  session_id:
    type: string
  text_summary:
    type: string
  text_embedding:
    type: array
    items:
      type: number
  image_embedding:
    type: array
    items:
      type: number
    description: Global CLIP embedding of the image.
```

### Fine Store
```yaml
type: object
properties:
  entry_id:
    type: string
  session_id:
    type: string
  object_captions:
    type: array
    items:
      type: string
    description: List of captions for detected objects.
  bounding_boxes:
    type: array
    items:
      type: object
      properties:
        x_min: { type: number }
        y_min: { type: number }
        x_max: { type: number }
        y_max: { type: number }
        label: { type: string }
  multimodal_embedding:
    type: array
    items:
      type: number
    description: Fused embedding of text and box coordinates.
  detection_status:
    type: string
    enum: [success, fallback]
    description: "success" if YOLO ran, "fallback" if global embedding used.
  recall_score:
    type: number
    description: Object detection recall (if ground truth available).
```

## Metrics Schema

**Output**: Generated by `code/evaluation.py` and `code/stats.py`
**File**: `artifacts/results/metrics.json`

```yaml
type: object
properties:
  strategy:
    type: string
    enum: [Coarse, Medium, Fine]
  metrics:
    type: object
    properties:
      accuracy:
        type: number
        description: Proportion of correct answers.
      retrieval_latency_ms:
        type: number
      inference_latency_ms:
        type: number
      peak_ram_mb:
        type: number
  statistical_comparison:
    type: object
    properties:
      test_name:
        type: string
        const: "Wilcoxon Signed-Rank"
      statistic:
        type: number
      p_value:
        type: number
      significant:
        type: boolean
      effect_size:
        type: number
      sample_size:
        type: integer
      power_warning:
        type: boolean
        description: True if n < 30.
```
