# Implementation Plan: Predicting Molecular Interactions in Polymer Composites with Graph Neural Networks

**Branch**: `[001-polymer-interaction-gnn]` | **Date**: 2024-01-15 | **Spec**: `specs/001-predictor-interaction-gnn/spec.md`

## Summary

We will build a reproducible, CPU‑only pipeline that (1) downloads molecular graph data from the MolNet benchmark, (2) cross‑references it with verified chemistry sources for adhesion‑energy measurements, (3) constructs heterogeneous graphs for polymer‑filler interfaces, (4) trains a 3‑layer Graph **Attention** Network (GAT) on these graphs, and (5) validates the model with a full re-training permutation test (a sufficient number of iterations, reduced subset), gradient-based attribution, and VIF reporting on hand-crafted descriptors. All steps respect the -hour runtime and Limited RAM constrains the research question, the method, and the references, but specific low-level empirical values such as exact memory counts are deferred to the implementation phase. of GitHub Actions free-tier.

**Critical Note on Spec Alignment**: This plan addresses scientific inaccuracies in the source spec. 
- The plan implements a **GAT** (not GCN) to satisfy the attention requirement, noting that the spec (FR-003) must be updated.
- The plan enforces a **hard abort** if adhesion energy is missing, rejecting the spec's proxy fallback (E-DATA-001) as scientifically invalid.
- The plan clarifies that **attention does not handle collinearity** (VIF); it handles feature weighting. VIF is reported separately. This requires an update to FR-007 in the spec.

## Phase 0 – Data Acquisition & Validation (FR‑001, FR‑002, SC‑001)

1. `code/data/download.py` fetches MolNet via `datasets.load_dataset('molnet', ...)` and records SHA256 checksums.  
2. The script verifies the presence of required columns (`polymer_smiles`, `filler_smiles`, `adhesion_energy`).  
3. **Critical**: If `adhesion_energy` is missing or <100 valid rows, the pipeline aborts with exit code **E‑DATA‑001**. **No proxy metrics are used.** The spec's fallback path is scientifically invalid and rejected.

## Phase 1 – Graph Construction (FR‑002, VII)

`code/data/graph_build.py` converts each SMILES pair to a heterogeneous PyG graph, encoding atom type, charge, hybridization as node attributes and bond type, order, `is_interaction` as edge attributes. 
- **Mandatory Output**: This script **explicitly generates** `analysis/topology_audit.md` documenting node/edge fidelity and any pruning.
- Outputs `data/processed/graphs.pt`.

## Phase 2 – Model Definition (FR‑003, FR‑007)

`code/models/gat.py` defines a **‑layer Graph Attention Network (GAT)**:
- `GATConv(in_channels, hidden, heads=4, concat=False)`
- ReLU + Dropout () after each layer.
- Global mean pooling → Linear → scalar output.
- **Clarification**: Attention mechanisms provide robust feature weighting. They **do not** statistically control for collinearity (VIF). VIF is handled separately in Phase 4.

## Phase 3 – Training (FR‑004, SC‑002)

`code/models/train.py`:
- /20 split, batch ≤32, Adam lr [qualitative setting].
- Fixed `random_seed = 42`.
- Checkpoint periodically throughout training. to `results/checkpoint_{epoch}.pt`.
- Early stop if runtime exceeds a predefined threshold.; resume allowed; hard timeout at several hours.

## Phase 4 – Statistical Validation (FR‑005, FR‑006, FR‑007, FR‑008, SC‑003, SC‑004, SC‑005)

1. **Permutation Test** (`analysis/perm_test.py`):  
   - **Methodology**: Full re-training of the GAT on **permuted labels** (not frozen model evaluation) to establish a valid null distribution.  
   - **Feasibility**: Reduced to **A fixed number of iterations sufficient for convergence will be employed.** on a **reduced subset (100 samples)** for **10 epochs** per iteration. Total epochs = 1000. This fits the -hour limit.  
   - Compute observed MSE from the full‑data trained model; p‑value = proportion of permuted MSE ≤ observed MSE.  

2. **Multiple‑Comparison Correction**:  
   - `stats.csv` records p‑values for **MSE** and **MAE**.  
   - If `stats.csv` contains >1 p‑value (i.e., multiple metrics), apply **Bonferroni** (or Holm) correction; corrected p‑values stored in `corrected_p_value` column.

3. **Gradient‑Based Attribution** (`analysis/attribution.py`):  
   - Integrated Gradients on a held‑out test set of graphs.  
   - Aggregate importance scores for topological features (node degree, edge connectivity, graph density).  
   - Report features with std > 0.1.

4. **Collinearity Reporting** (`analysis/collinearity.py`):  
   - Compute VIF on **handcrafted descriptors** (degree, clustering coefficient, density, average shortest path) **extracted before graph construction**.  
   - **Clarification**: VIF is **not** calculated on GNN latent embeddings (which are non-linear and correlated by design).  
   - Include VIF scores in `results/stats.csv`. No training halt based on VIF.

## Phase 5 – Power Analysis (SC‑002 justification)

`analysis/power_analysis.md` documents an a priori power analysis. 
- **Assumption**: Medium effect size (f² = 0.15), α = 0.05, Power = 0.80.
- **Result**: Required N will be determined based on power analysis.. Targeting ≥ 500 pairs provides > 0.95 power for medium effects.
- **Limitation**: Acknowledges that for small effects, the sample size may be underpowered, but it is sufficient for the stated medium-effect hypothesis.

## Phase 6 – State Hashing (Principle V)

After each major artifact creation (`curated_dataset.csv`, `graphs.pt`, `model.pt`, `stats.csv`), `utils/hash_state.py` **explicitly computes** SHA256 hashes and updates `state/projects/PROJ-413-predicting-molecular-interactions-in-pol.yaml` under `artifact_hashes`. This ensures version‑discipline and reproducibility.

## Phase 7 – Reporting & Packaging

All results (`performance.json`, `stats.csv`, `attribution.json`, `topology_audit.md`) are stored under `results/` or `analysis/`. The final manuscript will reference these files exclusively (Single Source of Truth).
