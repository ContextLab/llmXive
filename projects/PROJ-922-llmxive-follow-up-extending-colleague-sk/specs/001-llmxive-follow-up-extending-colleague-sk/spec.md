# Specification: llmXive follow-up: extending "COLLEAGUE.SKILL: Automated AI Skill Generation via Expert Knowledge Disentanglement"

## Project Overview
This project investigates the efficacy of decoupling "Capability" (what an AI can do) from "Behavior" (how it acts) in prompt engineering. We compare a Monolithic prompt approach against a Separated Tracks approach to determine if separation reduces hallucination rates while maintaining or improving style consistency.

## User Stories
- **US1**: Execute CPU-tractable LLM inference with decoupled prompt architecture.
- **US2**: Evaluate outputs against deterministic ground-truth rules.
- **US3**: Aggregate results and perform statistical comparison.

## Functional Requirements
- **FR-001**: The system must support CPU-only inference with OOM protection.
- **FR-002**: The system must generate synthetic expert profiles and task scenarios.
- **FR-003**: The system must support three prompt conditions: Monolithic, Separated Tracks, Generic Baseline.
- **FR-004**: The system must calculate Heuristic Adherence, Style Consistency, and Hallucination Rate.
- **FR-005**: Hallucination detection must include multi-hop reasoning checks via logic verification (Z3).
- **FR-006**: The system must perform statistical analysis using a Generalized Linear Mixed Model (GLMM) to handle mixed outcomes (binary/continuous).
 - *Amendment from T000a*: The statistical method is explicitly defined as a GLMM, not a Linear Mixed-Effects Model (LMM), to accommodate binary outcomes (Heuristic Adherence, Hallucination Rate) and continuous outcomes (Style Consistency) simultaneously within a unified framework.
- **FR-007**: The system must apply Holm-Bonferroni correction for multiple comparisons.
- **FR-008**: The system must perform sensitivity analysis on style consistency thresholds.

## Non-Functional Requirements
- **NFR-001**: All inference runs must complete within 300 seconds per task on a CPU-only runner.
- **NFR-002**: The system must be reproducible with a fixed random seed (42).

## Assumptions and Constraints
- **Scale Assumption**: The study is conducted on a reduced scale of **10 profiles × 50 tasks × 3 conditions = 1,500 runs**.
 - *Amendment from T000a*: The original assumption of "50 profiles × 200 tasks" is replaced with "10 profiles × 50 tasks". This reduction is necessary to ensure GLMM convergence within the computational limits (RAM and runtime) of the free-tier CI runner (CPU-only). The reduced scale is statistically powered to detect a moderate effect size (Cohen's d) with sufficient random-effect levels (10 profiles, 50 tasks).
- **Hardware Constraint**: The primary execution environment is a CPU-only runner. GPU-accelerated tasks must be scaled down or offloaded to a free-tier GPU environment (e.g., Kaggle) if strictly required, but the primary method is CPU-tractable.
- **Data Integrity**: All data generation must be deterministic and reproducible. Synthetic data is used only for expert profiles and task scenarios where real-world equivalents are not programmatically accessible, but inference outputs must be generated by a real model, not simulated.

## Data Model
- **Profile**: {id, domain, capability_rules, behavior_keywords}
- **Task**: {id, domain, prompt, expected_output_type, evaluation_rules}
- **InferenceOutput**: {profile_id, task_id, condition, latency, success_flag, output_text}
- **EvaluationScore**: {profile_id, task_id, condition, heuristic_adherence, hallucination_rate, style_consistency}

## Statistical Analysis Plan
- **Model**: Generalized Linear Mixed Model (GLMM).
- **Formula**: `metric ~ condition + (1 | profile_id) + (1 | task_id)`
- **Fixed Effects**: Condition (Monolithic, Separated, Generic).
- **Random Effects**: Profile ID, Task ID.
- **Outcomes**:
 - Heuristic Adherence: Binomial (Logit link)
 - Hallucination Rate: Binomial (Logit link)
 - Style Consistency: Gaussian (Identity link)
- **Corrections**: Holm-Bonferroni for multiple comparisons across metrics.
- **Non-Inferiority**: A non-inferiority margin of 0.05 (5 percentage points) is applied to Style Consistency comparisons as per SC-002.

## Dependencies
- `transformers` (or `llama-cpp-python` for CPU quantization)
- `statsmodels` (for GLMM)
- `pandas`, `numpy`
- `z3-solver` (for logic verification)
- `sympy` (for math verification)
