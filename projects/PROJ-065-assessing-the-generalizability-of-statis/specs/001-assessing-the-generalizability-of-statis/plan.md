# Implementation Plan: Assessing the Generalizability of Statistical Significance in Pre-Registered Studies

**Branch**: `001-assessing-the-generalizability-of-statis` | **Date**: 2026-06-28 | **Spec**: `specs/001-assessing-the-generalizability-of-statis/spec.md`

## Summary

This project assesses the fragility of statistical significance in pre-registered studies by re-running analyses on raw data via bootstrap resampling. The system ingests data from the Open Science Framework (OSF) or generates synthetic datasets mimicking the statistical properties of target studies, executes stratified bootstrap iterations (1,000 baseline + alternative specifications), and calculates "Sampling Stability" and "Model Specification Sensitivity" rates. The output includes a forest plot of stability rates across disciplines (psychology, economics, biology) and a meta-analysis of heterogeneity (I²). The implementation prioritizes CPU feasibility on a constrained multi-core CPU with limited RAM. and strict reproducibility.

## Technical Context

**Language/Version**: Python 3.11  
**Primary Dependencies**: `pandas`, `numpy`, `scipy`, `statsmodels`, `requests`, `matplotlib`, `seaborn`, `pyyaml`  
**Storage**: Local filesystem (`data/raw/`, `data/processed/`, `outputs/`)  
**Testing**: `pytest`  
**Target Platform**: Linux (GitHub Actions Free Tier: 2 CPU, 7GB RAM, 14GB Disk)  
**Project Type**: Data Analysis Pipeline / CLI  
**Performance Goals**: Process 50 studies within 6 hours total runtime; single study bootstrap < 10 mins. Adaptive subsampling (N<=2000) and iteration reduction (500) will be triggered if projected runtime exceeds 5.5h.  
**Constraints**: No GPU; no heavy LLM inference; memory usage < 6GB peak; strict random seed pinning.  
**Scale/Scope**: A sufficient number of studies, a sufficient number of bootstrap iterations per study to ensure stability, ~5 alternative model specs per study.

## Complexity Tracking

| Violation | Why Needed | Simpler Alternative Rejected Because |
|-----------|------------|-------------------------------------|
| **Multiple Model Specs** | Required by FR-003 and Constitution Principle VI to distinguish sampling noise from model choice. | Single-model bootstrap would fail to measure "Model Specification Sensitivity," rendering the generalizability claim weak. |
| **Cross-Domain Filtering** | Required by FR-001 and Constitution Principle VII. | Analyzing a single domain would not support "generalizability" claims across disciplines. |
| **Bootstrap Intensity** | Required by FR-002 ([deferred] iterations) for statistical power. | Fewer iterations (<100) would yield unstable stability rate estimates, failing SC-001. |
| **Adaptive Runtime** | Required to meet 6h limit on free-tier CI. | Fixed 1,000 iterations for all studies might exceed 6h if N is large; adaptive subsampling ensures completion. |

## Phase Execution Order

1.  **Data Ingestion (FR-001)**: Download and parse OSF data OR generate synthetic datasets mimicking target studies. Validate variables.
2.  **Bootstrap Execution (FR-002, FR-003)**: Run stratified resampling for baseline and alternative specs.
3.  **Metric Calculation (FR-004, FR-007)**: Compute Stability Rates and Threshold Sensitivity.
4.  **Aggregation & Visualization (FR-005, FR-006)**: Execute `meta_analysis.py` to generate forest plots and I² statistics.
5.  **Versioning & Reporting**: Execute `record_artifact_hash()` to update `state/projects/...yaml`. Compile `summary_report.pdf` and `baseline_metrics.csv`.

### Versioning Mechanism Detail
After Phase 4 completes, `code/main.py` will:
1.  Identify all files in `outputs/` and `data/processed/`.
2.  Compute SHA-256 hash for each file.
3.  Update `state/projects/PROJ-065-assessing-the-generalizability-of-statis.yaml` with the new `artifact_hashes` map.
4.  Update the `updated_at` timestamp.
This ensures the SSoT is updated automatically and non-negotiably.

### Runtime Feasibility Confirmation
The plan targets a substantial number of studies.. The `bootstrap_engine.py` will:
-  Check total projected runtime before starting a study.
-  If projected time > 5.5h, reduce iterations to 500 for remaining studies.
-  If N > 2000, subsample to 2000 for bootstrap iterations.
This guarantees completion within the allocated time limit while maximizing data usage.
