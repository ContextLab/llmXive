# Glossary

## Terms

- **Compiler Optimization**: The process of modifying code to improve performance, reduce size, or both, without changing its functionality.
- **Latency**: The time taken to execute a kernel operation.
- **Numerical Stability**: The property of an algorithm to avoid excessive rounding errors or overflow/underflow.
- **L2 Relative Error**: A metric measuring the difference between two vectors, normalized by the magnitude of the reference vector.
- **Max Absolute Difference**: The maximum absolute difference between corresponding elements of two vectors.
- **Pareto Frontier**: A set of optimal solutions where no objective can be improved without worsening another.
- **Welch's t-test**: A statistical test used to compare the means of two independent samples with potentially unequal variances.
- **Block Averaging**: A technique to reduce noise by averaging measurements over fixed blocks.
- **Downsampling**: Reducing the dimensionality of input data (e.g., 768x768 to 512x512) to fit within memory constraints.
- **NaN (Not a Number)**: A special floating-point value representing undefined or unrepresentable values.
- **Manifest**: A file containing hashes of all artifacts to ensure reproducibility.

## Acronyms

- **LLM**: Large Language Model.
- **MatMul**: Matrix Multiplication.
- **ReLU**: Rectified Linear Unit (not used here, but common in LLMs).
- **CPU**: Central Processing Unit.
- **GPU**: Graphics Processing Unit.
- **CSV**: Comma-Separated Values.
- **JSON**: JavaScript Object Notation.
- **SHA-256**: Secure Hash Algorithm 256-bit.
