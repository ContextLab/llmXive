# Self-improving LLM: recursive architecture refinement and re-training

Given a model + source code + weights, continually prompt the model to make improvements to its own architecture, re-train itself, and (when satisfied that the new version is working better than itself), replace itself with its improved version. Then repeat the process.

**Keywords**: LLMs, self-improvement, singularity, recursion
