## Summary

This project investigates the relationship between asynchronous communication delays (specifically response-time variance) and perceived team cohesion in distributed open-source software teams. The technical approach involves: (1) ingesting GitHub event data (issues, PRs, comments) to derive temporal and structural metrics; (2) applying VADER sentiment analysis to text content, combined with network metrics (reciprocity, density), to derive a composite cohesion proxy; (3) performing Spearman correlation and controlled linear regression (with interaction terms) to test hypotheses; and (4) validating the proxy against manual coding of relational indicators. The implementation is constrained to run on a free-tier GitHub Actions runner (limited CPU and RAM, no GPU).

## Complexity Tracking

| Violation | Why Needed | Simpler Alternative Rejected Because |
| :--- | :--- | :--- |
| **Multi-modal Validation (FR-009)** | Required by spec to establish construct validity of the cohesion proxy. | A single-pass VADER analysis would fail SC-005 (validity against manual coding) and violate Constitution Principle II (Verified Accuracy) regarding proxy claims. |
| **VIF Check & Halting (FR-008)** | Required to prevent unstable regression coefficients due to collinearity. | Proceeding with high VIF would produce misleading causal inferences, violating the "Associational only" assumption and statistical rigor requirements. |
| **Modality Separation (Const. VI)** | Strict isolation of timestamp and text processing. | Combined processing risks data leakage where temporal patterns could influence text tokenization or vice versa, compromising the "phenomenon-vs-method" validation. |
| **Structural Cohesion Metrics** | Required to address construct validity (methodology-b2dd1644). | Simple sentiment averaging fails to capture relational dynamics (trust, reciprocity) essential to the "cohesion" construct. |
| **Weighted Variance Aggregation** | Required to address statistical instability (methodology-17d46215). | Median of variances is biased by low-count pairs; weighted mean ensures robust team-level representation. |
