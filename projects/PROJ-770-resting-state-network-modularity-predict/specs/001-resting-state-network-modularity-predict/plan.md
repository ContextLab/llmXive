# Implementation Plan: Resting-State Network Modularity Predicts Social Network Size

**Branch**: `001-gene-regulation` | **Date**: 2026-06-28 | **Spec**: `spec.md`
**Input**: Feature specification from `/specs/001-gene-regulation/spec.md`

## Summary

This project investigates the association between resting-state functional network modularity (quantified by the Louvain quality index Q) and self-reported social network size in healthy adults. The technical approach involves downloading a sampled subset of the HCP 1200 Subjects Release, preprocessing fMRI time series (band-pass filtering, nuisance regression), constructing functional connectivity graphs using the Schaefer atlas, computing modularity Q, and fitting a linear regression model controlling for age, sex, motion, and total connectivity strength.

**Critical Constraint**: The analysis is strictly limited to CPU-only execution on GitHub Actions free-tier runners (limited CPU, constrained RAM). The pipeline requires **verified availability** of both fMRI NIfTI files and behavioral data for the same subjects. If the verified data sources do not contain the required fMRI data for the sampled subjects, the pipeline will halt with a "Data Unavailable" error. No synthetic or mismatched data sources will be used.

## Constitution Check

*GATE: Must pass before Phase 0 research. Re-check after Phase 1 design.*

1.  **I. Reproducibility**: The plan mandates pinning random seeds in `code/` and fetching data from canonical HCP sources (via verified HCP S3 or verified HuggingFace mirrors) on every run. The `requirements.txt` will pin all dependencies.
2.  **II. Verified Accuracy**: All dataset citations in `research.md` will strictly reference the URLs provided in the `# Verified datasets` block of the user message. If the verified URLs do not contain the required fMRI NIfTI data, the plan will halt rather than use unverified/synthetic sources.
3.  **III. Data Hygiene**: Raw data downloads will be stored in `data/raw/` with checksums recorded. Derived data (processed matrices, Q values) will be written to `data/processed/` as new files. No in-place modifications.
4.  **IV. Single Source of Truth**: All statistical results in the final output will be generated by the `code/` scripts and written to `data/results/`. The paper will reference these files, not hand-typed values.
5.  **V. Versioning Discipline**: Artifacts will be tracked via content hashes. The plan includes a step to generate and record checksums in the project state file.
6.  **VI. Neuroimaging-Survey Independence**: The pipeline will treat fMRI preprocessing and behavioral data ingestion as separate streams until the final merge step (row-wise join on `subject_id`), preventing cross-contamination. The merge will fail if `subject_id` exists in one stream but not the other.
7.  **VII. Null Result Equivalence**: The statistical analysis plan explicitly includes reporting coefficients and p-values regardless of significance, framing findings as associational.

*Note on Reviewer Feedback (Geoffrey West): The reviewer suggests modeling the "metabolic cost" of neural connections. The plan addresses this by including "total connectivity strength" (sum of absolute edge weights) as a *secondary* covariate. However, due to potential collinearity with Modularity Q, a Variance Inflation Factor (VIF) check is mandated. If VIF > 5, the primary model will be re-run without this covariate, and the metabolic cost hypothesis will be treated as an exploratory analysis.*
