# Research: Statistical Properties of Simulated Black Hole Mergers

## Dataset Strategy

This project requires two datasets: observational data from the Gravitational Wave Transient Catalog (GWTC) and a simulation dataset for comparison.

### Observational Data (GWTC-1 & GWTC-2)

The observational data consists of posterior samples for binary black hole mergers.
- **Source**: GWTC-1 (DOI: 10.5281/zenodo.3966973) and GWTC-2 (DOI: 10.5281/zenodo.3966974).
- **Status**: The specific Zenodo record URLs for these DOIs are currently unreachable (HTTP 404) in the verified dataset check.
- **Fallback Strategy**: As per FR-001 and the spec's edge cases, the pipeline will attempt to download from the canonical Zenodo DOI URLs. If the URL returns a 404, the system will retry with exponential backoff (1s, 2s, 4s) up to 3 times. If the source remains unavailable after retries, the pipeline will fail with an explicit error message identifying the missing DOI, rather than proceeding with invalid data.
- **Note**: The spec assumes these DOIs will be accessible. If they remain permanently unreachable, the project cannot proceed with real observational data, and a simulation-only study would require a formal spec amendment.

### Simulation Data

The project requires a dataset with resolved component masses, mass ratios, and effective spins.
- **Primary Source**: A dedicated binary black hole population synthesis catalog (e.g., COSMOS or a GWTC-3-like synthetic set).
- **Verified Sources**: No verified external source for a GWTC-3-like or COSMOS BBH spin/mass catalog was found in the provided dataset list.
- **Fallback Strategy**: Per FR-002, if no external verified source exists, the system MUST generate a synthetic catalog based on a specific physical hypothesis: **"Power-law spin distribution"**.
  - **Generation Method**: Synthetic events will be generated by sampling component masses from a power-law distribution and effective spins from a uniform or truncated normal distribution, ensuring the schema matches the observational data (mass_ratio, effective_spin, component_mass_1, component_mass_2).
  - **Rationale**: This ensures the KS test (FR-005) has a valid distributional comparison, unlike scalar merger rates which lack the necessary variables.

## Statistical Methodology

### Kernel Density Estimation (KDE)
- **Method**: 1D KDE for `mass_ratio` and `effective_spin`.
- **Bandwidth**: Selected via Scott's rule (default for `scipy.stats.gaussian_kde`), which scales with sample size $n$ as $n^{-1/5}$.
- **Justification**: Scott's rule is robust for sample sizes $n \ge 30$, which is the expected scale for GWTC catalogs.

### Kolmogorov-Smirnov (KS) Test
- **Hypothesis**: $H_0$: The observational and simulation distributions are identical.
- **Correction**: Bonferroni correction applied for multiple comparisons (2 tests: mass_ratio, effective_spin). Adjusted $\alpha = 0.05 / 2 = 0.025$.
- **Selection Bias**: The analysis will apply a correction for detection efficiency (FR-016) by weighting samples or using a selection function if available in the posterior metadata. If not, the limitation will be explicitly logged.

### Sensitivity Analysis
- **Scope**: Sweep $\alpha \in \{0.04, 0.05, 0.06\}$.
- **Output**: Report "borderline" flags if significance status flips across this range.

### Power Analysis
- **Metric**: Minimum Detectable Effect Size (MDES) calculated for the given sample sizes.
- **Limitation**: If simulation $N < 0.5 \times$ observational $N$, a power limitation note is logged (FR-010).

## Computational Feasibility

- **Hardware**: GitHub Actions free-tier (2 CPU, 7 GB RAM).
- **Memory**: Data processing uses `pandas` and `numpy` with streaming where possible. The full dataset (~100 events) fits easily in RAM.
- **Runtime**: All operations (download, preprocess, KDE, KS, plot) are $O(N \log N)$ or $O(N)$ and will complete in <30 minutes, well within the 6-hour limit.
- **Dependencies**: `scipy`, `numpy`, `pandas`, `matplotlib` are CPU-optimized and do not require GPU.

## Risks & Mitigations

| Risk | Mitigation |
|------|------------|
| Zenodo 404 (GWTC-1/2) | Retry with exponential backoff; fail explicitly if unreachable. |
| No simulation source | Generate synthetic data via Power-law hypothesis (FR-002). |
| Small sample size | Log power limitation; report MDES (FR-010). |
| Selection bias | Log limitation if correction cannot be applied (FR-016). |
