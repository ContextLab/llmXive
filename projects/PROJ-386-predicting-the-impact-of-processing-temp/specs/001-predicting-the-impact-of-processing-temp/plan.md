# Implementation Plan: Predicting the Impact of Processing Temperature on the Grain Size of Rolled Aluminum Alloys

**Branch**: `001-gene-regulation` | **Date**: 2026-07-02 | **Spec**: `specs/001-gene-regulation/spec.md`
**Input**: Feature specification from `/specs/001-gene-regulation/spec.md`

## Summary

This project implements a computational pipeline to predict grain size in rolled aluminum alloys based on processing temperature and alloy composition. The system performs strict schema pre-checks on public datasets (NOMAD, OpenML, etc.) to verify the presence of critical variables (Temperature, Composition, Grain Size) before ingestion. It then executes a two-stage modeling approach: (1) a baseline linear regression with interaction terms (Temp × Element) to establish main effects, and (2) a Random Forest regressor with grid search to capture non-linear modulation. The pipeline includes rigorous diagnostics for collinearity, sensitivity analysis on decision thresholds, and confounder checks, all constrained to run on a GitHub Actions free-tier CPU runner within 5 hours.

**Critical Note on Data Feasibility**: The project relies on the "Verified datasets" block provided in the spec. If these sources (specifically the NOMAD structure CSV) lack the required *processing* variables (rolling temperature, grain size), the pipeline will halt with a "Data Missing" report. This outcome is treated as a valid scientific finding: "Current public structural databases do not contain the necessary process data for this prediction task."

## Confounder Control Strategy

To address the observational nature of the data (where Alloy Series determines both Composition and typical Processing Temperature):

1.  **Residualization**: Before modeling the "Temperature Impact", we will first regress `Grain Size` against `Alloy Series` (one-hot encoded) and `Composition` (Mg, Si, Cu, Zn). The residuals from this model represent the "unexplained" variance.
2.  **Secondary Modeling**: We will then model these residuals against `Temperature` and `Temperature × Composition` interactions. This isolates the temperature effect from the confounding alloy series.
3.  **Stratified Group K-Fold**: Cross-validation will be performed with `groups=Alloy Series` to ensure that all data from a specific alloy series is either in the training or test set, preventing leakage of series-specific behaviors.
4.  **Framing**: All results are framed as **Conditional Associations** (e.g., "Within a fixed alloy series, higher temperature is associated with...") rather than causal impacts.

## Complexity Tracking

| Violation | Why Needed | Simpler Alternative Rejected Because |
|-----------|------------|-------------------------------------|
| **Two-stage modeling (Linear + RF)** | Required by FR-003 and FR-004 to distinguish main effects from non-linear interactions. | A single RF model would obscure the baseline contribution of linear terms, violating the research goal of quantifying "modulation" vs. "main effects". |
| **Schema Pre-checks** | Required by FR-001 to prevent wasted compute on datasets missing critical variables (Temperature/Grain Size). | Attempting to download and parse large files before checking schema would waste the 5-hour timeout and memory budget. |
| **Hard Timeout Enforcement** | Required by FR-007 and Assumption about compute environment. | Without explicit timeout logic, a grid search on a large dataset could exceed the 6-hour CI limit, causing a hard failure. |
| **Residualization** | Required to address confounding by Alloy Series. | Simple inclusion of `Alloy Series` as a feature may absorb the temperature effect entirely, rendering the temperature coefficient uninterpretable. |
| **Stratified Group K-Fold** | Required to prevent leakage of alloy-specific behaviors. | Standard random split may leak similar alloy compositions between train and test, inflating R². |


## Scientific Validity & Limitations

- **Associational Nature**: The project explicitly acknowledges that without random assignment or independent experimental validation, all findings are **associational**. The "Impact" is defined as the change in predicted grain size *conditional* on the observed data distribution.
- **Data Limitation**: If the provided verified datasets (NOMAD structure CSV) lack rolling process parameters, the project will halt with a "Data Missing" report. This is a valid scientific outcome indicating the unavailability of process data in current public structural repositories.
- **Unmeasured Confounders**: Variables like strain rate and cooling rate are likely missing. The model will attribute variance to temperature/composition interactions *conditional* on the observed data. The final report will explicitly state this limitation.

## Success Criteria Alignment

- **SC-001**: Success is defined as R² improvement > 0.05 **OR** relative improvement > 10% over baseline, AND statistical significance (p < 0.05) of interaction terms in the linear model.
- **SC-002**: Sensitivity analysis results are measured against the stability of the top-5 significant interaction terms across the threshold sweep {, low, medium}.
- **SC-003**: Dataset variable completeness is measured against the requirement for temperature, composition, and grain size fields.
- **SC-004**: Computational feasibility is measured against the standard GitHub Actions free-tier time limit and GB RAM constraint.
- **SC-005**: Collinearity diagnostic output is measured against the correlation matrix to verify that no two predictors have a correlation coefficient > 0.8 without appropriate descriptive framing.
