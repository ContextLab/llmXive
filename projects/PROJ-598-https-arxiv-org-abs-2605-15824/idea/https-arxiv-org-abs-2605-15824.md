# FashionChameleon: Towards Real-Time and Interactive Human‑Garment Video Customization  

**Field**: computer science  

## Research question  

How does inference latency (measured in frames per second) affect the visual fidelity of garment‑swap videos produced by real‑time video‑customization systems?  

## Motivation  

Interactive garment‑swap video tools promise immersive shopping experiences, but low‑latency operation may force algorithmic shortcuts that degrade visual quality. Understanding the trade‑off between speed and fidelity is essential for designing systems that are both responsive and realistic, guiding engineers on where latency reductions are acceptable and where they hurt user perception.  

## Related work  

- [FashionChameleon: Towards Real-Time and Interactive Human‑Garment Video Customization (2026)](https://arxiv.org/abs/2605.15824) — Introduces a real‑time pipeline for garment video customization but provides limited analysis of the speed‑quality trade‑off.  
- [Tunnel Try‑on: Excavating Spatial‑temporal Tunnels for High‑quality Virtual Try‑on in Videos (2024)](https://arxiv.org/abs/2404.17571) — Presents a high‑quality video‑try‑on method that prioritises visual detail over latency, serving as a quality‑focused baseline.  
- [Scalable and Realistic Virtual Try‑on Application for Foundation Makeup with Kubelka‑Munk Theory (2025)](https://arxiv.org/abs/2507.07333) — Demonstrates commercial‑grade AR try‑on pipelines, highlighting the importance of scalability and real‑time performance in consumer‑facing applications.  

## Expected results  

We anticipate observing a negative correlation between FPS and perceptual quality metrics (e.g., higher FPS → higher LPIPS, lower SSIM). A statistically significant regression slope (p < 0.05) would confirm that latency reductions come at a measurable cost to visual fidelity, while a non‑significant slope would suggest that current optimisations preserve quality.  

## Methodology sketch  

- **Data acquisition**  
  - Download the publicly available *FashionVideo* dataset (≈ 30 short videos with garment annotations) from https://github.com/DeepFashionLab/FashionVideo (≈ 200 MB).  
- **Baseline implementation**  
  - Clone the open‑source *Tunnel‑Tryon* code (https://github.com/VT-Research/TunnelTryon).  
  - Install required Python packages via `requirements.txt`.  
- **Latency‑controlled variants**  
  - Create three model configurations: (a) full‑size model, (b) 50 % channel reduction, (c) 75 % channel reduction.  
  - For each config, run inference on the same 20‑video test split, measuring wall‑clock time with Python’s `time.perf_counter()` and computing FPS (frames / seconds).  
- **Quality evaluation**  
  - Compute LPIPS (using the `lpips` library) and SSIM (using `skimage.metrics.structural_similarity`) between generated videos and the ground‑truth target videos.  
  - Aggregate scores per configuration (mean ± std over the 20 videos).  
- **Statistical analysis**  
  - Perform a Spearman rank correlation between FPS and each quality metric.  
  - Fit a linear regression model (quality ~ FPS) and report slope, 95 % confidence interval, and p‑value.  
- **Reproducibility**  
  - All scripts, configuration files, and a `README.md` will be version‑controlled in a GitHub repository.  
  - The entire pipeline (download → inference → evaluation) is designed to run on a GitHub Actions free‑tier runner (≤ 7 GB RAM, ≤ 6 h).
