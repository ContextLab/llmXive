## Unresolved concerns

- Task T027 (Implement `code/03_alignment.py`) is placed in Phase 4 (US2) but depends on the output of T019 (Raw JSON reports from US1). While Phase 4 is listed after Phase 3, the task description for T027 does not explicitly state it consumes the `data/raw` artifacts generated by T019. More critically, T027 implements the alignment logic which is a prerequisite for T032 (Metrics). The current grouping implies T027 is part of US2, but US2's 'Independent Test' relies on 'aligned pairs' which are the output of T027. However, T032 (US3) explicitly requires 'aligned pairs' as input. The dependency chain US1(T019) -> US2(T027) -> US3(T032) is correct, but T027's description 'Implement `code/03_alignment.py` to align tool issues with human annotations' implies it needs the human annotations (T022-T026) AND the tool issues (T019). The ordering within Phase 4 (T022-T026 before T027) is correct, but the separation of T027 from the metrics phase (US3) where its output is consumed creates a semantic gap. The task T027 produces the artifact `data/processed/aligned_pairs.json` (T029), which is the direct input for T032. This is a valid producer-consumer chain, but the 'Independent Test' for US2 claims it can be tested independently. This is only true if mock data is used. If real data is required, US2 cannot be tested independently of US1's successful completion. The 'Independent Test' claim for US2 is semantically violated if it requires real US1 output.
- Task T032 (Implement `code/04_metrics.py`) depends on the output of T029 (`data/processed/aligned_pairs.json`). T029 is in Phase 4 (US2). T032 is in Phase 5 (US3). The ordering is correct (Phase 4 before Phase 5). However, T032 also depends on T019 (Raw JSON reports) for the tool issues. The dependency on T019 is indirect (via T027/T029). The task T032 description 'calculate Precision, Recall... per tool' implies it needs the tool issues (from T019) and the aligned pairs (from T029). The current order (T019 -> T027 -> T029 -> T032) is valid. However, T037 (Capture-Recapture) is listed in Phase 5. Capture-Recapture requires the overlap between tool findings (T019) and human annotations (T022-T026). The task T037 description 'Implement Capture-Recapture sensitivity analysis' is correct, but it relies on the same inputs as T032. The ordering is fine, but the 'Independent Test' for US3 claims it can be tested on a 'sample dataset'. This sample dataset must include the aligned pairs. If the sample dataset is generated by US1 and US2, then US3 is not independent. If the sample dataset is mocked, the test is valid. The concern is the ambiguity of 'sample dataset' in the test description. It should explicitly state 'mocked aligned pairs' or 'sample data generated by US1/US2' to clarify the dependency.

The current order (T011 before T013) is acceptable for TDD, but the description of T011 should clarify that it tests the *expected* interface, not the implemented one.
