# Environment Configuration Setup Guide

## Overview
This project uses environment variables to manage API keys and configuration settings.
Sensitive information should never be hardcoded in source files.

## Quick Start

1. **Create your `.env` file:**
 ```bash
 cp.env.example.env
 ```

2. **Add your API keys:**
 Edit `.env` and replace the placeholder values with your actual API keys:
 ```
 HUGGINGFACE_TOKEN=your_actual_token_here
 NCBI_API_KEY=your_actual_key_here
 ```

3. **Verify setup:**
 ```bash
 python code/utils/env_manager.py
 ```

## Required API Keys

### Hugging Face Token
- **Purpose**: Access to datasets hosted on Hugging Face
- **Get it**: https://huggingface.co/settings/tokens
- **Required for**: T011a (Strategy A data fetch)

### NCBI API Key
- **Purpose**: Higher rate limits for NCBI E-utilities (SRA, Entrez)
- **Get it**: https://ncbiinsights.ncbi.nlm.nih.gov/2017/11/02/new-api-keys-for-the-e-utilities/
- **Required for**: T011b (Strategy B FASTQ download)

## Optional Configuration

You can customize the following settings in your `.env` file:

- `RANDOM_SEED`: Integer for reproducible random operations (default: 42)
- `MAX_WORKERS`: Number of parallel workers (default: 4)
- `REQUEST_TIMEOUT`: Network request timeout in seconds (default: 60)
- `CACHE_DIR`: Directory for caching downloaded data (default: data/cache)

## Security Best Practices

1. **Never commit `.env` files** to version control
2. **Use `.gitignore`**: The project's `.gitignore` should exclude `.env`
3. **Rotate keys regularly**: Update API keys periodically
4. **Use environment variables in CI/CD**: Store keys securely in your CI system
5. **Limit key permissions**: Use minimal required permissions for each key

## Troubleshooting

### "Missing required API keys" error
- Ensure your `.env` file exists in the project root
- Verify the file contains the required keys with non-empty values
- Check for typos in key names (case-sensitive)

### "Environment variables not loading"
- Ensure `python-dotenv` is installed: `pip install python-dotenv`
- Check file permissions on `.env`
- Verify the `.env` file format (no extra whitespace around `=`)

### Running tests
Tests for environment management are in `tests/test_env_manager.py`:
```bash
python -m pytest tests/test_env_manager.py -v
```
