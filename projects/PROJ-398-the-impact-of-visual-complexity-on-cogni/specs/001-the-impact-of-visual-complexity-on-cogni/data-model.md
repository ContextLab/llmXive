# Data Model: The Impact of Visual Complexity on Cognitive Load During Remote Meetings

## Overview

This document defines the data structures used throughout the project. All data is stored in `data/` with raw data preserved and derived data versioned. The model supports the extraction of visual metrics, the simulation of participant sessions, and the storage of statistical analysis results.

## Entities

### 1. BackgroundFrame

Represents a single frame from a meeting video or a static background image.

**Attributes**:
- `frame_id`: Unique identifier (string).
- `file_path`: Relative path to the raw image file.
- `entropy`: Shannon entropy of the grayscale image (float).
- `color_variance`: Variance of the RGB channels (float).
- `object_count`: Number of objects detected by YOLOv8n (integer).
- `complexity_score`: Composite score (optional, derived).
- `checksum`: SHA-256 hash of the raw file.

**Source**: Generated by `01_extract_metrics.py`.
**Destination**: `data/processed/metrics.json`.

### 2. ParticipantSession

Represents one trial or session for a simulated participant.

**Attributes**:
- `session_id`: Unique identifier (string).
- `participant_id`: Unique participant identifier (string).
- `stimulus_id`: Reference to `BackgroundFrame.frame_id`.
- `stimulus_complexity`: Pre-computed complexity metric (float).
- `task_difficulty`: Simulated difficulty level (float, 0-1).
- `order_position`: Position in the sequence (integer).
- `nasa_tlx_score`: Subjective cognitive load score (float, 0-100).
- `reaction_time_ms`: Mean reaction time (float, ms).
- `accuracy_pct`: Accuracy percentage (float, 0-100).
- `is_missing`: Boolean flag for missing data.
- `excluded`: Boolean flag for exclusion (e.g., attention check fail).

**Source**: Generated by `02_generate_synthetic_data.py`.
**Destination**: `data/processed/sessions.csv`.

### 3. AnalysisResult

Represents the output of the statistical model.

**Attributes**:
- `analysis_id`: Unique identifier.
- `model_formula`: String representation of the LMM formula.
- `predictor`: Name of the predictor variable (e.g., "entropy").
- `estimate`: Fixed effect coefficient (float).
- `std_err`: Standard error (float).
- `t_value`: T-statistic (float).
- `p_value`: Raw p-value (float).
- `p_value_adj`: Benjamini-Hochberg adjusted p-value (float).
- `ci_lower`: 95% Confidence Interval lower bound (float).
- `ci_upper`: 95% Confidence Interval upper bound (float).
- `vif`: Variance Inflation Factor for the predictor (float).
- `effect_size`: Cohen's d or similar (float).
- `significance_flag`: Boolean (True if `p_value_adj < alpha`).

**Source**: Generated by `03_analyze.py`.
**Destination**: `data/processed/analysis_results.csv`.

## Data Flow

1. **Raw Input**: `data/raw/stimuli/*.jpg` (Background images).
2. **Metric Extraction**: `01_extract_metrics.py` reads raw images -> writes `data/processed/metrics.json`.
3. **Session Generation**: `02_generate_synthetic_data.py` reads `metrics.json` -> writes `data/processed/sessions.csv`.
4. **Statistical Analysis**: `03_analyze.py` reads `sessions.csv` and `metrics.json` -> writes `data/processed/analysis_results.csv`.

## Constraints

- **Immutability**: Raw images in `data/raw/stimuli` must never be modified.
- **Checksums**: Every file in `data/` must have a corresponding checksum in `state/`.
- **No PII**: `participant_id` is synthetic and non-reversible.
- **Format**: CSV for tabular data, JSON for nested metrics.
