# Quickstart: Exploring the Correlation Between Musical Preference and Personality Traits

## 1. Prerequisites

*   Python 3.11+
*   `pip` or `conda`
*   Access to GitHub Actions (for CI) or local environment

## 2. Installation

1.  Clone the repository.
2.  Create a virtual environment:
    ```bash
    python -m venv venv
    source venv/bin/activate  # On Windows: venv\Scripts\activate
    ```
3.  Install dependencies:
    ```bash
    pip install -r requirements.txt
    ```
    *Dependencies include: pandas, numpy, scipy, statsmodels, seaborn, matplotlib, openml, requests, pytest, pingouin.*

## 3. Data Setup

**CRITICAL NOTE ON DATA STRATEGY**: Due to the lack of a verified public merged dataset containing both BFI-2 and Last.fm data for the same users, the primary execution path for this project uses a **Verified Synthetic Proxy**. This dataset is generated by `code/synthetic_data.py` using a fixed random seed to ensure reproducibility. The pipeline does NOT attempt to join OpenML BFI-2 and Last.fm datasets by default, as this join has been determined to fail (no common users).

1.  Create the data directories:
    ```bash
    mkdir -p data/raw data/processed results
    ```
2.  **Generate the Verified Synthetic Dataset** (Primary Path):
    ```bash
    python code/synthetic_data.py
    ```
    *This script generates `data/processed/merged_data.csv` containing simulated but structurally valid BFI-2 and genre preference data. This is the required input for the analysis pipeline.*

3.  *(Optional, Fallback)* **Attempt Real Data Join**:
    If you wish to attempt the original strategy (which is expected to fail or produce very low N), you can run:
    ```bash
    python code/data_ingestion.py
    ```
    *Note: This script will attempt to fetch OpenML BFI-2 and Last.fm data. If no common `user_id` is found, it will log a critical error and exit. Do not use this output for the primary analysis unless a valid merge is confirmed.*

## 4. Running the Analysis

1.  Execute the analysis pipeline (uses `data/processed/merged_data.csv`):
    ```bash
    python code/analysis.py
    ```
    *This generates `data/processed/analysis_results.csv` and `results/coefficient_deltas.csv`.*

2.  Generate visualizations and the final report:
    ```bash
    python code/visualization.py
    ```
    *This creates `results/correlation_heatmap.png` and `results/results_report.csv`.*

## 5. Verification

1.  **Check Output Files**:
    *   `data/processed/merged_data.csv`: Contains the verified synthetic dataset (or real data if the join succeeded).
    *   `data/processed/analysis_results.csv`: Contains correlation and regression results.
    *   `results/results_report.csv`: Contains effect sizes and significance flags.
    *   `results/correlation_heatmap.png`: Visual heatmap of correlations.

2.  **Run Tests**:
    ```bash
    pytest tests/
    ```
    *Tests verify schema compliance, data integrity, and statistical logic.*

## 6. Troubleshooting

*   **Data Download Failed**: If you ran `data_ingestion.py` and it failed, this is expected behavior due to the lack of overlapping users. Please use the synthetic data path (Step 3, Bullet 2) instead.
*   **Memory Error**: If the dataset is too large, reduce the sample size in `config.py` or enable streaming in `data_ingestion.py`.
*   **Statistical Warnings**: If `statsmodels` or `pingouin` warns about collinearity, check the `code/analysis.py` logs for VIF values > 5.
