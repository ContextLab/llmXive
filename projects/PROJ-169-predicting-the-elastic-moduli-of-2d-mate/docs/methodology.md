# Methodology: Surrogate Modeling for Elastic Moduli Prediction

## Overview

This project implements a **Structure-Only Surrogate Model** to predict the elastic moduli (Young's modulus, Shear modulus, and Poisson's ratio) of two-dimensional (2D) materials. The model is a **Graph Neural Network (GNN)** that learns to interpolate pre-computed Density Functional Theory (DFT) results.

It is critical to distinguish between the **first-principles calculations** that generated the training data and the **machine learning surrogate** that predicts properties from structure. The surrogate model does **not** solve the Schrödinger equation, calculate electron densities, or perform quantum mechanical simulations. It is a statistical mapping function trained on existing DFT data.

## Distinction: First-Principles vs. Surrogate Methods

### First-Principles (DFT)
The ground truth data used in this project originates from high-fidelity quantum mechanical simulations performed by external repositories (e.g., Materials Project, AFLOW). These simulations solve the many-body Schrödinger equation (approximated via DFT) to determine the electronic structure and total energy of a material.

- **Mechanism**: Solves for electron density $\rho(r)$ and wavefunctions $\psi$.
- **Output**: Elastic tensors derived from stress-strain relationships calculated from energy derivatives.
- **Cost**: Computationally expensive (hours to days per material on HPC clusters).
- **Role in this project**: Provides the *training labels* (target values) and *input structures* (CIF files).

### Surrogate (Machine Learning)
The model implemented here is a **surrogate** (or emulator) that approximates the input-output relationship of the DFT calculations.

- **Mechanism**: A Graph Neural Network (GNN) processes the crystal structure as a graph of atoms (nodes) and bonds (edges). It learns non-linear mappings from structural descriptors to elastic moduli.
- **Output**: Predicted elastic moduli based on learned correlations.
- **Cost**: Negligible (milliseconds per material on CPU/GPU).
- **Role in this project**: Rapidly predicts properties for new 2D materials within the chemical space covered by the training data.

**Key Limitation**: As a surrogate, this model is an **interpolator**, not an extrapolator. It cannot discover new physics or predict properties for materials significantly outside the chemical and structural space of the training DFT data. It does not "understand" the Hamiltonian; it recognizes patterns in the data generated by the Hamiltonian.

## Data Sources

The training data is sourced from established, public DFT databases. This project adheres to the principle of using a **single canonical source** per execution run to ensure reproducibility and consistency.

1. **Materials Project**
 - **Reference**: Jain, A. et al. "Commentary: The Materials Project: A materials genome approach to accelerating materials innovation." *APL Materials* 1, 011002 (2013).
 - **Access**: API-based retrieval of CIF structures and computed elastic tensors.
 - **DFT Settings**: Typically GGA-PBE functional, specific cutoff energies, and k-point densities as defined by the repository.

2. **AFLOW**
 - **Reference**: Curtarolo, S. et al. "AFLOW: An Automatic Framework for Large-Scale Materials Discovery." *Computational Materials Science* 58, 227–235 (2012).
 - **Access**: API or bulk download of crystallographic data and elastic properties.
 - **DFT Settings**: Standardized protocols for high-throughput DFT calculations.

*Note: The specific source used for a given run is determined by the configuration in `code/ingest/download.py`.*

## Pipeline Architecture

The methodology follows a strict four-stage pipeline:

1. **Ingestion & Parsing**
 - Download CIF files and elastic tensors from the selected DFT source.
 - Parse crystallographic data into `MaterialGraph` objects using `pymatgen`.
 - **Filtering**: Strictly exclude non-2D materials and entries with missing or invalid 6-component elastic tensors.

2. **Graph Construction**
 - Convert 2D material structures into graph representations.
 - **Nodes**: Atoms, encoded with elemental properties (atomic number, electronegativity, etc.).
 - **Edges**: Bonds, encoded with distance and angular features.
 - **Targets**: Young's, Shear, and Poisson's ratios extracted from the elastic tensor.

3. **Surrogate Model Training**
 - **Architecture**: Lightweight GNN (2-3 layers, hidden dimension ≤ 64) optimized for CPU execution.
 - **Splitting**: Stratified splitting by chemical prototype/space group to ensure **inter-family generalization** testing.
 - **Objective**: Minimize Mean Absolute Percentage Error (MAPE) against held-out DFT values.
 - **Constraint**: Early stopping and memory limits (≤ 7GB RAM) enforced.

4. **Evaluation & Analysis**
 - **Metrics**: MAPE, RMSE, and R² scores reported for Young's, Shear, and Poisson's ratios.
 - **Generalization**: Explicit measurement of performance drop on unseen material families (inter-family) vs. seen families (intra-family).
 - **Interpretability**: SHAP values and ablation studies identify which structural descriptors most strongly influence predictions.

## Compliance & Limitations

- **Terminology**: This project explicitly avoids terms like "First-Principles Prediction" or "Quantum Mechanical Simulation" when referring to the ML model. The model is a **surrogate interpolator**.
- **Extrapolation**: The model cannot predict properties for materials outside the training distribution. Predictions for novel chemistries are unreliable.
- **Physics Discovery**: The model identifies statistical correlations in DFT data, not fundamental physical laws. It does not solve the Schrödinger equation.
- **Reproducibility**: All runs are seeded, and the exact DFT source used is logged to ensure results can be reproduced.
