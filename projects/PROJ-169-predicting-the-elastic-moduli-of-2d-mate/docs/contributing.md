# Contributing to the Structure-Only Surrogate Model for 2D Material Elastic Moduli

Thank you for your interest in contributing to this project. This document outlines the guidelines, workflow, and terminology standards for all contributors.

## 🚨 Critical Terminology Guidelines

**READ THIS FIRST:** This project implements a **Machine Learning Surrogate Model** that interpolates pre-computed Density Functional Theory (DFT) data. It does **NOT** solve the Schrödinger equation, perform first-principles calculations, or discover new quantum mechanical laws.

### Forbidden Terms
The following terms are **strictly prohibited** when describing the ML model, its outputs, or its methodology:
- "First-Principles" (or "First Principles")
- "Schrödinger equation" (unless explicitly contrasting with the surrogate method)
- "Hamiltonian" (unless explicitly contrasting with the surrogate method)
- "Ab initio"
- "Quantum mechanical calculation" (in the context of the model's operation)
- "Physics discovery"

### Required Terminology
Use the following terms instead:
- "Surrogate Model" or "ML Surrogate"
- "Interpolation of DFT data"
- "Statistical correlation"
- "Predicted values" (never "calculated values" in the physics sense)
- "Structure-Only" (emphasizing the input data scope)

**Example of Correct Usage:**
> "The model predicts elastic moduli by interpolating patterns learned from pre-computed DFT datasets (Materials Project/AFLOW). These are statistical correlations, not solutions to the Schrödinger equation."

**Example of Incorrect Usage (VIOLATION):**
> "Our model performs first-principles calculations to derive the Hamiltonian and solve for the elastic moduli." (This is scientifically false and will be rejected.)

## Code Style and Quality

We enforce strict code quality standards to ensure reproducibility and scientific integrity.

- **Linting**: We use `ruff` with strict rules (E, F, W, I) and a line length of 88.
 ```bash
 ruff check code/
 ```
- **Formatting**: We use `black` with a line length of 88.
 ```bash
 black code/
 ```
- **Type Hinting**: All new functions and classes must include type hints.
- **Docstrings**: Follow the Google style guide. Every public function must have a docstring describing its purpose, arguments, and return values.

## Scientific Integrity and Disclaimers

Every output artifact generated by this project (JSON logs, Markdown reports, CSVs) must include the following disclaimer in its metadata or header:

> "These results are derived from a machine learning surrogate model interpolating pre-computed DFT data. They do not represent first-principles calculations or solutions to the Schrödinger equation."

This is a mandatory requirement enforced by the `code/utils/terminology_scanner.py` and `code/analysis/report_generator.py`. Do not remove or alter this text.

## Workflow

1. **Fork** the repository.
2. **Create a feature branch** from `main` (e.g., `feature/us3-ablation-study`).
3. **Implement your changes** ensuring they adhere to the terminology guidelines above.
4. **Run the linter and formatter**:
 ```bash
 ruff check code/
 black code/
 ```
5. **Run the terminology scanner** to ensure no forbidden terms were introduced:
 ```bash
 python code/utils/terminology_scanner.py
 ```
6. **Run tests** to ensure no regressions:
 ```bash
 pytest tests/
 ```
7. **Submit a Pull Request** with a clear description of your changes.

## Review Process

All Pull Requests will be reviewed for:
- Code quality and style compliance.
- Adherence to the **Terminology Guidelines** (strictly enforced).
- Scientific accuracy (ensuring claims match the "Surrogate" nature of the model).
- Reproducibility (seeds pinned, data sources documented).

If your PR uses forbidden terminology or makes false claims about "first-principles" capabilities, it will be rejected immediately.

## Questions?

If you are unsure about terminology or scientific claims, please open an issue or ask in the pull request discussion before proceeding. Refer to `docs/methodology.md` for the detailed distinction between First-Principles and Surrogate methods.
