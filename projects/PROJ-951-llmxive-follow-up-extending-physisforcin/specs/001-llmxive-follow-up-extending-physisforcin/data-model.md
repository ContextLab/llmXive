# Data Model: llmXive follow-up: extending "PhysisForcing: Physics Reinforced World Simulator for Robotic Manipula"

## 1. Overview

This document defines the data structures, schemas, and flow for the `001-llmxive-physs-filter` project. All data artifacts are stored under `projects/PROJ-951-llmxive-follow-up-extending-physisforcin/data/`.

## 2. Entity Definitions

### 2.1 VideoSample
A single generated robotic manipulation video with associated metadata.
- **Origin**: Generated by Wan2.1.
- **Status**: `raw` (pre-filter), `curated` (post-filter), or `control` (random subset).
- **Key Fields**: `video_id`, `prompt`, `physics_score` (composite), `continuity_score`, `contact_score`, `pass_status`.
- **Schema**: Refer to `contracts/video_sample.schema.yaml`.

### 2.2 CuratedDataset
A collection of `VideoSample` entities that have passed the physics filter.
- **Constraint**: All samples must have `physics_score` Ôëą 60th percentile of the batch.
- **Size**: Target n Ôëą 50 (if starting n=1000).
- **Schema**: Refer to `contracts/curated_dataset.schema.yaml`.

### 2.3 ControlDataset
A collection of `VideoSample` entities selected randomly from the raw set (same size as Curated).
- **Constraint**: Selected via random seed to ensure independence.
- **Size**: Target n Ôëą 50.

### 2.4 TrainedModel
The state of a large-scale parameter diffusion model after training.
- **Format**: PyTorch state dict (`.pt`).
- **Metadata**: Training config, epoch count, final loss, `dataset_type` (curated/control).

### 2.5 BenchmarkResult
The outcome of evaluating a model on R-Bench or PAI-Bench.
- **Fields**: `model_id`, `benchmark_name`, `score`, `p_value`, `n_samples`, `evaluation_engine` (must be "MuJoCo").
- **Schema**: Refer to `contracts/benchmark_result.schema.yaml`.

### 2.6 MuJoCoValidationResult
The outcome of validating the PyBullet filter against MuJoCo.
- **Fields**: `correlation_coefficient`, `distinctness_metric`, `pass_status`.
- **Schema**: Refer to `contracts/mujo_co_validation_result.schema.yaml`.

## 3. Data Flow

1.  **Ingestion**: Prompts loaded from `Rapidata` parquet Ôćĺ `src/generation/prompts.py` Ôćĺ `data/prompts.jsonl`.
2.  **Generation**: Wan2.1 generates videos Ôćĺ `data/raw/` (MP4) + `data/raw/metadata.jsonl`.
3.  **Filtering**: PyBullet scores videos Ôćĺ `data/curated/` (subset of MP4) + `data/curated/scores.parquet`.
4.  **Control**: Random subset selected Ôćĺ `data/control/` (subset of MP4) + `data/control/scores.parquet`.
5.  **Validation**: MuJoCo scores curated videos Ôćĺ `data/validation/mujo_co_validation_result.json` (T018).
6.  **Training**: Curated/Control data Ôćĺ `src/training/trainer.py` Ôćĺ `models/filtered_model.pt`, `models/control_model.pt`.
7.  **Benchmarking**: Trained models Ôćĺ `src/evaluation/` (MuJoCo) Ôćĺ `data/eval/results.json`.
8.  **Stats**: Benchmark results Ôćĺ `src/evaluation/stats.py` Ôćĺ `data/eval/stats_report.json`.

## 4. File Formats

- **Videos**: `.mp4` (H.264 codec, 24fps, 512x512).
- **Metadata**: `.jsonl` (JSON Lines) for streaming compatibility.
- **Scores**: `.parquet` (for efficient columnar access).
- **Results**: `.json` (for easy parsing by reporting tools).

## 5. Data Hygiene Rules

- **Immutability**: Files in `data/raw/` are never modified.
- **Checksums**: Every file in `data/` must have a corresponding SHA-256 hash recorded in `state.yaml`.
- **No PII**: All prompts are synthetic; no real-world PII is used.
