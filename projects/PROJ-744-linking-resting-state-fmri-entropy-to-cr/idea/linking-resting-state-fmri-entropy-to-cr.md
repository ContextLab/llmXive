# Linking Resting‑State fMRI Entropy to Creative Problem Solving  

**Field**: neuroscience  

## Research question  

Do individual differences in whole‑brain resting‑state fMRI entropy predict performance on a standardized divergent‑thinking task (Alternative Uses Test)?

## Motivation  

Resting‑state neural dynamics exhibit scale‑free, fractal‑like fluctuations that have been linked to cognitive flexibility. However, it remains unclear whether these intrinsic complexity signatures relate to higher‑order creative abilities. Demonstrating such a link would provide a mechanistic marker of creativity that can be accessed non‑invasively and could inform interventions aimed at enhancing creative problem solving.

## Related work  

- [Fractal‑driven distortion of resting state functional networks in fMRI: a simulation study (2012)](https://arxiv.org/abs/1208.0924) — shows that fractal (scale‑invariant) properties shape resting‑state functional connectivity, supporting entropy as a meaningful measure of intrinsic neural complexity.  
- [Resting‑State EEG Network Profiles Associated with Creative Engagement and Creative Self‑Efficacy (2025)](https://arxiv.org/abs/2510.22364) — reports associations between intrinsic EEG network topology and self‑rated creative engagement, providing a precedent for linking resting‑state network metrics to creativity.  
- [A Possible Role for Entropy in Creative Cognition (2016)](https://arxiv.org/abs/1611.03605) — argues conceptually that entropy‑based analyses can capture the stochastic processes underlying creative thought, motivating empirical testing with neuroimaging data.

## Expected results  

We anticipate a statistically significant relationship (p < 0.05, corrected) between higher whole‑brain rs‑fMRI entropy and better Alternative Uses Test scores, after controlling for age, sex, and head‑motion. A null finding (no reliable association) would also be informative, indicating that entropy alone does not capture the neural substrate of creative cognition.

## Methodology sketch  

- **Data acquisition**: Download the pre‑processed resting‑state fMRI 4‑D volumes and behavioral phenotypes (Alternative Uses Test scores) from the Human Connectome Project (HCP) via the OpenNeuro/HCP S3 bucket (≈2 GB).  
- **Parcellation**: Apply the HCP multimodal cortical atlas (360 parcels) to extract voxel time series per parcel.  
- **Entropy computation**: Compute multiscale sample entropy (MSE) for each parcel’s time series using a vectorized Python implementation (e.g., `pyentropy`).  
- **Aggregation**: Derive (a) a global mean entropy across all parcels and (b) network‑specific averages for canonical networks (DMN, FPN, CON, etc.).  
- **Covariate extraction**: Pull age, sex, and framewise displacement (head‑motion) from the HCP phenotypic file.  
- **Statistical modeling**: Fit linear mixed‑effects models (via `statsmodels` or `lme4` in R) with entropy metrics as fixed effects, participant ID as random intercept, and covariates as additional fixed effects.  
- **Multiple‑comparison correction**: Apply Benjamini‑Hochberg FDR correction across the set of network‑specific tests.  
- **Robustness checks**: (i) Repeat analyses using alternative entropy parameters (different template lengths, scales); (ii) Test reverse‑causality by swapping predictor/predicted roles; (iii) Verify that results persist when excluding participants with excessive motion (>0.2 mm).  
- **Runtime management**: All steps are scripted to run sequentially; entropy calculation is parallelized over parcels but limited to ≤4 CPU cores to stay within GitHub Actions limits. Expected total wall‑time ≤ 45 minutes.
