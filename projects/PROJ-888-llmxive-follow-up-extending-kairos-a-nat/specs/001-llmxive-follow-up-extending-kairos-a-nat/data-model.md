# Data Model: llmXive follow-up: extending "Kairos: A Native World Model Stack for Physical AI"

## Overview

This document defines the data structures, schemas, and relationships used throughout the project. It ensures alignment with the "Single Source of Truth" (Constitution Principle IV) and "Data Hygiene" (Principle III) principles.

## Key Entities

### 1. DiscreteStateVector
Represents the quantized state of the embodied agent at a single time step.
- **Source**: Derived from `LIBERO` HDF5 `states` array.
- **Format**: JSON (serialized).
- **Fields**:
    - `timestamp`: float (seconds)
    - `bit_depth`: int (4, 8, or 16)
    - `noise_std`: float (0.0, 0.1, 0.3, 0.5)
    - `state_vector`: list[int] (The quantized integer values)
    - `metadata`: dict (Scene ID, episode ID)

### 2. PredictionHorizon
Defines the target length for long-horizon forecasting.
- **Values**: 100, 500, and 1000 steps (per Constitution Principle VII).
- **Usage**: Used to slice input/output sequences for training and evaluation.

### 3. ErrorMetric
Composite record of model performance.
- **Fields**:
    - `run_id`: str (Unique identifier for the independent run)
    - `bit_depth`: int
    - `noise_std`: float
    - `horizon`: int
    - `mse`: float (Mean Squared Error between decoded prediction and continuous ground truth)
    - `cumulative_error_rate`: float (MSE growth per step)
    - `credible_interval`: dict (Lower and upper bounds from BHM)
    - `is_significant`: bool (True if credible interval excludes zero)

## Data Flow

1.  **Raw Data**: `data/raw/libero_*.h5` (Checksummed)
2.  **Processed Data**: `data/processed/discrete_vectors_*.json` (Generated by `quantize.py`)
3.  **Model Checkpoints**: `data/models/checkpoint_epoch_*.pt` (CPU-compatible)
4.  **Results**: `data/results/error_metrics.csv` (Aggregated statistics)

## Data Hygiene Rules

- **Checksumming**: Every file in `data/raw/` is checksummed (SHA-256) upon download.
- **Immutability**: Raw HDF5 files are never modified. Quantization outputs are written to new files with unique hashes.
- **Derivation Log**: `data/derivation_log.json` records the exact parameters (bit-depth, noise, seed) used to generate each processed file.
- **PII Scan**: All generated JSON files are scanned for PII (unlikely, but enforced by Constitution Principle III).
