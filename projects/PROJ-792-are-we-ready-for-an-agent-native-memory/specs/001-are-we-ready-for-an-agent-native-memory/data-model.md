# Data Model: Are We Ready For An Agent-Native Memory System?

## Overview

This document defines the data structures used for ingestion, intermediate processing, and final output in the reproduction pipeline. All data is processed in-memory (with streaming for ingestion) and persisted as JSON/Parquet artifacts.

## Entities

### 1. Raw Dataset (Ingested)
The raw input data loaded from the verified HuggingFace sources.
- **Source**: `MixSub-LLaMA`, `OOM-Test`.
- **Format**: Parquet (streamed).
- **Schema**:
    - `id`: string (unique identifier)
    - `text`: string (content to be memorized)
    - `label`: string (ground truth for retrieval/extraction)
    - `context_length`: integer (calculated length of text)

### 2. Synthetic Update Protocol (Injected Data)
A programmatic layer that injects "update" events into the static dataset to simulate the Maintenance module.
- **Schema**:
    - `original_id`: string (reference to the raw dataset item)
    - `update_instruction`: string (e.g., "Change 'X' to 'Y'")
    - `expected_state`: string (the ground truth text after update)
    - `update_type`: string (e.g., "Correction", "Addition", "Deletion")

### 3. Memory Artifact (Internal State)
The structured representation of a memory system's state after ingestion.
- **Vector Store**:
    - `embeddings`: numpy array (float32, shape: [N, D])
    - `documents`: list of strings (indexed by ID)
- **Graph Store**:
    - `nodes`: dict {id: {type, text}}
    - `edges`: list of tuples (source_id, target_id, relation)
- **Rule-Based**:
    - `keywords`: set of strings
    - `patterns`: list of regex patterns

### 4. Metric Artifact (Output)
The raw results generated by the evaluation modules.
- **Type**: JSON/CSV
- **Schema**:
    - `system_name`: string (e.g., "VectorStore-MiniLM")
    - `module`: string (e.g., "Retrieval", "Maintenance")
    - `dataset_subset`: string (e.g., "MixSub-10%")
    - `metric_name`: string (e.g., "Precision@5", "UpdateSuccessRate")
    - `value`: float (0.0 - 1.0)
    - `retrieval_cost_units`: float (e.g., number of embedding calls)
    - `maintenance_cost_units`: float (e.g., number of write operations)
    - `timestamp`: string (ISO8601)
    - `downsampled`: boolean (true if memory guard triggered)
    - `update_protocol_applied`: boolean (true if Maintenance was tested via synthetic protocol)

### 5. Summary Report (Final Output)
The aggregated analysis document.
- **Type**: Markdown
- **Content**:
    - Table of systems sorted by `Retrieval Efficiency` and `Maintenance Efficiency`.
    - Comparison of Local vs. Global maintenance strategies (with caveats).
    - Log of any edge cases triggered.

## Data Flow

1.  **Ingestion**: `loader.py` reads Parquet -> `Raw Dataset` (streamed).
2.  **Preprocessing**: `preprocessor.py` filters/samples -> `Processed Batch`.
3.  **Synthetic Update Injection**: `maintenance.py` generates `Synthetic Update Protocol` data for the `Processed Batch`.
4.  **Ingestion into Memory**: `modules/representation.py` creates `Memory Artifact`.
5.  **Evaluation**: 
    - `modules/retrieval.py` queries `Memory Artifact` -> generates `Metric Artifact` rows for Retrieval.
    - `modules/maintenance.py` applies `Synthetic Update Protocol` -> generates `Metric Artifact` rows for Maintenance.
6.  **Aggregation**: `aggregator.py` reads `Metric Artifact` -> generates `Summary Report`.

## Constraints

- **Memory**: `Memory Artifact` must not exceed 6GB. If `embeddings` array size * 4 bytes > 5GB, trigger downsampling.
- **Precision**: All float metrics must be rounded to 4 decimal places to ensure reproducibility and reduce file size.
- **Validation**: All `Metric Artifact` values must be within [0.0, 1.0]. Values outside this range trigger a validation error.
- **Cost Floor**: Cost values used in efficiency calculations must have a minimum floor of 1 to prevent division by zero.
