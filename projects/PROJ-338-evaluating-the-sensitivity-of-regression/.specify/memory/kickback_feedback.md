## Unresolved concerns

- The plan defines IQR and Z-score outlier removal using 'Intersection logic' (remove row only if outlier in ALL continuous features). This is statistically unsound for multivariate data. A point can be an extreme outlier in a single dimension (e.g., a house with 0 bedrooms) while being normal in others. Intersection logic will fail to remove these univariate outliers, rendering the 'sensitivity' analysis blind to the most common type of data error. The methodology should use 'Union logic' (remove if outlier in ANY feature) or a multivariate distance metric (e.g., Mahalanobis) to validly assess sensitivity to outlier removal.
- The proposed Binomial test ($H_0$: proportion = 0.5) is methodologically invalid for the stated research question. The null hypothesis assumes that 'flips' (changes in significance) are random coin flips between strategies. However, if a strategy is *better* (e.g., removes true noise), we expect a systematic shift, not a 50/50 random split. Testing against 0.5 does not answer 'which strategy is more stable' or 'does the choice matter'; it only tests if the strategies are equally likely to flip, which is a trivial and likely false assumption. The analysis should instead compare the *magnitude* of coefficient changes or use a paired test (e.g., Wilcoxon signed-rank) on the absolute delta of coefficients across datasets to detect systematic differences.
- The plan applies Cook's Distance (>4/n) for outlier removal. Cook's Distance identifies *influential* points, not necessarily *outliers* in the feature space. Removing influential points changes the regression fit by definition (it removes the leverage). Comparing IQR/Z-score (feature-space outliers) against Cook's (influence-based) conflates two different concepts. The 'sensitivity' measured here will be a mix of 'sensitivity to feature outliers' and 'sensitivity to influential points'. The methodology should either separate these analyses or explicitly frame the comparison as 'Sensitivity to Feature Outliers vs. Sensitivity to Influential Points' rather than treating them as equivalent 'outlier removal strategies'.
- The plan relies on the `ucimlrepo` library to fetch 15 UCI regression datasets without specifying which 15. The library contains hundreds of datasets, many of which are classification tasks, time-series, or have missing target variables. Without a hardcoded list of specific dataset IDs (e.g., 'california_housing', 'wine_quality_red', 'airfoil_self_noise') in `config.py` or the plan, the pipeline cannot deterministically select the '15 regression datasets' required by the spec. This risks the pipeline failing to find 15 valid regression targets or selecting inappropriate datasets (e.g., classification) that break the OLS regression logic.

- The Binomial test hypothesis (H0: proportion of flips = 0.5) is statistically unsound for this design. A 'flip' (p < 0.05 vs p >= 0.05) is a binary outcome of a continuous p-value distribution. Under the null hypothesis that outlier removal has *no systematic effect* on significance, the probability of a flip is not 0.5; it depends entirely on the baseline power and the distribution of p-values near the 0.05 threshold in the raw data. If most raw p-values are 0.99, removing outliers is unlikely to flip them to <0.05, making the expected flip rate near 0, not 0.5. Testing against 0.5 creates a tautological or meaningless null that does not represent 'randomness' of the cleaning process.
- The 'Intersection logic' for IQR and Z-score (remove row only if outlier in ALL continuous features) is methodologically flawed for multivariate regression. In high-dimensional spaces, the probability of a point being an outlier in *all* dimensions simultaneously approaches zero, rendering the outlier removal step ineffective (i.e., it will remove almost no data). This makes the 'sensitivity' analysis trivial: if no data is removed, the 'Cleaned' model is identical to the 'Raw' model, and the result is a tautology (no change). The standard approach is 'Union logic' (remove if outlier in ANY feature) or Mahalanobis distance, not intersection.
- The Binomial test treats each dataset as an independent Bernoulli trial for 'flips'. However, the 'flip' event is defined per *variable* within a dataset, then aggregated. The plan is ambiguous on whether the unit of analysis for the Binomial test is 'datasets with at least one flip' or 'total number of variable-flips across all datasets'. If it's the latter, the independence assumption is violated because variables within a dataset are correlated (collinearity). If it's the former, the sample size (N=15) is extremely low for a Binomial test to have power, and the definition of 'flip' per dataset is arbitrary (1 flip vs 10 flips treated equally).
