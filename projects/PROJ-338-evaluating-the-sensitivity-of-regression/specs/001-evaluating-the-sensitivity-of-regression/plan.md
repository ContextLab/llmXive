# Implementation Plan: Evaluating the Sensitivity of Regression Models to Outlier Removal Strategies

**Branch**: `001-evaluating-sensitivity-of-regression-models` | **Date**: 2026-07-08 | **Spec**: `specs/001-evaluating-the-sensitivity-of-regression-models/spec.md`
**Input**: Feature specification from `specs/001-evaluating-the-sensitivity-of-regression-models/spec.md`

## Summary

This project implements a reproducible statistical pipeline to evaluate how Ordinary Least Squares (OLS) regression results (coefficients, p-values, R², RMSE) change under different outlier removal strategies (IQR, Z-score, Cook's Distance). The system will download regression datasets from the UCI Machine Learning Repository, preprocess them, run the core analysis loop (Raw vs. Cleaned), perform Wilcoxon Signed-Rank and Paired t-tests on metric deltas, sweep IQR thresholds, and generate a PDF report with visualizations. The implementation strictly adheres to CPU-only constraints (free-tier CI) and frames all findings as associational.

**Methodological Correction Note**: The plan corrects the source spec's requirement for "Intersection logic" (remove if outlier in ALL features) to "Union logic" (remove if outlier in ANY feature) for IQR and Z-score, as intersection logic is statistically unsound for multivariate data. Similarly, the plan replaces the spec's "Binomial test" with Wilcoxon Signed-Rank and Paired t-tests, as the Binomial test's null hypothesis (p=0.5) is methodologically invalid for this design. These changes are flagged for spec kickback.

## Complexity Tracking

| Violation | Why Needed | Simpler Alternative Rejected Because |
|-----------|------------|-------------------------------------|
| **Wilcoxon Signed-Rank & Paired t-test** | Required to statistically validate if metric changes are systematic (replaces invalid Binomial test). | A simple count or Binomial test is insufficient; comparing magnitude of changes requires tests on continuous deltas. |
| **IQR Sweep** | Required by FR-006 to demonstrate threshold stability. | A single threshold (1.5) is arbitrary and risks artifact-based conclusions. |
| **VIF Calculation** | Required by FR-011 to flag collinearity. | Ignoring collinearity could lead to misinterpretation of coefficient instability as outlier sensitivity. |
| **Union Logic for Outliers** | Required by methodology panel to correctly identify univariate outliers (replaces intersection logic). | Intersection logic was found to be statistically unsound for multivariate data, removing almost no outliers. |

## Compute Feasibility
- **Environment**: GitHub Actions Free Tier (2 CPU, 7GB RAM).
- **Strategy**:
  - Datasets are small (UCI regression datasets are typically < 1MB to 10MB).
  - OLS is $O(n \cdot p^2)$, trivial for $n < 10,000$ and $p < 100$.
  - No GPU required. `statsmodels` and `scikit-learn` run efficiently on CPU.
  - Total runtime estimated < 30 minutes for 15 datasets + sweep.
  - **6-Hour Buffer**: The 6-hour limit explicitly includes network latency, retry logic overhead (up to 2 retries per dataset), and CI job overhead, ensuring robustness against transient network issues.
