However, the prior kickback concerns (science-class) explicitly state this null hypothesis is methodologically invalid for this research question. The current FR-005 and SC-001/SC-004 still mandate this invalid test. The current FR-004 treats them as equivalent 'outlier removal strategies' without distinguishing the analysis. The spec must explicitly separate the analysis of 'feature-space sensitivity' (IQR/Z) from 'influence-based sensitivity' (Cook's) or reframe the story to acknowledge this conflation.
- User Story 2 requires applying 'Intersection logic' for IQR/Z-score. The kickback concerns (science-class) explicitly state this is 'statistically unsound' and 'methodologically flawed' for multivariate data, likely resulting in no data removal. The spec must be revised to use 'Union logic' or a multivariate metric (e.g., Mahalanobis) to ensure the sensitivity analysis is valid.
- User Story 2 requires processing 'a list of 15 valid UCI dataset identifiers'. The kickback concerns (science-class) state that the plan lacks a 'hardcoded list of specific dataset IDs', making the selection non-deterministic and unverifiable. - FR-004 mandates 'Intersection logic' for IQR and Z-score. As noted in the kickback, this logic is methodologically flawed for multivariate regression. The requirement must be updated to reflect the corrected logic (Union or Mahalanobis) to align with the valid research methodology.
- FR-005 mandates a 'Binomial test' with H0: proportion = 0.5. The kickback concerns identify this as statistically unsound. The requirement must be updated to specify a valid statistical test (e.g., Wilcoxon signed-rank on coefficient changes) to ensure the research question can be answered.

Cook's Distance identifies points that *change* the fit by definition; removing them guarantees a change in coefficients. Comparing this to feature-space outliers (IQR) creates a tautological result where Cook's Distance will always show 'high sensitivity'. The requirement must separate these analyses or explicitly define the comparison metric to account for this methodological difference, otherwise the 'sensitivity' result is uninterpretable.

- FR-004 mandates 'intersection logic' (remove row only if outlier in ALL continuous features) for IQR and Z-score. In multivariate regression, this logic is statistically unsound: the probability of a point being an outlier in *all* dimensions simultaneously is near zero, rendering the removal step ineffective. - FR-005 requires a Binomial test with H0: proportion of flips = 0.5. This null hypothesis is scientifically invalid for this design. A 'flip' (p<0.05 vs p>=0.05) is not a random coin flip; its probability depends on the baseline power and the distribution of p-values near the threshold. If raw p-values are mostly 0.99, the expected flip rate is near 0, not 0.5. Testing against 0.5 does not measure 'sensitivity' or 'stability' but rather tests a false assumption of randomness. The analysis must instead compare the magnitude of coefficient changes (e.g., Wilcoxon signed-rank on absolute delta) or test if the flip rate differs from the baseline expectation derived from the raw data distribution.
- FR-004 conflates 'outlier removal' (IQR/Z-score, feature-space) with 'influential point removal' (Cook's Distance, model-space). Cook's Distance identifies points that *change* the fit by definition; removing them guarantees a change in coefficients. Comparing this to feature-space outlier removal conflates two distinct concepts. The 'sensitivity' measured will be a mix of 'sensitivity to feature outliers' and 'sensitivity to influential points'. The spec must either separate these analyses (e.g., 'Sensitivity to Feature Outliers' vs 'Sensitivity to Influential Points') or explicitly frame the comparison as such, rather than treating them as equivalent strategies for the same research question.
- The Binomial test in US-2 treats each dataset as an independent Bernoulli trial for 'flips'. However, the 'flip' event is defined per *variable* within a dataset. If the unit of analysis is 'total variable-flips', the independence assumption is violated due to collinearity between variables within a dataset. If the unit is 'datasets with at least one flip', the sample size (N=15) is too low for a Binomial test to have statistical power, and the definition of 'flip' per dataset is arbitrary (1 flip vs 10 flips treated equally). The statistical design must clarify the unit of analysis and use a method appropriate for the dependence structure (e.g., mixed-effects models or permutation tests).
- FR-001 requires downloading datasets via HTTP GET without specifying *which* 15 datasets. This risks the pipeline failing or selecting inappropriate datasets, violating the assumption of n > 30 and continuous targets required for OLS. The spec must enumerate the specific 15 dataset IDs to ensure reproducibility and validity.

The sensitivity sweep (1.0 to 2.0) is good, but the spec should explicitly state that the methodology assumes the data is approximately Gaussian or that the sweep is sufficient to capture non-Gaussian behavior. Without this, the 'sensitivity' results may be artifacts of the threshold choice rather than true data properties.
