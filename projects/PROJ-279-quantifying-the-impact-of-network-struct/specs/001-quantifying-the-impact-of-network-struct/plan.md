# Implementation Plan: Quantifying the Impact of Network Structure on Heat Transport in Amorphous Silicon

**Input**: Feature specification from `/specs/001-quantify-heat-transport/spec.md`

## Summary

This feature implements a **Tiered Execution** pipeline to quantify the correlation between topological descriptors and thermal conductivity in amorphous silicon (a-Si). The workflow supports two modes:
1.  **Full Mode**: If a dataset with pre-calculated thermal conductivity (k) and independent vibrational data (VDOS) is found, it tests all hypotheses (H-001 to H-004).
2.  **Structure-Only Mode**: If k or VDOS is missing, the pipeline skips regression (H-001, H-002) and focuses on topological characterization (H-003, H-004), marking the thermal conductivity hypotheses as "Untestable" rather than failing.

The approach involves downloading MD trajectories, constructing atomic graphs, computing topological descriptors (ring statistics, bond order), and optionally vibrational descriptors (if pre-calculated). Statistical analysis includes Ridge/Random Forest regression with strict dimensionality reduction (PCA/Lasso) and stability selection to handle small sample sizes. The study adheres to strict data hygiene, reproducibility, and independence validation constraints defined in the project constitution.

## Constitution Check

*GATE: Must pass before Phase 0 research. Re-check after Phase 1 design.*

1.  **Principle I (Reproducibility)**: Plan mandates pinning `requirements.txt`, setting random seeds in `code/`, and fetching datasets from canonical sources (Zenodo/HuggingFace) on every run.
2.  **Principle II (Verified Accuracy)**: Plan includes a step to validate citations against primary sources. **Distinction**: The 'Verified Accuracy' gate applies to the specific data modality found. If the dataset lacks thermal conductivity, the gate for H-001/H-002 is marked 'N/A', but the project proceeds with Structure-Only Mode for H-003/H-004, avoiding a hard block on the entire project.
3.  **Principle III (Data Hygiene)**: Plan requires checksumming all files in `data/` and recording them in `state/`. No in-place modification of raw data; derivations produce new files.
4.  **Principle IV (Single Source of Truth)**: All figures and statistics will be generated by `code/` scripts, ensuring they trace back to `data/` rows. No hand-typed numbers in `paper/`.
5.  **Principle V (Versioning Discipline)**: Artifacts will carry content hashes; `state/` YAML will be updated on artifact changes.
6.  **Principle VI (Finite-Size Effect Validation)**: **Strict Compliance**: For any system size < 1000 atoms, the pipeline MUST either demonstrate convergence (if multiple sizes are available) OR explicitly flag the result as "Preliminary - Unverified Convergence" and **exclude** it from the primary hypothesis test (H-001/H-002). Passive flagging is replaced with active exclusion.
7.  **Principle VII (Topological Descriptor Consistency)**: Plan explicitly includes the sensitivity analysis over cutoff radii {,, 3.2} Å as required by FR-002 and the constitution.

## Complexity Tracking

| Violation | Why Needed | Simpler Alternative Rejected Because |
|-----------|------------|-------------------------------------|
| Tiered Execution (Full vs. Structure-Only) | Required to prevent total project failure if thermal conductivity data is missing (data_resources-c4dceb4e). | A single-path pipeline would abort entirely, yielding no scientific value if the specific 'verified' dataset is unavailable. |
| Stability Selection & PCA | Required to address overfitting in small N (methodology-0a07cdcd). | Standard regression on N < 30 with ~15 features leads to unstable, non-reproducible feature importance. |
| Covariate Control (Density) | Required to isolate topological effects from density (methodology-332fa9fe). | Without controlling for density, correlations may be spurious proxies for local packing. |
| No Internal VDOS Calculation | Required to avoid construct validity risks (methodology-3351023c). | Mass-spring approximations fail to capture anharmonic scattering, invalidating the vibrational hypothesis test. |
| Active Convergence Check (Principle VI) | Required to satisfy Constitution Principle VI (plan_consistency-eacfc645). | Passive flagging does not demonstrate convergence; exclusion is required for unverified small systems. |

## Hypothesis Execution Logic

- **If** `thermal_conductivity` and `vdos` are present in metadata:
    - Execute Full Mode: Run all regression and correlation tests (H-001, H-002).
    - Validate source independence (distinct ensemble).
- **Else**:
    - Execute Structure-Only Mode: Skip regression.
    - Report H-003 (Ring Stats) and H-004 (Feature Importance on synthetic/relative targets if applicable, otherwise descriptive).
    - Mark H-001/H-002 as "Untestable" in final report.
