## Unresolved concerns

- The plan states 'ingests external entrainment metrics (Phase-Locking Values)' but the research.md admits the only candidate dataset (neurofusion/eeg-restingstate) is labeled 'restingstate' and likely lacks rhythmic entrainment metrics. Correlating resting-state topology with non-existent or mismatched 'entrainment' metrics invalidates the causal/associational claim. The plan must explicitly define the fallback to simulated data as the *only* valid path if real matched data is absent, rather than assuming the pipeline will 'halt' and leave the research question unanswered.
- The plan proposes calculating VIF between Clustering Coefficient and Path Length to detect collinearity, but then proceeds to correlate *each* metric independently with entrainment. If VIF > 5, the metrics are not independent predictors. The methodology fails to address how to interpret the results of two highly collinear predictors in a bivariate correlation framework without a multivariate model (e.g., multiple regression) to disentangle their unique contributions, risking spurious attribution of effect to one metric over the other.
- The research plan relies on an 'Inner Join' of HCP fMRI and external EEG datasets. The research.md explicitly notes these datasets likely have 'disjoint subject IDs'. If the inner join results in N=0 or N < 30 (as warned), the study has no statistical power to detect the effect. The methodology does not propose a mechanism to *generate* the necessary matched data (e.g., simulation) if the join fails, effectively making the research question unanswerable with the proposed data sources.
- SC-002 requires a comparative bar chart showing the absolute difference in effect sizes for *both* AAL and Power 264 against Schaefer. The plan's 'Complexity Tracking' and 'Project Structure' sections mention sensitivity analysis for AAL/Power but do not explicitly define a task or output element that generates a *single* comparative bar chart visualizing the absolute difference for *both* alternative atlases simultaneously. The plan mentions 'Bar charts for SC-002' in `code/viz.py` but lacks the specific logic to compute and plot the 'absolute difference' metric for the Power 264 scheme as a distinct bar in the same chart, risking partial coverage of the success criterion.
- The plan relies on 'HCP-Diffusion-datas' (Lexica.art) and 'HCP-flat' for fMRI connectivity matrices, and 'neurofusion/eeg-restingstate' for entrainment metrics. These datasets are either unrelated (Lexica.art), generic resting-state without rhythmic stimuli, or lack the specific 'Phase-Locking Value' (PLV) column required for the hypothesis. The plan admits this mismatch but proposes a 'Simulated Data Fallback' as the primary execution path. A computational pipeline testing a specific biological hypothesis (topology vs. entrainment) cannot be validated using simulated data, as the correlation results will be artifacts of the simulation parameters, not empirical findings. The dataset strategy fails to identify a real, accessible source containing both HCP fMRI connectivity and rhythmic entrainment metrics for the same subjects.
- The plan assumes the pipeline will 'ingest external entrainment metrics (Phase-Locking Values)' from a dataset that does not exist in the cited sources. The 'neurofusion/eeg-restingstate' dataset contains generic resting-state events, not rhythmic stimulation protocols required to calculate PLV. Without a real dataset providing the dependent variable (entrainment strength), the core analysis (Spearman correlation) cannot be performed on empirical data, rendering the research question unanswerable with the proposed resources.
- Constitution Principle II (Verified Accuracy) is marked 'Pass' in the plan, but the Research document (research.md) explicitly identifies the primary HCP URL as a likely placeholder ('Lexica.art') and the EEG dataset as potentially lacking the required 'rhythmic entrainment' metric. The plan must explicitly reference the 'Simulated Data Fallback' strategy mentioned in research.md to ensure the pipeline remains reproducible if verified datasets are unavailable, aligning with the Constitution's requirement for reproducibility.

- The plan proposes correlating Clustering Coefficient (CC) and Characteristic Path Length (CPL) against Entrainment Strength. However, CC and CPL are mathematically coupled in most graph regimes (often inversely correlated). The plan treats them as independent predictors in separate Spearman tests (H1, H2) without a multivariate model (e.g., multiple regression) to disentangle their effects. If VIF > 5, the plan only 'flags' collinearity but proceeds with univariate tests, which renders the individual p-values statistically uninterpretable regarding the unique contribution of each metric. This is a fundamental validity issue in the analysis design.

The research.md explicitly notes the 'neurofusion/eeg-restingstate' dataset is labeled 'restingstate' and likely lacks 'rhythmic entrainment' metrics (PLV to rhythmic stimuli). The plan's fallback is 'simulated data'. Correlating real fMRI topology with simulated entrainment data is scientifically invalid for testing the hypothesis about *neural* entrainment; it only validates the code pipeline. The core scientific question cannot be answered with the proposed data strategy.
- The plan calculates VIF between CC and CPL to check for collinearity. VIF is a diagnostic for a *multivariate* regression model. The plan, however, performs two separate *univariate* Spearman correlations (CC vs Entrainment, CPL vs Entrainment). Calculating VIF between two predictors without fitting a model that includes both as predictors of the outcome is methodologically inconsistent. If the goal is to test unique effects, a multiple regression (or partial correlation) is required, not separate univariate tests with a post-hoc VIF check.
