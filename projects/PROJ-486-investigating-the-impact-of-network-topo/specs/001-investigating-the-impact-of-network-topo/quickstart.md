# Quickstart: Investigating the Impact of Network Topology on Neural Entrainment to Rhythmic Stimuli

## Prerequisites

*   Python 3.11+
*   `pip`
*   Git

## Installation

1.  **Clone and Navigate**:
    ```bash
    git clone <repo-url>
    cd projects/PROJ-486-investigating-the-impact-of-network-topo
    ```

2.  **Create Virtual Environment**:
    ```bash
    python -m venv venv
    source venv/bin/activate  # On Windows: venv\Scripts\activate
    ```

3.  **Install Dependencies**:
    ```bash
    pip install -r requirements.txt
    ```

## Running the Pipeline

The pipeline is designed to run end-to-end. It will automatically detect missing real data and switch to the **Simulated Data Fallback** (Null Hypothesis Mode by default).

1.  **Execute Main Script**:
    ```bash
    python code/main.py
    ```
    *   *Expected Behavior*:
        *   Checks for `data/raw/entrainment_metrics.csv`.
        *   If missing or N < 30, generates synthetic data (label: "Simulated", r=0 for scientific mode).
        *   Computes graph metrics (Schaefer, AAL, Power).
        *   Runs MLR with Holm-Bonferroni correction and Orthogonalization (if VIF > 5).
        *   Generates visualizations.

2.  **Run Sensitivity Analysis**:
    ```bash
    python code/main.py --atlas AAL
    ```
    *   *Expected Behavior*: Runs analysis with AAL atlas and generates comparative bar chart.

3.  **View Results**:
    *   **CSVs**: `data/processed/joined_data.csv`, `data/processed/correlation_results.csv`.
    *   **Visualizations**: `data/visualizations/scatter_topology_entrainment.png`, `data/visualizations/sensitivity_bar_chart.png`.
    *   **Report**: `docs/report.md` (generated by the analysis step).

## Testing

Run the unit and integration tests:

```bash
pytest tests/
```

*   `tests/unit/test_simulation.py`: Verifies Cholesky decomposition and correlation generation (r=0 and r=0.5).
*   `tests/unit/test_analysis.py`: Verifies Holm-Bonferroni, VIF, and Orthogonalization logic.
*   `tests/unit/test_validation.py`: Verifies "Invalid Entrainment Data" error path (SC-005).
*   `tests/integration/test_pipeline.py`: Runs the full flow and checks output file existence.

## Troubleshooting

*   **"Invalid Entrainment Data"**: Ensure `data/raw/entrainment_metrics.csv` exists and has columns `subject_id` and `entrainment_metric` (numeric).
*   **"Power Warning"**: If N < 30, the system automatically switches to simulated data. Check logs for "Simulated" label.
*   **Collinearity Warning**: If `collinearity_warning` is True, individual p-values are reported for the **orthogonalized** model. Check `data/processed/correlation_results.csv` for the VIF value.
