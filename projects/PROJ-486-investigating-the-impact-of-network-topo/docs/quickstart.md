# Quickstart Guide: Investigating the Impact of Network Topology on Neural Entrainment

This guide provides instructions for setting up and running the `llmXive` pipeline for project **PROJ-486**.

## Prerequisites

- Python 3.9 or higher
- pip (Python package installer)
- A terminal/command line interface

## Installation

1. **Clone the repository** (if not already done):
 ```bash
 git clone <repository-url>
 cd PROJ-486-investigating-the-impact-of-network-topo
 ```

2. **Create a virtual environment** (recommended):
 ```bash
 python -m venv venv
 source venv/bin/activate # On Windows: venv\Scripts\activate
 ```

3. **Install dependencies**:
 ```bash
 pip install -r requirements.txt
 ```

## Data Requirements

The pipeline expects data in the following directory structure:

- `data/raw/`: Contains raw input data.
 - `hcp_correlation_matrices.csv`: Resting-state fMRI correlation matrices (generated by T006 or simulated fallback).
 - `entrainment_metrics.csv`: External entrainment strength metrics (generated by T012c if real data is unavailable).
- `data/processed/`: Contains intermediate and final processed data.
 - `metric_flags.json`: Flags for non-informative metrics.
 - `correlation_results.csv`: Final correlation and regression results.
- `data/visualizations/`: Contains generated plots.
 - `scatter_topology_entrainment.png`: Scatter plot with 95% confidence intervals.
 - `sensitivity_comparison.png`: Comparative bar chart for atlas sensitivity.

**Note**: If real HCP data is unavailable, the pipeline automatically triggers the **Simulated Data Fallback (FR-009)** to generate synthetic data with a target correlation of r=0.5.

## Execution

### Running the Full Pipeline

Execute the main orchestration script to run the entire analysis pipeline:

```bash
python code/main.py
```

This will:
1. Check for data gaps and trigger simulation if necessary.
2. Calculate network topology metrics (Clustering Coefficient, Path Length).
3. Perform correlation analysis and Multiple Linear Regression (MLR).
4. Apply Holm-Bonferroni correction for multiple comparisons.
5. Generate visualizations and save results to `data/`.

### Running Sensitivity Analysis

To re-run the analysis with alternative parcellation schemes (AAL, Power 264):

```bash
python code/main.py --sensitivity
```

This generates a comparative result table and a sensitivity bar chart.

### Running Tests

To verify the implementation:

```bash
pytest tests/
```

## Output Files

After successful execution, the following files will be available:

- `data/processed/correlation_results.csv`: Detailed statistical results including raw p-values, adjusted p-values, VIF values, and collinearity warnings.
- `data/visualizations/scatter_topology_entrainment.png`: Visualization of the relationship between topology metrics and entrainment strength.
- `data/visualizations/sensitivity_comparison.png`: Comparison of effect sizes across different atlases.

## Troubleshooting

- **Missing Data**: If the pipeline reports "Data Gap", it will automatically attempt to generate simulated data. Ensure you have internet connectivity if the simulation requires fetching external packages.
- **Import Errors**: Verify that all dependencies in `requirements.txt` are installed correctly.
- **Memory Issues**: For large datasets, ensure sufficient RAM is available. The pipeline is optimized for CPU-only execution.
