## Core Schemas

### 1. Task Catalog (`data/benchmarks/processed/catalog.json`)

A normalized JSON file containing all benchmark tasks.

**Schema:**
```json
{
 "task_id": "string",
 "prompt": "string",
 "human_solution": "string",
 "test_suite_path": "string",
 "difficulty": "easy|medium|hard",
 "code_patterns": ["loops", "conditionals", "recursion",...],
 "dataset_source": "mbpp|humaneval"
}
```

**Validation:**
- All tasks must have `task_id`, `prompt`, `human_solution`, `test_suite_path`, `difficulty`, and `code_patterns`.
- If `contracts/task_catalog.schema.yaml` exists, the catalog is validated against it.

### 2. Coverage Report (`data/coverage_reports/{task_id}.json`)

Per-task coverage results generated by `coverage_runner.py`.

**Success Schema:**
```json
{
 "task_id": "string",
 "status": "success",
 "line_coverage": float,
 "branch_coverage": float | "N/A",
 "timestamp": "ISO8601 string",
 "model_used": "string",
 "execution_time_seconds": float
}
```

**Failure Schema:**
```json
{
 "task_id": "string",
 "status": "failed",
 "error_message": "string",
 "timestamp": "ISO8601 string"
}
```

**Notes:**
- `branch_coverage` is `"N/A"` for HumanEval tasks (which lack branch coverage data).
- Errors are logged with full tracebacks in `error_message`.

### 3. Statistical Summary (`data/processed/stats_summary.csv`)

Aggregated statistical analysis results from `analyzer.py`.

**Columns:**
- `mean_llm`: Mean coverage for LLM-generated code
- `mean_human`: Mean coverage for human-written code
- `mean_diff`: Mean difference (LLM - Human)
- `p_value`: P-value from statistical test
- `cohen_d`: Effect size (Cohen's d)
- `test_type`: "t-test" or "Wilcoxon" (based on normality test)

**Normality Test:**
- Shapiro-Wilk test is performed first.
- If p < 0.05, Wilcoxon signed-rank test is used; otherwise, paired t-test.

### 4. Corrected P-values (`data/processed/corrected_pvalues.csv`)

Family-wise error corrected p-values for subgroup hypothesis tests.

**Columns:**
- `subgroup`: Name of the subgroup (e.g., "loops", "conditionals")
- `raw_p_value`: Original p-value
- `bonferroni_corrected`: Bonferroni-corrected p-value
- `holm_bonferroni_corrected`: Holm-Bonferroni-corrected p-value

**Exclusion:**
- Sensitivity analysis thresholds are explicitly excluded from correction (FR-011).

### 5. Sensitivity Report (`data/processed/sensitivity_report.csv`)

Sensitivity analysis across significance thresholds.

**Columns:**
- `threshold`: Significance threshold (0.01, 0.05, 0.10, 0.15, 0.20, 0.25)
- `significant_count`: Number of tasks with p < threshold
- `exclusion_rate`: Exclusion rate at this threshold

**Exclusion:**
- These thresholds are not subject to family-wise error correction.

### 6. Stratified Reports (`outputs/stratified_*.csv`)

Stratified analysis by difficulty and code patterns.

**Example: `stratified_loops.csv`**
**Columns:**
- `task_id`: Task identifier
- `difficulty`: easy|medium|hard
- `pattern`: loops
- `line_coverage`: float
- `branch_coverage`: float | "N/A"
- `branch_coverage_gap`: float (LLM - Human, excludes "N/A" tasks)

**Example: `stratified_summary.csv`**
**Columns:**
- `pattern`: Code pattern (loops, conditionals, etc.)
- `difficulty`: easy|medium|hard
- `mean_gap`: Mean branch-coverage gap
- `count`: Number of tasks in this group

**Constraint:**
- HumanEval tasks (`branch_coverage == "N/A"`) are excluded from branch coverage gap calculations.

### 7. Visualizations (`outputs/*.png`)

Generated plots with standardized formatting.

**Specifications:**
- Resolution: 800x600 pixels
- Includes axis labels, legends, and pattern annotations
- File naming: `coverage_by_pattern_<pattern>.png`

## Data Flow

1. **Ingestion:** `dataset_loader.py` downloads and validates raw data.
2. **Generation:** `llm_generator.py` produces code for each task.
3. **Coverage:** `coverage_runner.py` executes tests and generates coverage reports.
4. **Analysis:** `analyzer.py` performs statistical tests and generates summaries.
5. **Stratification:** `visualizer.py` creates stratified reports and visualizations.

## Validation Rules

- **HumanEval Branch Coverage:** Tasks with `task_id` prefix "HumanEval/" or `dataset_source`="humaneval" must have `branch_coverage` set to "N/A".
- **VIF Calculation:** Variance Inflation Factor is calculated only when `--model-method regression` is selected.
- **Error Reporting:** All failures must be recorded in `data/coverage_reports/{task_id}.json` with `status: "failed"`.

## Dependencies

- `pandas`: Data manipulation and analysis
- `scipy`: Statistical tests (Shapiro-Wilk, t-test, Wilcoxon)
- `statsmodels`: Linear mixed models (if applicable)
- `numpy`: Numerical operations
