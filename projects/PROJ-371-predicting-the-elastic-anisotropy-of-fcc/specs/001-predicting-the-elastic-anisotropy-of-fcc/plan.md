# Implementation Plan: Predicting the Elastic Anisotropy of FCC Metals from Composition

**Branch**: `001-elastic-anisotropy` | **Date**: 2024-05-21 | **Spec**: `specs/001-elastic-anisotropy/spec.md`
**Input**: Feature specification from `/specs/001-elastic-anisotropy/spec.md`

## Summary

This project implements a CPU-only data pipeline and regression modeling framework to predict the elastic anisotropy factor ($A_1$) of Face-Centered Cubic (FCC) metals based on compositional descriptors. The system ingests elastic constants ($C_{11}, C_{12}, C_{44}$) from public DFT databases (Materials Project, AFLOWlib) and curated benchmarks (MatBench), computes atomic-level features (radius variance, electronegativity, VEC), trains Random Forest, Gradient Boosting, and Linear models with strict complexity controls, and validates predictions against physical bounds ($0 < A_1 < 3$) and robustness criteria (sensitivity analysis on outlier removal). The project explicitly addresses the risk of chemical similarity leakage via Leave-One-Element-Out (LOEO) cross-validation and acknowledges the exploratory nature of the results.

## Constitutional Compliance Note

This project acknowledges that with a small sample size, a perfect stratified split by element is statistically invalid (empty test sets). However, to adhere to the *spirit* of Constitution Principle VII, we employ LOEO. We explicitly state that residual leakage for chemically similar elements remains a limitation, and all R² metrics are interpreted as 'interpolation capability' within the training space. This approach balances statistical validity with the physical constraint of the dataset.

## Complexity Tracking

| Violation | Why Needed | Simpler Alternative Rejected Because |
| :--- | :--- | :--- |
| **LOEO vs. Random Split** | Spec assumes random split, but Constitution VII requires stratification to prevent leakage. Random split on N~100 with elemental features causes severe leakage. | Random split inflates R² via memorization of elemental trends. LOEO is the standard compromise for small N in materials informatics. |
| **Sensitivity Analysis** (3 thresholds) | Required by FR-005 to ensure robustness of R² metrics against arbitrary outlier removal. | Single threshold would not demonstrate model stability; standard practice in materials informatics requires this sweep. |
| **API Key Requirement** | Materials Project requires an API key for bulk queries. | Public endpoints without keys are rate-limited and insufficient for the required data volume. |
