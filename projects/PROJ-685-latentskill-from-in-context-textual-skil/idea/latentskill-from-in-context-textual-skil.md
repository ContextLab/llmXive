# LatentSkill: From In-Context Textual Skills to In-Weight Latent Skills for LLM Agents  

**Field**: linguistics  

## Research question  

Do weight‑space latent skill embeddings (learned via lightweight adapters) achieve higher task success rates while reducing prompt length compared to traditional in‑context textual skill prompting, when evaluated on standard LLM‑agent benchmarks?  

## Motivation  

LLM agents that rely on textual skill libraries must repeatedly inject long skill descriptions into the prompt, inflating context size and exposing proprietary procedures. Embedding skills directly in model weights promises compact, reusable representations, but it is unclear whether this compression preserves or improves functional performance. Answering this question would clarify a core trade‑off for computational‑linguistic systems that aim to balance efficiency, privacy, and capability.  

## Related work  

- [LatentSkill: From In-Context Textual Skills to In-Weight Latent Skills for LLM Agents (2026)](https://arxiv.org/abs/2606.06087) — Introduces the latent‑skill concept and reports preliminary gains in context overhead and performance.  
- [Skill-as-Pseudocode: Refactoring Skill Libraries to Pseudocode for LLM Agents (2026)](https://arxiv.org/abs/2605.27955) — Shows that textual skill representations impose parsing burdens on agents and motivates more structured encodings.  
- [Externalization in LLM Agents: A Unified Review of Memory, Skills, Protocols and Harness Engineering (2026)](https://arxiv.org/abs/2604.08224) — Provides a broad survey of skill‑externalization strategies, highlighting the need for weight‑based skill storage.  

## Expected results  

We anticipate that latent‑skill adapters will (a) cut average prompt length by ≥30 % relative to full textual skill insertion, and (b) improve benchmark success rates by ≥5 % absolute over the in‑context baseline. Statistical significance will be demonstrated with paired bootstrap tests (p < 0.05) across multiple random seeds.  

## Methodology sketch  

- **Data acquisition**  
  1. Download the publicly released “SkillBank” skill collection (≈5 K curated skill snippets) from the authors’ GitHub release.  
  2. Obtain two LLM‑agent benchmark suites: **ALFWorld** (v1.0) and **MiniWoB‑Lite** from HuggingFace Datasets.  
- **Baseline implementation**  
  3. Implement an in‑context prompting baseline that concatenates the full textual skill description into the prompt for each task episode.  
- **Latent‑skill construction**  
  4. For each skill, fine‑tune a LoRA adapter (rank = 8) on a 1‑sentence “skill‑execution” dataset generated by prompting a frozen Llama‑2‑7B model to perform the skill on synthetic inputs.  
  5. Store each adapter as a separate weight patch; the agent loads the required patch at runtime instead of appending text.  
- **Evaluation protocol**  
  6. Run 30 random seeds for each benchmark, measuring (i) **success rate** (episode completion), (ii) **average prompt length** (tokens), and (iii) **inference latency**.  
  7. Compare latent‑skill vs. in‑context baseline using paired bootstrap resampling to obtain confidence intervals and p‑values.  
- **Ablation & analysis**  
  8. Perform an ablation where adapters are randomly shuffled across tasks to verify that performance gains stem from the correct skill‑weight mapping.  
  9. Measure “skill leakage” by probing the model’s logits for skill‑specific tokens when the adapter is *not* loaded.  
- **Reproducibility**  
  10. Archive all data URLs, code (Python 3.11, Transformers 4.41), and environment specifications (requirements.txt) in a public GitHub repository with CI‑based unit tests for adapter loading and episode execution.
