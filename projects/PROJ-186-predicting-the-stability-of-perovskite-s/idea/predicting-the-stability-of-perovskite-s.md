# Predicting the Stability of Perovskite Structures Using Machine Learning

**Field**: materials science

## Research question

How do compositional and structural descriptors (Goldschmidt tolerance factor, ionic radius mismatch, electronegativity differences) predict thermodynamic stability across the compositional space of ABX₃ perovskite compounds, and can data‑driven models generalize this relationship to novel, experimentally uncharacterized compositions?

## Motivation

Perovskite solar cells achieve record efficiencies but degrade quickly under ambient conditions, limiting commercial viability. High‑throughput first‑principles calculations of stability are computationally expensive, while experimental screening is slow and costly. A lightweight, descriptor‑based machine‑learning predictor would enable rapid virtual screening of thousands of candidate ABX₃ compositions, focusing experimental effort on the most promising, durable materials.

- **Machine Learning‑Driven Crystal System Prediction for Perovskites Using Augmented X‑ray Diffraction Data (2026)** – Demonstrates how machine‑learning models can predict crystallographic attributes of perovskites from diffraction data, providing a methodological precedent for applying data‑driven techniques to perovskite property prediction.

## Expected results

- A regression model (e.g., RandomForestRegressor) achieving cross‑validated RMSE ≤ 0.15 eV/atom on decomposition energy (or ≤ 30 °C on predicted decomposition temperature) for a held‑out test set of known perovskites.  
- Feature‑importance analysis highlighting tolerance factor, ionic‑radius mismatch, and electronegativity differences as the primary predictors, confirming chemical intuition.  
- A ranked list of ≥ 200 hypothetical ABX₃ compositions (generated by combinatorial substitution of A, B, X from the Inorganic Crystal Structure Database) with predicted stability scores; the top 10 candidates will be flagged for experimental follow‑up.

## Methodology sketch

- **Data acquisition**  
  - Query the Materials Project REST API (`https://materialsproject.org/open`) to download all entries labelled ABX₃.  
  - Retrieve formation energies, decomposition energies, and any reported decomposition temperatures from the Materials Project and the Open Quantum Materials Database (OQMD) releases.  

- **Descriptor calculation**  
  - Compute Goldschmidt tolerance factor *t* and octahedral factor *μ* for each composition.  
  - Extract ionic radii, Pauling electronegativity, and common oxidation states using `pymatgen` periodic‑table utilities.  
  - Derive compositional statistics (mean, variance) for these properties and store everything in a CSV file (< 200 MB).  

- **Data cleaning & splitting**  
  - Discard entries with missing target values or ambiguous oxidation states.  
  - Perform an 80/20 train‑test split stratified by a stability label (stable vs unstable) derived from decomposition energy thresholds.  

- **Model training**  
  - Train a `RandomForestRegressor` (n_estimators = 300, max_depth = 15) using scikit‑learn.  
  - Conduct a 5‑fold cross‑validation grid search over `max_depth ∈ {10,15,20}` and `min_samples_leaf ∈ {1,2,4}`.  

- **Evaluation**  
  - Compute RMSE, MAE, and R² on the held‑out test set.  
  - Produce a predicted‑vs‑true scatter plot (saved as PNG).  
  - Extract and plot feature importances (bar chart).  

- **Virtual screening**  
  - Enumerate a combinatorial library of ~5 000 ABX₃ formulas by pairing reasonable A‑site cations (alkali, alkaline‑earth, post‑transition metals) with B‑site cations (transition metals, Sn, Ge) and X = Cl, Br, I.  
  - Calculate the same descriptor set for each candidate.  
  - Apply the trained model to predict stability; retain compounds with predicted decomposition energy < ‑0.1 eV/atom (or predicted temperature > 150 °C).  

- **Result packaging**  
  - Create a markdown table of the top 20 candidates with predicted stability values and descriptor summaries.  
  - Save all artifacts (CSV files, trained model pickle, figures) under `results/` for downstream inspection.
