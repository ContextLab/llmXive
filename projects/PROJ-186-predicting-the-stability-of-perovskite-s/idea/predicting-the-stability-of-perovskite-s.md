# Predicting the Stability of Perovskite Structures Using Machine Learning  

**Field**: materials science  

## Research question  

Can a lightweight machine‑learning model trained on publicly available compositional and structural descriptors accurately predict the thermodynamic stability (e.g., decomposition temperature or formation energy) of novel ABX₃ perovskite compounds?  

## Motivation  

Perovskite solar cells promise high efficiencies but their commercial deployment is hampered by rapid degradation under ambient conditions. Experimental stability screening is slow and costly, while high‑throughput first‑principles calculations require substantial compute resources. A data‑driven predictor that relies only on easily obtained descriptors (ionic radii, electronegativity, Goldschmidt tolerance factor, etc.) would enable rapid virtual screening of thousands of candidate compositions, focusing experimental effort on the most promising, durable perovskites.

## Expected results  

- A regression model (e.g., RandomForestRegressor) achieving a cross‑validated RMSE ≤ 0.15 eV/atom on decomposition energy (or ≤ 30 °C on predicted decomposition temperature) for a held‑out test set of known perovskites.  
- Feature‑importance analysis showing that tolerance factor, ionic radius mismatch, and electronegativity difference are the dominant predictors, confirming chemical intuition.  
- A ranked list of ≥ 200 hypothetical ABX₃ compositions (generated by combinatorial substitution of A, B, X from the Inorganic Crystal Structure Database) with predicted stability scores, of which the top 10 are flagged for experimental follow‑up.  

## Methodology sketch  

1. **Data acquisition**  
   - Download perovskite entries (ABX₃) from the Materials Project (MP) via its public REST API (https://materialsproject.org/open).  
   - Retrieve formation energies, decomposition energies, and experimentally reported decomposition temperatures (when available) from the MP and the Open Quantum Materials Database (OQMD) releases.  

2. **Descriptor calculation**  
   - For each composition, compute:  
     * Goldschmidt tolerance factor (t) and octahedral factor (μ).  
     * Ionic radii, Pauling electronegativity, oxidation states (via `pymatgen` periodic table).  
     * Simple compositional statistics (mean, variance of the above properties).  
   - Store descriptors in a CSV file; all calculations run on the CPU and require < 200 MB RAM.  

3. **Data cleaning & splitting**  
   - Remove entries with missing target values or ambiguous oxidation states.  
   - Perform an 80/20 train‑test split stratified by stability class (stable vs. unstable).  

4. **Model training**  
   - Use scikit‑learn’s `RandomForestRegressor` (n_estimators = 300, max_depth = 15) – fits comfortably in ≤ 2 GB RAM and < 10 min on the runner.  
   - Optimize hyperparameters with a 5‑fold cross‑validation grid (max_depth ∈ {10,15,20}; min_samples_leaf ∈ {1,2,4}).  

5. **Evaluation**  
   - Compute RMSE, MAE, and R² on the test set.  
   - Plot predicted vs. true stability values (matplotlib, saved as PNG).  
   - Extract feature importances and generate a bar chart.  

6. **Virtual screening**  
   - Generate a combinatorial library of 5 000 ABX₃ formulas by enumerating all reasonable A‑site (alkali, alkaline‑earth, post‑transition metals) and B‑site (transition metals, Sn, Ge) cations with X = Cl, Br, I.  
   - Compute the same descriptor set for each candidate.  
   - Apply the trained model to predict stability; retain compounds with predicted decomposition energy < ‑0.1 eV/atom (or temperature > 150 °C).  

7. **Result packaging**  
   - Produce a markdown table of the top 20 candidates with predicted values and descriptor summaries.  
   - Save all artifacts (CSV, model pickle, figures) to the repository’s `results/` folder for downstream inspection.
