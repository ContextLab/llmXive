# Research Configuration: EvoMem-Conflict Filtering

## Overview
This document serves as the primary configuration source for the EvoMem-Conflict Filtering feasibility study.
It defines the statistical parameters, dataset sources, and experimental constraints.

## Statistical Power Analysis
The sample size $N$ is calculated based on the following parameters:
- Effect Size (Cohen's h): 0.2 (Small effect)
- Power (1 - β): 0.80
- Significance Level (α): 0.05
- Test Type: Two-tailed test for difference in proportions

## Calculated Parameters
sample_size: 157

## Experimental Constraints
- **Model Constraints**: CPU-only execution, default precision (no 8-bit/4-bit quantization).
- **Time Limit**: Execution must complete within the standard GitHub Actions runner time limit (6 hours).
- **Memory**: Fits within standard CPU runner memory limits (approx. 16GB).

## Data Sources
- **Synthetic Pairs**: Generated by `src/data/generators/synthetic_pairs.py` into `data/raw/synthetic_pairs.json`.
- **Benchmark Tasks**: Retrieved from `Terminal-Bench-Evo` or generated synthetically via `src/data/benchmarks/terminal_bench_evo.py` into `data/raw/terminal_bench_evo.jsonl`.

## Heuristic Configuration
- **Base Model**: `distilbert-base-uncased`
- **Conflict Threshold**: 0.90 (Softmax probability > 0.90 indicates conflict)
- **Fallback Strategy**: If no conflicts detected, retrieve latest state + 2 most recent non-conflict patches.

## Sensitivity Analysis
- **Thresholds**: [0.6, 0.7, 0.8, 0.9, 0.95]
- **Model Sizes**: ['distilbert-base-uncased', 'bert-base-uncased']
