# Implementation Plan: Testing the Isotropy of Cosmic Expansion with Type Ia Supernova Data

**Branch**: `001-testing-isotropy-cosmic-expansion` | **Date**: 2026-06-29 | **Spec**: `specs/001-testing-isotropy-of-cosmic-expansion/spec.md`
**Input**: Feature specification from `/specs/001-testing-isotropy-of-cosmic-expansion/spec.md`

## Summary

This project implements a statistical test for the isotropy of cosmic expansion using the Pantheon+ Type Ia Supernova dataset. The core methodology involves calculating distance‑modulus residuals against a flat ΛCDM model, projecting these residuals onto a HEALPix grid (Nside = 32), and extracting dipole (ℓ = 1) and quadrupole (ℓ = 2) amplitudes via a **pseudo‑Cₗ method with MASTER correction**. Statistical significance is determined by comparing observed amplitudes against a null distribution generated by rotating celestial coordinates (preserving redshift and uncertainties) with a fixed random seed. All steps are designed for CPU‑only execution on a GitHub Actions free‑tier runner.

## Detailed Methodology

### 1. Residual Calculation (FR‑002)

- Cosmological parameters (`H0`, `Ωm`, `ΩΛ`) are read from `data/metadata.json`. If missing, defaults to the values published with Pantheon+ (H₀ ≈ 67.4 km s⁻¹ Mpc⁻¹, Ωₘ ≈ 0.315, Ω_Λ = 1‑Ωₘ).
- The theoretical distance modulus μ_th(z) is obtained by numerically integrating  
  \[
  d_L(z) = (1+z)\,\frac{c}{H_0}\int_0^z \frac{dz'}{E(z')},
  \quad
  E(z)=\sqrt{\Omega_m(1+z)^3+\Omega_\Lambda},
  \]  
  using `scipy.integrate.quad` with absolute tolerance = 1e‑6.
- Residuals Δμ = μ_obs − μ_th are stored in `data/processed/supernovae_clean.csv`.

### 2. HEALPix Projection & Pseudo‑Cₗ (FR‑003, FR‑004)

1. **Pixelisation**  
   - Use `healpy.pixelfunc.ang2pix` with `nside=32` (≈ 3096 pixels).  
   - Each supernova contributes its residual weighted by the inverse variance (w_i = 1/σ_i²).  
   - For pixel *p*:  
     \[
     \bar{Δμ}_p = \frac{\sum_{i\in p} w_i Δμ_i}{\sum_{i\in p} w_i},
     \quad
     \sigma_p^2 = \frac{1}{\sum_{i\in p} w_i}.
     \]  
   - Pixels with zero weight are masked (`mask[p]=False`).

2. **Pseudo‑Cₗ Estimation**  
   - Compute the masked map’s pseudo‑Cₗ with `healpy.anafast(map, lmax=3, pol=False, mask=mask)`.  
   - Obtain the mask’s power spectrum `W_ℓ = healpy.anafast(mask, lmax=3)`.

3. **MASTER Mode‑Coupling Matrix**  
   - Build the coupling matrix **M** using `healpy.sphtfunc.get_mll_matrix(lmax=3, mask=mask)`.  
   - Regularise **M** via Tikhonov: **M_reg** = **M** + λ I, with λ = 1e‑6.  
   - Perform an SVD on **M_reg** and retain singular values > 1e‑12; invert the truncated matrix to obtain **M⁻¹**.

4. **Unbiased Cₗ**  
   - Unbiased estimate: **Ĉ = M⁻¹ · C̃**, where **C̃** are the pseudo‑Cₗ.  
   - Dipole amplitude: \(A_1 = \sqrt{(2ℓ+1)Ĉ_{ℓ=1}/4π}\).  
   - Quadrupole amplitude: analogous for ℓ = 2.

### 3. Uniform Rotation‑Matrix Null Simulations (FR‑005)

- **Uniform SO(3) sampling**: generate three independent uniform random numbers u₁,u₂,u₃ ∈ [0,1). Compute Euler angles:  
  \[
  φ = 2πu_1,\quad
  θ = \arccos(1-2u_2),\quad
  ψ = 2πu_3.
  \]  
  Pass (φ,θ,ψ) to `healpy.rotator.Rotator` to obtain a rotation matrix.  
- Apply the rotation to each (RA, Dec) pair, leaving redshift and μ unchanged.  
- Re‑project the rotated residuals onto the same HEALPix grid and repeat the pseudo‑Cₗ + MASTER pipeline to obtain dipole/quadrupole amplitudes for each mock catalog.  
- Seed is fixed for reproducibility.

### 4. Power Analysis (SC‑002)

The detectable dipole amplitude for a given significance α and desired power 1‑β is:  

\[
A_{\text{thresh}} = \frac{(Z_{1-β}+Z_{1-α})\,σ_A}{\sqrt{N_{\text{eff}}}},
\]

where \(σ_A\) is the per‑SN residual uncertainty (~0.1 mag), and \(N_{\text{eff}} = \frac{(\sum w_i)^2}{\sum w_i^2}\) is the effective number of independent measurements (≈ 1300 for the Pantheon+ sample after cuts). Using Z_{0.8}=0.84 and Z_{0.95}=1.645 gives a threshold ≈ 0.04 mag; thus a dipole of 0.05 mag yields > 80 % power.

### 5. Runtime Monitoring & Fallback (SC‑004)

- Parameter `N_SIM_MAX = 10_000`.  
- After every fixed batch of simulations, the elapsed wall‑clock time is checked.  
- If elapsed > 5 h, the remaining simulation count is reduced by **[deferred]** (floor) and a warning is logged (`reports/runtime_log.txt`).
- The final p‑value is computed from however many simulations were actually performed; the variance of the p‑value estimate is reported.

### 6. Reproducibility Validation (SC‑002)

- Generate a synthetic isotropic HEALPix map with a known injected dipole amplitude of a specified magnitude..  
- Run the full pseudo‑Cₗ + MASTER pipeline; the recovered dipole must match the injected value within 1e‑3 mag.  
- This test is executed in `tests/integration/test_canonical.py` and must pass before the main analysis runs.

## Risk Mitigation

- **Sparse Mask Ill‑Conditioning**: Regularisation λ = 1e‑6 and SVD cutoff > 1e‑12 guarantee an invertible mode‑coupling matrix.  
- **Circular Validation**: The null distribution tests *directional alignment* of the dipole rather than its mere existence; we explicitly state this limitation in the paper.  
- **Runtime Exceedance**: Dynamic fallback (Section 5) ensures the job never exceeds the 6‑hour CI limit.  
- **Redshift Peculiar Velocities**: Applying a cut z > 0.02 removes SNe where peculiar velocities could dominate (> 300 km s⁻¹), mitigating bias.
