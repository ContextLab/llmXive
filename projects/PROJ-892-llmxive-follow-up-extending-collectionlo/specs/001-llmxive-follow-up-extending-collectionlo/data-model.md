# Data Model: Quantization Robustness of Multi-Effect LoRA Adapters

## 1. Overview

This document defines the data structures, schemas, and relationships used in the `001-lora-quantization-robustness` feature. All data is persisted in `data/` and tracked in `state/`.

## 2. Entity Definitions

### 2.1 EffectAdapter
A record describing a specific effect within the CollectionLoRA adapter.
- `effect_id`: Unique identifier (e.g., "oil_painting").
- `weight_matrix_path`: Path to the LoRA weight file for this effect.
- `subspace_rank`: Integer, effective rank computed via SVD.
- `prompt_template`: The text prompt used to trigger this effect.

### 2.2 ReferenceImage
A ground-truth image generated by the FP16 baseline.
- `image_id`: Unique identifier (e.g., "oil_painting_fp16_001").
- `effect_id`: Link to EffectAdapter.
- `prompt`: The text prompt used.
- `image_path`: Path to the generated image file.
- `clip_embedding`: Array of floats (512-dim).
- `sha256`: Content hash of the image file.

### 2.3 GenerationResult
The output of a single generation run (FP16, INT8, or INT4).
- `run_id`: Unique identifier.
- `effect_id`: Link to EffectAdapter.
- `quantization_level`: Enum {FP16, INT8, INT4}.
- `image_path`: Path to generated image.
- `clip_embedding`: Array of floats.
- `cosine_similarity`: Float (vs prompt embedding).
- `lpips_distance`: Float (vs ReferenceImage).
- `cesr_score`: Float (Max similarity to other effect reference images).
- `delta_cesr`: Float (cesr_score - baseline_cesr_score for same effect/prompt).
- `sha256`: Content hash.

### 2.4 AnalysisMetric
Aggregated statistical results.
- `metric_id`: Unique identifier.
- `comparison`: e.g., "INT4_vs_FP16".
- `mean_delta`: Float.
- `credible_interval_lower`: Float.
- `credible_interval_upper`: Float.
- `posterior_prob`: Float.
- `correlation_coefficient`: Float (for rank vs bleeding).
- `correlation_ci_lower`: Float.
- `correlation_ci_upper`: Float.
- `status`: Enum {Valid, Underpowered, PredictorInsufficientVariance}.

### 2.5 StateRecord
Versioning and integrity record.
- `artifact_name`: Name of the file.
- `sha256`: Hash.
- `timestamp`: ISO 8601.
- `version`: Semantic version of the pipeline.

## 3. Data Flow

1.  **Input**: `config.yaml` (Prompts, Seeds).
2.  **Generation**:
    - Load Model -> Quantize -> Generate -> Save `GenerationResult`.
3.  **Embedding**:
    - Load CLIP -> Extract Embeddings -> Update `GenerationResult`.
4.  **Metrics**:
    - Compute CosSim, LPIPS, CESR (Max), Delta CESR -> Update `GenerationResult`.
5.  **Analysis**:
    - Load `GenerationResult` -> Run Bayesian Model -> Save `AnalysisMetric`.
6.  **State**:
    - Hash all files -> Update `state/artifacts.yaml`.

## 5. Constraints

- **Immutability**: Once generated, image files in `data/generated/` are never modified.
- **Checksums**: All files in `data/` must have a corresponding entry in `state/artifacts.yaml`.
- **Reproducibility**: `config.yaml` must contain fixed seeds.
