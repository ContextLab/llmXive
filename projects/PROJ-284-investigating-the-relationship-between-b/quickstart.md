# Quickstart – End‑to‑End Run‑Book

This document describes the commands that drive the full analysis pipeline.
Each command is intended to run in a fresh CI environment and must exit
with status code ``0`` while producing the artifacts declared in the
project specifications.

## 1️⃣ Download & Preprocess

```bash
python code/main.py download_preprocess --subjects 10
```

*Downloads a small subset of HCP subjects (or the synthetic fallback) and
runs the preprocessing pipeline. The resulting NIfTI files are stored under
``data/processed/``.*

## 2️⃣ Extract Metrics

```bash
python code/main.py extract_metrics
```

*Computes functional connectivity matrices, extracts graph‑theoretic
metrics, and writes the raw metric table to ``data/analysis/metrics.csv``.*

## 3️⃣ Run Analyses (PCA, Correlations, …)

```bash
python code/main.py analyze
```

*Performs PCA on the metric set, runs the correlation analyses (including
FDR correction), and writes the following files:

- ``data/analysis/pca_loadings.csv``
- ``data/analysis/factor_scores.csv``
- ``data/analysis/correlation_results.csv``
- ``data/analysis/full_metrics.csv`` *(generated by the helper script)*
*

The ``full_metrics.csv`` file is produced by the helper script
``code/analysis/generate_full_metrics.py`` which merges the raw metrics
with the PCA factor scores.

## 4️⃣ Visualisation & Report Generation

```bash
python code/main.py viz_report
```

*Creates the scatter‑plot and network‑diagram figures and assembles the
final Markdown report under ``reports/``.*

## 5️⃣ Verify Output

After the pipeline finishes, you can confirm that the critical artifact
exists:

```bash
test -f data/analysis/full_metrics.csv && echo "✅ full_metrics.csv is present"
```

If the test passes, the end‑to‑end run‑book has succeeded.
