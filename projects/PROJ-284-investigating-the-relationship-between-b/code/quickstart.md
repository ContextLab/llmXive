# Quickstart – end‑to‑end run‑book

This document contains the commands that are executed by the CI validation
pipeline. Each command must exit with status ``0`` and produce the files
declared in the specification.

## Steps

1. **Download & preprocess**
 ```bash
 python code/main.py download_preprocess
 ```

2. **Extract metrics**
 ```bash
 python code/main.py extract_metrics
 ```

3. **Analyze** – this step merges the raw metrics with the PCA factor scores
 and writes the final table required by FR‑004/FR‑005.
 ```bash
 python code/main.py analyze
 ```

 *If, for any reason, the `analyze` sub‑command does not invoke the
 merging logic, the script can also be run directly:*
 ```bash
 python code/analysis/correlations.py
 ```

4. **Visualization & report generation**
 ```bash
 python code/main.py viz_report
 ```

After the pipeline finishes, you should find the following artefacts
under the `data/analysis/` directory:

- `subject_metrics.csv` – raw network metrics per subject (generated by T021/T022)
- `pca_loadings.csv` – PCA component loadings (T023a)
- `factor_scores.csv` – PCA factor scores per subject (T023a)
- **`full_metrics.csv`** – raw metrics **plus** PCA scores (produced by T023b)

The presence of `full_metrics.csv` confirms that the merge step has
succeeded and that downstream reporting modules can access the complete
dataset.
