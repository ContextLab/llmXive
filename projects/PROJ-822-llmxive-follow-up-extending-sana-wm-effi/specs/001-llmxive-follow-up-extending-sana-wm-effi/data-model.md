# Data Model: llmXive follow-up: extending "SANA-WM: Efficient Minute-Scale World Modeling with Hybrid Linear Diff"

## Entity Definitions

### 1. SyntheticTrajectory
Represents a single ground-truth 6-DoF camera motion sequence.
- **ID**: Unique string identifier (e.g., `traj_001`).
- **Type**: Enum (`constant_velocity`, `sinusoidal`, `chaotic`, `high_freq`).
- **Parameters**: JSON object containing kinematic parameters (velocity, frequency, amplitude, chaos_seed).
- **Duration**: Float (seconds).
- **Frames**: List of `FramePose` objects.
- **TextureSeed**: Integer seed for procedural texture generation (3D Perlin Noise).

### 2. FramePose
Represents the camera pose at a specific timestamp.
- **Timestamp**: Float (seconds from start).
- **Position**: Tuple `(x, y, z)` in meters.
- **Rotation**: Tuple `(roll, pitch, yaw)` in radians.
- **Source**: Generated deterministically by `trajectory_generator.py`.

### 3. GeneratedVideo
The output video file produced by the model.
- **ID**: Links to `SyntheticTrajectory.ID`.
- **ConditionType**: Enum (`symbolic`, `learned_baseline`).
- **Path**: Relative path to `.mp4` file.
- **Metadata**: JSON containing generation parameters (resolution, fps, model_version, quantization_type).

### 4. PoseEstimationResult
The output of the COLMAP estimator for a specific video.
- **VideoID**: Links to `GeneratedVideo.ID`.
- **FrameID**: Integer frame index.
- **EstimatedPose**: Tuple `(x, y, z, roll, pitch, yaw)`.
- **Confidence**: Float (0.0 - 1.0).
- **Status**: Enum (`valid`, `invalid_nan`, `invalid_low_conf`, `invalid_black_frame`).

### 5. GeometricMetric
Aggregated error metric for a trajectory.
- **TrajectoryID**: Links to `SyntheticTrajectory.ID`.
- **ConditionType**: Enum (`symbolic`, `learned_baseline`).
- **MeanEuclideanError**: Float (meters). **Note**: Calculated after Procrustes scale alignment.
- **ScaleAlignedError**: Boolean (True if Procrustes alignment was applied).
- **SSIM**: Float (0.0 - 1.0).
- **ValidFrameCount**: Integer.
- **ExcludedFrameCount**: Integer.
- **IsFailure**: Boolean (True if >50% frames invalid; error capped at max value).
- **CappedErrorValue**: Float (Maximum error value assigned if `IsFailure` is True).

### 6. StatisticalTestResult
The final output of the hypothesis test.
- **TestType**: String ("paired_t_test" or "permutation_test").
- **Metric**: String ("mean_euclidean_error").
- **N**: Integer (sample size, e.g., 500).
- **TStatistic**: Float (or `permutation_statistic`).
- **PValue**: Float.
- **DegreesOfFreedom**: Integer (or `n_permutations`).
- **Conclusion**: String ("significant", "not_significant").
- **FailureHandlingMethod**: String ("capped_max_error" or "excluded").

### 7. COLMAPConfig
Configuration file for the pose estimator.
- **FeatureExtractor**: String ("AKAZE").
- **MatchingStrategy**: String ("exhaustive").
- **MinInlierRatio**: Float (0.8).
- **Source**: Generated by `evaluation/pose_estimator.py` to enforce FR-007.

## Data Flow

1.  **Generation**: `trajectory_generator.py` -> `SyntheticTrajectory` (JSON) + `FramePose` (List).
2.  **Inference**: `SyntheticTrajectory` + `SymbolicEncoder` -> `GeneratedVideo` (MP4).
3.  **Estimation**: `GeneratedVideo` + `COLMAP` (using `COLMAPConfig`) -> `PoseEstimationResult` (JSON).
4.  **Alignment**: `PoseEstimationResult` + `SyntheticTrajectory` -> `ScaleAlignedPose` (JSON).
5.  **Evaluation**: `ScaleAlignedPose` -> `GeometricMetric` (JSON).
6.  **Analysis**: `GeometricMetric` (all trajectories) -> `StatisticalTestResult` (JSON).

## Storage Schema

- **Input Data**: `data/synthetic/` (JSON files for trajectories).
- **Generated Artifacts**: `data/generated/` (MP4 files, organized by ID).
- **Intermediate Results**: `data/estimates/` (JSON for pose estimations).
- **Final Results**: `data/results/` (JSON for metrics and statistical tests).
- **Configuration**: `data/config/colmap_config.yaml` (Versioned and checksummed).

## Constraints

- **Floating Point**: All pose coordinates stored as 64-bit floats.
- **Timestamps**: Monotonically increasing.
- **Validity**: `Confidence` < 0.8 or `Status` != `valid` frames are excluded from `MeanEuclideanError` calculation.
- **Black Frame**: If `std_dev(pixel_intensity)` < 1e-5, frame is marked `invalid_black_frame` and excluded.
- **Failure Handling**: If `ValidFrameCount` < 50% of total frames, `IsFailure` = True, and `MeanEuclideanError` is set to `CappedErrorValue` (e.g., 100.0).
- **Scale Alignment**: `MeanEuclideanError` is computed on Procrustes-aligned poses.
- **COLMAP Config**: Must match FR-007 (AKAZE, exhaustive, 0.8).
