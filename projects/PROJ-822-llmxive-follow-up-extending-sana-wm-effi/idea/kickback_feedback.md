## Unresolved concerns

- User Story 3 defines a baseline using 'learned text-to-image encoder (after converting the symbolic trajectory to a text description)'. This introduces a confound (translation fidelity) not addressed by any Functional Requirement. No FR exists to specify the text-generation method, its accuracy, or how to isolate the encoder's geometric bias from the translation error. The story's goal (isolating architectural bias) is unbacked by a concrete requirement to control this variable.
- User Story 3 requires calculating 'Euclidean distance between generated and ground-truth poses'. However, the spec lacks any Functional Requirement mandating a 'scale-alignment' or 'Procrustes analysis' step. Without this, comparing relative COLMAP poses to absolute ground-truth poses is mathematically invalid. The story's success criteria (geometric error) cannot be met as described because the necessary alignment requirement is missing.
- User Story 3 assumes COLMAP will successfully recover poses from synthetic video. There is no Functional Requirement specifying the texture generation method (e.g., 'must use temporally consistent procedural noise') required to ensure COLMAP feature tracking. The story's validity depends on this unbacked assumption. If the video is black or static, the metric fails, but no FR exists to prevent or handle this specific failure mode in the data generation phase.
- User Story 3 proposes a paired t-test on 500 trajectories. There is no Functional Requirement addressing the statistical handling of 'censored data' (trajectories where the symbolic encoder fails to generate features, leading to excluded frames). The story assumes a valid distribution of 500 differences, but no FR exists to define the protocol for trajectories with mismatched valid frame counts or high exclusion rates, risking an invalid statistical test.
- FR-003 text is malformed: '...on a CPU-only runtime (A small number of cores, ~7GB RAM The research question: [Not provided in passage]...'. The requirement text is broken by an unremoved template placeholder and incomplete sentence structure, making the specific resource constraint ambiguous.

Given the sensitivity of COLMAP to these parameters, the requirement should explicitly state that these parameters are fixed for *both* symbolic and learned baseline runs to ensure a fair comparison, as implied by the story but not stated in the FR.

However, the Research Question asks to compare 'learned semantic priors' vs 'architectural inductive biases'. Converting symbolic rules to text introduces a 'translation fidelity' confound. The spec claims to isolate the encoder's effect but the method described (US-3) conflates encoder performance with text-generation performance.

The spec fails to define the necessary 'scale-alignment' step (e.g., Procrustes analysis) required to make Euclidean distance computable between relative (COLMAP) and absolute (kinematic) coordinates.
