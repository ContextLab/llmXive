## Summary

This project implements a symbolic-latent planner to extend the Geometric Action Model (GAM). The core innovation replaces the learned neural predictor with a differentiable symbolic solver operating **exclusively in the frozen multi-dimensional latent space** of the Geometric Foundation Model (GFM). The system generates a synthetic "topology-shift" test set using PyBullet (novel kinematic chains and deformable materials absent from training), executes the symbolic inference pipeline on CPU-only hardware, and performs comparative statistical analysis (Wilcoxon Signed-Rank Test, Fisher's Exact Test, Kaplan-Meier Survival Analysis) against the baseline GAM to validate zero-shot generalization and latency improvements.

## Technical Context

**Language/Version**: Python  
**Primary Dependencies**: `pybullet` (physics simulation), `torch` (CPU-only GFM inference), `cvxpy`/`diffcp` (differentiable convex optimization for symbolic solver), `scipy` (statistical tests), `numpy`, `pandas`.  
**Storage**: Local filesystem for `data/` (synthetic test sets, logs, results) and `code/` (scripts, models).  
**Testing**: `pytest` for unit tests (solver constraints, latent encoding), integration tests for full pipeline execution on CI.  
**Target Platform**: GitHub Actions free-tier runner (A small number of CPU cores, limited RAM, Linux x86_64).  
**Project Type**: Research computational experiment / CLI tool.  
**Performance Goals**: Complete 50 trials (symbolic + baseline) within 6 hours; inference latency < 300s per step (timeout constraint).  
**Constraints**: No GPU/CUDA; no 8-bit/4-bit quantization; strict memory usage (<7GB); strict adherence to frozen GFM weights (no fine-tuning).  
**Scale/Scope**: distinct novel topologies; A series of trials comprising both symbolic and baseline conditions will be conducted.; A fixed number of timesteps per trial is employed.

## Complexity Tracking

| Violation | Why Needed | Simpler Alternative Rejected Because |
|-----------|------------|-------------------------------------|
| **Differentiable Solver (DiffTaichi/CVXPY)** | Required to enforce physical constraints (rigid/soft body) in 3D space while maintaining gradient flow for the symbolic logic *within the solver*. | Standard non-differentiable solvers (e.g., `scipy.optimize`) cannot be integrated into the "symbolic-latent" pipeline as required by FR-003 to verify constraint satisfaction. Note: Gradients do NOT flow through the frozen GFM decoder. |
| **Synthetic Test Set Generation** | Real-world datasets lack the specific "novel topology" (e.g., 5-link hinge) required for the zero-shot hypothesis. | Using existing benchmarks (e.g., standard PyBullet tasks) would violate Constitution Principle VII (Zero-Shot Topology Generalization) as these topologies are likely in the training distribution. |
