# Research: Consciousness Bootstrapping: Self-Aware AI Through Recursive Introspection

## Executive Summary

This research investigates whether a recursive self-attention mechanism, which allows a language model to attend to its own confidence distribution from previous generation steps, can enhance meta-cognitive capabilities such as self-consistency, error detection, and uncertainty calibration. The study compares a modified small transformer architecture (e.g., Qwen) against a standard baseline trained on a subset of the Pile dataset. Evaluation is performed on MMLU, GSMK, and Self-Consistency benchmarks, with statistical significance tested via paired t-tests across five random seeds. The hypothesis is that the recursive architecture improves the correlation between self-consistency (stability) and accuracy (calibration), rather than implying awareness through consistency alone.

## Methodology

### 1. Model Architecture

* **Base Model**: Qwen1.5-0.5B (or similar <300M parameter model) to ensure fit within 7 GB RAM and 2-hour CPU budget. TinyLlama is too large for the recursive module overhead on CPU.
* **Recursive Module**: A custom `TemporalRecursiveAttention` layer.
 * **Mechanism**: At generation step $t$, the model computes a confidence distribution $C_{t-1}$ (softmax probability) of the previous token. This distribution is projected via a learnable linear layer and concatenated with the hidden state $H_t$ before the attention mechanism. This is fully differentiable.
 * **Depth**: Max recursion depth = 2 (as per FR-001).
* **Confidence Head**: A separate linear projection from the hidden state to a scalar, trained to predict the external teacher label (correctness). This ensures the target is independent of the model's own generation.
* **Baseline**: Standard small transformer without the recursive module.

### 2. Training Regime

* **Objective**: Joint Loss = Cross-Entropy + Confidence-Prediction Loss.
 * **Cross-Entropy**: Standard next-token prediction.
 * **Confidence Loss**: The model is trained to predict the correctness label provided by the frozen teacher model (external truth). This breaks the circular validation loop where the model trains on its own consistency.
* **Hyperparameters**:
 * Epochs: [deferred] (target: 100k tokens processed).
 * Batch Size: A minimal value selected to fit available RAM.
 * Learning Rate: [deferred] (standard magnitude).
 * Optimizer: AdamW.
 * Gradient Checkpointing: Enabled to reduce memory.
* **Seeds**: 5 distinct random seeds for reproducibility.
* **Pre-computation**: Teacher labels are generated once and stored in `data/processed/`. Training uses these labels directly.

### 3. Evaluation Protocol

* **Metrics**:
 * **Self-Consistency**: Majority vote accuracy over 10 generated paths (Temp=0.7, Top_p=0.9). *Note: This measures stability, not awareness.*
 * **Uncertainty Calibration**: Brier Score and Expected Calibration Error (ECE) against ground truth.
 * **Error Detection**: ROC-AUC of confidence scores vs. correctness.
 * **Correlation**: Pearson correlation between consistency score and accuracy. *Hypothesis: Recursive model has higher correlation.*
* **Control**: A "shuffled-attention" baseline where the recursive module's input is randomized to isolate the effect of temporal coherence.

### 4. Statistical Analysis

* **Test**: Paired t-test comparing Recursive vs. Baseline across 5 seeds.
* **Correction**: Bonferroni correction for 3 primary metrics (Consistency, Calibration, Error Detection).
* **Sensitivity**: Sweep confidence thresholds {0.4, 0.5, 0.6} to analyze error rate stability.
* **Power**: Acknowledged that with n=5 and a small dataset, statistical power is low. The analysis will focus on effect sizes (Cohen's d) and treat p-values as exploratory.

## Decision Rationale

**Why CPU-only?** The project must run on GitHub Actions free-tier (no GPU). Training 1.1B models on CPU is slow but feasible for 100k tokens if batch size is 1 and epochs are limited. However, 1.1B is too large for the recursive module; we use a smaller model (<300M).
**Why Teacher-Student Distillation?** Training on ground truth for confidence is impossible without labels. Using a frozen teacher model to generate pseudo-labels provides an external signal, breaking the tautological loop of self-consistency proxies.
**Why Small Model?** TinyLlama exceeds the RAM limit with the recursive module. A small-scale model fits and is sufficient to demonstrate architectural effects on stability and calibration.
**Why Correlation over Consistency?** Consistency alone does not imply awareness. The hypothesis is that recursion improves the *correlation* between stability and accuracy, indicating better meta-cognitive alignment.

## Limitations

* **Compute**: A very small training corpus is considered limited in scale. The model may not converge fully. This is a necessary trade-off for the 2-hour CI budget.
* **Model Size**: Small-scale models on CPU with limited RAM are tight. If OOM occurs, the run fails (as per spec), but we will optimize by using gradient checkpointing and reducing context length.
* **Philosophical Scope**: As noted by Socrates-simulated, this measures "knowing the shape of the shadow" (statistical consistency) rather than "knowing the good" (true understanding). The project scope is strictly limited to the measurable metrics defined in FR-004 and FR-005, with a focus on the correlation between stability and accuracy.
* **Statistical Power**: With n=5 and a small dataset, statistical power is low. Results are exploratory and focused on effect sizes.
