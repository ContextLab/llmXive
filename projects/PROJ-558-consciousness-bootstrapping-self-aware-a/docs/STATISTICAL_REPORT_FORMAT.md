# Statistical Report Format Specification

This document defines the structure and content of the statistical analysis outputs
generated by the Consciousness Bootstrapping pipeline (Task T024-T027).

## Overview

The statistical analysis module (`code/analysis/stats.py`) produces two primary
artifacts:
1. `artifacts/results/statistical_report.json`: A comprehensive JSON report containing
 hypothesis test results, effect sizes, and confidence intervals.
2. `artifacts/results/sensitivity_analysis.csv`: A CSV file detailing the performance
 of the model across different confidence thresholds.

## Statistical Report JSON Structure

The `statistical_report.json` file follows this schema:

```json
{
 "metadata": {
 "generated_at": "ISO-8601 timestamp",
 "seeds_analyzed": [123, 456, 789],
 "seeds_excluded": [101],
 "exclusion_reason": "Non-converged confidence loss"
 },
 "self_consistency": {
 "recursive_model": {
 "mean": 0.75,
 "std": 0.05,
 "n": 3
 },
 "baseline_model": {
 "mean": 0.62,
 "std": 0.04,
 "n": 3
 },
 "percentage_difference": 20.97,
 "paired_ttest": {
 "t_statistic": 4.12,
 "p_value": 0.025,
 "significant_at_alpha_0.05": true,
 "bonferroni_corrected_p_value": 0.075,
 "significant_after_bonferroni": false
 },
 "effect_size": {
 "cohens_d": 2.15,
 "interpretation": "Large effect"
 },
 "confidence_interval": {
 "mean_difference": [0.05, 0.21],
 "confidence_level": 0.95
 }
 },
 "calibration_metrics": {
 "brier_score": {
 "recursive": { "mean": 0.12, "p_value": 0.03 },
 "baseline": { "mean": 0.18, "p_value": 0.03 }
 },
 "ece": {
 "recursive": { "mean": 0.05, "p_value": 0.04 },
 "baseline": { "mean": 0.11, "p_value": 0.04 }
 }
 },
 "sensitivity_analysis_summary": {
 "thresholds_tested": [0.4, 0.5, 0.6],
 "optimal_threshold": 0.5,
 "optimal_f1_score": 0.82
 }
}
```

### Field Definitions

#### Metadata
- `generated_at`: ISO-8601 formatted timestamp of report generation.
- `seeds_analyzed`: List of random seed integers that successfully converged.
- `seeds_excluded`: List of seeds excluded due to non-convergence (e.g., confidence loss did not stabilize).
- `exclusion_reason`: String describing why seeds were excluded.

#### Self-Consistency Analysis
- `recursive_model` / `baseline_model`: Descriptive statistics (mean, std, count) for each model type.
- `percentage_difference`: The percentage difference in self-consistency scores calculated as:
 `((recursive_mean - baseline_mean) / baseline_mean) * 100`. This satisfies spec.md SC-001.
- `paired_ttest`: Results of the paired t-test comparing recursive vs. baseline scores across seeds.
 - `t_statistic`: The t-value from the test.
 - `p_value`: The raw p-value.
 - `significant_at_alpha_0.05`: Boolean indicating if p < 0.05.
 - `bonferroni_corrected_p_value`: P-value adjusted for multiple comparisons (Bonferroni correction).
 - `significant_after_bonferroni`: Boolean indicating if corrected p < 0.05.
- `effect_size`: Cohen's d calculation and qualitative interpretation (Small, Medium, Large).
- `confidence_interval`: 95% confidence interval for the mean difference.

#### Calibration Metrics
- `brier_score`: Comparison of Brier scores (lower is better).
- `ece`: Expected Calibration Error comparison (lower is better).

## Sensitivity Analysis CSV Structure

The `sensitivity_analysis.csv` file contains the following columns:

| Column | Type | Description |
|--------|------|-------------|
| threshold | float | The confidence threshold tested (0.4, 0.5, or 0.6) |
| false_positive_rate | float | Rate of false positives at this threshold |
| false_negative_rate | float | Rate of false negatives at this threshold |
| precision | float | Precision of the confidence predictor |
| recall | float | Recall of the confidence predictor |
| f1_score | float | Harmonic mean of precision and recall |

### Example Row

```csv
0.5,0.08,0.12,0.88,0.85,0.86
```

## Generation Process

1. **Data Loading**: The `load_evaluation_results_from_json` function reads all
 `evaluation_results_*.json` files from `artifacts/results/`.
2. **Filtering**: Seeds with non-converged confidence loss are excluded via
 `filter_converged_seeds`.
3. **Statistical Tests**: Paired t-tests, Bonferroni correction, and Cohen's d
 are calculated using `scipy.stats` and custom implementations.
4. **Sensitivity Sweep**: The `run_sensitivity_analysis` function iterates over
 the discrete set {0.4, 0.5, 0.6} to compute error rates.
5. **Report Generation**: `generate_statistical_report` assembles the final JSON
 and CSV artifacts.

## Usage

To regenerate these reports:

```bash
python code/analysis/stats.py \
 --input-dir artifacts/results \
 --output-dir artifacts/results \
 --thresholds 0.4 0.5 0.6
```

## Notes on Interpretation

- **Statistical Significance**: A result is considered statistically significant
 if the Bonferroni-corrected p-value is less than 0.05.
- **Effect Size**: Cohen's d values > 0.8 are interpreted as "Large", > 0.5 as
 "Medium", and > 0.2 as "Small".
- **Percentage Difference**: Positive values indicate the recursive model outperformed
 the baseline; negative values indicate the opposite.
