## Unresolved concerns

- The plan proposes Leave-One-Out Cross-Validation (LOOCV) for N < 100. For regression tasks with small N, LOOCV produces extremely high variance in performance estimates and is statistically inferior to k-fold CV (e.g., 5 or 10). This undermines the 'statistical rigor' claim in FR-006 and the validity of the R² confidence intervals.
- The plan states: 'If N < 100: LOOCV... If N >= 100: 5-Fold CV'. This creates a discontinuity in the evaluation methodology based on an arbitrary threshold. The choice of validation strategy should be consistent or based on a power analysis, not a hard cutoff that changes the statistical properties of the estimate mid-stream.
- The plan mentions 'Stratified 80/20 split based on binned yield strength'. Stratification on a continuous target (yield strength) requires binning, which introduces arbitrary bin boundaries. For small datasets (N ~80), this can lead to empty bins or highly imbalanced splits, invalidating the 'stratified' claim and potentially introducing selection bias.
- The plan calculates 'Mixing Enthalpy' using binary interaction parameters ($\Omega_{ij}$). The source of these parameters is not specified. If these are taken from a different dataset or literature with different conditions, they introduce a confound (data source mismatch) that is not modeled or controlled for, threatening construct validity.
- FR-001 requires downloading 'Materials Project data' in addition to the MPEA database. The plan.md, research.md, and data-model.md exclusively reference the MPEA database (DOI 10.1038/s41597-020-00768-9) and the 'MPEA_DB' source. There is no corresponding task, data source, or ingestion logic for the Materials Project dataset mentioned in FR-001. This leaves the 'Materials Project' portion of FR-001 unbacked.
- FR-003 mandates calculating 'electronegativity difference'. While the research.md lists this descriptor in Section 2, the plan.md 'Technical Context' and 'Project Structure' (specifically `code/features.py`) do not explicitly list 'electronegativity difference' as a calculated feature, only 'atomic radius mismatch, VEC, mixing entropy/enthalpy'. The data-model.md `CompositionalDescriptor` entity includes `electronegativity_diff`, but the plan's implementation steps lack a specific task or phase to compute this specific descriptor, creating a gap between the requirement and the planned execution steps.
- FR-004 requires a 'stratified 80/20 train-test split based on binned compositional ranges'. The research.md Section 3 mentions 'Stratified 80/20 split based on binned yield strength'. The plan.md does not specify the stratification variable. Stratifying by 'yield strength' (a target variable) is standard, but the spec explicitly requires stratification by 'binned compositional ranges' (features). The plan lacks a specific task to implement this distinct stratification logic, risking a mismatch with the spec's requirement.
- FR-005 specifies validation strategies: 'LOOCV for N < 100, 5-fold CV for N ≥ 100'. The research.md Section 3 states: 'If N < 100: LOOCV... If N ≥ 100: 5-Fold Cross-Validation'. However, the plan.md 'Technical Context' and 'Summary' do not explicitly mention this conditional logic or the N=100 threshold for switching validation methods. The plan only mentions 'training/evaluating regression models' generally. The specific conditional logic required by FR-005 is missing from the plan's explicit steps.

The plan assumes the DOI resolves to a standard repository and that the dataset contains ≥80 BCC alloys. However, the MPEA dataset (Multi-principal Element Alloys) is a general repository; it does not guarantee a specific subset of BCC alloys with yield strength values. Relying on a general dataset without a verified, filtered subset or a specific query path that guarantees the required modality (yield strength) and phase (BCC) creates a high risk that the dataset is inappropriate for the specific research question. The plan must verify that the specific DOI contains the required BCC+YieldStrength subset or identify a more specific, verified dataset.

The fallback strategy ('halt with Data Scarcity Warning') is a valid safety mechanism, but the primary strategy relies on an unverified assumption about the MPEA DOI's content structure. The plan must either confirm the MPEA DOI contains the specific BCC yield strength data or identify an alternative verified source before proceeding, as the current dataset choice is unproven for the specific question.

- The plan proposes using Isometric Log-Ratio (ILR) transformation on compositional data (FR-003.1) to solve multicollinearity, but then explicitly lists 'delta_radius', 'vec', 'mixing_entropy', and 'mixing_enthalpy' as separate predictors alongside 'ilr_features' in the data model. This creates a fundamental independence violation: these descriptors (δ, VEC, etc.) are deterministic functions of the raw composition. If ILR is applied to the composition, the raw composition is mathematically redundant. Including both the ILR coordinates AND the derived scalar descriptors (which are also derived from the same composition) in the same model without a clear theoretical justification for why the scalar descriptors add information *beyond* the ILR basis (which spans the simplex) risks a form of data leakage or redundant feature engineering that inflates apparent feature importance. The plan must clarify if the scalars are intended to replace the ILR features or be added to them, and justify the statistical independence of the signal.
- The calculation of Mixing Enthalpy (ΔH_mix) relies on binary interaction parameters (Ω_ij) which are often derived from the same thermodynamic databases used to generate the training data or are themselves estimated from the same elemental properties used for VEC/δ. If the source dataset (MPEA) was generated using CALPHAD methods that utilize these exact interaction parameters, using ΔH_mix as a predictor creates a circular validation where the model is essentially 'learning' the thermodynamic model used to generate the data, rather than discovering a new empirical relationship. The plan must verify that the target variable (yield strength) is experimentally measured and independent of the thermodynamic parameters used to calculate the predictors.
- The validation strategy proposes 'Stratified 80/20 split based on binned yield strength' (FR-004) but also 'LOOCV for N < 100'. These are mutually exclusive strategies for the same dataset. If N < 100, LOOCV is used, making the 80/20 split irrelevant. If N >= 100, the 80/20 split is used. However, for small N (e.g., 80-99), LOOCV has extremely high variance in the estimate of generalization error compared to repeated k-fold CV. The plan should specify a consistent cross-validation strategy (e.g., 5-fold repeated 10 times) that is robust for small N, rather than switching to LOOCV which can be unstable for regression tasks with small sample sizes.
