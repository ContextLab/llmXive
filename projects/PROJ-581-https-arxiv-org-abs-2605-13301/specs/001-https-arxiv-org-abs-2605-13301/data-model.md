# Data Model: Reproduce & Validate SU-01 Olympiad Reasoning

## Overview

This document defines the data structures used in the `SU-01` reproduction pipeline, focusing on the input datasets, intermediate generation artifacts, and the final evaluation results. The data model is designed to be lightweight and compatible with JSON/JSONL formats for easy parsing in a CPU-only environment.

## Input Data Models

### 1. Olympiad Problem (JSONL)
The primary input format for the inference engine.

```yaml
# Schema: olympiad_problem
type: object
properties:
  problem_id:
    type: string
    description: Unique identifier for the problem (e.g., "IMO-2025-P1").
  problem_text:
    type: string
    description: The full text of the mathematical problem.
  ground_truth:
    type: string
    description: The expected answer or solution key.
  difficulty_level:
    type: string
    enum: ["Easy", "Medium", "Hard"]
    description: Optional difficulty tag.
required:
  - problem_id
  - problem_text
  - ground_truth
```

### 2. Model Configuration (YAML/JSON)
Configuration for the inference run.

```yaml
# Schema: inference_config
type: object
properties:
  model_path:
    type: string
    description: Path to the model weights (local or HuggingFace ID).
  device:
    type: string
    enum: ["cpu"]
    description: Target device (strictly CPU).
  max_new_tokens:
    type: integer
    description: Maximum tokens to generate.
  temperature:
    type: number
    description: Sampling temperature.
  seed:
    type: integer
    description: Random seed for reproducibility.
required:
  - model_path
  - device
```

## Output Data Models

### 1. Generated Solution (JSONL)
Output from `direct_gen.py`.

```yaml
# Schema: generated_solution
type: object
properties:
  problem_id:
    type: string
    description: Reference to the input problem.
  generated_text:
    type: string
    description: The full text generated by the model.
  tokens_used:
    type: integer
    description: Number of tokens generated.
  generation_time_sec:
    type: number
    description: Time taken for generation.
  status:
    type: string
    enum: ["SUCCESS", "TIMEOUT", "ERROR", "SKIPPED"]
    description: Status of the generation.
required:
  - problem_id
  - generated_text
  - status
```

### 2. Evaluation Result (JSON)
Output from `eval_verifiable_answer.py`.

```yaml
# Schema: evaluation_result
type: object
properties:
  problem_id:
    type: string
    description: Reference to the input problem.
  is_correct:
    type: boolean
    description: Whether the generated answer matches the ground truth.
  reason:
    type: string
    description: Explanation for pass/fail (e.g., "Answer mismatch", "Timeout").
  confidence_score:
    type: number
    description: Optional confidence score from the verifier.
  timestamp:
    type: string
    format: date-time
    description: Time of evaluation.
required:
  - problem_id
  - is_correct
  - reason
```

### 3. Aggregated Report (JSON)
Final summary of the run.

```yaml
# Schema: aggregated_report
type: object
properties:
  run_id:
    type: string
    description: Unique identifier for the run.
  total_problems:
    type: integer
    description: Total number of problems attempted.
  passed_count:
    type: integer
    description: Number of correct answers.
  failed_count:
    type: integer
    description: Number of failed attempts.
  timeout_count:
    type: integer
    description: Number of timeouts.
  skipped_count:
    type: integer
    description: Number of skipped problems (e.g., missing weights).
  avg_generation_time_sec:
    type: number
    description: Average generation time.
  verdict:
    type: string
    enum: ["Pipeline Validated", "Pipeline Validated, Performance Claim Unverifiable", "Failed"]
    description: Overall verdict of the reproduction.
  notes:
    type: string
    description: Additional context (e.g., "Model weights missing, used fallback logic only").
required:
  - run_id
  - total_problems
  - passed_count
  - verdict
```

## Data Flow

1.  **Input**: `imo25.jsonl` / `usamo2026.jsonl` -> **Inference Engine** -> `generated_solutions.jsonl`
2.  **Processing**: `generated_solutions.jsonl` + `ground_truth` -> **Verifier** -> `evaluation_results.json`
3.  **Aggregation**: `evaluation_results.json` -> **Report Generator** -> `aggregated_report.json` + `reproduction_report.md`
