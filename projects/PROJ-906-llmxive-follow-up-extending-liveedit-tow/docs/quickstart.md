# Quickstart Guide: llmXive Automated Science Pipeline

This guide outlines the steps to execute the full research pipeline for the LiveEdit follow-up project.

## Prerequisites

- Python 3.9+
- Install dependencies:
 ```bash
 pip install -r code/requirements.txt
 ```

## Execution Steps

The pipeline is executed in stages via `code/main.py`.

### 1. Data Preparation
Downloads the dataset (DAVIS) and stratifies clips based on motion complexity.
```bash
python code/main.py --stage data_prep --dataset davis --stratify
```

### 2. Flow Computation
Computes optical flow for stratification and model warping.
```bash
python code/main.py --stage flow_compute --method farneback
```

### 3. Inference (Baseline)
Runs the baseline LiveEdit model.
```bash
python code/main.py --stage inference --model baseline
```

### 4. Inference (Flow-Coherence)
Runs the modified Flow-Coherence model.
```bash
python code/main.py --stage inference --model flow_coherence
```

### 5. Analysis
Performs statistical tests (KS, Piecewise Regression) and sensitivity analysis.
```bash
python code/main.py --stage analysis --sweep 0.01,0.05,0.1
```

### 6. Reporting (T017)
Generates the final JSON reports for baseline, flow, and analysis results.
```bash
python code/main.py --stage report
```

## Output Artifacts

- `data/metrics/baseline_results.json`: Baseline performance metrics.
- `data/metrics/flow_results.json`: Flow-Coherence performance metrics.
- `data/metrics/analysis_results.json`: Statistical analysis summary.
- `results/summary.md`: Final human-readable report (generated by T032).

## Troubleshooting

- **Missing Data**: Ensure `data_prep` and `flow_compute` stages complete successfully before running inference.
- **Memory Errors**: Reduce the dataset size or clip duration in `code/config.py` if OOM occurs.
- **Import Errors**: Verify `code/` is in your `PYTHONPATH` or run from the project root.
