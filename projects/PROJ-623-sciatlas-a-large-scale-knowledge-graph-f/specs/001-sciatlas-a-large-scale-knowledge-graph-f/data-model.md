# Data Model: Reproduce & Validate SciAtlas Knowledge Graph

## 1. Overview

This document defines the data model for the SciAtlas reproduction pipeline. It specifies the schema for `PaperRecord`, `SearchQuery`, and `ReproductionArtifact`, ensuring that the validation scripts (`artifact_validator.py`, `gold_standard_validator.py`) can enforce the requirements of `FR-002` and `US-2`.

## 2. Core Entities

### 2.1. PaperRecord
Represents a single academic paper retrieved from the SciAtlas knowledge graph (via OpenAlex).

| Field | Type | Required | Description |
|-------|------|----------|-------------|
| `title` | String | Yes | Title of the paper. Must be ≥ 5 characters. |
| `abstract` | String | Yes | Abstract of the paper. Must be ≥ 20 characters. |
| `authors` | List[String] | No | List of author names. |
| `year` | Integer | No | Publication year. |
| `source_id` | String | Yes | Unique identifier (DOI or ArXiv ID). |
| `entities` | List[String] | No | Extracted entities (e.g., methods, materials). |
| `experimental_evidence` | Boolean | No | Flag indicating if the paper contains experimental evidence (derived from Gold Standard NLP classifier). |
| `doi` | String | No | Digital Object Identifier. |
| `arxiv_id` | String | No | ArXiv identifier. |

**Constraints**:
- `title` and `abstract` must not be empty or whitespace-only.
- **Completeness Rule**: At least one of `doi` or `arxiv_id` must be present for a record to be considered "complete" (per `US-2`).
- `experimental_evidence` is a derived field, not native to the API.

### 2.2. SearchQuery
Represents the input to the SciAtlas pipeline.

| Field | Type | Required | Description |
|-------|------|----------|-------------|
| `query_string` | String | Yes | The user-defined search query (e.g., "graph neural networks"). |
| `limit` | Integer | Yes | Maximum number of results to retrieve (default: 100). |
| `filters` | Dict | No | Optional filters (e.g., year range, entity type). |

### 2.3. ReproductionArtifact
The final output file generated by the pipeline, containing search results or analysis summaries.

| Field | Type | Required | Description |
|-------|------|----------|-------------|
| `run_id` | String | Yes | Unique identifier for the run. |
| `query` | SearchQuery | Yes | The search query used. |
| `records` | List[PaperRecord] | Yes | List of retrieved paper records. |
| `status` | String | Yes | Execution status ("success", "partial", "failed", "data_unavailable"). |
| `log_path` | String | No | Path to the `reproduction_log.txt` file. |
| `validation_results` | Dict | No | Summary of validation checks (e.g., `completeness_score`, `recall_at_10`). |
| `data_source` | String | No | Indicates the source used ("openalex", "semantic_scholar", "mock"). |

## 3. Relationships

- A `ReproductionArtifact` contains zero or more `PaperRecord` instances.
- A `SearchQuery` triggers the generation of one `ReproductionArtifact`.
- `PaperRecord` entities are linked to the SciAtlas knowledge graph via `source_id`.

## 4. Data Flow

1. **Input**: `SearchQuery` is passed to `run_sciatlas.py`.
2. **Processing**: The script queries the OpenAlex API, retrieves `PaperRecord` instances, and logs API calls.
3. **Output**: A `ReproductionArtifact` (JSON) is generated.
4. **Validation**: `artifact_validator.py` parses the artifact and checks against the schema. `gold_standard_validator.py` checks retrieval accuracy.

## 5. Edge Cases & Handling

- **Empty Results**: If the API returns 0 results, the `ReproductionArtifact` will have an empty `records` list and `status` set to "partial". A specific "No results found" artifact is generated.
- **Missing Fields**: If a `PaperRecord` lacks `title` or `abstract`, it is flagged as "invalid" in the `validation_results`.
- **Network Failure**: If the API fails after retries, the `ReproductionArtifact` is not generated, and the system exits with a specific error code.
- **Data Unavailable**: If all APIs fail, `status` is set to "data_unavailable" and `data_source` is "mock" (if mock data is used for schema testing).
