## Summary

This feature implements a machine learning pipeline to predict molecular reactivity rankings for SN1, SN2, and Diels-Alder reactions using the USPTO-MIT reaction dataset. The approach involves ingesting raw reaction data, filtering by reaction templates, extracting structural features via RDKit, applying rigorous two-stage dimensionality reduction (Variance Thresholding + SelectKBest) to prevent overfitting on high-dimensional fingerprints, training a CPU-based XGBoost model with 5-fold cross-validation and Leave-One-Scaffold-Out validation, and performing a permutation test with 1,000 iterations to establish statistical significance. The entire pipeline is constrained to run within 30 minutes on a 2-core, 7GB RAM GitHub Actions free-tier runner through optimized batching and parameter tuning.

## Complexity Tracking

| Violation | Why Needed | Simpler Alternative Rejected Because |
|-----------|------------|-------------------------------------|
| Batch processing logic | Memory constraints (7GB) | Processing k+ reactions in one pass risks OOM; chunking is mandatory. |
| Template matching | Specific reaction class filtering | Simple keyword search is insufficient for SN1/SN2/Diels-Alder distinction; requires chemical graph analysis. |
| Permutation test | Statistical significance (FR-005) | Standard p-values from model output do not account for data leakage or chance in this specific observational setup; permutation is required. |
| Leave-One-Scaffold-Out (LOSO) | Generalization to new chemistry | 5-fold CV on mixed classes conflates classification with regression; LOSO ensures the model learns intrinsic reactivity, not just class identity. |
| **Dimensionality Reduction** | **Overfitting risk (High-Dim Fingerprints)** | **Direct use of 2048-bit fingerprints with <1,000 samples per class leads to the "curse of dimensionality"; SelectKBest + Variance Thresholding is mandatory to reduce feature count to a manageable ratio (k=100).** |
