## Unresolved concerns

- The plan proposes using 'Conformal Prediction' (FR-009) for Domain of Applicability (DoA) but defines the screening threshold as 'predicted log10(Rc) < 4.0'. Conformal Prediction generates prediction intervals (lower/upper bounds) for a given confidence level, not a binary 'in/out' flag based on a fixed target value. The methodology conflates 'uncertainty quantification' (is the prediction reliable?) with 'performance thresholding' (is the material good enough?). If a candidate has a predicted mean of 3.9 but a 95% CI of [2.0, 5.8], the current logic might flag it as 'good' (mean < 4.0) while the DoA logic might flag it as 'high risk' (wide interval). The plan lacks a decision rule for how to rank candidates when the mean is good but the uncertainty is high, or how to integrate the DoA status into the final ranking (e.g., penalizing high-risk candidates).
- The plan states: 'If p < 0.05, the model will be retrained with a weighted loss function (weights ∝ 1/σ̂²)'. This creates a circular dependency in the methodology. To calculate weights inversely proportional to variance (σ̂²), one must first estimate the variance of the residuals. However, the Breusch-Pagan test is performed on the *initial* model's residuals. The plan does not specify how to estimate the *local* variance (σ̂²) for the weighted loss retraining step. Standard weighted least squares requires known or estimated variances for each observation. Simply re-weighting by the inverse of the *global* residual variance from the first pass does not address heteroscedasticity (which implies variance changes with X). The methodology needs to specify a method for estimating local variance (e.g., using a separate variance model, binning residuals, or using the absolute residuals from the first pass as a proxy) before the weighted retraining can be valid.
- The screening strategy generates ~4,000 candidates and filters by 'predicted log10(Rc) < 4.0'. The spec (FR-009) requires flagging 'high extrapolation risk' for candidates outside the Domain of Applicability. The methodology does not define how the 'Domain of Applicability' is constructed for the *screening* set. Conformal Prediction typically requires a calibration set. If the calibration set is the training data, the DoA is defined by the training distribution. However, the plan does not explicitly state that the screening candidates are checked against the *convex hull* or *Mahalanobis distance* of the *training* feature space before prediction. Without this explicit check, the 'high extrapolation risk' flag (FR-009) cannot be reliably generated, as the model might be making predictions in regions where the physics-based descriptors have no support in the training data, rendering the prediction scientifically invalid regardless of the confidence interval.
- FR-002 requires computing 'valence electron count' (VEC) as an elemental descriptor. The plan.md 'Feature Engineering Strategy' lists 'Valence Electron Concentration (VEC)' but does not explicitly mention 'valence electron count' as a raw descriptor, only as part of the weighted mean. While VEC is often used interchangeably, the spec explicitly lists 'valence electron count' as a required output descriptor alongside interaction features. The plan should explicitly confirm the calculation of the raw 'valence electron count' per element or clarify that 'VEC' in the plan covers this requirement to avoid ambiguity in implementation.
- SC-002 requires measuring 'Feature importance rankings' against 'SHAP global importance values'. The plan.md 'Model Strategy' and 'Technical Context' sections mention Random Forest, Gradient Boosting, and bootstrapped ensembles, but make no mention of SHAP (SHapley Additive exPlanations) or any feature importance analysis. There is no phase, script, or output artifact in the plan dedicated to generating SHAP values or measuring feature importance. This success criterion is unbacked by the current plan.
- SC-003 requires measuring 'Novelty of discovered candidates' against 'existing literature'. The plan.md 'Screening Strategy' and 'Output' sections describe generating candidates, predicting GFA, and filtering/ranking them. However, there is no mechanism, script, or data source described in the plan to compare these candidates against 'existing literature' to measure novelty. The plan outputs a `verification_requests.json` but does not include a step to query literature or a database of known alloys to perform the novelty check required by SC-003.
- The plan cites 'GFA-D2 Pilot' with URL '.../pilot_flags/resolve/main/data.csv'. The filename 'pilot_flags' strongly suggests the dataset contains binary flags (e.g., glass-forming vs. crystalline) rather than the continuous critical cooling rate ($R_c$) values required to train a regression model for $log_{10}(R_c)$. If the dataset only contains binary labels, the regression task (FR-003) and the specific target variable ($log_{10}(R_c)$) cannot be computed, rendering the core methodology infeasible. The plan must verify the dataset actually contains continuous $R_c$ measurements, not just classification flags.

- The plan proposes retraining with weights 'inversely proportional to the predicted variance' ($\hat{\sigma}^2$) to fix heteroscedasticity. This is circular: the model's predicted variance is a function of its own inputs and current parameters. Using the model's own error estimates to weight its training creates a tautological loop where the model reinforces its existing bias rather than learning the true error structure. The weights must be derived from an external estimate of local variance (e.g., residuals from a preliminary fit binned by feature space, or a separate variance model), not the model's own predictions.
- The plan filters candidates by 'predicted $log_{10}(R_c) < 4.0$' to identify high GFA. In metallic glass literature, a *lower* critical cooling rate ($R_c$) indicates *better* glass-forming ability (easier to form glass). However, the plan defines the target as $log_{10}(R_c)$ and seeks to minimize it. While the direction is physically correct (minimize $R_c$), the threshold value of 4.0 ($10^4$ K/s) is arbitrary without empirical justification from the specific dataset distribution. If the dataset's $R_c$ values are all $> 10^5$, this filter yields zero candidates. The plan must verify the dataset's range or define the threshold relative to the dataset's quantiles (e.g., 'top 10% lowest $R_c$') rather than an absolute physical constant that may not apply to the specific alloy systems in the pilot data.
- The plan calculates 'Atomic Size Mismatch ($\delta$)' and 'Electronegativity Variance ($\Delta\chi$)' as global scalar features for the entire alloy. While standard, these are often insufficient for capturing the complex local atomic environments that dictate GFA in ternary systems. The plan does not include pairwise or triplet interaction descriptors (e.g., specific element pair mismatches) which are often critical for distinguishing between similar ternary compositions. Relying solely on global weighted means/variances may result in a model that cannot distinguish between chemically distinct but globally similar alloys, leading to poor generalization in the screening phase.
