# Research: Assessing the Validity of p-Values in High-Dimensional Data

## Overview

This research phase defines the methodology, data strategy, and statistical rigor required to answer the core question: *How do standard p-values deviate from uniformity under high-dimensional, non-independent, non-normal conditions?*

## Statistical Methodology

### 1. Data Generation (FR-001)
- **Correlation**:
 - *Primary*: Generate covariance matrix $\Sigma$ using a **Compound Symmetry** structure: $\Sigma_{ii} = 1$, $\Sigma_{ij} = \rho$ for $i \neq j$. This serves as the baseline for controlled violation.
 - *Secondary*: To address construct validity and ensure generalization to sparse/local correlation structures (e.g., gene expression), we will also simulate with an **AR(1)** structure: $\Sigma_{ij} = \rho^{|i-j|}$.
- **Distribution**:
 - *Normal*: Multivariate Normal.
 - *Heavy-tailed*: Multivariate t-distribution (df=3).
 - *Skewed*: Multivariate Skew-Normal.
- **Null Condition**: All features have mean 0. No group differences.
- **Dimensions**: Sweep $n \in \{50, 100, 200, 500\}$ and $p \in \{500, 1000, 2000, 5000\}$.

### 2. Hypothesis Testing (FR-002, FR-003)
- **Tests**: One-sample t-test (comparing feature mean to 0) and F-test (variance ratio if applicable).
- **Variance Estimator**: We use the **standard sample variance** estimator. This choice is intentional: we are studying the deviation of the standard t-test *as typically implemented*. The permutation reference uses the same estimator on permuted data, ensuring a fair comparison. We acknowledge that in high dimensions ($p \gg n$), this estimator can be unstable, and this instability is part of the phenomenon under investigation.
- **Execution**: Apply test to each of the $p$ features independently.
- **Output**: Vector of $p$ p-values per iteration.

### 3. Gold Standard Reference (FR-004)
- **Method**: Permutation test using **sign-flipping**.
- **Procedure**: For one-sample tests, we employ **sign-flipping** (randomly multiplying observations by -1) rather than label shuffling. This preserves the correlation structure of the noise vector while enforcing the null hypothesis of zero mean. Standard label shuffling assumes exchangeability which may not hold for dependent data; sign-flipping is exact for symmetric distributions under the null and preserves the correlation structure of the noise, making it the appropriate reference for one-sample tests in high dimensions.
- **Permutation Count**: We perform **10,000 permutations** per simulation iteration to generate a robust reference distribution of p-values.
- **Reference Construction**: The "Gold Standard" reference distribution is the **Empirical Cumulative Distribution Function (ECDF)** of the p-values generated by the sign-flipping procedure. To ensure the KS statistic measures deviation from the true null rather than noise from finite permutations, we apply **Kernel Density Estimation (KDE)** to smooth the empirical null before computing the KS statistic. The bandwidth $h$ will be selected using **Silverman's rule of thumb** ($h = 0.9 \times \min(\sigma, IQR/1.34) \times n^{-1/5}$) to ensure reproducibility.
- **Rationale**: The permutation test is not a "Gold Standard" for the standard test's p-values (which are invalid), but a method to estimate the *true* null distribution of the test statistic under the specific correlation structure. We compare the standard test's p-value distribution to this estimated distribution to quantify the deviation caused by assumption violations.

### 4. Deviation Quantification (FR-004, FR-005, FR-007)
- **Metric**: Kolmogorov-Smirnov (KS) statistic between the empirical distribution of standard test p-values and:
 1. The theoretical Uniform(0,1) distribution (to assess absolute validity).
 2. The permutation-derived ECDF (to measure deviation caused by assumption violations).
- **Visualization**: QQ-plots (Standard vs. Permutation).
- **Sensitivity**: Sweep $\rho \in \{0.0, 0.1, 0.3, 0.5, 0.7, 0.9\}$.
- **Confidence Intervals**: For every KS statistic reported, we will compute a **bootstrap-derived 95% confidence interval** as required by Constitution Principle VII. This is implemented in `analyze_pvalues.py`.

## Statistical Rigor & Assumptions

### Multiple Comparison Correction
- Since we are analyzing the *distribution* of p-values (not individual tests), we do not apply FDR correction to the p-values themselves. Instead, we test the *uniformity* of the distribution using the KS test.
- **Correction**: If multiple KS tests are run (e.g., one per $\rho$), we are testing multiple hypotheses (is the p-value distribution uniform for each $\rho$?). In this case, we will apply a **Bonferroni correction** to the significance threshold of the KS test itself (e.g., $\alpha_{corrected} = 0.05 / 6$) to control the family-wise error rate for the uniformity tests.

### Sample Size / Power Justification
- **Target**: Detect a KS statistic deviation > 0.05 with power $\ge 0.8$.
- **Strategy**: Run **1000 simulation iterations** per setting (fixed design parameter).
- **Permutation Count**: 10,000 permutations per iteration.
- **Limitation**: If the deviation is subtle (KS < 0.02), 1000 iterations may not provide sufficient power. This will be acknowledged as a limitation in the final report.

### Causal Inference & Measurement Validity
- **Observational Nature**: This is a simulation study, not an observational study of real data. The "causal" claim is that *violating assumptions causes p-value invalidity*.
- **Identification**: The identification strategy is the controlled generation of data where the only variable changing is the correlation structure or distribution type.
- **Validity**: The instruments (t-test, F-test) are standard. The validity of the "Gold Standard" relies on the correctness of the sign-flipping implementation.

### Collinearity
- **Issue**: High correlation ($\rho \to 1$) implies collinearity.
- **Handling**: We do **not** claim independent effects of correlated features. The analysis treats the *set* of p-values as a distribution. We explicitly acknowledge that as $\rho \to 1$, the effective dimensionality drops, and the t-test's degrees of freedom assumptions become increasingly violated.

## Compute Feasibility

- **Hardware**: GitHub Actions Free Tier (limited CPU, constrained RAM).
- **Constraints**:
 - Matrix inversion ($O(p^3)$) for $p=5000$ is expensive. We will use Cholesky decomposition or regularization ($\epsilon = 10^{-6}$) to ensure stability.
 - Memory: Storing $5000 \times 500$ float64 matrix requires ~20 MB. 1000 iterations fits easily in RAM.
 - Time: 1000 iterations $\times$ 10,000 permutations $\times$ 5000 tests $\approx 5 \times 10^{10}$ operations. This is computationally intensive.
 - **Optimization**: We will vectorize the permutation step where possible.
 - **Fallback**: If runtime exceeds 5 hours, we will reduce the permutation count to **5,000** per iteration. This trade-off is documented as a potential source of increased noise in the reference distribution.
- **Estimated Runtime**: Target < 4 hours with vectorized operations; fallback to 5,000 permutations if needed.

## Decision Rationale

- **Why Sign-Flipping?** Standard permutation (label shuffling) assumes exchangeability which may not hold for dependent data. Sign-flipping is exact for symmetric distributions under the null and preserves the correlation structure of the noise, making it the appropriate "Gold Standard" for one-sample tests in high dimensions.
- **Why Synthetic Data?** Real high-dimensional data (e.g., gene expression) has unknown ground truth. To measure "deviation from uniformity," we must know the null is true. Synthetic data is the only way to guarantee this.
- **Why AR(1) Structure?** To ensure findings generalize beyond the artificial Compound Symmetry structure, we include AR(1) as a secondary regime representing sparse, local correlations common in real data.
