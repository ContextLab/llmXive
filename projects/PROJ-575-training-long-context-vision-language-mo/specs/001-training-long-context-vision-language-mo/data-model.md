# Data Model: Reproduce & Validate Long-Context VLM Training with MMLongBench

## 1. Overview

This document defines the data structures used for the evaluation run, benchmark results, and scaling analysis. These structures map directly to the JSON artifacts generated by the `eval.py` script and the `scaling_analysis.py` module.

## 2. Entity Definitions

### 2.1 EvaluationRun

Represents a single execution of the evaluation pipeline.

*   `run_id`: Unique identifier (UUID).
*   `timestamp`: ISO 8601 timestamp of the run.
*   `sample_size`: Integer count of samples evaluated.
*   `context_lengths`: List of integers (e.g., `[32000, 64000, 128000]`).
*   `model_baseline`: Name of the baseline model (e.g., "Qwen2.5-VL-7B").
*   `model_target`: Name of the target model.
*   `hardware_config`: String describing the environment (e.g., "2-CPU-7GB-RAM-4bit").
*   `quantization_level`: String (e.g., "4-bit", "float16").

### 2.2 BenchmarkResult

Represents the score of a model on a specific task at a specific context length.

*   `task_id`: Identifier for the task category (e.g., "long-doc-vqa").
*   `context_length`: Integer context window size.
*   `score`: Float score for the target model (raw accuracy 0-100 or normalized 0-1).
*   `baseline_score`: Float score for the baseline model.
*   `improvement_pct`: Calculated percentage improvement: `((score - baseline_score) / baseline_score) * 100`.
*   `status`: String ("success", "failed", "oom").
*   `metric_scale`: String ("accuracy", "normalized") to clarify the score scale.

### 2.3 ScalingAnalysis

Represents the aggregated analysis of performance across context lengths.

*   `slope_coefficient`: Float from regression of score vs. log(context_length).
*   `r_squared`: Float goodness-of-fit metric.
*   `trend_classification`: String ("linear", "sublinear", "superlinear").
*   `p_value`: Float (null if not applicable due to low power).
*   `multiple_comparison_corrected`: Boolean (always `false` for this plan due to low power).
*   `limitation_note`: String describing the statistical power constraints (e.g., "n=5 context lengths, n=10 samples; descriptive only").

## 3. Data Flow

1.  **Input**: `MMLongBench` dataset (test split) loaded via `datasets` library.
2.  **Processing**: `eval.py` iterates over samples, loads the model (4-bit), and computes scores.
3.  **Output 1**: `results/evaluation_run.json` containing a single `EvaluationRun` object.
4.  **Output 2**: `results/sample_results.json` containing a list of `BenchmarkResult` objects.
5.  **Aggregation**: `scaling_analysis.py` reads `sample_results.json`, performs regression, and outputs `scaling_report.json` containing `ScalingAnalysis` data.

## 4. Constraints

*   **Data Volume**: The dataset is subsetted to `--sample-size=10` to fit within 7 GB RAM.
*   **Precision**: Scores are stored as floats (64-bit) to ensure precision in percentage calculations.
*   **Metric Scale**: The `metric_scale` field must be explicitly defined to avoid tautological interpretations of the `Retention` metric.
*   **Validation**: All output JSON files must conform to the schemas defined in `contracts/`.
