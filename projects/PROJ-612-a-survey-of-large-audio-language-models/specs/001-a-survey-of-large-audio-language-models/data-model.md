# Data Model: Survey of Large Audio Language Models – Hallucination Analysis

## 1. Overview
This document defines the data structures used for audio samples, model outputs, hallucination labels, and statistical results. All data is stored in machine-readable formats (CSV, JSON, YAML) to ensure reproducibility (Constitution Principle I).

## 2. Entity Definitions

### AudioSample
Represents a single audio clip processed by the pipeline.
- `id`: Unique identifier (string).
- `domain`: One of `speech`, `music`, `environmental`.
- `duration_seconds`: Float.
- `source_url`: String (original dataset URL).
- `ground_truth_labels`: Dict (e.g., `{"instrument": "piano", "speaker": "unknown"}`).
- `status`: One of `processed`, `skipped` (if >10s or missing metadata).
- `metadata_completeness`: Float (0.0 to 1.0, percentage of fields present).

### Caption
Text output generated by an LALM.
- `sample_id`: String (FK to AudioSample).
- `model_name`: String.
- `text`: String (raw output).
- `timestamp`: ISO8601.

### HallucinationLabel
Binary classification of a caption.
- `sample_id`: String.
- `model_name`: String.
- `is_hallucinated`: Boolean.
- `detected_contradictions`: List of strings (e.g., "piano" vs "guitar").
- `detector_version`: String.
- `match_type`: String (one of `exact`, `fuzzy`, `synonym`).

### TrainingDataEstimate
Metadata about model training.
- `model_name`: String.
- `domain`: One of `speech`, `music`, `environmental`.
- `volume_estimate`: String or Float (e.g., "10k hours" or 10000).
- `proxy_type`: String (one of `hours`, `tokens`, `ordinal`).
- `source_citation`: String (URL or paper title).
- `uncertainty`: One of `high`, `medium`, `low`.

### CorrelationResult
Output of the statistical analysis.
- `domain_pair`: String (e.g., "speech-music").
- `correlation_coefficient`: Float.
- `interpretation`: String (associational only).
- `rank_order_description`: String (e.g., "Env < Music < Speech").
- `sensitivity_stable`: Boolean (true if trend holds across permutations).
- `framing`: String (must be "associational").
- `sample_size`: Integer (N=3).

### HumanJudgment
Output of human validation.
- `sample_id`: String.
- `annotator_id`: String.
- `judgment`: Boolean (true=hallucinated).
- `platform`: String (e.g., "mturk").

## 3. File Formats

### `data/processed/hallucination_rates.csv`
Columns: `domain`, `total_samples`, `hallucinated_count`, `rate`, `ci_lower`, `ci_upper`, `ci_width`, `metadata_completeness_avg`.

### `data/processed/training_data_estimates.yaml`
YAML list of `TrainingDataEstimate` objects.

### `results/correlation_report.json`
JSON object containing `correlation_coefficient`, `interpretation`, `rank_order_description`, `sensitivity_stable`, `framing`, and `limitations`.

### `state/...yaml`
Contains SHA-256 checksums for all raw data files.

## 4. Validation Rules
- **Domain**: Must be one of `speech`, `music`, `environmental`.
- **Duration**: Must be ≤10.0 for processed samples.
- **Rate**: Must be between 0.0 and 1.0.
- **Citation**: All `source_citation` fields must reference a verified URL or a primary paper title.
- **Correlation**: `p_value` and `confidence_interval` fields MUST NOT be present (N=3 invalidity).
