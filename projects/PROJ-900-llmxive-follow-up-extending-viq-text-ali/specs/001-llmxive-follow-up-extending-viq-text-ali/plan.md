# Implementation Plan: llmXive follow-up: extending "ViQ: Text-Aligned Visual Quantized Representations at Any Resolution"

## Summary

This feature implements a rigorous evaluation of the "ViQ" (Text-Aligned Visual Quantized Representations) hypothesis regarding resolution invariance. The technical approach involves training a VQ-VAE codebook exclusively on low-resolution (64x64) COCO image-text pairs using CPU-only hardware, then evaluating its reconstruction fidelity and semantic alignment on high-resolution (1024x1024) images from ImageNet-1K and a subset of COCO (for semantic ground truth). The plan strictly adheres to the project's constraints: no GPU usage, strict separation of training and evaluation resolutions, and non-circular semantic validation using a frozen CLIP text encoder.

*Note: ChestX-ray14 has been removed from scope due to lack of verified source and CI compatibility risks. The spec's FR-004 (bilinearly upsampled ground truth) and SC-005 (one-sample t-test) are flagged as scientifically unsound and require amendment to align with the native-resolution evaluation and paired statistical testing described below.*

## Technical Context

**Language/Version**: Python 3.11

**Target Platform**: Linux (GitHub Actions free-tier runner: CPU, 7GB RAM, no GPU).  
**Project Type**: Computational research / Machine Learning Experiment.  
**Performance Goals**: Training loop converges on 64x64 sample within 6 hours; inference on 1024x1024 batch completes within memory limits.  
**Constraints**: Strict CPU-only execution; no CUDA; dataset subsampling required to fit constrained memory resources; no modification of raw data.  
**Scale/Scope**: Training on a subset of COCO (approx. a representative number of images); Evaluation on N=200 images (ImageNet + COCO subset).

## Spec Amendments Required

The following items in `spec.md` conflict with the scientifically sound implementation plan and must be amended:
1.  **FR-004**: Requires comparison against "bilinearly upsampled 64x64 ground truth". **Correction**: Must compare against native 1024x1024 ground truth to avoid interpolation bias. The upsampled target introduces a confound that invalidates the fidelity measurement.
2.  **US-2**: Ambiguously states "calculates a PSNR value" without specifying the target and includes ChestX-ray14. **Correction**: Must specify native 1024x1024 target and remove ChestX-ray14.
3.  **FR-003**: Includes "ChestX-ray14". **Correction**: Must be removed as the dataset is not in the verified block.
4.  **SC-005**: Mandates "one-sample t-test". **Correction**: Must be "paired t-test" or "Wilcoxon signed-rank test" to account for variance in both groups.
5.  **US-2**: Requires evaluation on ChestX-ray14. **Correction**: Remove ChestX-ray14; rely on ImageNet-1K and COCO for evaluation.
6.  **General**: The plan explicitly requires a memory fallback strategy (batch size 1, dynamic N reduction) not detailed in the spec.
