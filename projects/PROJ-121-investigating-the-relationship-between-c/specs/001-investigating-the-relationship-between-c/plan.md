# Implementation Plan: Cosmic Ray Anisotropy Solar‑Cycle Modulation

**Branch**: `001-cosmic-ray-anisotropy-solar-cycle` | **Date**: 2026-06-24 | **Spec**: `spec.md`

## Summary

This plan implements a reproducible, CPU-only pipeline to investigate the correlation between cosmic ray anisotropy (dipole amplitude/phase) and solar cycle variations (sunspot number, solar wind, IMF) using IceCube and Pierre Auger data (-2020). The system downloads verified datasets from official open-data portals, validates temporal coverage, bins events into configurable intervals (default duration), computes relative anisotropy (correcting for detector acceptance), and fits spherical harmonic coefficients (ℓ≤2). Statistical significance is tested using Lomb-Scargle, block-bootstrap, and Monte-Carlo shuffle (shuffling solar proxy as per spec) with Bonferroni correction. The plan includes a blind validation step and generates a LaTeX report.

## Constitution Check

| Principle | Compliance Status | Implementation Detail |
|-----------|-------------------|-----------------------|
| I. Reproducibility | **COMPLIANT** | All scripts use pinned `requirements.txt`, fixed random seeds for reproducibility, and fetch data from canonical URLs. |
| II. Verified Accuracy | **COMPLIANT** | Citations in `research.md` and `data-model.md` are restricted to the "Verified datasets" block. **Data download steps explicitly map to the sources listed in research.md's "Verified datasets" block.** |
| III. Data Hygiene | **COMPLIANT** | SHA-256 checksums verified on download. Raw data immutable; derivatives in `data/processed`. |
| IV. Single Source of Truth | **COMPLIANT** | All figures/stats in report trace to specific CSV rows generated by `analyze_correlation.py`. |
| V. Versioning Discipline | **COMPLIANT** | `requirements.txt` pins versions. Artifact hashes stored in state YAML. |
| VI. Multi-Detector Consistency | **COMPLIANT** | Analysis runs independently for IceCube and Auger. Combined analysis uses Fisher's Method (meta-analysis) to avoid selection bias. **Significance threshold is strictly α=0.0017 (Bonferroni-corrected) as per spec, not a colloquial '3σ' threshold.** |
| VII. Statistical Analysis Protocol | **COMPLIANT** | Lomb-Scargle, block-bootstrap (multiple bins), Monte-Carlo shuffle (multiple permutations, shuffling solar proxy as per spec) implemented. **For a default bin of specified duration, block length is set to twice the bin duration, satisfying the constitution's intent and spec's formula.** |
