# Data Model: Reproduce & Validate SpatialClaw

## Entities & Relationships

### 1. Reproduction Run
A single execution instance.
- **ID**: UUID or timestamp-based string.
- **Dataset**: Name and subset size.
- **Model**: Configuration used.
- **Status**: `running`, `completed`, `failed`, `aborted`.
- **Resource Usage**: Peak RAM (MB), CPU Time (s).

### 2. Run Artifact (JSON)
Structured output from the agent.
- **Run ID**: Link to Reproduction Run.
- **Samples**: List of processed samples.
  - **Sample ID**: Unique identifier.
  - **Input**: Question/Task description.
  - **Intermediate Steps**: List of code blocks and VLM responses.
  - **Final Answer**: The agent's final output.
  - **Execution Status**: `success`, `error`, `unanswerable`.
  - **Accuracy**: Boolean (if ground truth available).

### 3. Validation Report
Post-run analysis.
- **Run ID**: Link to Reproduction Run.
- **Accuracy Deviation**: Calculated difference from paper baseline.
- **Pass/Fail Status**: Based on tolerance and resource limits.
- **Warnings**: List of issues (e.g., "Model Mismatch", "Low Sample Size").

## Data Flow

1.  **Input**: `BLINK` dataset (Parquet) -> `spatial_agent/entrypoints/run.py`.
2.  **Processing**: Agent iterates samples -> Generates code -> Executes code -> Collects results.
3.  **Output**: `results/blink_subset_X.json` (Run Artifact).
4.  **Validation**: `results/...json` + Paper Baseline -> `results/validation_report.json`.
5.  **Logging**: All steps -> `logs/run.log`.

## Schema Definitions

The following schemas define the contract for the artifacts generated by this feature.

- **Run Artifact**: `contracts/run_artifact.schema.yaml`
- **Validation Report**: `contracts/validation_report.schema.yaml`
