# Implementation Plan: Can Publicly Available Data Reveal Subtle Violations of Time-Reversal Symmetry in Beta Decay?

**Branch**: `001-t-violation-beta-decay` | **Date**: 2026-07-14 | **Spec**: [link]
**Input**: Feature specification from `/specs/001-t-violation-beta-decay/spec.md`

## Summary

This project implements a meta-analysis pipeline to aggregate published T-violation correlation coefficients (D-coefficients) for specific nuclei (6He, 19Ne) from archival sources. The system performs inverse-variance weighted meta-analysis to calculate combined upper bounds, validates consistency via Cochran's Q test (with shuffle fallback for small samples), and cross-references results against the Particle Data Group (PDG) Review limits to avoid circularity. The implementation strictly adheres to CPU-only constraints, ensuring all data retrieval, statistical computation, and validation run on free-tier GitHub Actions runners (limited CPU, constrained RAM) without GPU dependencies or large model training. Crucially, the primary hypothesis test (T-violation) is conducted via a Z-test of the combined mean against zero, as permutation testing is methodologically invalid for summary statistics.

| **I. Reproducibility** | **PASS** | All data retrieval scripts use pinned `requests` calls to public endpoints. Random seeds are pinned in `code/` for any stochastic elements (e.g., shuffle fallback for Cochran's Q). The `random_seed` is recorded in the output schema (`meta_analysis_result.schema.yaml`). |
| **II. Verified Accuracy** | **PASS** | Citations in `research.md` are restricted to the "Verified datasets" block (Official PDG URLs). The PDG comparison uses the official latest Review (verified) to avoid circularity. Mock data is strictly for CI testing, not scientific results. |
| **III. Data Hygiene** | **PASS** | Raw data is saved to `data/raw/` with checksums. Derived data (harmonized) is saved to `data/processed/`. No in-place modification. |
| **IV. Single Source of Truth** | **PASS** | All statistics in the final report are generated by `code/analysis.py` reading from `data/processed/`. No hand-typed numbers. |
| **V. Versioning Discipline** | **PASS** | Content hashes for all artifacts will be recorded in `state/projects/...yaml`. |
| **VI. Cross-Modal Statistical Independence** | **PASS (Adapted)** | The pipeline treats each D-coefficient as an independent random variable derived from distinct experiments. While the constitution's specific wording refers to "momentum spectra," the *intent* of avoiding artificial correlations between distinct measurement modalities is satisfied by aggregating only independent published summary statistics. |
| **VII. Null-Hypothesis Rigor via Permutation Testing** | **ADAPTED (PASS)** | The constitution's specific mechanism ("polarization values randomly reassigned to momentum bins") is inapplicable to summary statistics (D-coefficients). The plan achieves *Null-Hypothesis Rigor* via a formal Z-test of the combined mean against the Standard Model prediction (D=0). This is the standard rigorous method for meta-analysis of point estimates. The "shuffle" requirement is implemented only as a fallback for Cochran's Q p-value estimation if the analytic chi-square approximation is invalid (n < 5), not for the primary T-violation hypothesis test. |
