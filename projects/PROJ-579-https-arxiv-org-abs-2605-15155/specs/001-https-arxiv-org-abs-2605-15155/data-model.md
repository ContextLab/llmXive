# Data Model: Self-Distilled Agentic Reinforcement Learning (SDAR) Reproduction

## 1. Overview

This document defines the data artifacts generated by the SDAR reproduction pipeline. All data is derived from the actual execution of `external/SDAR/agent_system/train.py` and `eval.py` (or their mock equivalents if Thor fails). No synthetic data is generated. The data model ensures traceability from raw logs to final statistical reports, satisfying the "Single Source of Truth" and "Data Hygiene" principles.

## 2. Data Flow

1.  **Raw Logs**: Generated by `train.py` and `eval.py` (stdout/stderr and JSON logs).
2.  **Parsing**: `code/parse_logs.py` transforms raw logs into structured JSON/CSV. This script also updates `state/projects/...yaml` with new checksums.
3.  **Analysis**: `code/analyze_results.py` generates the statistical report (diagnostic only).
4.  **Final Report**: Aggregated in `docs/reproducibility/reproducibility_report.md`.

## 3. Artifact Definitions

### 3.1 Raw Training Logs (`data/raw/train_log_<seed>.json`)
- **Source**: `external/SDAR/agent_system/train.py` (or mock output)
- **Format**: JSON Lines (one object per step)
- **Fields**:
  - `step`: Integer, current training step.
  - `seed`: Integer, random seed used.
  - `method`: String, "SDAR" or "PPO".
  - `gate_loss`: Float, SDAR gating loss (KL divergence).
  - `rl_loss`: Float, Reinforcement Learning loss.
  - `total_loss`: Float, Sum of losses.
  - `gate_activation`: Boolean, whether the gate chose the distillation path.
  - `teacher_update`: Boolean, whether a teacher update occurred.
  - `timestamp`: ISO 8601 string.

### 3.2 Raw Evaluation Logs (`data/raw/eval_log_<seed>.json`)
- **Source**: `external/SDAR/agent_system/eval.py` (or mock output)
- **Format**: JSON Lines (one object per task)
- **Fields**:
  - `task_id`: String, unique task identifier (from fixed set).
  - `seed`: Integer, random seed used.
  - `method`: String, "SDAR" or "PPO".
  - `success`: Boolean, `True` if task completed.
  - `steps_taken`: Integer, number of actions taken.
  - `timeout`: Boolean, `True` if task exceeded 60s.
  - `reward`: Float, final reward value.

### 3.3 Parsed Results (`data/processed/sdar_results.csv`)
- **Source**: `code/parse_logs.py`
- **Format**: CSV
- **Fields**:
  - `seed`: Integer
  - `method`: String ("SDAR" or "PPO")
  - `final_success_rate`: Float (0.0 to 1.0)
  - `avg_gate_loss`: Float (for SDAR only)
  - `avg_rl_loss`: Float
  - `total_steps`: Integer
  - `gate_activation_rate`: Float (percentage of steps where gate activated)

### 3.4 Statistical Report (`data/processed/statistical_analysis.json`)
- **Source**: `code/analyze_results.py`
- **Format**: JSON
- **Fields**:
  - `test_type`: String ("paired_t_test" or "wilcoxon")
  - `sdar_mean`: Float
  - `ppo_mean`: Float
  - `p_value`: Float
  - `significant`: Boolean (p < 0.05)
  - `n_seeds`: Integer (5)
  - `note`: String (e.g., "Diagnostic only; low power due to N=5 and 1000 steps")

## 4. Data Hygiene & Versioning

- **Checksums**: All files in `data/raw/` and `data/processed/` are checksummed (SHA-256) and recorded in `data/checksums.yaml`.
- **State Update**: The `parse_logs.py` script MUST update `state/projects/...yaml` with the new artifact hashes and `updated_at` timestamp upon successful generation.
- **Immutability**: Raw logs are never modified. Parsed artifacts are overwritten only if the source raw logs change.
- **PII**: No Personally Identifiable Information is expected in ALFWorld logs. If any user-specific tokens appear in logs, they will be masked by `parse_logs.py`.

## 5. Assumptions

- The `external/SDAR` codebase outputs logs in a parseable format (JSON or structured text). If the format differs, `parse_logs.py` will be updated to handle the actual output structure.
- The ALFWorld environment returns a consistent `success` flag structure.
- If the Thor binary fails, the mock environment will generate logs with the same schema to allow the pipeline to complete.
