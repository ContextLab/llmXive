# Data Model: llmXive follow-up: extending "Orca: The World is in Your Mind"

## Overview
This document defines the data schemas, transformations, and storage formats for the project. All data is stored in `data/` with raw data immutable and processed data derived.

## Entities

### 1. PhysicalScenario
Represents a single curated video clip and its metadata.
- **Attributes**:
  - `video_id`: Unique identifier (string).
  - `original_prompt`: The original caption/prompt for the video.
  - `interaction_type`: Category of physical interaction (e.g., "falling", "collision").
  - `counterfactual_prompt`: The manually annotated counterfactual prompt (e.g., "remove gravity").
  - `status`: "valid", "skipped", "ambiguous".
  - `file_path`: Path to the raw video file (if available locally) or URL.

### 2. LatentVector
Represents the frozen Orca world latent representation.
- **Attributes**:
  - `scenario_id`: FK to `PhysicalScenario`.
  - `vector_type`: "full", "conscious", "unconscious".
  - `embedding`: Array of floats (shape: [dim]).
  - `is_valid`: Boolean (True if extraction succeeded).

### 3. CounterfactualEdit
Represents the transformation applied to a `LatentVector`.
- **Attributes**:
  - `scenario_id`: FK to `PhysicalScenario`.
  - `original_vector_id`: FK to `LatentVector`.
  - `operation`: "subtraction", "masking".
  - `concept_token`: The token or vector component manipulated.
  - `modified_vector`: Array of floats (shape: [dim]).
  - `flag_ambiguous`: Boolean (True if the prompt was ambiguous).

### 4. GroundTruthLabel
Represents the verified outcome from the physics engine.
- **Attributes**:
  - `scenario_id`: FK to `PhysicalScenario`.
  - `outcome`: Binary label (e.g., "falls", "floats").
  - `simulation_id`: Unique ID for the physics simulation run (generated by `physics_verify.py` using a deterministic hash).
  - `verified`: Boolean (True if the vector arithmetic prediction matched the simulation).

### 5. ReadoutModelOutput
Represents the result of a trained model.
- **Attributes**:
  - `model_type`: "DecisionTree", "LinearProbe".
  - `input_type`: "Latent", "Pixel", "Unconscious".
  - `scenario_id`: FK to `PhysicalScenario`.
  - `predicted_outcome`: Predicted label.
  - `actual_outcome`: Ground truth label.
  - `correct`: Boolean.

## Storage Formats

### Raw Data (`data/raw/`)
- **Format**: Parquet (from HuggingFace).
- **Integrity**: Checksum recorded in `state/...yaml`.

### Processed Data (`data/processed/`)
- **Format**: CSV/JSON.
- **Files**:
  - `curated_scenarios.csv`: List of valid scenarios.
  - `latents.parquet`: Extracted latent vectors.
  - `counterfactuals.csv`: Modified vectors and edit metadata.
  - `physics_labels.csv`: Ground truth from simulation (N=50 subset).
  - `model_results.csv`: Aggregated prediction results.

### Logs (`data/logs/`)
- **Files**:
  - `pipeline.log`: General execution log.
  - `failed_scenarios.log`: Ambiguous prompts and skipped files.
  - `memory_profile.log`: RAM usage over time.

## Data Flow Diagram
1. **Raw Parquet** -> `download_orca.py` -> **Filtered CSV** (Physical interactions only).
2. **Filtered CSV** -> `extract_latents.py` -> **Latents Parquet** + **Counterfactuals CSV**.
3. **Counterfactuals CSV** -> `physics_verify.py` -> **Physics Labels CSV** (N=50 subset).
4. **Latents + Physics Labels** -> `train_readout.py` -> **Model Results CSV**.
5. **Model Results CSV** -> `stats.py` -> **Final Metrics JSON**.
