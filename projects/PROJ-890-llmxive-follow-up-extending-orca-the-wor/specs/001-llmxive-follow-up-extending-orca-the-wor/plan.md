## Summary

This feature implements a CPU-tractable research pipeline to test the hypothesis that the "Orca" multimodal model's latent space encodes causal priors regarding physical interactions better than raw pixel data. The approach involves: (1) curating a subset of physical interaction clips from the Orca dataset; (2) extracting frozen latent vectors; (3) **probing to identify concept-specific vector dimensions** (e.g., gravity); (4) applying symbolic counterfactual edits (vector arithmetic/masking) to simulate physical law changes; (5) validating these edits against an independent physics engine simulation (PyBullet/MuJoCo) on a subset (N=50); and (6) training lightweight `DecisionTreeClassifier` and `LinearProbe` models.

**Critical Distinction**: The pipeline distinguishes between:
1.  **Descriptive Analysis (N=450)**: Training models on original physical outcomes (ground truth from video labels) to establish a correlation baseline.
2.  **Causal Consistency Test (N=50)**: Training models on *simulated counterfactual* outcomes (ground truth from physics engine) to test if the latent space aligns with physics engine logic.

The pipeline strictly adheres to CPU-only constraints (limited vCPU, constrained RAM) and handles data edge cases (corruption, ambiguity, validation failure) gracefully. If the vector arithmetic validation fails, the pipeline **blocks the causal analysis branch** but continues with the descriptive analysis.

## Complexity Tracking

| Violation | Why Needed | Simpler Alternative Rejected Because |
|-----------|------------|-------------------------------------|
| Dual-Model Comparison (Latent vs. Pixel) | Required by FR-003 and FR-004 to isolate the causal advantage of the latent space. | A single model would fail to prove the hypothesis that "conscious scaffolding" provides a unique causal signal over raw correlation. |
| Physics Engine Validation (FR-009) | Required to break the tautological loop of logical determination and provide ground truth for counterfactuals. | Logical simulation alone is insufficient per Assumptions and Constitution Principle VI; an independent engine is mandatory for scientific validity. |
| Adaptive Batch Sizing | Required by SC-005 and Edge Cases to handle the 7GB RAM limit on variable video sizes. | Fixed batch sizes risk OOM crashes on larger clips, violating the "runnable on free CI" constraint. |
| Ambiguity Logging & Filtering | Required by Edge Cases and FR-003 to handle semantic mismatches without halting the pipeline. | A hard fail on any ambiguous prompt would prevent the analysis of valid data. |
| Conditional Causal Gate | Required to handle cases where vector arithmetic fails to map to physics logic. | A hard halt would waste resources on descriptive analysis; a soft gate allows the pipeline to complete descriptive tasks while flagging the causal claim as unverified. |

## Task Order & Dependencies

The following task sequence ensures data is prepared, validated, and filtered before training, addressing the dependency concerns.

1.  **T010**: **Data Curation** (Download & Filter).
    *   *Input*: Verified Dataset URL.
    *   *Output*: `data/raw/scenarios.csv` (N=450).
2.  **T011**: **Latent Extraction**.
    *   *Input*: `data/raw/scenarios.csv`.
    *   *Output*: `data/processed/latents.npy`.
3.  **T019**: **Concept Localization**.
    *   *Input*: `data/processed/latents.npy`, `data/metadata/manual_annotations.csv`.
    *   *Action*: Probe latents to identify vector dimensions corresponding to physical concepts (e.g., gravity).
    *   *Output*: `data/processed/concept_vectors.json`.
4.  **T021**: **Counterfactual Injection (Descriptive)**.
    *   *Input*: `data/processed/latents.npy`, `data/processed/concept_vectors.json`.
    *   *Action*: Apply vector arithmetic to ALL clips (N=450) to generate $z_{cf}$.
    *   *Output*: `data/processed/edited_latents.npy` (All clips).
5.  **T020**: **Physics Simulation (Subset)**.
    *   *Input*: `data/raw/scenarios.csv` (Subset N=50).
    *   *Output*: `data/processed/physics_labels.csv` (Ground Truth for N=50).
6.  **T026a**: **Vector Arithmetic Validation (Gate)**.
    *   *Input*: `data/processed/edited_latents.npy` (Subset N=50), `data/processed/physics_labels.csv`.
    *   *Action*: Compare $z_{cf}$ predictions against Physics Engine outcomes. Calculate Accuracy.
    *   *Output*: `data/processed/validation_status.json` (Accuracy score, `causal_gate_status` flag).
    *   *Logic*: If Accuracy < [deferred], set `causal_gate_status = "blocked"`. Pipeline continues to descriptive tasks.
7.  **T022**: **Data Filtering & Curation**.
    *   *Input*: `data/processed/edited_latents.npy`, `data/processed/validation_status.json`, `data/raw/scenarios.csv`, `data/processed/concept_vectors.json`.
    *   *Action*: 
        *   Filter out ambiguous clips (FR-003).
        *   If `causal_gate_status` is "blocked", exclude the N=50 subset from causal training.
    *   *Output*: `data/processed/filtered_training_set.csv`.
8.  **T023**: **Latent Model Training (Causal)**.
    *   *Input*: `data/processed/filtered_training_set.csv` (Subset N=50 if gate passed).
    *   *Action*: Train `DecisionTreeClassifier` on $z_{cf}$ using physics labels.
    *   *Output*: `data/models/latent_causal.pkl`.
    *   *Dependency*: T022, T026a (Gate Pass).
9.  **T024**: **Pixel Baseline Training (Descriptive & Causal)**.
    *   *Input*: `data/processed/filtered_training_set.csv`.
    *   *Action*: 
        *   **Descriptive Mode**: Train on N=450 using *original* physical outcomes (labels from video metadata).
        *   **Causal Mode**: Train on N=50 (if gate passed) using *simulated* outcomes (labels from physics engine).
    *   *Output*: `data/models/pixel_descriptive.pkl`, `data/models/pixel_causal.pkl`.
10. **T025**: **Statistical Comparison**.
    *   *Input*: `data/models/latent_causal.pkl`, `data/models/pixel_causal.pkl`.
    *   *Action*: Paired t-test on N=50 subset (if gate passed).
    *   *Output*: `data/results/metrics.json`.
11. **T039**: **Performance Benchmark & Refactor Trigger**.
    *   *Input*: `data/results/metrics.json`.
    *   *Action*: Check runtime/memory. If > 6h or > 7GB, log `REFACTOR_REQUIRED`.
    *   *Output*: `data/logs/benchmark_status.json`.
