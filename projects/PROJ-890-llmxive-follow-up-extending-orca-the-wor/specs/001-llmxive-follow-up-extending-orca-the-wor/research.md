# Research: llmXive follow-up: extending "Orca: The World is in Your Mind"

## Research Question
Does the "conscious" linguistic scaffolding in the Orca world model provide an *associational advantage* for predicting physical interactions under counterfactual edits, compared to raw pixel data?

**Note**: The study explicitly avoids claiming "causal priors" in the dataset. It tests whether the latent representation, when manipulated via vector arithmetic, predicts the *simulated* counterfactual outcome better than a pixel baseline predicts the *original* outcome (or fails to predict the counterfactual).

### Secondary Data: Physics Simulation
- **Source**: Synthetic generation via `mujoco`.
- **Rationale**: Ground truth for counterfactuals (e.g., "remove gravity") is generated by *rendering* the scenario in an independent engine, ensuring no tautology with the model's training data. The video is generated *from* the simulation, not the other way around.

## Methodology

### Phase 0: Reference-Validator Gate & Audit Logging
1. **Validation**: Invoke the Reference-Validator Agent to check all dataset URLs against the "Verified datasets" block.
2. **Logging**: Initialize `failed_scenarios.log` and `pipeline.log` (FR-008).

### Phase 1: Data Curation & Disentanglement Probe
1. **Download**: Stream the Orca dataset (or synthetic fallback). Filter for clips containing physical interactions.
2. **Disentanglement Probe**: Run a preliminary analysis to verify the existence of distinct "conscious" (linguistic) and "unconscious" latent components. If disentanglement fails, halt.

### Phase 2: Latent Extraction & Counterfactual Injection
1. **Latent Extraction**:
 - Load the frozen Orca model (CPU).
 - Extract the "conscious" and "unconscious" latent vectors for each frame.
 - **Handling Ambiguity**: If a counterfactual prompt cannot be mapped to a known vector component, apply a zero-vector mask and flag the sample in `failed_scenarios.log`.
2. **Counterfactual Generation**:
 - Compute $z_{cf} = z - v_{concept}$ (vector arithmetic) or $z_{cf} = z \odot m$ (masking).
 - Store $z_{cf}$ alongside the original $z$.

### Phase 3: Ground Truth Generation (Synthetic)
1. **Simulation**: For a subset of clips, *render* the scenario in `mujoco` with the counterfactual condition (e.g., $g=0$).
2. **Labeling**: Record the outcome (e.g., "floats") as the ground truth label. The `simulation_id` is generated deterministically by `physics_verify.py`.
3. **Output**: Write verified labels to `data/processed/physics_labels.csv`.

### Phase 4: Vector Arithmetic Validation
1. **Hold-out**: Split the N=50 subset into training (40) and hold-out (10) sets.
2. **Training**: Train a readout model on the training set to predict the physics outcome.
3. **Validation**: Evaluate on the hold-out set. Calculate SC-008 (validity > 90%).
4. **Independence**: This step is decoupled from the main model training to avoid tautology.

### Phase 5: Model Training
1. **Main Model (N=450)**: Train `DecisionTreeClassifier` on *original* video labels (e.g., "falls") using original latents.
2. **Counterfactual Model (N=50)**: Train `DecisionTreeClassifier` on *counterfactual* labels using $z_{cf}$.
3. **Pixel Baseline**: Downsample raw video frames to a reduced grayscale resolution suitable for efficient processing. Train `DecisionTreeClassifier` on original frames to predict *original* outcomes.
4. **Ablation 1 (Method)**: Replace `DecisionTree` with `LogisticRegression` (Linear Probe) on $z_{cf}$.
5. **Ablation 2 (Scaffolding)**: Train `DecisionTree` on "unconscious" latents only.

### Phase 6: Statistical Analysis
1. **Metric**: Calculate "Counterfactual Sensitivity Score" (correlation between edit magnitude and prediction change).
2. **Test**: Compare Latent Model Sensitivity vs. Pixel Baseline Sensitivity (expected to be near zero).
3. **Correction**: Apply Bonferroni correction for multiple comparisons.
4. **Significance**: Threshold $p < 0.05$ (adjusted).
5. **Output**: Generate `stats_report.json` with `verified_ratio` (SC-007).

### Phase 7: Versioning & State Update
1. **Hashing**: Compute content hashes for all artifacts.
2. **State Update**: Update `state/projects/PROJ-890-llmxive-follow-up-extending-orca-the-wor.yaml` with hashes and timestamps.

## Assumptions & Limitations
- **Dataset Availability**: The study assumes the Orca video dataset is accessible or a verified synthetic fallback is available. If neither is found, the project halts.
- **Vector Arithmetic Validity**: Assumes that subtracting a concept vector from the latent space effectively removes that concept. This is validated by FR-010 on a hold-out set.
- **CPU Constraints**: The Orca model is assumed to be small enough to run on CPU without quantization. If not, the plan must be revised to use a smaller proxy model or a distilled version.
- **Physics Simulation Fidelity**: The simplified physics engine may not perfectly match the real-world video, introducing noise in the ground truth labels.

## Success Criteria (Revisited)
- **SC-001**: Latent Model Sensitivity > Pixel Baseline Sensitivity.
- **SC-002**: Statistical significance ($p < 0.05$) after correction.
- **SC-003**: Robustness across Decision Tree and Linear Probe.
- **SC-004**: Significant drop in accuracy when "conscious" scaffolding is removed.
- **SC-005**: Execution within 6 hours on 2 vCPU, 7GB RAM.
- **SC-006**: No GPU libraries imported.

- **SC-008**: Vector arithmetic validity > 90% on hold-out set.
