# Data Model: llmXive follow-up: extending "OmniDirector: General Multi-Shot Camera Cloning without Cross-Paired D"

## 1. Overview

This document defines the data structures used throughout the `001-gene-regulation` feature. It ensures consistency between the ingestion pipeline, the geometric solver, and the statistical analysis modules.

## 2. Key Entities

### 2.1 GridFrame
Represents a single video frame containing the "empty" camera grid.
- `frame_id`: Unique identifier (string).
- `sequence_id`: Identifier for the source video sequence.
- `image_path`: Relative path to the image file.
- `timestamp`: Frame timestamp (float).
- `intrinsics`: Camera matrix ($3 \times 3$) as a list of lists.
- `grid_lines`: List of detected line segments (optional, generated by solver).
- `intersections`: List of 2D intersection points (optional, generated by solver).

### 2.2 CameraPose
Ground-truth or estimated camera parameters.
- `rotation`: $3 \times 3$ rotation matrix or Rodrigues vector.
- `translation`: $3 \times 1$ translation vector.
- `source`: "ground_truth" or "estimated".
- `confidence`: Float (0.0–1.0) indicating solver confidence.
- `scale_factor`: Float (1.0 by default). Represents the ambiguity in scale for planar targets.

### 2.3 ReconstructedBox
Estimated 3D bounding box dimensions.
- `sequence_id`: Identifier for the sequence.
- `height`: Floor-to-ceiling height (float, relative units unless scaled).
- `width`: Room width (float, relative units unless scaled).
- `depth`: Room depth (float, relative units unless scaled).
- `aspect_ratio`: Width / Depth (float, scale-invariant).
- `error_metrics`: Dictionary containing absolute differences vs. ground truth (relative or absolute depending on context).
- `scale_anchored`: Boolean. True if a canonical height was assumed to resolve scale.

### 2.4 ComplexityMetric
Quantitative measure of camera motion.
- `sequence_id`: Identifier for the sequence.
- `radial_motion_deg`: Total radial motion in degrees.
- `z_velocity_avg`: Average Z-axis velocity.
- `motion_type`: Enum ("orbit", "dolly", "linear", "static").
- `complexity_score`: Normalized score (0.0–1.0).

## 3. Data Flow

1. **Raw Data** (OmniDirector/Synthetic) $\to$ **Ingestion Module** $\to$ **FilteredDataset** (CSV + Images).
2. **FilteredDataset** $\to$ **Solver Module** $\to$ **PoseEstimates** (JSON/CSV).
3. **PoseEstimates** + **GroundTruth** $\to$ **Analysis Module** $\to$ **ReconstructionResults** (CSV).
4. **ReconstructionResults** $\to$ **Validation Module** $\to$ **FinalReport** (JSON).

## 4. Storage Schema

All derived data is stored in `data/processed/`.
- `filtered_sequences.csv`: List of retained sequences and their metadata.
- `poses_estimated.json`: Per-frame pose estimates (including `scale_factor`).
- `reconstruction_results.csv`: Final dimensions, aspect ratios, and error metrics.
- `validation_report.json`: Statistical summaries and aspect ratio validations.

## 5. Constraints

- **Immutability**: Raw data in `data/raw/` is never modified.
- **Checksums**: All files in `data/processed/` are checksummed upon creation.
- **Format**: Numerical data stored in double precision (float64).
- **Scale Ambiguity**: `ReconstructedBox` must always include `scale_anchored` flag and `aspect_ratio` to distinguish between relative and absolute dimensions.
