# Implementation Plan: Investigating Statistical Properties of Simulated Dark Matter Halos

**Branch**: `[001-dark-matter-halo-statistics]` | **Date**: 2025-01-15 | **Spec**: `spec.md`
**Input**: Feature specification for analyzing IllustrisTNG and Millennium simulations.

## Summary

This project implements a statistical analysis pipeline to investigate deviations from NFW framework predictions in dark matter halos. The system attempts to download public catalogs from IllustrisTNG (TNG100-1) and Millennium. **Critical Note**: Real data acquisition is currently blocked due to the unavailability of verified URLs for full particle catalogs. 

Consequently, the project scope is currently **Pipeline Validation & Methodology Stress-Testing**. The implementation falls back to a **Synthetic Data Generator** with **Controlled Deviations** (intentionally offset from NFW predictions) to validate the pipeline's ability to detect statistical anomalies. Final scientific conclusions regarding TNG/Millennium deviations are blocked pending real data availability.

The implementation is constrained to CPU-only execution on free-tier CI (limited CPU cores, constrained RAM, limited disk space). The project utilizes **Mass-Stratified Sampling** for halos and **Memory-Mapped Sparse Particle Streams** for environment calculations to fit within memory limits.

## Complexity Tracking

| Violation | Why Needed | Simpler Alternative Rejected Because |
|-----------|------------|-------------------------------------|
| **Mass-Stratified Sampling** | Full TNG + Millennium catalogs require substantial RAM. | Random sampling without stratification biases the mass function and rare high-mass halos, invalidating KS tests. |
| **Chunked Streaming** | Processing full dataset in memory causes OOM. | Loading all data at once exceeds the available RAM limit on CI.. |
| **Memory-Mapped Particle Stream** | Full particle catalog for overdensity (FR-003) exceeds RAM. | Loading all particles into memory for cKDTree construction exceeds available RAM capacity. |
| **Subsampled Plummer-Softened Potential** | Direct N-body potential summation is O(N²) and too slow. | Simplified uniform sphere approximation (U ≈ -G M²/R) is physically incorrect and invalidates spin measurement validity. |
| **Controlled Deviations** | Synthetic data mimicking NFW perfectly cannot test deviation detection. | Perfectly matching NFW would make the pipeline blind to deviations (tautology). |
| **Multiple Hypothesis Correction** | Multiple tests (3 metrics × 3 bins) require FWER control. | Uncorrected p-values inflate Type I error. |

## Phases

### Phase 0: Data Acquisition Attempt & Gap Handling (FR-001)
1. **Attempt Real Download**: Execute `code/data/download.py` to fetch TNG100-1 and Millennium catalogs via official API.
2. **Gap Detection**: If API fails (403, timeout, or no verified URL), log "Data Gap" and trigger **Synthetic Fallback**.
3. **Synthetic Generation**: If real data is unavailable, generate synthetic catalogs with:
   - **Mass-Stratified Sampling**: Ensure representative mass distribution.
   - **Controlled Deviations**: Inject known offsets to NFW parameters to test pipeline sensitivity.
   - **Sparse Particle Stream**: Generate a particle table that is written to disk in chunks and accessed via memory-mapped arrays (mmap) to satisfy FR-003 without exceeding RAM.
4. **Checksum & Log**: Record checksums of synthetic data and the "Data Gap Report".

### Phase 1: Pre-processing & Metric Computation (FR-002 to FR-006)
1. **Filtering**: Retain halos with ≥300 particles (FR-002).
2. **Chunked Processing**: Process data in chunks of a size constrained by available RAM.
3. **Overdensity (FR-003)**: Compute local overdensity using `cKDTree` on the **memory-mapped particle stream** with periodic boundary wrapping (top-hat). The tree is built on a subsample or incrementally to fit memory.
4. **Shape (FR-004)**: Compute inertia tensor and `s=c/a`.
5. **Spin (FR-005)**: Compute `λ = J|E|^(1/2)/(GM^(5/2))` using **Subsampled Plummer-Softened Potential** (summing interactions over N=500 random particles) to ensure physical validity and CPU feasibility.
6. **Concentration (FR-006)**: Fit NFW profile via `scipy.optimize.curve_fit`. Log convergence status (success/fail). Track **Convergence Rate** (SC-004).

### Phase 2: Statistical Analysis (FR-007 to FR-011)
1. **Binning**: Bin by Mass (3 bins) and Environment (Low/High, Δ < 200 vs ≥ 200).
2. **KS Tests**: Two-sample KS tests for Shape, Spin, Concentration across bins.
3. **Correction**: Apply Benjamini-Hochberg correction for multiple comparisons.
4. **Spearman**: Compute mass-concentration correlation.
5. **Bullock Comparison**: Compare measured relation to Bullock et al. (2001) fit.
6. **Convergence Rate**: Calculate percentage of halos with successful NFW fits (SC-004).

### Phase 3: Visualization & Reporting (FR-012)
1. **Plots**: Generate scatter plots, KDE curves, heatmaps.
2. **Results**: Save `results/statistics.json` with all p-values, effect sizes, and convergence rates.
3. **Data Gap Report**: Document the status of real data acquisition and the nature of synthetic deviations used.

## Compute Feasibility

- **RAM**: < 2 GB via **Chunked Streaming** (k halos/chunk) and **Memory-Mapped Particle Streams** for FR-003.
- **CPU**: 2 Cores sufficient for O(N log N) tree operations (on subsamples) and O(N) subsampled potential summation.
- **Disk**: < 14 GB via streaming writes and minimal intermediate storage.
- **Time**: < 6 hours for full pipeline on synthetic data (real data time depends on download speed).
