# Implementation Plan: llmXive follow-up: extending "Agentic Abstention: Do Agents Know When to Stop Instead of Act?"

**Branch**: `001-llmxive-abstention-meta-critic` | **Date**: 2026-07-11 | **Spec**: `spec.md`
**Input**: Feature specification from `specs/001-llmxive-abstention-meta-critic/spec.md`

## Summary

This feature implements a "Meta-Critic" system to predict optimal agent stopping (abstention) decisions using only low-level state features (e.g., token usage, error counts, turn number) rather than full semantic context. The system ingests the "Agentic Abstention" benchmark (Environment-based subset) or a validated Synthetic Agent Simulator, extracts features including a lightweight semantic proxy (embedding distance), trains a CPU-optimized gradient-boosted tree classifier (XGBoost/LightGBM), and evaluates performance against a locally simulated full-context baseline. The plan ensures all Functional Requirements (FR-001 to FR-007) and Success Criteria (SC-001 to SC-005) are addressed, with strict adherence to CPU-only constraints and statistical rigor, including survival analysis to handle censored data.

## Technical Context

**Language/Version**: Python  
**Primary Dependencies**: `xgboost` (or `lightgbm`), `sentence-transformers` (v2.2, `all-MiniLM-L6-v2`), `pandas`, `scikit-learn`, `pyyaml`, `numpy`, `lifelines` (for survival analysis)  
**Storage**: Local files (Parquet/CSV) in `data/`  
**Testing**: `pytest`  
**Target Platform**: Linux (GitHub Actions free-tier: CPU, ~7 GB RAM)  
**Project Type**: Research/CLI tool  
**Performance Goals**: Complete training and evaluation within 6 hours on 2 CPU cores; memory usage < 7 GB.  
**Constraints**: No GPU/CUDA; no large-LLM inference; no deep-net training; strict separation of state features from full semantic context in the Meta-Critic input.  
**Scale/Scope**: Representative subset of the "Agentic Abstention" benchmark (Environment-based subset) or Synthetic Agent Simulator; training on sampled data to fit CPU constraints.

> Domain-specific empirical specifics (exact counts, dataset sizes, measured quantities) are deferred to the research/implementation phase.

## Constitution Check

*Gates determined based on constitution file*

- **I. Reproducibility**: Plan mandates pinned seeds (a representative subset) in `code/`, canonical dataset sources (verified URLs) or verified synthetic generators, and isolated virtualenvs via `requirements.txt`.
- **II. Verified Accuracy**: **Conditional**. The "Agentic Abstention" benchmark is currently unverified. The plan proceeds with a Synthetic Agent Simulator as the primary execution path, explicitly noting this limitation. If a verified real dataset is found, it will replace the synthetic source.
- **III. Data Hygiene**: Plan requires checksumming of raw data (or synthetic generation logs) in `data/`, immutable derivations (new files for features), and PII scanning.
- **IV. Single Source of Truth**: Metrics (Token Consumption, Latency, Recall) are generated by `code/analysis/statistical_tests.py` (implementing Survival Analysis and cost-weighted utility) and logged to `data/` before inclusion in reports.
- **V. Versioning Discipline**: Content hashes for `data/` and `code/` will be tracked in `state/projects/PROJ-918-llmxive-follow-up-extending-agentic-abst.yaml`.
- **VI. Decoupled Stopping Logic Validation**: **Verified via Schema**. The exclusion of full semantic context is enforced by `contracts/dataset.schema.yaml`, which explicitly forbids text columns in the feature vector.
- **VII. CPU-Optimized Inference Benchmarking**: All training and evaluation steps are designed for 2-core CPU execution; no GPU-specific libraries or quantization are used.
