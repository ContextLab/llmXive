# Quickstart: Qwen‑VLA Cross‑Embodiment Transfer Study

This guide walks a new contributor through a full end‑to‑end run on the GitHub Actions CI runner (or a comparable local Linux environment).

## Prerequisites
- Python 3.11 (recommended via `pyenv` or `conda`).
- Git 2.40+
- Access to the **verified** Open X‑Embodiment subset and LIBERO benchmarks (must be added to `data/` as described in the *Dataset Strategy* section).

## 1. Clone the Repository
```bash
git clone
cd qwen-vla-cross-embodiment
```

## 2. Set Up the Virtual Environment
```bash
python -m venv.venv
source.venv/bin/activate
pip install -r requirements.txt
```

## 3. Verify Dataset Availability
```bash
# This will raise an informative error if the required datasets are missing.
python code/check_datasets.py
```

## 4. Pre‑process the Data
```bash
python code/preprocess_actions.py \
 --input data/open_x_embodiment_raw/ \
 --output data/subset.parquet \
 --ratio 1.0 # use full cross‑embodiment data; change for ablation
```
The script writes `data/metadata.yaml` with the SHA‑256 checksum.

## 5. Train the Cross‑Embodiment Model
```bash
python code/train.py \
 --data data/subset.parquet \
 --ratio 1.0 \
 --seeds 0 1 2 3 4 \
 --output_dir models/
```
- Training respects the 6 h wall‑time limit; a `"TIMEOUT_WARNING"` is logged if exceeded.
- Checkpoints appear as `models/checkpoint_1.0_seed{N}.pt`.

## 6. Train the Single‑Embodiment Baseline
```bash
python code/train.py \
 --data data/subset_single_embodiment.parquet \
 --ratio 0.0 \
 --seeds 0 1 2 3 4 \
 --output_dir models/
```

## 7. Evaluate on LIBERO Benchmarks
```bash
python code/evaluate.py \
 --checkpoints models/ \
 --benchmarks LIBERO-Spatial LIBERO-Object \
 --platforms Franka UR5 \
 --seeds 0 1 2 3 4 \
 --output_dir results/
```
Outputs `results/evaluation_{benchmark}_{seed}.json`.

## 8. Run Statistical Analyses
```bash
python code/statistics.py \
 --eval_dir results/ \
 --output_dir analysis/
```
Generates `analysis/wilcoxon_report.yaml` and `analysis/ablation_report.yaml`.

## 9. Render the Reproducibility Manifest
```bash
python code/render_manifest.py
# Produces manifest.yaml and a human‑readable manifest.md
```

## 10. Inspect the Final Report
Open `analysis/report.md` (generated by `code/generate_report.py`) to see:
- Mean success‑rates with 95 % confidence intervals.
- Wilcoxon p‑value and Holm‑Bonferroni corrected decision.
- Ablation plot (`ablation.png`).

## 11. CI Execution
The repository contains a GitHub Actions workflow (`.github/workflows/ci.yml`) that runs the exact sequence above on the hosted runner. Push a branch to trigger the workflow and monitor the artifacts under the “Actions” tab.
