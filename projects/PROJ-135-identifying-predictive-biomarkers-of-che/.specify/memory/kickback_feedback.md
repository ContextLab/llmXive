## Unresolved concerns

- FR-013 (Spec) mandates a fixed split of data into 'discovery' and 'training' sets. The Plan explicitly removed this split in favor of Nested CV. Consequently, no task exists to implement the fixed split logic required by FR-013. This is a missing task for a spec requirement.
- FR-007 (Spec) requires building 'tumor-type-specific' models. The Plan explicitly rejects this for a 'Pan-Cancer Model'. No task exists to implement the tumor-type-specific models mandated by FR-007. This is a missing task for a spec requirement.
- FR-005 (Spec) requires DESeq2 execution on the 'discovery set'. Since the 'discovery set' (FR-013) is removed in the Plan, no task implements DE on a specific discovery subset. Tasks T022/T023 implement DE on the full data inside CV, leaving the FR-005 requirement unmet.
- FR-014 (Spec) allows 'ComBat-seq OR quantile matching'. The Plan explicitly forbids ComBat-seq. No task implements ComBat-seq as the alternative option required by the spec's 'OR' condition, potentially leaving the spec requirement unfulfilled if Quantile Normalization alone is insufficient for RNA-seq batch effects.
- FR-006 (Spec) requires a fallback to union of top genes if intersection is 0, AND flagging this in 'results/summary.md'. Task T025 implements the fallback logic but lacks an explicit step to 'flag in results/summary.md'. The flagging action is missing from the task list.
- FR-001 (Spec) requires downloading data for '≥3 tumor types'. The Plan's Data Feasibility Gate allows proceeding with '≥2'. No task explicitly enforces the initial download of ≥3 types as a hard requirement before the gate, potentially violating the spec's initial scope.
- Task T027 references '[deferred]' for the gene limit. While the Plan uses this marker, the task description should explicitly state that the specific number is determined by the 'top [deferred] most variable genes' logic to ensure the implementer knows to calculate this dynamically rather than hardcoding a placeholder.
- T022 and T023 (DESeq2 execution) are placed in Phase 4 (US2), but the Plan explicitly states DE must run 'inside the inner loop' of the Nested CV in Phase 5 (US3) to prevent data leakage. Placing DE in US2 creates a consumer-before-producer violation where US3 (Modeling) consumes a pre-computed, static gene list that was not selected within the CV folds, violating the data-flow dependency required for valid nested CV.
- T024-T028 (Meta-analysis and Gene Panel generation) are in Phase 4, but the Plan's 'Pan-Cancer Model' approach (Phase 5) requires the gene panel to be derived from the intersection of DE results *inside* the CV loop or as a pre-step to the pooled model. If the panel is generated in Phase 4 on the full dataset, Phase 5 (T032) cannot perform 'Internal Feature Selection' as required by the Plan's complexity tracking, breaking the dependency chain for the nested CV logic.
- T018 (Quantile Normalization) is in Phase 3 (US1), but the Plan specifies this is for 'cross-platform alignment' of GEO microarray data with TCGA RNA-seq data *before model application*. If US1 only processes TCGA data (as implied by 'subset of 2 cancer types' in the test), the alignment logic is premature. The dependency on the specific mix of GEO/TCGA data required for this normalization step is not established until US2/US3 data acquisition is confirmed, creating a potential ordering violation if US1 runs in isolation without the full dataset context.
- T031, T032, and T033 are listed as sequential tasks in Phase 5, but T032 (Nested CV) and T033 (Pan-Cancer Model) are logically distinct execution paths in the Plan (one is internal validation, one is final training). The current ordering implies T033 depends on T032 completing, but the Plan suggests the Pan-Cancer model uses the *same* gene panel derived earlier. Clarify if T033 is a separate run or if the dependency is strictly on the gene panel artifact, not the CV execution.
- Task T014 ('Implement Data Feasibility Gate... halt if <2 tumor types remain') lacks a concrete deliverable artifact. It describes a control flow action (halt) but does not name the specific output file, log entry format, or error code that confirms the gate was executed and the halt condition met. An implementer cannot deterministically verify 'halt' without a named artifact (e.g., 'Write `data/feasibility_gate_status.json` with `status: 'halted'` and `reason: 'insufficient_types'`').
- Task T025 ('...flag in `results/summary.md`') is ambiguous. The task does not specify the exact format, key, or section within `results/summary.md` where the fallback flag must appear. Without a concrete string or JSON key (e.g., 'Add `fallback_reason: "intersection_empty"` to the `meta_analysis` section of `results/summary.md`'), the implementer cannot deterministically satisfy the verification step.
- Task T034 ('...halt if <2 tumor types remain') suffers from the same issue as T014. It describes a runtime behavior (halt) without defining the concrete artifact (log file, exit code, or status JSON) that proves the halt occurred. The task must name the specific artifact produced upon the halt condition.
- Task T027 ('Limit analysis to top [deferred] most variable genes') is too coarse and relies on a placeholder value. While '[deferred]' is a sanctioned marker for empirical values, the task does not specify *how* the variable count is determined (e.g., 'Calculate variance across all samples, sort descending, and select the top N genes where N is defined in `config.py` as `MAX_VARIANCE_GENES`'). The task needs to name the configuration key or logic that resolves the '[deferred]' value to be executable.
- Task T040 ('Enforce runtime timeout... and memory limit') describes a mechanism but lacks a concrete deliverable for verification. It does not specify the artifact that records the enforcement (e.g., 'Write `results/runtime_metrics.json` with `timeout_triggered: boolean` and `peak_memory_mb: float`'). Without this, the task cannot be verified as 'done' other than by observing a crash, which is not a deterministic deliverable.
- Tasks T023 and T032 implement 'Nested Cross-Validation with Internal Feature Selection' on the full dataset, explicitly removing the fixed 'discovery set' split mandated by FR-013. FR-013 requires a specific data split (discovery for gene selection, training for model fitting) to prevent circular validation. The task description 'execute DE inside the inner loop' replaces the spec's required architecture without a documented staging exception, constituting a semantic weakening of the requirement.
- Task T031 and T033 describe building a 'Pan-Cancer Model' on 'pooled data'. FR-007 explicitly mandates building 'elastic-net logistic regression models... within each cancer type'. The task replaces the per-tumor-type model architecture with a single pooled model, violating the spec's structural requirement for validating cross-tumor generalizability via leave-one-cancer-type-out on specific type models.
- Task T018 implements 'Quantile Normalization' to align GEO microarray data with TCGA RNA-seq. FR-014 mandates 'ComBat-seq OR quantile matching'. The task description implies a hard constraint to use *only* Quantile Normalization, potentially violating the spec's allowance for ComBat-seq (which is valid for RNA-seq batch effects). The task does not reflect the 'OR' condition or the conditional logic required if ComBat-seq is preferred for specific batch scenarios.
- Spec US2 Acceptance Scenario 2 requires that if the intersection yields 0 genes, the system MUST 'flag this in results/summary.md'. Tasks T024 and T025 implement the intersection/union logic but lack an explicit verification step or task description confirming the 'flag in results/summary.md' action is triggered. This drops a specific acceptance criterion from the task verification scope.
- Task T038 states: 'Handle class imbalance... if responder ratio <20%'. The Spec's Edge Cases mandate: 'The system MUST use stratified k-fold cross-validation AND, if the responder ratio is <20%, apply cost-sensitive learning'. The task description ambiguously links stratified k-fold to the <20% condition, potentially omitting the mandatory stratified k-fold for cases where the ratio is >20% but stratification is still required for valid CV.
