# Implementation Plan: Identifying Predictive Biomarkers of Chemotherapy Response in Public Cancer Datasets

**Branch**: `001-chemo-biomarker-discovery` | **Date**: 2024-01-15 | **Spec**: `specs/001-chemo-biomarker-discovery/spec.md`

## Summary

This feature implements a computational pipeline to identify gene-expression signatures predictive of chemotherapy response across multiple tumor types. The approach involves: (1) acquiring TCGA RNA-seq and GEO microarray data with a **Data Feasibility Gate** to verify response labels, (2) harmonizing identifiers and normalizing via **Quantile Normalization** for cross-platform alignment (avoiding ComBat-seq on microarrays), (3) performing **Nested Cross-Validation with Internal Feature Selection** to identify cross-tumor biomarkers without data leakage, (4) meta-analyzing p-values using Stouffer's method for gene ranking, and (5) training a single **Pan-Cancer Elastic-Net Logistic Regression** model on pooled data. The pipeline is designed to run entirely on CPU within GitHub Actions free-tier constraints (≤6h, ≤7GB RAM) by limiting analysis to the top [deferred] most variable genes.

## Technical Context

**Language/Version**: Python 3.11  
**Primary Dependencies**: `pandas`, `numpy`, `scikit-learn`, `statsmodels`, `rpy2` (for DESeq2), `biopython`, `requests`, `scipy`  
**Storage**: Local filesystem (`data/raw/`, `data/processed/`, `results/`)  
**Testing**: `pytest` with contract tests against YAML schemas  
**Target Platform**: Linux (GitHub Actions free-tier runner)  
**Project Type**: Data analysis pipeline / Computational biology  
**Performance Goals**: ≤6 hours runtime, ≤7 GB RAM usage  
**Constraints**: No GPU, no CUDA, no heavy LLM inference, no deep learning training from scratch.  
**Scale/Scope**: ≥2 tumor types with response labels, ≤10,000 most variable genes, ≤50 genes in final panel.

> Note: DESeq2 is accessed via `rpy2` for RNA-seq DE. For microarray data, we use standard log2 transformation followed by Quantile Normalization. All R packages are installed in a conda environment compatible with CPU-only execution.

## Data Feasibility Gate

*Critical Pre-Condition for Analysis*

1.  **TCGA Check**: Upon download, the system MUST verify that clinical metadata for each tumor type contains explicit chemotherapy response annotations (e.g., RECIST, CR/PR). If a tumor type lacks these labels, it is excluded from analysis.
2.  **GEO Check**: The system MUST verify that selected GEO datasets contain response labels. If the preferred datasets (GSE, GSE42752) are unavailable in verified mirrors, the system MUST dynamically select the next available verified GEO dataset with response labels.
3.  **Minimum Requirement**: The pipeline MUST halt with an error if fewer than 2 tumor types (with response labels) remain after filtering.

## Complexity Tracking

| Violation | Why Needed | Simpler Alternative Rejected Because |
| :--- | :--- | :--- |
| **R/RPy2 Integration** | DESeq2 is the gold standard for RNA-seq DE and not natively available in Python. | Pure Python alternatives (e.g., `edgeR` in `rpy2` or custom GLMs) lack the specific VST and Wald test implementations required by FR-005 and SC-007. |
| **Nested Cross-Validation with Internal Feature Selection** | Required to avoid data leakage. Feature selection (DE) must occur *inside* the inner loop. | A fixed Discovery/Training split creates circular validation where the model is trained on data that influenced feature selection. |
| **Pan-Cancer Model** | To test the "Cross-Tumor" hypothesis, a single model must be trained on pooled data using the meta-analyzed panel. | Separate tumor-specific models do not test the generalizability of a *single* biomarker panel across tumors. |
| **Quantile Normalization** | Required for cross-platform alignment (RNA-seq vs Microarray). | ComBat-seq is invalid for microarray data due to distributional mismatch (negative binomial vs continuous). |

## Runtime & Resource Mitigation

- **Gene Limit**: Analysis restricted to the top subset of most variable genes across all datasets.
- **Sequential Processing**: Tumor types processed sequentially to save RAM.
- **Timeout Watchdog**: `utils.py` implements a watchdog that halts execution if runtime exceeds a predefined threshold.
- **Memory Profiling**: `psutil` used to monitor RAM usage; process killed if >6.5 GB.
