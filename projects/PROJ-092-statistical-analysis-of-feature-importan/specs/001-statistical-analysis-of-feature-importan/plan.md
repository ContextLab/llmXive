# Implementation Plan: Statistical Analysis of Feature Importance Drift in Pre-trained Models

**Branch**: `001-feature-importance-drift` | **Date**: 2024-05-21 | **Spec**: `specs/001-feature-importance-drift/spec.md`

## Summary

This project implements a statistical pipeline to quantify and test the significance of feature importance drift in Random Forest models over time. Using the UCI Electricity Load Diagrams dataset, the system will split data into sequential 30-day windows, train baseline models, compute permutation importance, and analyze the drift in feature rankings using Spearman rank correlation. 

**Statistical Methodology Update**: The analysis employs a two-stage test: (1) A **Magnitude Test** comparing observed mean correlation against a null distribution generated by permuting feature importance vectors across windows to test if drift > 0. (2) A **Trend Test** using block permutation of the derived rho sequence to test for monotonic trends. 

All computations are optimized to fit within CPU-only CI constraints (2 cores, **4 GB RAM**, no GPU).

## Complexity Tracking

| Violation | Why Needed | Simpler Alternative Rejected Because |
| :--- | :--- | :--- |
| **Block Permutation Test (FR-008)** | Required due to small sample size (n=5-6 windows) where asymptotic tests fail. | Asymptotic Mann-Kendall p-values are unreliable for n < 10; simple t-tests ignore temporal autocorrelation. |
| **Windowed Training (FR-002)** | Essential to capture temporal drift; a single global model cannot detect drift. | Global model would average out local shifts, failing the core research question. |
| **R² Threshold Skip (FR-003b)** | Prevents garbage importance scores from failing models from contaminating drift metrics. | Including invalid models would introduce noise indistinguishable from drift. |
| **Two-Stage Statistical Test** | Required to distinguish between "magnitude of drift" and "trend of drift" (Methodology Concern). | A single test conflates stability and trend, leading to invalid conclusions. |
