# Data Model Specification

This document describes the core data entities used throughout the `llmXive` pipeline for investigating the impact of network structure on neural avalanche dynamics.

## Overview

The pipeline processes diffusion MRI (dMRI) structural connectomes and simulated EEG time-series. The data model is designed to support the flow from raw data acquisition through preprocessing, simulation, analysis, and reporting.

## Core Entities

### 1. Participant

Represents a single subject in the study.

| Attribute | Type | Description |
|:--- |:--- |:--- |
| `subject_id` | `str` | Unique identifier (e.g., "sub-001") |
| `age` | `Optional[int]` | Age of the participant (if available) |
| `sex` | `Optional[str]` | Sex of the participant (if available) |
| `qc_passed` | `bool` | Whether the participant passed quality control checks |
| `qc_reasons` | `List[str]` | List of reasons for QC failure if applicable |

**Usage**:
- Instantiated in `code/data/models.py`.
- Used as the primary key for storing and retrieving data in `code/data/store.py`.
- Referenced in `code/analysis/metrics.py` and `code/analysis/avalanches.py` to load subject-specific data.

### 2. StructuralConnectome

Represents the structural brain network derived from dMRI tractography.

| Attribute | Type | Description |
|:--- |:--- |:--- |
| `subject_id` | `str` | Reference to the owning Participant |
| `adjacency_matrix` | `np.ndarray` | Node-by-node weighted adjacency matrix (N x N) |
| `node_labels` | `List[str]` | List of region-of-interest (ROI) labels (e.g., HCP-MMP1.0) |
| `parcellation` | `str` | Name of the parcellation scheme used (e.g., "HCP-MMP1.0") |
| `file_path` | `Path` | Path to the stored matrix file (`.npy` or `.csv`) |

**Usage**:
- Generated by `code/data/preprocess_dMRI.py` using `tck2connectome` logic.
- Loaded by `code/analysis/metrics.py` to compute degree, clustering, and rich-club coefficients.
- Used by `code/data/simulate_EEG.py` as the connectivity kernel for the Wilson-Cowan equations.

### 3. AvalancheRecord

Represents a detected neural avalanche event from the EEG time-series.

| Attribute | Type | Description |
|:--- |:--- |:--- |
| `subject_id` | `str` | Reference to the owning Participant |
| `start_time` | `float` | Start time of the avalanche (seconds) |
| `end_time` | `float` | End time of the avalanche (seconds) |
| `size` | `int` | Total number of active time bins (or summed amplitude) |
| `duration` | `float` | Duration of the avalanche (seconds) |
| `peak_amplitude` | `float` | Maximum amplitude observed during the event |

**Usage**:
- Generated by `code/analysis/avalanches.py` via `detect_avalanches`.
- Aggregated in `code/analysis/fitting.py` to fit power-law distributions.
- Exported in `code/analysis/export_metrics.py` for statistical analysis.

## Data Flow Diagram

1. **Input**: Raw dMRI data (ds003813) -> `code/data/download.py`
2. **Preprocessing**: Tractography -> `code/data/preprocess_dMRI.py` -> `StructuralConnectome`
3. **Simulation**: `StructuralConnectome` + Wilson-Cowan params -> `code/data/simulate_EEG.py` -> `EEG Time-Series`
4. **QC**: `StructuralConnectome` + `EEG` -> `code/data/quality_control.py` -> `Participant` (QC status)
5. **Storage**: `StructuralConnectome` + `EEG` -> `code/data/store.py`
6. **Analysis**:
 - `StructuralConnectome` -> `code/analysis/metrics.py` -> Network Metrics
 - `EEG` -> `code/analysis/avalanches.py` -> `AvalancheRecord` list
7. **Fitting**: `AvalancheRecord` -> `code/analysis/fitting.py` -> Power-law parameters
8. **Stats**: Network Metrics + Avalanche Exponents -> `code/analysis/stats.py` -> Correlation results
9. **Output**: `data/results/correlation_report.csv`

## File Paths and Conventions

- **Raw Data**: `data/raw/<subject_id>/`
- **Processed Data**: `data/processed/<subject_id>/`
 - Connectome: `connectome_matrix.npy`
 - EEG: `eeg_simulated.npy`
- **Results**: `data/results/`
 - Metrics: `metrics_export.csv`
 - Correlations: `correlation_report.csv`
- **Figures**: `figures/`

## API Integration Notes

- All data loading functions in `code/data/store.py` return typed objects or DataFrames compatible with the analysis modules.
- The `Participant` dataclass in `code/data/models.py` is the central anchor for linking structural and functional data.
- Error handling is centralized in `code/utils/logger.py`; data loading failures raise `DataLoadError`.
