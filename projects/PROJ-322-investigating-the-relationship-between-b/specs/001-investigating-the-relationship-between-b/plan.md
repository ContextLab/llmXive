# Implementation Plan: Investigating the Relationship Between Brain Network Reconfiguration and Recovery from Mild Traumatic Brain Injury

**Branch**: `001-brain-network-reconfiguration` | **Date**: 2023-10-27 | **Spec**: `specs/001-brain-network-reconfiguration/spec.md`
**Input**: Feature specification from `/specs/001-investigating-the-relationship-between-b/spec.md`

## Summary

This project implements a computational pipeline to investigate the relationship between brain network reconfiguration (measured by global efficiency, local efficiency, and modularity) and recovery from mild traumatic brain injury (mTBI). 

**CRITICAL STATUS**: The primary scientific hypothesis (correlation with cognitive recovery scores) is **BLOCKED** due to the lack of a verified dataset containing both longitudinal mTBI fMRI data and clinical cognitive scores (e.g., Post-Concussion Symptom Scale). The provided "Verified datasets" block contains no valid source for the dependent variable.

Consequently, the project will execute in **Methodology Validation Mode** only. This mode uses synthetic data to verify the correctness of the data ingestion, preprocessing, graph metric calculation, and statistical modeling code. No scientific results (correlation coefficients, p-values) will be reported as empirical findings. The pipeline will explicitly flag all outputs as "Synthetic Validation" and halt scientific analysis until a verified source for cognitive scores is identified.

## Phase Breakdown & Requirement Mapping

### Phase 0: Data Acquisition & Feasibility Check
- **Goal**: Download data, verify variables (cognitive scores, time points), and check sample size.
- **Mapped Requirements**: FR-001, FR-009, Assumption (Data Availability).
- **Logic**: 
  - **BLOCK CONDITION**: If the dataset lacks cognitive scores (PCS), the pipeline **HALTS** scientific analysis. 
  - **CONTINGENCY**: Switch to **Methodology Validation Mode** using synthetic data. 
  - If `n < 20` (in synthetic or real data), trigger contingency (FR-009) to switch to bootstrapping or report pilot status.
  - If variables missing, log "Data Gap" in `data/results/gaps.json` and halt scientific output.

### Phase 1: Preprocessing & Graph Construction
- **Goal**: Apply minimal confound regression, parcellate with AAL, compute connectivity matrices, apply sparsity threshold.
- **Mapped Requirements**: FR-001, FR-002, FR-008.
- **Logic**: Handle missing time points (Edge Case, as defined in the spec's Edge Cases section) by excluding from longitudinal model but logging. Handle AAL failure (Edge Case, as defined in the spec's Edge Cases section) by skipping subject.

### Phase 2: Statistical Modeling
- **Goal**: Fit LMM (Global/Local Efficiency, Modularity ~ Cognitive Score + Time + (1|Subject)).
- **Mapped Requirements**: FR-003, FR-006, SC-001.
- **Logic**: 
  - **Collinearity Check**: Calculate VIF for predictors. If VIF > 5, perform PCA on graph metrics and use PC1 as the predictor instead of individual metrics (to avoid tautological fitting).
  - If VIF > 5 and PCA is not applicable, report joint relationship descriptively (FR-006).
  - Handle non-convergence (Edge Case) by skipping subject.
  - **Conditional Execution**: If in "Methodology Validation Mode", the model runs on synthetic data and outputs are flagged as "Synthetic Validation".

### Phase 3: Robustness & Validation
- **Goal**: Run 1,000 permutations, sweep thresholds.
- **Mapped Requirements**: FR-004, FR-005, SC-002, SC-003, Constitution Principle VI.
- **Logic**: 
  - Ensure runtime warning at a predefined early threshold, hard stop at a predefined maximum duration. (FR-003, SC-004).
  - **Validation Note**: Permutation p-values from synthetic data are artifacts and will not be reported as scientific results.

### Phase 4: Reporting
- **Goal**: Generate final JSON/CSV with all metrics, p-values, and limitations.
- **Mapped Requirements**: FR-007 (Independent metric check), SC-005.
- **Logic**: 
  - **FR-007 Compliance**: If an independent metric (e.g., Return-to-Work) is missing, output a `validation_gap` flag and skip the secondary validation step.
  - Explicitly state if independent validation metric is missing.
  - If in "Methodology Validation Mode", the output must include a `mode: "synthetic_validation"` flag and a disclaimer that no scientific hypothesis was tested.

## Methodology Validation Mode (Contingency)

If the primary dataset lacks cognitive scores (which is the current state):
1.  **Generate Synthetic Data**: Create synthetic fMRI time series and cognitive scores with known correlations (seeded) for testing the pipeline.
2.  **Run Pipeline**: Execute all phases on synthetic data.
3.  **Output**: Generate results with a `is_synthetic: true` flag.
4.  **No Scientific Claim**: The output will explicitly state: "This analysis validates the pipeline code. No scientific hypothesis regarding mTBI recovery was tested due to missing data."
