# Assessing Dataset Imbalance Effects on Materials Property Predictions  

**Field**: materials science  

## Research question  

How does the degree of compositional and target‑property imbalance in publicly available materials databases influence the predictive accuracy and feature‑importance attribution of supervised regression models for material‑property prediction?  

## Motivation  

Public repositories such as the Materials Project, OQMD, and AFLOW contain orders of magnitude more data for some chemistries and property ranges than for others. If models trained on these skewed collections systematically under‑perform on under‑represented regions, downstream materials‑design decisions could be biased. Quantifying this effect will reveal whether simple re‑sampling or weighting schemes are needed before deploying data‑driven discovery pipelines.  

## Related work  

- [Orbital Graph Convolutional Neural Network for Material Property Prediction (2020)](https://arxiv.org/abs/2008.06415) — Demonstrates state‑of‑the‑art graph‑based models for predicting formation energy, band gap, etc., providing a high‑performance baseline against which imbalance effects can be compared.  
- [Queue-based Resampling for Online Class Imbalance Learning (2018)](https://arxiv.org/abs/1809.10388) — Introduces a resampling strategy for handling severe class imbalance in streaming settings; the technique can be adapted to create balanced training subsets for regression on materials data.  
- [A New Strategy of Cost‑Free Learning in the Class Imbalance Problem (2013)](https://arxiv.org/abs/1307.5730) — Provides a theoretical framework for learning without explicit cost matrices under imbalance, informing the design of weighting schemes in our experiments.  

## Expected results  

We expect to observe a measurable degradation in MAE/R² for under‑represented chemistries when models are trained on the original skewed datasets, and a corresponding improvement after balanced resampling or class‑weighting. SHAP‑based feature‑importance patterns are also anticipated to shift, indicating that imbalance can distort the inferred physical drivers. Statistical significance will be assessed across multiple random splits (e.g., paired t‑test, α = 0.05).  

## Methodology sketch  

- **Data acquisition**: Download the latest releases of Materials Project, OQMD, and AFLOW (≤ 5 GB total) via their public REST APIs or Zenodo mirrors.  
- **Pre‑processing**:  
  1. Compute compositional descriptors (e.g., element fractions, Magpie features).  
  2. Select target properties (formation energy, band gap, bulk modulus).  
- **Imbalance quantification**:  
  3. Bin the target property and/or key compositional descriptors; calculate the frequency distribution to obtain an “imbalance score” (e.g., inverse of bin counts).  
- **Dataset construction**:  
  4. Create two training sets per property:  
     - *Original*: retains the native skewed distribution.  
     - *Balanced*: apply stratified undersampling/oversampling (or the queue‑based resampling method) to achieve approximately uniform bin counts.  
  5. Reserve a stratified test set that preserves the original imbalance, ensuring that under‑represented bins are represented.  
- **Model training**:  
  6. Train Random Forest and Gradient Boosting regressors (scikit‑learn) on each training set using identical hyper‑parameters.  
- **Evaluation**:  
  7. Compute MAE, RMSE, and R² on the test set for each model.  
  8. Perform paired statistical tests (paired t‑test or Wilcoxon signed‑rank) across 10 different random seeds to assess significance of performance differences.  
- **Interpretability analysis**:  
  9. Generate SHAP values for each trained model; compare the top‑10 feature importance rankings between original‑skewed and balanced models to detect distortion due to imbalance.  
- **Guideline derivation**:  
  10. Summarize findings into practical recommendations (e.g., when to apply weighting vs. resampling) for future materials‑property ML projects.
