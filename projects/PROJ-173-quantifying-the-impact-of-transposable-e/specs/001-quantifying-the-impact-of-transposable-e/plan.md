# Implementation Plan: Quantifying the Impact of Transposable Element Activity on Gene Expression Variation in Drosophila

**Branch**: `001-gene-regulation` | **Date**: 2024-01-15 | **Spec**: `specs/001-quantifying-the-impact-of-transposable-e/spec.md`
**Input**: Feature specification from `/specs/001-quantifying-the-impact-of-transposable-e/spec.md`

## Summary

This project implements a statistical pipeline to quantify the association between Transposable Element (TE) presence and gene expression variation in *Drosophila* (DGRP lines). As no verified public dataset exists containing matched TE-genotype and RNA-seq data for the same lines, the system generates a **Mock/Synthetic Dataset** (FR-001) to validate the pipeline logic. The core analysis employs linear models (`gene_expression ~ TE_presence + PC1 + PC2 + PC3`) to control for population structure, applies Benjamini-Hochberg correction for multiple testing, and performs Freedman-Lane permutation testing (1000 iterations, with dynamic fallback) to establish a null distribution. The pipeline is designed to run entirely on CPU-only CI (GitHub Actions free tier) with strict memory and time constraints.

## Technical Context

**Language/Version**: Python 3.10+  
**Primary Dependencies**: `pandas`, `numpy`, `scipy`, `statsmodels`, `scikit-learn`, `pyyaml`, `pytest`, `shapely` (for coordinate logic)  
**Storage**: In-memory DataFrames (sampled to fit available RAM), CSV/Parquet for intermediate outputs.  
**Testing**: `pytest` with contract validation against `contracts/*.schema.yaml`.  
**Target Platform**: Linux (GitHub Actions free-tier runner: CPU, standard RAM allocation, no GPU.).  
**Project Type**: Computational biology CLI / analysis pipeline.  
**Performance Goals**: Complete 1000 permutation iterations on ~50-100 lines and ~ TE-gene pairs within 6 hours (with dynamic fallback to a variable number of iterations if time permits).  
**Constraints**: No GPU; no deep learning; data must be sampled or synthetic to fit RAM; strict adherence to spec-defined thresholds (VIF > 5, FDR < 0.05).  
**Scale/Scope**: A moderate number of synthetic lines, A large number of TE-gene pairs, A sufficient number of permutations will be employed to ensure the robustness of the statistical inference. (or dynamic fallback).

## Complexity Tracking

| Violation | Why Needed | Simpler Alternative Rejected Because |
| :--- | :--- | :--- |
| **Freedman-Lane Permutation** | Required by FR-006 to preserve LD structure between TE and PCs while shuffling residuals. | Simple label shuffling would break the correlation structure between TEs and population PCs, leading to invalid null distributions. |
| **VIF Diagnostics** | Required by FR-007 to flag collinearity. | Standard linear regression output does not automatically flag collinearity; explicit VIF calculation is needed. |
| **Mock Data Generation** | Required by FR-001 as no verified public dataset exists. | Using a real dataset not containing both TE and RNA-seq for the same lines would violate the "Verified Accuracy" and "Data Hygiene" principles. |
| **Dynamic Iteration Fallback** | Required to prevent CI timeout on 2-core runners. | Fixed 1000 iterations may exceed 6 hours if per-fit time is >10ms; fallback ensures pipeline completion. |
