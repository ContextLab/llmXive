# Research: Quantifying the Impact of Transposable Element Activity on Gene Expression Variation in Drosophila

## Dataset Strategy

The core scientific question requires a DGRP genotype VCF with TE insertion calls **and** TE‑aware RNA‑seq quantifications for the same lines. As of the current date, **no publicly verified dataset meeting both criteria exists** (FlyBase FTP and SRA/ modENCODE provide either genotype *or* expression, but not a combined TE‑aware matrix). Consequently:

| Data Type | Description | Source / Loader | Availability |
|---|---|---|---|
| **DGRP Genotypes (TE calls)** | VCF with SNPs and TE presence/absence per line. | `ftp://ftp.flybase.net/genomes/Drosophila/DGRP/` (unverified for TE calls). | **Unavailable for public CI** |
| **RNA‑seq TE‑aware Expression** | Gene + TE TPM matrix generated by TEtranscripts (or equivalent). | User‑provided FASTQ → `TEtranscripts` (or mock matrix). | **Unavailable for public CI** |
| **Gene Models (Release 6)** | GTF with gene coordinates. | FlyBase FTP (standard GTF). | **Unavailable for public CI** |
| **Independent Replication Dataset** | ModENCODE or other tissue‑specific DGRP expression. | Verified URLs listed in the "Verified datasets" block (none currently include TE‑aware quantification). | **Unavailable** |

**Implication:** The pipeline can be fully exercised on **synthetic/mock data** that conforms to `dataset.schema.yaml`. When a real, verified dataset becomes available, users can point `config.yaml` to the actual files and the same workflow will operate without code changes. The research question cannot be conclusively answered on public data at present; the current scope is a methodological demonstration.

## Statistical Methodology

### 1. Population Structure Control
* **PCA** on genome‑wide SNPs (LD‑pruned) using `sklearn.decomposition.PCA`.  
* **Covariates:** PC1, PC2, PC3 (fixed per Constitution Principle VI).  
* **R² Metric:** `results/population_structure_control.csv` reports R² with PCs, without PCs, and the reduction (SC‑004).

### 2. TE‑Gene Pair Definition
* **Proximity:** TE insertion ≤5 kb upstream/downstream of gene TSS/TES (FR‑002).  
* **Ambiguity:** If a TE falls within the window of multiple genes, `ambiguous_flag = True` is set; the pair is retained and reported (addressing Edge Case).  
* **Monomorphic Filter:** Exclude TEs with frequency < 5 % or > 95 % (FR‑008).  
* **Power Filter:** Compute Minor Allele Count (MAC). Enforce **minimum MAF = 0.10** (≥5 carriers in a 50‑line sample). Pairs failing this are flagged (`power_flag`) and omitted from the primary OLS analysis; rare‑variant signals can be explored via the optional burden test (Task T019). This satisfies the reviewer‑requested power analysis (methodology‑009f4fdf).
* **Selection Bias Note:** The reported 'significant' associations apply only to 'polymorphic TEs with intermediate frequency' due to the power constraint. The optional burden test (Task T019) addresses rare variants.

### 3. Association Model
* **Model:** `gene_expression ~ TE_presence + PC1 + PC2 + PC3` (OLS via `statsmodels`).  
* **Outputs:** Effect size (log₂‑fold change), 95 % CI, unadjusted p‑value, adjusted p‑value (BH).  
* **Collinearity:** VIF calculated; `vif_flag = True` if VIF > 5 (FR‑007).  
* **Mixed-Model Consideration:** OLS with PCs is the primary approach due to CI constraints. For full LD‑aware inference, a mixed‑model (e.g., EMMAX/REML) would be preferable; this is noted as future work when computational resources allow (addresses scientific_soundness‑0328bbb4).

### 4. Multiple‑Testing Correction
* **Method:** Benjamini–Hochberg applied **across all tested TE‑gene pairs**, including ambiguous ones. BH is robust to the positive dependence expected from spatially correlated TE insertions, so a hierarchical procedure is unnecessary (addresses methodology‑3addd2de).

### 5. Robustness Validation – Permutation Testing
* **Algorithm:** **Freedman‑Lane residual permutation** (preserves covariate structure).  
    1. Fit reduced model `Y ~ PC1 + PC2 + PC3`.  
    2. Extract residuals, shuffle them, add back to fitted values → permuted response `Y_perm`.  
    3. Re‑fit full model `Y_perm ~ TE_presence + PC1 + PC2 + PC3`.  
    4. Record the TE coefficient t‑statistic.  
* **Iterations:** ≥100 permutations (configurable).  
* **Null Plot:** `results/plots/null_distribution.png` shows the histogram of null t‑statistics, a vertical line for the observed statistic, and a highlighted 95th‑percentile line (SC‑005).  
* **Validity:** This method correctly simulates the null hypothesis of 'no TE effect' while preserving the correlation between TE and PCs, satisfying SC‑005.

### 6. Replication Analysis
* Run the same linear model on an independent expression dataset (if provided).  
* Generate a comparison table (`results/replication/comparison.tsv`) with original effect size, replication effect size, direction concordance flag, and replication p‑value (FR‑010).  
* Missing lines are excluded per test (FR‑009).  

### 7. Rare‑Variant Sensitivity (Scientific Soundness)
* Rare TE insertions (MAF < 10�%) are excluded from the primary OLS analysis.  
* An optional **burden test** aggregates rare TE insertions per gene and tests the aggregate effect (Task T019). This provides a principled way to assess rare‑variant impact (addresses methodology‑c0a8a510 and scientific_soundness‑fdbadfaf).

### 8. TE‑Aware Quantification (Scientific Soundness)
* The pipeline mandates that pre‑quantified inputs include a `quantification_method` metadata field set to `"TEaware"`. If this flag is missing, the pipeline rejects the input to prevent the circular validation risk of testing gene counts against TE presence using data that excluded TE reads.

## Computational Feasibility

* **Environment:** GitHub Actions free tier (2 CPU, ~7 GB RAM, ≤6 h).  
* **Data Size:** Mock dataset ≤ 2 GB; real data expected to be comparable after sub‑sampling.  
* **Runtime:** Estimated ≤ 5 h for full pipeline on mock data (including 100 permutations).  
* **Memory:** Chunked loading; peak < 4 GB.

## Decision Log

| Decision | Rationale |
|---|---|
| **Use Freedman‑Lane Permutation** | Preserves LD between TE and PCs; satisfies SC‑005. |
| **Power‑Based MAF Threshold** | Guarantees ≥5 carriers in 50 lines, mitigating under‑power for rare TEs (methodology‑009f4fdf). |
| **Ambiguous Pair Handling** | Flags (`ambiguous_flag`) rather than discarding, preserving transparency (Edge Case). |
| **Mixed‑Model Not Implemented** | CI resource limits; OLS with PCs is sufficient for demonstration; mixed‑model noted for future work. |
| **Real Data Unavailable** | Explicitly acknowledge that the core question cannot be answered without a verified dataset; pipeline remains useful for methodological validation. |
| **Rare‑Variant Burden Test** | Provides a principled alternative to simple exclusion, addressing scientific soundness concerns. |
| **TE‑Aware Input Validation** | Mandates `quantification_method == "TEaware"` metadata to ensure scientific validity of the input data. |
