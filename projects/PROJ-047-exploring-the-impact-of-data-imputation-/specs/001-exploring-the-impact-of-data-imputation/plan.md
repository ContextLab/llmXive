# Implementation Plan: Exploring the Impact of Data Imputation Methods on Causal Inference

**Branch**: `001-data-imputation-mnar-impact` | **Date**: 2026-07-10 | **Spec**: `specs/001-data-imputation-mnar-impact/spec.md`
**Input**: Feature specification from `specs/001-data-imputation-mnar-impact/spec.md`

## Summary

This project implements a simulation study to quantify the bias introduced by standard imputation methods (Mean, KNN, MICE) when applied to data with Missing Not At Random (MNAR) mechanisms in causal inference settings. The system generates synthetic Structural Causal Models (SCMs) with known ground-truth Average Treatment Effects (ATE) and injects missingness dependent on unobserved outcomes. It then applies imputation, estimates causal effects via IPW and PSM, and performs rigorous statistical analysis (ANOVA/Friedman, Bootstrap, Sensitivity Analysis) to identify failure thresholds. The entire pipeline is constrained to CPU-only execution on GitHub Actions free-tier runners.

## Complexity Tracking

| Violation | Why Needed | Simpler Alternative Rejected Because |
|-----------|------------|-------------------------------------|
| **Dual Estimator Strategy (IPW + PSM)** | Required by Constitution Principle VII to distinguish imputation error from causal identification error. | Using only one estimator (e.g., OLS) would conflate the failure modes of imputation with the specific biases of the causal method, failing the robustness verification requirement. |
| **Statistical Decision Tree (Shapiro-Wilk -> ANOVA/Friedman -> Bootstrap)** | Required by FR-006 to ensure valid inference under varying distribution shapes of bias. | A fixed test (e.g., always ANOVA) would yield invalid p-values if bias distributions are non-normal, violating statistical rigor. |
| **Sensitivity Analysis (Beta Sweep)** | Required by FR-007/SC-005 to identify failure thresholds. | A single-point analysis (e.g., $\beta=0.5$) would not reveal the monotonic trend or the specific point of method failure. |
