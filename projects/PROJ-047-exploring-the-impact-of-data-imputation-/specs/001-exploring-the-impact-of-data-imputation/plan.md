# Implementation Plan: Exploring the Impact of Data Imputation Methods on Causal Inference

**Branch**: `001-explore-imputation-impact` | **Date**: 2026-07-10 | **Spec**: `specs/001-exploring-the-impact-of-data-imputation/spec.md`
**Input**: Feature specification from `/specs/001-exploring-the-impact-of-data-imputation/spec.md`

## Summary

This project implements a reproducible simulation pipeline to quantify the bias introduced by standard imputation methods (Mean, KNN, MICE) when estimating causal effects (ATE) under Missing Not At Random (MNAR) mechanisms. The core technical approach involves generating synthetic structural causal models (SCM) with known ground-truth ATEs, injecting MNAR missingness dependent on the outcome variable, applying imputation, and estimating the ATE using Inverse Probability Weighting (IPW) and Propensity Score Matching (PSM). 

Crucially, the study distinguishes between **Total Deviation** (error from ground truth) and **Imputation Bias** (error relative to an Oracle baseline). It employs a **Repeated-Measures ART ANOVA** (or Linear Mixed-Effects Model) to account for replication-level variance and includes a **Confidence Interval Coverage** analysis. The sensitivity analysis now controls for missingness rates by solving for the intercept $\alpha$ given a target missingness proportion.

## Technical Context

**Language/Version**: Python 3.11  
**Primary Dependencies**: `numpy`, `pandas`, `scikit-learn`, `statsmodels`, `fancyimpute` (or `sklearn.impute` fallback for CPU), `art-test` (for ART ANOVA), `matplotlib`, `seaborn`, `linearmodels` (for Mixed Effects)  
**Storage**: Local files (CSV/Parquet) under `data/`; no external database required.  
**Testing**: `pytest` (unit tests for data generation logic, integration tests for full pipeline runs on small seeds).  
**Target Platform**: Linux (GitHub Actions free-tier runner: CPU, 7GB RAM).  
**Project Type**: Research simulation library / CLI tool.  
**Performance Goals**: Complete 200 replications (N=1000) within 6 hours on CPU-only runner.  
**Constraints**: No GPU usage; all libraries must have CPU wheels; memory usage < 7GB; no external API calls for data (synthetic generation only).  
**Scale/Scope**: A series of replications across multiple imputation methods, multiple estimators, and multiple MNAR strengths.

## Complexity Tracking

| Violation | Why Needed | Simpler Alternative Rejected Because |
|-----------|------------|-------------------------------------|
| **Repeated-Measures ART ANOVA** | Required by Spec (FR-006) and Constitution (Principle VII) to handle non-normality/heteroscedasticity and the repeated-measures structure (200 replications). Standard ANOVA assumes independence, which is violated here. | Standard ANOVA would yield invalid p-values due to non-normal residuals and correlated errors across methods within the same seed. |
| **Sensitivity Sweep ($\beta$ & $\alpha$)** | Required by Spec (FR-007) and to address the missingness rate confounder. | A single $\beta$ or fixed $\alpha$ would conflate missingness proportion with MNAR strength, making the breakdown point uninterpretable. |
| **Dual Estimators (IPW & PSM) + Baselines** | Required by Spec (FR-004) to isolate imputation effects. Baselines (Oracle/Complete Case) are added to separate estimator failure from imputation failure. | Using only one estimator or no baseline would conflate imputation bias with estimator-specific bias, failing to isolate the variable of interest. |
| **CI Coverage Analysis** | Required by Constitution (Principle VII) to assess validity. | Point estimates alone do not capture the uncertainty or calibration of the method under MNAR. |


## Risks & Mitigations

- **Risk**: MICE fails to converge.
  - **Mitigation**: Catch exception, log seed, retry with A sufficient number of iterations will be performed to ensure convergence of the optimization process, as determined by the stopping criteria defined in the methodology. or exclude from ANOVA with a flag.
- **Risk**: Extreme sparsity (>90% missing) causes estimator failure.
  - **Mitigation**: Check missingness rate; if >90%, flag result as "unreliable" but do not crash.
- **Risk**: VIF > 10 in confounders.
  - **Mitigation**: Regenerate $X$ if VIF > 10 detected.
- **Risk**: Bias variance is trivial (deterministic).
  - **Mitigation**: Analyze variance of bias across replications; if variance is near zero, report as "deterministic error" rather than statistical distribution.
