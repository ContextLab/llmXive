## Unresolved concerns

- FR-006 and SC-004 require a specific decision tree (Shapiro-Wilk → ANOVA/Friedman) for statistical testing. Task T028 implements the logic but splits the 'skewed distribution' bootstrap fallback into a separate task T034. This fragmentation risks the primary test (ANOVA/Friedman) executing without the necessary robust alternative if data is skewed but normal (Shapiro-Wilk p >= 0.05), leaving the spec requirement for a 'robust alternative' conditionally unimplemented in a single coherent workflow.
- FR-007 and SC-005 require a sensitivity analysis confirming a monotonic trend (Spearman ρ > 0.9, p < 0.05). Task T030 computes the correlation but lacks an explicit verification step to assert the trend condition or flag failure. Task T035 handles regression for coverage (SC-002) but is not integrated into the bias trend analysis, leaving the specific 'monotonicity confirmation' of FR-007/SC-005 partially unimplemented as a single testable outcome.
- SC-002 requires verifying a statistically significant negative slope (p < 0.05) in the coverage-vs-beta regression. Task T035 implements the regression logic but lacks a specific verification step or output assertion that explicitly checks for the 'negative slope' condition or flags failure if the slope is positive/non-significant, leaving the success criterion unmeasurable by the task's completion condition.
