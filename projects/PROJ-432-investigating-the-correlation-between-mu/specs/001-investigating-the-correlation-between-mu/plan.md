# Implementation Plan: Investigating the Correlation Between Muon Flux and Atmospheric Temperature Profiles

**Branch**: `001-muon-temp-correlation` | **Date**: 2026-06-26 | **Spec**: `specs/001-muon-temp-correlation/spec.md`
**Input**: Feature specification from `specs/001-muon-temp-correlation/spec.md`

## Summary

This project implements a statistical analysis pipeline to investigate the correlation between IceCube muon flux and atmospheric Effective Temperature ($T_{eff}$) derived from ERA5 reanalysis data. The approach involves ingesting time-series data, aligning temporal resolutions, calculating $T_{eff}$ via weighted integration of vertical temperature profiles, and performing Pearson/Spearman correlation and linear regression analysis. The pipeline includes sensitivity analysis on weighting parameters and seasonal stratification (Summer vs. Winter). All methods are designed to run on CPU-only CI environments.

## Phase Breakdown

### Phase 0: Data Ingestion, Validation & Alignment (FR-001, FR-002, FR-007)
1.  **Download**: Fetch IceCube muon flux (via `icecube_daily_counts` loader or fallback) and ERA5 temperature (via `cdsapi` loader).
2.  **Validate**: Check for `date` column and non-negative `count` in IceCube. Check for `pressure` and `temperature` in ERA5.
3.  **Energy Filter**: Filter IceCube data for `energy > 1 TeV` (if available).
4.  **Align**: Resample to daily bins. Drop dates where either source is missing.
5.  **Overlap Check**: If intersection is empty, trigger **Fallback Strategy** (load NOAA). If still empty, output `mismatch_report.json` and exit.
6.  **Log**: Record exclusion counts and dates to `logs/alignment.log`.

### Phase 1: Effective Temperature Calculation (FR-008)
1.  **Weighting**: Implement $T_{eff}$ using the **Grieder (1985)** weight function ($z_{peak}=15$km, $\sigma=5$km).
2.  **Interpolation**: Use linear interpolation *only* for missing pressure levels within the integration range.
3.  **Output**: Generate `aligned_dataset.csv` with `date`, `muon_count`, `t_eff_value`, `energy_flag`.

### Phase 2: Statistical Analysis (FR-003, FR-004)
1.  **Pre-whitening**: Fit AR(1) model to `muon_count` and `t_eff_value`. Use residuals for correlation.
2.  **Correlation**: Compute Pearson ($r$) and Spearman ($\rho$) on residuals.
3.  **Regression**: Fit OLS model on residuals. Apply **Newey-West** standard errors for p-values and CIs.
4.  **Validation**: Check for outliers (>3$\sigma$) and report impact.

### Phase 3: Sensitivity & Stratification (FR-005, FR-006)
1.  **Power Check**: If `len(summer) < 30` or `len(winter) < 30`, skip Fisher's r-to-z test and report "Insufficient Data".
2.  **Stratification**: If power check passes, split data and run separate correlations.
3.  **Sensitivity**: Sweep weighting function parameters. Compare $r$ and slope.

### Phase 4: Reporting & Validation
1.  **Output**: Generate summary JSON/CSV with all metrics.
2.  **Contract Check**: Validate output against `contracts/analysis_output.schema.yaml`.
3.  **Reproducibility**: Run end-to-end test with fixed seed.
