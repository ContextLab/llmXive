# Implementation Plan: Predicting Battery Electrolyte Decomposition Products via DFT and Machine Learning

**Branch**: `001-battery-electrolyte-decomposition` | **Date**: 2026-07-10 | **Spec**: `specs/001-battery-electrolyte-decomposition/spec.md`
**Input**: Feature specification from `/specs/001-battery-electrolyte-decomposition/spec.md`

## Summary

This feature implements a computational pipeline to predict battery electrolyte decomposition energies using Density Functional Theory (DFT) data and Machine Learning (Random Forest). The system ingests pre-computed molecular structures and energies from verified HuggingFace DFT datasets, extracts electronic (HOMO/LUMO) and geometric descriptors, calculates decomposition energies across electrochemical potentials (0V, 2V, 4V) using a **hardcoded reaction mechanism lookup table**, and trains a Random Forest Regressor. The model's feature importance is analyzed across potential bins, and predictions are validated against **held-out DFT data** (internal consistency) due to the absence of a verified experimental onset potential dataset. The implementation strictly adheres to CPU-only constraints for GitHub Actions runners.

3.  **Target Calculation (FR-002)**: Calculate $E_{decomp}$ for $\phi \in \{0, 2, 4\}$ V using **hardcoded reaction mechanisms** (lookup table).
4.  **Model Training (FR-004)**: Train **single global** Random Forest with 5-fold CV on A standard train-test split will be employed, with the majority of the data allocated for training and a smaller portion reserved for testing, following established protocols in the field (DOI/arXiv/author-year). (Potential as a feature).
5.  **Feature Importance (FR-005)**: Generate permutation importance per potential bin (conditioned on potential).
6.  **Validation (FR-006 - Fallback)**: Compare predictions against held-out DFT data (Internal Consistency). **Flag**: External experimental validation is not implementable.
7.  **Sensitivity Analysis (FR-007)**: Sweep threshold $\{0.45, 0.50, 0.55\}$ eV.
