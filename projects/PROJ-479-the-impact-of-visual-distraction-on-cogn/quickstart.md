# Quickstart Guide: Visual Distraction & Cognitive Control Study

This guide provides instructions for setting up and running the research pipeline to analyze the impact of visual distraction on cognitive control in remote work environments.

## Prerequisites

- Python 3.9+
- pip package manager
- CPU-only environment (GPU not required)

## Installation

1. Clone the repository:
 ```bash
 git clone <repository-url>
 cd PROJ-479-the-impact-of-visual-distraction-on-cogn
 ```

2. Install dependencies:
 ```bash
 pip install -r code/requirements.txt
 ```

## Pipeline Execution

The pipeline consists of four main stages. Run them in order:

1. **Data Acquisition** (T015, T015b, T020):
 ```bash
 python code/01_data_acquisition.py
 ```
 This generates the merged dataset `data/processed/merged_data.csv` and raw artifacts.

2. **Visual Metric Extraction** (T024-T028):
 ```bash
 python code/02_visual_metrics.py
 ```
 This computes edge density, color entropy, and object counts, saving results to `data/processed/final_analysis_data.csv`.

3. **Statistical Analysis** (T031-T044):
 ```bash
 python code/03_analysis.py
 ```
 This performs correlations, VIF checks, PCA (if needed), and bootstrap resampling, outputting to `results/statistics/` and `results/sensitivity/`.

4. **Visualization** (T038):
 ```bash
 python code/04_visualization.py
 ```
 This generates scatter plots and the visual summary report.

5. **Final Report** (T045):
 The final report `results/report.md` is generated automatically by the analysis and visualization steps.

## Data Source Selection

The pipeline supports both real public datasets and a synthetic data fallback. The selection logic is handled automatically in `code/01_data_acquisition.py`.

### Real Dataset Path

The acquisition script first attempts to download participant-level cognitive data (Stroop/Flanker tasks) linked to workspace images from verified sources:
- HuggingFace Datasets (IDs: `StroopTaskDataset`, `FlankerDataset`)
- OpenML

**Verification Steps**:
1. The script checks for the existence of these datasets programmatically.
2. If a dataset is found and downloaded successfully, it is used for all subsequent steps.
3. The script validates that the downloaded data contains the required columns (`reaction_time`, `accuracy`, `image_path`) and has sufficient participants (N ≥ 100).
4. If validation passes, the `synthetic_fallback` flag remains `False`, and the pipeline proceeds with real data.

### Synthetic Data Path

If no linked public dataset is found (verified by check) or the download fails, the system sets the `synthetic_fallback` flag to `True` and proceeds with the following generation process:

1. **Cognitive Data Generation**:
 - Generates N ≥ 100 synthetic participant records.
 - Simulates a negative correlation between visual complexity and cognitive performance (reaction time/accuracy) based on configurable `target_correlation` parameters.
 - Uses a configurable `literature_source` parameter to derive correlation structures.

2. **Image Generation**:
 - Simultaneously generates synthetic workspace images using the `Pillow` library.
 - Images include varied lighting conditions, room types, and demographic metadata.
 - **Atomic Generation**: Participant IDs in the CSV match the filenames of the generated images exactly within a single execution step.

3. **Validation**:
 - Immediately after generation, the script computes edge density on a sample of images.
 - If the standard deviation of edge density is 0 (indicating no variance), the script raises a `ValueError` and halts execution to prevent invalid data generation.

## Interpretation of Results

All findings generated by this pipeline are **associational**. No causal claims are made regarding the impact of visual distraction on cognitive control.

### Associational Framing

- **Correlation vs. Causation**: The statistical methods used (Pearson correlation, linear regression) identify relationships between variables but do not establish causality.
- **Confounding Variables**: While the analysis controls for known visual metrics (edge density, color entropy, object count), unmeasured confounders may exist.
- **Generalizability**: Results are specific to the dataset used (real or synthetic) and the specific conditions of the study.

### Real Dataset Interpretation

When using real data:
- Results reflect actual observed associations in the specific dataset downloaded.
- External validity depends on the representativeness of the source dataset.
- Confidence intervals and p-values indicate the strength and reliability of the observed associations.

### Synthetic Data Interpretation

When using synthetic data:
- Results validate the pipeline's ability to detect the pre-specified correlation structure.
- Synthetic data is generated to exhibit a known negative correlation; findings should confirm this theoretical relationship.
- Synthetic results are primarily useful for methodological validation and do not generalize to real-world populations without further empirical testing.

## Output Artifacts

- `data/processed/merged_data.csv`: Combined participant and cognitive data.
- `data/processed/final_analysis_data.csv`: Data with visual metrics, filtered for NaN object counts.
- `results/statistics/statistics.json`: Correlation coefficients, p-values, and confidence intervals.
- `results/statistics/vif_report.json`: Variance Inflation Factor scores.
- `results/statistics/power_analysis_report.md`: Power analysis methodology and results.
- `results/sensitivity/bootstrap_results.json`: Bootstrap resampling confidence intervals.
- `results/plots/`: Scatter plots with trend lines for significant correlations.
- `results/report.md`: Comprehensive final report including methods citations, alpha threshold justification, and multiplicity tables.

## Troubleshooting

- **Missing Dependencies**: Ensure all packages in `code/requirements.txt` are installed.
- **Image Processing Failures**: Check `data/raw/` for corrupted images; the pipeline logs these errors and skips affected records.
- **Zero Variance in Synthetic Data**: If the synthetic data generation fails variance checks, verify the `target_correlation` and image generation parameters in `code/01_data_acquisition.py`.
