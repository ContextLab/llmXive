# Implementation Plan: Exploring the Impact of Molecular Dynamics Simulation Parameters on Predicted Protein-Ligand Binding Affinity

**Branch**: `001-md-simulation-params` | **Date**: 2023-10-27 | **Spec**: `specs/001-md-simulation-params/spec.md`
**Input**: Feature specification from `/specs/001-md-simulation-params/spec.md`

## Summary

This feature implements a CPU-tractable pipeline to execute Molecular Dynamics (MD) simulations on a subsampled set of protein-ligand complexes, calculate binding free energy estimates (MM-PBSA/GBSA), and perform statistical sensitivity analysis using a **Linear Mixed-Effects Model (LMM)** to determine the impact of simulation parameters (force field, duration) on prediction accuracy. The system strictly adheres to the CI runner time limit (CPU, constrained RAM) by subsampling to [deferred: target 10] complexes and using short simulation durations (0.5ns, 1.0ns, 1.5ns) with lightweight MD engines (OpenMM).

**Note on Constraints**: This plan explicitly deviates from Constitution Principle VI regarding duration (5ns, 10ns, 20ns) and temperature (300K, 310K) and force fields (ff19SB) due to hardware constraints. The selected parameters (0.5-1.5ns, 300K, ff14SB/CHARMM36m) are the maximum feasible set for the free-tier CI environment. This deviation is flagged for a future spec update to align the requirements with the implementation reality.

| **Short durations (0.5ns, 1.0ns, 1.5ns)** | Required to ensure individual jobs complete within 5 minutes. | Longer durations (5ns+) would cause CI timeouts, preventing data generation. (Deviation from Constitution VI). |
| **OpenMM over GROMACS** | OpenMM has better Python integration and CPU-only wheels available via `pip`. | GROMACS often requires complex compilation or GPU-specific binaries in CI environments. |
| **LMM over ANOVA** | Required to handle the "Complex" as a random effect and small N. | Standard ANOVA on aggregated RMSE is statistically invalid for this design. |

## Statistical Design Notes

- **Response Variable**: Absolute Error (AE) per complex per parameter set ($| \Delta G_{pred} - \Delta G_{exp} |$).
- **Model**: Linear Mixed-Effects Model (LMM) with `Complex` as a random intercept.
- **Fixed Effects**: Force Field, Duration.
- **Output**: Fixed effect estimates, confidence intervals, and variance components.
- **RMSE**: Calculated post-hoc as a summary metric of the model residuals, not as the input to the model.
- **Power**: N=10 is low. The study is exploratory. Results will be framed as effect size estimates with wide confidence intervals, not definitive hypothesis tests.

## Risk Mitigation

- **Timeout**: Hard timeout per run (simulation + MM-PBSA). If exceeded, the run is marked "failed" and logged. This is stricter than the spec's time limit to ensure the 6-hour total budget is met.
- **Memory**: Trajectory files truncated to ≤ 20 frames for MM-PBSA.
- **Statistical Power**: N=10 is low. The study is exploratory. Results will be framed as effect size estimates with wide confidence intervals, not definitive hypothesis tests.

## Scientific Caveat

- **MM-PBSA Bias**: The plan acknowledges that MM-PBSA is known to have systematic bias that may dominate duration effects in short simulations.. The analysis will report effect sizes with wide confidence intervals and acknowledge that 'duration' may show no significant effect due to the short timescale and methodological bias.
- **Observational Framing**: The study is purely observational; therefore, any observed correlations between parameter settings and prediction accuracy are interpreted as associational, not causal.
