# Implementation Plan: Predicting Plant Stress Response from Publicly Available Proteomic Data

**Branch**: `001-predict-plant-stress-response` | **Date**: 2026-06-27 | **Spec**: `specs/001-predicting-plant-stress-response/spec.md`
**Input**: Feature specification from `/specs/001-predicting-plant-stress-response/spec.md`

## Summary

This feature implements a computational pipeline to train machine learning models (Random Forest, SVR) that predict plant gene expression patterns from proteomic data under abiotic stresses (drought, salinity, heat). The approach involves ingesting public datasets, harmonizing identifiers via `mygene` (with `biomaRt` fallback), imputing missing values using the MinProb algorithm (LCM) with a rigorous detection limit estimation protocol, and evaluating cross-stress generalization on a CPU-only GitHub Actions runner. A critical "Stress-Blind Baseline" is implemented to distinguish universal regulatory mapping from stress-class memorization. Due to small sample sizes in public datasets (n < 10), the plan utilizes Leave-One-Out Cross-Validation (LOOCV) for within-stress evaluation.

## Complexity Tracking

| Violation | Why Needed | Simpler Alternative Rejected Because |
|-----------|------------|-------------------------------------|
| **mygene vs biomaRt** | `mygene` is Python-native and CPU-efficient; `biomaRt` is a fallback for specific plant orthology coverage. | Pure Python alternatives (e.g., `mygene`) lack the specific plant orthology coverage required for accurate UniProt-Ensembl mapping in Arabidopsis/Rice/Wheat *unless* `biomaRt` is used. |
| **MinProb Imputation** | Missing values in proteomics are often "missing not at random" (below detection limit). | Mean/median imputation would bias results by inflating low-abundance proteins, violating biological reality. `impyute` library provides MinProb. |
| **Stress-Blind Baseline** | Required to distinguish universal mapping from stress-class memorization. | Simple k-fold CV would only test within-stress generalization, failing to address the core research question of cross-stress prediction. |
| **LOOCV vs 5-Fold** | Public datasets often have n < 10 per condition; K-fold CV yields test sets of size 2, making R² variance too high. | 5-fold CV is statistically unreliable for small n. LOOCV maximizes training data and provides a more stable (though potentially optimistic) estimate for small samples. |
| **Target-Permutation Test** | Required to validate the regression model (Protein->Gene) rather than stress classification. | Shuffling stress labels is invalid for regression validation; shuffling target values is the correct null model. |

## Implementation Phases

### Phase 0: Data Acquisition & Verification (Critical Path)

1.  **Reference-Validator Integration**:
    *   Execute `Reference-Validator` agent on the manually curated accession list (from `data/config.yaml`).
    *   Verify each accession against the primary source (NCBI/ProteomeXchange) to ensure it exists and contains the required species/stress metadata.
    *   *Constitution II Compliance*: Only verified accessions proceed.
2.  **Dataset Selection (FR-001)**:
    *   Query NCBI/ProteomeXchange for all datasets matching `[species] AND [stress]`.
    *   Compare sample sizes (n) across all results.
    *   Select the dataset with the **largest n**.
    *   **Tie-Breaker**: If n is equal, select the dataset with the **earliest publication date**.
    *   Log the selection criteria and the rejected datasets.
3.  **Data Ingestion**:
    *   Download raw files using `wget`/`curl`.
    *   Calculate SHA-256 checksums and record in `data/raw/`.
    *   **Fatal Failure Mode**: If no paired datasets (proteomic + transcriptomic) are found for the requested species/stress, halt with error `E001_NO_PAIRED_DATA` and log the "No Paired Data" condition.
    *   **Pivot Strategy**: If paired data is unavailable, log a warning and proceed to a "Correlation Meta-Analysis" of separate studies (if feasible) or halt.

### Phase 1: Data Preprocessing & Harmonization

1.  **Normalization & Filtering (FR-002)**:
    *   Filter proteins detected in < 50% of samples within a stress condition.
    *   **Detection Limit Estimation (Critical)**:
        *   For each protein column, calculate the detection limit (DL) using the following hierarchy:
            1.  **Standard**: If the dataset provides a reported DL, use it.
            2.  **Percentile**: If >5 non-missing values exist, estimate DL as a low percentile of non-missing values.
            3.  **Small Sample Fallback**: If a small number of non-missing values exist, use the global minimum non-zero value across the entire dataset as the DL.
            4.  **All Missing**: If 0 non-missing values exist, **drop the column** and log the protein ID. Do not attempt imputation.
    *   Apply **MinProb** imputation (via `impyute`) using the estimated DL for Left-Censored Missing values.
2.  **Identifier Mapping (FR-003)**:
    *   Attempt mapping using `mygene` (Python).
    *   **Fallback**: If `mygene` success rate < 80%, switch to `biomaRt` (R) using the `plants_mart` dataset.
    *   *Verification*: Check availability of `plants_mart` in the environment. If unavailable, use a pre-computed mapping table from a validated source (e.g., Planta database).
    *   Log unmatched rows and drop them.
3.  **Data Completeness Calculation (SC-004)**:
    *   Calculate `Data Completeness %` = (Retained Datasets / Initial Query Results) * 100.
    *   Record this metric in `results/data_completeness.json`.

### Phase 2: Modeling & Validation

1.  **Stress-Blind Baseline Implementation (Addressing Methodology-a8545604)**:
    *   **Algorithm**: For each protein feature, regress out the `StressCondition` effect using a linear model: `Protein_residual = Protein - (Stress_Coeff * Stress_Label)`.
    *   Train the main models on these **residualized features** to ensure the model learns regulatory logic (Protein->Gene) independent of the stress class.
    *   Compare performance against a baseline model trained on raw features to quantify the "stress signature" contribution.
2.  **Model Training (FR-004)**:
    *   Train `RandomForestRegressor` and `SVR`.
    *   **Within-Stress Validation (LOOCV)**: Due to small sample sizes (n < 10), use **Leave-One-Out Cross-Validation (LOOCV)** instead of 5-fold CV.
        *   *Rationale*: 5-fold CV on n=10 yields test sets of size 2, leading to high variance in R². LOOCV maximizes training data and provides a more stable estimate.
        *   *Statistical Note*: Acknowledge that LOOCV may be optimistic; report confidence intervals via bootstrap if n permits, otherwise report LOOCV standard error.
    *   **Within-Fold Preprocessing (SC-005)**: All normalization, imputation, and feature selection MUST be performed **inside** the training fold to prevent data leakage.
    *   Hyperparameters: `n_estimators=100`, `max_depth=10` (RF); `kernel='rbf'`, `C=1.0` (SVR).
3.  **Target-Permutation Control Test (FR-005)**:
    *   **Method**: Shuffle the **target gene expression values** (not stress labels) relative to the predictors.
    *   Retrain the model on the shuffled data using the same LOOCV strategy.
    *   Compare R² scores: Real Data R² must be significantly higher than Shuffled Data R² (p < 0.05).
4.  **Cross-Stress Evaluation (FR-005)**:
    *   Train on Drought, Test on Salinity/Heat.
    *   Report R² and RMSE for each stress pair.

### Phase 3: Reporting & Versioning

1.  **Visualization (FR-007)**:
    *   Generate scatter plots, heatmaps, and feature importance bars.
2.  **Metrics Logging (FR-008)**:
    *   Log runtime and memory usage to `runtime_metrics.json`.
3.  **Versioning & State Update (Constitution V)**:
    *   Hash all artifacts in `results/` and `data/processed/`.
    *   Update `state.yaml` with new `updated_at` timestamp and artifact hashes.
    *   *Constitution V Compliance*: Ensure `state.yaml` reflects the current state of all artifacts.

## Risk Mitigation

| Risk | Mitigation |
| :--- | :--- |
| **No Paired Data Found** | Pipeline halts early with `E001_NO_PAIRED_DATA`. Research phase will attempt to find a single "gold standard" dataset manually or pivot to meta-analysis. |
| **Identifier Mapping Failure** | If < 50% of proteins map to genes, the dataset is discarded for that stress type. Fallback to pre-computed mapping table. |
| **Runtime Exceeds 6h** | Implement a "checkpoint" mechanism: save intermediate data after ingestion. If time is low, skip complex SVR and only run RF. |
| **LCM Imputation Failure** | If a column is all missing, drop it and log. Use MinProb from `impyute` with strict DL estimation. |
| **Stress Confounding** | Use Stress-Blind Baseline (residualization) to isolate regulatory signals. |
| **Low Statistical Power** | Acknowledge n < 10 limitation in report. Use LOOCV to maximize data usage. Aggregate results across multiple studies if available. |
