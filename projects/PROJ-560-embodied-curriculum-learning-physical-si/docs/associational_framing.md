# Associational Framing Policy

## Purpose

This document outlines the mandatory policy for framing statistical results
in the Embodied Curriculum Learning project. It ensures that all automated
outputs and generated reports adhere to the project's methodological constraints
(FR-003).

## Core Principle

**No Causal Claims**: The system must never claim that one variable "causes"
or "determines" another. All findings are to be described as "associations"
or "correlations" observed in the data.

## Implementation Details

### Automated Output Generation

The `stats_engine.py` module includes a `frame_inference` function that
explicitly labels all findings. This function ensures that:
1. The term "causal" is never used in the final report.
2. The term "association" is used to describe the relationship between
 instruction type and learning gain.
3. A standard disclaimer is appended to all statistical summaries.

### Example Output Text

**Correct (Associational)**:
> "The analysis reveals a statistically significant association between
> embodied instruction and higher learning gains (p < 0.05)."

**Incorrect (Causal)**:
> "Embodied instruction causes higher learning gains."

## Rationale

The data collected in this project (whether from public sources or synthetic
generation) does not support causal inference without further experimental
design controls (e.g., randomization, control for confounders). By strictly
framing results as associational, we prevent the misinterpretation of
correlational data as causal evidence.

## Compliance

All automated reports generated by `cli.py` must include the `associational_framing`
flag set to `true` in the JSON output. Any manual reports derived from this
data must adhere to the same policy.
