# Statistical Methods and Associational Framing

## Introduction

This document details the statistical methods employed in the Embodied Curriculum Learning project. It emphasizes the **associational nature** of all findings, ensuring that no causal claims are made.

## Data Processing

### Gain Score Calculation

The primary metric of interest is the **learning gain**, calculated as the difference between post-intervention and pre-intervention scores:

$$ \text{Gain} = \text{Post\_Score} - \text{Pre\_Score} $$

Rows with missing values in either `pre_test_score` or `post_test_score` are excluded from the analysis. A log of skipped records is maintained in `data/derivation_logs/skipped_records.log`.

### Handling Missing Instruction Types

Per FR-008, if the `instruction_type` column is missing in the input dataset, the system automatically invokes the `SyntheticDataGenerator`. This ensures a deterministic fallback without manual intervention, allowing the pipeline to proceed with a labeled dataset for validation purposes.

## Statistical Inference

### Independent Samples T-Test

To compare the mean learning gains between two instruction groups (e.g., "embodied" vs. "static"), the system performs an independent samples t-test.

1. **Homogeneity of Variance**: Levene's test is conducted to determine if the variances of the two groups are equal.
2. **Test Selection**:
 - If variances are equal (Levene's p > 0.05), **Student's t-test** is used.
 - If variances are unequal (Levene's p ≤ 0.05), **Welch's t-test** is used.

The null hypothesis ($H_0$) is that the mean gains of the two groups are equal. The alternative hypothesis ($H_1$) is that they are different.

### Effect Size: Cohen's d

To quantify the magnitude of the difference, Cohen's d is calculated:

$$ d = \frac{\bar{X}_1 - \bar{X}_2}{s_{pooled}} $$

Where $s_{pooled}$ is the pooled standard deviation.
- Small effect: $d \approx 0.2$
- Medium effect: $d \approx 0.5$
- Large effect: $d \approx 0.8$

### Confidence Intervals

95% confidence intervals are calculated for the difference in means and the effect size, providing a range of plausible values for the population parameter.

### Multiple Comparisons: Bonferroni Correction

When testing multiple concepts or hypotheses, the risk of Type I errors increases. The **Bonferroni correction** is applied:

$$ \alpha_{corrected} = \frac{\alpha}{k} $$

Where $k$ is the number of tests performed. This ensures the family-wise error rate remains at the desired level (e.g., 0.05).

### Power Analysis

The system calculates the **achieved power** of the t-test given the sample size, effect size, and significance level.
- Power < 0.80 is flagged as "underpowered," indicating a higher risk of Type II errors (failing to detect a true effect).

### Collinearity Diagnostics

If covariates are included in the analysis, the system checks for multicollinearity. A correlation coefficient $|r| > 0.8$ between predictors triggers a diagnostic warning, as high collinearity can inflate standard errors and destabilize estimates.

## Sensitivity Analysis

To assess the robustness of the findings, a **sensitivity sweep** is performed. The analysis is re-run across a range of significance thresholds (e.g., 0.05, 0.01, 0.001).
- If the sample size (N) is less than 30, the sweep is skipped, and a warning is issued.
- If the effect size drops below a substantively meaningful threshold at any point in the sweep, a `robustness_warning` is set to `true`.

## Associational Framing (FR-003)

**Critical Methodological Note**:

All statistical findings generated by this system are explicitly framed as **associational**. The analysis identifies statistical correlations between instruction types and learning outcomes.

- The system **does not** claim that one instruction method "causes" better learning outcomes.
- The system **does not** use terms like "teaching efficacy" or "causal impact" in its output.
- All reports include a disclaimer stating that the results represent associations observed in the data, subject to confounding variables and study design limitations.

This framing adheres to FR-003, ensuring that the project remains scientifically rigorous and avoids over-interpreting correlational data as causal evidence.

## Output Format

Results are aggregated into an `AnalysisResult` object and written to `data/processed/results.json`. The JSON structure includes:
- `t_statistic`: The calculated t-value.
- `p_value`: The p-value from the t-test.
- `effect_size`: Cohen's d.
- `confidence_interval`: [lower, upper] bounds.
- `power`: Calculated power.
- `associational_framing`: A boolean flag and text note confirming the non-causal interpretation.
- `robustness_warning`: Boolean indicating sensitivity to threshold changes.
