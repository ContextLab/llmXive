# Implementation Plan: Predicting Personal Sleep Quality from Resting‑State fMRI Connectivity

**Branch**: `[001-predict-sleep-quality]` | **Date**: 2026-06-25 | **Spec**: `specs/001-predict-sleep-quality/spec.md`

## Summary

This project implements a reproducible, CPU-tractable pipeline to predict personal sleep quality scores from resting-state functional connectivity (rs-fMRI) matrices derived from the Human Connectome Project (HCP) large-scale release. The technical approach involves ingesting minimally preprocessed HCP data, parcellating time series using the Schaefer atlas, computing Fisher-z transformed connectivity vectors, and training an Elastic-Net regression model within a nested cross-validation framework. 

**Critical Methodological Correction**: To prevent data leakage and ensure statistical validity, Variance Thresholding and PCA are **NOT** performed on the full dataset. Instead, these dimensionality reduction steps are fitted **strictly within the training fold** of every cross-validation iteration and every permutation test iteration. 

To meet the 5-hour compute constraint on GitHub Actions (2 vCPU, 7 GB RAM) while maintaining the requirement for a sufficient number of permutations of the full pipeline, the permutation test will be executed on a **stratified subset** of 100 subjects. The main model evaluation will run on the full set of subjects. This trade-off is documented to preserve the statistical validity of the null hypothesis testing (re-fitting features) while remaining computationally feasible.

**Note on Estimates**: All numerical claims in this plan (e.g., subject counts, feature dimensions) are **provisional projections** based on the spec assumptions. The Single Source of Truth (SSoT) for all final metrics is the `ResultReport.json` generated by the code.

## Complexity Tracking

| Violation | Why Needed | Simpler Alternative Rejected Because |
|-----------|------------|-------------------------------------|
| **Nested CV (multiple folds)** | Required by FR-004 and SC-006 to prevent overfitting during hyperparameter tuning. | Simple hold-out or single CV would inflate performance estimates in high-dimensional (p >> n) neuroimaging data. |
| **Dynamic Feature Selection** | Required to prevent data leakage (SC-002). Variance Thresholding and PCA must be fitted *within* the training fold of every permutation. | Pre-computing features on the full dataset (static matrix) would invalidate the null hypothesis and inflate Type I error rates. |
| **Permutation Test ()** | Required by FR-006 for family-wise error correction. | Analytical p-values assume normality which is often violated; permutation provides a robust non-parametric alternative. |
| **Stratified Subsampling for Permutations** | Required to meet the 5-hour compute constraint on 2 vCPU. | Running [deferred] full nested CV loops on a large cohort of subjects would exceed 5 hours. Using a subset (100 subjects) preserves the *method* (re-fitting features) while meeting time limits. |

## Implementation Phases

### Phase 1: Data Ingestion & Preprocessing
1. **Download**: Fetch HCP minimally preprocessed CIFTI files and `hcp1200_behavioral_data.csv`. Verify checksums.
2. **Filter**: Identify subjects with valid sleep data. Calculate **Success Rate** (Target: ≥80%). If <80%, log warning and halt (SC-001).
3. **Preprocess**: Parcellate using Schaefer et al.

The study investigates whether [research question], employing [method]. Prior work by Schaefer et al. suggests a significant relationship in this domain.. Apply band-pass filter (low-frequency cutoff).
4. **Feature Engineering**: Compute Fisher-z correlation matrices. **Do NOT apply variance thresholding or PCA yet.** Store raw vectors.

### Phase 2: Model Training & Evaluation (Full Set)
1. **Nested CV**: Run Outer/Inner Cross-Validation on the full dataset (~-300 subjects).
2. **Dynamic Reduction**: Inside the *inner* loop, apply Variance Thresholding and PCA **only on the training fold**.
3. **Metrics**: Aggregate Pearson r and R². Generate bootstrap CI on the full set.

### Phase 3: Permutation Testing (Subset)
1. **Subset Selection**: Select a stratified random subset of subjects (preserving sleep score distribution).
2. **Loop**: Run a sufficient number of permutations.
   - Shuffle labels.
   - **Re-run the entire nested CV pipeline** (including dynamic feature selection) on the subset.
   - Record null R².
3. **P-Value**: Calculate empirical p-value from the null distribution.

### Phase 4: Sensitivity Analysis (SC-003)
1. **Grid Search**: Iterate through variance thresholds {, 0.01, 0.02} and PCA retention {, 0.90, 0.85}.
. **Execution**: Run the full nested CV (Phase) for each of the combinations.
3. **Reporting**: Log R² for each combination in `ResultReport.json`.

### Phase 5: Visualization & Validation (SC-004)
1. **Extraction**: Map non-zero coefficients to brain edges.
2. **Plotting**: Generate brain surface plot with a representative subset of the strongest edges.
3. **Validation**: Verify plot file exists and contains ≥50 edges. If fail, retry or log error. Target success rate ≥95%.

### Phase 6: Final Reporting
1. **Generate**: `ResultReport.json` with all metrics, p-values, CIs, and paths.
2. **Log**: Structured JSON log of the entire run.
3. **Verify**: Run contract tests against `result.schema.yaml`.

## Risks & Mitigations

| Risk | Impact | Mitigation |
|------|--------|------------|
| **Compute Time Exceeds 5h** | CI Job Failure. | Use Stratified Subsampling for permutations. Implement early stopping if time > 4h. |
| **Data Leakage** | Invalid p-values. | Enforce 'Fit-Within-Loop' for all dimensionality reduction. |
| **Low Power** | False negatives. | Explicitly report power calculation. Frame as exploratory. |
| **Missing Sleep Data** | No outcome. | Validate column existence in `hcp1200_behavioral_data.csv` immediately. |
