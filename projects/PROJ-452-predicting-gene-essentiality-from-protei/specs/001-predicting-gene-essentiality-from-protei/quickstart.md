# Quickstart: Predicting Gene Essentiality from Protein Interaction Network Topology

## Prerequisites
- Python 3.11+
- `pip` or `conda`
- Access to a Unix-like environment (Linux/macOS/WSL)
- Internet access (to fetch data from STRING, DEG, OpenTree APIs)

## Installation

1.  **Clone the repository** (if applicable) or navigate to the project root.
2.  **Create a virtual environment**:
    ```bash
    python -m venv venv
    source venv/bin/activate  # On Windows: venv\Scripts\activate
    ```
3.  **Install dependencies**:
    ```bash
    pip install -r code/requirements.txt
    ```
    *(Note: `requirements.txt` will be generated by the Implementer Agent, but will include: `networkx`, `pandas`, `scipy`, `statsmodels`, `requests`, `pyyaml`, `numpy`, `pytest`, `dendropy`, `biopython`)*

## Data Setup

The pipeline expects data in `data/raw/`. The `main.py` script will automatically fetch data from the official APIs (STRING, DEG, OpenTree) defined in `research.md`.

1.  **Run the pipeline** (Data fetch is automatic):
    ```bash
    python code/main.py
    ```
    *This will download the necessary files from the verified URLs and perform the analysis.*

2.  **Verify data presence**:
    Ensure `data/raw/` contains the downloaded files.

## Running the Analysis

Execute the full pipeline:

```bash
python code/main.py
```

**Configuration**:
Edit `code/config.py` to modify:
- `ORGANISMS`: List of organism IDs (default: ["9606", "10090", ...]).
- `THRESHOLDS`: List of confidence scores (default: [500, 700, 900]).
- `PHYLOGENY_FILE`: Path to the Newick tree (optional; if missing, OpenTree API is used).

## Expected Outputs

After completion, check the `results/` directory:
- `results/correlations.json`: Per-organism correlation stats (includes `mapping_coverage_percent`).
- `results/pgls_results.json`: Comparative statistics (if tree available).
- `results/sensitivity_report.md`: Markdown report of threshold sensitivity.
- `results/null_distribution/`: Empirical p-values from label permutation.

## Testing

Run the test suite to verify correctness:

```bash
pytest tests/ -v
```

**Contract Tests**:
Ensure outputs match the YAML schemas in `contracts/`:
```bash
pytest tests/contract/test_schemas.py -v
```
*Note: The pipeline will fail if these contract tests fail.*

## Troubleshooting

- **Memory Error**: Reduce the number of organisms or sample the network (modify `config.py`).
- **No Data Found**: The API may not have data for the requested organism IDs. Check logs for "Skipping organism: [ID] - No overlapping genes".
- **PGLS Skipped**: If no phylogenetic tree is found, PGLS is skipped with a warning.
- **Rate Limiting**: If API calls fail, the script will retry with exponential backoff.
