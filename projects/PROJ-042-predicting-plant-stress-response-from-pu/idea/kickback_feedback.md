## Unresolved concerns

- FR-003 states: 'retain only genes present in the intersection of [deferred] of the selected datasets'. This contradicts Constitution Principle VII which mandates 'Only genes present in all integrated datasets are retained'. The use of '[deferred]' here implies a variable threshold (e.g., 80%) rather than the strict 100% intersection required by the Constitution. This creates a silent scope creep where the spec allows a relaxation of the constitutional constraint without explicit anchoring to a story that justifies this deviation.
- FR-004 states: 'retain the top [deferred] most variable genes'. While the assumption section mentions a sensitivity analysis, the requirement itself lacks a concrete anchor to a specific User Story that defines the *minimum* variable gene count or the *method* for determining the 'top [deferred]' value. Without a specific story defining the selection logic (e.g., 'US-X: System MUST select top N genes where N is determined by...'), this requirement is unanchored and risks implementation ambiguity.
- FR-017 requires updating `state/projects/<PROJ-ID>/projects.yaml`. While it cites 'See US-1', US-1 is about 'Cross-Dataset Stress Signature Separability'. Updating a state file is a system maintenance task, not a direct fulfillment of the research hypothesis in US-1. It should be anchored to a story about 'Project State Management' or 'Versioning Discipline' (if such a story exists), or explicitly noted as a constitutional compliance requirement (Constitution Principle V) rather than a functional requirement of the research logic.
- FR-018 mandates framing findings as 'ASSOCIATIONAL'. It cites 'See US-1'. However, US-1 is about 'Separability' and 'Generalization'. While related, the specific constraint of 'associational framing' is a direct consequence of the study design (observational) and Constitution Principle I (Reproducibility/Methodology). It is weakly anchored to US-1 and might be better served by a specific story about 'Reporting Standards' or 'Scientific Integrity' to ensure the constraint is not lost if US-1 is refactored.
- FR-003 states: 'retain only genes present in the intersection of [deferred] of the selected datasets'. The phrase 'intersection of [deferred]' is semantically contradictory. 'Intersection' implies a specific set operation (100% overlap), while '[deferred]' implies a missing parameter. This conflicts with Constitution Principle VII which mandates 'Only genes present in all integrated datasets are retained'. The spec must explicitly state 'intersection of ALL selected datasets' or remove the '[deferred]' marker if the threshold is fixed at 100%.
- FR-004 states: 'retain the top [deferred] most variable genes'. This creates a conflict with the 'Independent Test' in User Story 2 which specifies: 'calculates the Jaccard index of the top 50 genes'. If the feature selection threshold is deferred/unknown, the specific test case of 'top 50' cannot be validated against the system's output unless the system defaults to 50 or the test is generalized. The spec must either fix the number (e.g., 'top 1500' as per FR-004's error condition) or clarify that the 'top 50' in US-2 is a subset of the deferred top-K.
- Acceptance Scenario 3 defines an 'Ambiguous' range as 'strictly between 0.25 and 0.60 (exclusive of 0.25, inclusive of 0.60)'. However, the text says 'inclusive of 0.60' but the logic for 'above 0.60' in Scenario 1 implies 0.60 is the success threshold. If 0.60 is inclusive in the 'Ambiguous' bucket, then a result of exactly 0.60 triggers 'Ambiguous' and halts analysis, contradicting Scenario 1 which says 'above 0.60' (implying >0.60) is success. The boundary condition at 0.60 is contradictory: is 0.60 a pass or an ambiguous halt?
- The 'FeatureSpace' entity definition states: 'The intersection of gene identifiers present in **[deferred]** of datasets'. This repeats the semantic error found in FR-003. '[deferred]' is a placeholder for a value, not a quantity of datasets. It should read 'present in ALL selected datasets' to align with Constitution Principle VII and FR-003's intent.
- FR-015 states: 'The system MUST halt execution if the p-value < 0.05 indicates significant confounding.' However, the 'Edge Cases' section states: 'If significant confounding is detected, the system MUST exclude that dataset... and log a warning.' FR-015 mandates a hard HALT, while the Edge Cases mandate an EXCLUDE and CONTINUE (with a warning). These are mutually exclusive behaviors for the same condition (confounded batch/condition).
- FR-003 mentions 'using `biopython`' for gene identifier harmonization. While `biopython` is a valid library, gene identifier mapping (e.g., Ensembl to Symbol, or cross-species orthology) is typically handled by `mygene`, `biomart`, or `gprofiler` APIs, not standard `biopython` functions which focus on sequence manipulation. This may be a terminology error or an implementation impossibility if `biopython` alone cannot perform the required cross-species mapping. The spec should verify the tool's capability or use a more precise library name.
- The threshold 'strictly below 0.25' for random chance is mathematically incorrect for a 4-class problem. Random chance is exactly 0.25 (1/4). A model performing at random chance will achieve ~0.25, not strictly below it. The criterion 'strictly below 0.25' is impossible to satisfy by a random classifier and creates a logical contradiction in the test definition. It should be '≤ 0.25' or 'not significantly greater than 0.25'.
- The condition 'accuracy falls strictly between 0.25 and 0.60 (exclusive of 0.25, inclusive of 0.60)' creates a boundary ambiguity. If accuracy is exactly 0.25, it falls into Scenario 1 (random). If it is 0.60, it falls into Scenario 3 (Ambiguous). However, the logic for Scenario 1 says 'strictly below 0.25', which excludes 0.25. This leaves the exact value 0.25 undefined in the acceptance criteria (not < 0.25, not > 0.25, not >= 0.60). The ranges must be partitioned to cover all real numbers without gaps or overlaps.
- The Jaccard index threshold '> 0.30' is an arbitrary empirical value without a defined statistical basis or reference to a null distribution. While the scenario mentions 'statistically significant overlap', it does not define the test (e.g., permutation test of gene sets) or the p-value threshold required to claim significance. A Jaccard of 0.31 might be noise if the gene universe is small. The criterion must specify the statistical test and p-value (e.g., 'Jaccard > 0.30 AND p < 0.05 via permutation test').
- The requirement 'distinct clusters must form for at least 3 out of 4 stress types with a Silhouette Score > 0.4' is ambiguous regarding the aggregation method. Does the system calculate one global Silhouette Score for the entire embedding, or individual scores per cluster? If global, a score > 0.4 might be driven by 2 strong clusters and 2 weak ones. If per-cluster, the spec must state 'for at least 3 clusters, the individual Silhouette Score > 0.4'. The current phrasing is not verifiable.
- The phrase 'intersection of [deferred] of the selected datasets' is syntactically invalid for a functional requirement. '[deferred]' is a placeholder for a number (e.g., '5'), but the requirement needs to specify the *logic* of the intersection (e.g., 'intersection of genes present in ALL selected datasets'). The current text is a template artifact, not a verifiable rule. It must be resolved to 'intersection of genes present in all selected datasets' to be testable.
- Similar to FR-003, 'top [deferred] most variable genes' contains a placeholder. While the 'Assumptions' section mentions a sensitivity analysis, the FR itself must define the *mechanism* for determining 'top [deferred]'. Is it a fixed number (e.g., 1500) or a dynamic threshold (e.g., top 10% variance)? Without a concrete rule or a defined variable name (e.g., 'N_genes'), the requirement is not implementable or testable.
- The requirement 'p-value < 0.05 to claim statistical significance' is verifiable, but the method for generating the null distribution is critical. The text says 'labels are shuffled strictly within each dataset'. This is a specific procedure. However, the requirement does not specify the number of permutations (e.g., 1000) in the FR itself, only in the User Story. FRs should be self-contained. If the FR relies on the User Story for the permutation count, it is not independently verifiable. The FR must state '1,000 iterations' or reference a constant.
- The requirement 'halt execution if the p-value < 0.05 indicates significant confounding' is verifiable, but the contingency table construction is ambiguous. 'Rows=Batch (Dataset ID) and Columns=Stress Type' implies a specific table structure. However, if a dataset has 0 samples for a stress type, the table is sparse. The requirement must specify how to handle zero-count cells (e.g., 'using Fisher's Exact Test with continuity correction' or 'excluding zero-count cells') to ensure the test is reproducible.
- SC-001 states 'measured against the baseline random chance (equal probability for the number of classes, 1/4 = 0.25)'. This is a definition, not a measurement procedure. It should be rephrased to 'Within-dataset accuracy is measured against the theoretical random baseline of 0.25 (1/4 classes) to determine if the signal is statistically distinguishable from chance (p < 0.05 via permutation test).' The current phrasing is vague on *how* the comparison is made (e.g., is it a simple subtraction or a statistical test?).
- SC-003 'measured against a random permutation baseline' is vague. It does not specify the metric (Jaccard? Overlap count?) or the statistical test. It should be 'Feature importance overlap (Jaccard index) is measured against a null distribution generated by permuting gene labels 1,000 times to calculate a p-value < 0.05.' The current text is not a measurable outcome.
- The spec defines a 'random chance baseline' of 0.25 (1/4 classes) for the accuracy threshold. However, the research question and methodology explicitly require a 'Leave-One-Dataset-Out' (LODO) strategy where the held-out dataset may not contain all 4 stress types (e.g., if a dataset only has drought and heat). The baseline must be dynamic (1/N_classes_in_held_out) or the spec must mandate that only datasets with all 4 classes are used, otherwise the 0.25 threshold is scientifically invalid for partial-class datasets.
- The spec defines an 'Ambiguous' range as (0.25, 0.60). This assumes a fixed 4-class problem. If the held-out dataset has fewer classes (e.g., 2 classes -> 0.50 random chance), the 0.25 lower bound is meaningless. The spec must align the ambiguity threshold with the dynamic random chance baseline of the specific held-out dataset to avoid false 'Ambiguous' flags or missed signals.
- The spec requires a Silhouette Score > 0.4 for 'at least 3 out of 4 stress types'. If the held-out dataset only contains 2 stress types (due to data availability), this requirement becomes impossible to satisfy, causing the pipeline to fail or the result to be invalid. The spec must either enforce that all datasets contain all 4 stress types (FR-001) or make the '3 out of 4' condition dynamic based on the actual classes present in the test set.
- FR-003 states: 'retain only genes present in the intersection of [deferred] of the selected datasets'. The phrase 'intersection of [deferred]' is syntactically ambiguous and does not clearly map to the Constitution Principle VII requirement of 'genes present in ALL integrated datasets'. It should explicitly state 'intersection of ALL selected datasets' to avoid implementation ambiguity regarding the [deferred] placeholder.
- FR-011 mandates a 'label-permutation test (1,000 iterations)' where labels are shuffled 'strictly within each dataset'. The spec does not define the null distribution generation strategy for the LODO cross-dataset accuracy. Shuffling within datasets preserves the batch structure but may not generate a valid null distribution for cross-dataset generalization if the batch effect is the primary driver. The spec needs to clarify if the permutation is applied to the *training* labels before training, or the *test* labels, to ensure the p-value correctly tests the 'generalization' hypothesis against chance.
- FR-015 requires a Fisher's Exact Test for 'confounded batch and condition' and to 'halt execution if p-value < 0.05'. However, the spec does not specify the contingency table construction for datasets with >2 stress types (e.g., 4x4 table). Fisher's Exact Test is computationally intractable for large tables. The spec must define the specific statistical test or approximation (e.g., Chi-squared with Yates' correction, or a permutation-based independence test) suitable for multi-class contingency tables to ensure the requirement is implementable.
- The scenario mentions a 'post-hoc power analysis' but does not specify the input parameters (effect size, alpha, power) or the specific metric (e.g., MDES) to be calculated. While FR-014 mentions MDES, the acceptance scenario should explicitly state that the system reports the MDES value to distinguish 'No Signal' from 'Insufficient Sample Size'.
- The validation target (cross-dataset accuracy) is compared against a random baseline (0.25) but lacks a null distribution derived from the specific data structure. Without a label-permutation test that respects the dataset hierarchy (shuffling labels *within* datasets, not across), the 'accuracy > 0.60' claim cannot distinguish between biological signal and residual batch effects that correlate with stress types in the specific dataset mix. The spec mandates a permutation test in FR-011 but fails to define the null generation strategy in the acceptance criteria, risking a tautological validation where any non-random accuracy is claimed as 'generalization' without proving it exceeds the chance level of the specific dataset configuration.
- The claim that a Jaccard index > 0.30 between within-dataset and cross-dataset rankings validates 'universal stress responders' is scientifically unsound without a stability baseline. In high-dimensional transcriptomics, feature importance is notoriously unstable due to multicollinearity. A Jaccard index of 0.30 could easily arise from random noise or correlated gene modules if the top 50 genes are not independently validated. The spec requires bootstrapping (FR-013) but does not mandate that the Jaccard index itself be compared against a null distribution generated by permuting feature labels or using a stability selection threshold (e.g., selection frequency > 0.7) to distinguish robust biomarkers from noise-driven importance.
- The unsupervised validation relies on a Silhouette Score > 0.4 on uncorrected data. However, in transcriptomic meta-analysis, uncorrected data often clusters by dataset (batch) rather than biology. If the Silhouette Score is calculated on the raw embedding, a high score might indicate strong batch separation (dataset-specific artifacts) rather than stress-type separation. The spec fails to require that the Silhouette Score be calculated *conditional* on stress type while controlling for dataset, or that a 'dataset-label' Silhouette Score be reported to ensure the clustering is not driven by batch effects. Without this, the validation target is confounded with the very artifact (batch) the study aims to distinguish from biology.
- FR-011 mandates a label-permutation test where labels are shuffled 'strictly within each dataset'. While this is the correct approach to preserve batch structure, the spec does not specify how the null distribution is aggregated across the Leave-One-Dataset-Out (LODO) folds. If the permutation is performed only on the training set or only on the test set, the resulting p-value does not validly test the cross-dataset generalization hypothesis. The null distribution must be generated by permuting labels in the *held-out* test set (relative to the training set's learned decision boundary) to assess if the observed accuracy exceeds chance *given the specific training data*. The current phrasing is ambiguous and risks an invalid statistical test.
- The spec mandates normalization via TPM (Transcripts Per Million) for RNA-seq count matrices. However, TPM is a within-sample normalization method that does not correct for library size differences *between* samples in a way that is suitable for cross-dataset comparison of absolute expression levels. For differential expression and cross-dataset classification, methods like TMM (Trimmed Mean of M-values) or DESeq2's median-of-ratios are standard because they account for composition bias. Using TPM across disparate datasets (different species, platforms) without a between-sample normalization step (like quantile normalization or ComBat, which is explicitly avoided) may introduce systematic biases that confound the stress signal with technical artifacts, invalidating the cross-dataset comparison.
- FR-003 requires retaining 'only genes present in the intersection of [deferred] of the selected datasets'. While this aligns with Constitution Principle VII, the spec does not address the biological validity of this intersection. If the selected datasets include different species (e.g., Arabidopsis and Maize), the gene intersection will be limited to highly conserved orthologs, potentially excluding stress-specific genes that are species-specific. The spec assumes the intersection is sufficient for analysis but does not mandate a check for the biological relevance or coverage of the intersection (e.g., 'Does the intersection cover known stress pathways?'). If the intersection is too small or biased towards housekeeping genes, the resulting model may fail to detect stress signatures not because they don't exist, but because the feature space is biologically impoverished.
- FR-015 mandates a Fisher's Exact Test for 'confounded batch and condition' and halts if p < 0.05. However, in a meta-analysis of public datasets, it is highly likely that some datasets will have imbalanced designs (e.g., more drought samples than heat samples). A strict p < 0.05 threshold for confounding may exclude too many datasets, leaving insufficient data for the LODO strategy. The spec does not provide a mechanism to handle partial confounding (e.g., weighting samples or using a less stringent threshold) or to distinguish between 'complete confounding' (one dataset has only one stress type) and 'partial confounding' (imbalanced but present). A strict halt on any significant confounding may render the study infeasible.
