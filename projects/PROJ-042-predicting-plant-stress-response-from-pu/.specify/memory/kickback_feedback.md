## Unresolved concerns

- The plan states: 'A formal a priori power analysis is impossible... The study is observational.' While true that a priori power analysis is hard with fixed public data, the plan lacks a post-hoc power analysis or a minimum detectable effect size (MDES) calculation. Without this, a null result (accuracy ~25%) cannot distinguish between 'no biological signal' and 'insufficient sample size to detect a weak signal'.
- The plan relies on Random Forest's robustness to multicollinearity and explicitly rejects VIF diagnostics. However, in high-dimensional transcriptomics (1500 genes), correlated features can lead to unstable feature importance rankings. The plan proposes Shapley values but does not specify a stability metric (e.g., bootstrapped importance variance) to distinguish robust biomarkers from noise-driven importance.
- The 'leave-one-dataset-out' (LODO) strategy is proposed to test generalization. However, with only ~5 datasets, LODO results will have extremely high variance. The plan does not include a permutation test baseline (shuffling stress labels across datasets) to establish if the observed cross-dataset accuracy is significantly better than chance given the small N of datasets.
- The plan mentions applying Benjamini-Hochberg FDR correction to p-values from permutation tests of accuracy. However, it does not define the null distribution generation strategy for these permutation tests (e.g., permuting labels within datasets vs. across datasets). Permuting across datasets in a LODO setup is methodologically complex and risks breaking the batch structure, potentially invalidating the p-values.
- The plan lists GSE30047, GSE40677, GSE51148, GSE59991, and GSE66904 as 'Plant Stress' datasets but provides no evidence these specific accessions contain the required 4 distinct stress types (drought, salinity, heat, cold) with sufficient sample sizes. GSE40677 is known to be a human cancer dataset (TCGA), not plant stress. Using a human dataset for a plant stress hypothesis is a fundamental dataset mismatch that invalidates the analysis.
- The plan assumes '5 GEO datasets' will cover 'drought, salinity, heat, cold' stress types. Without verifying that each accession contains samples for ALL four stress types (or at least a balanced subset), the 'leave-one-dataset-out' cross-validation for multi-class classification is infeasible. If a dataset only contains 'drought' samples, it cannot be used to test generalization across stress types.
- The plan relies on direct NCBI FTP fetch for datasets without verified URLs. While the resolver guarantees URL reachability, it does not guarantee the dataset content matches the 'Plant Stress' hypothesis. The plan must explicitly confirm that the cited GEO accessions are indeed plant transcriptomics studies before proceeding, as human or animal datasets (e.g., GSE40677) would render the pipeline useless for the stated research question.
- Constitution Principle II (Verified Accuracy) is marked PASS, but the plan explicitly states that for accessions GSE40677, etc., there is 'NO verified source' and the strategy is to 'fetch directly from NCBI GEO via wget/curl without fabricating a HuggingFace URL'. The Constitution requires 'Every external citation... MUST be verified by the Reference-Validator Agent against the primary source'. The plan's fallback strategy (direct FTP) bypasses the verification gate defined in the Constitution, creating a conflict between the plan's execution path and the constitutional requirement for verified citations.
- Constitution Principle VII (Transcriptomic Data Harmonization) states: 'Only genes present in all integrated datasets are retained for modeling'. However, the plan (and spec FR-003) states: 'retain only genes present in the union of ≥80% of the selected datasets'. The plan's 80% threshold contradicts the Constitution's 'all integrated datasets' (100%) requirement. This is a direct violation of the constitutional constraint on feature consistency.
- Constitution Principle V (Versioning Discipline) requires updating the project state file `state/projects/...yaml` upon completion. The plan mentions 'update the project state file' in the summary but does not explicitly detail the mechanism or the specific file path in the 'Technical Context' or 'Project Structure' sections, unlike the explicit path in the Constitution. While likely implied, the plan should explicitly reference the state file path to ensure the 'Versioning Discipline' check is fully satisfied in the design.
- The plan validates 'universal stress signatures' using a 'leave-one-dataset-out' strategy on data that has already undergone 'batch correction' (ComBat/Harmony) in the preprocessing pipeline (Data Model, Data Flow). ComBat explicitly removes dataset-specific variance to align distributions. If the model is trained on batch-corrected data, the 'generalization' it achieves is trivially guaranteed by the algorithm's definition (removing the very artifact the test seeks to measure). This creates a circular validation where the predictor (corrected features) and the validation target (cross-dataset accuracy) are not independent; the correction algorithm forces the target to be predictable. The validation must be performed on data *before* batch correction, or the batch correction must be applied *within* the cross-validation loop (train on uncorrected, correct only the test set relative to train), which is not described.
- The plan relies on Random Forest feature importance to identify 'universal stress responders' (US-2) but explicitly rejects VIF diagnostics or explicit collinearity checks, assuming RF handles it. In transcriptomics, top variable genes are often co-regulated modules (e.g., heat shock proteins). If the 'top 1500' genes are highly correlated, RF importance scores will be distributed arbitrarily among them. Claiming these specific genes are 'biomarkers' based on importance scores without verifying their independence or stability across folds is scientifically unsound; the result is a 'module' signal, not a specific gene signal. The plan must include a stability selection step (e.g., bootstrapping feature importance) to distinguish robust biomarkers from correlated noise.
- The plan proposes using ComBat/Harmony for batch correction on a dataset with only ~5 GEO series. ComBat requires sufficient samples per batch to estimate batch effects reliably. With only 5 datasets, the 'batch' variable is confounded with the specific biological conditions of those few studies. If one dataset contains only 'drought' and another only 'heat', ComBat will remove the biological signal (stress type) along with the batch effect, or fail to converge. The plan lacks a check for 'confounded batch and condition' which is a critical failure mode in small-sample meta-analysis.
