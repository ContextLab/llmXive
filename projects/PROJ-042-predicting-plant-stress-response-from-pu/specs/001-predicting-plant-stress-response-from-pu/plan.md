# Implementation Plan: 001-gene-regulation

**Branch**: `001-gene-regulation` | **Date**: 2026-07-08 | **Spec**: `specs/001-gene-regulation/spec.md`

## Summary

This feature implements a computational pipeline to determine if distinct abiotic stress types (drought, salinity, heat, cold) induce separable transcriptional signatures in plants that generalize across independent public datasets. The approach involves downloading raw RNA‑seq count matrices from NCBI GEO, verifying content (plant species, stress labels), harmonizing gene identifiers, filtering, normalizing, and training a Random Forest classifier. 

**Critical Methodological Updates**:
1.  **Strict Dataset Filtering**: The pipeline fetches **only** verified plant stress datasets (GSE30047, GSE51148, GSE59991, GSE66904). GSE human cancer datasets are explicitly excluded and never fetched.
2.  **Intra-fold Batch Correction**: To prevent circular validation, batch correction (ComBat) is applied **within** the Leave-One-Dataset-Out (LODO) cross-validation loop. Correction parameters are estimated on the training fold and applied to the held-out test fold.
3.  **Global Permutation Baseline**: A null distribution is generated by permuting stress labels *across* datasets (preserving batch structure) to establish if cross-dataset accuracy is significantly better than chance given the small N of datasets.
4. **Constitution Alignment**: Gene retention strictly follows Constitution Principle VII ([deferred] intersection of *included* datasets), overriding the [deferred] union rule in FR-003 (flagged as a spec-root cause error).

All methods are constrained to CPU‑only execution on GitHub Actions free‑tier runners (2 CPU, ~7 GB RAM, ≤6 h).

## Detailed Phase Mapping (FR / SC Coverage)

| Phase | FR / SC | Description |
|-------|----------|-------------|
| 0 – Dataset Download & Verification | FR‑001, FR‑012 | `loader.py` downloads **only** verified plant accessions (GSE30047, GSE51148, GSE59991, GSE66904). Verifies organism = plant, ensures ≥2 stress types, computes SHA‑256 checksum. **GSE40677 is excluded**. |
| 1 – Pre‑processing | FR‑002, FR‑003, FR‑004, FR‑011 | `preprocess.py` filters genes with <1 CPM in <3 samples, TPM normalizes, maps IDs via Biopython. **Retains only genes present in [deferred] of included datasets** (Constitution Principle VII). |
| 1.5 – Label Space Harmonization | (New) | Verifies that the union of included datasets contains all target stress types (drought, salinity, heat, cold). If a dataset lacks a required stress type, it is excluded from the LODO loop. |
| 1.6 – Pre-ComBat Confounding Check | (New) | Runs a chi-square test of independence between Batch (Dataset) and Condition (Stress). If p < 0.01 (high confounding), the pipeline halts with a specific error to prevent biological signal removal. |
| 2 – Feature Selection | FR‑004 | Select the top-ranked most variable genes (variance across all samples). If fewer than 1500 remain, pipeline aborts with “Feature Space Insufficient”. |
| 3 – Model Training & Evaluation | FR‑005, FR‑006, FR‑007, FR‑009, FR‑010 | `rf_classifier.py` runs LODO CV. **Intra-fold Batch Correction**: (a) estimate ComBat parameters on training set, (b) apply to test set, (c) train Random Forest, (d) compute metrics, (e) extract feature importances. **Stability Selection**: 1000 bootstraps, top‑k=50, stability score > 0.8. **Global Permutation Baseline**: Permute labels *across* datasets to establish null distribution. |
| 4 – Unsupervised Validation | FR‑008 | `unsupervised.py` runs UMAP on the **batch‑corrected test set** of each LODO fold, computes Silhouette Score, visualizes clusters. |
| 5 – Post‑hoc Power & MDES | (New) | `utils.metrics.py` bootstraps the observed accuracy to estimate the Minimum Detectable Effect Size at [deferred] power; reports post‑hoc power estimate. |
| 6 – Reporting & State Update | (No explicit FR) | Writes `results/model_metrics.json`, `results/embedding_coords.csv`, updates `state/projects/PROJ-042-predicting-plant-stress-response-from-pu.yaml` with content hashes of all artifacts. |

All functional requirements (FR‑001 – FR‑012) and success criteria (SC‑001 – SC‑005) are covered by the above phases.

## Decision Log

| Decision | Rationale |
|----------|-----------|
| Direct NCBI GEO fetch | No verified HuggingFace URLs; programmatic verification of primary source satisfies Constitution Principle II. |
| [deferred] Gene Intersection | Required by Constitution Principle VII; overrides FR-003 ([deferred] union) which is a spec-root cause error. |
| LODO CV with intra‑fold batch correction | Guarantees genuine cross‑dataset generalization testing (Principle VI) and prevents circular validation. |
| Global Permutation Baseline | Establishes significance of cross-dataset accuracy given small N (N=5) by shuffling labels across datasets. |
| Stability Selection & Stability Score | Provides a robust, quantitative metric for biomarker reliability, addressing collinearity concerns. |
| Post‑hoc power & MDES | Enables interpretation of null results (distinguish “no signal” vs. “underpowered”). |
| Pre-ComBat Confounding Check | Proactive gate to prevent removal of biological signal when batch and condition are aligned. |
| Exclusion of non‑plant or single‑stress datasets | Ensures relevance to the research question and avoids invalid model training. |
