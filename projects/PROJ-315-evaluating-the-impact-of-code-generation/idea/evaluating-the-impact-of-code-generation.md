# Evaluating the Impact of Code Generation on Code Review Quality

**Field**: computer science

## Research question

Does code generated by Large Language Models (LLMs) receive different code review feedback (in terms of comment depth, critical issue identification, and time to merge) compared to human-written code in open-source repositories?

## Motivation

As LLMs become integral to software development, understanding their impact on downstream quality assurance processes is critical. This research addresses the gap in empirical evidence regarding whether automated code generation alters reviewer behavior or exposes different defect patterns, which informs CI/CD pipeline design and tooling adoption.

## Related work

- [DeepCodeSeek: Real-Time API Retrieval for Context-Aware Code Generation (2025)](http://arxiv.org/abs/2509.25716v1) — Provides context on modern code generation techniques that influence the code being reviewed.
- [FRANC: A Lightweight Framework for High-Quality Code Generation (2023)](http://arxiv.org/abs/2307.08220v2) — Establishes baseline metrics and frameworks for assessing the functional quality of generated code.
- [Understanding Code Patterns - Analysis, Interpretation & Measurement (2011)](http://arxiv.org/abs/1106.6159v1) — Offers foundational methodologies for measuring code quality and patterns relevant to review analysis.

## Expected results

We expect to find that LLM-generated code receives a higher volume of superficial comments but fewer deep architectural critiques compared to human code. Confirmation will require statistically significant differences in review sentiment scores and merge latency across a sample of at least 1,000 pull requests.

## Methodology sketch

- **Data Acquisition**: Download a public dataset of GitHub Pull Requests (e.g., via HuggingFace Datasets library or GitHub Archive BigQuery export) containing PR code diffs and associated review comments.
- **Labeling**: Filter commits for LLM-generated content using heuristics (e.g., commit message keywords like "copilot", "generated") or by comparing against known LLM code generation benchmarks.
- **Feature Extraction**: Compute code complexity metrics (cyclomatic complexity, lines of code) using static analysis tools (e.g., `radon`, `lizard`) and review metrics (comment count, sentiment score via `textblob`).
- **Computation**: Perform data aggregation using `pandas` on the local runner (7GB RAM limit); ensure chunking if dataset exceeds memory.
- **Statistical Analysis**: Apply Mann-Whitney U tests to compare review depth and merge time between LLM and human groups, controlling for project size and complexity.
- **Visualization**: Generate summary plots (boxplots, histograms) using `matplotlib` to illustrate distribution differences.
- **Validation**: Run sanity checks on data integrity (missing values, outlier detection) before statistical testing.
