# Data Model: Evaluating the Impact of Code Generation on Code Review Quality

## Entities & Relationships

### Pull Request (PR)
- **Attributes**:
  - `pr_id`: Unique identifier (string)
  - `code_diff`: Full code diff text (string)
  - `review_comments`: List of review comment texts (list of strings)
  - `merge_timestamp`: Timestamp of merge (datetime)
  - `project_metadata`: Repository name, language, etc. (dict)
  - `code_group`: Classification label ("Keyword-Associated" or "Human") (string)
  - `comment_count`: Number of review comments (integer)
  - `sentiment_score`: Average sentiment score of review comments (float)
  - `merge_latency`: Time from PR creation to merge in hours (float)
  - `cyclomatic_complexity`: Average cyclomatic complexity of code files (float)
  - `lines_of_code`: Total lines of code in PR (integer)
  - `classification_accuracy`: Estimated accuracy of heuristic classification (float, from manual audit)

### Code Group
- **Definition**: Binary classification based on commit message keyword matching.
- **Rules**:
  - "Keyword-Associated" if commit message contains ≥2 keywords from predefined list.
  - "Human" otherwise.
- **Keywords**: ["copilot", "generated by", "llm", "ai-generated", "codex", "starcoder"]
- **Note**: This label represents "PRs with LLM-related keywords" and is used as a proxy for "LLM-generated code" due to dataset limitations.

### Review Metrics
- **Derived from**: `review_comments`
- **Metrics**:
  - `comment_count`: len(review_comments)
  - `sentiment_score`: mean(textblob.sentiment.polarity for each comment)

### Complexity Metrics
- **Derived from**: `code_diff`
- **Metrics**:
  - `cyclomatic_complexity`: Mean of radon CC results across files
  - `lines_of_code`: Sum of lines across all files
- **Usage**: Reported descriptively to understand the nature of the code; **NOT** used as a covariate in regression to avoid mediator bias.

## Data Flow

1. **Fetch**: Download raw parquet from HuggingFace.
2. **Preprocess**:
   - Filter records with missing required fields.
   - Compute review metrics (comment count, sentiment).
   - Compute complexity metrics (cyclomatic complexity, lines of code).
   - Classify PRs using keyword heuristics (`code/labeling/classify.py`).
3. **Audit & Calibrate**:
   - Perform manual audit on a sample to estimate `classification_accuracy` and `sentiment_validity_correlation`.
   - Calculate `effective_sample_size` based on noise rate.
4. **Analyze**:
   - Run Mann-Whitney U tests.
   - Run Descriptive Stratification for complexity (no regression control).
   - Perform sensitivity analysis on keyword threshold.
5. **Report**: Generate visualizations and final report with validity margins.

## Data Contracts

- **Input Schema**: Verified via `contracts/dataset.schema.yaml`.
- **Output Schema**: Verified via `contracts/output.schema.yaml`.
- **Checksums**: All raw and processed files checksummed and recorded in state file via `code/utils/hash_artifacts.py`.

## Assumptions

- Commit messages contain sufficient keywords to classify "Keyword-Associated" vs "Human" PRs.
- Review comments are available in the dataset for ≥95% of PRs.
- Code complexity tools can process ≥90% of code files.
- TextBlob scores correlate with human-coded "tone" (correlation factor reported).
