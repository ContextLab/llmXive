\section*{Reproducibility Checklist}
\label{sec:reproducibility}
\begin{itemize}
\item \textbf{Random seeds:} All experiments use fixed random seeds (42) for NumPy, PyTorch, and environment initialization to ensure deterministic behavior where applicable.
\item \textbf{LLM calls:} Temperature is set to 0.0 for policy execution to ensure deterministic outputs. Temperature=0.7 is only used for data augmentation tasks where stochasticity is beneficial.
\item \textbf{Environment resets:} All environment initializations use seeded random number generators for consistent state initialization across runs.
\item \textbf{Policy execution:} Inference is deterministic given fixed seeds and model checkpoints.
\item \textbf{Code and configuration:} All code, configurations, and experiment logs are publicly available at \url{https://github.com/...}. Exact commit hashes for each experiment are documented in the reproducibility appendix.
\end{itemize}

\section*{Limitations}
While \textsc{RATs} is an initial step toward playful agentic robot learning, its evaluation remains primarily simulation-based, requiring larger-scale physical deployment to validate robust sim-to-real transfer. Moreover, play-time skill acquisition is constrained by the diversity of available simulation environments, limiting the range of objects, dynamics, and affordances the agent can practice. Additionally, improper skill reuse can hurt performance when retrieved skills do not fit the downstream task, showing the need for better retrieval and context-aware selection. Finally, \textsc{RATs} increases inference costs, relies heavily on VLM verification, and remains bounded by primitive-level control APIs, which limits dexterous manipulation and motivates lighter feedback mechanisms and richer low-level controllers for real-world scaling.
