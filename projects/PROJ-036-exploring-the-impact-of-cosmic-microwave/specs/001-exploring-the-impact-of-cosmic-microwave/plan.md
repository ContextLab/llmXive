# Implementation Plan: Exploring the Impact of Cosmic Microwave Background Anomalies on Early Universe Simulations

**Branch**: `001-cmb-anomaly-lss-impact` | **Date**: 2026-06-25 | **Spec**: `specs/001-cmb-anomaly-lss-impact/spec.md`
**Input**: Feature specification from `specs/001-cmb-anomaly-lss-impact/spec.md`

## Summary

This feature implements a computational pipeline to investigate how specific Cosmic Microwave Background (CMB) anomalies (Cold Spot, low quadrupole) alter predicted Large-Scale Structure (LSS) statistics. The approach involves downloading validated Planck CMB maps (PR3), generating anomaly-modified initial conditions via CAMB with a **Phase-Injected Mode** strategy (replacing the scientifically unsound global modulation), running paired CPU-tractable N-body simulations (a particle count sufficient for the study's resolution requirements, 250 Mpc/h box) to ensure CI feasibility, and performing **Diagnostic Statistical Comparisons** (KS, Chi-squared) between anomaly and control runs. The implementation strictly adheres to the project's runtime and RAM constraints on GitHub Actions free-tier runners. All statistical outputs are framed as **diagnostic metrics** (descriptive) rather than inferential hypothesis tests due to the N=1 limitation.

## Constitution Check

*GATE: Must pass before Phase 0 research. Re-check after Phase 1 design.*

- **I. Reproducibility**: The plan mandates pinned random seeds in `code/`, checksum validation for all downloaded Planck data (FR-001), and a `requirements.txt` with exact versions. The pipeline is designed to run end-to-end on a fresh CI runner.
- **II. Verified Accuracy**: All citations in `research.md` will be restricted to the "# Verified datasets" block provided in the user message. The plan explicitly avoids inventing dataset URLs. A **Verified Accuracy Gate** (CI step) will validate these citations before allowing progress.
- **III. Data Hygiene**: Raw Planck maps will be checksummed and stored in `data/raw/`. Derived files (initial conditions, snapshots) will be stored in `data/derived/` with explicit derivation logs. No in-place modification of raw data.
- **IV. Single Source of Truth**: All statistical outputs (p-values, delta values) will be generated by code and stored in `data/results/`. The `paper/` will reference these artifacts directly, not hand-typed numbers.
- **V. Versioning Discipline**: Every artifact (map, IC, snapshot, result) will carry a content hash recorded in `state/`. **CI Integration**: The GitHub Actions workflow includes a mandatory step (`code/utils/versioning.py --update-state`) that calculates and writes these hashes to `state/projects/.../artifact_hashes.yaml` before the job completes. This enforces Constitution Principle V.
- **VI. Simulation Initial Condition Integrity**: The plan enforces separate configuration files for the anomaly-modified power spectrum vs. standard ΛCDM. The exact CAMB version and parameter file used will be logged in the simulation metadata.
- **VII. Statistical Significance Adherence**: The plan strictly implements the pre-defined KS and Chi-squared tests. However, due to N=1, the p-values are treated as **diagnostic metrics** (descriptive of deviation magnitude) rather than inferential significance tests. Sensitivity analysis over a range of small values is included as per the spec's assumption, but the primary claim relies on the fixed threshold as a diagnostic cutoff.

## Complexity Tracking

| Violation | Why Needed | Simpler Alternative Rejected Because |
|-----------|------------|-------------------------------------|
| **Phase-Injected Mode** (vs. global modulation) | The Cold Spot is a specific spatial phase configuration, not a global amplitude change. Global modulation fails to test the specific anomaly hypothesis. Phase injection preserves the non-Gaussian spatial structure required for the Cold Spot test. | Global modulation only changes σ8, which is a trivial test of amplitude, not the specific Cold Spot spatial configuration. |
| **Finite Volume Approximation** | The CMB quadrupole (ℓ=2) has a wavelength [deferred] Mpc/h, far exceeding the 250 Mpc/h box. The simulation cannot physically contain this mode. | A full-sky simulation is computationally infeasible. The plan explicitly treats the box modes as the *fundamental periodic modes* and injects the anomaly as a *local gradient approximation* (boundary condition), acknowledging the finite volume confound as a limitation rather than hiding it. |
| **Diagnostic Statistics** (vs. Inferential Tests) | N=1 (single realization) makes p-value calculation for "significance" statistically invalid. | Standard hypothesis testing requires ensembles (N≥5) to estimate variance. With N=1, p-values are purely descriptive metrics of deviation magnitude, not inferential evidence. |
| **Downsized Simulation** (128³, 250 Mpc/h) | 256³/500 Mpc/h exceeds 7GB RAM and 12h runtime on CI. | A smaller box/particle count is required to fit the hardware constraints while still resolving large-scale voids (>10 Mpc/h). |

## Compute Feasibility & CI Integration

- **Hardware**: GitHub Actions free-tier (2 CPU, 7GB RAM, 14GB Disk).
- **Strategy**:
  - **No GPU**: All simulations run on CPU.
  - **Memory Management**: 128³ particles in 250 Mpc/h box fits within 7GB RAM.
  - **Runtime**: Target ≤ 12 hours for paired runs.
  - **Versioning**: The GitHub Actions workflow includes a mandatory step: `python code/utils/versioning.py --update-state`. This script calculates SHA256 hashes for all artifacts in `data/` and `code/` and writes them to `state/projects/.../artifact_hashes.yaml` **before** the job completes. This enforces Constitution Principle V.
  - **Verified Accuracy**: The workflow includes a step `python code/utils/validate_citations.py` that checks all URLs in `research.md` against the `# Verified datasets` block. If any are missing or unreachable, the job fails. This enforces Constitution Principle II.

## Risk Mitigation

- **Data Unavailability**: If Planck PR is unavailable, the system retries a limited number of times with a fixed backoff interval. If still unavailable, the job fails with a clear error.
- **Memory Overflow**: If RAM usage exceeds 6GB, the simulation aborts and logs a "Memory Limit Exceeded" error. The plan assumes a sufficiently large grid resolution is safe, but this is a hard stop.
- **Runtime Exceeded**: If the simulation exceeds a reasonable duration, the GitHub Actions job is terminated. The plan assumes 128³ fits, but this is a hard limit.
- **Scientific Limitation**: The plan explicitly documents the "Finite Volume Confound" and "N=1 Power Limitation" in the `research.md` and `paper/` sections. Results are framed as "Diagnostic Case Study" not "Statistical Discovery".
- **Spec Contradiction**: The spec's Success Criteria (SC-001, SC-002) demand p-values for significance, but the N=1 constraint makes this impossible. The plan implements the output but frames it as a "Diagnostic Metric" and flags this as a "Spec-Root Cause" contradiction requiring a kickback to update the spec text.
