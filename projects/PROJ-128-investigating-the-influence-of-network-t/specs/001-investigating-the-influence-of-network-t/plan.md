# Implementation Plan: Investigating the Influence of Network Topology on Spontaneous Brain Activity Patterns

**Input**: Feature specification from `specs/001-gene-regulation/spec.md`

## Summary

This project investigates whether topological properties of structural brain networks (derived from dMRI) associate with the prevalence, stability, and switching speed of recurrent functional activity patterns (derived from fMRI). The technical approach involves processing preprocessed HCP data to extract graph metrics (global efficiency, clustering, modularity) and dynamic functional states using a **Leave-One-Out (LOO) K-Means** strategy to ensure statistical independence. The implementation adheres to strict CPU-only constraints (GitHub Actions free tier) and explicitly frames all findings as associational, not causal. The study is explicitly framed as a **Methodological Pilot** to validate the pipeline and estimate effect sizes, acknowledging power limitations with a constrained sample size.

## Independence Enforcement Mechanism

To satisfy Constitution Principle VI (Structural-Functional Independence) and address reviewer concerns about circularity:
1.  **Data Loaders**: `preprocess/structural.py` and `preprocess/functional.py` use distinct, isolated data loading classes. No shared state or buffers exist between them.
2.  **Leave-One-Out (LOO) Clustering**: The dynamic state centroids are **NOT** derived from the full cohort. Instead, for each subject `i`, the state centroids are calculated by clustering the windowed matrices of all subjects `j != i`. Subject `i`'s metrics are then derived by assigning their windowed matrices to these `N-1` centroids. This ensures the target variable (dynamic stability) is mathematically independent of the subject's own data.
3.  **No Shared Preprocessing**: Structural matrices are derived from dMRI tractography; functional time series are derived from fMRI BOLD signals. No preprocessing step (e.g., motion correction, normalization) is shared or reused between modalities.

## Complexity Tracking

| Violation | Why Needed | Simpler Alternative Rejected Because |
|-----------|------------|-------------------------------------|
| LOO Clustering | Required to break circular dependency between subject data and state definition. | Cohort-wide clustering (creates tautology where subject defines their own stability). |
| Fixed Threshold (%) | Required to avoid p-hacking in threshold selection. | Selecting threshold post-hoc based on small-worldness (methodologically invalid). |
| Power Limitation Acknowledgement | Required for scientific honesty given N~50. | Claiming high power (methodologically invalid). |
| Permutation Test | Required to validate p-values for small N. | Standard parametric tests only (risk of inflated Type I error). |
