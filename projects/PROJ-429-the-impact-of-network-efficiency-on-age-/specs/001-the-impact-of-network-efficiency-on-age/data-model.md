# Data Model: The Impact of Network Efficiency on Age-Related Changes in Resting-State EEG

## Overview

This document defines the data structures used throughout the project, ensuring consistency across preprocessing, analysis, and reporting. All data flows are traceable to raw sources, and derived artifacts maintain strict versioning via `trace_id`.

## Entity Definitions

### Participant

Represents a single individual in the study.

| Attribute | Type | Description | Source |
|-----------|------|-------------|--------|
| `participant_id` | str | Unique identifier | PhysioNet/EEG dataset |
| `age` | int | Age in years | Demographics |
| `sex` | str | Biological sex (M/F) | Demographics |
| `education_years` | int | Years of formal education | Demographics |
| `cognitive_score` | float | MMSE/MoCA score (or NaN) | Cognitive assessment |
| `cognitive_instrument` | str | Instrument used (e.g., "MMSE", "MoCA") | Cognitive assessment |
| `signal_quality_flag` | str | "Valid", "Low SNR", "Excessive Artifact" | Preprocessing |
| `exclusion_reason` | str | Reason for exclusion (if any) | Preprocessing |
| `trace_id` | str | SHA-256(source_file + code_hash) | Generated by `download.py`/`preprocess.py` |

### NetworkMetric

Computed graph-theoretical metrics for a participant.

| Attribute | Type | Description | Source |
|-----------|------|-------------|--------|
| `participant_id` | str | Foreign key to Participant | Derived |
| `path_length_auc` | float | Area Under Curve of path length (0.05-0.20 density) | Network computation |
| `clustering_coeff_auc` | float | Area Under Curve of clustering coefficient | Network computation |
| `global_efficiency_auc` | float | Area Under Curve of global efficiency (avg of 1/d_ij) | Network computation |
| `local_efficiency_auc` | float | Area Under Curve of local efficiency | Network computation |
| `modularity_auc` | float | Area Under Curve of modularity | Network computation |
| `density_threshold` | float | Threshold used for sensitivity check (0.1) | Network computation |
| `coherence_band` | str | Frequency band used (e.g., "alpha") | Network computation |
| `trace_id` | str | SHA-256(source_file + code_hash) | Generated by `metrics.py` |

### CorrelationResult

Statistical relationship between a metric and a variable.

| Attribute | Type | Description | Source |
|-----------|------|-------------|--------|
| `metric_name` | str | Name of network metric (e.g., "global_efficiency_auc") | Derived |
| `variable_name` | str | Variable tested (age, cognitive_score) | Derived |
| `correlation_coefficient` | float | Spearman r | Statistical analysis |
| `p_value` | float | Uncorrected p-value | Statistical analysis |
| `corrected_p_value` | float | Bonferroni/FDR corrected p-value | Statistical analysis |
| `sample_size` | int | N used in test | Statistical analysis |
| `trace_id` | str | SHA-256(source_file + code_hash) | Generated by `correlation.py` |

### RegressionResult

Results from multiple regression analysis (Cognition ~ Efficiency + Covariates).

| Attribute | Type | Description | Source |
|-----------|------|-------------|--------|
| `outcome_variable` | str | Dependent variable (cognitive_score) | Derived |
| `predictor` | str | Independent variable (network metric) | Derived |
| `coefficient` | float | Regression coefficient | Statistical analysis |
| `std_error` | float | Standard error | Statistical analysis |
| `p_value` | float | p-value for coefficient | Statistical analysis |
| `vif` | float | Variance Inflation Factor | Statistical analysis |
| `adjusted_r2` | float | Adjusted R-squared | Statistical analysis |
| `trace_id` | str | SHA-256(source_file + code_hash) | Generated by `regression.py` |

## Data Flow

1. **Raw Data**: Downloaded from TUH PhysioNet; checksummed; stored in `data/raw/`.
2. **Preprocessed Data**: Filtered, ICA-cleaned, epoching (10s for connectivity); stored in `data/processed/`.
3. **Network Metrics**: Computed from preprocessed data (AUC approach); stored in `data/results/metrics.csv`.
4. **Statistical Results**: Correlation/regression outputs; stored in `data/results/stats.csv`.
5. **Visualizations**: Generated from statistical results; stored in `data/results/plots/`.
6. **State**: Hashes and timestamps stored in `state/version_map.yaml`.

## Constraints

- **Immutability**: Raw data files are never modified; all transformations create new files.
- **Checksums**: Every file in `data/` has a recorded SHA-256 hash in `state/version_map.yaml`.
- **PII Protection**: No personally identifiable information is stored; participant IDs are anonymized.
- **Versioning**: All artifacts carry `trace_id` and content hashes; changes trigger timestamp updates.
- **Traceability**: Every row in output CSVs includes a `trace_id` linking to the exact source file and code block hash.
