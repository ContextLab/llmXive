## 1. Research Question
Does appending machine-generated "Symbolic-Physics" constraints to text prompts significantly improve **2D Geometric Coherence** (defined as alignment between generated image bounding boxes and `pymunk`-derived 2D projections) in images generated by CPU-optimized diffusion models, compared to text-only prompts or length-matched noise controls?

## 2. Methodology Overview

### 2.1 Study Design
- **Type**: Experimental, comparative study.
- **Groups**:
  - **Baseline**: Text-only prompts.
  - **Experimental**: Text + Natural Language descriptors of `pymunk`-derived physics constraints.
  - **Control (Length-Matched)**: Text + Random text of equal length/complexity (to control for prompt length effects).
- **Sample Size**: N=100 scenes per group (Total N=300). *Note: N=500 is the theoretical target but is computationally infeasible on the free-tier; N=100 is the feasible pilot size.*
- **Randomization**: Scenes are randomized into groups; identical random seeds used for generation within matched triplets (Baseline, Exp, Control) where possible.
- **Unit of Analysis**: Individual generated image.
- **Primary Outcome**: "Prompt Adherence Rate" (1 - Violation Rate), where Violation = mismatch between detected 2D box and `pymunk` 2D projection.
- **Statistical Test**: Two-proportion z-test (or Fisher's Exact Test if expected cell counts < 5).

### 2.2 Dataset Strategy
The study does not use an external pre-existing dataset for the *visual* generation task, as the scenes are synthetic and generated from text descriptions.
- **Input Data**: A curated list of 100 complex scene descriptions (e.g., "a cup balancing on a tilted book"). This list is generated/curated by the researcher and stored in `data/raw/scene_descriptions.csv`.
- **Verified Datasets**: No external image datasets are used for training or evaluation. The `YOLOv8n` model is a pre-trained weights file loaded from the `ultralytics` library (standard weights, no external dataset download required for inference).
- **Note**: The "Verified datasets" block in the input message lists text/CUDA datasets which are irrelevant to this specific visual generation study. We will not use them.

### 2.3 Technical Stack & Feasibility (CPU-Only)
- **Physics Simulation**: `pymunk` (pure Python/C extension). Runs on CPU. Complexity: O(N) per scene. Feasible for 100 scenes in < 15 minutes.
- **Image Generation**:
  - **Model**: LCM-LoRA (Latent Consistency Models) with `diffusers`.
  - **Hardware**: GitHub Actions Free Runner (2 CPU, ~7GB RAM).
  - **Optimization**:
    - Use `torch.no_grad()`.
    - Use `model.to("cpu")`.
    - Use `float32` (avoid 8-bit quantization which often requires CUDA).
    - Batch size = 1.
    - Resolution: 512x512.
    - Inference time per image: ~ minutes on CPU (LCM 4-8 steps).
    - Total time estimate: 300 images * 1.5 min = 450 mins (7.5 hours). *Risk*: Slightly exceeds 6h.
    - **Mitigation**: Use `LCM-LoRA` with 4 steps (fastest) and optimize pipeline (e.g., pre-load model, avoid reload). If pilot > 6h, N will be reduced to 80 per group.
    - **Decision**: Proceed with `LCM-LoRA` + `diffusers` on CPU. If runtime exceeds 6h in pilot, N is reduced to 80.
- **Evaluation**:
  - **Detector**: `YOLOv8n` (Nano version). Small, fast, runs on CPU.
  - **Logic**: Rule-based comparison of detected boxes vs. JSON constraints.

### 2.4 Statistical Rigor
- **Power Analysis**:
 - **Goal**: Detect a moderate reduction in violation rate (e.g., from [deferred] to [deferred]).
  - **Alpha**: 0.05.
  - **Power**: With N=100 per group, power is estimated at ~0.45-0.55 for moderate effect sizes. *Limitation*: The study is a pilot; it is underpowered to definitively reject the null hypothesis. The goal is to estimate effect size and feasibility.
  - **Measurement Noise Correction**: The power analysis accounts for the detector's False Negative Rate (FNR). If FNR > 5% (estimated via a small validation set), the effect size assumption is adjusted conservatively (e.g., looking for a larger delta) to avoid overestimating power.
- **Multiple Comparisons**: Three groups (Baseline, Exp, Control). Primary comparison is Exp vs. Baseline. Secondary is Exp vs. Control. Bonferroni correction applied if both are tested (alpha = 0.025).
- **Causal Inference**: Observational regarding the model's internal state. Claims are framed as "association between prompt conditioning and output geometry."
- **Measurement Validity**:
  - **Physics Simulation**: Validated by `pymunk` documentation and logical consistency checks.
  - **Object Detection**: `YOLOv8n` is a standard, validated detector. **Construct Validity**: The metric "2D Geometric Coherence" is explicitly defined as alignment with the `pymunk` 2D projection, not 3D physical reality. We acknowledge that a 3D-correct image might appear as a 2D violation due to perspective, and vice versa. This is a known limitation of the automated pipeline.
  - **Detector Calibration**: A small validation set (N=10) will be manually inspected to estimate FNR. If FNR > 5%, a conservative correction factor will be applied (as per FR-010).

### 2.5 Limitations & Assumptions
- **Circular Validation**: The study measures adherence to the `pymunk` instruction. If the model follows the prompt, it matches the `pymunk` output by definition. This is a study of **Instruction Adherence**, not independent physical truth.
- **2D Projection**: `pymunk` 2D projections may not perfectly match 3D diffusion outputs due to camera angle. A manual spot-check (N=20) will be performed to validate the 2D assumption.
- **Model Fidelity**: LCM-LoRA has lower geometric fidelity than standard SD. The study measures *relative* improvement within the LCM regime.
- **Compute Constraint**: N=100 is the maximum feasible sample size for the free-tier. Power is limited.

## 3. Risk Management
- **Compute Time**: The primary risk is the 6-hour CPU limit.
  - *Mitigation*: Use `LCM-LoRA` with 4 steps. Pilot run with N=10. If > 2 mins/image, reduce N to 80.
- **Memory**: 7GB RAM limit.
  - *Mitigation*: Load model once, process images sequentially. Clear memory after each image.
- **Seed Locking**: Diffusion models may not be perfectly deterministic across prompt changes.
  - *Mitigation*: Use `LCM-LoRA` which is more deterministic. If deviation > 1 pixel, use the fallback mechanism (N=5 candidates, select best match).
- **Dataset Fit**: No external dataset needed; synthetic scenes ensure variable control.
