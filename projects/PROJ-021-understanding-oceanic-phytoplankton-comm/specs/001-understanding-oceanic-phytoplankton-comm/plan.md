# Implementation Plan: Understanding Oceanic Phytoplankton Communities through Remote Sensing and Oceanographic Data

**Branch**: `001-phytoplankton-vlm-analysis` | **Date**: 2026-07-14 | **Spec**: `specs/001-phytoplankton-vlm-analysis/spec.md`
**Input**: Feature specification from `specs/001-phytoplankton-vlm-analysis/spec.md`

## Summary

This project implements a multi-modal analysis pipeline to quantify how oceanographic drivers (temperature, salinity, nutrients) influence phytoplankton distribution. The approach ingests **verified real-world data** from the SeaBASS dataset (which includes satellite ocean color patches and in-situ measurements), aligns them, and trains two models: a Random Forest baseline and a lightweight CLIP-based Vision-Language Model (VLM). The pipeline strictly adheres to CPU-only constraints (limited RAM, bounded runtime) and prioritizes data integrity and statistical rigor.

## Technical Context

**Language/Version**: Python 3.11  
**Primary Dependencies**: `pandas`, `numpy`, `scikit-learn`, `torch` (CPU-only), `transformers` (CPU-optimized), `xarray`, `netCDF4`, `matplotlib`, `seaborn`, `datasets`  
**Storage**: Local file system (`data/`), CSV/NetCDF intermediate artifacts  
**Testing**: `pytest` (contract tests against schema, integration tests for pipeline)  
**Target Platform**: GitHub Actions Free Tier (Linux, 2 CPU, ~7GB RAM)  
**Project Type**: Computational Data Science / Machine Learning Pipeline  
**Performance Goals**: Total runtime ≤ 6 hours, Peak RAM ≤ 7GB, Data alignment missingness ≤ 5%  
**Constraints**: No GPU, no 8-bit quantization, no large-LLM inference, strict memory limits  
**Scale/Scope**: Global ocean basins (subsetted via SeaBASS verification), Several years of monthly composites (subsetted to fit RAM)

> **Dataset Fit Confirmation**: The "Verified datasets" block confirms the availability of the **SeaBASS** dataset on HuggingFace, which contains the necessary multi-modal data: satellite ocean color pixels (derived from MODIS) and in-situ measurements (Chl-a, SST, Salinity). The plan uses this verified source to satisfy FR-001, FR-002, and FR-003 without resorting to synthetic data.

## Complexity Tracking

| Violation | Why Needed | Simpler Alternative Rejected Because |
| :--- | :--- | :--- |
| **VLM Training on CPU** | Required by FR-003 to test novel approach. | A pure RF model would be simpler, but the spec explicitly requires comparing VLM performance to assess if multi-modal architectures offer value in oceanography. |
| **Multi-Source Alignment** | Required by FR-001 (MODIS + Reanalysis + In-situ). | Using only one data source would violate the "Multi-Modal Data Independence" principle and fail to capture the full driver set (temp, salinity, nutrients). |
| **Sampling Strategy** | Required to fit 7GB RAM / 14GB Disk. | Processing full global high-resolution monthly data for extended periods would exceed disk and RAM limits, violating SC-002 and Constitution Principle VII. |
