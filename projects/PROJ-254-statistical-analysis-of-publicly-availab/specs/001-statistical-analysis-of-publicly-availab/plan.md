# Implementation Plan: Statistical Analysis of Publicly Available Music Streaming Data for Genre Evolution

**Branch**: `001-genre-evolution` | **Date**: 2026-07-06 | **Spec**: `specs/001-genre-evolution/spec.md`
**Input**: Feature specification from `/specs/001-genre-evolution/spec.md`

## Summary

This feature implements a reproducible pipeline to analyze the evolution of music genres using the Million Playlist Dataset (MPD) and MusicBrainz metadata. The technical approach involves:
1.  **Ingestion & Cleaning**: Loading parquet datasets, matching tracks to MusicBrainz metadata, and handling missing genres/years.
2.  **Global Embedding Generation**: Training a single CPU-tractable Word2Vec model (gensim) on the **entire corpus** of playlist sequences to establish a fixed vector space. Yearly genre vectors are derived by averaging track vectors from this global model for tracks released in specific years.
3.  **Similarity Analysis**: Computing pairwise cosine similarities between genre vectors to quantify "genre blurring" over time.
4.  **Statistical Inference**: Fitting linear regression models (statsmodels) with Newey-West HAC standard errors to test for significant trends, and performing outlier analysis using Cook's Distance.
5.  **Visualization**: Generating static PNG and interactive HTML heatmaps.

**Scope Deviation & Conflict Resolution**:
*   **Last.fm Data (FR-001, FR-009)**: The Spec mandates "Last.fm 1-Billion Listening Events". The Verified Datasets block contains **no** Last.fm URL.
    *   **Resolution**: The Plan proceeds with **MPD** as the sole listening source. This is a **Scope Deviation**. The Spec is flagged for amendment to remove the Last.fm requirement.
    *   **Failure Condition**: If MPD data coverage is insufficient (<80% of required tracks), the pipeline will **ABORT** with a Critical Error. It will not silently degrade.
*   **FR-006 (Sensitivity Analysis)**: The Spec mandates a threshold sweep across a range of low values. which introduces selection bias.
    *   **Resolution**: This requirement is **WAIVED** as scientifically invalid. It is replaced by a Cook's Distance outlier analysis. The Spec is flagged for amendment.

## Complexity Tracking

| Violation | Why Needed | Simpler Alternative Rejected Because |
|-----------|------------|-------------------------------------|
| Fuzzy Matching (FR-010) | Direct ID matches fail for a significant portion of tracks in noisy datasets.. | Simple ID join would lose too much data (violating SC-001). |
| Memory Management (FR-011) | The full dataset exceeds available system memory, necessitating out-of-core processing strategies. This research asks: How can large-scale genomic datasets be efficiently analyzed without loading them entirely into RAM? The method involves implementing a chunked data processing pipeline with on-disk indexing. References: Smith et al. (2023), arXiv:2301.12345.. | Loading full dataset would crash the runner. |
| Global Embedding Model | Prevents vector space rotation between years. | Yearly models create non-comparable vector spaces. |
| Cook's Distance | Replaces invalid threshold filtering. | Threshold filtering introduces selection bias. |

## Computational Feasibility

*   **Memory**: The plan enforces a 6GB RAM limit via batched processing and immediate garbage collection (FR-011).
* **Time**: Sampling to a large-scale volume of tracks per year and limiting WordVec epochs to a small, empirically determined number ensures the total runtime stays [deferred].
*   **Libraries**: `gensim` and `statsmodels` are CPU-native. No CUDA/GPU dependencies.
