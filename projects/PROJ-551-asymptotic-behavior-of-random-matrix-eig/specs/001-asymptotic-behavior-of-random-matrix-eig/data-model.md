# Data Model: Asymptotic Behavior of Random Matrix Eigenvalues with Sparse Perturbations

## Overview

This document defines the data structures for the project, ensuring alignment with the `spec.md` requirements (FR-001 to FR-010) and the project constitution (Principles III, IV, VI). All data is synthetic, generated by the simulation engine.

## Entities

### 1. SimulationRun
Represents a single execution of the simulation for a specific set of parameters.

| Field | Type | Description | Source |
| :--- | :--- | :--- | :--- |
| `run_id` | `string` | Unique UUID for the run. | Generated |
| `timestamp` | `iso8601` | Time of execution. | System |
| `seed` | `integer` | Random seed used for reproducibility. | Config |
| `N` | `integer` | Matrix dimension ($100 \le N \le 2000$). | Input |
| `k` | `integer` | Perturbation rank ($1, 2, 5$). | Input |
| `theta` | `float` | Spectral norm of perturbation ($1.0 \le \theta \le 3.0$). | Input |
| `pattern_type` | `string` | "diagonal", "block-sparse", "random-sparse". | Input |
| `support_density` | `float` | $p$ (for random sparse). Null for others. | Input |
| `computation_time_ms` | `float` | Time taken for eigenvalue computation. | Metric |
| `empirical_bulk_edge` | `float` | The 99th percentile of the computed spectrum (Tracy-Widom adjusted). | Computed |

### 2. EigenvalueResult
Stores the computed eigenvalues and outlier status for a `SimulationRun`.

| Field | Type | Description | Source |
| :--- | :--- | :--- | :--- |
| `run_id` | `string` | FK to `SimulationRun`. | Generated |
| `eigenvalue_index` | `integer` | Rank of eigenvalue (1 = largest). | Computed |
| `value` | `float` | The eigenvalue $\lambda$. | Computed |
| `is_outlier` | `boolean` | True if $\lambda > \text{empirical\_bulk\_edge} + 0.05$. | Derived |
| `theoretical_bbp` | `float` | Expected outlier location $\theta + 1/\theta$ (if $\theta > 1$). | Derived (Reference Only) |
| `deviation` | `float` | $\lambda - \text{theoretical\_bbp}$. | Derived |

### 3. AggregatedThreshold
Summary statistics for a specific configuration ($N, k, \text{pattern}, \theta$).

| Field | Type | Description | Source |
| :--- | :--- | :--- | :--- |
| `config_id` | `string` | Hash of ($N, k, \text{pattern}, \theta$). | Derived |
| `N` | `integer` | Dimension. | Input |
| `k` | `integer` | Rank. | Input |
| `pattern_type` | `string` | Pattern. | Input |
| `theta` | `float` | Norm. | Input |
| `total_runs` | `integer` | Number of simulations (target 100). | Count |
| `outlier_count` | `integer` | Number of runs with at least one outlier. | Count |
| `outlier_probability` | `float` | `outlier_count / total_runs`. | Derived |
| `mean_deviation` | `float` | Mean of `deviation` for outlier runs. | Derived |
| `std_deviation` | `float` | Std dev of `deviation`. | Derived |
| `theta_c_estimate` | `float` | Estimated critical threshold from sigmoid fit. | Derived |
| `theta_c_confidence` | `float` | Confidence interval width for $\theta_c$. | Derived |

## Data Flow

1. **Generation**: `SimulationRun` parameters are generated by `main.py`.
2. **Computation**: `eigen_solver.py` computes eigenvalues, producing `EigenvalueResult`.
3. **Aggregation**: `threshold_sweep.py` aggregates results into `AggregatedThreshold`, including curve fitting for $\theta_c$.
4. **Storage**: All artifacts written to `data/processed/` as Parquet/CSV with checksums.

## Constraints & Validations

- **N Range**: $100 \le N \le 2000$.
- **Theta Range**: $1.0 \le \theta \le 3.0$.
- **Outlier Logic**: Must strictly follow FR-004 ($> \text{empirical\_bulk\_edge} + 0.05$).
- **Reproducibility**: `run_id` must be deterministic given `seed` and inputs.
- **Numerical Stability**: Eigenvalues must be real (within tolerance $1e-10$).
- **Structural Fidelity**: `pattern_type` and `support_density` must be recorded to satisfy Constitution Principle VII.
