## Summary

This feature implements the **Foundation Protocol**, a middleware coordination layer designed to improve the efficiency and robustness of heterogeneous autonomous agents in a simulated multi-agent ecosystem. The technical approach involves:
1.  Building a Python middleware library (`foundation_protocol/`) that wraps message passing with signing, routing, and checkpointing.
2.  Implementing three benchmark environments: Hanabi (cooperative), SPEAR-style auditing (fault-tolerance), and Resource Allocation (scalability).
3.  Creating two agent types: a **pre-trained** PPO learner (using `stable-baselines3` CPU mode, inference-only) and a rule-based planner.
4.  Running paired experiments comparing **Foundation Protocol** (Middleware + Optimized Logic) vs. **Native Direct Communication** (Optimized Logic, No Middleware) over 30 random seeds.
5.  Performing statistical analysis (Linear Mixed-Effects Models, McNemar's test) with Bonferroni correction.

**Critical Design Decision**: To isolate the cost of the middleware layer, the **Baseline** condition uses the **same optimized binary serialization and agent logic** as the Intervention, but executes message passing via direct function calls (bypassing the middleware routing/signing layer). This prevents the "tautology" where the middleware is present in both arms and ensures the comparison validly measures the overhead of the protocol layer itself. The baseline is explicitly **Native Direct Communication**, not a "Legacy Protocol" with its own overhead.

The implementation strictly adheres to the **CPU-only constraint** (no GPU, no large LLMs) and targets execution within 6 hours on a GitHub Actions free-tier runner.

**Language/Version**: Python 3.10  
**Primary Dependencies**: `pettingzoo`, `stable-baselines3` (CPU build, inference-only), `numpy`, `pandas`, `scipy`, `ed25519`, `pyyaml`, `jinja2` (for report generation), `statsmodels` (for LMM).

**Testing**: `pytest` with `pytest-cov`, `pytest-randomly`, and `jsonschema` for contract validation.  
**Target Platform**: Linux (GitHub Actions runner).  
**Project Type**: Research Simulation / Library.  
**Performance Goals**: Run 30 seeds √ó 3 tasks √ó 2 protocols within ‚â§ 6 hours. Memory usage < 7 GB.  
**Constraints**: No GPU; no deep learning training from scratch (use pre-trained PPO); deterministic seeds; CPU-only cryptographic signing.  
**Scale/Scope**: benchmark tasks, 3 agent types, 30 seeds per condition.  
**Baseline Protocol**: **Native Direct Communication**. The comparison is strictly between "Middleware + Agent" vs. "Agent with Direct Calls". Legacy protocols (MCP, A2A) are mentioned as context but are **not** used as the statistical baseline to avoid confounding overhead.

## Complexity Tracking

| Violation | Why Needed | Simpler Alternative Rejected Because |
|-----------|------------|-------------------------------------|
| **3 Benchmark Tasks** | Required by US-3 to measure scalability across different interaction patterns (cooperative, auditing, resource). | Using only Hanabi would fail to validate robustness (US-2) and bandwidth efficiency (US-3) across diverse workloads. |
| **Native Direct Baseline** | Required to isolate middleware overhead. The Baseline uses the same optimized logic but bypasses the middleware. | Using a "Legacy Protocol" with its own overhead would confound the comparison (Protocol Logic vs. Logic Quality vs. Protocol Overhead). |
| **Statistical Sensitivity Sweep** | Required by FR-007 to demonstrate robustness of findings. | A single alpha threshold (0.05) is insufficient to prove stability of results under varying significance levels. |
| **Linear Mixed-Effects Models** | Required to handle agent heterogeneity and interaction effects. | Simple paired t-tests would mask heterogeneity and violate independence assumptions. |

## Implementation Phases

### Phase 0: Research & Dataset Verification
1.  **Verify Datasets**: Confirm availability of `pettingzoo` (Hanabi) and verify `code/benchmarks/resource_alloc_runner.py` (IRM4MLS implementation) and `code/data/generate_spear.py` (SPEAR) as the verified artifacts in `data/verified_datasets.yaml`. Specifically, identify and verify the open-source implementation of the IRM4MLS methodology.
2.  **Protocol Analysis**: Review open-source implementations of MCP v1 to understand the contrast with Native Direct Communication.
3.  **CPU Feasibility Check**: Validate that `stable-baselines3` PPO **inference** (not training) fits within 7GB RAM and 6h runtime on 2 cores.

### Phase 1: Data Model & Contracts
1.  Define `MessageEnvelope` schema (JSON/Protobuf).
2.  Define `MetricRecord` schema for CSV output.
3.  Generate `contracts/*.schema.yaml` files.
4.  Draft `quickstart.md` for environment setup.

### Phase 2: Core Implementation
1.  Implement `foundation_protocol/middleware.py` (routing, signing, checkpointing).
2.  Implement `foundation_protocol/direct_comm.py` for Native Direct Communication (Baseline).
3.  Implement `agents/` (Pre-trained PPO, Rule-based, Heuristic).
4.  Implement `benchmarks/` runners for Hanabi, SPEAR, Resource Allocation.
5.  Implement `experiments/crash_injector.py` (FR-004).
6.  Implement `code/benchmarks/resource_alloc_runner.py` (IRM4MLS simulation).
7.  Implement `code/data/generate_spear.py` (SPEAR synthetic data).
8.  **Verify Logic Equivalence**: Run `test_direct_vs_middleware.py` to ensure `foundation_protocol/direct_comm.py` (Baseline) and `foundation_protocol/middleware.py` (Intervention) execute identical agent logic, differing only in the communication layer.
9.  **Artifact Hashing**: Implement `scripts/hash_artifacts.py` to generate and persist artifact hashes in `state/` (Constitution Principle V).

### Phase 3: Experimentation & Analysis
1.  Write `run_simulation.py` to orchestrate 30 seeds √ó 2 protocols (Foundation vs. Native Direct).
2.  Implement `stats_analyzer.py` (LMM, McNemar, Bonferroni, Cohen's d).
3.  **Validate Outputs**: Run `pytest` with schema validation against `contracts/metrics.schema.yaml` and `contracts/dataset.schema.yaml` (Plan Consistency).
4.  Execute full experiment suite.
5.  Generate `make report` (LaTeX PDF + Zenodo archive).

### Phase 4: Verification & Reporting
1.  Run reproducibility check (re-run 1 seed, verify checksums).
2.  Validate all citations against primary sources using `data/verified_datasets.yaml`.
3.  Finalize `plan.md` and `research.md`.
