# Implementation Plan: Evaluating the Robustness of Common Statistical Tests to Non-Independence in Public Datasets

**Branch**: `001-evaluating-robustness-tests` | **Date**: 2023-10-27 | **Spec**: `spec.md`

## Summary

This project implements a Monte Carlo simulation framework to quantify the inflation of Type I error rates and the reduction of statistical power for common tests (t-test, ANOVA, chi-squared) when applied to public datasets with injected non-independence structures (AR(1), block bootstrap, spatial). The approach prioritizes CPU-only execution on GitHub Actions free-tier runners (limited cores, constrained RAM) by using vectorized NumPy operations, stratified sampling, and strict memory management. The plan ensures full coverage of all Functional Requirements (FR-001 to FR-008) and Success Criteria (SC-001 to SC-005) by mapping every requirement to a specific simulation phase and output artifact.

**Critical Methodological Update**: To address scientific soundness concerns, the null hypothesis construction now follows a strict "Generate-then-Inject" paradigm. We generate synthetic data under the true null (independence) first, then inject dependency structures. This ensures the null is true by construction while the i.i.d. assumption is violated, preventing tautological error rate estimates. The previous "Inject-then-Permute" approach described in earlier drafts is explicitly rejected as it fails to generate a valid null distribution under non-independence.

## Technical Context

**Language/Version**: Python 3.10+  
**Primary Dependencies**: `numpy`, `scipy`, `pandas`, `statsmodels`, `matplotlib`, `seaborn`, `pyyaml`  
**Storage**: Local file system (`data/`, `results/`) with checksum verification  
**Testing**: `pytest` (unit tests for injection logic, integration tests for pipeline)  
**Target Platform**: Linux (GitHub Actions `ubuntu-latest`)  
**Project Type**: Computational research / CLI pipeline  
**Performance Goals**: Complete 10,000 replications per configuration within 6 hours; memory usage < 6GB peak.  
**Constraints**: No GPU; no deep learning; strict adherence to verified dataset URLs; deterministic random seeds.  
**Scale/Scope**: test types × dependency structures × dependency strengths × datasets (plus synthetic nulls) = ~ simulation configurations (plus power analysis variations).
