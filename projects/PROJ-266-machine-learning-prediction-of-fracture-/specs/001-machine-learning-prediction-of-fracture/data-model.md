# Data Model: Machine Learning Prediction of Fracture Toughness from Microstructure Images

## Entity Definitions

### MicrostructureImage
Represents a single micrograph of a metallic alloy.
*   `image_id`: Unique identifier (string).
*   `file_path`: Path to the raw image file (string).
*   `alloy_family`: Categorical label ("steel", "al", "ti").
*   `original_dimensions`: Tuple (width, height).
*   `processed_path`: Path to the 128x128 grayscale image.
*   `intensity_mean`: Float (mean pixel value after normalization).
*   `intensity_std`: Float (std dev of pixel value).
*   `synthetic_source`: Boolean (True if generated by `synthetic_gen.py`).

### FractureToughnessRecord
Links an image to its mechanical property.
*   `record_id`: Unique identifier (string).
*   `image_id`: Foreign key to `MicrostructureImage`.
*   `k_ic_value`: Float (Fracture toughness in MPa√m).
*   `source`: String (e.g., "Synthetic Generator", "User Provided").

### ModelEvaluationResult
Output of a model run.
*   `run_id`: Unique identifier (string).
*   `model_type`: String ("CNN", "LinearRegression", "RandomForest").
*   `seed`: Integer (Random seed used).
*   `r_squared`: Float.
*   `mae`: Float (Mean Absolute Error).
*   `rmse`: Float (Root Mean Squared Error).
*   `alloy_family_metrics`: Dictionary (e.g., `{"steel": {"r2": 0.8, "mae": 1.2}, ...}`).
*   `permutation_statistic`: Float (Permutation test statistic).
*   `permutation_p_value`: Float (Permutation test p-value).

### AttributionMap
InputXGrad output.
*   `attribution_id`: Unique identifier.
*   `image_id`: Foreign key.
*   `heatmap_path`: Path to the saved heatmap image.
*   `overlay_path`: Path to the image with heatmap overlay.
*   `augmentation_seed`: Integer.
*   `iou_score`: Float (Intersection over Union with other augmentations).

## Data Flow

1.  **Ingestion**: Raw images + CSV $\to$ `data/raw/`. Checksums generated. (Or Synthetic Data Generation).
2.  **Preprocessing**: Raw $\to$ `data/processed/`. Images resized, grayscale, normalized. Split metadata generated.
3.  **Feature Extraction**: `data/processed/` $\to$ Numpy arrays (for baselines). Features are automatically extracted from raw pixels.
4.  **Training**: `data/processed/` + Seeds $\to$ Model Weights + `ModelEvaluationResult` (JSON).
5.  **Attribution**: `data/processed/` + Model Weights $\to$ `data/explainability/` (Heatmaps + IoU).
