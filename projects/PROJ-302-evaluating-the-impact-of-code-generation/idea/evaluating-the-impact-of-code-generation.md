# Evaluating the Impact of Code Generation on Code Review Time: A Large-Scale Study

**Field**: computer science

## Research question

Does code generated by large language models (LLMs) require significantly more review time than human-written code when controlling for code complexity and file size?

## Motivation

LLM-powered code generation tools are rapidly adopted in software development, yet their downstream effects on code review efficiency remain poorly understood. If LLM-generated code consistently requires longer review times, claimed productivity gains may be offset by increased review overhead. This study addresses that gap by quantifying the review-time differential at scale.

## Expected results

We expect LLM-generated code to require 10–30% longer review times compared to matched human-written code, primarily due to subtle semantic errors and style inconsistencies. Statistical significance will be confirmed using paired t-tests or Mann-Whitney U tests with p < 0.05. Effect sizes (Cohen's d) will quantify the practical magnitude of the difference.

## Methodology sketch

- **Data acquisition**: Download GitHub repository metadata and commit history via GitHub REST API (https://api.github.com). Focus on repositories with ≥1,000 stars and ≥100 pull requests for sufficient review activity data.
- **LLM code injection**: Use a public code-generation model (e.g., CodeLlama from HuggingFace) to generate equivalent code snippets for a stratified random subset of commits (n ≈ 500). Store synthetic commits separately to avoid polluting production data.
- **Feature extraction**: For each pull request, extract review timestamps (first comment, first approval, merge time), file size (lines of code), cyclomatic complexity (via `radon` Python package), and author metadata.
- **Matching strategy**: Pair LLM-generated code commits with human-written commits using propensity score matching on file size, complexity, and repository activity level to reduce confounding.
- **Statistical analysis**: Compute mean/median review time per pair. Apply paired t-test if normality holds (Shapiro-Wilk test), otherwise Mann-Whitney U test. Calculate 95% confidence intervals and effect sizes.
- **Sensitivity analysis**: Repeat analysis across multiple repositories and code complexity tiers to assess generalizability.
- **Visualization**: Generate box plots and CDF curves of review-time distributions for human vs. LLM-generated code using `matplotlib`/`seaborn`.
- **Reproducibility**: Package all scripts, data processing pipelines, and analysis notebooks in a GitHub repository with a `requirements.txt` and Dockerfile for environment replication.
