# Data Model: Investigating the Relationship Between Brain Network Topology and Susceptibility to Visual Illusions

## 1. Entity-Relationship Overview

The data model is designed to support the pipeline from raw acquisition to statistical reporting.

### Entities
1.  **Subject**: The core unit of analysis.
2.  **RawFMRI**: The raw NIfTI file and metadata.
3.  **PreprocessedFMRI**: The output of fMRIPrep.
4.  **TimeSeries**: The extracted BOLD signal matrix.
5.  **ConnectivityMatrix**: The functional connectivity matrix.
6.  **TopologyMetrics**: The computed graph theory metrics (raw).
7.  **TopologyComponents**: The PCA-reduced orthogonal components (derived).
8.  **IllusionScore**: The behavioral scores.
9.  **AnalysisResult**: The final correlation statistics.

## 2. Schema Definitions

### Subject
*   `subject_id` (string, PK): Unique identifier (e.g., "sub-001").
*   `study_id` (string): External ID linking to behavioral data.
*   `age` (int): Optional demographic.
*   `sex` (string): Optional demographic.
*   `exclusion_reason` (string): If excluded, reason (e.g., "Motion > 0.5mm").

### PreprocessedFMRI
*   `subject_id` (string, FK): Links to Subject.
*   `file_path` (string): Path to preprocessed NIfTI.
*   `confounds_path` (string): Path to confounds TSV.
*   `mean_fd` (float): Mean Framewise Displacement.
*   `status` (string): "success" | "failed".

### TimeSeries
*   `subject_id` (string, FK): Links to Subject.
*   `roi_count` (int): Should be 200.
*   `timepoints` (int): Number of volumes.
*   `matrix_path` (string): Path to `.npy` or `.csv` matrix.

### ConnectivityMatrix
*   `subject_id` (string, FK): Links to Subject.
*   `matrix_path` (string): Path to 200x200 matrix.
*   `threshold` (float): Correlation threshold used.

### TopologyMetrics
*   `subject_id` (string, FK): Links to Subject.
*   `modularity` (float): [0, 1].
*   `path_length` (float): > 0. `NaN` if disconnected.
*   `clustering_coefficient` (float): [0, 1].
*   `global_efficiency` (float): > 0.
*   `small_worldness` (float): > 0.
*   `computation_timestamp` (datetime).

### TopologyComponents
*   `subject_id` (string, FK): Links to Subject.
*   `components` (list of objects): A dynamic list of PCA components retained for this subject.
    *   `component_index` (int): The index of the component (0, 1, 2...).
    *   `component_value` (float): The score of the subject on this component.
    *   `variance_explained` (float): The proportion of variance explained by this component (stored globally or per run).
    *   *Note*: The semantic label (e.g., "Integration") is assigned dynamically by the analysis script based on loadings, not stored as a fixed field here.

### IllusionScore
*   `subject_id` (string, FK): Links to Subject.
*   `muller_lyer_score` (float): Susceptibility metric.
*   `ponzo_score` (float): Susceptibility metric.
*   `task_duration_sec` (float).
*   `completion_date` (datetime).

### AnalysisResult
*   `component_id` (int): The PCA component index tested.
*   `illusion_type` (string): e.g., "muller_lyer", "ponzo".
*   `correlation_coef` (float).
*   `p_value` (float).
*   `p_value_fdr` (float).
*   `effect_size_r` (float).
*   `is_significant` (boolean): True if `p_value_fdr` < 0.05.

## 3. Data Flow & Constraints

1.  **Ingestion**: Raw data is checksummed upon download from OpenNeuro.
2.  **Preprocessing**: `PreprocessedFMRI` is generated from `RawFMRI`. `mean_fd` must be < 0.5 for inclusion in analysis.
3.  **Extraction**: `TimeSeries` is derived from `PreprocessedFMRI` using Schaefer 200 atlas.
4.  **Computation**: `ConnectivityMatrix` and `TopologyMetrics` are derived from `TimeSeries`.
5.  **Dimensionality Reduction**: `TopologyComponents` are derived from `TopologyMetrics` via PCA. The number of components is dynamic (retaining >80% variance).
6.  **Behavioral**: `IllusionScore` is extracted from OpenNeuro behavioral files and linked by `subject_id`.
7.  **Analysis**: `AnalysisResult` is generated by joining `TopologyComponents` and `IllusionScore` on `subject_id`. Subjects with missing data in either column are dropped for that specific test.

## 4. Edge Case Handling

*   **Disconnected Graph**: If `path_length` is infinite, it is stored as `NaN`. The PCA and correlation analysis for that subject excludes this metric (or imputes if appropriate, but exclusion is preferred).
*   **Missing Behavioral Data**: If `IllusionScore` is missing for a subject with valid fMRI, the subject is excluded from the correlation table but logged in `preprocessing_log.csv`.
*   **Single Module**: If `modularity` is 0, it is recorded as 0.0. No artificial partitioning is applied.
*   **Variable Component Count**: The system must handle subjects where PCA yields 2 components vs. 3 components. The `components` list in `TopologyComponents` and the `dataset_schema` will reflect the actual count for each subject.
