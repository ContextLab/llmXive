# Implementation Plan: Predicting Individual Differences in Sensory Processing Speed from Resting‑State EEG Power Spectra

**Branch**: `001-predict-sensory-speed-from-eeg` | **Date**: 2026-06-25 | **Spec**: `spec.md`

## Summary

This feature implements a computational pipeline to predict individual differences in sensory processing speed (proxied by median reaction time) from resting-state EEG power spectra. The approach involves downloading and preprocessing public EEG and behavioral data, extracting canonical frequency band power features (delta, theta, alpha, low-beta, high-beta, gamma), and fitting linear regression and LASSO models with 5-fold cross-validation. The plan strictly adheres to CPU-only constraints for GitHub Actions free-tier runners and addresses every Functional Requirement (FR) and Success Criterion (SC) in the specification.

**CRITICAL DATA FEASIBILITY WARNING**: The spec assumes a single dataset containing both resting-state EEG and simple reaction-time (RT) tasks for the same participants. The verified sources (PhysioNet EEG and 'RTs Cleaned') are separate and likely contain different cognitive tasks (Motor Imagery vs. Simple RT). **This plan includes a mandatory Phase 0.5 Data Feasibility Gate.** If the datasets cannot be linked by valid participant IDs AND demographic/task metadata, or if the cognitive states do not align, the pipeline will **HALT** and generate a formal `data/processed/feasibility_report.md` stating the hypothesis cannot be tested with the current verified data. No modeling will proceed on mismatched or disjoint data.

## Phase Breakdown

### Phase 0: Data Acquisition & Integrity (FR-001)
*   **Action**: Download raw EEG and behavioral data from verified sources.
*   **Constraint Check**: Verify file integrity (checksums) before processing.
*   **Output**: Raw files in `data/raw/`.

### Phase 0.5: Data Feasibility Gate (CRITICAL)
*   **Action**: Attempt to join EEG and RT datasets on `participant_id`.
*   **Action**: Verify that the RT dataset contains a **simple reaction-time task** (not motor imagery) and that demographic metadata (age, sex) matches between sources to prevent spurious ID collisions.
*   **Decision**: If no matched participants are found, or if the cognitive states are incompatible, **HALT** the pipeline and generate `data/processed/feasibility_report.md` stating the hypothesis cannot be tested with the current verified data.
*   **Output**: `data/processed/feasibility_report.md` (if halted) or proceed to Phase 1.

### Phase 1: Preprocessing & Feature Extraction (FR-002, FR-003, FR-004, FR-010)
*   **Action**: Apply band-pass (1-40Hz) and notch filters. Reject channels with variance > 3 SD. Exclude participants if the **ratio of rejected channels (channels_rejected / total_channels) > 0.30**, matching the invariant in `data-model.md`.
* **Action**: Compute Welch's PSD on continuous 5-minute epochs using **2-second windows** with **[deferred] overlap** (per Constitution Principle VI). The **overlap parameter is [deferred]** in the spec but fixed here to [deferred] for standardization; code will default to [deferred] unless overridden by config.
*   **Action**: Extract median RT, exclude outliers (<100ms, >2000ms), exclude participants if <70% trials remain.
*   **Action**: Calculate relative power (band/total) and **apply Centered Log-Ratio (CLR) transformation** to handle compositional data constraints before modeling.
*   **Output**: `data/processed/features.csv` (with CLR-transformed features).

### Phase 2: Primary Modeling & Hypothesis Testing (FR-005, FR-006, FR-007, FR-011, FR-012)
*   **Action**: **Calculate Minimum Detectable Effect Size (MDES)** for the observed N to detect R²=0.10 with **power ≥ 0.80**. Report this alongside results. If N is insufficient, flag results as exploratory.
*   **Action**: Fit Linear Regression and LASSO (k-fold cross-validation, lambda tuned for min RMSE). **Calculate and log Adjusted R²** for both models to `data/processed/model_results.json`. **Log optimal lambda value** to ensure reproducibility (Constitution Principle IV).
*   **Action**: Perform Pearson correlations with Bonferroni correction across multiple frequency bands.
*   **Action**: Run a permutation test for R² significance. **Method**: Shuffle labels **only on the held-out test set** to validate predictive validity without data leakage. Acknowledge that small test set N may reduce permutation power.
*   **Action**: Explore non-linear interactions (polynomial terms for alpha and beta, of a specified degree). **Compare Adjusted R² using an F-test**. **Output**: A specific `data/processed/non_linear_comparison.json` table listing linear vs. polynomial R², F-statistic, and p-value to satisfy FR-012.
*   **Output**: `data/processed/model_results.json`, `data/processed/correlations.csv`, `data/processed/non_linear_comparison.json`.

### Phase 3: Robustness & Sensitivity (FR-008, FR-009)
*   **Action**: Re-run pipeline with 2s windows (if primary used 4s) and without ICA. Compare R² stability.
*   **Action**: Sweep p-value threshold (low to high) for correlation significance counts.
*   **Output**: `data/processed/robustness_report.csv`, `data/processed/sensitivity_plot.png`.

### Phase 4: Reporting (SC-001 to SC-005)
*   **Action**: Aggregate all metrics into a final summary report.
*   **Output**: `data/processed/final_report.md`.

## Compute Feasibility Note
The plan explicitly avoids GPU-dependent libraries. `mne` and `scikit-learn` are used in CPU-only mode. Data is processed in chunks if necessary to stay within 7 GB RAM. A large-scale permutation test is optimized using vectorized numpy operations to ensure completion within the 6-hour limit on a 2-core runner.
