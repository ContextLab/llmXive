# Implementation Plan: Statistical Analysis of Flight Delay Distributions

**Branch**: `001-flight-delay-distributions` | **Date**: 2023-10-27 | **Spec**: `specs/001-flight-delay-distributions/spec.md`
**Input**: Feature specification from `/specs/001-flight-delay-distributions/spec.md`

## Summary

This feature implements a statistical pipeline to determine if flight delay times follow heavy-tailed distributions (Pareto, Log-Normal) versus short-tailed models (Exponential, Gamma, Weibull). The system downloads Bureau of Transportation Statistics (BTS) data for a specified year from the official BTS transtats.bts.gov endpoint using a chunked streaming strategy to respect the 7 GB RAM limit. It performs rigorous data hygiene (handling zero-inflation, anomalies, and data errors), fits five parametric models via Maximum Likelihood Estimation (MLE) on a common tail subset for hypothesis testing, and validates the "heavy-tail" hypothesis using Hill estimators, log-log survival plots, and Vuong tests. The pipeline is designed to run on CPU-only GitHub Actions runners with strict memory (≤7 GB) and time (≤6 h) constraints.

## Implementation Phases

### Phase 0: Data Acquisition and Pre-processing (US-1)

- **T001**: Download BTS 2022 On-Time Performance data from `transtats.bts.gov` using chunked streaming to avoid memory overflow.
- **T002**: Parse CSV chunks, calculate `total_delay = ArrDelay + DepDelay`, treat missing values as 0 (per FR-002), remove negative values.
- **T003**: Flag anomalies (`delay > 1440`) and data errors (`delay > 10000`). Exclude data errors from primary analysis; retain anomalies.
- **T004**: Calculate retention rate (`valid_records / total_downloaded`) and report in summary.
- **T005**: Generate `data/processed/cleaned_delays.csv` and `data/results/summary.json` (including retention rate).
- **T006**: Perform Zero-Inflation Sensitivity Analysis: create a subset excluding `total_delay == 0` and store as `data/processed/cleaned_delays_no_zero.csv`.

### Phase 1: Model Fitting and Goodness-of-Fit (US-2)

- **T007**: Fit Exponential, Gamma, Log-Normal, Weibull, Pareto to the *full* cleaned dataset (for bulk fit).
- **T008**: **Component Comparison**: Generate side-by-side histograms and compute KS p-value between `total_delay` and individual `ArrDelay`/`DepDelay`. Save to `data/results/component_comparison.json`.
- **T009**: Estimate `x_min` using the Clauset et al. (2009) method (KS minimization over grid) on the tail.
- **T010**: Refit all 5 models to the *tail subset* (`delay >= x_min`) for fair comparison.
- **T011**: Calculate AIC, BIC, KS, AD for all models on the tail subset.
- **T012**: Perform Vuong test comparing best heavy-tailed (Pareto) vs best short-tailed (Log-Normal/Gamma) on the tail subset. Save to `data/results/vuong_test.json`.
- **T013**: Save `data/results/model_comparison.json` with all metrics.

### Phase 2: Heavy-Tail Diagnostics (US-3)

- **T014**: Estimate `x_min` (if not already done) and apply Hill estimator on top k records (k/n <= 0.1).
- **T015**: Perform Hill stability analysis: compute alpha for k in range, identify 'stable plateau' (slope ~ 0). If no plateau, flag as unstable.
- **T016**: Generate log-log survival plot and calculate R² via OLS.
- **T017**: Perform Log-Normal discrimination test (curvature check) to distinguish Pareto from Log-Normal.
- **T018**: Perform Tail KS test (goodness-of-fit) on the tail subset. Save to `data/results/tail_ks.json`.
- **T019**: Generate QQ-plot for the best-fitting model.
- **T020**: Save `data/results/tail_index_estimate.json` (Hill, R², stability, curvature).

### Phase 3: Model Selection and Reporting

- **T021**: **Model Rejection Logic**: If best AIC model fails tail diagnostics (R² < 0.95 or unstable Hill), flag as rejected and select next best candidate.
- **T022**: Generate final summary report including causality disclaimer (associational only).
- **T023**: Save all artifacts with checksums. Verify against `contracts/` schemas.

### Phase 4: Verification

- **T024**: Run `pytest` on unit and integration tests.
- **T025**: Verify all output files exist and match schema.
