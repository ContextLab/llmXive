## Unresolved concerns

- The plan proposes using a paired t-test to compare MAE between the clean model and the perturbed (ablation) model. However, the ablation strategy (plan.md: Phase 3, Step 1) describes perturbing features 'during the forward pass' (inference-time noise) rather than retraining the model with missing features. This creates a mismatch: a paired t-test on inference-time perturbations measures sensitivity to noise, not the 'predictive contribution' of the feature learned during training. To validly measure feature importance via ablation, the model must be retrained without the feature (or with it zeroed out during training), or a different metric (e.g., permutation importance on the test set) must be used. The current design conflates robustness with feature importance.
- The plan states: 'If Phase 0 profiling shows > 4h for 10k samples, the plan will switch to a stratified sample of 20k-30k molecules.' This introduces a post-hoc sample size decision based on runtime rather than statistical power. If the sample is reduced to meet the 6-hour CPU constraint, the power to detect small effect sizes (e.g., < 0.05 eV MAE difference) may drop below acceptable levels (e.g., 0.8). The plan lacks a pre-defined minimum sample size required for the intended statistical tests, risking a study that is computationally feasible but statistically underpowered to answer the research question.
- The plan assumes a linear relationship (Ionization Energy ≈ -HOMO) if the specific 'ionization_energy' column is missing, citing Koopmans' theorem. However, Koopmans' theorem is an approximation that often fails for organic molecules due to electron correlation and relaxation effects, leading to systematic errors (often > 1 eV). Using this approximation as a primary target without explicitly modeling the error or validating the correlation coefficient (R²) between HOMO and IE in the specific QM9 subset introduces a significant construct validity risk. The measurement (HOMO) may not validly represent the construct (Ionization Energy) for the intended conclusions.
- The plan assumes the primary QM9 dataset contains a column named 'ionization_energy' or can be trivially mapped to 'HOMO'/'gap'. However, the standard QM9 dataset (from the original paper and most canonical sources like HuggingFace 'qm9' or 'qm9-130k') provides 'HOMO' (Highest Occupied Molecular Orbital) and 'LUMO' (Lowest Occupied Molecular Orbital) energies, but NOT the vertical ionization energy directly. While Koopmans' theorem approximates IE ≈ -HOMO, this is an approximation with known errors for organic molecules. The plan treats this mapping as a simple column rename ('handle missing values as per FR-001') rather than a scientific transformation requiring validation. If the dataset lacks the specific 'ionization_energy' label, the research question (predicting ionization energy) cannot be answered directly without this theoretical approximation, which changes the nature of the study from 'prediction' to 'approximation validation'.
- Phase 0 lists 'Dataset Verification' as a step to 'Verify column existence for ionization_energy... If missing, map to HOMO or gap if scientifically valid'. This defers a critical methodological decision to the implementation phase. If the dataset does not contain the target variable, the entire hypothesis (that 2D GNNs predict ionization energy) is untestable with the current data source. The plan must explicitly state the scientific validity of using -HOMO as a proxy for Ionization Energy in the context of the QM9 dataset *before* implementation, or identify a dataset that actually contains measured/calculated ionization energies (e.g., a subset of QM9 with specific IE calculations if available, or a different dataset like the 'QM9-IE' variant if it exists). Relying on a fallback mapping without prior validation risks the project failing to answer the research question.
- The note 'The spec assumes it contains "ionization energy" values' is vague. The plan should explicitly cite the specific column name in the referenced HuggingFace dataset (e.g., 'HOMO' or 'gap') and the theoretical justification (Koopmans' theorem) for using it as a proxy, rather than treating it as a missing value issue to be handled by code.
- Constitution Principle II (Verified Accuracy) is marked 'Compliant' with the action 'citations validated against primary sources.' However, the plan explicitly defers dataset verification to 'Phase 0: Research & Data Verification' (Step 1: 'Verify column existence...'). The plan does not describe a mechanism to validate the dataset URL or title against the primary source *before* the research phase begins, creating a logical gap where the plan claims compliance with a gate that is scheduled to happen later. This risks violating the 'Verified Accuracy' gate if the dataset is invalid.
- Phase 0 Step 1 states: 'If missing, map to HOMO or gap... or flag as blocking gap.' This contradicts Constitution Principle VII (Scaffold-Based Generalization) and the Spec's FR-001/SC-001 which strictly define the target as 'ionization energy.' The plan allows a scope change (reframing the target) without a constitutional amendment or explicit approval workflow, potentially invalidating the research question if the fallback is used.
- Phase 2 Step 1 mentions implementing 'GCN or GAT'. The data-model.md defines specific node/edge features but does not specify the exact architecture choice. While flexibility is good, the plan should explicitly state that the final architecture choice will be documented in the `data-model.md` or `contracts` to ensure the 'Single Source of Truth' (Constitution Principle IV) is maintained for the model definition.
- Phase 3 Step 3 mentions 'Integrated Gradients or simple gradient magnitude'. The `contracts/model_output.schema.yaml` and `data-model.md` define `AttributionMap` but do not specify the algorithm. The plan should reference the specific contract or data model section that will define the chosen attribution method to ensure consistency between the implementation plan and the data model.
- The ablation study plan (perturbing bond features with Gaussian noise) validates feature importance by measuring the drop in MAE. However, the 'ground truth' for this validation is the model's own prediction error on the *same* test set. If the model has learned spurious correlations or if the noise simply disrupts the model's ability to fit the training distribution without reflecting true chemical causality, the 'increase in MAE' is a tautological confirmation that the model relies on those features, not that those features are chemically causal. The plan lacks an independent chemical validation (e.g., comparing perturbation effects against known chemical shifts or experimental ionization energy trends for specific functional groups) to distinguish between 'model reliance' and 'chemical reality'.
- The plan proposes comparing the GNN (2D) against a DFTB baseline (3D) to claim 'accuracy comparable to semi-empirical methods'. This is a category error in validation. DFTB is a physics-based solver that *calculates* energy from 3D geometry; the GNN *predicts* energy from 2D topology. Comparing their MAE against the same experimental target (QM9) is valid, but the plan frames the DFTB result as a 'baseline' for the GNN's *mechanism*. If DFTB outperforms GNN, it proves 3D geometry matters, not that the GNN failed to capture 2D signal. If GNN matches DFTB, it proves 2D is sufficient. The plan's ablation (noise on 2D features) does not validate the *independence* of the 2D signal from the 3D ground truth; it only validates internal model sensitivity. A true validation of the '2D sufficiency' hypothesis requires showing that the GNN's errors are uncorrelated with the *residuals* of the DFTB calculation (i.e., where DFTB fails due to 3D effects, does GNN also fail?).
- The plan proposes using a paired t-test to compare MAE between the clean model and the perturbed model. However, MAE is a scalar aggregate metric calculated over the entire test set. A 'paired' t-test requires paired observations (e.g., error on molecule X for model A vs. model B). While one can pair the *errors* per molecule, the plan describes perturbing features *during the forward pass* (likely implying a single run with noise added to the input features). If the noise is stochastic, the 'perturbed MAE' is a random variable. The plan does not specify if the ablation involves multiple stochastic runs to generate a distribution of MAE values for the perturbed model, which is required for a valid statistical test against the deterministic clean model MAE. Without multiple runs, the p-value is meaningless.
