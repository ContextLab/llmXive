## Unresolved concerns

- User Story 2 (CPU-Constrained GNN Training) lacks a direct Functional Requirement (FR) or Success Criterion (SC) that explicitly anchors to the 'ablation study' acceptance scenario #2 (perturbing bond features with Gaussian noise). While FR-005 mentions ablation, it does not explicitly reference the specific 'Gaussian noise (σ=0.1)' constraint defined in the US-2 acceptance scenario, creating a potential gap in traceability for the specific experimental protocol.
- User Story 3 (Model Evaluation) acceptance scenario #3 (error correlation with molecule size/flexibility) is not explicitly anchored by any FR or SC. FR-006 covers baseline comparison, and FR-007 covers attribution, but neither explicitly mandates the analysis of error distribution against molecular size or flexibility, leaving this specific research goal unbacked by a functional requirement.
- SC-004 (generalization capability, gap < 15%) is anchored to FR-004 (scaffold splitting) and US-2. However, FR-004 only mandates the *implementation* of the split, not the *measurement* of the performance gap. There is no FR explicitly stating 'System MUST calculate and report the MAE gap between train and test sets' to support SC-004's measurement requirement, creating a traceability gap between the metric and the functional capability.
- FR-006 mandates comparing the GNN against a 'DFTB baseline (run on RDKit ETKDG-generated 3D conformers)'. This directly contradicts Constitution Principle VI ('Structure-Only Graph Fidelity'), which prohibits the model from relying on 3D coordinates unless justified by a specific ablation study. The spec requires a 3D-dependent baseline for the primary comparison, violating the core hypothesis constraint that the study must rely 'exclusively on 2D graph representations'.
- FR-005 requires ablation by 'perturbing specific structural features (e.g., adding Gaussian noise σ=0.1 to bond feature vectors)'. This contradicts the research goal of measuring 'predictive contribution' (feature importance) as defined in the User Story 2 motivation. Perturbing inputs at inference time measures robustness/sensitivity, not the learned contribution of features. A valid ablation for feature importance requires retraining without the feature or zeroing it during training, creating a logical contradiction between the method (noise injection) and the goal (feature importance).
- The Assumptions section states: 'The QM9 dataset () contains all necessary variables (SMILES, ionization energy)...'. The URL is missing (empty parentheses), and the assumption treats 'ionization energy' as a direct column. This contradicts the known schema of QM9 (which provides HOMO/LUMO, not direct IE) and the research question's need to validate the Koopmans' theorem approximation. The spec assumes a data reality that likely does not exist, creating a fatal scope risk where the primary target variable is undefined.
- Inconsistency in resource constraints: User Story 2 states the system must run on a '2-core, 7 GB RAM runner' and 'without GPU acceleration', but Acceptance Scenario 3 explicitly checks for 'GPU memory usage (if any) is zero'. While the 'if any' clause mitigates this, the phrasing is contradictory to the explicit 'CPU-only' constraint established in the story description and FR-003. It should be removed for clarity.
- FR-008 defines a 'hard timeout of 12 hours for the entire training and evaluation pipeline, and a hard timeout of 6 hours specifically for the training phase'. This creates a logical impossibility if the evaluation phase (which includes ablation and attribution) takes longer than 6 hours, as the total (12h) allows for it, but the training phase limit (6h) is a subset. The phrasing implies the training phase is a separate constraint, but if training takes 6h and evaluation takes 7h, the total is 13h (violating FR-008) even though training was within its limit. The constraints need to be mathematically consistent (e.g., Training <= 6h, Total <= 12h implies Evaluation <= 6h).
- SC-004 states: 'the gap must not exceed 15% relative MAE increase'. This is a pass/fail threshold for a research hypothesis, not a measurable outcome. Research success criteria should measure the *observed* gap to validate the hypothesis, not pre-assert a specific value as a requirement for the system to 'pass'. If the gap is 16%, the system is 'failed' but the research question is still answered. This conflates a success metric with a system requirement.
- US-2, Scenario 2 requires: 'MAE increases by a statistically significant margin (p-value < 0.05 from a paired t-test)'. This is not a verifiable system behavior but a statistical outcome dependent on the specific dataset and model convergence. A system cannot be 'built' to guarantee a p-value < 0.05; it can only perform the calculation. If the p-value is 0.06, the system is working correctly, but the scenario fails. This makes the acceptance criteria untestable as a binary system check.
- FR-005 mandates: 'System MUST perform ablation studies... and re-evaluating model performance'. It does not specify the *method* of ablation (e.g., retraining vs. inference-time perturbation). As noted in the kickback concerns, inference-time noise measures robustness, not feature importance. Without specifying the ablation protocol (e.g., 'retrain with feature zeroed'), the requirement is ambiguous and cannot be verified as meeting the research goal of 'predictive contribution'.
- FR-006 requires comparison against a 'DFTB baseline (run on RDKit ETKDG-generated 3D conformers)'. The spec does not define the DFTB parameters (e.g., Hamiltonian, basis set) or the specific ETKDG parameters. Without these, the baseline is not reproducible, and the comparison is not verifiable. Two implementations could produce vastly different DFTB results, making the 'comparison' unverifiable.
- US-1, Scenario 3 states: 'Then the resulting training, validation, and test sets are disjoint in terms of molecular scaffolds'. While the intent is clear, 'disjoint' is a binary state that requires a specific definition of 'scaffold' (e.g., Bemis-Murcko, specific ring systems) to be verifiable. Without defining the scaffold extraction algorithm in the spec, a tester cannot deterministically verify this condition.
- The Assumptions section lists 'The QM9 dataset () contains all necessary variables'. The URL is missing (empty parentheses). While this is an assumption, the lack of a specific source makes the assumption unverifiable. It should cite the specific HuggingFace repo or Figshare ID to be actionable.
- The spec mandates comparing the GNN against a DFTB baseline (FR-006) and measuring success against it (SC-001). However, the upstream Idea explicitly defines the research question as determining if 'structure-only models' (2D) can approximate accuracy, and the Methodology Sketch lists DFTB only as a comparison point, not a required baseline for the *validity* of the 2D hypothesis. More critically, the Idea assumes QM9 contains 'ionization energies', while the Plan (kicked back) revealed QM9 likely only has HOMO/LUMO. The spec fails to resolve this construct validity gap: it demands prediction of 'ionization energy' (FR-001) without specifying the scientific transformation (Koopmans' theorem: IE ≈ -HOMO) required to generate the target label from the available data. This creates a fatal mismatch where the spec requires a target variable that does not exist in the source data without a defined scientific proxy, risking the entire research question's validity.
- The spec requires ablation studies by 'perturbing specific structural features (e.g., adding Gaussian noise σ=0.1 to bond feature vectors)' (FR-005, US-2). The upstream Idea asks to identify 'which structural features... carry the most predictive signal'. The kicked-back plan correctly identified that inference-time noise perturbation measures *robustness*, not *feature importance* (predictive contribution). To validly answer the research question, the spec must require retraining the model without the feature (or with it zeroed out during training) or using a permutation importance method on the test set. The current spec conflates robustness with feature importance, failing to address the core research question.
- The spec requires a paired t-test to compare MAE between clean and perturbed models (US-2, SC-002). However, the ablation method defined (inference-time noise) produces a single deterministic MAE for the clean model and a stochastic MAE for the perturbed model (if noise is applied per run). A valid paired t-test requires paired observations (errors per molecule) across multiple stochastic runs to establish a distribution. The spec does not mandate multiple stochastic runs or a specific statistical aggregation method, rendering the proposed statistical test invalid for the defined ablation strategy.
- The spec imposes a hard 6-hour training limit and 12-hour pipeline limit (FR-008, US-2) but lacks a pre-defined minimum sample size or power analysis. The kicked-back plan noted that reducing the sample size to meet the time constraint risks statistical underpowering. The spec must explicitly state the minimum sample size required to detect the expected effect sizes (e.g., < 0.05 eV MAE difference) with acceptable power (e.g., 0.8), or define a fallback statistical method for smaller samples, to ensure the study can validly answer the research question within the time constraints.
- The Assumptions section states 'The QM9 dataset () contains all necessary variables (SMILES, ionization energy)'. This is factually incorrect for the standard QM9 dataset which provides HOMO/LUMO, not direct ionization energy. The spec must update this assumption to explicitly acknowledge the use of Koopmans' theorem (IE ≈ -HOMO) as the scientific transformation to derive the target variable, or identify a specific dataset variant that contains direct IE values, to ensure construct validity.
- FR-005 and US-2 define ablation as 'perturbing specific structural features (e.g., adding Gaussian noise...)' during inference. This measures model robustness/sensitivity to noise, not the 'predictive contribution' of features learned during training. Valid feature importance via ablation requires retraining the model without the feature or zeroing it out during training. The current design conflates robustness with causal feature importance, rendering the validation of SC-002 scientifically invalid.
- The spec assumes QM9 contains 'ionization energy' or that 'HOMO' is a valid proxy via Koopmans' theorem without validation. Koopmans' theorem (IE ≈ -HOMO) is an approximation with known systematic errors (>1 eV) in organic molecules due to electron correlation. Treating HOMO as the ground truth for 'Ionization Energy' without explicitly validating the correlation (R²) or modeling the error constitutes a construct validity failure. The research question (predicting IE) cannot be answered if the target variable is actually HOMO with unquantified bias.
- The validation design compares a 2D GNN against a DFTB (3D) baseline to claim 'accuracy comparable to semi-empirical methods'. This is a category error. DFTB calculates energy from 3D geometry; the GNN predicts from 2D topology. Comparing their MAE against the same target validates accuracy, but the spec frames DFTB as a baseline for the GNN's *mechanism*. If DFTB outperforms GNN, it proves 3D matters, not that GNN failed. The spec lacks a method to validate the *independence* of the 2D signal from 3D effects (e.g., checking if GNN errors correlate with DFTB residuals), making the '2D sufficiency' hypothesis untestable with the current design.
- The spec requires a 'paired t-test' on MAE between clean and perturbed models. However, the ablation (noise injection) is described as a single inference-time event or a stochastic process without specifying multiple runs. A paired t-test requires paired observations (errors per molecule) across multiple stochastic runs to generate a distribution. Without specifying multiple runs to establish the variance of the perturbed MAE, the p-value is statistically meaningless. The statistical method is mismatched to the experimental design.
- SC-004 sets a hard threshold: 'gap must not exceed 15% relative MAE increase'. This is an arbitrary empirical claim presented as a success criterion without statistical power analysis. If the sample size is reduced to meet CPU constraints (as hinted in the kickback comments), the study may be underpowered to detect this specific effect size, making the 'pass/fail' criterion scientifically unsound. The spec defers sample size decisions to runtime, risking a study that is computationally feasible but statistically incapable of answering the research question.
