# Implementation Plan: Statistical Analysis of Publicly Available Weather Data for Extreme Event Prediction

**Branch**: `001-extreme-weather-spatial-analysis` | **Date**: 2026-07-07 | **Spec**: `specs/001-extreme-weather-spatial-analysis/spec.md`
**Input**: Feature specification from `specs/001-extreme-weather-spatial-analysis/spec.md`

## Summary

This feature implements a statistical pipeline to analyze spatial dependence in extreme weather events using NOAA GHCN-Daily data. Due to computational constraints on the GitHub Actions free tier (2 CPU, 7GB RAM), the originally specified Brown-Resnick max-stable process (FR-004) has been pivoted to a **Spatial Generalized Pareto Distribution (Spatial-GPD) model with a Gaussian Copula** for dependence. This approach preserves the core research question (spatial predictive gain) while ensuring feasibility. The system ingests raw daily data, defines extremes via a peaks-over-threshold method (a high percentile on historical training data), and compares an independent GPD baseline against the Spatial-GPD model. Evaluation uses Brier scores and RMSE on a recent multi-year test set, with a "Global Dependence, Local Prediction" strategy for Leave-One-Station-Out (LOSO) cross-validation to meet the 6-hour compute limit.

> **Methodological Pivot Notice**: The implementation deviates from the literal requirement of FR-004 (Brown-Resnick) and FR-006 (standard LOSO re-fitting) due to computational infeasibility on the target platform. The plan satisfies the *intent* of these requirements (spatial modeling, generalization assessment) using a scientifically sound, CPU-tractable alternative. A kickback is flagged to update the spec to reflect this pivot.

## Implementation Phases

### Phase 0: Validation & Hygiene (Constitutional Gates)
1.  **Phase 0.1: Reference Validation**: Run `scripts/validate_citations.py` to verify the NOAA GHCN-Daily URL against the primary source. **HALT** if validation fails.
2.  **Phase 0.5: Data Hygiene & Versioning**: Run `scripts/hygiene_check.py` to compute SHA-256 hashes for raw data and write them to `state/projects/...yaml`.

### Phase 1: Data Ingestion & Preprocessing
1.  **Ingestion**: Download raw GHCN-Daily data. Verify date range (2000-2020). **HALT** if range is incomplete.
2.  **Preprocessing**: Filter stations (>15% missing, >30 day gaps). Impute short gaps. Calculate high percentile thresholds (2000-2015).
3.  **Extreme Event Definition**: Generate `ExtremeEvent` records (exceedances).

### Phase 2: Modeling
1.  **Baseline**: Fit Independent GPD for each station.
2.  **Spatial**: Fit Spatial-GPD (marginals + Gaussian Copula dependence). **Subsample** time series if fit time > 2 hours.
3.  **Fallback**: If Spatial-GPD fails to converge, log `FALLBACK: CONVERGENCE_ERROR` and use Independent GPD (flagged as `fallback_gpd`).

### Phase 3: Evaluation & Visualization
1.  **Metrics**: Calculate Brier score, RMSE, Coverage.
2.  **Cross-Validation**: Perform LOSO using Global Dependence, Local Prediction.
3.  **Visualization**: Generate variograms, QQ-plots, maps.

### Phase 4: Reporting
1.  **Output**: Generate `outputs/metrics.json` and `outputs/plots/`.
2.  **Audit**: Verify output schema and fallback logs.
