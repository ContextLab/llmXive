# Data Directory Structure

This directory contains all data artifacts for the Material Degradation Pathways project.

## Structure

- `raw/`: Unmodified data downloaded from external sources (e.g., Zenodo).
- `processed/`: Cleaned, filtered, and engineered datasets ready for modeling.
- `contracts/`: Validation artifacts and reference vectors (e.g., literature_vector.json).

## Provenance

All data files must include a checksum and source URL in their respective README or metadata files.
Do not modify files in `raw/`. All transformations should be reproducible via the scripts in `code/`.

## Subdirectory Descriptions

### raw/
Contains original datasets as downloaded from external repositories (Zenodo, etc.).
Files here are immutable and serve as the single source of truth for all downstream processing.
Expected files:
- `corrosion_alloys_raw.csv`: Raw corrosion data from Zenodo (T013)

### processed/
Contains cleaned, filtered, and feature-engineered datasets.
Files here are generated by scripts in `code/` and should be reproducible.
Expected files:
- `cleaned_alloys.csv`: Filtered and imputed alloy data (T018)
- `retention_audit.json`: Statistics on data retention after filtering (T018)
- `train_set.parquet`: Training split based on alloy class (T019)
- `test_ood_set.parquet`: Out-of-distribution test split (T019)

### contracts/
Contains validation artifacts, reference vectors, and ground truth definitions.
Files here are used for model validation and scientific contract verification.
Expected files:
- `literature_vector.json`: Aggregated feature importance from literature review (T009)

## Checksums

All critical data files should have corresponding `.sha256` checksum files for integrity verification.
Checksums are generated using the `calculate_sha256` function from `code/utils.py`.
