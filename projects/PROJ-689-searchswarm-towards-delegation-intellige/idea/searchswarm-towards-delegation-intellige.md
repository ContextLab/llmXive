# SearchSwarm: Towards Delegation Intelligence in Agentic LLMs for Long‑Horizon Deep Research

**Field**: computer science

## Research question

How does hierarchical delegation among agentic LLMs affect success rates on long‑horizon research benchmarks compared with a flat (non‑delegating) agent architecture?

## Motivation

Long‑horizon autonomous research tasks quickly exceed the finite context windows of current LLMs, forcing agents to truncate or forget earlier reasoning steps. A promising remedy is to delegate subtasks to specialized sub‑agents, thereby offloading context and allowing deeper reasoning. However, it is unclear whether hierarchical delegation yields measurable gains in task success beyond the overhead of managing multiple agents. Clarifying this relationship would guide the design of scalable, agentic research assistants.

## Related work

- [SearchSwarm: Towards Delegation Intelligence in Agentic LLMs for Long‑Horizon Deep Research (2026)](https://arxiv.org/abs/2606.09730) — Introduces the SearchSwarm framework that spawns sub‑agents to manage context, providing the conceptual foundation for hierarchical delegation.  
- [LongCoT: Benchmarking Long‑Horizon Chain‑of‑Thought Reasoning (2026)](https://arxiv.org/abs/2604.14140) — Defines a benchmark suite for evaluating LLM reasoning over extended horizons, which we will use as the primary evaluation set.  
- [Hierarchical Multi‑agent Large Language Model Reasoning for Autonomous Functional Materials Discovery (2025)](https://arxiv.org/abs/2512.13930) — Shows that hierarchical multi‑agent reasoning can improve scientific discovery tasks, supporting the hypothesis that delegation benefits complex domains.  
- [A Survey of Multi‑Agent Deep Reinforcement Learning with Communication (2022)](https://arxiv.org/abs/2203.08975) — Reviews communication mechanisms that enable coordination among agents, informing our design of message passing between delegator and sub‑agents.  
- [Learning From Failure: Integrating Negative Examples when Fine‑tuning Large Language Models as Agents (2024)](https://arxiv.org/abs/2402.11651) — Demonstrates that exposing agents to failed attempts improves robustness, which we will incorporate when training the delegator’s decision policy.

## Expected results

We anticipate that a hierarchical delegation architecture will achieve a statistically higher average success score (≥ 5 % absolute improvement) on the LongCoT benchmark than a flat agent of comparable size. Significance will be confirmed with paired t‑tests (α = 0.05) across multiple random seeds, and effect sizes will be reported. If the gain is not significant, the result will still clarify the trade‑off between delegation overhead and reasoning depth.

## Methodology sketch

- **Data acquisition**  
  - Download the LongCoT benchmark dataset (`wget https://arxiv.org/pdf/2604.14140.pdf`) and extract the JSON test cases.  
  - Obtain the Open‑Source Llama‑2‑7B model weights via HuggingFace (`git lfs clone https://huggingface.co/meta-llama/Llama-2-7b-hf`).  

- **Baseline implementation (flat agent)**  
  - Fine‑tune Llama‑2‑7B on the LongCoT training split using the standard LoRA approach (learning rate = 2e‑4, 3 epochs, batch size = 8).  
  - Evaluate the fine‑tuned model on the test split, recording per‑question correctness and overall success rate.

- **Hierarchical delegation system**  
  - Implement a delegator module that, given a task description, predicts whether to solve directly or to spawn a sub‑agent (binary classifier trained on a small synthetic delegation dataset).  
  - Sub‑agents are identical fine‑tuned Llama‑2‑7B instances but receive only the delegated sub‑task context.  
  - Use the communication protocol described in the 2022 survey to exchange summaries between delegator and sub‑agents.  
  - Incorporate negative‑example training (from the 2024 paper) to improve the delegator’s decision policy.

- **Evaluation**  
  - Run both systems on the LongCoT test set using the same random seeds (5 seeds).  
  - Compute per‑seed success rates, aggregate mean ± standard deviation.  
  - Perform paired t‑tests comparing hierarchical vs. flat scores per seed.  
  - Report confidence intervals and Cohen’s d effect sizes.

- **Resource constraints**  
  - All steps run on a GitHub Actions runner (2 CPU cores, ≤ 7 GB RAM).  
  - Model inference is performed with `bitsandbytes` 4‑bit quantization to stay within memory limits.  
  - Training is limited to ≤ 30 minutes per epoch, ensuring total job time < 6 hours.
