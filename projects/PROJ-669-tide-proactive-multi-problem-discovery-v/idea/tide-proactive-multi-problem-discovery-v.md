# TIDE: Proactive Multi-Problem Discovery via Template-Guided Iteration

**Field**: linguistics

## Research question

How does template‑guided iterative prompting influence the number and linguistic diversity of latent problems identified in unstructured text corpora compared with a single‑shot prompting baseline?

## Motivation

Current language‑assistant agents react only to explicit user queries, leaving many latent problems in documents unnoticed. A proactive, template‑driven iterative approach could surface a broader set of issues, but it is unclear whether this yields genuinely richer and more relevant problem statements than a one‑shot prompt. Establishing the empirical impact of such iteration will inform the design of future assistive systems that aim to discover hidden linguistic or pragmatic problems.

## Related work

- [TIDE: Proactive Multi-Problem Discovery via Template-Guided Iteration (2026)](https://arxiv.org/abs/2606.04743) — Introduces the template‑guided iterative framework and reports preliminary performance gains, but lacks systematic evaluation of diversity and independent relevance validation.  
- [The Art of Socratic Inquiry: A Framework for Proactive Template‑Guided Therapeutic Conversation Generation (2026)](https://arxiv.org/abs/2602.01598) — Demonstrates how structured template prompts can steer large language models toward deeper, multi‑turn inquiry, providing a methodological precedent for our iterative prompting design.  
- [Empirical Discovery in Linguistics (1995)](https://arxiv.org/abs/cmp-lg/9506023) — Presents an early discovery system for detecting linguistic correspondences, offering a baseline for evaluating problem‑discovery systems against classic, non‑iterative approaches.

## Expected results

We anticipate that the template‑guided iterative method will uncover a larger quantity of problem statements and exhibit higher lexical and syntactic diversity than the single‑shot baseline. Success will be confirmed if (a) the mean relevance score (human‑rated on a 5‑point Likert scale) is statistically higher (p < 0.05, paired t‑test) and (b) diversity metrics (type‑token ratio, POS‑n‑gram entropy) are significantly greater (p < 0.05). A null result—no significant difference—will still be informative, indicating that iteration may not add value beyond well‑crafted single prompts.

## Methodology sketch

- **Data acquisition**  
  - Download the “Reddit Comments” dataset (HuggingFace `reddit_comments`) and the “StackExchange Posts” dataset (HuggingFace `stackexchange`) via `datasets` library (≈ 2 GB total).  
  - Randomly sample 10 000 text snippets (≈ 200 MB) from each source for analysis.

- **Template construction**  
  - Derive a set of 5‑10 probing templates from the Socratic‑inquiry framework (e.g., “What assumptions underlie …?”, “Can you identify any contradictions in …?”).  
  - Encode templates as system prompts for the LLM.

- **Iterative prompting pipeline**  
  - Use an open‑source 7B‑parameter LLM (e.g., Llama‑2‑7B) via the HuggingFace `transformers` library with 4‑bit quantization to stay within 7 GB RAM.  
  - For each text snippet:  
    1. Apply the first template and collect generated “problem” statements.  
    2. Feed the top‑k statements back into the next template, repeating for up to 3 iterations or until no new statements appear.  
  - Record all generated problem statements per snippet.

- **Baseline condition**  
  - Run a single‑shot prompt (“List any problems you notice in the following text: …”) with the same LLM on the identical snippets.

- **Human relevance annotation (independent validation)**  
  - Export all problem statements to a CSV and upload to a crowdsourcing platform (e.g., Amazon MTurk).  
  - Each statement receives three independent relevance ratings (1 = irrelevant, 5 = highly relevant).  
  - Compute the mean rating per statement; aggregate per condition.

- **Diversity measurement**  
  - Compute type‑token ratio (TTR) and part‑of‑speech n‑gram entropy for the pooled statements of each condition.

- **Statistical analysis**  
  - Perform paired t‑tests comparing (a) mean relevance scores and (b) diversity metrics between iterative and baseline conditions.  
  - Report effect sizes (Cohen’s d) and 95 % confidence intervals.

- **Reproducibility packaging**  
  - All scripts, environment file (`environment.yml`), and random seeds are version‑controlled in a public GitHub repository.  
  - Data sampling code includes the exact `seed=42` used for reproducibility.
