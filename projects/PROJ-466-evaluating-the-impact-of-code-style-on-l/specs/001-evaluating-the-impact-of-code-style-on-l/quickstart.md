# Quickstart Guide: Evaluating the Impact of Code Style on LLM Code Generation Diversity

This guide provides instructions for setting up, running, and analyzing the `llmXive` pipeline for evaluating how code style constraints affect LLM code generation diversity.

## Prerequisites

- **Python**: 3.11+
- **Hardware**: Recommended 16GB+ RAM (for dynamic batch sizing and dataset loading)
- **Dependencies**: `transformers`, `torch`, `datasets`, `networkx`, `scipy`, `pandas`, `pytest`, `pyyaml`, `numpy`

## Setup

1. **Clone the repository** (if applicable) and navigate to the project root.
2. **Create a virtual environment**:
 ```bash
 python -m venv venv
 source venv/bin/activate # On Windows: venv\Scripts\activate
 ```
3. **Install dependencies**:
 ```bash
 pip install -r code/requirements.txt
 ```
 *Note: `requirements.txt` is located in the `code/` directory as per project structure.*

4. **Verify environment**:
 ```bash
 python -c "import datasets; import torch; import networkx; print('Dependencies OK')"
 ```

## Configuration

The pipeline uses a YAML configuration file located at `config/analysis.yaml`.

- **Seeds**: Default is 42.
- **Timeouts**: Default per-task limit is 5 minutes.
- **Batch Size**: Automatically adjusted based on available memory.
- **Thresholds**: Define statistical significance thresholds (α).

To modify settings, edit `config/analysis.yaml`:
```yaml
seeds:
 generation: 42
limits:
 task_timeout_seconds: 300
 max_memory_gb: 7.0
thresholds:
 alpha: 0.05
 bias_magnitude: 0.15
```

## Running the Pipeline

The entire pipeline is orchestrated by `code/main.py`.

### Full Execution
Run the complete workflow (Generation → Testing → Metrics → Statistics → Report):
```bash
python code/main.py
```

### Output Artifacts
Upon successful completion, the following files will be generated in `data/processed/`:

- `samples_all.csv`: All generated code samples (including those that failed tests).
- `samples_valid.csv`: Only samples that passed the HumanEval unit tests.
- `metrics_all.csv`: Structural diversity metrics for all samples.
- `metrics_valid.csv`: Structural diversity metrics for valid samples only.
- `report.html` / `report.pdf`: Final analysis report containing statistical tests, bias flags, and sensitivity plots.

## Data Sources

- **HumanEval Dataset**: The pipeline automatically downloads the `openai/human-eval` dataset from Hugging Face Datasets on the first run. It is cached in `data/raw/humaneval/`.
- **No Synthetic Data**: All code samples are generated by the configured LLM; no synthetic or placeholder data is used.

## Troubleshooting

### Memory Errors
If you encounter `CUDA Out of Memory` or similar errors:
- The pipeline includes a dynamic batch sizing mechanism (`code/generation/generator.py`). It will automatically reduce batch size if memory usage exceeds the threshold defined in `config/analysis.yaml` (default 7GB).
- Logs for memory reductions are stored in `data/memory_log.json`.

### Timeouts
If a task generation exceeds the timeout (default 5m):
- The task is skipped, and a timeout error is logged.
- Check `data/memory_log.json` or the console output for `TaskTimeoutError`.

### Model Incapability
If the pass rate for any style group drops below 1%:
- The pipeline halts immediately with a "Model Incapability" warning.
- This prevents entry into Phase 4 (Metrics) to avoid analyzing non-functional code.

### Verification
To verify the setup without running the full generation (which requires an LLM API):
```bash
# Run unit tests
pytest tests/unit/

# Run integration tests (requires LLM access)
pytest tests/integration/
```

## Expected Runtime

- **Small Subset (e.g., 10 tasks)**: ~15-30 minutes
- **Full HumanEval (164 tasks)**: ~4-6 hours (depending on LLM latency and batch size)
