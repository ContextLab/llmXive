# Data Model: The Impact of Linguistic Complexity on Trust in AI-Generated Text

## Overview

This document defines the data schema for the project, ensuring alignment with the project constitution (Data Hygiene, Metric Dimensionality) and functional requirements (FR-001 to FR-008). All data is stored in CSV/Parquet format with checksums.

## Entities

### 1. TextSample

Represents a unique text instance generated by the LLM.

| Field | Type | Description | Source |
|-------|------|-------------|--------|
| `sample_id` | str | Unique identifier (e.g., `TS-001`) | Generated |
| `raw_text` | str | The generated text content | `generate_text.py` |
| `source_id` | str | Identifier of the source Wikipedia paragraph | `generate_text.py` |
| `flesch_kincaid` | float | Flesch-Kincaid readability score | `generate_text.py` |
| `mtld` | float | Measure of Lexical Diversity | `generate_text.py` |
| `avg_sentence_length` | float | Average words per sentence | `generate_text.py` |
| `generation_seed` | int | Random seed used for generation | `generate_text.py` |
| `prompt_variation` | str | Prompt variation used (e.g., "simple", "academic") | `generate_text.py` |

**Constraints**:
- `flesch_kincaid` range: 5.0–12.0 (SC-001).
- `mtld` and `avg_sentence_length` must be non-negative.

### 2. ParticipantResponse

Represents a single participant's interaction with the survey.

| Field | Type | Description | Source |
|-------|------|-------------|--------|
| `response_id` | str | Unique identifier (e.g., `PR-001`) | Generated |
| `participant_id` | str | Anonymized participant ID (e.g., `P-123`) | Prolific / Survey |
| `sample_id` | str | Foreign key to `TextSample.sample_id` | Survey |
| `trust_rating` | int | Likert trust rating (1–5) | Survey |
| `attention_check_passed` | bool | True if attention check passed | `collect_trust.py` |
| `straight_lining_detected` | bool | True if participant gave identical ratings for all items | `collect_trust.py` |
| `submission_time` | datetime | Timestamp of submission | Survey |

**Constraints**:
- `trust_rating` ∈ {1, 2, 3, 4, 5}.
- `attention_check_passed` and `straight_lining_detected` used for filtering (FR-004).

### 3. PCAFactors

Derived dataset containing orthogonal complexity factors.

| Field | Type | Description | Source |
|-------|------|-------------|--------|
| `sample_id` | str | Foreign key to `TextSample.sample_id` | PCA Pipeline |
| `syntactic_factor` | float | PCA-derived score for syntactic complexity (FK + Sentence Length) | `analyze.py` |
| `lexical_factor` | float | PCA-derived score for lexical diversity (MTLD) | `analyze.py` |

**Constraints**:
- `syntactic_factor` and `lexical_factor` are orthogonal (correlation ≈ 0).

### 4. AnalysisResult

Represents the output of the statistical model for a given complexity factor.

| Field | Type | Description | Source |
|-------|------|-------------|--------|
| `result_id` | str | Unique identifier (e.g., `AR-SYNT-001`) | Generated |
| `factor_name` | str | Name of complexity factor (e.g., "syntactic_factor") | `analyze.py` |
| `quadratic_coefficient` | float | Coefficient for factor² | `analyze.py` |
| `quadratic_p_value` | float | p-value for quadratic term | `analyze.py` |
| `linear_coefficient` | float | Coefficient for factor | `analyze.py` |
| `r_squared` | float | R-squared of the model | `analyze.py` |
| `fdr_adjusted_q_value` | float | Benjamini-Hochberg adjusted q-value | `analyze.py` |
| `significant_at_fdr` | bool | True if q < 0.05 | `analyze.py` |
| `plot_path` | str | Path to the generated plot (e.g., `figures/syntactic_plot.png`) | `analyze.py` |
| `post_hoc_effect_size` | float | Minimum detectable effect size (f²) | `analyze.py` |
| `ordinal_logistic_consistent` | bool | True if ordinal logistic regression yields same conclusion | `analyze.py` |

**Constraints**:
- `quadratic_p_value` ∈ [0, 1].
- `r_squared` ∈ [0, 1].

## Data Flow

1. **Generation**: `generate_text.py` → `data/raw/generated_text.csv` (TextSample).
2. **Collection**: `collect_trust.py` / Prolific → `data/raw/trust_responses.csv` (ParticipantResponse).
3. **Cleaning**: Filter attention checks / straight-lining → `data/processed/cleaned_responses.csv`.
4. **PCA**: `analyze.py` (PCA step) → `data/processed/pca_factors.csv`.
5. **Merge**: Join TextSample + cleaned responses + PCA factors → `data/processed/analysis_inputs.csv`.
6. **Analysis**: `analyze.py` (LME step) → `data/outputs/regression_results.json` + `figures/`.

## Checksums & Versioning

- Every file in `data/` is checksummed (SHA-256).
- Checksums recorded in `state/projects/PROJ-521-the-impact-of-linguistic-complexity-on-t.yaml`.
- No in-place modifications; derivations create new files.
