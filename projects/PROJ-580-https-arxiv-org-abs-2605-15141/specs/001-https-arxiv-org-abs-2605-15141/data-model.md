# Data Model: Causal Forcing++ Validation

## 1. Overview
This document defines the data schemas for the validation artifacts and the internal data structures used during the CPU-only execution.

## 2. Artifact Schema
The validation process produces two primary artifacts: a generated video and a training checkpoint.

### 2.1 Video Artifact
- **Format**: `.mp4` (H.264) or `.gif`.
- **Content**: A sequence of frames generated by the diffusion model.
- **Properties**:
  - Minimum size: 1KB.
  - Codec: Valid video codec (verified by `ffprobe`).
  - Resolution: As defined in config (default 256x256 or 512x512).

### 2.2 Checkpoint Artifact
- **Format**: `.pt` or `.pth` (PyTorch `torch.save`).
- **Content**: Model state dictionary (`state_dict`).
- **Properties**:
  - Minimum size: 100KB.
  - Keys: Must contain standard model keys (e.g., `model.diffusion_model`, `model.text_encoder`).

## 3. Internal Data Structures

### 3.1 Synthetic Training Data
A `TensorDataset` containing random tensors.
- **Input Shape**: `(B, C, H, W, F)` where:
  - `B` (Batch): 1
  - `C` (Channels): 16 (typical for diffusion latent)
  - `H, W` (Spatial): 32x32 (downsampled for speed) or 64x64.
  - `F` (Frames): 16.
- **Target Shape**: Same as input or noise (depending on loss function).

### 3.2 Validation Report
A JSON structure summarizing the run.
- **Fields**:
  - `inference_status`: "SUCCESS" | "FAIL" | "SKIP".
  - `inference_artifact_path`: String (relative path).
  - `training_status`: "SUCCESS" | "FAIL" | "SKIP".
  - `training_checkpoint_path`: String (relative path).
  - `loss_values`: List[float] (first 5 steps).
  - `peak_memory_mb`: Integer.
  - `total_runtime_seconds`: Float.
  - `errors`: List[String] (if any).

## 4. Configuration Schema
The YAML configuration files used for overriding defaults.

### 4.1 Inference Override
```yaml
model:
  name: "wan_t2v_1_3B"
  device: "cpu"
  precision: "float16" # Attempt to save memory
pipeline:
  num_steps: 2
  frames: 16
  resolution: [256, 256]
  seed: 42
```

### 4.2 Training Override
```yaml
model:
  name: "wan_t2v_1_3B"
  device: "cpu"
  precision: "float16"
training:
  max_steps: 5
  batch_size: 1
  use_dummy_data: true
  dummy_data_shape: [1, 16, 32, 32, 16]
```
