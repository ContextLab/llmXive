# ABot‑Earth 0.5: Generative 3D Earth Model  

**Field**: computer science  

## Research question  

How does conditioning a generative 3D Earth model on multi‑view high‑resolution satellite imagery affect geometric continuity across adjacent tiles, compared with conditioning on single‑view imagery?  

## Motivation  

Current generative Earth‑model pipelines often train on a single view per tile, which can cause seams when tiles are stitched together. Improving inter‑tile continuity would enable more reliable large‑scale visualizations and downstream simulations. The question isolates the impact of multi‑view conditioning, a concrete gap in existing generative‑terrain literature.  

## Related work  

- [ABot‑Earth 0.5: Generative 3D Earth Model (2026)](https://arxiv.org/abs/2606.09967) — Introduces a diffusion‑based 3D generative framework that conditions on satellite imagery but does not isolate the effect of multi‑view conditioning on tile continuity.  
- [InfiniteDiffusion: Bridging Learned Fidelity and Procedural Utility for Open‑World Terrain Generation (2025)](https://arxiv.org/abs/2512.08309) — Proposes a diffusion model for open‑world terrain and discusses fidelity versus procedural scalability, providing a baseline for single‑view conditioning.  

## Expected results  

We anticipate that the multi‑view‑conditioned model will achieve lower Chamfer‑distance scores (≥ 15 % reduction) on overlapping border regions than the single‑view baseline, demonstrating improved geometric continuity. A non‑significant difference would suggest that additional views offer limited benefit for this metric.  

## Methodology sketch  

- **Data acquisition**  
  - Download Sentinel‑2 (10 m GSD) and Landsat 8 (30 m GSD) imagery for a 10 km × 10 km test region from the AWS Open Data Registry.  
  - Extract overlapping 256 × 256 px tiles; for each tile collect up to four orthogonal views (north‑south‑east‑west) using the same geographic footprint.  
- **Pre‑processing**  
  - Align and resample all views to a common grid; generate corresponding height‑map proxies from publicly available SRTM DEM (30 m) for evaluation only.  
- **Model training (two variants)**  
  1. **Single‑view model** – train a lightweight diffusion network (e.g., 2‑stage UNet, 5 M parameters) on one randomly selected view per tile.  
  2. **Multi‑view model** – same architecture, but input concatenates the four aligned views.  
  - Use the HuggingFace `datasets` library to stream data; train for 5 epochs (≈ 2 h on a 2‑core GHA runner with GPU‑emulated CPU inference).  
- **Generation**  
  - Generate 3D point‑cloud tiles for the test region with each model; keep a 10 % overlap between adjacent tiles.  
- **Continuity evaluation**  
  - Compute Chamfer distance between overlapping point clouds of neighboring tiles (both directions).  
  - Aggregate mean and standard deviation across all borders.  
- **Statistical analysis**  
  - Perform a paired t‑test on the per‑border Chamfer scores (null hypothesis: no difference).  
  - Report effect size (Cohen’s d) and 95 % confidence intervals.  
- **Reproducibility**  
  - All scripts, environment (`requirements.txt`), and random seeds are version‑controlled in a public GitHub repository.  
  - Data URLs and checksums are listed in the README.
