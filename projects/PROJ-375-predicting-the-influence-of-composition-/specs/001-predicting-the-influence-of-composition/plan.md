# Implementation Plan: Predicting the Influence of Composition on the Thermal Expansion of Metallic Glasses

**Branch**: `001-gene-regulation` | **Date**: 2026-07-20 | **Spec**: `specs/001-gene-regulation/spec.md`

## Summary

This project implements a computational pipeline to predict the Coefficient of Thermal Expansion (CTE) of metallic glasses based solely on their chemical composition. The approach involves ingesting data from public repositories (Materials Project, AFLOWlib) or a verified open-access fallback dataset (Zhang et al.,), deriving compositional descriptors (atomic radius, electronegativity, VEC, size mismatch), and training baseline machine learning models (Linear Regression, Random Forest) under strict CPU resource constraints. The plan ensures statistical rigor via permutation testing (if N ≥ 50) or qualitative trend analysis (if N < 50) and strict adherence to the project constitution regarding reproducibility and data hygiene.

## Complexity Tracking

| Violation | Why Needed | Simpler Alternative Rejected Because |
|-----------|------------|-------------------------------------|
| **Stratified Split by Alloy Family** | Essential to prevent data leakage and ensure models generalize across different base metals (Zr, Pd, Fe). | Random split would likely create training sets dominated by one family, leading to biased performance metrics. |
| **Permutation Testing (Conditional)** | Required by SC-002 to validate that R² > 0.3 is not due to random chance *if N ≥ 50*. | Standard cross-validation alone does not provide a p-value against the null hypothesis. |
| **VIF-based Feature Exclusion** | Required to handle the mathematical coupling between `mean_atomic_radius` and `size_mismatch`. | Including both without checking VIF leads to multicollinearity, invalidating independent effect claims. |
| **Fallback Research Question** | Required to ensure the project yields a valid result even if N < 50. | A hard halt would render the project a failure; a qualitative analysis provides valid scientific insight. |
| **Divergence Analysis** | Required to detect non-linear effects. | Forcing a match between linear correlation and non-linear importance is scientifically unsound. |

## Implementation Phases

### Phase 0: Data Acquisition & Validation
1. **Attempt API Fetch**: Try Materials Project and AFLOWlib APIs using environment variables.
 * **Constraint**: Must collect **all** available valid entries from APIs, even if count is 0.
2. **Fallback Fetch**: If APIs return < 50 entries OR fail, download the verified Zenodo dataset (Zhang et al.).
3. **Validation**: Check for `N ≥ 50`.
 * **If N ≥ 50**: Proceed to Phase 1.
 * **If 20 ≤ N < 50**: Proceed to Phase 1 with Hold-Out validation (skip 5-fold).
 * **If N < 20**: Proceed to Phase 1 with Leave-One-Out (LOO) validation.
 * **If N = 0**: Trigger **Phase 0.5 (No Data Termination)**.

### Phase 0.5: No Data Termination (If N = 0)
1. Log error: "No valid metallic glass entries found in API or Zenodo."
2. Generate `results/metrics.json` with `status: "no_data"`.
3. End pipeline with exit code 0 (valid completion, not a crash).

### Phase 0.6: Qualitative Trend Analysis (If N < 50)
1. Skip permutation testing and 5-fold CV.
2. Perform simple linear regression and correlation analysis.
3. Generate feature rankings and report "Qualitative Trend" status.
4. End pipeline with `status: "qualitative"` (valid completion).

### Phase 1: Feature Engineering
1. Calculate descriptors: `mean_atomic_radius`, `electronegativity_var`, `vec`.
2. Calculate `size_mismatch` and run VIF check against `mean_atomic_radius`.
3. If VIF > 5.0, exclude `size_mismatch` from model input (log warning).
4. Save processed dataset to `data/processed/clean_mg_data.parquet`.

### Phase 2: Model Training (If N ≥ 20)
1. **Splitting**:
 * **If N ≥ 50**: 5-Fold Stratified by `alloy_family`.
 * *Fallback*: If any family has < 5 samples, group into "Other" or revert to random split.
 * **If 20 ≤ N < 50**: k-Fold or Hold-Out (random split)
