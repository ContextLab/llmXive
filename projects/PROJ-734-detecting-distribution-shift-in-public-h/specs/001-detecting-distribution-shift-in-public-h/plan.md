# Implementation Plan: Detecting Distribution Shift in Public Health Surveillance Data via Kernel Two‑Sample Tests

**Branch**: `001-detect-distribution-shift` | **Date**: 2026-06-25 | **Spec**: `spec.md`

## Summary

This feature implements a reproducible pipeline to detect distributional shifts in CDC FluView weekly Influenza-like Illness (ILI) rates using the Gaussian-kernel Maximum Mean Discrepancy (MMD) two-sample test. The system compares MMD performance against baseline change-point detection methods (Pettitt test and Bayesian Online Change-Point Detection) and evaluates robustness via sensitivity analysis on kernel bandwidth, window length, and week-alignment tolerance. The pipeline runs entirely on CPU, adhering to strict memory and time constraints for GitHub Actions free-tier runners.

**Critical Note on Data Availability**: This pipeline strictly requires real CDC FluView and CDC Virological/Hospitalization data to produce final results. If these datasets are unavailable, the pipeline will halt with a clear error (E-NO-DATA) and will not produce the final `report.pdf` with metrics. Synthetic data is used *only* for unit testing the pipeline logic, not for final study results.

**Scale/Scope**: A multi-year weekly ILI dataset comprising a substantial volume of observations.; sensitivity grid multiple configurations (2 bandwidths x 3 windows x 3 tolerances).

**Structure Decision**: Single project structure (Option 1) chosen. The scope is a linear data pipeline with distinct modules for download, preprocessing, detection, and evaluation. This minimizes overhead and aligns with the CPU-only, single-runner execution model.

**Configuration Validation**: The `code/config.yaml` file is validated against `contracts/config.schema.yaml` at runtime startup. If validation fails, the pipeline halts with an error.

**Data Fetch Strategy**: The pipeline attempts to download real CDC FluView and Virological/Hospitalization data from verified sources. If these sources are unavailable, the pipeline halts with error code `E-NO-DATA`. Synthetic data is used *only* for unit tests in `tests/unit/`.

**Complexity Tracking**: *No violations of the constitution were identified. The complexity of the sensitivity analysis (FR-007, FR-010) is managed by parallelizing grid points within the 30-minute window and auto-reducing permutations if necessary (FR-008).*
