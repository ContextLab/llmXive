## Summary

This project implements a Convolutional Neural Network (CNN) to predict three molecular electronic properties (dipole moment, polarizability, HOMO-LUMO gap) from vibrational IR spectra. The pipeline ingests QM and IR-spectra data, aligns them via InChIKey, preprocesses spectra to a fixed grid, trains a multi-head regression model on a CPU-only runner, and evaluates performance using MAE, R², and statistical TOST (equivalence) tests. The implementation strictly adheres to the runtime and memory constraints of GitHub Actions free-tier runners.

## Complexity Tracking

| Violation | Why Needed | Simpler Alternative Rejected Because |
|-----------|------------|-------------------------------------|
| **Multi-head Architecture** | Research question requires independent prediction of multiple distinct electronic properties. | A single-output model would conflate the properties, violating Principle VII (Multi-Head Regression Traceability). |
| **Independent Validation / Fallback** | Required to avoid circular validation (US-4). | If no verified external dataset exists, a 'Domain Shift Simulation' (synthetic noise) is used to test robustness, preventing the plan from stalling while maintaining scientific rigor. |
| **Strict CPU Constraints** | Must run on GitHub Actions free tier. | GPU training or large models would fail the runtime/memory constraints, blocking `research_complete`. |
| **Runtime Enforcement** | FR-006 requires hard termination at 6h. | Without a `timeout` wrapper, a hanging process could exceed the CI limit, causing a failed job rather than a controlled stop. |

## Implementation Phases

### Phase 1: Data Ingestion & Alignment (FR-001, FR-002)
1. **Download**: Fetch QM9 and IR-Spectra datasets.
2. **DFT Verification**: Check metadata for DFT functional/basis set. If mismatched, flag as "Domain Shift" candidate.
3. **Alignment**: Inner join on `InChIKey`. Log discarded count.
4. **Coverage Audit**: Compare property distributions (KS-test) between full QM9 and aligned subset to detect selection bias.
5. **Preprocessing**: Interpolate to 400–4000 cm⁻¹ (1 cm⁻¹), Gaussian smoothing (σ=2), normalize unit area.
6. **Validation**: `tests/test_preprocessing.py` verifies grid fidelity and artifact absence.

### Phase 2: Model Training (FR-003, FR-004)
1. **Architecture**: 1-D CNN with 3 heads.
2. **Training**: CPU-only, Adam, early stopping.
3. **Runtime Enforcement**: `utils/timeout_wrapper.py` kills process if > 6 hours.

### Phase 3: Evaluation & Validation (FR-005, FR-007)
1. **Metrics**: Compute MAE, R² per property.
2. **Statistical Analysis**:
   - **TOST**: Test if mean error is within a practical threshold (e.g., ±0.1 Debye).
   - **Multivariate**: Hotelling's T² test for joint error distribution.
3. **Independent Validation**:
   - If external dataset (different DFT) exists: Evaluate and compare MAE (≤20% tolerance).
   - Else: Run Domain Shift Simulation (noise injection) and report robustness metrics.
4. **State Update**: `utils/update_state.py` hashes artifacts and updates `state/...yaml`.

### Phase 4: Reporting
1. Generate `results/evaluation_metrics.json`.
2. Write final report referencing JSON metrics only (Principle IV).
