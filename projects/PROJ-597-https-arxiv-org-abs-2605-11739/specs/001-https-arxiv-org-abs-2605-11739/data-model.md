# Data Model: Reproduce & Validate EffOPD On-Policy Distillation

## 1. Data Flow Overview

1.  **Input**: Raw datasets (`gsm8k`, `aime24`) fetched via Hugging Face `datasets` library.
2.  **Preprocessing**: Data is sampled and formatted into JSONL files matching the `EffOPD` expectations. A data adapter maps HF keys (`question`/`answer`, `problem`/`solution`) to internal schema.
3.  **Processing**:
    *   **Simulated Step**: A single training step is performed to generate Delta W (W_new - W_old). Output: `artifacts/delta_w.pt`.
    *   **SFT Baseline Step**: A standard SFT step is performed to generate SFT_Delta W for comparison. Output: `artifacts/sft_delta_w.pt`.
    *   **Null Baseline**: A random Gaussian matrix is generated for comparison.
    *   `svd.py`: Loads Delta W, performs SVD, outputs singular values.
    *   `upd_rank.py`: Analyzes Delta W, outputs rank concentration.
    *   `reasoning_eval.py`: Generates solutions, runs grader, outputs Pass@k metrics.
    *   `baseline_comparison`: Compares EffOPD vs SFT vs Null metrics.
4.  **Output**: CSV/JSON artifacts stored in `artifacts/`.

## 2. Schema Definitions

### 2.1 Input Dataset Schema (Internal)

| Field | Type | Description | Source Keys (GSM8K) | Source Keys (AIME24) |
| :--- | :--- | :--- | :--- | :--- |
| `id` | `string` | Unique problem identifier | `question` (hash) | `problem` (hash) |
| `question` | `string` | The problem statement (normalized) | `question` | `problem` |
| `answer` | `string` | The ground truth solution (normalized) | `answer` | `solution` |
| `metadata` | `object` | Optional metadata (difficulty, source) | `gsm8k` / `aime24` | `gsm8k` / `aime24` |

### 2.2 Delta W Input Schema (for SVD/Rank)
*Source: `artifacts/delta_w.pt` generated by `simulate_step.py`*

| Field | Type | Description | Constraints |
| :--- | :--- | :--- | :--- |
| `tensor` | `torch.Tensor` | Flattened weight update matrix (Delta W = W_new - W_old) | Shape: `(num_params,)` or `(layer_id, flattened_dim)` |
| `layer_id` | `integer` | Identifier of the model layer | $\ge 0$ |
| `norm` | `float` | L2 norm of the update | $> 0$ (non-zero update required) |

### 2.3 SVD Output Schema (`svd_results.csv`)
*Input: Delta W (Update Matrix)*

| Column | Type | Description | Constraints |
| :--- | :--- | :--- | :--- |
| `layer_id` | `integer` | Identifier of the model layer | $\ge 0$ |
| `singular_value` | `float` | Singular value $\sigma_i$ of Delta W | $> 0$ |
| `index` | `integer` | Rank index (0-based) | $\ge 0$ |
| `cumulative_energy` | `float` | Cumulative sum of $\sigma^2$ | $[0.0, 1.0]$ |

### 2.4 Rank Concentration Output Schema (`upd_rank_results.csv`)

| Column | Type | Description | Constraints |
| :--- | :--- | :--- | :--- |
| `layer_id` | `integer` | Identifier of the model layer | $\ge 0$ |
| `rank` | `integer` | Effective rank of the update matrix (Delta W) | $\ge 1$ |
| `concentration_score` | `float` | Score indicating low-rank concentration | $[0.0, 1.0]$ |

### 2.5 Baseline Comparison Output Schema (`baseline_comparison.json`)
*New artifact to capture comparative validation logic.*

| Field | Type | Description | Constraints |
| :--- | :--- | :--- | :--- |
| `effopd_concentration` | `float` | Concentration score of EffOPD update | $[0.0, 1.0]$ |
| `sft_concentration` | `float` | Concentration score of SFT baseline | $[0.0, 1.0]$ |
| `null_concentration` | `float` | Concentration score of Null baseline | $[0.0, 1.0]$ |
| `delta_effopd_sft` | `float` | Difference (EffOPD - SFT) | N/A |
| `delta_effopd_null` | `float` | Difference (EffOPD - Null) | N/A |
| `flops_effopd` | `integer` | Theoretical FLOPs for EffOPD step | $> 0$ |
| `flops_sft` | `integer` | Theoretical FLOPs for SFT step | $> 0$ |
| `efficiency_ratio` | `float` | Ratio of FLOPs (SFT / EffOPD) | $> 0$ |

### 2.6 Reasoning Evaluation Output Schema (`reasoning_results.json`)

| Field | Type | Description | Constraints |
| :--- | :--- | :--- | :--- |
| `problem_id` | `string` | Problem identifier | Non-empty |
| `Pass@1` | `float` | Fraction of correct solutions (k=1) | $[0.0, 1.0]$ |
| `Pass@5` | `float` | Fraction of correct solutions (k=5) | $[0.0, 1.0]$ |
| `Pass@10` | `float` | Fraction of correct solutions (k=10) | $[0.0, 1.0]$ |
| `total_problems` | `integer` | Number of problems evaluated | $> 0$ |
| `execution_time` | `float` | Total time in seconds | $> 0$ |

## 3. Data Validation Rules

*   **Missing Data**: If a dataset row lacks `question` or `answer` (or `problem`/`solution`), it is skipped, and a warning is logged.
*   **Memory Guard**: Data loading is capped at a fixed token limit. If exceeded, the script samples fewer rows.
*   **Numerical Stability**: Singular values are clipped to a minimum of $1e-10$ to avoid division by zero in energy calculations.
*   **Delta W Validation**: The update matrix must be non-zero (norm > 0) before SVD is performed.
*   **Tensor Format**: `artifacts/delta_w.pt` must be a valid PyTorch tensor file containing the flattened weight update.
