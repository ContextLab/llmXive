## Summary

This project implements a computational pipeline to analyze the Neural Correlates of Error Monitoring using the Navigation Error Corpus. The core objective is to quantify the **associational** relationship between Medial Frontal Negativity (MFN) amplitude (mean amplitude in 200-400ms) and directional error magnitude. The technical approach involves downloading the dataset, applying MNE-Python for EEG preprocessing (filtering, ICA), extracting MFN features using mean amplitude, calculating behavioral error metrics via angular deviation, and fitting Linear Mixed-Effects models (with GAM fallback via AIC comparison) using `statsmodels` and `pymer4`. The pipeline includes mandatory robustness checks (threshold sensitivity, VIF diagnostics, multiple-comparison correction) and ensures full reproducibility on CPU-only GitHub Actions runners.

## Specific Algorithmic Details

### Angular Deviation Calculation (FR-003)
To satisfy FR-003, the `preprocess.py` script will implement the following logic:
1.  **Inputs**: Participant heading vector $H$ and optimal path vector $O$ at the moment of error onset.
2.  **Calculation**: Compute the angle $\theta$ between $H$ and $O$ using the dot product formula: $\theta = \arccos(\frac{H \cdot O}{\|H\| \|O\|})$.
3.  **Output**: Convert $\theta$ to degrees. This value is the `error_magnitude_deg`.
4.  **Edge Case**: If vectors are zero-length, log a warning and skip the event.

### MFN Extraction (FR-004)
To satisfy FR-004 and address noise concerns:
1.  **Window**: 200ms to 400ms post-error.
2.  **Metric**: **Mean Amplitude** (average of all samples in the window) at electrodes FCz, Cz, Fz.
3.  **Baseline**: Corrected to the pre-error period (-200ms to 0ms).
4.  **Note**: While the spec mentions "peak", mean amplitude is methodologically preferred for single-trial ERP analysis to reduce noise sensitivity. If peak is strictly required by the spec's final interpretation, the code will compute both but report mean as the primary metric for robustness.

### Compute Feasibility Validation (SC-005)
The `analysis.py` script will include a `validate_feasibility()` step:
1.  **Start/End Time**: Log timestamps for each major phase.
2.  **Memory**: Use `psutil` to log peak RSS memory usage.
3.  **Report**: Write a `results/diagnostics/feasibility_report.json` containing `runtime_seconds` and `peak_memory_mb`.
4.  **Gate**: If `runtime_seconds > 21600` (6h) or `peak_memory_mb > 7168`, the script raises a `FeasibilityError` and logs a warning.
