## 2. Measurement Precision Standards

| Invariant | Data Type | Precision Standard | Rationale |
|-----------|-----------|--------------------|-----------|
| **Crossing Number (c)** | Integer | Exact (no rounding) | By definition the crossing number is a non‑negative integer; any deviation would indicate a parsing error. |
| **Braid Index (b)** | Integer | Exact (no rounding) | The braid index is also an integer; tabulated values are taken directly from the Knot Atlas without modification. |
| **Hyperbolic Volume (V)** | Float | Rounded to three decimal places (0.001) | Volumes are computed numerically in KnotInfo; the source reports up to 6‑digit precision, but for reproducibility we store values rounded to the nearest 0.001 to avoid spurious variability. |
| **Alternating Classification** | Categorical (`alternating`, `non‑alternating`, `unclassifiable`) | Exact string match | Tie‑breaking rules (see `docs/reproducibility/tie_breaking_rules.md`) enforce deterministic assignment. |

### 2.1 Precision Validation

The script `code/analysis/precision.py` (function `dummy_precision_check`) validates that:

* All crossing numbers and braid indices are whole numbers.
* Hyperbolic volumes contain at most three decimal places.
* No invariant exceeds the allowed null‑percentage thresholds (≤ 0.5 % for integer fields, ≤ 5 % for continuous fields).

Results of this validation are recorded in `docs/reproducibility/precision_validation.md` and summarized in the scatter‑plot `data/plots/crossing_vs_braid.png`.
