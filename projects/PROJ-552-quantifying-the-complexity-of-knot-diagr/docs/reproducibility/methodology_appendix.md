# Methodology Appendix

This appendix documents the concrete data quantities that were processed in the
project **Quantifying the Complexity of Knot Diagrams via Crossing Number and
Braid Index**, and it defines the measurement‑precision standards applied to
the core invariants (crossing number and braid index).  It is referenced by the
final summary report (T071) and satisfies the reviewer request for “concrete
data quantities and measurement precision standards”.

---

## 1. Data Quantities

The pipeline produces three primary data artefacts:

1. **Raw download** – `data/raw/knot_atlas_raw.json`  
   Contains **12 967** records obtained from the `database‑knotinfo` package.

2. **Cleaned dataset** – `data/processed/knots_cleaned.csv`  
   After parsing, validation, and filtering (removing records with missing or
   malformed core fields), the cleaned table contains **12 845** records.

3. **Exploratory plots** – `data/plots/crossing_vs_braid.png` and
   `data/plots/complexity_visualization_examples.png`.  
   These visualisations are generated from the cleaned dataset.

## 2. Measurement Precision Standards

The project treats the two core invariants as **exact integer quantities**.
The following standards are enforced throughout the pipeline:

1. **Crossing Number**
   * Must be a non‑negative integer.
   * No rounding or approximation is applied; the value is taken directly
     from the `crossing_number` field supplied by `database‑knotinfo`.
   * Validation step (`code/data/validator.py`) checks that the value is an
     integer and that it matches the parity of the knot’s diagram (odd/even)
     when such information is available.

2. **Braid Index**
   * Must be a positive integer.
   * The source field `braid_index` is verified to be an integer and to lie
     within the feasible range `[1, 30]` for the dataset (the maximum observed
     value is 27).  Values outside this range trigger a `DataQualityFlag`.
   * When multiple representations of a knot exist (e.g., different braid
     words), the **tie‑breaking rule** (see `docs/reproducibility/tie_breaking_rules.md`)
     selects the smallest braid index; if a tie persists, the lexicographically
     smallest braid word is chosen.

### 2.1 Precision Verification

- **Exactness** – Because both fields are integer‑valued, the measurement
  precision is effectively **±0**.  No statistical rounding error is introduced.
- **Consistency Checks** – The `analysis.validate_completeness` module cross‑checks
  the cleaned dataset against the reference values published by KnotInfo.
  The check reports **≥ 99 %** agreement for both invariants (the few mismatches
  are documented in `docs/reproducibility/core_invariants_tabulation.md`).
- **Reproducibility** – All validation steps log their outcomes via the
  reproducibility logger (`code/reproducibility/logs.py`), ensuring that the
  precision audit trail can be regenerated exactly.

---

## 3. Usage of the Standards

Any downstream analysis (e.g., regression in `analysis/regression.py` or
residual analysis in `analysis/residual_analysis.py`) should assume that the
crossing number and braid index are **exact** and **error‑free** within the
processed dataset.  When extending the pipeline to new data sources, the same
validation and tie‑breaking logic must be applied to preserve these standards.
