# Missing Data Handling Details

The dataset contains several fields that may be missing for some knot records.
We adopt the following strategy throughout the analysis pipeline:

1. **Flagging** – During parsing (`code/data/parser.py`) any missing required
   attribute is recorded in the `validation_flags.json` file. The flag includes
   the record identifier and the missing field name.
2. **Imputation** – For quantitative attributes that are optional (e.g., `hyperbolic_volume`),
   we impute missing values using the median of the observed values for the same
   knot family. This is implemented in `code/analysis/data_quality.py`.
3. **Exclusion** – Records with missing critical identifiers (e.g., `knot_id`,
   `crossing_number`) are excluded from downstream statistical models. The
   exclusion criteria are applied in `code/analysis/composite_metric.py`.
4. **Reporting** – A summary of missingness is generated by the
   `data_quality_report` module and stored in `data/processed/validation_flags.json`.

All scripts respect the `MISSING_DATA_POLICY` environment variable, allowing
researchers to switch between imputation and exclusion modes without changing
code.
