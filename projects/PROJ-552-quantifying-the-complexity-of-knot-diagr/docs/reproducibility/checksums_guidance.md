# Checksums and Version Control

All checksum manifests (`data/checksums.csv`, `data/checksums.json`, `data/checksums.sha256`) are committed to the repository.  
They are generated by `code/reproducibility/run_checksums.py` and should be regenerated whenever source data changes.  
The CI pipeline runs this script and fails if the committed checksums differ from the freshly computed ones, ensuring integrity.
To update checksums locally:
```bash
    python -m code.reproducibility.run_checksums
```
Commit the updated files together with the data changes.
