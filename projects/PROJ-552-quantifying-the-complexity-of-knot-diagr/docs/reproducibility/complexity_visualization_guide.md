# Complexity Visualization Guide

## Overview

This document describes the visualization examples generated to demonstrate how
complexity metrics (crossing number and braid index) map to knot diagram features.
These visualizations address reviewer feedback from dan-rockmore-simulated regarding
human-readable interpretation of complexity metrics.

## Generated Artifacts

### Primary Visualization

**File**: `data/plots/complexity_visualization_examples.png`

**Purpose**: Multi-panel figure showing relationships between crossing number,
braid index, and other knot invariants.

**Resolution**: 1200x900 pixels (configurable via figsize parameter)

**Generated by**: `code/analysis/complexity_visualization.py`

## Visualization Panels

The main visualization consists of 6 panels:

### Panel 1: Correlation Heatmap

Shows the correlation matrix between complexity metrics (crossing number, braid
index, hyperbolic volume). This helps identify which metrics provide independent
information about knot complexity.

**Interpretation**:
- Strong positive correlation between crossing number and braid index (expected,
 since braid index ≤ crossing number)
- Hyperbolic volume may show weaker correlation, indicating independent geometric
 information

### Panel 2: Crossing Number vs Braid Index Scatter

Scatter plot with alternating and non-alternating knots shown in different colors.
Includes reference line showing the mathematical constraint braid index ≤ crossing
number.

**Interpretation**:
- Points cluster near the diagonal for alternating knots
- Non-alternating knots may show more deviation from the diagonal
- The constraint line provides a visual bound

### Panel 3: Crossing Number Distribution

Histogram with KDE showing the distribution of crossing numbers in the dataset.

**Interpretation**:
- Shows census composition (more knots at higher crossing numbers)
- Helps understand data density across complexity ranges

### Panel 4: Braid Index by Crossing Number

Boxplot showing braid index distribution for each crossing number value.

**Interpretation**:
- Shows how braid index varies within each crossing number class
- Reveals whether braid index is tightly constrained or variable

### Panel 5: Complexity Feature Examples

Bubble chart where point size represents braid index and color represents
crossing number.

**Interpretation**:
- Visual encoding of two complexity metrics simultaneously
- Larger bubbles = higher braid index
- Color gradient = crossing number

### Panel 6: Summary Statistics

Text box with key dataset statistics and correlation values.

**Content**:
- Dataset size
- Range of crossing numbers and braid indices
- Mean values
- Correlation coefficient
- Alternating/non-alternating split

## Usage

### Running the Visualization

```bash
cd code/analysis
python complexity_visualization.py
```

### Programmatic Usage

```python
from analysis.complexity_visualization import (
 load_cleaned_knots,
 generate_complexity_visualization_examples
)

# Load data
df = load_cleaned_knots(Path("data/processed/knots_cleaned.csv"))

# Generate visualization
output_path = generate_complexity_visualization_examples(
 df,
 Path("data/plots/complexity_visualization_examples.png"),
 figsize=(16, 12)
)
```

## Reviewer Alignment

This visualization directly addresses dan-rockmore-simulated's feedback about
providing human-readable complexity interpretations. The multi-panel approach
shows:

1. **Quantitative relationships**: Correlations and distributions
2. **Structural patterns**: How metrics relate across the census
3. **Concrete examples**: Visual encoding of complexity values
4. **Statistical context**: Summary statistics for interpretation

## Maintenance

To modify visualization appearance:

1. Adjust `figsize` parameter in `generate_complexity_visualization_examples()`
2. Modify individual panel functions (e.g., `create_crossing_braid_scatter()`)
3. Update color schemes in seaborn/matplotlib calls

All changes should maintain the 2x3 grid layout for consistency.
