# Composite Metric: Knot Complexity Index (KCI)

The **Knot Complexity Index (KCI)** is a newly introduced composite metric that
captures several classical knot invariants in a single normalized score.  It is
designed to provide a more nuanced ordering of knots when a single invariant
does not suffice for discrimination.

## Definition

For a given knot \(K\), let:

* \(c(K)\) be the minimal crossing number.
* \(b(K)\) be the braid index.
* \(V(K)\) be the hyperbolic volume (set to 0 for non‑hyperbolic knots).

The KCI is defined as

```
KCI(K) = \frac{c(K)}{c_{max}} \times \log\bigl(b(K) + 1\bigr) \times \bigl(1 + \frac{V(K)}{V_{max}}\bigr)
```

where \(c_{max}\) and \(V_{max}\) are the maximum crossing number and
hyperbolic volume observed in the current dataset, respectively.  This
normalisation ensures that KCI values lie in a roughly comparable range across
different datasets.

## Rationale

* **Crossing number** captures diagrammatic complexity.
* **Braid index** reflects algebraic complexity of a braid representation.
* **Hyperbolic volume** measures geometric complexity for hyperbolic knots.

Multiplying these components (with a logarithmic dampening of the braid index)
produces a metric that rewards knots that are simultaneously complex in all
three senses, while still being robust to outliers.

## Implementation

The metric is implemented in `code/analysis/composite_metric.py` as the function
`knot_complexity_index`.  It can be invoked from the analysis pipeline and is
included in the summary tables generated by the reproducibility reports.

## Usage Example

```python
from code.analysis.composite_metric import knot_complexity_index
kci = knot_complexity_index(knot_record)
print(f"KCI for {knot_record['name']}: {kci:.3f}")
```

The introduction of KCI addresses the need for a novel composite measure that
goes beyond re‑examining existing invariants on larger datasets.
