# File Naming Conventions

All Python modules in this project should follow **snake_case** naming.
Modules that combine nouns and verbs (e.g., `validation_status_generator.py`)
are considered non‑conforming. The preferred name for that functionality is
`validation_status_gen.py`, which already exists in `code/analysis/`.

If you encounter a module with a mixed naming style, rename it to the
snake_case version and update any import statements accordingly.

This document serves as a reference for contributors to maintain consistent
file‑type conventions throughout the codebase.
