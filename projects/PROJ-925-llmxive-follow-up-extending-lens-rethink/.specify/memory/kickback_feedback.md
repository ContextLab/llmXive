## Unresolved concerns

- The plan assumes the `coco-img-caption-pairs` dataset contains a `human_rating` column to compute the deviation target ($| 	ext{CLIP} - 	ext{Human} |$). However, standard COCO datasets contain image/caption pairs but lack explicit human preference ratings. If this column is absent (as is typical), the target variable $Y$ cannot be computed, rendering the entire regression analysis impossible. The plan relies on a 'loud failure' rather than a valid alternative data source or a redefined target variable, which breaks the causal chain required to answer the research question.
- The plan operationalizes 'Semantic Entropy' as $\ln(\text{Perplexity})$ from a BERT model. This is a construct validity failure. Semantic entropy (Farquhar et al., 2024) measures the uncertainty of a model's *output distribution* over multiple generations, whereas BERT perplexity measures the likelihood of a *single* token sequence. These are distinct constructs; using perplexity as a proxy for semantic entropy in the context of 'alignment gap' analysis may measure syntactic fluency or vocabulary diversity rather than the intended semantic uncertainty, invalidating the study's conclusions about 'semantic' drivers of the gap.
- The study design is purely observational (using existing dataset pairs) but aims to identify 'predictors' of the alignment gap. While the plan correctly frames this as 'associational', the lack of any confound control (e.g., controlling for caption length, image complexity, or domain) means any observed correlation between linguistic features and deviation could be spurious. For instance, if complex captions are more likely to be generated by newer models (which have different CLIP/Human alignment profiles), the 'syntactic complexity' feature is merely a proxy for model version. The plan fails to model or rule out these obvious confounds.
- FR-006 mandates a permutation-based significance test with 'at least 1,000 iterations' and Benjamini-Hochberg correction. The plan.md and research.md mention 'permutation importance' and 'Benjamini-Hochberg' generally, but do not explicitly commit to the 1,000 iteration count or the specific FDR control mechanism in the implementation strategy. Without this explicit commitment in the plan, the success criterion SC-004 (p < 0.05 after BH correction) cannot be guaranteed to be measured correctly.
- SC-005 requires measuring sensitivity of feature importance rankings against a 'sweep of random seeds'. The plan.md mentions a 'Stability Loop' in the Complexity Tracking table and 'stability_metrics.json' in the data model, but the research.md section on 'Methodological Rigor' does not explicitly define the loop mechanism (e.g., 'iterate over N seeds, retrain, aggregate ranks'). The plan lacks a concrete description of how the 'sweep' is executed, risking a failure to measure the specific success criterion.
