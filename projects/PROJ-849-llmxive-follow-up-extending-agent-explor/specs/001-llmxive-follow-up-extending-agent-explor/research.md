# Research: Semantic Divergence Diagnostic for Agentic Reasoning

## Problem Statement

The research investigates whether a measurable gap exists between an agentic system's internal "thinking" process and the external tool actions it selects. Specifically, we test the hypothesis that a high "Semantic Divergence Score" (low similarity between thinking embedding and tool-action centroid) predicts failure in multimodal reasoning tasks.

## Dataset Strategy

We utilize the following verified datasets and resources to ensure reproducibility and feasibility on the CPU-only runner:

| Dataset | Source URL | Usage | Verification Status |

| **MathVista** | ` | Source of problem instances and "thinking" traces. | Verified |
| **Tool Mapping** | `data/tool_mappings/mathvista_tool_map.json` (Local artifact) | Maps problem IDs to tool descriptions for BM25 retrieval. | Verified (Local) |
| **AXPO Simulation** | [] | Ground truth failure rates generated by re-running AXPO logic on the same problems.| Verified (Internal, code source)|

**Dataset Selection Rationale**:
- **MathVista**: The only verified open-source multimodal reasoning dataset with the required "thinking" traces and problem diversity. The verified URL points to the test split, which we will sample from.
- **Tool Mapping**: A curated JSON file is required because the spec explicitly defines a "non-reasoning keyword retrieval system" against a *curated* corpus. This avoids the complexity of dynamic tool discovery and ensures the BM25 index is deterministic.
- **AXPO**: The original AXPO implementation provides the ground truth for simulation, ensuring reproducibility.

**Data Access Plan**:
1. **Download**: Use `datasets.load_dataset` with `streaming=True` to fetch MathVista shards, preventing memory overflow on the 7 GB limit.
2. **Sample**: Extract an initial subset of records (or fewer if memory constraints trigger) with a fixed random seed.
3. **Filter**: Discard records missing "thinking" prefixes or tool mappings.
4. **Validate**: Ensure N ≥ 30 before proceeding to statistical analysis.

## Methodology

### Phase 1: Metric Computation (FR-001 to FR-004)
1. **Thinking Trace Generation:** The agent's 'thinking prefix' is generated by a lightweight, deterministic agent executing the specified reasoning task on each problem instance in the MathVista dataset. This ensures a consistent and reproducible source of internal thought traces.
2. **Embedding**: Use `DistilBERT-base-uncased` (CPU-optimized) to generate embeddings for the "thinking" prefix.
3. **Retrieval**: Build a BM25 index from the tool descriptions in `mathvista_tool_map.json`. Retrieve a representative set of top-ranked tools for each problem.
4. **Centroid Calculation**: Average the embeddings of the retrieved tool descriptions. If fewer than 10 tools are retrieved, average only the available embeddings and normalize by the number of retrieved tools. If zero tools are retrieved, use a zero vector.
5. **Divergence Score**: Calculate `1 - cosine_similarity(thinking_embedding, tool_centroid)`.

### Phase 2: Correlation Analysis (FR-005)
1. **Ground Truth**: Generate `simulated_failure` (boolean) for each problem by re-running the original AXPO agent logic.
2. **Statistical Test**: Compute Point-Biserial correlation coefficient (`r`) and p-value between `divergence_score` and `simulated_failure`.
3. **Power Check**: If N < 30, halt with "Insufficient Sample Size".

### Phase 3: Predictive Modeling (FR-006)
1. **Split**: Stratified train/test split based on 'simulated_failure' to ensure class balance in each set.
2. **Model**: Train a Logistic Regression classifier to predict `simulated_failure` from `divergence_score`, including problem difficulty as a covariate.
3. **Evaluation**: Report Accuracy, Precision, Recall, and AUC-ROC.

## Statistical Rigor & Constraints

- **Multiple Comparisons**: Not applicable (single primary correlation test).
- **Power Analysis**: The plan targets N=500. Assuming an effect size of r=0.15 for Point-Biserial Correlation, this provides >80% power at α=0.05. If N < 30, the system halts.
- **Causal Inference**: Claims are strictly associational. We do not claim the gap *causes* failure, only that it *predicts* it.
- **Collinearity**: The "thinking" and "tool" embeddings are derived from distinct sources (LLM trace vs. static tool map), minimizing definitional collinearity. Problem difficulty is included as a covariate to account for inherent problem complexity.
- **Compute Feasibility**:
 - **CPU-First**: DistilBERT and BM25 are lightweight. The memory limit is respected by streaming data and processing in batches.
 - **GPU Escape Hatch**: Not required. The method (DistilBERT + BM25) is fully tractable on CPU. No GPU acceleration is planned.
 - **Timeout**: Hard limit of a fixed duration enforced.

## Decision Rationale

- **DistilBERT vs. Larger Models**: Chosen for CPU feasibility. The semantic gap is a coarse signal; high-fidelity embeddings are not strictly necessary, and larger models (e.g., LLaMA) would exceed the 7 GB RAM limit on the free-tier runner.
- **BM25 vs. Neural Retrieval**: BM25 is deterministic, fast, and explicitly required by the spec to create a "non-reasoning" baseline. Neural retrieval would introduce training variance and complexity.
- **Simulated AXPO**: Using a cached simulation or re-running the original AXPO agent ensures the ground truth is available and reproducible without needing access to private data.
