# Research: Visual Detail and False Memory Susceptibility

## Executive Summary

This research plan outlines the computational investigation of how visual detail (enhanced vs. reduced) influences false memory susceptibility. The study employs a repeated-measures design where participants view baseline images, undergo a distractor task, and complete a recognition test containing true and false details. Due to the lack of a verified URL for the Visual Genome dataset in the provided context, the implementation will utilize a **synthetic image generation pipeline** that mimics the complexity distribution of Visual Genome. This ensures the project remains reproducible and runnable on CPU-only infrastructure while strictly adhering to the "Dataset-variable fit" requirement. The study is framed as a **Computational Proxy Validation**, testing the pipeline's ability to detect effects of a known magnitude in a controlled environment.

## Methodological Rigor

### Operationalization of Visual Detail
To ensure the mock generator is a valid proxy for the perceptual construct of "visual detail," the `baseline_complexity_score` is derived from a weighted sum of:
1.  **Object Count**: Number of distinct objects detected in the synthetic scene.
2.  **Texture Variance**: Standard deviation of pixel intensity in texture regions.
These values are normalized to a 0.0 - 1.0 scale. The `source_type` field in metadata explicitly marks these as "mock" to maintain transparency.

### Statistical Analysis Plan
1.  **Primary Test**: Repeated-measures ANOVA (Condition: Enhanced vs. Reduced vs. Baseline) on False Memory Rate.
    -   **Correction**: Bonferroni correction applied for multiple comparisons if >1 contrast is tested.
    -   **Assumptions**: Sphericity will be tested (Mauchly's test); Greenhouse-Geisser correction applied if violated.
    -   **Effect Size**: Partial eta-squared ($\eta_p^2$) will be reported.
2.  **Synthetic Response Logic**:
    -   The false memory rate is **not** hardcoded. It is calculated as:
        $$Rate = \beta_0 + \beta_1(Condition) + \beta_2(Complexity) + \beta_3(Condition \times Complexity) + \epsilon$$
    -   Where $\beta_3$ represents the interaction between manipulation and complexity, ensuring the ANOVA tests a modeled interaction rather than a simple main effect.
3.  **Power Analysis**:
    -   **Goal**: Detect a medium effect size ($f = 0.25$) with $\alpha = 0.05$ and Power = 0.80.
    -   **Estimated N**: ~50 complete sessions (per SC-002).
    -   **Limitation**: As this is a simulation, power is theoretical. Real-world recruitment may require adjustment.
4.  **Causal Inference**:
    -   **Observational Note**: The design is *not* a randomized controlled trial of "detail" in the wild; it is a controlled experiment where "detail" is manipulated. Claims will be framed as **associational** regarding the *effect of the manipulation* on memory, acknowledging that the manipulation itself is the independent variable.
    -   **Collinearity**: If multiple complexity metrics are used (e.g., object count vs. texture), they will be checked for multicollinearity (VIF). If high, they will be reported descriptively, and the primary analysis will use the single `baseline_complexity_score`.

### Measurement Validity & Calibration
-   **False Memory Paradigm**: Uses the recognition paradigm (Loftus et al., 1978) adapted for visual stimuli. The "false" items are lures that are semantically or perceptually related but absent.
-   **Calibration Step**: Before main analysis, the synthetic response parameters ($\beta_0, \beta_1, \dots$) are tuned to match effect sizes reported in the Loftus et al. (1978) literature. This ensures the "false memory rate" approximates the target construct.
-   **Null Model Validation**: A "flat" simulation (no effect) is run to ensure the ANOVA correctly returns a non-significant result, proving the test is not tautological.
-   **Instrument**: The recognition questions are generated based on the object list of the baseline image.
-   **Limitation**: The synthetic nature of the images means the "false memory" is a model output, not a human cognitive error. The validity depends on the realism of the probabilistic model used in the simulation.

## Decision Rationale

| Decision | Rationale |
| :--- | :--- |
| **Use Mock Visual Genome** | The "Verified datasets" block states **NO verified source found** for Visual Genome. Using an unverified URL would violate the "Verified Accuracy" gate. A mock generator ensures the code runs, variables are present, and reproducibility is maintained. |
| **CPU-Only Implementation** | GitHub Actions free tier (limited CPU and RAM resources) prohibits GPU training. Image manipulation (PIL) and statistical analysis (scipy) are CPU-tractable. |
| **Simulated Participants** | Real recruitment requires IRB approval (Constitution VI). For the `planned` stage, a simulation allows testing the ANOVA pipeline and power calculations without ethical overhead. |
| **Bonferroni Correction** | Required by FR-006 and SC-005 to control family-wise error rate when testing multiple contrasts. |
| **Computational Proxy Framing** | The study is explicitly framed as validating the *pipeline's ability to detect effects* in a controlled environment, acknowledging that real-world validation requires real stimuli. |

## Ethical Considerations

-   **IRB**: No human data will be collected in this computational phase. The code will include a placeholder `consent_form.md` and a flag `--require-irb` that blocks real data collection until a valid IRB number is provided.
-   **Privacy**: All generated participant IDs are pseudonymous hashes. No PII is stored.
-   **Data Hygiene**: All generated data is checksummed. Raw mock data is preserved; processed data is derived.

## Limitations and Scope

-   **Semantic Complexity**: The mock generator simulates the *statistical* properties of visual detail (object count, texture) but lacks the semantic richness of real-world scenes. The study tests the *computational proxy* of visual detail, not the absolute perceptual experience.
-   **Human Cognition**: The "false memory" is a model output based on calibrated parameters, not a measure of actual human cognitive error. The results validate the *pipeline's sensitivity* to detect effects of a specified magnitude.
-   **External Validity**: Findings are limited to the simulated environment. Real-world generalization requires future studies with actual human participants and real stimuli.
