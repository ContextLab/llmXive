# Data Model: The Effects of Gamified Habit Tracking

## Overview
This document defines the data structures used throughout the pipeline, from raw ingestion to final reporting. It ensures alignment between `research.md` (methodology) and the implementation.

## Entity Definitions

### 1. User Profile
An anonymized individual participating in the study.
-   `user_id`: String (UUID) - Unique identifier.
-   `conscientiousness_score`: Float (0.0 - 100.0) - Normalized Big Five score.
-   `achievement_score`: Float (0.0 - 100.0) - Normalized Need for Achievement score.
-   `gamified_status`: Integer (0 or 1) - 1 if assigned to gamified condition (Randomly Assigned), 0 otherwise.
-   `consent_verified`: Boolean - True if consent artifacts exist.

### 2. Behavioral Log (Daily)
Raw event data before aggregation.
-   `user_id`: String (FK)
-   `date`: Date (YYYY-MM-DD)
-   `event_type`: String (e.g., "check_in", "task_complete")
-   `app_id`: String (Identifier for the habit tool used)

### 3. Weekly Aggregation
Derived dataset for modeling.
-   `user_id`: String (FK)
-   `week_number`: Integer (1, 2, ..., N)
-   `adherence_flag`: Integer (0 or 1) - 1 if `daily_events > 0` in that week.
-   `streak_length`: Integer - Current consecutive weeks of adherence.
-   `dropout_status`: Integer (0=active, 1=dropout, 2=censored) - **Derived post-hoc** from the "3 consecutive weeks of non-adherence" rule applied to the `adherence_flag` sequence.

### 4. Analysis Result
Output of statistical tests.
-   `test_type`: String (e.g., "MixedEffects", "CoxPH", "LogRank")
-   `coefficient`: Float
-   `std_error`: Float
-   `p_value`: Float
-   `p_value_corrected`: Float (FDR adjusted)
-   `confidence_interval_lower`: Float
-   `confidence_interval_upper`: Float
-   `sample_size`: Integer

## Data Flow
1.  **Raw**: `data/raw/synthetic_data.csv` (Generated by `synthetic_generator.py`)
2.  **Processed**: `data/processed/merged_data.csv` (Weekly Aggregation)
3.  **Results**: `data/reports/final_analysis.html` (Visuals + Stats)

## Constraints
-   **Missing Data**: Any user with missing `conscientiousness_score` or `gamified_status` is excluded (FR-001).
-   **Censoring**: Users who do not drop out by the end of the study period are censored.
-   **Minimum Sample**: Pipeline halts if `len(merged_data) < 100` or `non_gamified_count < 30`.
-   **Event Threshold**: Survival analysis halts if `dropout_events < 10` per group.
-   **Model Identifiability**: The mixed-effects model MUST use random slopes `(gamified_status | user_id)` to avoid collinearity between the between-subject predictor and the random intercept.
