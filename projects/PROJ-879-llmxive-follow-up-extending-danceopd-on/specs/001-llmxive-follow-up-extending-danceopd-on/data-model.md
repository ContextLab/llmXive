# Data Model: llmXive follow-up: extending "DanceOPD: On-Policy Generative Field Distillation"

## Overview

This document defines the data structures used throughout the project. All data is stored in Parquet format for efficient I/O and type safety, except for intermediate model artifacts (`.pkl`) and generated images (`.png`).

## Core Entities

### 1. TeacherRoutingDataset

The ground-truth dataset generated by the DanceOPD teacher model.

| Column | Type | Description | Constraints |
| :--- | :--- | :--- | :--- |
| `id` | `string` | Unique identifier for the sample (UUID). | Unique, Non-null |
| `prompt_embedding` | `list[float]` | The embedding vector of the input prompt. | Length = `embedding_dim` (e.g., 512) |
| `noise_level` | `float` | The noise level (sigma) at the current diffusion step. | $0.0 \le noise\_level \le 1.0$ |
| `routing_label` | `string` | The expert field ID selected by the teacher. | Must match known expert IDs (e.g., `expert_text_to_image`) |
| `velocity_vector` | `list[float]` | The velocity vector output by the selected expert. | Length = `latent_dim` |
| `source_dataset` | `string` | Source of the original image (e.g., `imagenet`, `laion`). | Enum: `['imagenet', 'laion']` |
| `step_index` | `int` | The diffusion step index. | $0 \le step\_index \le max\_steps$ |

### 2. DecisionTreeModel

A serialized scikit-learn Decision Tree.

| Attribute | Type | Description |
| :--- | :--- | :--- |
| `model_id` | `string` | Identifier (e.g., `tree_depth_5`). |
| `max_depth` | `int` | The `max_depth` hyperparameter used. |
| `accuracy` | `float` | Routing consistency on the test set. |
| `artifact_path` | `string` | Path to the `.pkl` file. |

### 3. InferenceResult

The result of a single image generation attempt using a static tree.

| Column | Type | Description |
| :--- | :--- | :--- |
| `sample_id` | `string` | Reference to the original sample. |
| `model_id` | `string` | The tree model used (or `teacher_baseline`). |
| `predicted_routing` | `string` | The routing label predicted by the tree. |
| `true_routing` | `string` | The ground truth routing label. |
| `match` | `boolean` | `True` if `predicted_routing == true_routing`. |
| `fid_score` | `float` | FID score for the generated image (relative to a reference set). |
| `clip_score` | `float` | CLIP Score for the generated image. |
| `image_path` | `string` | Path to the generated PNG. |

## Data Flow

1.  **Ingestion**: Raw ImageNet/LAION data (Parquet) $\to$ `download.py`.
2.  **Transformation**: `generate_teacher.py` $\to$ `TeacherRoutingDataset.parquet`.
3.  **Split**: `preprocess.py` $\to$ `train.parquet`, `test.parquet`.
4.  **Training**: `train_tree.py` $\to$ `models/trained_trees/tree_depth_X.pkl`.
5.  **Evaluation**: `inference.py` $\to$ `InferenceResult` table (CSV/Parquet) + `results/images/`.
6.  **Analysis**: `stats.py` $\to$ `results/statistical_tests.json`.

## Integrity Constraints

*   **Checksums**: All files in `data/raw` and `data/processed` must have a corresponding `.sha256` file.
*   **Immutability**: Files in `data/raw` and `data/processed` are never modified.
*   **Schema Validation**: All Parquet files are validated against the contracts defined in `contracts/` before processing.
