|---------|-----------------------|--------|-------------------|

- When real data are present, the full analysis (preprocessing, modeling, permutation testing, route‑shift analysis) runs and produces scientifically valid results.

## Methodology Overview

### 1. Phenology Metric Calculation
- **First Arrival**: 5th percentile of detection dates (DOY).  
- **Median Arrival**: 56th percentile (the median).  
- **Stopover Duration**: Difference between the 10th and 90th percentiles of DOY = `stopover_duration`.  
- **Sparse Data Handling**: Cells with **< 5** observations **must** mark **`data_quality="insufficient"`** and be excluded from modeling. A **sensitivity analysis** will re‑run the **same** pipeline with thresholds **3** and **7** to assess robustness.

### 2. Bias‑Mitigation: Tail‑Preserving **Stratified** Sampling
This **required** adaptation **(FR‑002‑S)** is derived from the plan to avoid bias toward densely sampled cells.  
- Quantile‑based bins on `first_arrival` (deciles).  
- Oversample cells in the **lowest [deferred]** decile by a factor of 2.
- Assign inverse‑probability weight = 0.5 to oversampled cells, weight = 1.0 otherwise.  
- Weights are passed to the GAMM via `sample_weight=`.

### 3. Unified Spatial Model (GAMM with Matérn GP)
- **Fixed smooth terms**: temperature, precipitation, extreme weather index, observer effort (mandatory).  
- **Random effects**: species‑level random intercepts and slopes for temperature.  
- **Spatial component**: Matérn Gaussian Process (`sklearn.gaussian_process.GaussianProcessRegressor`, ν = 1.5) applied to latitude/longitude.  
- **Model formula** (explicitly stated for reproducibility):  

```
phenology_metric ~
    s(temp) + s(precip) + s(extreme_idx) + s(effort) +
    (1 + temp | species) +
    GP_spatial(lon, lat; kernel=Matérn(nu=1.5))
```

- **Confounding control**: observer effort is included; habitat change is optional if ancillary land‑cover data become available.  

### 4. Permutation Testing & FDR
- **Full [deferred] shuffles** per coefficient generate empirical null distributions.
- **Early‑stop flag**: after 100 shuffles, if interim p < 0.001 the term is flagged “highly significant” for reporting, **but the remaining shuffles continue** so the final p‑value always reflects all 10,000 permutations.  
- **Benjamini–Hochberg** correction applied to the final p‑values, preserving the nominal false‑discovery rate.  

### 5. Route Shift Analysis
- Weekly centroids computed per species‑year; geodesic distances on a spherical Earth quantify shift vectors.  
- **Permutation test** (10,000 shuffles) yields p‑values for shift magnitude.  
- **Bootstrap** (1,000 resamples) provides 95 % confidence intervals for shift magnitude and direction.  
- **Validity note**: centroids are a simplification; for species with ≥ 500 observations an optional kernel‑density trajectory analysis is performed to verify that centroid‑based results are not driven by outliers.

### 6. Computational Feasibility
- All steps run on CPU‑only GitHub Actions free runners (2 CPU, ≤ 7 GB RAM).  
- Chunked I/O and limited parallelism keep memory < 6 GB.  
- Estimated total runtime ≤ 5.5 h (see Phase 5 in the implementation plan).  

## Success Criteria (Re‑stated)

- **SC‑001** – Power analysis (Section 4) shows ≥ 80 % power to detect a standardized effect size of 0.3 with 10,000 permutations.  
- **SC‑002** – Proportion of “insufficient data” cells reported; sensitivity analysis with thresholds 5/7 ensures robustness.  
- **SC‑003** – GAMM convergence rate logged; target ≥ 90 % success (based on pilot runs).  
- **SC‑004** – 95 % CI width for phenology shift ≤ [deferred] days (to be evaluated on real data).  
- **SC‑005‑A** – Full 10,000 permutation shuffles are always executed (early‑stop only flags).  
- **SC‑005‑B** – Total runtime ≤ 6 h on the free‑tier runner (validated by CI job).

# Data Model: Bird Migration and Climate Analysis

## Overview

This document defines the data structures used throughout the pipeline. All data is stored in CSV or Parquet format, with checksums recorded in the project state.

## Entity Definitions

### 1. MigrationRecord (Raw)
Represents a single bird observation.
- `species` (str): Scientific name (e.g., "Turdus migratorius").
- `latitude` (float): Decimal degrees (WGS84).
- `longitude` (float): Decimal degrees (WGS84).
- `date` (date): YYYY‑MM‑DD.
- `count` (int): Number of individuals observed.
- `checklist_id` (str): Unique eBird checklist ID.
- `grid_cell` (str): "lat_lon" string (e.g., "45.5_-122.0").

### 2. ClimateVariable (Raw/Interim)
Represents climate measurements for a grid cell and week.
- `grid_cell` (str): "lat_lon" string.
- `week` (int): ISO week number (1‑52).
- `year` (int): Year (2020‑2024).
- `mean_temp` (float): Mean temperature (°C).
- `total_precip` (float): Total precipitation (mm).
- `extreme_idx` (float): Extreme weather index (0‑1).
- `imputed` (bool): True if value was interpolated.

### 3. PhenologyMetric (Processed)
Computed metrics per species, grid cell, year.
- `species` (str).
- `grid_cell` (str).
- `year` (int).
- `first_arrival` (int): Day of year (1‑365).
- `median_arrival` (int): Day of year.
- `stopover_duration` (float): Days between 10th and 90th percentile.
- `n_obs` (int): Number of observations used.
- `data_quality` (str): "sufficient" or "insufficient".
- `sample_weight` (float): Inverse‑probability weight used in modeling (1.0 for regular cells, 0.5 for oversampled early‑arrival cells).

### 4. ModelOutput (Result)
Results from the Unified Spatial Model.
- `species_id` (str).
- `grid_cell` (str).
- `model_id` (str): Unique identifier for the model run.
- `term` (str): Predictor name (e.g., "temp_smooth").
- `estimate` (float): Coefficient or smooth effect estimate.
- `p_value` (float): Permutation‑derived p‑value based on **[deferred]** shuffles.
- `fdr_q_value` (float): Benjamini‑Hochberg corrected q‑value.
- `aic` (float).
- `r_squared` (float).
- `convergence_status` (str): "success" or "failure".
- `moran_i` (float): Moran’s I of residuals (diagnostic only).
- `gp_applied` (bool): Always `true` for the Unified Spatial Model.
- `spatial_method` (str): `"matérn_gp"` indicating the Matérn Gaussian Process used.
- `weights_applied` (bool): Indicates whether inverse‑probability
